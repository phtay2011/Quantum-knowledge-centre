Payment Date Offset, Security Instrument

Available in the deal when Formula Timing is FRN-CGL PERIODIC with Overnight Index .

Payment Date Offset has two values:

Enter the number of days by which to offset the payment dates for this deal. Select a number from 0 (zero) to 99. The default is the value (if any) specified on the Instrument.

Analytics adds the Payment Date Offset directly to the Interest Effective Date which is calculated using the Interest Calculation Business Days Convention. The INTEREST Action Dates value changes according to the Payment Date Offset Days value. The REPAY PRINCIPAL Action Dates value does not.

Quantum bases business days on the Currency and Location settings, and Action Dates are the dates for the following items in Quantum deal's Deal Map tab: INTEREST, PRINCIPAL, REPAY PRINCIPAL, NOTIONAL, and REPAY NOTIONAL.

When both the Payment Date Offset Days and Suspension are non-zero in a deal, the system applies the following logic in the Deal Map:

  • the system applies Payment Date Offset Days to all INTEREST cash flows, except the final INTEREST cash flow;
  • the system applies Suspension (if greater than 0) to calculate all the Rate Set Action Dates. It also uses Suspension to calculate the Action Date of the last Rate Set only.

Payment Date Offset is not supported by revaluation methods that are used in Quantum's Reporting > Deal Revaluation page and the Reporting > Position - Query page.

The resulting dates may be after a deal's Maturity Date.

The Effective Date is not affected by this setting.

Payment Date Offset with Allow Asset Backed - You can apply Payment Date Offset to deals with Allow Asset Backed selected, when Formula Timing is FRN-CGL PERIODIC. You can choose between Calendar or Business Days. The Payment Date Offset value should be between 0 and 99.

NOTE: Changing this setting is a financial change. To change it after saving a deal requires a deal Reset.
