Dealing Analysis Codes (Commodities)

Create and maintain any Analysis Codes specific to Commodity Swap and Exposure deals - by setting Scope to Commodity Swap Deals.

TIP: Use Analysis Code Defaulting Rules to set default values that will initially populate these Analysis Code fields (on the deal page's Analysis Codes tab) for deals matching specific rule criteria.

Perform the following procedures to create and maintain Analysis Codes for Commodity Swap and Exposure deals.

To implement any Analysis Code setup changes in Analytics, you must run the Analysis Code Replication wizard.

To define a new Analysis Code specifically for Commodity deals

  1. In the site menu's Configuration submenu, select Analysis Code.

    The Analysis Code page opens.

  2. Click New.
  3. Enter the mandatory unique Name for the analysis code throughout the system.
  4. Select a Scope of Commodity Swap Deals.
  5. Enter a text Description description of the Analysis code; up to 40 characters long.
  6. In the Options area, select or clear the following check boxes as required.
    • Use predefined values (available only when Data type is TEXT) - select this option to display the Predefined Values grid for entering one or more analysis code values for selection when entering the relevant type of data record; clear the option to have no predefined values.
    • Allow values other than predefined (available only when Use predefined values is selected) - select this option to allow users to enter free-format analysis code values (in addition to any predefined values); clear it to allow users to select from the list of predefined values only.
    • Warn when entered value is not predefined (available only when Accept predefined values only is selected) -- select this option to request user confirmation before they can save a newly entered code value; clear it if the system should automatically save new values for this analysis code.
    • Mandatory - select this option to force users to enter a value for this analysis code when entering the relevant type of data record; clear it to make analysis code entry optional.
    • Show on flow details during Bank Reconciliation - select this option to enable users to select this analysis code as (1) a display column in the Cash Flows grid on the Flows tab of the Prior Day query page, and (2) a Cash Flow field to match with any of the generic Bank Flows fields In the Passes tab's Criteria grid on the Detail Match Script page.
  7. If Use predefined values is selected, then In the Predefined Values grid menu, select New and enter a valid value for the Analysis Code; repeat as required (to remove a value, click the x button at the right of the value field).

    NOTE:If Allow values other than predefined is cleared, you must enter at least one predefined value.

  8. Data type is AMOUNT, enter the number of Decimal places for amounts that will be entered under this Analysis Code.
  9. If Data type is NUMBER or AMOUNT, optionally enter Maximum and Minimum limits to be enforced for values that will be entered under this Analysis Code.
  10. Click Save.

    Your new Analysis Code is stored in the database. The Analysis Codes tab on the Commodity Swap and Commodity Exposure dealing pages will now display the code field, labeled with the Name you entered in Step 3.

    NOTE:Any Analysis Codes with Scope set to Deals will also be displayed.

To edit the currently displayed Analysis Code

  1. Enter or select new values as required (see the procedure above).

    TIP:Name and Data type cannot be changed on a saved record.

  2. Click Save.

    Your changes are implemented. Note that:

    • When you remove a Predefined Value, any occurrences of that value in any affected data are replaced by NONE.
    • When you change the Scope - except when you change it to (all) Deals is equivalent to a delete and add. The Analysis Code is removed from all Commodity Swap and Commodity Exposure deal records and pages it previously applied to (including all backdated occurrences) and the field will no longer appear on the Analysis Codes tab of those pages; conversely, the code is added to the data records and application pages it now applies to.

To delete the currently displayed Analysis Code

  1. In the page menu, select Delete.
  2. Click OK to confirm.

    The Analysis Code is removed from all Commodity Swap and Commodity Exposure deal records and pages it previously applied to (including all backdated occurrences), and the field will no longer appear on the Analysis Codes tab of those pages.
