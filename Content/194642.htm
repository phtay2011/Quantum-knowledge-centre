Import function on Dealing pages

(First introduced in Release SP 6.7.1, August 2018.)

You can now import deal data files directly from Quantum Dealing pages, subject to the Module Security and Resource Security permissions assigned to your User Name.

NOTE: The importing of deal data files using the Deal import Service and the Upload File function on the Deal Import Status page remain unchanged (see History below). The Import Deals operational setting applies only to that importing method (NOT to the new Dealing page function).

Implementation

User page

On the User Details tab under User Rights, the Can import data option has been added, as highlighted below. This option must be selected for each User who needs the Import function to be available on Dealing pages.

This option has also been added to the User tab on the Static Data Transfer spreadsheet.

Module Security

Each User who needs to use the Import function on any Dealing pages must be assigned (or inherit) Inquire, Add, Update, and Delete Module Security permissions for each of those Dealing modules (or be exempt from Module Security).

Resource Security

If Resource Security is enabled, the data a user can import will be subject to the permissions applicable to that User name and the relevant resources in each case.

Dealing pages

Subject to the required security settings (see above), an Import function will appear in the Browse menu of applicable Dealing pages, as highlighted below.

When you select this function, an Import dialog box is displayed for you to browse for and enter a File to Upload name and location. This file may contain data records for any type of deal, in a supported format (QXT, QID, or tab-formatted).

When you enter a file name and click OK, deal data records in the file are imported into the Quantum database provided the data is valid and you have the required security rights in each case. When the import operation is finished, you will receive a status notification (see below), where:

  • Deal file imported successfully indicates that all deal records were imported from the file,
  • Deal file imported Partially indicates that only some records were imported, or
  • Deal file failed to import indicates that no records were imported.

If Deal Approval applies to a deal you imported in this way, the Approve dialog box shows your User name as the Creator.

Clicking on an imported successfully notification returns you to the Dealing page from which you did that import. Clicking on a failed to import or imported partially notification downloads the Import Error log file for that run, which details the reasons for failed records.

Restrictions and limitations

The Import function is not currently supported for Analytics Pricing pages.

History

Previously, the importing of deal data could be done only from the Deal Import Status page, and only by users with Upload File Module Security rights for that module.

TIP: This method still applies. It uses a service that picks up files arriving in the file system and uploads them using the Service Account User, who is typically exempt from Module and Resource security. So, for deals imported in this way, both the Approve dialog box (if applicable) and the Message Log indicate the Service Account user.
