Utility

Utility menu items significantly different from Version 6 are summarized in the following subsections.

Service Health

Use these submenu items to access the:

  • Import status pages for all services that handle importation of files, or
  • Scheduled Jobs page.

Import Status pages

Use these pages to check if any files are in error, move files back to the processing queue, delete files, or manually upload files for all services that handle importation of files - namely:

  • Bank File Import,
  • Confirmation Match import,
  • Deal Import, and
  • Static Data Import.

An example of the Bank File Import Status page is shown below.


Scheduled Jobs page

You can use this page to perform the following limited processing and management of Scheduler jobs:

  • run or re-run a job using its current settings, and
  • view job history and the associated logs.

NOTES:

  • Scheduler must be integrated with Quantum before you can use this page.
  • You can view and run only jobs covered by a predefined Custom Module record for their Scheduler category, and for which you have the relevant Module Security permissions.
  • Prior to Version 6, you could execute Scheduler jobs only from the Scheduler Server.
  • You must still use Scheduler to create, maintain, and schedule jobs and tasks. See Batch Processing.

An example Scheduled Jobs page is shown below.

Back Office

Bank Account Conversion and Counterparty Delivery Instruction Conversion

Separate pages apply for these utilities (prior to Version 6, these were two modes within the FMOVE program).

Set Statement Details

For selected bank or stock accounts, you can use this page to:

  • set the start date for the first Statement run,
  • set the next Statement number (for example, to restart from 1 at the beginning of each year), or
  • delete the last Account Statement query Update run.

(Prior to Version 6, the Bank Statement Date program allowed you only to set the start date for the first Statement run, and when run, cleared all existing Statement run results for your selected accounts.)

EFTs

New from Release 6.6.1, use this page when exceptions arise and you need to reprocess previously sent Electronic payment output (EFT reprocessing is no longer available from the EFTs Processing page).

Structured Address Migration

CAUTION: Ensure that static data approval in data security is disabled while migrating unstructured addresses as not all unapproved static can be approved in bulk.

Use the utility Structured Address Migration, or create a Workflow Report page based on the utility, to suit the specific new address structure, and migrate each set of addresses.

In the utility you can update one or more of the addresses in bulk according to the logic defined in the workflow report. Select items that suit the proposed mapping criteria, then Save Structured Address.

You can also use the Quick Menu option named Edit Address. This menu option uses a Bulk Update action which allows you to modify the details of the structured address' individual lines.
