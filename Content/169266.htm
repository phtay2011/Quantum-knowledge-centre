Index Rates (Analytics)

Use the Index Rate page to create and maintain a rate series specifically for indexed instruments in Analytics - for example, see About TIPS (Treasury Inflation-Protected Securities).

  • Use the Rate Definition area to view and maintain the Name value for an Analytics Index Rate record.
  • Use the Rates grid to enter and/or change individual Effective Date and Index rate values for the current Rates record.

NOTE:

  • The Rate Definition Name alone identifies a unique Index Rate record.
  • A Rate Name Record must be created before you can select a Rate Definition Name.
  • Each Rate record may have one or more effective dates, each with a different Period Start Date and Index value. Unlike other rates, each Effective Date is listed in the grid (because there is only one rate value per effective date).

Rate Definition area

Use this area to view and maintain the defining values for an Analytics Index Rate record.

To list Rate records - click the page browse button (the top one) to open the browse window. (Click the Configure link to change browse list properties.)

To add a new (empty) Rate record - in the page menu, select New; in the New Rate Definition dialog box select a new Name (one set up as an Analytics Rate Name record); then click Save.

Each Rate record may have one or more effective dates, each with a different Period Start Date and Index value. Unlike other rates, each Effective Date is listed in the Rates grid (because there is only one rate value per effective date).

To view and edit another Rates record - in the main browse list, select the Rates record you want and add, edit, and/or remove complete rate sets and their individual rate entries (grid rows) as required.

To delete a Rate record - in the main browse list, select the Rates record you want to remove; in the Rates menu, select Delete; then click OK to confirm.

Field Descriptions

Name - the name of the required rate record (the Code value of an existing Analytics Rate Name record).

Rates grid

Use the Rates grid to enter and/or change individual Effective Date and Index rate values for the current Rates record.

To add a single Effective Date and Rate entry to the current Index Rate record:

  1. In the grid menu, select New*.
  2. In the New Rate dialog box, enter new Effective Date, Description, Period Start Date, Published Date (where applicable) and Index values as required (see Field Descriptions below).
  3. Click Save.

    The new Effective Date and Rate entry is added to the Rates grid.

*TIP: You can also add a new row by selecting an existing one, selecting Copy in the grid menu, changing the values as required, and clicking Save.

To edit an Effective Date and Rate entry in the current Index Rate record:

  1. Click a cell value in the entry's grid row (or select the row you want to edit and in the grid menu, select Edit).
  2. In the Edit Rate dialog box, change the values as required (see the Field Descriptions and Cautions below).
  3. Click Save.

    Your changes are shown in the Rates grid.

To remove an Effective Date and Rate entry from the current Index Rate record:

  1. Select the grid row you want to remove.
  2. In the grid menu, select Delete.
  3. In the Confirm Delete dialog box,click OK to confirm.

    The entry is removed from the Rates grid.

Field Descriptions

Effective Date - the earliest date from which the Index record can be used in Index calculations. The first index record's Effective Date is usually the Start Date for the Rate Series.

NOTE: Effective Dates from the Index Rate Series table are not used in any calculations for Instruments using the INDEXED BOND Formula. The Effective Date for an Indexed Bond deal is calculated on deal save, using a combination of the available Index Rates and the Index Calculator Definition associated with the Instrument.

CAUTION: DO NOT change this value for past periods with Index values previously used in current and past deals.

Description - the month and year of the index.

Period Start Date - the start of the index period. The published Index value applies to the period starting from this date and ending on the day before the next Index record's Period Start Date.

TIP: When first setting up an Index Series, enter the Start Date of the first period.

Published Date - the date each existing Index value is actually published (except the two latest-dated placeholder records), where the month is the one after the Period Start Date month in each case.

Index - the value of the index, as published by the relevant government or external entity - for example, a CPI for use with a TIPS (Treasury Inflation-Protected Security) instrument. This index will be interpreted differently by the respective Indexed Formulas.

CAUTION: Make sure that the index value is correct before an Update Index processing run. To correct an index series after Update Index has run, you will need to change the index value here, re-save all affected deals,and then run Update Index again. The system always uses the most recent figure to update an index.
