Permissions

In the Module Security tab, configure a user's access rights to pages (modules) and the available menu actions on those modules.

These access rights are called permissions. For each of the permissions that follows, we name the module the permission applies to and, in SEE ALSO, we list other permissions that you might use in the module.

To add module permissions to a page (module)

  1. Open Configuration > Group and select the Module Security tab.
  2. Select Apply security for specific modules.
  3. Click Add and select the names of the modules in Quantum or Analytics to add module permissions to. Click OK.
  4. To assign the same standard permissions to all modules, click the tick at the top left hand side of the table of Secured Modules, then click Bulk Update. The Bulk Update Standard Permissions dialog box opens.
  5. Select the standard permissions that you want to apply to all modules, and click Add.
  6. To assign permissions to individual modules, select the circle at the top left hand side of the table of Secured Modules, which deselects all modules.
  7. Click one module name to open the Edit Module Permissions dialog box for that module.
  8. Add permissions for that module and click OK.
  9. Continue to add permissions (assign permissions) one at a time to specific modules.
  10. Save.

In a similar way, add permissions at the user level by opening Configuration > User.

Adhoc Instructions

Enables Adhoc Instruction and Third Party modes in the page menu of the Counterparty Instructions page for cash flows, when this page is accessed from the Settlements or a Dealing page allowing such access.

NOTE: The applicable Module name is Counterparty Delivery Instructions Change.
The Adhoc Instructions permission allows any changes on the counterparty instruction, for example, Adhoc, Third Party, or change of account. The permission also allows changes to Delivery Narrative, Bank to Bank Instructions, Payment Category, Overseas Bank, Details of Charges, Sender’s Charges, Receiver’s Charges and Charges Comments.
Adhoc is only applicable for cash flows.

Configure Public Queries and Views

When enabled, allows the user to do the following:

  • create, configure, save, copy, and delete public queries;
  • keep and delete public reports created by other users; and
  • select Public in the Access Permission area of the query's Properties tab.
    (This option is selected and read-only by default when the user does not have Configure Private Queries permission).

NOTE:

  • When you enable Query resource filtering for Resource Security, Resource Security and Module Security work together to allow or restrict actions within queries and reports.
  • If a user does not have either Configure Private Queries or Configure Public Queries permission for the module, they will not be able to configure any queries.
