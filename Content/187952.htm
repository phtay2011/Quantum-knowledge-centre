Setting Rates for Floating deals

Floating rate deals require the setting of rates from time to time. The same process is used for rate-setting floating Money Market deals, Security deals, and Swap deal floating legs, with the following exceptions.

  • Deals in Hedge Relationships

    CAUTION: DO NOT change a rate entry that has any earlier UNACTIONED rates shown if the deal is (or may be) part of a Hedge Relationship. If you do, those earlier Rate Sets will also be actioned ( a warning message informs you of that). If you save these multiple Rate Sets, Effectiveness Testing will give incorrect results, and the subsequent hedge revaluations will be incorrect. If you want to use this deal in a Hedge Relationship, apply Rate Sets one at a time, in chronological order - that is, change only the first UNACTIONED rate and save the deal, then re-open the deal, change the next one, save the deal again, and so on.

  • Range Accrual Money Market Deals

    Rate setting must be done in sequence - that is, previous rate sets must have been ACTIONED. (With other deals, you can Rate Set a period when there are UNACTIONED previous periods. Those previous periods will automatically be set to ACTIONED.) See Rate Setting for Range Accruals.

The following deal setting restrictions and recommendations apply to Ratesets:

  • Rateset Frequency - must be the same as, or less often than, the Coupon Frequency if Formula Timing is:
    • DISCOUNT PERIODIC, or
    • FRN-CGL - except that if Multiple Rateset is selected for the instrument, any frequency is allowed.
  • First Rateset - must be set to the Start or Settlement Date if a Rateset event is to occur on that date.
  • Days Offset (some Formula Timing settings only) - for markets where there is a spot days offset, gives the number of days the rate review date is ahead of the effective interest period.
  • Separate Rateset Location - select to enable separate locations for ratesets.
  • Rateset Location - select one or more locations to apply specifically to Ratesets for the deal.

To perform a single* Rate Set action

TIP: To Rate Set multiple deals, use the bulk Rate Set Deals page.

  1. Do one of the following.
    • On the Dealing page when the required Floating deal is displayed: select Rate Set from the page menu. The Rate Set dialog box is displayed (see the example below).
    • On the Settlements Processing page when query run results show the required Rate item: click on a value in the Rate item row and select Rate Set from the popup menu. The applicable Dealing page is opened in a separate window, then the Rate Set dialog boxdialog box is displayed, as in the example below.

  2. Click on an item in the applicable row (or select the row, and in the grid menu select Edit).

    The Edit Rate Set dialog box is displayed, as in the example below.

  3. Edit either the Base Rate or Rate field.

    The Amount and the other rate field are automatically recalculated (applying the Margin, if one applies).

  4. Select or clear the Apply rate to subsequent rate sets option for rate fill-down when you change the rate for an already Actioned item, as follows:
    • select the option to apply that rate on all subsequent rate entries (including those already Actioned) and recalculate the amounts, or
    • clear the option to have no impact on any other Rate Sets.
  5. Click OK.

TIP: The new rate is able to be reviewed and changed by editing the relevant RATE SET component row in the Map Items grid on the Deal Map tab. All changes are also recorded in the Audit History.
