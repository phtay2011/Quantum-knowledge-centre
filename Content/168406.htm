Settlements Processing Functions

After you run a query, deal events matching your specified filters and options are listed. Depending on your run option (Run Query or Run Query & Net), cash flows meeting the criteria for netting will either be shown as proposed for netting or already netted.

Two function menus are available for result list items.

  • Grid menu functions apply when one or more items are selected (click the selection box at the left of the row/s and click the menu button above the grid).
  • Quick menu functions apply to only one item (click a value in the item row to show this menu).

The table below summarizes the functions available, which menu/s, and the types of flows or other events they apply to.

NOTES:

  • You must have the required security rights to perform most functions. These rights are determined by settings on the relevant User or User Group record's Module Security tab.
  • The only available actions on flows currently Proposed for Netting are either Net as Proposed or Clear Net Proposal (TIP: Add Type to the View to identify these - and other - flow types).
  • If a cash flow is linked to a stock flow, it is part of a "package" of linked flows and has no stand-alone cash flow rights. Actioning any one flow in such a linked package actions all flows in that package, so all flows have the same rights as that of a stock flow (for example, you cannot Net).

    Linked stock flows and cash flows are grouped together in the Settlements result list.
  • When you action a cash flow that creates a SWIFT MT101, MT103, or MT103+ Payment message, a Bank Account Adjustment will be automatically created when the payment is output (by the EFT processing application), to reflect the charges to be applied.

    This system-generated Bank Account Adjustment is always in the same currency as the payment - if required, the amount is converted using the currency Translation Rate Type specified for the Bank Account.
  • You cannot perform any action on a cash flow that is associated with one or more previous un-actioned RateSet events and linked to a Deal Approval control rule record with Allow Settle set to NO for the RATESET action.
