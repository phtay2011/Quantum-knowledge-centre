Calculation Properties (Adaptiv)

Use the Calculation Properties page to set AA Studio Calculation properties from within Quantum.

You can specify different sets of properties, each with an identifying Code and Name, the applicable Query Type, and your specified Properties grid entries. In each grid entry row, enter a valid Adaptiv Calculation Property Name and Parameters value). For details of these, see the Adaptiv Studio documentation or help.

To apply a set of saved Properties to a query Definition, enter its Name in the Calculation Properties field in the query's Assumptions Definition record. To change it for the current query run, edit the Calculation Properties field on the query's Assumptions tab.

In each case, only Calculation Properties that match the Query type are available for your selection.

Where Property Names/Parameters specified in your selected Calculation Properties record duplicate values specified:

  • on the Assumptions tab - the Assumptions tab values take precedence, or.
  • in the AAI.AAIS.exe.config file - the Calculation Properties record values take precedence.

Any Adaptiv errors in Property Name/Parameters are returned to the user at query run time and recorsed in thje Message Log as usual.

Module Security

Module Security for this page is covered by the assigned permissions under Adaptiv Analytics Static Configuration - Analytics.

Example

If you specify the above Calculation properties record for an Adaptiv Historical VaR query, you will be setting:

  • Decay Factor to 1, and
  • InflationRate and InterestRate Price Factor Return Methods to Absolute and Historical respectively (leaving the remaining Return Methods set to the default of Proportional).
