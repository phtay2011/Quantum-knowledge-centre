Imported Market Values

Depending on the type of revaluation required, the methodology used to revalue a deal is determined by settings in:

  • Static > Accounting Rules
  • Rules > Management (Management Reporting Control)
  • the Linked Deals tab on the Accounting > Hedge Relationship page

Revaluation calculations in Quantum

Generally, Quantum calculates a market value and a book value for each deal. It then calculates the difference between these to determine the incremental gain/loss on revaluation. If General Ledger postings are requested, they are automatically posted. See Understanding Revaluation and Pricing.

Alternatively, the system can apply an imported market value to a deal, which is either derived from Analytics or another external source. All of the above calculations take place, except for the calculation of Market Value, and gain/loss is derived relative to the imported value. For instruments set up to use this feature, a market value may be derived either from Analytics or from another external source.

NOTE: Market Revaluation importation is separate from the main importation methods (covering deals, static data, rules, rates, and bank data).

Standard security is provided through the COM interface.

Applying an imported market value

Quantum provides a range of revaluation methods for market instruments. If an instrument requires a method other than these, you can - for selected Transaction Types - import a market value.

The following advantages apply to deals that can be revalued using an imported market value:

  • If there is no suitable revaluation method already available for a particular deal, you can apply an imported market value. This extends the range of deals that can be revalued and accounted for, and gives you access to a virtually unlimited range of market revaluations.
  • Full back office processing, accounting, and reporting is available to non-standard deals such as commodities (input as FX Deals), complex embedded derivatives, and other deals that could not previously be revalued. You can enter your own market value for any deal.
  • You can import immediate, actual market values as often as they are required.
  • You can choose to add or update amounts individually or as part of a batch, manually or with varying degrees of automation.
  • You can store different "sets" of market values for each deal, covering each type of revaluation that might be required.
  • Freed of the need to calculate market values, Quantum can proceed directly to the calculation of book value and the calculation (and posting) of profit/loss.

To apply an imported market value

  1. Set up an instrument with a revaluation method of Imported or Imported Separate.
  2. Select the instrument when entering a deal. There are certain deals that use imported market values.
  3. Add the necessary Imported Revaluation Record to the database using the Imported Valuations page.

    NOTE: Zero is a valid market value.

  4. Run a Reporting > Position page or a Reporting > Deal Revaluation page to revalue the deal and report on it. See Revaluing and reporting on deals with imported market values.

Different records for a single deal

You can enter multiple market values for a single deal. Independent records can exist for:

  • Different revaluation dates over the life of the deal
  • The various types of revaluation that may be required (Accounting or Management mode, revaluation as part of a Hedge Relationship, or all of these)

Quantum draws on the appropriate 'set' of records as required.

NOTE: There cannot be two identical records for a single day. To change the revaluation amount for an existing record, recall the record, edit the amount, and update the record.

For more information about revaluation as part of hedge accounting, see Hedge Accounting, imported market values
