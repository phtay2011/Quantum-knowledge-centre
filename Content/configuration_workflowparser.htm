Workflow Parser page

Use this page to define a parser that can be used to map text file input to Quantum data - for example, to create an Actual Cash Flow database record based on a fee invoice received from a regulatory authority. Such a parser will be invoked only if and when:

  • its Name is specified in a Data Transfer Location (DTL) record's Parser field,
  • a Transfer action specifying that DTL is performed on a report created via the Workflow Report page,
  • the system-wide Process Workflows option is selected on the Operational Settings page,
  • the Static Data Import Service is running, and
  • incoming text input is found - for example, a file is found in the WorkflowIn\Import folder at the Location specified in the DTL record.

NOTE: You cannot modify nor delete an in-use parser- that is, one specified in a Data Transfer Location (DTL) record.

Data items

Name - the internal name that the system uses to recognize the parser record.

Input Format - select Delimited, XML, or Text.

Delimiter (available when Input Format is Delimited) - select Comma, Pipe, Semicolon, or Tab.

Description - a short description of the use of the parser.

Data has a header row (available when Input Format is Delimited) - by default, this check box is checked. To parse a delimited file without a header row, deselect the check box. Once the columns are generated, the check box becomes read-only.

Data Starts at Line (available when Input Format is Delimited, or Text) - specify the line number from which data starts, so that you can distinguish and parse the system header and data appropriately.

By default, the value is set to 1. Click 1 to open the Expression Builder window and change the value. Once the columns are generated, this field becomes read-only.

Data Ends at Line (available when Input Format is Delimited, or Text) - specify the line number from which data ends, so that you can distinguish and parse the system footer and data appropriately. By default, the value is the following expression - Count(Lines([Raw Data])). Click it to open the Expression Builder window and change the value. Once the columns are generated, this field becomes read-only.

Data has multiple items (available when Input Format is set to Text) - by default, this check box is deselected. When you check the check box, the Item Separator field is shown.

Item Separator parses the data. Based on the item separator value specified here, data set is available in the next row. If the value specified in the Item Separator field is not present in the Sample data, the entire data is considered as one item.

Parser tab - displays either a Payload area (initially), or Columns and Calculated Columns grids after you click Generate Columns and enter or paste correctly formatted data based on the specified Input Format.

You can parse a fileâ€™s header (Before Data) and footer (After Data) by adding an expression in the calculated column. These items are available in Expression Builder > Context > Data Fields.

Preview tab - displays a Sample Data box which displays data you entered on the Parser tab (if any), and the Parsed Columns grid, which displays the parsed output for the Sample Data (if any).

For more details of each tab, click on the tab while the help panel is open. Or click the Knowledge Center link below.
