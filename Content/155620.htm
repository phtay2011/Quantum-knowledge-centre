Query Page Common Features

In general, each page has query, view, and report areas (some special-purpose reports do not).

The page menu provides functions you can perform on the current Query and/or Report (click to open), and, in some cases, functions to create new queries.

NOTE: The view area has a separate menu with functions to create new views or to reconfigure, copy, save, and delete the current view.

Query area

Where available, use this area to:

  • select an existing Query or create a new one,
  • enter specific parameter values for a run of the currently selected Query, and
  • configure different Queries by adding parameters and setting default values for them.

NOTE: Query parameters and configuration options in Analytics (and Adaptiv if installed) differ from those for Quantum.

Report area

This area displays the report results produced when you run a Query. Where applicable, you can:

  • see a summary of the parameters used for the query run;
  • page through the results grid or use the free-format text Search tool or partial number (~) search to find specific items;
  • sort on any column;
  • add a color and a description (tool tip) to cells; and
  • select a different view of the results or change the current view parameters, then click Apply to change the view accordingly (click Save to keep any changes).

NOTE: If you do not explicitly save them, report run results are kept until they expire.

About views

There is usually a Default view and a Crystal view.

You can configure different views and save them for re-use. You can rearrange columns in the grid, add new columns, sort on multiple columns, and add filters to view different subsets of the result set (without having to re-run the Query).

You can configure Table, Pivot, or Chart view types.
