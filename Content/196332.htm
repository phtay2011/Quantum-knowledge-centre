Defining Fee Estimation Parameters (when applying True-Ups)

True Up options (on the Other Details tab) are not available if you have specified:

  • on the Fee deal page's Calculation tab - a Base Template that has a Transaction Fee Category (Invoice Dates and True-Ups are not available with Transaction Fees),
  • on the Schedule tab - an Invoice Schedule grid item with Frequency set to Same Day, or
  • on the Schedule tab - Pay fee cashflow in advance is selected.

    (If Suppress true up cashflow processing is selected, True Ups are still generated, but are not used to adjust cash flows.)

When the Invoice date occurs prior to the Calculation End date, the system will calculate an estimated fee cash flow amount for the period between the Invoice date and Calculation End Date (inclusively). The True Up is the adjustment made in the next calculation period for the difference between this estimate and the actual amount for the current period.

TIP: You can manually change the estimated cash flow after it has been calculated (via the Process Fees page), but only if the corresponding G/L entries have not yet been processed.

To specify True Ups settings for the current Fee deal record

  1. Open the Other Details tab.
  2. In the True Ups area, select one of the following Estimation Methodology Based on settings for calculating the estimated fee cash flow amount.
    • Actual Accrued to Invoice Date - to divide the actual accrued fee amount from Calculation Start Date to Invoice Date by the number of days in that period, then multiply by the number of days from Invoice Date to Calculation End Date. You can optionally apply an additional Estimation Margin.
    • Reference Rate - to multiply the Fee base amount at Invoice Date by the applicable rate curve (specified by the selected Rate Name and Rate Type) over the period from Invoice Date to Calculation End Date. You can optionally apply an additional Estimation Margin.
    • Straight Line - to multiply the last known fee amount prior to the Invoice Date by the number of days from the Invoice Date (exclusive) to the Calculation Start Date (inclusive). (This is the default for a Manual fee.) You can optionally apply an additional Estimation Margin.
    • User Defined Amount - to use the value specified in the associated User Defined Amount field.
  3. If Estimation Methodology Based On is Actual Accrued to Invoice Date or Straight Line, optionally enter a percentage value for an additional margin rate to be applied to the estimated fee amount.
  4. if Estimation Methodology Based On is NOT User Defined Amount, select the Day Count convention to be used for calculating the correct number of days within the calculation period, so that, if the period between the Invoice Date and Calculation End Date:
  • includes the 31st of a month - that date is:
    • included if Day Count is Actual/360 or Actual/365, but
    • ignored if Day Count is 30/360;
    • includes the 28th of February - that day equates to:
      • one day if Day Count is Actual/360 or Actual/365, or
      • three days (28th, 29th, and 30th) if Day Count is 30/360;
    • includes the 29th of February (that is, in a leap year) - that day equates to:
      • One day if Day Count is Actual/360 or Actual/365, or
      • Two days (29th and 30th) if Day Count is 30/360.
  1. If Estimation Methodology Based On is User Defined Amount, enter the mandatory User Defined Amount for the estimated fee cash flows generated (via the Process Fees page) when applying this Fee deal record.
  2. If Estimation Methodology Based On is Reference Rate, select the Rate Name and Rate Type that together define the interest rate curve to be used to calculate fee cash flow amounts (based on the overnight rate).

    Only applicable Rate Names for Interest Rates are available - that is, those where:

    • Input Type is Yield,
    • at least one Date Interval is Overnight,
    • Currency is the same as for this Fee deal record, and
    • the Rate record's date is on or before first Invoice Date in this Fee deal record.
  3. Do one of the following:
    • clear the Suppress true up cash flow processing option to apply your True Up settings when generating fee cash flows (via the Process Fee page) (this is the default), or
    • selectSuppress true up cash flow processing to suppress application of True Ups when generating fee cash flows (True Ups will still be generated and viewable on the Process Fees page where applicable, but will not be applied to adjust cash flows).
