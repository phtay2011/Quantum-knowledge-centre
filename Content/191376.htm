Account Mapping File (qtmap.txt)

Use an Account Mapping file to map external account codes to the corresponding internal account codes in Quantum, where necessary.

NOTES:

  • All account codes must be represented exactly as they are found in the data file being imported and in Quantum. In particular, if the account code in Quantum has leading zeros, then the account code in the mapping file must also have leading zeros.
  • An Account Mapping file must always be present (even if it is empty).
  • The location of the QTmap.txt file must be configured using the AccountMapFileLocation parameter in the .ini configuration file for the relevant format.

 

CAUTION: DO NOT use Excel to edit an Account Mapping file that has a .csv file extension, because it will strip leading zeros off account codes, corrupting the mapping file. Use a text editor such as Notepad instead.

Each account map record is a comma-delimited record with the following format:

external account, external ccy, internal account, , exclude transactions flag

Where:

  • external account is an account in an import data file,
  • external ccy is an account currency in an import data file,
  • internal account is the Quantum target account,
  • exclude transactions flag* will toggle the processing of transactions for the account; set to true to exclude transactions; leave blank to process transactions normally.

    *NOTES:

    • The exclude transactions flag must be preceded by two commas (,,).
    • Only applicable for BAI, camt.053, camt.052, MT940, MT941, and MT942 parsers.
    • When set to true, transactions for the account are ignored, but account balances are updated from the import data file.
    • The exclude transactions flag completely stops Bank Importer from reading an account's transactions, resulting in significant processing time improvement over an instruction to exclude the account in the BKImport.ini configuration file (for example, FOR ACCOUNT 1234 SET EXCLUDE = TRUE), which will still read (but not process) each transaction for the account.

Examples

The following instruction will map all instances of the external account 019800000705 to the 00198801882996 account into Quantum

...
019800000705,BRL,00198801882996
...

The following instruction will exclude all bank flows when importing bank data* for the account 123-4567890-12 into Quantum.

...
123-4567890-12,EUR,123-4567890-12,,true
...
