Creating a New Report Page

This page contains some of the common procedures used to configure or edit a report created via the Workflow Report page.

To open the Workflow Report page

  1. In the Default perspective's Configuration menu under WORKFLOW, select Report.
  2. The Workflow Report page opens.

To display an existing record

  1. On the Workflow Report page, click browse .
  2. In the browse list, navigate to the item you want (or use the Search tool), and select it.

The current record settings appear.

TIP: Click on the thumbnails in the steps below to see example displays at the relevant stage of the worked example CUSBDR. To see all displays at once, go to the Example topic: Analytics report joined to Quantum static data.

To create a new page

  1. On the Workflow Report page, click New and enter or select the following mandatory settings.
    • Name - enter the unique module name for the target page (used in Module Security).
    • Title - enter the target page title (also used on browser tabs and in the site menu).
    • Url Name - enter the controller or last part of the target page URL (the new page's full URL must be unique).
    • Menu Area - select the area in the site menu where the target page menu item will appear (Analytics, Processing, Reporting, and so on)
    • Menu Group - enter the menu group under which the target page menu item will appear (an existing group or a new one).

  2. To set up a source, in the Sources grid menu, select New.

    The New Source dialog box appears, as below (in this example we will use all the data and not aggregate the data).

  3. Select All Data.
  4. In the Source browse list, navigate to and select a source, where the top-level folders are organized as follows:

    • Analytics contains Reporting, Static Data, and User Defined subfolders for accessing Analytics queries, static data, and custom table pages, respectively, and several Run... folders that contain test details;
    • APIs
    • Banking subfolders for accessing account balances
    • Database
    • Dealing subfolders for accessing those Quantum deal pages
    • Diagnostics contains Workflow Components and Workflow Component Problems (for reporting on all Workflow components and any errors and warnings on them)
    • Excel
    • Reporting contains Quantum reporting pages
    • Static Data subfolders for accessing those Quantum pages and all User data items

    The New Source dialog box opens again and shows your selected Source.

  5. In the Name field, enter your name for the source or accept the default (the Source value).
  6. In the Columns tab, click Add and in the Select Columns dialog box, select the data items that you want in the report's results, then click OK. (For Dealing, Static, Rates, and Reporting sources, you can also select linked items down to any level.)

    Each item selected becomes a row in the Columns grid.

  7. (Optional) To edit the displayed name for any selected data item: select its row in the Columns grid, and either use the grid menu to select Edit or click a linked value in the row; in the Edit Column dialog box edit the Display Name value; and click OK. Repeat as required.
  8. In the Edit Source or New Source dialog box, in the Settings area select Add.
  9. In the Select Setting dialog box (as below), select a required Setting name. This is a query parameter if the source is a query-type, otherwise it is a data item. Click OK.

    If there is more than one column/source, a Join tab appears. You can join any column from each source.

    TIP: For Dealing, Static, Rates, and Reporting sources, you may not need a Join, as you can select linked items down to any level.

  10. (Optional) To map data from one data source (the Source) to one already defined, so that data from both sources will appear in the target report results, either select the Join tab or open the Edit Source dialog box on any source but the first one.
  11. Use the browse list to select a previously defined source, for example the Left Source.
  12. Select the Type of Join: 
    • Left first - all from left, first match from source - all data from the left source plus matching data from the main source,
    • Left outer - all data from the left source , and matching data from the main source,
    • Inner - data from matching rows and no other rows,
    • Left anti - data from rows from the left source , and no other rows
  13. You can also add extra columns to join by selecting Add and using the Left Source Column and the Source Column (in any order). Click OK.

  14. To set up a filter, in the Filters grid menu, select New. The New Filter dialog box opens.
  15. Enter a Name for the filter and select the Type (Text, Date, Yes/No, Lookup, Amount, or Number); then - depending on the Type - select the Lookup Source and Display As format (Lookup, Dropdown, or Radio Buttons) and (if a Lookup) select or clear Allow multiple values as required, enter a Default Value or default radio button option if required, then click OK.

  16. Click OK. The New Filter dialog box closes and your new filters appear as new rows in the Filters grid. They also appear in the Report Filters folder in Expression Builder.

    TIP: To add or edit sources, click on one item in the Sources grid to open the Edit Source dialog box. To add or edit filters, click on one item in the Filters grid.

  17. To enable the system to resolve a filter, add it as a setting for one of the data sources. Select a source, and in the Edit Source dialog box, In the Settings area, click Add and select the filter to add. Repeat to add further query filters.
  18. (Optional) To set up an action you want to be available in the target report's view as a quick menu item or toolbar button: In the Actions grid menu select New and in the New Action dialog box:
    1. Select an Action Name. Click the top-level folder for the type of action (as below) and select the type of data to be actioned.
      • Create - to create a new record without opening the applicable page.
      • Delete - to delete a record.
      • Merge (where applicable) - to merge selected items.
      • Open - to open a new page.
      • Process - to perform a Deal, Request, o
      • Transfer - to output a file as defined by the selected Data Transfer Location record.
      • Update - to update a record.
    2. Enter a different Name for display, or use the default.
    3. In the Display In field, choose if the action will be implemented as a Quick Menu item or a Toolbar button.
    4. To set values as Inputs to the action: click Add (above the Inputs grid); select one or more items and click OK. Click each new item in the Inputs grid to open the Edit Setting dialog box; click the Add Expression link to open Expression Builder, create the required expression, Validate it, and click Done.
    5. (Create actions only) To optionally set values as Outputs resulting from the action (usually only one identifier/key value - for example, Deal Number for a Deal Create action): click Add (above the Outputs grid); select one or more items and click OK. Click each new item in the Outputs grid to open the Edit Output dialog box, select the Column for the output , and click OK.
  19. Click OK to close the New Action dialog box. Your added details are shown as a new row in the Actions grid.
  20. Repeat to add further actions.
  21. To enable data in spreadsheet form to be included as data sources, use the Structures grid menu. The Structures grid shows optional structures. The end user of the custom report can edit this data directly or via a spreadsheet application (as with Bulk Replace and Insert for Deal Map Items).
  22. On the Workflow Report page, click Save. The system validates your custom report and saves it.
  23. (Not applicable if you are exempt from Module Security) In the page menu, select Add to Current User to assign Module Security rights to your current User name.

    Alternatively, go to the Users page to assign rights to any users you are allowed to.

  24. To see the new Custom report in the site menu, refresh the page (press F5) or log out and in again.

  25. To try it out, go to the new page and run the query in the usual way: enter Filter values, configure Views, repeat earlier runs, and so on, as required.
