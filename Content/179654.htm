Importing Counterparty Account Rules

The Counterparty Account Defaults Importer supports the following QID/QXT-format fields.

Examples

NOTE: When using these format examples, separate each line of input with a line break. Separate by pressing Enter at the end of each line.

Insert a New Record

(QXT format)

@CounterpartyCashAccountRule
$new
Name = Rule 14895
Counterparty = SWIFTCNF Counterparty
Entities = ALL
TransactionTypes = ALL
Instruments = ALL
Facility = ALL
CashFlowType = ALL
Currency = USD
PaymentsReceipts = Both
SettlementOption = CLS
$$ADD Instructions#EffectiveDate = 10-Jul-2019;Bank = CLS Service Provider;AccountNumber = Bank CLS USD;DefaultPaymentMethod = NOEFT;AccountName = Bank CLS USD;AccountCurrency = USD
$Insert

Insert a New Record

(QID format)

EXENAME=CPACCDEF
$DoMenu=mfnew
mode=STOCK
cparty= Custodian Counterparty
trans_type=SE
sectype=Yield Statement
entity=THE Corporation
relationship=NONE
cflow_type=ALL
ccy=AUD
pay_rec=BOTH
effect_dt=1-Jan-2005
acc_no=CPSTK66663333
$DoMenu=mfinsert

Add an Effective Date Record

(QID format)

EXENAME=ACCDEF
$DoMenu=mfnew
mode=STOCK
cparty=Custodian Counterparty
trans_type=SE
sectype=Yield Statement
entity=THE Corporation
relationship=NONE
cflow_type=ALL
ccy=AUD
pay_rec=BOTH
effect_dt=31-Dec-2005
acc_no=CPSTK66668888
$DoMenu=mfmodify

(QXT format)

@CounterpartyCashAccountRule
$new

Counterparty = SWIFTCNF Counterparty
