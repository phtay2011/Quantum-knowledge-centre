Field structure, Layout XML Data record

In the Layout record, Field and FieldItem elements represent the actual message content. They provide the message structure and the linkage to the actual data.

Structure

<Field ...>
...
</Field>

Rules

A Section or a SubSection may contain one or many Field elements.

A Field element can contain zero, one, or many FieldItem elements - though not directly. FieldItem elements must be wrapped within some other nesting structure - see Supporting Elements and Supporting Attributes.

Attributes

All Field structure elements have the same attributes, as described in the following table.

Attribute

Required

Default

Description

colon

No

preserve

Defines how colons are handled in multi-line text:

  • collapse* - removes all leading colons of each line
  • preserve - does not remove any colons
  • remove - removes all colons inside the text

(Valid only with multiLine = "yes")

default

No

N/A

Defines the default value for this element if it contains an empty value.

formatType

No

N/A

Defines the formatting method to be used.

For more details, see Character Formatting Elements.

hyphen

No

preserve

Defines how hyphens are handled in multi-line text:

  • collapse* - removes all leading hyphens of each line
  • preserve - does not remove any hyphens
  • remove - removes all hyphens inside the text

(Valid only with multiLine = "yes")

id

No

N/A

Gives unique identification of the element

linefeed

No

preserve

Defines how line feeds are handled in multi-line text:

  • preserve - does not remove any user-entered line feeds
  • remove - removes all user-entered line feeds

(Valid only with multiLine = "yes")

mandatory

No

no

Defines whether or not the element is mandatory in the resulting message - possible values:

  • yes
  • no (for more details, see GlobalAttributes Element)
  • not_allowed
  • strict_conditional
    Child elements are mandatory if certain conditions are True, and excluded from the final message if those conditions are False.
  • loose_conditional
    Child elements are mandatory if certain conditions are True, and excluded from the final message if those conditions are False.

    That is, if it contains a valid value, it is mandatory; otherwise, it will be excluded.

map

No

N/A

Defines the mapping category to be used.

For more details, see Mapping Attributes.

maxLength

No

unlimited

Defines the maximum number of characters; works together with multiLine and maxLine.

For more details, see GlobalAttributes Element.

maxLine

No

unlimited

Defines the maximum number of lines allowed; can be used together with multiLine:

  • If maxLine is defined but multiLine is not, multiLine will be assumed to be yes.
  • If multiLine is defined as yes, but maxLine does not appear, then maxLine will be set to unlimited.

For more details, see GlobalAttributes Element.

multiLine

No

yes, no

Defines whether or not the element may contain values across multiple lines; works together with maxLine and maxLength.

For more details, see GlobalAttributes Element.

name

No

N/A

User-definable description of the element (optional)

nillable

No

no

Defines whether or not the element is allowed to contain an empty string as its value (see GlobalAttributes Element)

prefix

No

N/A

User-definable code, tag, or identification that can be used elsewhere

priorityPrefix

 

No

N/A

User-definable code, tag, or identification that can be used in a priorityGroup. Similar to prefix.

processMethod

No

N/A

Defines any additional processing method - possible values: count, repeat, repeat_asc, repeat_desc

For more details, see processMethod Attributes.

space

No

preserve

Defines how white space is handled in the element:

  • collapse - removes all leading and trailing spaces and replaces multiple spaces within text with one single space.
  • preserve - does not remove any spaces.
  • remove - removes all space inside the text.

For more details, see GlobalAttributes Element.

Where spaces are required to fill an entire field, the special attribute xml:space="preserve" must be used.

truncate

No

no

Defines whether or not excess characters will continue on a new line or will be truncated; can be used together with maxLength,with possible values of either yes or no.

  • If maxLength is defined but truncate is not, truncate will be assumed to be no.
  • If truncate is defined as yes, but maxLength does not appear, then maxLength will be set to unlimited.

validationID

No

N/A

Defines the associated warning or error message for the element

For more details, see Validation Attributes.

xpath

No

N/A

Defines an xpath expression that links to where the data can be found in the data source document.

*TIP: With the hyphen and colon attributes' collapse setting, removal is after the field content has been split into lines (according to the maxlength setting).

Examples

A Confirmation example

<Field id="15A" prefix="15A" mandatory="yes" nillable="yes"/>
<Field name="Custom Text">Hello World</Field>
<Field id="77D" prefix="77D" name="Additional Conditions"
  mandatory="conditional" validationID="BW0005">
</Field>
<Field id="30F" prefix="30F" mandatory="yes" processMethod="repeat"
  formatType="DATE_YYYYMMDD"
    xpath="//deal/cashflows/cashflow[@context='pay']/date"/>
</Field>
<FieldItem mandatory="yes" map="Day Count Fraction"
  xpath="/confirmation/action/deal/swapdeal/receive/couponDaysBasis"/>
