CashXplorer

Asynchronous query runs

(First introduced in Release SP 6.7.2, December 2018.)

CashXplorer queries now run asynchronously.

This means that your query runs to Refresh, Create, and Delete Data Versions no longer hold a connection open to the server until they finish - enabling you to perform other CashXplorer tasks. Notifications let you know when runs are completed (see below).

(Previously, you had to wh6> until your query run finished before you could perform other CashXplorer tasks, and if you left the page, you weren't notified when the query finished.)

Cash Concentration dialog box enhancements

Cash Concentration drill-down - change of perspective

(First introduced in Release SP 6.7.1, August 2018.)

The Currency, Bank, Account, and Account Entity column values within the Concentration Flow Drill Down Details dialog box are now derived from the opposite side's perspective - that is, from EITHER:

  • From Ccy, From Bank, From Account, and From Account Entity respectively, if the concentration flow amount is positive; OR...
  • To Ccy, To Bank, To Account, and To Account Entity respectively, if the concentration flow amount is negative.

NOTE: There is no change to how concentration flows are calculated; this is only an informational change and impacts only Concentration Flow drill down details. The column headings are also unchanged.

Cash Concentration dialog box - added Entity and Instrument columns

(First introduced in Release SP 6.7.1, August 2018.)

From Entity, To Entity, and Instrument columns have been added to the Cash Concentration Transfer dialog box for all three concentration types (Transfers, Bank Generated, and Cross Currency).

Cash Concentration dialog box - two Transfers tabs

(New from Release 6.8.0.)

The Transfers tab in the Cash Concentration Transfers dialog box has been replaced with two tabs, where:

  • the Today's Transfers tab displays only suggested transfers whose Settlement Date is today*, and
  • the Other Transfers tab displays all other suggested transfers (those whose Settlement Date is not today*).

* Today's date is determined by the user's location. If the CashXplorer Horizon date is Today and the Settlement Date of a Suggested Transfer is equal to Today’s date, then the transfer will be shown on the Today’s Transfers tab.

In addition, the Today's Transfers tab's Suggested Transfers grid now has a Cut-Off Time column for both the From Account and To Account.

This column shows the cut-off time for each transfer, based on the location settings on the applicable Cut-Off Time Rule record (see below). It remains empty where there is no relevant cut-off time rule for the transfer.

Cut-off Time Rules

(First introduced in Release SP 6.7.2, December 2018.)

The best-matching Cut-Off Time rule (if any) will apply to today's Cash Transfer and Actual Cash Flow deals created from the Cash Concentration page (either as added deals or Cash Concentration transfers), and if the specified cut-off time has passed, the Settlement Date will be adjusted according to that rule.

NOTE: The rule's Adjust settlement date option does not apply to Cash Concentration, and the Settlement dates for deals created in this way will not be modified by deal defaulting logic.

CashXplorer notifications

`(First introduced in Release SP 6.7.2, December 2018.)

The CashXplorer processes Run Query, Refresh Version, and Delete Data Version are now included in the Latest Notifications drop-down list, as shown in the example below.

When you click on a notification in the list, to open the Notifications page (as in the example below), you can then click on a CashXplorer notification to navigate to the CashXplorer page for the relevant process, displaying the output the notification relates to.

Notifications are also sent as e-mails to the address (if any) associated with the user account. The e-mail also contains a link back to the relevant page and/or output the notification relates to, which can be viewed while the email addressee is logged on.

TIP: When you configure email notifications for ALL users using the EmailNotificationsEnabledList key in the Web.AppSettings.config file, the processing key value now also covers CashXplorer notifications.

Local Time supported

(First introduced in Release SP 6.7.2, December 2018.)

CashXplorer now displays the User Location timezone setting rather than System date/times for:

  • Run Query - First Horizon Start Date and.
  • Worksheet Adjustment - Value Date.

Messages will still be logged using the database server date/time (for example, in the Report View's History tab, Message Log reports, and so on).
