About the Generate Fees process

After you start fee generation and validation passes successfully, the system does the following.
 

  1. Creates Fee Base Amounts:
    • from information stored in the Position History table (for Position fees, Facility fees, and Transaction fees),
    • for the specified active Fee Deals (or All),
    • over the date range you specify, and
    • according to the specified Base Reset dates in each Fee Deal schedule.
  2. Calculates the Fee cash flows.

    If the day prior to the Calculation End-date or Invoice Date (whichever is earlier) for a given Fee Deal is within the specified Start and End Date range, the system calculates a fee cash flow ready to be processed on the Calculation End Date or Invoice Date, as applicable.

    For each Fee Deal, the required cash flows are created.

    If True Ups apply for any Fee Deal, then where a scheduled Invoice Date falls in the specified date range, cash flows for estimated fees and for the True Up adjustments for the previous period are also generated where applicable. Note that:

    • Where Invoice Date is equal to Calculation End Date: the fee cash flow amount is calculated by summing up all the fee amounts generated from the fee base amounts for the entire calculation period, plus or minus any True Up adjustment from the previous period.
    • Where Invoice Date is prior to Calculation End Date: the fee cash flow amount is calculated by summing up all the fee amounts generated from the fee base amounts up to Invoice Date (exclusive) plus the Estimated Amount for the period between Invoice Date (inclusive) and Calculation End Date (exclusive), plus or minus any True Up adjustment from the previous period.

      If a non-zero True Up amount (negative or positive) is outstanding beyond the Maturity Date of the fee deal (the last calculation date), a cash flow entry will be created, but will not be processed unless you manually change its status. For more details, see Defining Fee Estimation Parameters (when applying True Ups).

    • If the Suppress true up cash flow processing option is selected (Other Details tab, Fee Deal page), True Ups will still be calculated and displayed if and where applicable, but will not be applied to adjust the cash flow amount.

    Fee cash flows will be updated if the cash flow's Deal Date and Payment Date are between the Start Date and End Date specified in the Generate Fees dialog box.

  3. Calculates the Fee Accounting Entries (if requested).

    If you selected the Calculate G/L Entries option, then the accounting entries are created for each Fee Base Amount, according to the specified G/L codes in each Fee Deal (Accounting tab) for Accrual, Calculation, Invoice, and Payment Dates within the date range you specify, as follows.

    NOTE:Accruals and amortizations occur on a straight-line basis and are aggregated for each accrual period (the period bounded by the Start Date and End Date you specify).

    If True Ups apply:

    • Where Invoice Dates equal to Calculation End Date, one Accrual entry is generated for the date range. The value of the accruals is the sum of the fee amounts for the date range, exclusive of the Invoice/Calculation End Date. If there is a True Up value from the previous period, the full amount is booked/recognized on the first day of the calculation period.
    • On the last calculation date (Maturity Date) of the fee deal, no G/L entries are created for a True Up value calculated, even if you process the cash flow.
    • Where Invoice Date is prior to Calculation End Date, the value of the accruals is the sum of the fee amounts up to the Invoice Date (exclusive) plus the daily Estimated Amount accrual plus the full amount of the True Up (to be booked on the Calculation Start Date).
    • When the True Up value is negative, the G/L entry will be reversed - the Credit code becomes the Debit code, and vice versa (G/L entry amounts are always positive).
    • In unusual circumstances the fee cash flow amount can be negative if there is a large True Up value or if the last calculation period is very short (for example, a stub period). When this occurs, the G/L entries created for Invoice, Calculation End, or Payment dates are reversed as required.
    • If you change any G/L codes for a Fee Deal (Accounting tab), the Generate Fees process uses those codes from that point forward only. You cannot use the new G/L codes retrospectively for any Fee Deal accounting entries already posted.

    G/L entries are recalculated as follows.

    • There must be no existing posted G/L Entries whose Start Date is greater than or equal to the Generate Fees Start Date. Also, no fee cash flows can have been posted into Quantum for the period specified.
    • Provided the above conditions apply, any existing unposted G/L entries whose Start Date and End Date are greater than or equal to the Generate Fees Start Date are deleted, and new G/L Entries are calculated for the period specified.
    • G/L Accrual or Amortize Entries are recalculated from the Start Date of the last accrual entry that currently exists.

      Example: Suppose a Fee Deal's last G/L Accrual Entry has a Start Date of 15 Oct 2019 and an End Date of 23 Oct 2019.

      If you re-run Generate Fees for that Fee Deal over the period from 20 to 27 Oct 2019, then the next G/L accrual entry for the Fee Deal will be from 24 to 27 Oct 2019.

    • The Generate Fees process does not create accounting reversals of previously generated G/L entries. These must be done manually in Quantum (using G/L Journal Entries), as required.

      NOTE:For more details on how Fee Generator handles accounting entries when it is re-run, see About Generate Fees reruns.
