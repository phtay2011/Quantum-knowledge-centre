action element, Data Template XML

Confirmations only.

The action element has a subset of the following major structural elements, depending on the message type.

<action>
  
  <deal>
   
    <subdeals/>
    <cashflows/>
    <flows/>
    
    <dealEvents/>
    
  </deal>
  <actionEvents/>
  <actionRatesets/>
  <actionRollOvers/>
</action>

Within this structure, there are many sources of information. The following sections define some of the more complex parts of the structure.

For more details, see the SeedXMLDataTemp_ALL.xml file provided in your Site's Confos folder. For specific message types, see the files named SeedXMLDataTemp_MTnnn.xml, where nnn is the message type - for example, SeedXMLDataTemp_MT300.xml, and so on.

Context attributes, XML Data Template

Confirmations only.

There are multiple instances of context attributes throughout the structure of the Action element. These attributes provide an easy way to identify and access parts of the structure, rather than searching the elements themselves.

This topic describes the Context Attributes for the following elements.

Element

Context Attribute

Definition

account

mode

Identifies the type of Bank Account (the mode in the Bank Accounts form): Bank, Broker, or Stock, as follows.

  • bankacc.bankacc_mode - 10860001 : mode = "actual"
  • bankacc.bankacc_mode - 10860002 : mode = "broker"
  • bankacc.bankacc_mode - 10860003 : mode = "stock"

analysisCode

name

The anaset.name value for that Analysis Code. This attribute provides a quick way of finding an Analysis Code.

Analysis Codes can be set generically at the Deals level, or more specifically at the Transaction Type level.

cashflow

context

Defines Pay and Receive cashflows (saves having to check the sign of the amount):

  • cflow.amount < 0 : context = "pay"
  • cflow.amount > 0 : context = "receive"

(No cashflows should have a zero amount.)

category

name

This attribute provides a quick way of finding a Category.

The structure for a Counterparty category is generic, but the underlying database structure has a different field for each category:

  • "Bank External" source="cparty.bank_ext"
  • "Deal External" source="cparty.deal_ext">
  • "Bank Internal" source="cparty.bank_int"
  • "Deal Internal" source="cparty.deal_int"
  • "Debtor/Creditor" source="cparty.debt_cred"
  • "Issuer/Acceptor" source="cparty.issuer"
  • "Issuing and Paying Agent" source="cparty.is_iss_pay_agent"
  • "Broker" source="cparty.fu_exe_bro"
  • "Clearing Broker" source="cparty.fu_clr_bro"
  • "SWIFT Financial Institution" source="cparty.swift_fi"
  • "Settlement Bank" source="cparty.bank_settl"
  • "Tax Authority" source="cparty.taxor"
  • "Custodian" source="cparty.is_stock_agent"

clearingFields

name

The currency.clear_cap value for that Clearing Field. This complex field contains details about more than one clearing field for some currencies, and needs to be retrieved with reference to the currency.clear_chk field.

This attribute provides a quick way of finding a Clearing Field.

flow

context

Currently, there are no context attributes for flow.

identifier

name

The paymethd.name value for that Identifier.

An Identifier can be set for Locations (locidents), Business Structures (entidents), and Counterparties (cpidents).

This attribute provides a quick way of finding an Identifier.

intermediary
ClearingField

name

The currency.clear_cap value for that Clearing Field. This complex field contains details about more than one clearing field for some currencies, and needs to be retrieved with reference to the currency.clear_chk field.

This attribute provides a quick way of finding a Clearing Field.

location

context

This is where multiple locations for a deal are stored in the dealloc table - for example, for Money Market and Swap Deals. There can be two types of location: Rate Set and Cashflow.

In the dealloc table, dealloc.loc_type defines the type of location:

  • Codeset Id: 10100001 means "cashflow" : context = "cashflow"
  • Codeset Id: 10100002 means "rateset"   : context = "rateset"

raComponent

context

The racmpt.cmpt_type value, which identifies the type of Repo Component, as saved:

  • cmpt_type: SEC   : context = "SEC"
  • cmpt_type: CASH : context = "CASH"

actionEvents element, Data Template XML

Confirmations only.

This element finds and populates the <actionEvents> structure for Events that are related to the current Action (this relationship is a functional one rather than a pure database one).

The <actionEvents> structure will be populated with the most relevant Events for the current Action.

Rules - DEAL Action

All Transaction types have a DEAL Action.

Currently, there is:

  • No direct relationship between DEAL Action and the Events
  • No requirement to populate <actionEvents> for a DEAL Action

Rules - CHANGE

Currently, there is:

  • No direct relationship between CHANGE Action and the Events (other than to image the previous Events)
  • No requirement to populate <actionEvents> for a CHANGE Action

Rules - DELETE

All Transaction types can have a DELETE Action.

Currently, there is:

  • No direct relationship between DELETE Action and the Events (other than to have no current Events)
  • No requirement to populate <actionEvents> for a DELETE Action

Rules - RATE SET

Transaction Types Affected

SI - Interest Rate Swap

SC - Cross Currency Swap

Requirement

There is a direct relationship between a RATE SET Action (Actions Table) and the RATE SET Event (Events/I Table).

A RATE SET Action will either link to a RATE SET Event (events table) or an Imaged Event (eventsi table) if it has subsequently been re-RATE SET (Amended).

The structure needs to be populated to show what the Event looked like when the RATE SET was performed.

For the given RATE SET Action, obtain the appropriate RATE SET Event and populate the structure.

Example

<actionEvent>

  <eventNumber>105808</eventNumber>
  <flowNumber>4</flowNumber>
  <actionNumber>53978</actionNumber>
  <actionDate>2005-01-07</actionDate>
  <effectiveDate>2005-01-07</effectiveDate>
  <comments>RATE SET</comments>
  <rateOrPrice>4.2</rateOrPrice>
  <margin>0.1</margin>
  <amount>-113917.81</amount>
  <flag>PAY</flag>
  <isCashflow>N</isCashflow>
  <isNotional>N</isNotional>
  <closeAmount>0</closeAmount>
  <confoStatus>R</confoStatus>
  <currency>

    <code>USD</code>
    <name>UNITED STATES DOLLAR</name>
    <daysConvention>ACTUAL/360</daysConvention>
    <cashflowNonBusinessDay>M</cashflowNonBusinessDay>

  </currency>

</actionEvent>

custom Element, Data Template File

Confirmations only.

Enables you to add, into the Data Template file, "hard-coded" values that will be available to all messages built using that Data Template file.

This is another alternative to adding these values in the Layout file for each message.

Each seed Data Template file will not contain any values for these custom fields. They are fully user-definable and are positioned in various parts of the structure.

Structure

<custom>
  <item name="..."/>
</custom>

actionRateSets Element, Data Template File

Confirmations only.

This element populates the actionRatesets structure when the currently linked Action is RATE SET.

Structure

<actionRateSets>

  <actionRateSet context="..." description="...">

    <eventNumber/>
    <rateSetDate/>
    <periodStartDate/>
    <periodEndDate/>
    <calcBasedOnAmount/>
    <rateOrPrice/>
    <margin/>
    <baseRateOrPrice/>
    <currency>

      <code source="..."/>
      <name source="...currency.name"/>
      <daysConvention source="..."/>
      <cashflowNonBusinessDay source="..."/>

    </currency>
    <calcAmount/>
    <netAmount/>
    <isCashflow>

  </actionRateSet>

</actionRateSets>

The applicable rules and attribute settings for Swap Deals are summarized in the following topics.

RATE SET Event, Data Template File

Confirmations only.

The RATE SET Event associated with the current RATE SET Action has the following attribute settings.

  • context is based on the events.flag value on the same side of the Swap (either PAY or REC).
    • context = "pay" where events.flag = 'PAY'
    • context = "receive" where events.flag = 'REC'
  • description is the events.comments value of the current RATE SET Event.
  • eventNumber is the events.event_no value of the current RATE SET Event.
  • rateSetDate is the events.action_dt value of the current RATE SET Event.
  • periodStartDate is the events.effect_dt value of the current RATE SET Event.
  • periodEndDate is calculated by looking for:
    • The next RATE SET Event with the events.flag value on the same side (either PAY or REC).
    • If there is no next RATE SET Event, look for the next INTEREST Event with the events.flag value on the same side (either PAY or REC). This should be the last INTEREST Event.
    • periodEndDate is the events.effect_dt value of the next-found RATE SET or INTEREST Event.
  • calcBasedOnAmount is calculated by:
    • Starting from the beginning of the deal and working forwards, up until an events.effect_dt value equal to or less than the periodStartDate value.
    • Calculate the Remaining Face Value/Notional Value as at periodStartDate. Items such as NOTIONAL, UNWIND NOTIONAL, REPAY NOTIONAL, PRINCIPAL, UNWIND PRINCIPAL, and REPAY PRINCIPAL need to be taken into account.
    • The events.amount value needs to be changed for each of these events. This needs to be calculated for the side of the Swap applicable to the RATE SET Event.
  • rateOrPrice is the events.rate value of the current RATE SET Event.
  • margin is the events.margin value of the current RATE SET Event.
  • baseRateOrPrice is the rateOrPrice value minus the margin value (the difference).
  • currency is the events.ccy value of the current RATE SET Event.
  • calcAmount is calculated by using similar logic to that of the deal program's Rate Set form. This shows the amount associated with a particular RATE SET, as follows.
    • Where there is one RATE SET to one INTEREST event, events.amount will be zero on the RATE SET, and non-zero on the INTEREST event. calcAmount is set to this non-zero amount.
    • Where there are many RATE SET events to one INTEREST event, events.amount will be zero on the RATE SET and non-zero on the INTEREST event. calcAmount is set to the contribution to the INTEREST by the particular RATE SET event.
    • Where there is one RATE SET to many INTEREST events, events.amount will be zero on the RATE SET event and non-zero on the INTEREST event. calcAmount is set to the total of the INTEREST that the RATE SET event causes.
  • netAmount is not applicable to the RATESET Event.
  • isCashflow is the events.cashflow value of the current RATE SET Event.

INTEREST Event, Data Template File

Confirmations only.

The INTEREST Event associated with the current RATE SET Action is obtained if:

  • The INTEREST Event has an events.effect_dt value equal to the periodEndDate value of the current RATE SET Event.
  • The INTEREST Event has the same events.flag value as the current RATE SET Event.

NOTE: The INTEREST event can have an events.comment value of 'INTEREST' or 'INTEREST (ACC)'.

Where obtained, the INTEREST Event associated with current RATE SET Action has the following attribute settings.

  • context is based on the events.flag value on the same side of the Swap (either PAY or REC).
    • context = "pay" where events.flag = 'PAY'
    • context = "receive" where events.flag = 'REC'
  • description is the events.comments value of the INTEREST Event.
  • eventNumber is the events.event_no value of the INTEREST Event.
  • rateSetDate is not applicable to the INTEREST Event.
  • periodStartDate is calculated by looking for:
    • The previous INTEREST Event with the events.flag value on the same side (either PAY or REC).
    • If there is no previous INTEREST Event, look for the previous RATE SET Event with the events.flag value on the same side (either PAY or REC). This should be the first (and only) RATE SET Event on that side.
    • periodStartDate is the events.effect_dt value of the previous found INTEREST or RATE SET Event.
  • periodEndDate is the events.effect_dt value of the INTEREST Event.
  • calc BasedOnAmount is not applicable to the INTEREST Event.
  • rateOrPrice is not applicable to the INTEREST Event.
  • margin is not applicable to the INTEREST Event.
  • baseRateOrPrice is not applicable to the INTEREST Event.
  • currency is the events.ccy value of the INTEREST Event.
  • calcAmount is the events.amount value of the INTEREST Event.
  • netAmount is the net amount of the INTEREST Events associated with the current RATE SET Action. Where there are INTEREST Events for both sides of the Swap on the same day and the currencies are the same, netAmount will get populated in this element if:
    • context = "pay" and the net amount is negative,

      -or-

    • context = "receive" and the net amount is positive.

    Otherwise it will be zero, or of there is only one INTEREST Event, then the netAmount value will be the same as the calcAmount value.

  • isCashflow is the events.cashflow value of the INTEREST Event.
