Enhancements to Weighted Average Closeout Processor

(First introduced in Release 6.4.1 SP, March 2017.)

Enhancements have been made to Weighted Average Closeout processing in this version as follows:

  • The Weighted Average Closeout Processor page functionality has been brought into line with other Processing pages. The Query area configures the portfolio, while the deals created by the processor are displayed in the View area.
  • The Weighted Average Closeout Processor page now enables you to order the processing of deals chronologically.
  • Notional deals are now created by the Service Account and a special/dedicated user account (WACOProc) is no longer required.
  • Scheduler has been updated to process Weighted Average Closeout (WAC) queries.

Implementation

The Weighted Average Closeout Processor page now looks and behaves like other Quantum Processing pages. The Query area enables you to configure and save the filtering criteria, while the View area displays the results of the query.

Order of Processing Deals

A new Deal Order option has been added to the Query Options area, which enables you to select the order in which deals are processed when the Weighted Average Closeout Processor is run.

  • Buy before sell (default) - Select this option to process deals grouped by Buy first and then by Sell. When this option is selected, deals are processed in the following order:
    • Settlement Date
    • Buy
    • Sell
    • Deal Number
  • Chronological - Select this option to process all deals sequentially. When this option is selected, deals are processed in the following order:
    • Settlement Date
    • Input date time
    • Deal Number

User Account Creating Deals

All notional deals, required for processing, are created by the Quantum Service Account. This ensures that the processing tasks are executed with the correct permissions.
