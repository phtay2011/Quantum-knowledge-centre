Importing Bank Data from ECHOS

If you want to import Bank data from ECHOS:

  • You must set up the Bank Import INI configuration file to enable this. For example, DataManager = Echos and Parser = Echos must be specified. For more details, see ECHOS Configuration Parameters and BkImport.ini Configuration file.
  • Bank importer must be running, using that Configuration file.
  • The ECHOS Message Listener Service must be running.

Process Overview

  1. The ECHOS Message Listener Service "listens" for any incoming statements and passes them to Quantum.
  2. The initial Message LifeCycle state of any received statements is set to Pending.

    Received statements can be viewed using the ECHOS Message Management page.

  3. When Bank Importer next looks for new messages, it will process any received messages with a LifeCycle state currently set to Pending.
  4. When Bank Importer has successfully uploaded these statements (applying the relevant ECHOS settings from the Configuration file,) the LifeCycle state is set to Completed.
  5. If the statement could not be uploaded successfully, the LifeCycle state is set to Error.

    With error messages, you can the ECHOS Message Management page to:

    • Change the LifeCycle state back to Pending if you want to try and upload that statement again.
    • Mark the statement as Not Required - for example, if it is a duplicate.
    • Delete the statement.
