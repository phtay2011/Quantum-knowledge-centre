Workflow Action page

Use this page to combine a sequence of actions into a new action with inputs and outputs and referenced data sources. That configured workflow action then becomes available for selection wherever an action can be chosen in the system

You can also define an action that has no included actions - for example, to only perform validations before a Delete is allowed.

The Workflow Action page has the following grids.

  • Inputs define what can come into the workflow action. These form the context of what can go into the embedded actions.
  • Outputs define what the embedded actions can output into and what the workflow action ultimately returns. Outputs are also part of the context and can be passed into subsequent actions).
  • Aggregated Inputs define the data set that goes into a workflow action. These aggregated inputs can be used in expressions (for example, in validations, or actions).
  • Calculated Outputs define outputs based on expressions applied to Sources, Inputs, and Outputs and evaluated when the Workflow action completes.
  • For each - Add For Each – use this expression field to create data sets that suit your business needs. The Workflow Action calculates this For Each expression for all rows. Grouping of the data is based on the expression result. If any record from the group doesn’t pass the validation, an error message is displayed in Message Log.
  • Validations are applied to the inputs and must all pass for the first action to be triggered.
  • Actions list the actions to be triggered, in order. A prerequisite condition must hold true for the action to execute. The action can apply to all lines in a data set, the first line, or the last line.
  • Sources can be used in the Calculated Output expressions.

When a workflow action has completed, the system returns the Success Message text. Any errors come from the embedded actions. When any of the actions fail, the database rollback happens.

NOTE:If any of the actions have resulted in a change outside of the database (e.g. a file was created or a message was sent), then this cannot be reverted.

You can also export an action (using the Browse window menu's Export function) In a supported format (Excel, CSV, XML, PDF, or QXT), and import one (QXT format) via the Static Data Import Status page.
