Workflow Pages

Use the Workflow Page module to modify the user interface of a deal-related core (original or "out-of-the-box") page, by editing its XML syntax, either changing the Core page or creating a new Variant of it.

You can:

  • change page titles and field labels;
  • restrict access to page elements such as fields, menu options, and tabs, by hiding or disabling them;
  • position analysis codes with other fields;
  • dynamically hide analysis codes when other fields are hidden;
  • add other data sources - for example, to provide values for validation and calculated columns;
  • add optional actions, with prerequisite conditions and output values;
  • add calculated fields with defining expressions;
  • add validation logic;
  • control access to variant pages through Module Security;
  • export your modified core and variant pages - for example, for importing into other databases;
  • add actions already present in a deal.

NOTE:

  • Select the Core option to modify a page for a module that is part of the core product. Only one core page can exist, and its URL cannot be changed.
  • Select the Variant option to create a new, modified version of a core page. You can create multiple variants of any core page, with each one having its own unique URL (and an entry in the same menu as the core module).
