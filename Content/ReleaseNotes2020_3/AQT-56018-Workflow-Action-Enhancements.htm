Workflow action enhancements

You can select the new actions described below on many pages where you can specify a workflow action - including on a Workflow Report, Workflow Page, Workflow Action, or Data Transfer Location (DTL) page, as applicable.

Deal customization actions added

You can now select deal customization actions for those deal types currently supporting them (for Cross Currency and Interest Rate Swaps, Money Market and Securities deals). These actions, available under Process in the Select Action dialog box, are:

  • Add Deal Map Item
  • Update Deal Map Item
  • Profile (only available for FRN-CGL Bond & Certificate Deposit formulas)
  • Delete Profiling (only available for FRN-CGL Bond & Certificate Deposit formulas)

The same restrictions apply as on the applicable Dealing page in each case. For example, the page you create will support only some Formula Timing options, and you can only add certain deal map items.

Create, update, and delete actions added

You can now select create, update, and delete actions for the following deal types and static data, plus the specific dealing actions indicated:

  • Counterparty and Counterparty Group
  • Entity and Entity Group
  • User
  • Currency Option deals – also Average Rate Fixing, Cash Exercise, Knock In, Knock Out, Lapse, Physical Exercise, Rollback
  • Cross Currency and Interest Rate Swap deals - also Unwind, RateSet, plus Customization actions where valid (see above)
  • Money Market and Security deals - also RateSet, Unwind, Closeout, Profile, plus Customization and Profiling actions where valid (see above)

Use workflow actions to process reports

When you configure a multi-select action on a workflow report (using a toolbar button), then you can select that report and action as a new workflow action under Process > Report Name > Action Name. The action has an input for each of the filters defined on the report.

Such workflow actions enable you to invoke, in bulk, other actions based on data retrieved by reports. When a report defines multiple actions on the same report, each action is represented with its own report-executing action.

When you invoke such an action:

  • the report runs with the specified input values passed into the filters, and the action executes on all results from the report,
  • the action runs synchronously,
  • any errors in running an action on a report return as they would on the original report, and
  • the system logs each error will in the message log.

Previously, you needed to do this by scheduling the report to process the action. Now you can perform this processing from a workflow action. For example, you could call one of these workflow actions on deal save, so that all other deals in the same deal set get updated.
