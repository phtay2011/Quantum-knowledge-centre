Adaptiv performance - Allow multiple calculation servers

When configuring Adaptiv settings on the Analytics Server during a Quantum installation, you can now configure one or more access URLs for its services.

Background

Some client sites schedule thousands of CVA, PFE, and HVaR queries to run overnight and to post output to dedicated Analytics reporting tables. All three query types can each process a large number of simulations or scenarios, so processing times may be up to several minutes each, even when you've selected only a small number of deals.

Such sites need the ability to run multiple Adaptiv calculation servers in order to process the expected load of queries that they will run on a scheduled basis overnight.

Configuration

When configuring the Adaptiv Analytics Settings dialog box (as part of an Analytics Server configuration during the Quantum installation), you specify the location of the Adaptiv Analytics Engine and one or more access URLs for its services.

If you require one server, enter its URL as in the following example.

If you require more servers, enter their URLs after the first (main) service, each preceded by a space and an identifying alphanumeric label in square brackets. (Do not put a label on the main service.) There is no maximum number of servers that you can configure, but they must each exist on different physical machines. An example of a Service Url entry for multiple servers is given below:

http://AB-WS100 [S1]AB-WS102 [S2}AB-WS103

Implementation

All Adaptiv queries and Adaptiv processes run from within Quantum will be run on the first (main) server. The only time we use the other services is when running Adaptiv queries through the command line where you can override the default service by specifying a Run On Server parameter in the Analytics Batch file, as below.

Example command line.

AGCommandLineSupport.exe ANAQUERY SITE=DEMO_AA_QT USERNAME=INUS PASSWORD=xxYYzz98 ANALYTICSBATCHFILENAME="C:\Projects\AABatchFiles\command_Query.txt"

command_Query.txt batch file example content:

/PARAMETER=(~\QA\CVA DB;[Keep Job File=true,Run On Server=S1])

/EXPORTTODB=(~\QA\CVA DB;ViewName=CVA DB;aa_rptoutput;TRUE;TRUE;TRUE;TRUE)

If you want to run the query on the main server, don't include the Run On Server parameter in your command line.
