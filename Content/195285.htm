Configuration and security setup for Commodities

Use the following pages to set up information relating to configuring pages or security.

Application page

Requirements

Analysis Codes

Create and maintain any Analysis Codes specific to:

  • Commodity static data records - by setting Scope to Commodity, or...
  • Commodity Swap and Exposure deals - by setting Scope to Commodity Swap Deals.

    Analysis Codes with Scope set to Deal also apply.

TIP: Analysis Codes with Scope set to Instrument Type apply to all instruments (including Commodity Swap and Exposure ones).

Crystal Mapping

Mandatory for Paper confirmations.

The seed Crystal report CMS Deal - supplied specifically for Commodity Swaps - must be mapped to the CONFFMT module (as must any other Crystal reports you want to use with Commodity Swap Paper confirmations).

Data Security

If the Enable Resource Security option is selected and (under Filtered Resources) the Instrument setting is Enabled, then Group- and User-level Resource Security permissions can be applied for all or individual Commodity Swap and Exposure instruments - see below.

Under Approval Level, if the Instrument setting is Single, then approval will be required (by a single Approver) for changes to any instruments (including Commodity Swap and Exposure ones).

Group

Resource Security tab / Instruments:

  • If Instrument Resource Security applies, set each non-exempt group's access type for all or individual Commodity Swap and Exposure instruments.

Module Security tab:

  • Set each non-exempt group's permissions for each Commodities-related module - see Module Security.

User

Resource Security tab / Instruments:

  • If Instrument Resource Security applies, for each non-exempt user not inheriting from a Group, set their access type for all or individual Commodity Swap and Exposure instruments.

Module Security tab:

  • For each non-exempt user not inheriting from a Group, set permissions for each Commodities-related module - see Module Security.
