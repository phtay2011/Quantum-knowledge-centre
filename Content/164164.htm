When to Use Set Base Currency Amounts

This utility is not mandatory.

In the normal course of events, it would not be part of your regular accounting processes because the G/L Close and G/L Interface pages perform the same functions. Also, each record contains enough information to find the translation rate and therefore to calculate the amount where a foreign currency amount does not have a base currency equivalent. In addition, if account balances are not stored up to the date required, the system determines the balance at that time.

However, you may need to use it in the following situations.

  • You are instructed by the FIS Help desk to run it to resolve an anomaly (for example, c below).
  • G/L Close or G/L Interface are not run regularly (or at all). Then use this utility to update the base amounts on a regular basis; otherwise the performance of Queries such as G/L Transactions and Currency Revaluation will deteriorate over time, because they will have to recalculate all G/L base amounts each time they are run.
  • Base amounts have been calculated and updated using incorrect FX-rates. Run this utility with Reset mode selected (with the required Force updates from date) to recalculate the base amounts (not possible on G/L entries that have been processed by G/L Interface).
  • You have a large number of G/L entries and have been advised to run the utility before running your accounting processes. This can greatly improve efficiency of Queries such as G/L Transactions and Currency Revaluation because they do no need to calculate base amounts.
