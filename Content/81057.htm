About the Data Template XML Data records (Confirmations)

The Data Template XML Data record for each Confirmation External Method:

  • defines which Quantum data is to be retrieved, depending on the Deal Action that has triggered the Confirmation message,
  • is accessible using the XML Data page,
  • contains XML code, providing a structure that is extendable and flexible to handle future needs, and
  • can use, or be based on, the Seed XML Data record supplied in Quantum for that message type and named SeedXMLDataTemp_MTnnn, where nnn is the message type - for example, SeedXMLDataTemp_MT300, and so on.

    TIP: You may also use the general record SeedXMLDataTemp_ALL.

For Confirmations, the Data Template XML Data records have the following general structure.

<confirmation>
  <message/>
  <action/>
  <system/>
</confirmation>

Where:

  • message: When a message is to be constructed, some of this information will be sourced from information about that message - for example, message ID or other references. It may also be necessary to use a previous message when constructing a new message.
  • action: Deal Confirmations are based on a Deal Action, which is the source point of all related information used to construct a message. The relationship flows from Action to the Deal, with a complex structure of associated information.
  • system: System-related information - for example, System Time.

The Data Template records include the system-generated unique end-to-end transaction reference (UETR).

NOTE: The SeedXMLDataTemp_ALL template record can accommodate data for sub-deals for the following deal types.

  • Interest Rate Options
  • Futures
  • Actual Cash Flows
  • Initial Margin
  • Bank Account Transfer
  • Stock Account Transfer
  • Letter of Credit
