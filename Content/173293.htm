Money Market and Securities Closeout Methods

When you input a Buy deal and a Sell deal based on the same tradeable instrument, closeout can occur.

To determine which deals are to be closed out, Quantum matches deals using the Closeout Method specified on the underlying Money Market or Securities instrument.

A Gain or Loss entry is posted to the deal that is fully closed out. In addition, closeout settlement entries are posted to the continuing deal.

Matching elements

For matching to occur, certain elements must be the same in both deals. The type of deal determines which elements need to be matched.

  • Acceptor/Issuer (Securities only)
  • Accrual Method
  • Coupon (no longer matching element for FRN Overnight Index Swaps)
  • Coupon dates
  • Coupon frequency
  • Coupon type
  • CUSIP (Securities only; see Note 1)
  • Dealer (if Closeouts by Dealer is selected in Business Structure)
  • Entity
  • Ex interest Days
  • Fixed/Floating
  • Formula / Timing
  • Instrument
  • Interest days
  • ISIN (Securities only; see Note 1)
  • Maturity Date
  • Taxes
  • Accrued Interest WHT (see Note 2)
  • Capital Discount/Premium WHT (see Note 2)

NOTE:
1 (Securities only) FRN closeouts are possible if you have specified a CUSIP or an ISIN. However, Quantum will not allow a closeout for an Asset-backed FRN with stock flows if the Dealt Factor on the closing deal differs from that currently applied on the original deal through the Deal Profiler (Deal Map tab).

2 The two WHT options apply only to instruments with the following settings:

  • Formula = BOND, Timing = PERIODIC (and for Securities only, Include embedded optionscleared), or
  • Formula = FRN-CGL, Allow asset backedcleared, Allow multiple ratesetscleared.

3 (Securities only) ILB with amortization closeouts are possible if the following conditions are satisifed:

  • Index is updated,
  • Amortization frequency is the same on both deals, and
  • Amortization dates in both deals are aligned.

Selecting a Closeout method

On the instrument page's Details tab, set Closeout or Unwind to Closeout, then select one of the following methods from the Closeout Method drop-down list.

  • CLOSEST YIELD - with this method:
    • Quantum first identifies matching deals in terms of the elements listed above, then matches them again in terms of yield rate,
    • deals with a yield most like that of the closing deal are selected, and
    • from these, earlier deals are selected first, and if there are still identical matches, Quantum selects the deal with the lowest deal number.
  • FIFO - with this method:
    • Deals are matched on a first-in, first-out basis, determined by the deal date,
    • the earliest deals are closed out first,
    • if two deals have the same deal date, then the deal with the earliest settlement date is used, and
    • if the deals also have the same settlement date, the lowest deal number is used first to closeout the deal.
  • FIFO SETTLE - with this method:
    • deals are matched by settlement date, and
    • deals dealt today for settlement today will be closed out ahead of deals dealt today for settlement at a future date.
  • LIFO - with this method:
    • deals are matched on a last-in, first-out basis, determined by the deal date, and
    • the latest deals are closed out first.
  • MATCHED - with this method:
    • you manually match deals using a page menu function (Match Deals for Closeout) and selecting them and the applicable amounts in a grid.
    • For more details, see Matched closeouts.
  • WEIGHTED AVG* (available only for Securities where the Accounting Rule for the instrument specifies Adjusted Book Value) - with this method:
    • accruals on Capital Premiums and Discounts are not supported,
    • deals are initially matched on a FIFO basis, and
    • Quantum then uses a Weighted Average method to calculate the book value of the portfolio, and to create the Profit and Loss General Ledger entries for the closeout.

      NOTE:

      • This functionality does not apply to currency revaluations, accruals, or mark to market calculations. These will continue to be calculated on a FIFO basis or on the actual individual deal level.
      • You could use Weighted Average Closeout Processing as an alternative to selecting this setting, but if you do and WEIGHTED AVG is selected on the underlying instrument, the deal will not be processed by Weighted Average Closeout Processing.
      • FIFO, LIFO, and MATCHED closeout methods are available for FRN Overnight Index Swaps.
      • See also Weighted Average Closeout formulas.

Closing out deals that use Deal Date accounting

You can close out deals before settlement date if they use Deal Date accounting (that is, if Balance sheet timing in Accounting Rules is set to Deal Date).

If you perform such a close out, the realized gain or loss is calculated by comparing the book values on Settlement Date of the closing deal, and the gain/loss is posted into the General Ledger on the Deal Date of the closing deal.

Matched Closeouts

NOTE: Tradable Yield Instruments (Bond-Rolling) do not support Matched Closeouts.

When you input a Buy deal and a Sell deal based on the same tradeable instrument, closeout can occur. If the Closeout Method for that instrument is MATCHED, then the page menu item Match Deals for Closeout function is available. When you select that menu item, the Match Deals for Closeout dialog box appears, with a grid listing all deals that are available to be closed out together with the current deal when it is saved. The listed deals have defining details in common with the current deal (Instrument, Entity, Currency, Maturity Date, and so on.

The available deals are grouped by their matched or unmatched status in ascending order based on their Deal date, Settlement date, and Deal Number.

Select the deals that you want to close out (along with the current deal) as follows:

  1. In the grid, click a linked value in a deal row (or select the row and in the grid menu select Edit).

    The Edit Match Deal for Closeout dialog box appears.

  2. In the Closeout Face Value field, either accept the default amount to be closed out or enter a lesser amount, and click OK.

    The grid row now shows that Closeout Face Value.

    NOTE: Entered amounts cannot exceed the Remaining Face Value.

  3. Repeat Steps 1 and 2 for other deals, as and if required.
  4. Click OK to close the Match Deals for Closeout dialog box and save the deal.

When you re-display any of the closed out deals, a badge at the top of the page shows either Fully Closed Out or Partially Closed Out with any Remaining Face Value, as applicable in each case.

Indexed Bond Closeouts

When Indexed Bonds are closed out, extra Deal Map components are required on the original deal.

Indexation Validation on Closeout

Quantum checks if the original deal has been indexed beyond the Closing Date. If not, the deal is not allowed to close out. If (or when) the applicable Index values are published, use Update Index to correctly adjust the deal's cash flows as required.

Full Closeout

A "pseudo" control point is inserted in the Deal Map for the closeout date, containing information as for a normal control point. This enables Quantum to calculate the valuation of the Instrument on the closeout date, and provides accounting information for accruing and amortizing. Quantum calculates the difference between the Straight Line accrual and a re-indexation on the closeout date, and then posts a balancing General Ledger entry.

  • A Closeby Cum/Ex n component is added to the original deal map (as for other deals), where n is the Deal number.
  • A CLOSEOUT INDEXATION item is entered into the deal map, with its Effective Date and Action Date set to the closeout date. This item is used to calculate the following three items stamped on the deal map (which also have their Effective Date and Action Date set to the closeout date):
    • INTEREST CALC
    • FACE VALUE INDEX CALC
    • DISCOUNT/PREMIUM CALC

The three items that would otherwise have made up the next control point will be left in the deal map, with their Effective Dates set to the closeout date (their Action Dates will be unchanged).These items will be relabeled as follows.

  • PRE CLOSEOUT INTEREST CALC
  • PRE CLOSEOUT FACE VALUE INDEX CALC
  • PRE CLOSEOUT DISCOUNT/PREMIUM CALC

Partial Closeout

This is similar to full closeout, with a pseudo control point being inserted on the closeout date. This pseudo control point contains information as for a normal control point (based on the original Face Value prior to closeout). See Full Closeout above for details of the components entered as part of the closeout.

However, the deal map is regenerated, so the subsequent control points are based on the remaining Face Value.
