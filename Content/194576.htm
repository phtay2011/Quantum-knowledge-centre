G/L account generation using filters

(First introduced in Release SP 6.7.2, December 2018; see also †Note below.)

When you generate General Ledger Accounts from a G/L Account Template, you can now filter on Currency and any fields specified in the Account Code Format.

Implementation

On the General Ledger Account Template page when you create or edit a template at the Accounting type level, you can select one or more specific currencies (or ALL) and specify an Account Code Format based on up to five field values, which may optionally include a Constant value and Currency field - as in the example below.

TIP: Fields available for selection in the Account Code Format Detail area are those related to Counterparty, Currency, Entity, Instrument, Relationship, and Strategy, including Analysis Codes.

When you select the Generate Accounts* function in the grid menu for such a template, you can now optionally filter on Currency and fields specified in the Account Code Format, as shown below (continuing our example from above).

*TIP: This function is not available if the option Create accounts when posted to for first time is selected on the Account template.

When specifying your filters on the Generate Accounts dialog box:

  • The number of G/L accounts that will be generated (with any filtering applied†) is indicated.

    †NOTE: Prior to Release 6.8.0, the maximum number of accounts was indicated, without regard to any filtering.

  • You can filter on Currency whether or not it is specified in the Account Code Format.
  • If specific currencies are specified in the template, you can filter on those currencies only.
  • Other filters default to ALL and you can optionally filter on any available values, for example, you might select four Entity Code values, as shown below.

When you click OK, G/L Accounts are generated accordingly. Select the template and select View GL Accounts in the grid menu to see them - as below for the currencies and entities specified in the example above (note also the Prefix, Constant, and Suffix values included in the codes).

The generated accounts initially inherit properties from the template. To view and possibly edit these properties for a listed account, click on a value in that account row to open the Edit General Ledger Account dialog box, as shown below for the first account listed in the example above.

On this dialog box you can optionally change the properties for the G/L account, or delete it, EXCEPT THAT:

  • you cannot change the Code itself, nor the Currency;
  • you cannot enable the Apply account level currency translation option if the account currency is the Base Currency for the Ledger (as defined on the General Ledger page); and
  • you cannot delete a G/L Account Code if it has already been posted to.

History

Previously, no filtering was possible when you generated G/L Accounts - all accounts for the currencies specified in the template were generated.
