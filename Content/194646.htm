Repo Deal Enhancements

Allow Dirty Pricing

(First introduced in Release SP 6.7.1, August 2018.)

You can now book a Repo deal with dirty pricing for Classic, Reverse, Fee Based Borrowing, and Fee Based Lending Repo formats.

Implementation

Security and Money Market instruments

On the Pricing tab under Repo Pricing, a new Price Type field is available for the BOND PERIODIC, INDEXED BOND, and INDEXED GILT formulas.

Select whether the Repo deals using this instrument in their security leg express the price as a price per hundred (PPH) on either the Capital value (the Clean price), or on the Settlement value (the Dirty price).

Repo deals

In the security leg of Classic, Reverse, Fee Based Borrowing, and Fee Based Lending Repos, you can select a Security or Money Market instrument with a Repo Price Type setting.

Exclusions

Dirty pricing is not currently supported for Buy/Sell Back, Sell/Buy Back, Security Borrowing, and Security Lending Repo formats.

Derived value in Security leg

(New from Release 6.8.0)

You can now specify which value will be the derived one in a Repo deal security leg, by using a new setting on the underlying instrument.

Implementation

On the Security (or Money Market) Instrument page's Pricing tab, when Formula Timing is BOND PERIODIC, INDEXED BOND PERIODIC, or INDEXED GILT PERIODIC, a Derived field now appears in the Repo Pricing area.

For the INDEXED BOND formula when Coupon Type is EXACT, this field is read-only and set to:

  • CAPITAL if Price Type is SETTLEMENT, or
  • SETTLEMENT if Price Type is CAPITAL.

In all other cases, you can select:

  • CAPITAL or INCOME if Price Type is SETTLEMENT, or
  • SETTLEMENT or INCOME if Price Type is CAPITAL.

This setting determines which value will be the derived one in Repo deal security legs based on the specified instrument.

History

Previously, the derived price value in a Repo deal's security leg was always as specified for the Security itself. For example, you could not derive the Repo price from the Capital proce if the Security deal was derived from the Settlement price.

Indexed Bond Security leg enhancements

(First introduced in Release SP 6.7.1, August 2018.)

Booking beyond Last Index Date

You can now book and modify Classic or Reverse Repo deals beyond the Last Index Date, where the Collateral leg is based on a Security instrument with Formula Timing of INDEXED BOND PERIODIC. This enables the deal to be saved using the last available Retail Price Index (RPI) value in the applied Index Rate series.

Coupon amounts and coupon interest will not be indexed-updated, as these are handled in the Collateral pool.

Indexed price (clean or dirty)

You can now enter an indexed Capital or Settlement Price for Classic, Reverse, Fee Based Borrowing, or Fee Based Lending Repo deals, where the Collateral leg is based on a Security instrument with:

  • Formula Timing (Details tab) set to INDEXED BOND PERIODIC,
  • Repo Price Type (Pricing tab) set to eitherCAPITAL or SETTLEMENT, and
  • the Repo Price Indexed option selected.

When you select such a Security instrument in a Repo deal leg*, the Deal Details tab shows an Indexed Capital Price or Indexed Settlement Price field for that leg, depending on the instrument's Repo Price Type setting (instead of the Capital Price or Settlement Price field when the Repo Price Indexed option is cleared on the Security instrument).

*TIP: Collateral leg for Classic or Reverse Repo, Benefit leg for Fee Based Borrowing or Lending Repo.

Fee-based repo - indexed bond in Benefit leg

In a Fee Based Borrowing or Lending Repo deal's Benefit panel, you can now select a Security instrument with Formula Timing (Details tab) set to INDEXED BOND PERIODIC.

(Selection of such an instrument in the Collateral leg is currently supported for Classic and Reverse Repos).

lndexed Gilt Security leg enhancements

(First introduced in Release SP 6.7.1, August 2018.)

New Yield Type supported

Repo deal legs specifying Security instruments with Formula Timing of INDEXED GILT PERIODIC now support the new Yield Type of UK SEMI ANNUAL, for the following Repo formats only:

  • Classic,
  • Reverse,
  • Fee Based Borrowing, and
  • Fee Based Lending.

The UK Government's Semi-Annual Yield formulas will be applied for such Repo deal legs.

For more details, see Indexed Gilt Securities - New Yield Type.

Extend Maturity Date

You can now extend the Maturity Date of some Repo deals to any future date before the Maturity Date of the collateral, where the Collateral leg is based on a Security instrument with Formula Timing of INDEXED GILT PERIODIC and the Repo Format is one of the following:

  • Classic,
  • Reverse,
  • Fee Based Borrowing, or
  • Fee Based Lending.

Restrictions and limitations

In the applied Index Rate series, a Retail Price Index (RPI) value must exist for the Repo Settlement Date. (The latest RPI will be used for Leg 2 pricing.)

NOTE: The Repo Maturity Date cannot be greater than Collateral leg's Maturity Date.

Settlement Date up to Effective Date of latest RPI

You can now save a Repo deal with a Collateral leg based on a Security instrument with Formula Timing of INDEXED GILT PERIODIC (and an eight-month lag) for any Settlement Date up to the Effective Date of the latest Retail Price Index (RPI) record in the applied Index Rate series.

History

Previously, you could not save such a Repo deal if the Settlement Date was NOT before the Period Start Date of the latest RPI record in the applied Index Rate series.
