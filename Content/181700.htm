Intraday Matching Enhancements

(First introduced in Release 6.2.1 SP, May 2016.)

Intraday Matching functionality has been extended to include most of the same features of Prior Day Matching.

From this release, a separate Intraday Match Script page no longer exists, and the Detail Match Script page is now used to create and maintain scripts for either Prior Day or Intraday Matching, or both.

Intraday Matching using a script can now therefore:

  • automatically generate adjustments*,
  • produce Probable and Possible match statuses,
  • be part of a script chain, and
  • optionally print an interim reconciliation report.

NOTE: The automatic generation of adjustments in Intraday Matching will be a major new feature for some users. For example, an Intraday Matching run can automatically create Actual Cash Flows for unmatched bankflows that are specially designated, so that these will then reconcile in the next day's Prior Day Matching run.

You can also apply the same rules to manual matching of Intraday bank flows and cash flows as for Prior Day flows - for example, to:

  • prevent matching of Intraday flows where the balance is not zero,
  • produce Probable and Possible match statuses,
  • force a match for a set of either cash flows or bank flows, provided they sum to zero.

    TIP: The flows must be selected in the respective grid on the Intraday Matching page after you have run a query. The Force as Matched menu item is available only when your current selection sums to zero. Any restrictions on matching over different Value Dates are ignored (that is, as specified by the Allow Flows to match operational setting and any Script Pass criteria for Value Date).

Exclusions and Restrictions

The separate Intraday Match Script page is no longer required and has been removed.

When a Detail level script is run for Intraday Matching:

  • the Rollback Reconciliation Mark option is ignored (Reconciliation Marking applies only to Prior Day Matching).

When a Script Chain is run for Intraday Matching:

  • any Balance script in the chain is ignored, and
  • any Mark As Reconciled step in the chain is also ignored.
