Statement Balance Code Parameters

Statement balance amounts in each statement record are identified by BAI summary codes that are defined in the BAI standard. By default, Bank Importer extracts balance amounts from these fields using the BAI summary codes specified in the BAI standard and listed in the following table. (You can also enable further balances by adding other (non-defaulted) parameters to the Configuration file - see Enabling Further Balances below.)

Balance Code Parameter

Default Summary Code

OpeningBalanceFieldCode

010

OpeningBalanceFieldCodeFallBack

(blank)*

ClosingBalanceFieldCode

015

ClosingBalanceFieldCodeFallBack

045

TotalCreditsFieldCode

100

TotalDebitsFieldCode

400

AvailableBalanceFieldCode

045

AvailableBalanceFieldCodeFallBack

(blank)*

CollectedBalanceFieldCode

089

CollectedBalanceFieldCodeFallback

(blank)*

OtherBalanceFieldCode

090

OtherBalanceFieldCodeFallback

(blank)*

*NOTE: The Fallback parameters specify a code to use to extract a balance if the first choice of code is not found. There is no default value specified for fallback parameters themselves : if the primary code is not found, no attempt is made to extract a fallback balance.

The default summary code used to extract each balance can be overridden by assigning a new summary code to the corresponding parameter name using the syntax:

balance parametercode = bai summary code

NOTE: If a balance parameter code is set to blank, then no attempt is made to extract the fallback balance.

Example

OpeningBalanceFieldCode = 060

Enabling Further Balances

You can also upload Current Available and/or Bank Value Date balances by explicitly adding the following parameters to the Configuration file as required (these balances are not defaults).

  • CurrentAvailableBalanceFieldCode
  • CurrentAvailableBalanceFieldCodeFallBack*
  • BankValueDateBalanceFieldCode
  • BankValueDateBalanceFieldCodeFallBack*

*TIP: See the Note about Fallback parameters above.

Example

CurrentAvailableBalanceFieldCode = 045
