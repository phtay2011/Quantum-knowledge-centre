Entities and Inter-Company Bank Account Defaults

In Inter-Company Bank Account Defaults, the Entities that you select as G/L Owner A and G/L Owner B identify which General Ledgers will exchange postings. Both of these Entities must be G/L Owners.

TIP: By default, the lookups in these fields list only those Entities that are G/L Owners, but if you select the Display all Entities option, all Entities are listed.

Transactions between Posting Entities

Always link posting (transaction-level) Entities at the G/L Owner level.

If a posting Entity is not a G/L Owner, it can be associated with one through Entity grouping in your Business Structure. In this case, you should name the G/L Owner, not the Entity itself, in the iNter-Company Bank Account Defaulting Rule. Quantum will then use the General Ledger of the associated G/L Owner for these Inter-Company postings.

Example

Entity 1 may be a transaction-level Entity and a G/L Owner, while Entity 2 is a transaction-level Entity but not a G/L Owner.

You cannot create an Inter-Company Bank Account Defaulting Rule between them directly. However, if Entity 2 is grouped under the Entity 5 group that is a a G/L Owner, you can create a rule record between Entity 1 and Entity 5, which will also control G/L postings for Entity 2 (and any other group members). Transactions between Non-Posting Entities

A non-posting Entity does not have a General Ledger, so there is no need to set up an Inter-Company Bank Account Defaulting Rule between two non-posting Entities.

Transactions between Non-Posting and Posting Entities

The creation of Inter-Company Bank Account Defaulting Rule records for Inter-Company cash flows in such cases is not supported.
