About Statement Runs

NOTE: While Quantum statement runs can help you to reconcile with the bank's (or Custodian's) actual statement detail, they are based solely on Quantum's record of account transactions.

For details of full Bank Reconciliation facilities in Quantum, including importation of actual bank statement details, matching, and online adjustment facilities, see Bank Reconciliation.

You can run statements at different reporting frequencies for different accounts - for example, daily statements, weekly statements, monthly statements, and so on. Create an account group for each reporting frequency, and set up Query parameters for each group.

For new statement runs - to ensure sequential operation:

  • if the Future dated cash flows option is cleared, only transactions after each account's latest Statement Date and up to and including the Query End Date are considered; or
  • if the Future dated cash flows option is selected, future-dated cash flows for an included account and specified Dealing instruments will be included, UNLESS you also select Exclude future dated deals, when cash flows for any deal with a Deal Date later than the query End Date will NOT be included; and
  • an Update run will increment the Last Statement Number* and update the latest Statement Date, and Last Statement Number for each account, while a Preview run will not.

*TIP The Account Statement Number length is nine digits, and after 999999999 it is automatically restarted at 1).

For statement re-runs:

  • only transactions already included in each account's latest statement OR added since are considered,
  • you cannot change the Query End Date,
  • an Update run will increment each included account's Last Statement Number and Last Statement Run Number, while a Preview run will not, and
  • neither type of re-run will update each included account's Last Statement Date.

Backdated deal additions, changes, and deletions between the run date and the cash flow date will be indicated as such in the run results.

NOTES:

  • Running Bank Account Statements does not calculate nor post interest. To include interest postings on your statements, you must first run Bank Interest Statements (for the relevant bank accounts) with the Post Interest option selected.
  • The Select Query or Run browse window includes Run Number and End Date and can be used as a method to repeat an earlier run.
  • Ensure your Expiry Strategy settings (Properties tab, Configure Query) reflect the appropriate time statements are to be kept.
  • An account is not included in your report output if its latest Statement Date is after the Query End Date (you will receive a warning message when this occurs).
  • Use the Include Statements for options to include or exclude:
    • Statements with no transactions in the statement period,
    • Statements with zero balance AND no transactions in the statement period, and/or
    • future-dated cash flows for one or more specified Instruments (with the additional option to include or exclude cash flows for deals with a Deal Date later than the query End Date).
  • Projected cash flows are not considered.

Rolling back the last statement Update run

You can use the Set Statement Details utility's Delete Last Run function to remove changes to Account data made by the last statement update run.

NOTE:

  • The Delete Last Run page menu function is NOT available if no updating statement run has yet occurred for any accounts.
  • Only the latest Account Statement Updating run will have changed the database (Previewing runs and query runs started from the Reporting page cannot).
  • That last statement run may have covered (and updated) EITHER one or more Bank Accounts OR Stock Accounts (but not both types). Its result set will still be available, but will no longer show the statement-related database values it changed.

You can also use the Set Statement Details utility to:

  • set an initial Start Date for the first statement for selected bank or stock accounts that have no previous statement runs, or
  • set the next statement number for selected accounts - for example, to restart from 1 at the beginning of each year to comply with SWIFT Usage Rules.
