Notes and Attachments

Document attachments are stored in, and retrieved from, a central document repository (which can be secured). This document repository can be configured to be on any server in the Quantum Six environment.

Because the documents are stored in a dedicated document repository, access to a document that is uploaded via Quantum is independent of any access permissions for locations in the file system, and Quantum Module Security is used to ensure that only valid users can access appropriate documents.

The Notes module in Module Security controls the availability and access to the functionality for notes and attachments with the standard module security permissions (that is, Inquire, Add, Change, and Delete).

NOTES:

  • At the very least, users must have the Inquire permission in order to view the available notes; they will however, have access to any of the editing or deleting functions.
  • Users with the Add permission can add new notes with attachments and add new attachments to existing notes; they will however, not be able to change the existing text of the note or remove its attachments (the additional Change and Delete permissions will be required to enable this functionality in the Note Editor).

File attachment functionality is available on every page that has Notes in the page menu. This applies to all Dealing pages, and to some Cash Management, Accounting, and static data pages (in the Default perspective only), as listed below.

Applicable Cash Management pages are:

  • Account Adjustment
  • Actual Cash Flow
  • Cash Transfer
  • Interest Adjustment
  • Projected Cash Flow
  • Stock Transfer

Applicable Accounting pages are:

  • Hedge Relationship

Applicable static data pages are:

  • Broker Account
  • Cash Account
  • Counterparty
  • Entity
  • Stock Account
  • User and User Group (accessed via the Configurationsite menu)

When any notes or attachments have been added to a record, the Notes badge appears at the top right of the page.

Permitted File Extensions

The default list of permitted file extensions is: dat, doc, docx, xls, xlsm, xlsx, ppt, pptx, pdf, png, jpg, qxt, qid, txt, xml, jrg, csv, sft, out, dat.

System administrators can add or remove file extensions from this list by editing the UploadWhiteList key in the Web.AppSettings.config file. However, there is a hard-coded blacklist of file extensions that can never be uploaded for security reasons - for example, exe and bat.

Location and Security of Attached Files

All attached files are uploaded into a single folder called Attachments under your site data folder. For security reasons, the files are renamed with unique untraceable file names while they are stored in this location.

When you download any of these files, the original file names are restored.

How to use

TIP: A record here refers to an item accessed via any page with Notes functions (as listed above).

To add a new Note to the current record

  1. Do one of the following:
    • In the page menu under NOTE, select New, OR
    • If the Notes badge is already showing, click on it to open the Notes dialog box and click New Note.
  2. In the New Note dialog box, enter your note text as required.
  3. (Optional) If you want to attach files to this note, click in the New Files to Attach field. In the file selection window, navigate to and select one or more files and click Open, then click OK. For more details, see To attach files to a note below.
  4. Click OK.

To attach files to the current record (at record level)

  1. In the page menu under NOTE, select Attach File.

    TIP: You can also attach files via the New Note function or the Notes badge.

  2. In the Attach File dialog box, click in the New Files to Attach field. In the file selection window, navigate to and select one or more files and click Open, then click OK.

    NOTES:

    • If a file with the same name has been attached before at record level, the previous file will be replaced.
    • Files with the same name may be attached to different notes - one per note, and/or one at the record level.
    • If you select a file with an extension that is not allowed, an Invalid file type message is returned. See Permitted File Extensions, above.

To view existing notes and attachments

  • Click the Notes badge, OR
  • in the page menu under NOTE, select View.

The Notes window lists attachments at the record level, followed by individual notes (latest first) with any attachments at the note level, as shown in the following example.

To view notes attached to Static Data on dealing pages

When you add a note to static data that is associated with deals (that is, Counterparty, Entity, Facility, Bank Account, and so on), you can select Show also in deal notes in the New Note dialog box, to make the note available for viewing on the dealing page. By default, this option is unchecked.

On a dealing page, when a note is available for an associated static data record, the Notes badge is displayed on the dealing page, and the Notes dialog box displays the notes for the deal itself, followed by the notes for the static data in the following order:

  • Counterparty
  • Entity
  • Facility
  • Bank Account

To edit an existing note and/or its attached files

  1. Open the Notes dialog box as above (by clicking the Notes badge OR in the page menu under NOTE, selecting View).
  2. In the Notes dialog box, click the edit link for the note you want to change.
  3. In the Edit Note dialog box, change the Note text if required.
  4. (Optional) If you want to attach more files to this note, click in the New Files to Attach field. In the file selection window, navigate to and select one or more files and click Open, then click OK.

    NOTES:

    • If a file with the same name has been attached to this note before, the previous file will be replaced.
    • Files with the same name may be attached to different notes - one per note, and/or one at the record level.
    • If you select a file with an extension that is not allowed, an Invalid file type message is returned. See Permitted File Extensions, above.
  5. (Optional) If you want to delete an attached file from this note, move the mouse pointer to the file name button in the Attachments field and click the x on that button.
  6. Click OK.
  7. Click Close.

To delete a note and its attached files (if any)

  1. Open the Notes dialog box as above (by clicking the Notes badge OR in the page menu under NOTE, selecting View).
  2. In the Notes dialog box, click the edit link for the note you want to delete.
  3. In the Edit Note dialog box, click Delete.
  4. Click Close.

To delete a file attached at the record level

  1. Open the Notes dialog box as above (by clicking the Notes badge OR in the page menu under NOTE, selecting View).
  2. In the Notes dialog box, click the edit link for the Attachments list (at the top).
  3. In the Edit Attachments dialog box, move the mouse pointer to the file name button in the Attachments field and click the x on that button.

    TIP: To delete all files attached at the record level, click Delete All.

  4. Click OK.
  5. Click Close.

Restrictions and Exclusions

Notes and Attachments are not supported for Analytics pages.
