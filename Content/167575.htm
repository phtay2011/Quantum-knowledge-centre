Futures and Options Closeouts

When you save a Futures or Exchange Traded Option (ETO) deal, the system attempts to close it out against one or more existing deals. It checks that certain criteria are met, then applies the closeout method specified on the Futures Control page.

Futures and Options closeout criteria

For Futures Exchange contracts to be closed out against each other, they must match in terms of:

  • Contract type (both must be OPTIONS or FUTURES)
  • Contract
  • Exchange
  • Entity
  • Delivery month
  • Strike price (ETO only)
  • Taxes

Brokers as closeout criteria

To include Executing and Clearing brokers as closeout criteria, make sure that the following Closeouts options are selected on the Futures Contract Details tab in Futures Control:

  • Include Executing Broker
  • Include Clearing Broker
  • Include strategy - if you select this option, the Strategy field will be treated as a financial change field. This means that if the strategy value of an existing deal is changed, the existing accounting (Accounting Rule, Posting Rule, and Account Templates) for that deal will be re-applied, and new accounting will be generated.

If you select this option, and if you set the Closeout criteria to:

1. LIFO or FIFO - strategy will be included when selecting deal/s to closeout.

2. MATCHED - strategy will be included when filtering available deals to present within the Matched closeout listing.

Closeout methods

To determine how the selection of deals for closeout should take place, choose a Closeout Method on the Futures Contract Details tab in Futures Control.

If you select …

Then …

LIFO or FIFO

The system selects matching deals automatically.

MATCHED

You select matching deals manually when saving a deal.

Changing a closeout method

If you change the closeout method for a contract type, open positions can be closed out using the new method. Closed positions are not affected by the changes.

Changing closeout matches

Once closeout matches are in place, you can change them by recalling the deal and:

  • editing only the matches, or
  • changing the deal significantly and then re-saving it - the system changes the closeout selection according to the selected closeout method.

Editing matches

You can recall a deal and manually change the closeout matches made against it if either of the following conditions are met:

  • the closeout method for the contract type is set to MATCHED in Futures Control (and the Modifiable Matches option is selected), or
  • the closeout method is FIFO or LIFO and the Modifiable Matches option is selected.

To make changes, select Modify Matches from the site menu.

Changing a deal

Closeout matches also change if the deal itself is significantly changed, where:

  • the closeout method is MATCHED - you select new matches manually when the deal is saved, or
  • the closeout method is FIFO or LIFO and the deal is re-saved - the new matches are again automatic (and therefore transparent). You can modify them afterward if the Modifiable Matches option is selected.

The deal number does not change when you re-save a deal.

For step-by-step information on changing closeout matches see Matching closeouts.

Deals available for closeout

The system selects closeouts from all open deals. A deal becomes available for matching again if an earlier closeout is negated. This can occur if the other deal is deleted, for example. Valid existing closeouts are not affected by the changes.

Hypothetical deals and closeouts

Quantum supports closeouts for Hypothetical Futures and Exchange Traded Options (ETO). However, these must be mirrored in the equivalent respective Live deals.

Reporting

When you run a Futures Margin query, the system again displays details of the contract, remaining lots, and the nominal value of the remaining lots. It revalues the Futures deals and, if instructed to do so, posts the appropriate General Ledger entries.

Running a Position-Query page revalues the deals, but does not post G/L entries.

Matching closeouts

The closeout method that applies for a Futures deal is determined by a setting on the Futures Control page.

When manual matching is possible, the Closeouts tab displays all deals that are available to be closed out. These have defining details in common - for example, Contract Type, Entity, and so on (see the list of closeout criteria under Futures and Options closeouts).

NOTE: You can select a deal that fully or partially closes out a current deal, or save a deal without accepting any of the closeouts matches offered.

The top of the Closeouts tab contains header information about the deal being saved. The lower part lists all open transactions against which the deal being saved can be closed out.

Defaults

When you select a row, the deal with the earlier deal date is regarded as the opening deal and is marked as such in the Open/Cls column. If both deals have the same deal date, the opening deal is, by default, the deal with the lower deal number. You can modify this.

The date of the closing deal is entered as the default Match Date.

Closeout Lots shows the number of lots being closed out by a selected deal. This defaults to the total number of lots eligible for the current match. This is the total.

The system automatically updates the Number of Lots Remaining (not yet closed out) and the Gain/Loss on the default number of closed lots. When you actively change the number of lots to be closed out, the system recalculates how many lots remain for each deal, and the gain or loss on each.

Previewing possible matches

To see the outcome of possible matches, select and deselect deals without saving.

To select a deal for closeout

  1. In the list of available deals, select the deal you want to close out.

    The Closeout Matches dialog box displays the default values (see above) and other contract information.

  2. in the dialog box, enter the number of lots that are to be closed out against the current deal.

    The defaulted values change accordingly. If the deal dates are the same for the matched deals, you can modify the setting in the Open/Cls field.

  3. Click OK.

To manually change closeout matches when a deal is changed

This procedure applies when the closeout method is MATCHED. For other closeout methods, any rematching is automatic and therefore transparent.

  1. Browse and select the deal.
  2. Make the required changes and Save.

    If the changes affect closeouts, a message appears warning you that the changes will invalidate existing closeouts.

  3. As you intend to change the closeouts, click OK to continue.

    (Otherwise you could cancel and return to the deal page.)

    NOTE: Existing closeouts are fully unwound, and the Closeout Matches dialog box displays possible matches. Previous matches are also shown, but they are no longer selected.

  4. Select new matches and, if necessary, change the number of lots to be closed out by each.
  5. Complete the deal save. The deal number does not change.

    NOTE: A deal becomes available for matching again if an earlier closeout is negated - by the deletion of the other deal, for example. Valid existing closeouts are not affected by this change. The system does not recalculate historically to take account of a deal which becomes available. That deal is considered only for subsequent closeouts.

To manually change closeout matches without changing other deal details

This procedure applies for any closeout method when the Modifiable Matches option is selected. This rematching is done manually.

  1. Browse and select the deal.
  2. In the site menu, select Modify Matches.

    The Closeout Matches dialog box displays existing matches, but they are no longer selected.

  3. Select which deals are to be closed out.
  4. Save the deal. The deal number does not change.
