Creating, Changing, and Deleting (Unfilled) Requests

To enter and maintain FX requests, open the FX Request or Money Market Request page and follow the general procedures below, as required.

Creating, making financial changes to, or deleting a Request can be subject to Approvals.

You can create a new Request in two ways:

  • from scratch, using the New button or menu function, or
  • by copying an existing one, using the Copy menu function.

In both cases:

  • you need Add rights to the relevant Request module,
  • you must enter at least the mandatory field values - see Quantum Request Pages,
  • you can optionally add a Note or attach a file as a note,
  • when you click Save, the Request - if validated successfully - is stored in the database, and
  • if an FX Request and the Replication process is active, the Request will be replicated into Analytics as an FX Deal with a Deal Type of Request).

To change a request's details:

  • you need Change rights to the relevant Request module,
  • you must retain the mandatory field values - see Quantum Request Pages,
  • you can optionally add a Note or attach a file as a note,
  • when you click Save, the Request - if validated successfully - is updated in the database, and
  • if an FX Request and the Replication process is active, the Request FX Deal will also be updated in Analytics.

To delete an unfilled request:

  • you need Delete rights to the relevant Request module,
  • you can optionally enter Reason text,
  • when you confirm the deletion, the Request is marked as deleted (both on the page and in the database), and
  • if an FX Request and the Replication process is active, the Request FX Deal will also be deleted in Analytics.
