Enterprise Reporting

NOTE: In v6.8.0, Enterprise Reporting was renamed the Enterprise Workflow Framework.

Your system administrator or FIS Consultant can now create and maintain site-specific reporting pages, which can combine data from multiple Quantum Six* data sources, and which can be placed anywhere in the Quantum menu structure. To the end user, these reporting pages will look and behave like any other core Quantum reporting page.

* It is planned to add external data sources in a near-future release.

Currently supported data sources are:

  • other Quantum reports,
  • Analytics reports,
  • Quantum deals,
  • static data (Quantum and Analytics),
  • Prior Day and Intraday bank balances from CashXplorer, and/or
  • Analytics custom tables.

Implementation

The new Custom Report page has been added to the Configuration menu (in the Default perspective). Use that page to create and maintain your enterprise reporting pages.

TIP: The Custom Report page should not be confused with the Customization page, which can be used only to customize the user interface of core system pages.

An example of the Custom Report page is shown below.

Use the Name field to enter a unique module name for the target page (used in Module Security) and the Title field for the page title. The Url Name gives the last part of the page URL.

Use the Menu Area and Menu Group fields to enter the area in the site menu where the target page will appear (Analytics, Processing, Reporting, and so on) and the menu group under which it will appear (also used in Module Security).

The Data Sources grid will have one or more entry rows, each with the following values:

  • Source - the data source used for the report;
  • Name - a name for the source (defaults to the Source value) - useful to shorten the default name or when the same data source is included twice for comparison;
  • Settings - settings used to reduce the target report's result set, either fixed values or linked to query filters on the target page;
  • Columns - selected columns from the data source to be available in your target report's result set; and
  • Joined To (optional when two or more sources exist) - mapping of fields to another data source so that both values will appear in your target report results when the joined items match.

The Filters grid shows filters to be displayed as Filters fields on the target reporting page. The filters are used by the data source settings to reduce the result set. A filter provides the end user with the ability to control the report result set.

The Actions grid shows optional additional actions made available in your target report's results view (as quick menu items or toolbar buttons). Defining actions for a report changes the reporting page into a processing page.

To see a new target page in the site menu, you must refresh the page (press F5) or log out and in again.

Examples

Some possible usages of enterprise reporting capabilities include:

  • Including Analytics valuation data in Quantum deal reports
  • Including market data in existing reports
  • Adding limit checks onto existing Quantum report data
  • Listing General Ledger entries for a Counterparty's live deals
  • Loan covenant monitoring
  • Blotter functionality (for example, FX deals/requests combined with current cross rates)
  • Failed payments by bank
  • Comparing start of month with end of month positions
  • Offsetting Actual Cash Flow deals with Projected Cash Flow deals

See an example of an Analytics report joined to Quantum static data.

Module Security

For accessing the Custom Report page itself, Custom Report module permissions apply.

For accessing an enterprise reporting page created from the Custom Report page:

  • for your User name and the current target page if you are not exempt (all permissions) - select Add to Current User in the page menu under MODULE SECURITY (you can remove that permission in the same way); or
  • for any user (selected permissions) - use the Module Security tab on the Users or Groups page as usual. The page will be labeled using the specified Menu Group and Name values.

History/Background

Previously, the data queried by each Quantum Six reporting application was fixed. While you could customize the user interface and filter the queried data and report results, you could not specify any other data sources.
