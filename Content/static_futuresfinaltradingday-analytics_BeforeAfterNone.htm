2nd Rule Applied (Analytics Futures Final Trading Days)

Indicates whether or not a second rule is to be applied; one of the following:

  • None - to allow the primary rule to remain unaffected,
  • Before* - to apply the second rule to find a date before the first rule set date,
  • After* - to apply the second rule to find a date after the first rule set date, or
  • Exception† - to apply the second rule to specify a single date in the year.

* The following fields display when 2nd Rule Applied is Before or After:

  • 2nd Day - the secondary rule day type: a day of the week, Actual, Actual (Preceding Month), Business Day, or End of Month;
  • 2nd Day No - used in conjunction with the 2nd Day value to specify the additional rule timing, when applicable. If 2nd Day is set to:
    • Monday through Sunday - set 2nd Day No to a week number in the month (1 through 4),
    • Actual - set 2nd Day No to a day of the month,
    • Actual (Preceding Month) - set 2nd Day No to a day of the preceding month,
    • Business Day - set 2nd Day No to a number of business days with which to offset the calculated delivery date, or
    • End of Month - set 2nd Day No to zero (type the numeral 0; do not delete the contents of the field); and
  • 2nd Business Day - the rule to determine the day to be used instead when the secondary rule results in a non-business day: Actual, Following, Modified Following, or Preceding.

† The following fields display when 2nd Rule Applied is Exception:

  • Month - used in conjunction with the adjacent Day setting to specify a single date in the year for the additional rule timing, when applicable;
  • Day - used in conjunction with the adjacent Month setting to specify a single date in the year for the additional rule timing; and
  • 2nd Business Day - the rule to determine the day to be used instead when the secondary rule results in a non-business day: Actual, Following, Modified Following, or Preceding.

    TIP: A common example is 24 December with the Preceding Business Day rule.
