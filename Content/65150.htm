Replicating Portfolios for At-Expiry Single Barrier Options

This topic shows the combinations of digital options and Vanilla caps and floors that replicate the payoff structure for the different types of single barrier option. The sum of the values in each combination can therefore can be used to value the barrier option at any time prior to maturity.

In some cases a restriction applies between the option's Barrier value (B) and Exercise price (X) to either ensure that a payoff is feasible or to ensure that the barrier option is distinct from an equivalent Vanilla cap or floor.

TIP:RT = Reference Rate at maturity.

Down & In Cap

The Replicating Portfolio consists of:

  1. A Put Digital option with:
    • Position = Long
    • Strike = B
    • Payoff = max[B-X,0], where B > X
  2. A Vanilla Floor option with:
    • Position = Short (will sell)
    • Strike = B
    • Payoff = - (max[B-RT,0]), where flow is negative for a sale
  3. A Vanilla Floor option with:
    • Position = Long
    • Strike = X
    • Payoff = max[X-RT,0]

Up & In Cap

The Replicating Portfolio consists of:

  1. A Vanilla Cap option with:
    • Position = Long
    • Strike = B
    • Payoff = max[RT-B,0]
  2. A Call Digital option with:
    • Position = Long
    • Strike = B
    • Payoff = max[B-X,0], where B > X

Proof for Down & In Cap

The following diagram represents the payoff (CF) for a Down & In Cap option, where the shaded areas indicate a zero payoff.

The payoff from the replicating portfolio exactly matches the payoff for the Down & In Cap for all potential values of the reference rate at maturity RT (corresponding to the three zones in the diagram), as shown below.

  • When RT > B, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike B): Payoff = max[B-X,0] = 0
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = 0
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = 0
  • When B > RT > X, Payoff = RT - X.

    The sum of the replicating portfolio elements is the same:

    • Put Digital option (Strike B): Payoff = max[B-X,0] = B-X
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = -(B-RT)
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = 0
    • Sum: (B-X) - (B-RT) + 0 = RT - X
  • When RT < X, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike B): Payoff = max[B-X,0] = B-X
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = -(B-RT)
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = X-RT
    • Sum: (B-X) - (B-RT) + X-RT = 0

A simple no-arbitrage argument can therefore be invoked to ensure that the current values of the Down & In Cap and the replicating portfolio must also be equal.

Proof for Down & In Floor

The following diagram represents the payoff (CF) for a Down & In Floor option, where the shaded areas indicate a zero payoff.

The floorlet is only in-the-money at expiration if the reference rate RT fixes at a level lower than the pre-specified barrier level B, in which case the payoff structure is the same as a Vanilla floorlet (max[X-RT,0]). A logical restriction on this type of instrument requires that the pre-specified barrier rate must be lower than the strike price X. Barrier floors with the barrier set above the strike rate are effectively the same as Vanilla floors.

The payoff from the replicating portfolio exactly matches the payoff for the Down & In Floor for all potential values of the reference rate at maturity (corresponding to the three zones in the diagram), as shown below.

  • When RT > X, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike B): Payoff = max[X-B,0] = 0
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = 0
  • When X > RT > B, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike B): Payoff = max[X-B,0] = 0
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = 0
  • When RT < B, Payoff = X-RT.

    The sum of the replicating portfolio elements is the same:

    • Put Digital option (Strike B): Payoff = max[X-B,0] = X-B
    • Vanilla Floor option (Strike B): Payoff = -(max[B-RT,0]) = (B-RT)
    • Sum: (X-B) + (B-RT) + 0 = X-RT
