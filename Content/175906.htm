Dual Curve Bootstrapping - Basis Swaps

The Dual Curve Bootstrapping option enables you to define and utilize two distinct curves when bootstrapping the forward curve, allowing separate projecting and discounting rate and margin curves to be applied in Swap valuations. These changes provide for more accurate and internally consistent valuations when different curves are being used.

This is because, when discounting using a separate curve, the projecting curve becomes an issue. It is no longer possible to derive accurate forward rates from the Swap Par curve, as that process relies on the fixed side of the swap for each maturity initiating at par, and the floating side resetting to par at each reset date.

Analytics utilizes discount factors from the discounting curve along with the fixed Swap Par rates from a separate projecting curve, to generate an implied forward curve that is consistent with the discount curve discount factors. This curve may then be used to project future interest in Swap valuations where separate curves are in place.

The Dual Curve Bootstrapping approach adopted by Analytics applies a method to build the forward curve by equating the fixed and floating sides of a hypothetical par swap, where both sides are discounted using the discount curve. This is an iterative process to solve for a forward rate that achieves parity by adjusting the zero rates/discount factors at each point along the curve.

To set up Dual Curve Bootstrapping for the Projecting curve

  1. In the Analytics perspective's Rates menu, select Zero Curve.

    The Zero Curve Rate Definition page opens.

  2. Click the Browse button and select the Zero Curve you want to apply Dual Curve Bootstrapping to.
  3. In the Settings area:
    1. (Recommended) In the Bootstrap Frequency drop-down list, select Natural.
    2. Select Apply dual curve bootstrapping.
  4. In the Dual Curve Settings area:
    1. In the Frequency drop-down list, select the frequency that represents the tenor of the floating index: Monthly, Quarterly, Semi-Annual, or Annual.

      Within the bootstrapping calculation, this frequency will be used to find forward rates on the floating side of the swap. As with single-curve methodologies, the Bootstrap Frequency setting in the Zero Curve Definition will be used to set the frequency on the fixed side of the swap. Where this is set to Natural, the Frequency column on the Swap rate record will be used.

    2. In the Day Count drop-down list, select the floating index day count convention to be applied on the floating leg.
    3. In the Discount Curve drop-down list, select the Discount Curve (Rate Name) to be used within the dual bootstrapping calculation.

      This lookup lists other Zero Curve definitions, and while it is expected this will be set to the curve used for discounting Swap cash flows, there is no system restriction in place. Only curves using traditional single-curve models will be available for selection (those with Apply dual curve bootstrappingcleared).

  5. Click Save.
  6. Repeat Steps 2 though 5 for any other Zero Curve Definitions requiring Dual Curve Bootstrapping.

Valuation Control

Zero Curve Definitions with the Apply dual curve bootstrapping option selected will be available in all Zero Curve lookups, for use in any valuation that utilizes a Zero Curve, for any transaction type.

To implement Dual Curve Bootstrapping for Swap valuations

  1. From the Analytics site menu, select Static & Rules, then Valuation.
  2. Select a record from the list using the browse button.
  3. Select the Interest Rate Swap or Cross Currency Swap tab.
  4. In the Rules grid, click any cell value link in an applicable row (or select the row and in the grid menu select Edit).

    The Edit Matching Criteria dialog box opens.

  5. In the Project Zero Curve lookup, select the appropriate projecting curve defined through the above process (one with Dual Curve set to Yes).
  6. In the Discount Zero Curve lookup, select the same curve set as the Dual Curve Discount Curve in your Projecting curve definition.
  7. Click OK.
  8. The grid row shows your changes.
  9. Repeat Steps 4 through 8 for other applicable rows in the Rules grid.
  10. In the page menu, select Save.

NOTE:  Yield or Deal Valuation Margins at the Par or Zero Curve level may be applied in the existing manner within Valuation Control, but a margin may not be applied to the discount curve used within the Dual Curve Bootstrapping. See Yield Margin for further detail.
