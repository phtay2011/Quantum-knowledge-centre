Set Statement Details

Use the Set Statement Details pageONLY to:

  • set the start date for the first Statement run (for selected bank or stock accounts that do not already have statements);
  • set the next Statement number for selected bank or stock accounts - for example, to restart from 1 at the beginning of each year to comply with SWIFT Usage Rules; or
  • delete the last Account Statement query Update run (with the option to view details of that run first).

To set the Start Date for selected accounts

  1. Select either Bank Accounts or Stock Accounts, as applicable.
  2. In the Account Number multi-select field, click the list button and in the Select Account Number dialog box, select one or more accounts (or ALL) and click OK.
  3. Select Set initial date* and enter the Start Date you want to apply to the next (and first) statement for each of the selected accounts.

    *TIP: This option is available only if no Statement run already exists for one or more of your selected accounts.

  4. Click Update.

    The first Statement Start Date is set for eligible Accounts (those with no existing Statement AND no Unapproved pending changes). The next Account Statements Reporting or Processing run including any of those accounts will include account details from your entered Start Date in each case.

    A popup message indicates the Update status as follows.

    • green - when all selected Accounts have been updated,
    • yellow - when not all Accounts could be updated, or
    • red - when none of the selected Accounts could be updated.

      TIP: Details of any accounts that were not updated are available in the Message Log.

To set the next Statement Number for selected accounts

  1. Select either Bank Accounts or Stock Accounts, as applicable.
  2. In the Account Number multi-select field, click the list button and in the Select Account Number dialog box, select one or more accounts (or ALL) and click OK.
  3. Select Set next number and enter the Statement Number you want to apply to the next statement for each of the selected accounts.
  4. Click Update.

    The next Statement Number is set for available Accounts (those with no Unapproved changes). The next Account Statements Reporting or Processing run including any of those accounts will show your entered Statement Number against those accounts.

    A popup message indicates the Update status as follows.

    • green - when all selected Accounts have been updated,
    • yellow - when not all Accounts could be updated, or
    • red - when none of the selected Accounts could be updated.

      TIP: Details of any accounts that were not updated are available in the Message Log.

To remove changes to Account data made by the last Statement Update run

  • This function is NOT available if no updating Statement run has yet occurred for any accounts.
  • Only the latest Account Statement Updating run will have changed the database (Previewing runs and query runs started from the Reporting page cannot).
  • That last Statement run may have covered (and updated) EITHER one or more Bank Accounts OR Stock Accounts (but not both types). Its result set will still be available, but will no longer show the statement-related database values it changed.
  1. In the page menu select Delete Last Run.

    The Delete Last Run dialog box is displayed, with the last Statement run number shown.

  2. (Optional) Click View Run.

    The Account Statement query page is opened in a separate window, displaying the last Statement run results. When you have finished reviewing these, close the window.

  3. To continue with the rolling-back of that Statement run, click OK (or click Cancel if you no longer want to delete it).

    In the Account record for each account that was included in that run, the read-only Last Statement Number field is reduced by 1 and:

    • if Last Statement Number is now zero - that field is no longer read-only and the associated Last Statement Date and Last Statement Run Number fields no longer appear; otherwise
    • if Last Statement Number is still above zero - the read-only Last Statement Date and Last Statement Run Number fields are reset to the date of that statement and its query run number, respectively.

    The Account Statements Reporting or Processing pages will still show the run, but it no longer qualifies as an Updating run and its run results (and any other query run results that include the affected accounts) will show the reduced Statement values as above for each of those accounts.

    TIP: The Delete Last Statement action is recorded in the audit log. Use Audit-Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.
