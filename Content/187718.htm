Configuring Message Queues

In order to use message queues for the import or export of Quantum data, the MessageQueueSettings.config file (located in the \config directory of the Site folder) must be configured for the Message Queue servers. Set up the SITE to point to the required Quantum site that you want to use.

NOTE: For a given Site section, a number of Message Queue servers can be configured.

Service Definition

Use the <service> definition to configure a queue service (for example, outputting from a Data Transfer Location to a WCF service).

Key

Description

Default Value

name

The service name. This must match the Name declaration in the service contract.

ServiceTest

namespace

The namespace declaration used for service contracts and associated callback contracts.

http://tempuri.org

methodName

This must match the methodName declaration in the [OperationContract] attribute of the Service Contract.

Use SendString for an operation contract with text content encoding.

Use SendBytes for an operation contract with binary content encoding.

SendString

parameterName

This must match the parameterName declaration in the [OperationContract] attribute of the Service Contract.

Use contentMessage for an operation contract with text content encoding.

Use byteMessage for an operation contract with binary content encoding.

contentMessage

Server Definition

Use the <server> definition to specify the queue connections. A Message Queue server server is configured using the following attributes:

Key

Description

Default Value

name

Used to distinguish one server from another (max 100 characters).

NOTE: This forms the name of the record in the Queue Connection browse list of the Data Transfer Location page.

N/A

uri

Connection to the Message Queue Broker.

ActiveMQ

activemq:tcp://<machine name>:<port>

MSMQ

msmq://localhost

IBM WebSphere

xms:wmq://WebSphereServer?QueueManagerName=QUEUENAME&amp;ChannelName=CHANNELNAME

Where:

  • WebSphereServer is the IP address of the machine hosting the IBM WebSphere MQ server,
  • QueueManagerName matches the queue manager you have created in IBM WebSphere (must be unique); and
  • ChannelName matches the IBM WebSphere channel created for the queue manager.

Windows Communication Foundation (WCF)

xms:wmq://127.255.255.255?QueueManagerName=QUEUENAME&amp;ChannelName=CHANNELNAME" deliveryMode="Persistent

Must match the attributes of the WCF service contract. See Data Transfer Location output to a WCF service for more information.

WebLogicConnection (Oracle WebLogic Server)**

weblogic://://${weblogicMachine}:${weblogicPort}

Where:

  • weblogicMachine is the IP address of the machine hosting the WebLogic server, and
  • weblogicPort is the port number for the WebLogic connection (7001 is the default).

For example:

weblogic://$localhost:7001

**See also:WebLogic DLL Installation for your Site below.

N/A

queueMachineName

Required for MSMQ.

TIPS:

  • For MSMQ, localhost can be specified using a period (that is, ".").
  • Private queues can be specified by suffixing \Private$ (for example, ServerName\Private$).

N/A

connectionfactory

Required for WebLogic Server only when Subdeployment for a specified Connection Factory is being used.

Gives the unique name of the Connection Factory, as defined in the WebLogic Server Console.

If omitted, the default Connection Factory is used.

N/A

deliveryMode

Persistent or NonPersistent.

NOTE:

  • Not all Queue Providers support this functionality.
    MSMQ only supports persistent queues whereas ActiveMQ supports both persistent and non-persistent queues.
  • Non-persistent queues only maintain their messages in memory and these messages will be lost after a service restart.

Persistent

username/password

The username and password (if required to be supplied for a queue).

N/A
destinationProperty

Use destination properties to change default destination settings.

NOTE: For now, only IBM WebSphere supports this functionality.

For example, if you want to change the xms target client destination property to MQ, add the following to your server IBM Websphere settings:

<destinationProperties>

<destinationProperty name="XMSTargetClient" value="1" />

<destinationProperties>

N/A

WebLogic DLL Installation for your Site

Before you can use Oracle WebLogic Server connections, you must install the DLL file WebLogic.Messaging.dll by copying it into the bin folder on every machine where Quantum is installed (Application Server, Interface Server, Web Server, and so on) and also into the QuantumWeb/bin folder on your installed Quantum Six Web Server.

You can find the DLL file in the folder where WebLogic Server is installed. The file path will be something like:

WebLogicInstallFolder\wlserver\modules\com.bea.weblogic.jms.dotnetclient\WebLogic.Messaging.dll

...where WebLogicInstallFolder is the folder where WebLogic Server is installed - for example, C:\Oracle\WebLogicServer\ on a local site.

For more details, contact your Quantum consultant.
