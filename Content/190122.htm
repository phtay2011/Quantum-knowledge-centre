Queries and Reporting in Quantum

Terminology in Quantum Six

  • Query - a set of parameters defining a run instance of the underlying application
  • Report - a full set of results from a Query run
  • View - a defined format for viewing report results
  • Explorer - the Analytics or Adaptiv application used to drill down through the structure of folders and saved queries (see Analytics Explorer)
  • Definitions - predefined Analytics and Adaptiv static data types whose records can be selected in queries to quickly provide reusable query configuration settings (see Query Definition Elements)

Reporting v Processing Modes (Quantum)

Some Quantum queries perform database processing - for example, the Accruals page and Position page in the Accounting menu. These have both a Preview button, so you can check before committing the database changes, and an Update button, to then commit the changes.

Some of the Processing queries also have a Reporting-only mode, available in the Reporting menu, that enables you to check results without updating the database.

TIP: For EFT output, there are three modes: Reporting, Processing, and Reprocessing, the latter being accessed via the Utility menu and used only to re-send previously sent messages, which Processing mode cannot do. For more details, see EFTs - Reporting, Processing, and Reprocessing Separated.

Resource Security

Query resource filtering will affect the functionality described in this section. See Applying Resource Security to Queries and Reports.

Creating Workflow Queries

Use the Workflow Report page to create and maintain site-specific reporting pages, which can combine data from multiple Quantum Six* data sources, and which can be placed anywhere in the Quantum menu structure. To the end user, these reporting pages will look and behave like any other core Quantum reporting page.

If you only want to modify the user interface of core dealing-related pages, use the Workflow Page page.
