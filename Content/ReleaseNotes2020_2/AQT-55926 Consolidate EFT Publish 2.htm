EFT - Publish to ECHOS option removed

The Echos Gateway Settings no longer support Publisher settings, so existing users will need to migrate Publisher settings before working. Alongside this there are changes to the Data Transfer Location (DTL) page and the External Method Sources (EMS) page.

NOTE: The ECHOSGatewaySettings.config still supports Listener settings.

Changes to DTL and EMS pages

In the Configuration > External Method Source > EFT Processing area, we removed the Type Export option Publish to ECHOS and added a new option, Queue.

The system now publishes to ECHOS by using a Data Transfer Location (DTL) page that you define and then select in the Destination field (previously labeled File Destination).

When Type Export is Create File, the Destination field options you may select are:

  • a site EFT folder name, or
  • any predefined DTL page that has Purpose set to Eft (File or Queue type, respectively).

The system outputs payment files to the location specified for an EFT Processing run, which can be any network location you have access to.

For more information about setting up the DTL page, search the Knowledge Center for "Configure and use the DTL to send EFT messages using QUEUE".

Changes to MessageQueueSettings.config

Please add text similar to the following to your MessageQueueSettings.config. For existing users, adding this to the MessageQueueSettings.config replaces the functionality of the current settings in the ECHOSMessageQueue.config.

For more information, search the Knowledge Center for "Configure and use the DTL to send EFT messages using QUEUE".

Upgrading from a prior release

The system DOES NOT remove a Publish To ECHOS setting from the Type Export field on any existing External Method Source records.

However, to edit or alter these existing records, change this setting because:

  • if you re-save with Publish To ECHOS selected, the system returns an error message, and
  • if the system processes an EFT using the External Method Source record, the system logs an error message.

The system cannot change the Type Export field to Queue for you, because you have to first set up a DTL page.

If using an External Method Source page with the same Destination as a DTL page's Location, set up the digital signature and encryption from the DTL page. The system hides these fields on the External Method Source page in this situation.

For more information, search the Knowledge Center for "Configure and use the DTL to Send EFT Messages using QUEUE".

Removed - "Publisher" in Quantum Configuration wizard

In the Quantum Configuration wizard, in the Echos Message Queue Settings dialog box, we have removed the Queue Type radio button Publisher.

We've also removed it from other configuration files, such as the Echos Gateway Settings.

NOTE: The Queue Type Listener remains supported.

View message received from ECHOS

To view messages received from ECHOS, open Processing > ECHOS and set up the ECHOS Messages page to show these messages. This is current functionality.
