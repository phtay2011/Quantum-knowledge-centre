XML Confirmation and EFT file maintenance

You can now view and edit Confirmation and EFT seed XML Template files and related files as data records within Quantum, via the new XML Data page.

The way the system uses the XML data remains unchanged. The system uses the data to specify the format and content of Confirmations and EFTs. It covers XTC Configuration, Data Templates, Layout XML, Stylesheets, "Includes" files, and ATS Transformations, as indicated by the record Type. Initially, when the site is first installed, each XML Data record accessible via the browse function corresponds to a seed file previously located in your site's Confos folder.

For more information about previous functionality, see History.

You can copy the seed data provided with your system (with a new Name) and then edit* - using an XML Rich Text editor - to create your own “non-seed” records, which you can use to provide different formatting and content for specific output messages, as required.

*NOTE: You can copy the system-supplied seed records, but you cannot edit them.

In the Electronic Confirmation Format rule Settings, you now select the XTC Configuration XML Data record for each deal action (instead of a physical .xtc file). Similarly, in the External Method Source for an EFT, you select - in the Transformation Configuration File field - an XTC Configuration XML Data record.

New or changed records are subject to Module Security and are audited.

Upgrading your database

The system replaces seed records when the database upgrades. Any non-seed records are not affected – although you should check for changes to the seed record they were copied from, in case those changes affect your non-seed ones.

Providing you have not changed any of the pre-existing XML formatting files during the upgrade process, you will not need to change existing Electronic Confirmation Format rules and External Method Source records for EFTs in your upgraded system, because the XML Data records are built from, and have the same names as, those files.

NOTE: The system does not automatically save the non-seed records to the database via upgrade. This is a manual process for now.

Restrictions and exclusions

QMT files that can optionally be used for stock accounts if not using the XML Framework are not included as XML Data Record Types.

History

Previously, Confirmation and EFT XML Template files and related data was available only in external XML-formatted files that could not be viewed, copied, or edited within Quantum.

NOTE: While the original seed files currently remain located in the installed Confos folder for reference purposes only, they are no longer used by Quantum.
