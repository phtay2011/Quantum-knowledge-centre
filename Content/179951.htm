SWIFT 2015 MT Release Standards

(First introduced in Release 6.1.6 SP, August 2015.)

Quantum Six now supports the following changes required by the 2015 SWIFT MT Release Standards.

Tag 32F Validation for White Metals

A new Validation rule - to prevent mismatching of white metals trades - has been added to the Seed XML Layout files for the following SWIFT message types:

  • MT600 Commodity Trade Confirmation,
  • MT601 Commodity Option Confirmation,
  • MT604 Commodity Transfer / Delivery Order (EFT), and
  • MT605 Commodity Notice to Receive (EFT).

The rule enforces the use of unit code GOZ for "white" metals (Silver, Palladium, and Platinum), and prevents the use of GOZ (and TOZ) for Gold. Specifically, the commodity indicated in the Type subfield of field 26C in Sequence A determines the value in the Unit subfield of field 32F in Sequences B and C. If Type* is.

  • PALL, PLAT, or SILV - then Unit is set to GOZ, or
  • GOLD - then Unit is set to UNT.

*TIP: The respective Commodity/Currency codes are XPD, XPT, XAG, and XAU.

Tag 59F Support in Payment Messages

In addition to the output options Field 59 and 59A (Beneficiary Customer) in MT101 and MT103 Payment messages, Field 59F is now also supported, and is included in the supplied Seed Layout XML file (SeedXMLLayout_EFT.xml).

The new formatting attribute prefix_line of a fielditem element (within a multi-line Combination element) supports continuation of a line where the field content either exceeds the specified line length (maxLength) OR has a line break, with each output line preceded by the attribute value.

Network Validated Rules:

  • the first line must start with number 1,
  • numbers must appear in numerical order,
  • number 2 must not be used without number 3, and
  • the first occurrence of number 3 must be followed by a valid ISO country code.

Usage Rules:

  • if the account number of the beneficiary customer is known, it must be stated in Account,
  • line numbers may be repeated, and
  • if number 2 is present, the first occurrence of number 3 must include the town in additional details.

Example 1

Simplified XML Layout file code snippet for Field 59F:

Counterparty Delivery Instructions settings:

  • Beneficiary Name

Benny Fishery with Long Name of 45 characters

  • IBAN Account

DK391234567ABCD

  • Postal Address

First Address line also very long with 51 characters
Second short line
Third line
Fourth line [not supported]

  • Country code

BE

  • Location name

Brussels

Resulting output for Field 59F:

:59F:/DK391234567ABCD
1/BENNY FISHERY WITH LONG NAME OF 4
1/5 CHARACTERS
2/FIRST ADDRESS LINE ALSO VERY LONG
2/ WITH 51 CHARACTERS
3/BE/BRUSSELS

In the output:

  • the Beneficiary Name is wrapped, so there are two lines with the 1/ prefix,
  • the first Postal Address line is also wrapped, so there are two lines with the 2/ prefix,
  • no further Postal Address lines are output as the maximum of four lines has been reached,
  • the Country Code has the specified (Mandatory) 3/ prefix, and
  • the Location Name has no prefix.
