Using the Workflow Template

This section contains procedures relating to using your Workflow Template.

To edit a template

  1. Open Configuration > Workflow > Template.
  2. In the browse list, select the template you want to edit.
  3. Click the input or expression.
  4. Select Edit from the grid menu.
  5. Edit the input's Name, Description, Type, and, optionally, the expression's Name and the mathematical Expression.
    • Inputs area- You can only edit the Type if the input is not used by an expression or by template text.
    • Expressions area - This can be any type of expression.
  6. Click OK.

To delete an input or expression

  1. Open Configuration > Workflow > Template.
  2. Click the input or unused expression.
  3. In the Input grid menu or Expression grid menu, select Delete.

The system deletes the input or unused expression with no warning. The system will not delete an expression that is in use.

To delete a template

The system will not delete a template that is in use - that is, one specified in a Data Transfer Location record.

  1. Open Configuration > Workflow > Template and select the template name.
  2. In the page menu, select Delete.
  3. Confirm that you want to delete the template.

To add sample data - Bulk Replace

  1. Open Configuration > Workflow > Template.
  2. In the page menu, select Bulk Replace. The Bulk Replace dialog box appears.
  3. Paste tab-delimited text into the field, for example:

    Effective Date®Action Date®Component®Amount

    2019-02-08®2019-11-08®INTEREST®2000000.00

    2019-02-08®2019-11-08®RATE SET®10.000000

  4. Click Replace from the drop-down toggle.
  5. The system validates that each row has the same number of tab characters, and that the tab-delimited values of the first row are unique. You cannot have the same column name twice.

The first row becomes the column names. Each column in the grid is text. The data in the grid is persisted with the template.

To add sample data - Import EFT Data

  1. Check that the user has the required module security permissions in the User > Module Security tab.
  2. Open Configuration > Workflow > Template > Workflow Template module > EFTs.
  3. If using one of the seeded workflow template pages, from the Browse list select the name. For example,

    • FIToFICustomerCreditTransfer - pacs.008

    • FinancialInstitutionCreditTransfer - pacs.009

  4. If creating your own workflow template, set up the workflow template with Output Format= XML and Purpose = EFTYou can:

    • enter sections of code that refer to the inputs into the text box (template area);
    • set up validations, and their error messages, referring to those inputs; and

    • set up expressions referring to those inputs.

  5. In the page menu, select Import EFT Data to import some seeded sample data and merge with some existing EFT-related static data.

To control the whitespace in the Preview tab

Use the following to control the whitespace in the output:

A template line is not output when it only contains whitespace and one of:

  • opening section - for example {{/if}}
  • closing section - for example {{/unless}}
  • {{else}}
  • {{~ use to remove whitespace before the instruction
  • ~}} use to remove whitespace after the instruction
  • ~ use with all of these:
    • {{Input}}
    • {{Expression}}
    • {{each}}

 For example,

<root>

{{#if door}}

<door open="

{{~DoorIsOpen}}" />

  {{/if}}

</root>

...renders as:

<root>

<door open="true" />

</root>

Example 1: In the Template tab's text box, enter the code with as much white space (blank lines) as you need, for example:

{{#if C1}}

{{Input1}

{{/#if}}

-------------

{{#each....

Example 2: If you do not want the white space to appear in the result in the Preview tab, edit to include the tilde (~) as shown:

{{~#if C1}}

{{Input1}

{{/#if}}

-------------

{{#each....

Example 3: In the Preview tab's text box, this removes the white space (blank lines) around the output that relates to the #if.

One

...

To set a default value for an input

If you expect to have some blank cells in the source data, then configure a default value into the appropriate row of the Inputs grid. If you then use the input in the Configuration > Template > Expression Builder, then you must always configure a default value for that input.

The default value when Type is Yes/No, Amount, and Number is 0 (zero).

To set up #each statements

  1. Open the Template tab.
  2. In the template's text area, set up nested #each statements. You can nest #each statements to any level. You can only nest the deepest #each statement without a collection.
  3. In Sort, select one or more inputs and/or expressions to set up the order of the inputs and expressions that the #each statement is over. The default direction is ascending.
  4. The data appears in the template's output file and Preview tab.

NOTE: To sort in reverse order, use -1*'[Amount], or to sort on the number of days to a date in the future, use Date(2200,1,1) - [Date].

Examples of invalid #each statements that throw a validation

  • Invalid: nesting on the same expression

{{#each Expression1}}

{{#each Expression1}}

<something>{{something}}</something>

{{/each}}

{{/each}}

  • Invalid: #each without a qualifier, not at the deepest level of nesting

{{#each}}

{{#each Expression1}}

<something>{{something}}</something>

{{/each}}

{{/each}}

  • Invalid: using the same input for the nested #each

{{#each Input1}}

{{#each Input1}}

<something>{{something}}</something>

{{/each}}

{{/each}}

  • Invalid: #each without an input, not at the deepest level of nesting

{{#each}}

{{#each Input1}}

<something>{{something}}</something>

{{/each}}

{{/each}}

To set up an @index keyword

  1. Open the Template tab.
  2. In the template's text area, set up an @index keyword in order to access the current index of the innermost loop it is used in.
    • You can use a parent path indicator (../) to access the index number of parent each loop. For example, @../index for the direct parent, @../../index for the parent above that.
    • Input and expression names cannot include @ symbol.

To test whether the input, expression or source column is blank

Use an #if statement to test whether the input, expression or source column is blank. When the input, expression, or source column is blank, the condition is false. Otherwise, the condition is true. This option is available when the Type = Text in the Inputs, Expressions, or Source grid.

Example 1

For a text input or expression called Name:

{{#if Name}}

<name>{{Name}}</name>

{{/if}}

Example 2

For a text column called Name from a source called Counterparty:

{{#if Counterparty.Name}}

<counterparty>{{Counterparty.Name}}</counterparty>

{{/if}}
