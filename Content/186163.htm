Requests and Approvals

Approvals will apply to Requests that match Deal Approval Rule entries, which can be "catch-all" rules or rules specific to FX or Money Market Requests.

Deal Approval Rule Settings

Applicable Matching Criteria for Requests are:

  • Transaction Type - ALL, FX Deals, or Money Market,
  • Deal Type - ALL or REQUEST,
  • Instrument - ALL, an FX instrument, or a Money Market instrument,
  • Action - ALL or REQUEST (also DELETE for Request deletion), and
  • other criteria - ALL or a matching setting.

    NOTE: Requests without a specified Entity, Counterparty, and/or Dealer will match to rules with ALL settings for the relevant parameters.

Applicable rule Settings are:

  • Approval - YES,
  • 1st Approval - not set to NONE,
  • 2nd Approval - required only for Dual approvals, and
  • Allow Processing - YES or NO to respectively allow or disallow Requests to be filled before they are fully approved.

Approval of Request Actions

Where a matching Deal Approval Rule exists:

  • qualifying Approver users can approve or reject Request creation, financial change, deletion, and resetting using either one at a time on the Request page or in bulk on the Deal Approval page, but
  • approvals are NOT required for:
    • Fill or Mark as Filled actions (but you can set up Approval Rules for actual deals that can be created by a Fill), nor for
    • Reset actions on Filled request without any financial change (the clearing of the link details alone is not a financial change).

On the Request page, when a First Approval is pending for a new Request Save or for a subsequent Change action:

  • fields are editable and changes can be saved without requiring a Reset,
  • the Reset menu item is disabled,
  • if Allow Processing is NO in the Approval Rule, the Fill and Mark as Filled menu items are also disabled, and
  • when the Request function is selected, the Request Approve dialog box displays:
    • a Pending Approval grid showing the status and details of the action pending approval, and
    • (if there are earlier actions) a second Approved Actions grid giving the status and details of the earlier approved actions.

When a First Approval is granted for an action and a Second Approval is also required:

  • the Request remains in the same state, and
  • the Request Approve dialog box displays details of the pending Second Approval and actioned First Approval.

When a Request becomes fully approved:

  • a Request Approved badge is displayed on the page,
  • fields on the Request page become read-only,
  • the Reset menu item is enabled, and
  • a Reset is required to make any further changes.

When you Reset a fully approved Request:

  • a dialog box is displayed for you to confirm or cancel the Reset,
  • if the Request is Filled, you are notified of the linked actual deal and that the Reset will disconnect it from the Request,
  • if you confirm the Reset:
    • the fields on the Request page are re-enabled and changes can be saved,
    • any changes to the Request details, if saved, will in turn need approval, and
    • any Fill details such as the fill deal number and fill date are removed from both the request and the previously linked actual deal,

      NOTE: A Change approval is NOT triggered by a Reset and Save on a Filled request alone (if a rule exists), as the clearing of the link details itself is not a financial change.

Who Can Approve or Reject Requests?

Users can action a First Approval when:

  • they have an Approver record with:
    • Deal Approver selected,
    • Deal Approval Signature List matching the 1st Approval setting on the Deal Approval Rule applicable to the Request, and
    • Deal Approval Limit Ccy, Limit Amount, and Entities settings are consistent with the applicable Deal Approval Rule; AND
  • they are not the Creator or Dealer of the Request, OR
  • they are the Creator - but Excluded to Approve is DEALER or NONE (on the Deal Approval Rule), OR
  • they are the Dealer - but Excluded to Approve is CREATOR or NONE.

Users can action a Second Approval when they meet the same criteria as above for First Approval, EXCEPT THAT:

  • their Approver record's Deal Approval Signature List matches the 2nd Approval setting on the Deal Approval Rule applicable to the Request, and
  • they are not the First Approver, OR
  • their Approver record's Deal Approval Authority is set to Both.
