Deal Approval page

Use this page to view and approve deals and their related action events. By default, only deal actions that you (the logged-on user) may approve will be displayed.

NOTE: The criteria established in Approval and Authorization Rules (Deal tab) determine which deals may require approval (and by whom). Approver records and their associated Signature Lists define user approval types and levels.

Filters (defaults - you can add others) - Enter values and make option selections to reduce or expand the result set.

(For details of a filter parameter, click on it when the help panel is showing.)

WHAT DO YOU WANT TO DO?

  • Select a different Query or previous run - Click on the current Query heading and select the Query or run in the browse window. By default, the system retains run results according to the Expiry Strategy settings for the Query configuration (on the Properties tab) EXCEPT THAT selecting Keep in the Page menu keeps the current run results only.
  • Reconfigure the Query or configure a new one - Click the Query Configure link, make your changes, add expressions, and save them.
  • Select a different result View - Click on the current View heading and select in the browse window.
  • Change the View or create a new one - Click the View Configure link, make your changes, and save them.
  • Run the current Query - Enter specific Filter and Option settings and click Run Query.

    TIPS:

    • Use relative dates in date fields so you can reuse queries in future runs.
    • The returned data is subject to Resource Security.
  • Export a standard report - Select Export in the page menu to export a report to CSV, PDF and XML formats.
  • Export a Crystal Report - Use the export control in a Crystal View to export a export a report to a variety of different formats.
  • Action multiple items - Select the items in the result grid (by clicking the check box in the first column of each item row) and click the applicable function button (Approve, Reject*, or Reject with Note* as appropriate), or select the required function in the grid menu.

    *TIP: The Reject function is not available if the system-wide option Require Rejection Notes is selected (on the Operational Settings page).

  • Action a single item - Click on a linked value in the item row and select the required function from the quick menu.
  • Open the Deal page for a single item - Click on a data value for the item and in the quick menu, select Deal.
