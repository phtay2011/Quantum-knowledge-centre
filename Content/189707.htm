Dashboards

You can now create pages containing the result sets of different query-driven modules and other data from the new Dashboard module.

History

Previously, composite pages could only be created by defining a View Source Code file (.cshtml) containing the required layout. This file had to be placed in the Quantum server.

NOTE: Composite page functionality is no longer available. See Exclusions and Restrictions below for more information.

Implementation

A new Dashboard page has been added to the Configuration menu and is controlled by standard module access permissions. Dashboard pages created from this page are, in turn, subject to module permissions, and must be enabled for individual users with the Inquire permission enabled.

Users with access to newly created dashboard pages will only see these pages in the site menu upon re-logging-in to Quantum.

The layout of a dashboard page is completely defined using the dashboard XML layout.

About Dashboard Pages

Dashboard pages are composed of panels and components which can be arranged within the page space in various combinations, according to the Dashboard XML Syntax.

The space available on a page for arrangement of components and panels is divided into six equal columns. By default, components will fill up all columns available in the page or panel they are part of, but a specific size can also be set. Quantum arranges the dashboard to fill up the columns available, according to their sequence and width as defined in the dashboard XML layout.

The panel acts as a sub-container, enabling you to group other panels and components within its space.

Components display information and data within the dashboard page. Components can be added to a panel or directly to the page. Components are as follows:

  • message - Displays text in the dashboard. The text can contain basic HTML.
  • external - Displays an external site in the dashboard.

    NOTE: External websites need to be whitelisted in the Web.AppSettings.config file's AllowedDashboardDomains key first, for reasons of security.

  • report - Display the result set of query-driven modules (Reporting, Processing, User Maintenance, and so on).
  • browse - Display page records from a page's browse list.

Component Heights

The dashboard XML element's pagesize attribute determines the height of the component (for example, a grid of pagesize= 5 will occupy the height of a grid with five rows even if it returns only three rows of data). All components occupy a fixed minimum height, regardless of the data they contain.

Focus Mode

Click in the component's header to display the single component within the full space of the page; click to toggle the view back to the full dashboard.

  • Use Focus Mode to act on the data displayed in a component. For example, in Focus Mode, a component grid displays all the rows returned by the query as displayed in the original page, along with any processing, view, or grid controls, if available.

Exclusions and Restrictions

Following this update, the Composite Pages functionality will no longer be available and you will have to recreate these pages using the Dashboard module.

The Rate Fix NDF’s, Rate Set Deals, Mature Deals, and Roll Deals pages cannot be included in a Dashboard.
