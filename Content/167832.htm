Deal Set, Rules

Use the Deal Set Rule page to create and maintain Deal Set Rules, which restrict the types of deals that can be put into different deal sets and whether those deals can be updated.

Changes to an existing Deal Set Rule immediately impact all existing Deal Sets that use that rule.

The Deal Set Rule page is divided into the following areas.

Header

This area contains the following fields:

  • Name - the unique key value identifying the record in the system (up to 25 characters),
  • Short Name - a shorter name that can be used for reporting where space is limited (up to 10 characters), and
  • Description - optional text for reference and reporting purposes (up to 40 characters).

Matching Criteria

This area's grid has one or more entries that restrict the deals that may be in a Deal Set specifying the Deal Set Rule Name. Each entry row specifies:

  • Transaction Type - either a single transaction type or ALL, and
  • Instrument - either a single instrument for the transaction type, or ALL.

Deals that do not match any grid entries for a Deal Set Rule cannot be put into any Deal Set specifying that rule.

Deals that do match a grid entry are subject to the rule's Intended Deal Set Use and On Update Outside Intended Use restrictions (see below).

Intended Deal Set Use

Use these settings to restrict whether deals can be updated outside of the intended use.

  • Maintenance Mode - select the intended type of system access for updating deal data - one of:
    • Interactive - via standard or variant* Quantum Dealing pages,
    • API - via an external Application Program Interface, or
    • Import - via the Deal Importer.

      *TIP: A variant page is one that is not part of the core Quantum Six product. Usually based on one or more core pages, it is customized for specialized use by certain clients. A simple example could be an adaptation of the Currency Option Instrument page to allow only Vanilla option types and with the Facility field relabeled Syndication.

  • Context (applicable when Maintenance Mode is Interactive or API) - an optional value that, if specified here, a variant page or API must pass through to qualify as an intended use of the Deal Set.

On Update Outside Intended Use

Use these settings to specify any restrictions on Deal Actions and Process Actions when this rule applies and system access does not meet the specified Intended Deal Set Use criteria.

In each case, select one of:

  • Are Allowed - to apply no restrictions,
  • Need Confirmation - to display a message when deals are updated or assigned to a Deal Set,
  • Are Blocked - to disallow deals to be updated or assigned to a Deal Set.

    When bothActions fields are set to Are Blocked, a badge is displayed at the top of the page that this rule applies only to deal sets for the specified use. This will be used in combination with the Matching Criteria to restrict Deal Set selection on Dealing pages to those deal sets that match the rules.

Example 1

If Maintenance Mode is API (deals are created via an automated process) and Deal Actions is Need Confirmation, then whenever deals in applicable Deal Sets are opened in Quantum, a warning message will appear and users must confirm to continue.

Example 2

If the mode is Interactive, Context is Payment Hub, and Deal Actions is set to Are Blocked, then only the variant page called Payment Hub can create deals in applicable Deal Sets. If any other page, any API, or the Importer attempts to save a deal using this rule, an error message is displayed: This deal can only be updated in context 'Payment Hub'.
