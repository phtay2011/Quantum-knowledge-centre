External Cash Flow Import CSV file format

This topic covers cash flow import formats.

The first header line of the CSV file contains the column name properties (separated by commas) to be set for each cash flow item.

The second and subsequent detail lines contain the cash flow information that is to be imported into the Analytics database.

Example

Line 1: dealno,itemdt,itemtype,effectdt,ccyid,amount....etc

Line 2: 0001,14/5/2014,Principal, 14/5/2014,AUD,1000000....etc

Line 3: 0001,14/5/2014,Fixed Rate,14/5/2014,AUD,5.00....etc

Line 4: 0001,15/5/2014,Repay Principal,15/5/2014,AUD,-1000000....etc

Line 5: 0001,1/6/2014,Interest,15/5/2014,AUD,-500....etc

Required Header Rows and values

The following are required for cash flow import csv files.

dealno

Used to match cash flow detals to the deal import csv.

NOTE The dealno information is expected to be in the same order in both the deal and cash flow csv files.

side

Required for Swap transaction types only; one of:

  • 1 - for Pay side
  • 2 - for Receive side

itemdt

Physical cash flow date

effectdt

Date for cash flow calculations

itemtype

Description of cash flow - for example Principal, Interest, and so on

Available values are those listed on the schedule table of the Analytics Pricing screens.

ccyid

Required for Foreign Exchange and Currency Swap transaction types.

For all other transaction types, the currency will default from the Instrument record.

bankaccid

If this field is not populated, the Analytics Bank Account defaults will be applied to the cash flow.

amount

Currency amount of the cash flow.
