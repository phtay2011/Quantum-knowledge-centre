CLS workflow enhancements

You can now use the same FX instrument in combination with Cut Off Time Rules to handle a Continuous Linked Settlement (CLS) workflow for FX deals; applying CLS-specific account defaults and generating MT300 or MT304 confirmation messages, depending on whether deal is CLS-settled deal or not.

Cut-Off Time Rules

Cut Off Time Rules include a new Scope specific to CLS for FX deals.

The presence of a CLS rule will determine the value for the Is CLS and Is Past CLS Cut Off properties on FX deals. It also determines which Bank accounts and Counterparty accounts are used.

When Is CLS is Yes for a new FX deal, a matching CLS Bank Account Defaulting Rule must be present for both currencies and FX Deals, while CLS Counterparty Account Defaulting Rules are optional (in both cases, rules for Transaction Type ALL are ignored).

FX Deal

Two new data items are available for FX deals: Is Past CLS Cut Off and Is CLS.

  • Both can be displayed in the deal browse list and used for filtering.
  • Is CLS is determined at deal save time and remains unchanged throughout the life of the deal, unless the FX Settlement Switch to function is initiated (on the Confirmations Processing page - see below).
  • When an applicable Cut-Off Time Rule is found for the deal and for each currency (or ALL), the deal's Value Date is compared to the rule Cut-Off Time and the Is Past CLS Cut Off and Is CLS values are set as follows.

  • also applies to FX deals created when you perform a PHYSICAL EXERCISE action on a Currency Option, and
  • does not affect the existing deal save validation options for CLS, as determined by the underlying instrument's CLS Settlement setting.

Bank Account and Counterparty Bank Account Defaulting Rules

Under Matching Criteria when Transaction Type is FX Deals, you can now select either CLS or Standard Settlement.

This means you no longer need to define rules at an instrument level. You can specify your CLS Settlement Provider's bank account for CLS-settled deals, and a different bank account for Standard Settlement.

Likewise, for Counterparty Account Defaults, you can set Payment Category to EFT Not Required for CLS-settled deals; and to Standard Settlement Instructions or Debit/Credit Advice for Standard Settlement.

CLS defaults will be used when Is CLS is set to Yes, either at deal save time when a matching CLS Cut-Off Time Rule is found, or when the FX Settlement Switch to function is used on the Confirmations Processing page (see below).

CLS rules are specific to FX deals and rules for Transaction Type ALL are ignored for CLS processing.

Electronic Confirmation Templates

New XML Template files are provided to cater for either MT300 or MT304 Confirmation message output depending on the Is CLS value.

Specify the new SeedXMLConfig_MT300_MT304_CLS.xtc XML Template Configuration file in your FX deal Electronic Confirmation Template Format Rules to generate an MT300 when Is CLS is No, and an MT304 when Is CLS is Yes. This Configuration file specifies the new XML Layout file that is required (SeedXMLLayout_MT300_MT304_CLS.xml).

Confirmations Processing

The new FX Settlement Switch to quick menu functions enable you to switch from CLS to standard settlement, and vice versa. The applicable action is available only on a single selected FX deal action when:

  • the latest deal action is either the initial DEAL action or a CHANGE (the latest action cannot be a natural action such as a PRE-DELIVERY for example);
  • the confirmation External Method is either Electronic, Misys (for Finastra), or SWIFT Confirmation; and
  • the current user has the Switch CLS and Inquire Module Security permissions for the FX Deal module (or is exempt).

When you perform an FX Settlement Switch function:

  • a CHANGE action is created, triggering a Cancel of the previous Confirmation (if one had been generated) and a new Confirmation message; and
  • the Bank Account and Counterparty Account for that deal's cash flows are changed from CLS to Standard Settlement defaults (or vice versa).

History

Previously the system did not capture whether a deal was past cut-off time or not; nor whether a deal was CLS-settled deal or not. You were required to set up specific account defaults and confirmation templates at an instrument level, duplicating each instrument to control workflow; and any change from CLS to Standard settlement or vice versa required a deal change to a different instrument.
