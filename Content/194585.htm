Automatch - Times reduced

(First introduced in Release 6.8.0)

When running scripts, Auto-match process times have been reduced.The balance script times were reduced by up to 70%.

Balance match script and script chain auditing

(First introduced in Release SP 6.7.1, August 2018.)

Changes to Bank Reconciliation Balance Match scripts and Script Chains are now audited (auditing of Detail Match script changes was introduced in Release 6.6.1).

For more details, see Auditing of Balance Scripts and Chains

Match ID uniqueness

(New from Release 6.8.0)

Match IDnumbers are now unique across all Bank Reconciliation matching.

(Previously, the Match ID sequence was unique per bank account.)

All existing match IDs will be migrated to the new sequence during the upgrade process. Newly generated match IDs will now use the next available number.

Match Script - Preceding Business Day option for End Date

(First introduced in Release SP 6.7.1, August 2018.)

In Balance and Detail Match scripts, you can now set the End Date for the data scope to the preceding business day.

This means that you no longer need more than one script to process your data when non-business days are involved (see the example under History below).

Implementation

On the Details tab of both the Balance Match Script and Detail Match Script pages, you can now select the Preceding Business Day relative date option in the End Date field.

When such a script runs against an account, the system will apply the non-business day settings for the account (from the Worksheet Settings tab) to determine the non-business day rules.

History

Previously, you may have needed multiple scripts with different End Dates to process your data when non-business days were involved.

A typical example was where you needed:

  • one script with End Date set to End of Last Week to process your Friday data when run on Monday, and
  • another script with End Date set to Yesterday to process your Monday through Thursday data each day when run on the next day (Tuesday to Friday).

(Now, only one script is needed to cover both cases.)

Prior Day Matching times reduced

(From Release 6.8.0)

In Prior Day Match, on the Flows tab:

  • the time to select/deselect a single bank flow or cash flow has been reduced by up to 60% - particularly if you have a large number of items in the grid,
  • the time to match/unmatch a single bank flow or cash flow has been reduced by 70-80%, and
  • the time to refresh the Flows grid has been reduced by up to 35% - particularly if you have a large number of items in the grid.
