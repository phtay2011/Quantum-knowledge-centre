Deal Cash Flow View Enhancements

(First introduced in Release 6.5.1 SP, September 2017.)

On all Dealing pages with a Cash Flows tab, you can now configure that tab's grid view to filter on, and/or include as displayed grid columns, the following new items:

  • Flow Lifecycle,
  • Flow Lifecycle State,
  • Flow Lifecycle Phase,
  • a badge representing the Flow Lifecycle Phase,
  • Payment Instruction,
  • Beneficiary,
  • Counterparty Account Number,
  • Counterparty Account Name,
  • Counterparty Bank, and
  • Net Number.

NOTE:

  • Columns for these items are not shown in the Default view - you must manually add them.
  • Cash Flow tab views are common across all deal types.

The badges provide an instantly recognizable visual indication of the Flow Lifecycle Phase.

Flow Lifecycle values indicate a cash flow's current stage in its processing lifecycle within your system, as descriptive text values - for example Created, Amended, Awaiting Settlement Action, and so on.

Flow Lifecycle State is a unique number corresponding to the Flow Lifecycle value.

Flow Lifecycle Phase values and their corresponding badges are listed below.

  • Confirmation
  • EFT Preparation
  • EFT Completion
  • Exception

TIPS: In the grid View configuration:

  • include the Flow Lifecycle Phase column to show the text values;
  • include the column to show the badges; and
  • when familiar with the badges and Flow Lifecycle State numbers, a View displaying only those columns significantly reduces overall grid width.
