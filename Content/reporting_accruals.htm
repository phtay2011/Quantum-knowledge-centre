Accruals Reporting page

In Reporting mode, this query enables you to preview accruals, calculating and listing General Ledger accrual entries based on the selection criteria prior to committing them to the ledger. The Accounting Rules determine the Accounting Treatment of the deals, including which components should be accrued. These include interest, capital premiums or discounts, option premiums, and forward point amortizations.

TIP: To commit accrual General Ledger postings to the database, run the Accruals query in Processing mode (via the Accounting menu).

WHAT DO YOU WANT TO DO?

  • Select a different Query or previous run - Click on the current Query heading and select the Query or run in the browse window. By default, the system retains run results according to the Expiry Strategy settings for the Query configuration (on the Properties tab) EXCEPT THAT selecting Keep in the Page menu keeps the current run results only.
  • Reconfigure the Query or configure a new one - Click the Query Configure link, make your changes, add expressions, and save them.
  • Select a different result View - Click on the current View heading and select in the browse window.
  • Change the View or create a new one - Click the View Configure link, make your changes, and save them.
  • Auto-run a query - In the Run Query drop-down list, select Continue on error, Stop on error, or Auto-run. If Auto-run:
    • In the dialog box select one of: Auto-run after a pause (and enter the Pause time), Auto-run immediately, or Update live on change in the system.
    • If the Expiry Strategy option is Keep a number of reports (Query Configure/Properties tab), the specified Maximum number of completed runs will be kept; otherwise, only the last two completed runs will be kept.
  • Run the current Query - Enter specific Filter and Option settings and click Run Query.

    TIP:

    • To show Filters and Options when hidden (as after a run), click the parameter summary header, select Change Parameters in the Run Query button's drop-down list, or click the Configure link.
    • Use relative dates in date fields so you can reuse queries in future runs.
  • Export a standard report - Select Export in the page menu to export a report to CSV, PDF and XML formats.
  • Export a Crystal Report - Use the export control in a Crystal View to export a export a report to a variety of different formats.
  • Publish the report - Select Publish in the page menu to make the report results available to other users.

    NOTE:

    • By default, the report results can only be viewed by the user who has run the query.
    • When a report is published, it will not expire as per the Expiry Strategy configured in its associated query.
  • Display previous run results - EITHER click the Previous Runs link (if present) OR click the Query heading, and select the previous run from the browse window or drop-down list.
  • Learn about Filters and Options

NOTE: The returned data is subject to Resource Security.
