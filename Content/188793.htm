Relationship to Accounting Types and Entities

Accounting, posting, and inter-company rules form part of your overall accounting set up. The relationship between these rules, and accounting types and entities determine how transactions are accounted for within Quantum.

Accounting and posting rules

You can define accounting and posting rules at any accounting type level or entity level.

NOTE: To define rules for a transactor entity, the entity must either own, or post to, a ledger.

  • Subsequent accounting types inherit all accounting and posting rules defined for the Master accounting type.
  • Entities that own ledgers inherit rules defined for the accounting type the ledger is based on.
  • Entities that post to the ledger of another entity (that is, their parent entity) inherit the posting rules defined for that entity.
  • Entities defined in Business Structure as group entities, and which do not own a ledger, only ever have locally defined rules. These rules are applied to any transactor entities within the group structure that own or post to a ledger.

Inter-company rules

An inter-company posting rule is defined between two entities. You can define a rule:

  • at group or transactor entity level, or...
  • for an entity without an associated ledger (that is, a non-posting entity).

You cannot define a rule between two non-posting entities; at least one must be a posting entity.

NOTE: You do not define Inter-Company rules for individual accounting types.

Transactor entities that post to the group entity's ledger inherit the inter-company rules defined for that group entity's ledger.

Transactor entities that own ledgers only ever have locally defined inter-company rules. They do not inherit rules defined at the group entity level.

Non-posting entities only ever have locally defined inter-company rules.

The relationship between inherited inter-company rules and counter entities

A counter entity is the entity with which another entity has an inter-company rule defined. For example, if OurCo USA has an inter-company rule defined with OurCo Europe, and the focus in the entity diagram is on OurCo USA, then OurCo Europe is referred to as the counter entity.

When the focus in the entity diagram is on OurCo Europe, then OurCo USA becomes the counter-entity.

If the counter entity is also a group entity that owns a ledger, then all transactor entities within the group structure that post to the group entity's ledger inherit the inter-company rule.

Rules and Inheritance

Quantum applies the concept of inheritance when determining rules. Rules you define on the Master accounting type can apply to all subsequent accounting types and any entity that is associated with the accounting type. When you define a subsequent accounting type or ledger, you can localize any inherited rules by overriding them.

To determine where a rule is inherited from

  • In Accounting Rule, on the applicable tab, the Inherited From column displays the accounting type or entity name from which the rule is inherited.

To override an inherited rule

You can override an inherited rule at any lower level.

  1. In Accounting Setup, on the applicable tab, create a new rule with the same condition as the inherited rule.
  2. Define the local treatment you want to apply to the rule.

The local rule replaces the inherited rule.

NOTE: If you delete the local rule, the inherited rule replaces it.
