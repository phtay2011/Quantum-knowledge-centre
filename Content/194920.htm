Indexed Bond enhancements

Indexed Bond coupon PPH rounding

You can now set PPH rounding specifically for coupons in Indexed Bond deals.

Implementation

On the Security instrument page when Formula Timing (on the Details tab) is set to INDEXED BOND PERIODIC, the new Coupon Rounding field is displayed (on the Pricing tab). Select the rounding method for the coupon component of the price (the accrued interest), for Security deals (or Repo deal legs) based on this instrument:

  • NO ROUNDING - to use the maximum number of decimal places (12). The adjacent Places field is unavailable,
  • ROUND (5/4) - to apply Swiss rounding up or down to the number of decimal places specified in the adjacent Places field,
  • ROUND UP - to always round up to the number of decimal places specified in the adjacent Places field, or...
  • ROUND DOWN - to always round up to the number of decimal places specified in the adjacent Places field.

History

Previously, coupon rounding for Indexed Bond deals was determined by (and therefore the same as) that specified for Income.

Indexed Bond pricing with unrounded coupons

You can now select whether to apply rounded or unrounded coupons in Indexed Bond deal price/yield calculations.

Implementation

On the Security (or Money Market) instrument page when Formula Timing (on the Details tab) is set to INDEXED BOND PERIODIC, the new Round coupon in pricing option is displayed (on the Pricing tab). Select the option to use rounded coupon values in price/yield calculations for deals based on the instrument; clear the option to use unrounded coupon values.

The default setting is:

  • cleared - for new deals, or...
  • selected - only after an upgrade from an earlier release, and only for deals existing prior to the upgrade.

History

Previously, rounded coupon values were always used in price/yield calculations for Indexed Bond deals.

Indexed Bonds - Repo price indexed option

(First introduced in Release SP 6.7.1, August 2018.)

You can now select whether or not, when an Indexed Bond Security instrument is used in a Repo deal leg, an indexed price may be entered for that leg.

Implementation

On the Security (or money Market) Instrument page when Formula Timing (Details tab) is set to INDEXED BOND PERIODIC, the new Price indexed option is displayed in the Repo Pricing area on the Pricing tab. Select the option to enable entry of an indexed price in Repo deal legs based on the Security instrument; clear the option for entry of un-indexed prices (the default).

Indexed Bond Yield when final rate known

Where the trade of an Index-Linked Bond deal occurs after the publication of the final Index Rate that determines the redemption payment, a different formula for calculating the (real) dirty price from the nominal yield can apply.

This formula matches that used by the UK Government.

Implementation

The new formula (see below) will only be used to calculate Yield for an Index-Linked Bond deal on the final coupon date when all of the following conditions apply:

  • in the underlying Security instrument:
    • Formula Timing is INDEXED BOND PERIODIC (Details tab),
    • Coupon Type is EXACT and the Final Period Pricing Method is COMPOUND YIELD (Pricing tab), and
    • the required Index Rate name is set (Other Details tab); also...
  • In the specified Index Rate record:
    • the Use Published Date option is selected, and
    • in the Index Rate Series grid, an Index rate value is available for the final period.

NOTE: The instrument's Round coupon in pricing option setting is not utilized in the final period calculation for Yield or Price when the final coupon is known.

Formula details

Where the trade occurs after the publication of the RPI that determines the redemption payment (Repay Principal in Quantum Six), the formula for calculating the (real) dirty price from the nominal yield will be given by:


The reference RPI for the first calendar day of any calendar month is the RPI for the calendar month falling three months earlier. For example, the reference RPI for 1 June corresponds to the RPI for March, the reference RPI for 1 July corresponds to the RPI for April, and so on. The reference RPI for any other day in the month is calculated by linear interpolation between the reference RPI applicable to the first calendar day of the month in which the day falls and the reference RPI applicable to the first calendar day of the month immediately following. Interpolated values for Ref RPISetDate should be rounded to the nearest 5th decimal place, as should values for IndexRatioSetDate.

Reference

Formulae for Calculating Gilt Prices from Yields, United Kingdom Debt Management Office, 3rd Edition, 16 March 2005
