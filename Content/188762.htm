VAT on Interest Cash Flows

Quantum now supports Value Added Tax (VAT) on Interest Cash Flows for Money Market and Security deals with the following Formula Timing settings:

  • YIELD PERIODIC,
  • PRINCIPAL & INTEREST ACTUAL END DATE, or
  • PRINCIPAL & INTEREST NOTIONAL END DATE.

Implementation

Static Data

On the Tax page, you can now select a Tax Type of VAT, as indicated in the following example.

TIP: You can specify a zero rate - for example, where VAT has been applied to a deal, but needs to be stopped after a certain date.

On the Posting Rule page's Money Market & Securities tab, VAT Credit and VAT Payable posting account fields have been added to the BALANCE SHEET tab, so you can post VAT Payments and Receipts to specified Tax Credit and Liability accounts respectively, and reconcile VAT debit and credit entries.

Quantum Rules

On the Bank Account Defaulting Rule page (under Matching Criteria):

  • if Transaction Type is set to Money Market* or ALL, then...
  • Cash Flow Type may be set to VAT or UNWIND VAT.

On the Counterparty Bank Account Defaulting Rule page (under Matching Criteria):

  • if Transaction Type is set to SE* or ALL, then...
  • Cash Flow Type may be set to VAT or UNWIND VAT.

On the Approval and Authorization Rule page, when you add or edit a rule on the Deal tab, then in the dialog box (under Matching Criteria):

  • if Transaction Type is set to Money Market* or ALL, then...
  • Action may be set to UPDATE VAT.

On the Confirmation Rule page, when you add or edit a rule on any tab (Deal, Settlement, or Deal Ticket), then in the dialog box (under Matching Criteria):

  • if Transaction Type is set to Money Market* or ALL, then...
  • Action may be set to UPDATE VAT.

On the Paper Confirmation Format page (under Matching Criteria):

  • if Transaction Type is set to SE* or ALL, then...
  • under Settings, the VAT field is made available to specify a Paper Confirmation Format file for Update VAT actions.

*TIP:Money Market or SE covers both Money Market and Security deals.

Quantum Instruments

On the Money Market Instrument and Security Instrument pages when the Formula Timing is YIELD PERIODIC, PRINCIPAL & INTEREST ACTUAL END DATE, or PRINCIPAL & INTEREST NOTIONAL END DATE: you can select, in the Tax Exempts field (on the Other details tab) one or more taxes whose Tax Type is VAT, as well as any others.

Dealing

On the Money Market Deal and Security Deal pages, when the selected Instrument's Formula Timing is YIELD PERIODIC, PRINCIPAL & INTEREST ACTUAL END DATE, or PRINCIPAL & INTEREST NOTIONAL END DATE: you can set the Taxes field (on the Other details tab) to a predefined tax whose Tax Type is VAT.

If you do select a VAT:

  • no other taxes may be specified for the deal;
  • the Deal Map tab shows VAT payment or receipt items, applying the tax rate to the interest;
  • the Cash Flows tab shows separate VAT payment or receipt cash flows, using the bank account set up on the applicable defaulting rule;
  • the G/L Entries tab shows VAT General ledger postings, using the G/L account set up for VAT payments or receipts on the applicable Posting rule;
  • the Update VAT action is available in the page menu, enabling authorized users to start, stop, or change VAT on the deal;
  • on a Rateset action, affected VAT amounts are recalculated based on the amended Interest amount; and
  • on an Unwind action, an Unwind VAT Deal Map item is created with an amount equal to the Accrued Interest times the VAT Rate.

Deal Revaluation

Any VAT cash flows are not included in the Market Value (Cash Flow NPV schedule) or Book Value when revaluing a deal using a Projected or Bills Revaluation method, so that the Gain/Loss does not include cash flows that have no market risk.

Module Security

The Update VAT permission can now be assigned to Users or Groups for the Money Market Deal and Securities modules.

History

Previously, VAT was not available as a separate tax type. Implementing VAT-type tax cash flows on interest required complex or cumbersome workarounds. For example, a tax record had to be set up for each Counterparty, which could require hundreds of tax records.
