Configure Query - Definition tab, Pools tab

Use the Definition tab to create, maintain, and view pools.

NOTE: All group and individual resources of each type are initially assigned to the system-generated Unallocated pool of that type. You can view, but cannot change, these pools - they always contain all resources of that type not yet allocated to any user-defined pool for the current query.

To view an existing pool - click and in the Browse window, select the pool.

To add a new pool from scratch - in the tab menu select New; enter a unique Name for the pool, select its Type, enter a Short Name and longer Description; in the tab menu select Apply Changes to show the Resources grid; add and remove items from the grid as below; then in the tab menu select Apply Changes.

  • To add group and/or individual resources to the current pool - click Add; in the Select Resources dialog box select one or more group (G) and/or individual (T) items of the specified Type; then click OK. Note that:
    • selecting Include All from the grid menu adds all available items, which can speed up the process; and
    • if you assign a group to a pool, all group and individual items below it in the hierarchy (its children) are assigned to the same pool, and any previous settings for those child items are overwritten.
  • To remove a resource item from the current pool - click the x button at the right of that item's grid row. The removed resource item is returned to the Unallocated pool.
  • To remove all resource items from the current pool - select Exclude All from the grid menu.

To add a new pool by copying an existing one - click and select the pool to copy, in the tab menu select Copy, enter a new Name, edit values, add and delete grid rows as required (see above), and in the tab menu select Apply Changes.

To edit an existing pool - click and select the pool; edit values, add and delete grid rows as required (see above), and in the tab menu select Apply Changes.

To delete a pool - click and select the pool, in the tab menu select Delete, and click OK when prompted to confirm. The resources previously assigned to that pool are returned to the Unallocated pool.
