Quantum Zero Curve

The Quantum Zero Curve is a blended curve that consists of a Cash Curve, an optional Futures Strip, and a Swap Curve. These components of a Zero Curve are defined on the relevant Interest Rate page and/or the Zero Curve Definition page.

The Zero Curve routines supported by Quantum are suitable for building:

  • Curves like the LIBOR-based curve, where the input rates are drawn from cash, futures, and swap data
  • Analytics-compatible curves (see below)

CAUTION: You must ensure that there is only one tenor point for each applicable date on the curve, or invalid results will occur. For example, choose either the 1-Year Cash Rate or the 1-Year Swap Rate for the 1-Year tenor point.

The Quantum Zero Curve is constructed by first processing the Cash Curve, then (optionally) the Futures Strip, and finally the Swap Curve. In summary, the processing involves:

  • Converting the par rates into equivalent rates with a 365 days basis and annual compounding frequency
  • Blending the converted rates with those calculated in the previous step

Most of the rates contained in the Quantum Zero Curve are usually derived from the Swap section of the curve. The procedure produces a swap-based zero rate starting six months from the valuation date and recurring every six months until the maturity date of the longest dated swap par rate.

Processing the Cash Curve Section of the Zero Curve

  1. Adjust cash par rates to 365 days basis.
  2. Convert cash par rates to zero rate equivalents.
  3. Convert computed zero rates to discount factors.

NOTE: The Cash Curve used for the Zero Curve must contain at least one point of any maturity.

Processing the Optional Futures Strip Section of the Zero Curve

  1. Calculate futures yield from inverted yields.
  2. Adjust futures yields to 365 days basis. See Adjust cash par rates to 365 days basis.
  3. Adjust futures yields for compounding frequency.
  4. Interpolate the initial futures point
  5. Bootstrap the remainder of the futures strip.
  6. Merge the cash and futures strip.

NOTE: If a Futures Strip is used, the maturity date of the longest-dated futures contract must exceed two years.

To set the expiration dates for the futures contracts (and therefore the number of days for each rate), see Futures and ETO Instruments and Futures Final Trading Days.

For a description of the User Input Zero Curve and User Input Discount Factor, see Interest Rates.

Processing the Swap Curve Section of the Zero Curve

  1. Calculate annual swap rates based on the par rates.
  2. Interpolate the initial swap (6 month) point
  3. Calculate initial swap points either when using a futures strip or without a futures strip.
  4. Calculate the remainder of the swap-based zero rates.
  5. Merge the rates into the blended Zero Curve.

NOTE: The Zero Curve calculation for the swap curve uses the Quadratic Bootstrap model.

Analytics-compatible curves

Quantum's interest rates functionality provides for Analytics compatibility by providing the following Model types in Blended mode.

  • Cubic Spline
  • Linear Par
  • Quadratic Variable

However, when you select any of these model types, you must, in the Blend/Zero Definition table (on the Zero Curve Definition page):

  • specify only one CASH row,
  • specify one SWAPS row and/or one FUTURES row,
  • set the Begin column in the CASH row to Overnight,
  • set the Begin column in the SWAPS row (if one) to 1 YEAR or 2 YEARS, depending on the available rates, and
  • set the Begin column in the FUTURES row (if one) to the date from which you want Futures rates to be applied.

    NOTE:
  • The Days Convention setting within each Par Curve is only used with the new Model types listed above. If the Model type is, Already Zero or Quadratic Semi-Annual, then the currency's Days Convention is used.
  • The ACTUAL/365LE Days Convention is mapped to ACTUAL/ACTUAL in Analytics (which does not currently support ACTUAL/365LE). If you are reconciling to Analytics, there will be minor differences in Leap years around Feb 29.

TIP: If you use any of the Model types listed above, then for each Par Curve you specify within the Blend/Zero Definition table, set Days Offset to 0, 1, or 2 days.
