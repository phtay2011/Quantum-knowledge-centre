About Layout XML Data records (EFTs)

The Layout XML Data record for an EFT External Method:

  • is specified in the name attribute of the Layout element in the XTC Configuration XML Data record,
  • is optional and if omitted, the Stylesheet record must also be omitted from the XTC record,
  • defines the content and business logic for the EFT message,
  • has structured content that is customizable for handling future needs, and
  • contains XML code.

For SWIFT

SeedXMLLayout_EFT is provided.

For TARGET - 2020.3 onwards

From 2020.3, we stop using the TARGET2.txt file and include the addressee details in the EFT XML Framework. You can define an external method for TARGET EFT payments.

For more information about TARGET, read Details of TARGET Addressee.

For TARGET - prior to 2020.3

SeedXMLLayout_eft_target2 is provided.

Elements and attributes

These are common to both Confirmations and EFTs.

See Layout file elements and attributes.

Example Layout XML Data record (EFT)

The following example structure will format only MT103 message types.

NOTE: The Data Template XML Data record contains XML code covering more than a thousand data fields that can be used to build the EFT.

<Message id="XML EFT" name="XML Formatted EFT"
    mandatory="no" validationID="BW0002B">
<IncludeDocument href="MyIncludes.xml"/>
<GlobalAttributes>
  ...
</GlobalAttributes>
<!--=================================================================
<!-- The presence of the ExternalReference block will update the 
<!-- External Reference field in the Quantum database to the same
<!-- value that is output. That field is used when updating the
<!-- Confirmation match status and message statuses).
<!-- In the example here, you must also:
<!-- (a) Define an IncludeItem with id="ExternalReferenceEft" in
<!-- the Includes XML Data record, and...
<!-- (b) Specify <Include id="ExternalReferenceConfo"> in the
<!-- relevant Field block in the Layout record.
<!--=================================================================
<ExternalReference>
  <Section>
    <Field id="External Reference">
    <Include id="ExternalReferenceEft">
  <Section>
</ExternalReference>
<Header id="103_HB" name="Basic Header Block" mandatory="strict_conditional">
  <Condition ref="is103"/>
  ...
</Header>
<Detail id="103_TB" prefix="4" name="Text Block" mandatory="strict_conditional">
  <Condition ref="is103"/> 
  <Section>
    <Field id="103_20" name="Sender's Reference" prefix="20" mandatory="yes">
      <Include id="ExternalReferenceEft"/>
    </Field>
    ...
  <Section id="A" name="General Information" mandatory="yes"
     prefix="16R/16S" qualifier="GENL">
    <SubSection id="C1" name="Interest Details"
        mandatory="strict_conditional" prefix="57A" validationID="BW002">
...
    </SubSection>
    ...
  </Section>
...
</Message>
