Revaluation Method (FX Accounting Rule)

(Available only when Revalue deals is selected.)

This setting gives the valuation method for FX deal revaluation postings (and determining Book Value).

TIPS:

  • The name of the revaluation method suggests the order in which calculations are performed. Compare for example, FORWARD NPV to NPV SEPARATE.
  • For an explanation of the currency used for revaluation calculations and posting gain/loss in each case, see Revaluation currency.
  • The interest rate curve used to discount the resulting gain/loss on revaluation is defined using the Interest Rate and FX Rate Type for the rule.
  • The system always uses a mid-rate during revaluation calculations, which is derived by averaging the Buy and Sell rates for the required date.
  • See also: Calculations and Postings during Hedge Reclassification.

Data items

  • FORWARD - to use the Forward rate to revalue the deal (for the period from the revaluation As At Date to the deal Value Date), and if necessary, to translate the resulting gain/loss to base currency.
  • FORWARD NPV - to use the closing Forward rate to revalue the deal, and to calculate the NPV of the resulting gain/loss (from the deal Value Date to the Position Date) after the revaluation calculations have been changed into the specified Revaluation Currency.
  • IMPORTED - to use an externally derived market value and calculated book value to calculate profit/loss (your system must be set up to Import market values). See also Hedge Accounting below.
  • IMPORTED SEP - as IMPORTEDexcept that separate imported values are used for each leg of the FX deal. See also Hedge Accounting below.
  • NPV SEPARATE - to post the resulting gain/loss in the respective deal currencies for each cash flow of the deal, with no Spot rate translation. The Market Value is the forward cash flow discounted back (Net Present Valued) to the Position Date. The Book Value on the Initial revaluation date is, for the:
    • Dominant currency side - the forward cash flow of that side discounted back to Deal Date, and
    • Non Dominant currency side - the Book Value of the Dominant currency side converted to the Non Dominant currency at the FX Deal's Spot Rate.

    The Book Value on subsequent revaluation dates is as above adjusted for any postings to date on each side of the FX Deal.

    NPV SEPARATE ALT - as NPV SEPARATEexcept that the Book Value on the Initial revaluation date is, for each side, the forward cash flow of that side only. The Book Value on subsequent revaluation dates is the same, but adjusted for any postings to date on each side of the FX Deal.

  • NPV SPOT - to calculate the NPV of the forward cash flows at the revaluation As At Date, converting the resulting gain/loss to the specified Revaluation Currency using the Spot Rate for the As At date.
  • SPOT - to calculate a deal Spot amount to the closing Spot rate, with the revaluation excluding any forward component of the FX Deal.

    TIP: Use the SPOT method when Forward Points are being accrued (and are already taken into account).

Hedge Accounting

When an FX deal is revalued as part of a Hedge Relationship (HR), the system uses the revaluation method specified on the HR page's Linked Deals tab.

If you select a Revaluation method of IMPORTED or IMPORTED SEP for the Accounting Rule, that will be the default revaluation method when the FX deal is linked to a Hedge Relationship for which Designated Risk is set to OVERALL MARKET RISK.

To change the revaluation method used for the FX deal within the HR, change the Revaluation Method column setting for the FX deal row on the HR page's Linked Deals tab.

See FX Deals - HR Revaluation Methods.
