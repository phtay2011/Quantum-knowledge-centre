Bulk Replace and Insert for Deal Map items

You can use a spreadsheet application (for example, Excel) to create and maintain multiple Deal Map entries, in conjunction with the Map Items grid menu's BULK Insert and Replace Forward functions (on the Deal Map tab), as follows.

If you amend any Deal Map items that have already been Actioned, or add items that impact other items already Actioned:

  • you may have to manually reprocess those items. This is indicated by the Flow Lifecycle values, which you can display in a grid column on the Cash Flows tab, in Settlements and Confirmations results grids, and use as a filter in deal or flow queries or result views (for more details, see Flow Lifecycles, States, and Phases); also...
  • depending on whether or not Deal Map item changes have a financial effect, the changed deal will be badged as Customized - Major or Customized - Minor, respectively.

TIP:

  • For customizing single Deal Map items, see Customizing Single Deal Map Items.
  • You can also edit directly in the Bulk Replace and Bulk Insert dialog boxes.

To replace the current entries

  1. In the Map Items grid menu under BULK, select Replace Forward.

    The Bulk Replace dialog box is displayed, showing the column headings and all valid entry rows, with all those rows and columns selected, as in the following example of a Money Market deal.

  2. Press CTRL-C.

    The selection is copied to the clipboard.

  3. In your spreadsheet application:
    1. Open a new workbook.
    2. Press CTRL-V (or select the Paste function).

      The selection is pasted into the equivalent spreadsheet rows and columns.

    3. Add*, edit†, and/or delete entry rows using spreadsheet functions as required.

      CAUTION:

      * Set the Event Number of new rows to 0 (zero) to include them as new deal map items.

      † Retain the Event Number of an existing deal map item to ensure that Quantum does not treat it as a new Deal Map item.

      For lists of valid items per deal type, refer to the See also topics below.

    4. Select all rows and columns you want to take back to Quantum.
    5. Press CTRL-C (or select the Copy function).
  4. In the Bulk Replace dialog box, press CTRL-V.

    Your spreadsheet selection is copied into the Bulk Replace dialog box, replacing the current selection.

    CAUTION: Ensure that the date and number formats match those used for Quantum (for example: DD/MM/YYYY for dates and N,NNN.NN for amounts).

  5. Click Replace.

    The dialog box closes and the Map Items grid now shows the entries from the spreadsheet.

    Error messages will appear if any entries are incorrect or incomplete.

  6. Fix or delete incorrect entries as necessary.

To insert new entries

  1. In the Map Items grid menu under BULK, select Insert.

    The Bulk Insert dialog box is displayed, showing only the column headings, with all columns selected.

  2. Press CTRL-C.

    The selection is copied to the clipboard.

  3. In your spreadsheet application:
    1. Open a new workbook.
    2. Press CTRL-V (or select the Paste function).

      The selection is pasted into the equivalent columns.

    3. Add*, edit†, and/or delete entry rows using spreadsheet functions as required.

      CAUTION:

      * Set the Event Number of new rows to 0 (zero) to include them as new deal map items.

      † Retain the Event Number of an existing deal map item to ensure that Quantum does not treat it as a new Deal Map item.

      For lists of valid items per deal type, refer to the See also topics below.

    4. Select all rows and columns you want to take back to Quantum.
    5. Press CTRL-C (or select the Copy function).
  4. In the Bulk Insert dialog box, press CTRL-V.

    Your spreadsheet selection is copied into the Bulk Replace dialog box.

    CAUTION: Ensure that the date and number formats match those used for Quantum (for example: DD/MM/YYYY for dates and N,NNN.NN for amounts).

  5. Click Insert.

    The dialog box closes and the Map Items grid now shows the entries from the spreadsheet.

    Error messages will appear if any entries are incorrect or incomplete.

  6. Fix or delete incorrect entries as necessary.
