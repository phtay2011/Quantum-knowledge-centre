Query Security - Precedence and Inheritance

The order of precedence and inheritance of Access rights follows the Browse structure on the Query Security page and is summarized in the tree diagram below.

By default, lower-level objects inherit their access security settings from their parent object, which inherits from their parent, and so on, back up the following tree structure, EXCEPT THAT for Definition objects, any folder structure is ignored.

NOTE: If you change an object's the Access level for a User/Group, any inheritance is broken and the Inherited From setting indicates <Not Inherited> for that User/Group and object (because you have explicitly made an assignment at the object level).
