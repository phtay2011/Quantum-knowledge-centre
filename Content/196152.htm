Structured Address fields

You can now break down address fields in Quantum into structured address fields, allowing better support of ISO20022 messages. With structured addresses enabled, you can configure the addresses across the entire system so they list one or more of the ISO20022 address tags below.

The tags are static and align with ISO20022; whereas the captions can be defined by the user.

NOTE: The address fields in Rules > Counterparty Contact pages are not structured as they are not used in electronically formatted messages such as confirmations and EFT's.

The system saves the structured address details in addition to your existing unstructured address details, meaning that enabling structured addresses does not force you to update existing confirmations, EFT's or reports. If you want to use structured addresses across your whole site, enable structured addresses, configure new addresses as structured addresses, AND manually migrate migrate existing unstructured addresses.

Migrating unstructured addresses

CAUTION: Ensure that static data approval in data security is disabled while migrating unstructured addresses as not all unapproved static can be approved in bulk.

Use the utility Structured Address Migration, or create a Workflow Report page based on the utility, to suit the specific new address structure, and migrate each set of addresses.

In the utility you can update one or more of the addresses in bulk according to the logic defined in the workflow report. Select items that suit the proposed mapping criteria, then Save Structured Address.

You can also use the Quick Menu option named Edit Address. This menu option uses a Bulk Update action which allows you to modify the details of the structured address' individual lines.

The workflow report provided uses the action as a quick menu item and will update only a single address in context when selected. You can create a copy of the report and add an additional toolbar action which will then update selected items in bulk.
