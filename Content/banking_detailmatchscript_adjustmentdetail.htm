Adjustment Detail, Passes tab, Detail Match Script page

These options are available when, for the selected pass:

  • Pass Type is Match, Match Status is Matched or Probable, the Criteria grid contains an Amount entry with a tolerance amount or percentage, and an Adjust Using type is selected; OR
  • Pass Type is Bankflow Only and an Adjust Using type is selected.

Use this area to specify static data values* for the adjustments that will be automatically generated when the script pass runs. The available data fields vary depending on the selected adjustment type (in Adjust Using), as indicated below.

*IMPORTANT NOTE: The starred settings below (Ticket Number through Counterparty) will be superseded if the corresponding Set From Bank Flow Record options are selected. These apply only to Match and Bankflow Only passes with Adjust Using set to ACTUAL CASHFLOW or PROJECTED CASHFLOW.

The bank flow Ticket Number, Instrument, and Entity fields must be populated by the Bank Statement Import process, using Data Manipulation commands (for example, as supplied via Transaction Rule Set records).

*Ticket Number (not applicable for Bankflow Adjustments) - optional transaction ticket information - for example, a reference for interfacing to external systems.

*Instrument (not applicable for Bank Account Adjustments, optional for Bankflow Adjustments, mandatory otherwise) - select an instrument the transaction is based on. Only instruments defined for the adjustment type are available.

*Entity (not applicable for Bank Account Adjustments and Transfers) - the entity (legal party) on your side of the transaction.

*Counterparty (not applicable for Bankflow Adjustments and Account Transfers) - the counterparty you are transacting with. The drop-down list includes all transaction-level Counterparties with Category set to Deal external or Deal internal in Counterparty Setup.

Dealer (not applicable for Bank Account Adjustments and Bankflow Adjustments, mandatory otherwise) - select a dealer; defaults to the current user if they are set up as a Dealer in User Maintenance.

Strategy (Actual or Projected Cash Flows only) - optionally select, for the adjusting deal, a Strategy group.

G/L Code (Bank Account Adjustments only) - the General Ledger code for the adjustment entry.

Comments (not applicable for Bankflow Adjustments and Account Transfers) - optional text giving further information about the adjustment.

Deal Number (Bankflow Adjustments only) - enter a nominal deal number for the adjusting bank flow.

Nett Number (Bankflow Adjustments only) - where applicable, enter a Net Number for the adjusting bank flow.

Check Number (Bankflow Adjustments only) - enter an optional check number for the adjusting bank flow.

Transaction Type (Bankflow Adjustments only) - select the Transaction Type of the adjusting bank flow.

Extra Information (Bankflow Adjustments only) - optionally enter text giving further information about the adjustment.

Source Entity (Bank Account Transfers only) - the entity owning the account the amount is to be transferred from.

Destination Entity (Bank Account Transfers only) - the entity owning the account the amount is to be transferred to.
