Importing Bank Files

The Bank Import Service processes import files in several data formats using specific parsers and any additional instructions, as specified in the BkImport.ini configuration file. You can use custom and dynamic import file name formats if required.

Submit bank account transactions and balances (or external cash flows related to a bank account) for processing to the Bank Import Service manually or automatically, as below. The imported data is then available for viewing and using in Bank Reconciliation and CashXplorer.

TIP: While you can also import Bank Account balances from external files using the Import Balances page, that method is totally independent from the Bank Import Service and Bank Importer.

Prerequisites

Before you can import Bank files, ensure that:

  • the required .ini configuration file and folder structure exist on your Application Server,
  • the format of files is correct for the specified parser,
  • the Import bank statements option is selected on the Operational Settings page (under Active Services), and
  • you have the required Module Security permissions.

For installation-related settings, see also the AGBankImportService.config file details under Configuring Data Importation Services.

Importing files manually

Use this method to update Quantum with a single set of external data, for example, when converting from another system.

To import a single file

  1. Open the Quantum Bank File Import Status page.
  2. In the page menu, select Upload File.
  3. Select the required import file and the AppData destination path.

    TIP: See AppData Folder for more information on configuring the AppData location.

  4. Click OK to upload the file.

    CAUTION: You cannot stop the import once started.

When the import run finishes, the page displays details of any records that failed validation.

Importing files automatically

This method enables you to synchronize with an external program.

Place the files you want to import in the appropriate folder. See Configuring Import Folders, and start the Bank Import service, if not running.

The Bank Import service regularly checks your specified input folder for new import files, and imports any new records as and when it finds them.

Validating and reporting

The Bank Import Service works as if the values in the import were entered manually by a user into the appropriate Quantum page. For this reason the system:

  • validates before saving each imported record,
  • skips records that fail validation and reports these to the Error Log, and
  • reports all actions (successful or unsuccessful) on the Bank File Import Status page.
