Counterparty Instructions function (Stock Flow)

(Quick menu function on a stock flow item)

For a stock flow, the Counterparty ("their") delivery instructions are those concerning the stock account you are delivering Securities stock into or receiving stock from - that is, the Counterparty Stock account.

You can view and possibly change these instructions from the Settlements page results grid by clicking a linked value in the appropriate Stock grid row and selecting Counterparty Instructions from the Quick menu. A separate Counterparty Instructions window opens for the purpose, displaying details from the Counterparty Stock account record.

You can also access Counterparty Instructions from the Settlement (Approval) page in the Processing menu in the same way, but for viewing purposes only.

  • You cannot change Counterparty Instructions for linked flows (cash and stock) that have a Payment Sent status of YES (Deliver/Receive against Payment).
  • To change instruction data, you require Modify SSI permission for the Counterparty Delivery Instruction Change module (Module Security tab, User or User Group page).
  • To only view the instructions, the Inquire permission for the Counterparty Delivery Instruction Change module is sufficient.
  • Changing the instructions for any flow can change the Approval requirements, because Payment Instructions is one of the Matching Criteria for Settlement Approval Rules.
  • Other restrictions may apply, as for cash flows. See the Restrictions subsection under Changing Counterparty Instructions.

The default instructions are those defined:

  • for Stock Account Transfers - on the Custodian Details tab on the Counterparty Stock Account page, or
  • for other types of stock flow - according to the applicable Counterparty Stock Account Defaulting Rule.

Header area

Amount

Read-only; the currency and amount of the stock flow.

Beneficiary Name

Read only; the deal counterparty the stock is being delivered to, or received from.

Account Number

The Counterparty Stock account number. To change, select from the drop-down list of Counterparty Stock Accounts in the currency of the stock flow.

This list may or may not be limited to stock accounts for the deal counterparty, depending on the setting of the system-wide option Display bank accounts for all counterparties (under Bank Accounts on the Operational Settings page).

TIP: Select another Counterparty's stock account to allow for settlement to an intermediary stock account.

If you do change the Account Number:

  • a warning message alerts you that it may be necessary to select an alternative SWIFT MT Format Template in Our Instructions, and
  • if the instructions are for a Stock Account Transfer, your change will be reflected on the Settlement Details tab for that Stock Transfer transaction - either in the Source or Destination area, depending on whether the stock flow is incoming or outgoing, respectively.

SWIFT DSS

An optional unique identifier for the Counterparty's custodian; defaults to that defined for the default Counterparty Stock account (see Custodian Details below).

Account Name

Read-only; the Counterparty Stock account name.

Custodian Details

TIP: These details relate to the custodian holding the counterparty stock account.

Custodian

Read-only; the name of the custodian.

If specifying an Intermediary Bank, this field is mandatory. However, with the SWIFT payment method AND provided a BIC code is present, N/A is valid.

BIC

Read-only; the Bank Identification Code for the custodian.

SWIFT DSS

Read-only; the DSS (Data Source Scheme) code value that will be output in the stock flow message when:

  • the Payment Method for the corresponding cash flows is SWIFT,
  • the custodian stock account has a Unique Identifier value that applies for this code, and
  • the custodian stock account's specified SWIFT MT Format Template indicates that this DSS code and the Unique Identifier are to be output.

For example: ECLR for Euroclear.

Account Number

Read-only; the account number that the custodian has with the Intermediary.

Required for Counterparties who do not (or cannot) hold an account directly with the custodian (the Delivery Agent), and therefore must go though an Intermediary.

Bank Name

Read-only; the name of the bank where the specified custodian account is held.

Address

Read-only; the postal address of the custodian.

Payment Category

Mandatory; the EFT output option for the selected Counterparty and the applicable payment method. Select Standard Settlement Instruction or EFT Not Required to specify whether or not to pass the stock flow on for EFT Processing, respectively.

Overseas Bank

Not applicable to a stock flow.

Clearing Fields area

Clearing code values

Any Clearing codes specified for the Counterparty Stock Account record appear here as read-only values, for information purposes only, each labeled with the name of the Clearing code, as specified on the Custodian's Stock account.

Delivery Narrative

Read-only; as specified on the Counterparty Stock Account record.
