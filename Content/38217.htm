Create at least one Repo instrument for every Repo format you want to use.

Stock flows and SWIFT Messages for Repo Deals

The system will generate stock flows for both legs of a Repo deal when the underlying instrument is set up to include stock flows, and will output SWIFT Confirmation messages as follows, depending also on the settings in Confirmation Template Defaults (Print/E-mail mode and/or XML mode).

  • SWIFT MT541 or MT543 messages to respectively receive or deliver stock against payment will be output for Classic, Reverse, Buy & Sell Back, and Sell & Buy Back Repo deals (the cash flow details are included).

    In these cases (and MT540/MT542 below), one message will be output for the first leg on the Stock Face Value, and one for the second leg on the Stock Repay Face Value (in the opposite direction). By contrast, only one stock flow message is output for settlement of a Money Market Deal (on the Stock Face Value).

  • SWIFT MT540 or MT542 messages to respectively receive or deliver stock free of payment will be output for Securities Borrowing, Securities Lending, Fee Based Borrowing, and Fee Based Lending Repo deals (any associated cash payments/receipts are handled independently of the stock flows).
  • SWIFT MT518 messages are output for confirmation of the opening leg only for all types of Repo deal where the underlying Money Market Security has stock flows (and where XML Confirmation templates are used).

For details of how to format SWIFT MT540-543 message output, see About SWIFT MT Format Templates.

Repo Life Cycle

Position Query recognizes Repo deals entered and settled on the same day ("intra-day").

The status of a Repo deal position (whether it is open or matured) is determined from the Actioned Status of the deal's final cash flow (either the Buy Back or Sell Back component, whichever occurs later).

This assessment is made according to the following rules:

  • If neither leg of the Repo is actioned in Settlements, then the deal is included in the position. It is treated as a forward start deal.
  • If the first leg is actioned, then the Repo is included in the position. It is treated as a settled deal.
  • If the second leg is actioned, then the deal is considered to have matured.

Repo Limit Calculation

When the system calculates the risk associated with a Repo Deal, cash- or notional cash-based Repos are handled in the same way, apart from the settlement cash flows. Revaluations are based on the cash component and not the underlying security.
