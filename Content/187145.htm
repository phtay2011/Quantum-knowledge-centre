Changing or Deleting a Filled Request

At times you may need to:

  • change an existing filled Request to correct errors or to add additional details (Reset action), or
  • delete a Request filled in error, removing it from any further processing or reporting (Delete action).

In both cases, the Fill details are removed from the Request and the previously linked deal.

These actions are enabled for a fully approved and filled Request when you have the corresponding rights to the particular Request page in Module Security (Change or Delete rights, respectively).

Resetting a Filled Request

When you select Reset, the Reset Request as Filled dialog box is displayed, with OK and Cancel buttons and the following message:

The request has been filled. Resetting the request will
disconnect the deal from the request.

Do you want to continue?

If you click OK:

  • details of the fill (deal number and fill date) are removed from the Request and the previously linked deal;
  • if an FX Request and the Replication process is active - then, in Analytics, the link between the FX Request and the FX Deal will be cleared and no linkage will remain at the database level; and
  • further changes to the Request details are allowed.

Any subsequent changes will be subject to any applicable Approval Rule, but the Reset action itself will NOT trigger a CHANGE approval if a rule exists, as the clearing of the link details alone is not a financial change.

Deleting a Filled Request

When you select Delete, the Delete Request dialog box is displayed, with Delete and Cancel buttons, a Reason text box for entering relevant information, and the following message (where nnnn is the linked actual deal number):

The request has been filledby deal nnnn. Deleting the request will
disconnect the deal from the request.

Do you want to continue?

If you click Delete, this action will occur immediately (as with deals, whether subject to an Approval Rule* or not), and:

  • details of the fill (deal number and fill date) are removed from the Request and the previously linked deal,
  • the Request database record is marked as Deleted and Unfilled,
  • a Deleted badge will show on the Request,
  • any Reason text you entered is attached as a Note and a Notes badge will also show on the Request, and
  • if an FX Request and the Replication process is active - then, in Analytics, the link between the FX Request and the FX Deal will also be cleared and no linkage will remain at the database level.
