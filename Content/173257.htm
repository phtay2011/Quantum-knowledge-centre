Unit Fee - Defaults, Rules

Use the Unit Fee Defaulting Rule page to list current Fees Defaulting records, and to create and maintain them. Each Rule record specifies up to three different fees to be defaulted onto Unit deals/holdings. For example, you might have a Broker Fee, Exchange Fee, and Stamp Duty.

Fees will be defaulted onto Unit Deals where the following key values on the deal match those of the most specific Fees Defaulting record:

  • Instrument
  • Trading Style (Exchange Traded or Over the Counter)
  • Exchange (where Trading Style is Exchange Traded)
  • Counterparty (where Trading Style is Over the Counter)
  • Broker (where Trading Style is Exchange Traded)
  • Security ID
  • Deal Actions (Buy, Sell, and/or Exercise)

TIP: You can select ALL for most of these default settings.

Where more than one Rule record applies, the most specific will be used. This enables you to set up a defaulting hierarchy.

You can define up to three different fees in each Rule record. For each fee, you can specify:

  • Fee Code
  • PAY/RECEIVE (Pay or Receive the fee)
  • Type (Fixed, 1 Tier, 2 Tiers, or 3 Tiers)
  • Fee Timing Method (On Deal Date or On Settlement Date)
  • Fee Currency
  • Fee Counterparty for Settlement
  • If Type is Fixed:
    • Method (PER DEAL or PER UNIT), and
    • Fee Amount
  • If Type is 1 Tier:
    • The limiting Gross Settlement Amount (after which no fee applies), and
    • Up To Fee in %: the percentage of the Gross Settlement amount giving the fee for amounts up to that limit.
  • If Type is 2 Tiers:
    • The limiting Gross Settlement Amount between the two tiers,
    • Up To Fee in %: the percentage of the Gross Settlement amount giving the fee amount for the first tier, and
    • Over Fee in %: the percentage of the Gross Settlement amount giving the fee amount for the second tier.
  • If Type is 3 Tiers:
    • A first pair of Gross Settlement Amount and Up To Fee in % fields giving the limiting amount between the first two tiers and the fee percentage for the first tier,
    • A second Gross Settlement Amount field for the limiting amount between the second and third tiers,
    • A second Up To Fee in % field for the percentage of the Gross Settlement amount giving the fee amount for the second tier, and
    • Over Fee in %: the percentage of the Gross Settlement amount giving the fee amount for the third tier.
