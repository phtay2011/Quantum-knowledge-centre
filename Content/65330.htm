Cap/Floor Pricing Examples

Some adjustments are required when the formulas for standard caplets/floorlets and digital options are applied to the various replicating components of a barrier caplet and floorlet. These mainly relate to the relationship between the computed per unit and position values. Each value is scaled to be consistent with the following definitions.

Per Unit Value

This represents the value of the option assuming that the notional principal of the deal is $1.00. For the standard caplet or floorlet component, the per unit value OVpu is therefore calculated as:

OVpu = BLK(•) * ( DIP / DIY )

...where:

BLK(•) = value returned from the Black model

DIP = number of days in reset period

DIY = number of days in year

To be consistent with the definition of the barrier cap or floor, per unit values for the digital option components are computed as:

OVpu = DIG(•) * ( DIP / DIY ) * FVpu

...where:

DIG(•) = "raw" value returned from digital option model (see About the Examples below)

FVpu = the cash-or-nothing payment made if the option finishes in the money.

For barrier caps and floors, this amount will typically be the difference between the barrier rate and the strike rate (or vice versa). See the following examples.

Position Value

This represents the actual full value of the option, computed by scaling the per unit value by an appropriate figure based on the actual face value of the deal. For both the standard caplet / floorlet components and the digital options, position values OVpos are simply determined as:

OVpos = OVpu * FV

...where FV is the stipulated face value of the deal.

These same scaling conventions are also used in the calculation of the per unit and position greeks.

About the Examples

For the vanilla cap and floor components, raw value is computed directly from the appropriate version of the Black model, without any scaling for the face value of the deal. That is, the raw values for caplets (RVcap) and floorlets (RVfloor) are computed as:

RVcap = DF(0,ti+1) * [ Fi * N(d1) - Fx * N(d2) ]

RVfloor = DF(0,ti+1) * [ Rx * N(-d2) - Fi * N(-d1) ]

NOTE: These are simplified versions of the equations for the full value of caplets and floorlets - see Standard Cap/Floor Pricing Models.

For standard (European) digital options, the payoff coincides with the option maturity date and should therefore be discounted back from that date. However, for the digital option component of a barrier option to be consistent with the standard cap or floor component, the discounting should reflect that fact that the payoff is assumed to occur at the end of the rate-set period, not at the maturity date of the optlet.

The raw digital option value VCON, prior to any adjustment to account for the deal face value, is computed as:

VCON = exp(-rT*) • N(Æd)

...where:

r = the riskless interest rate with maturity T

T* = time (in years) to the end of the optlet's reset period

N(Æd) = probability that price is above/below strike price at maturity

N(•) = cumulative normal distribution function

Æ = 1 for a Call option, -1 for a Put option
