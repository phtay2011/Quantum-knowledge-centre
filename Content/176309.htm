Setup requirements for hedge accounting

Hedge Accounting functionality requires a specific license and system-wide Operational Setting, then individual G/L Owner Entity settings, General Ledger accounts, and Posting Rules, as follows.

TIP: If you are introducing the Hedge Accounting process to an existing system, see Upgrading to Hedge Accounting below.

  1. On the Licensing page (under Accounting), ensure that Hedge Accounting is selected.

    If it cannot be selected and you are a system administrator with full rights, contact your FIS Consultant.

  2. On the Operational Settings page (under Accounting), ensure that Enable Hedge Accounting is selected.
  3. On the Entity page for the required G/L Owner Entities*, ensure that the Enable Hedge Accounting option is selected and a valid Transition Date* is entered (on the Accounting tab).

    *TIPS: To use the Hedge Accounting process, the Entity must either be a G/L Owner, or it must post to the G/L of another Entity.

    The Transition Date is the date on which hedge accounting is to begin for the Entity.

  4. On the G/L Account Template page for the required Accounting Types:
    • Add the following Profit and Loss accounts, with the Exclude from consolidated G/L position reports and Disallow use in G/L journals options cleared in each case:
      • Hedge Reclassification (current)
      • Hedge Reclassification (amortizing)
      • OCI Reclassification (P&L)
      • Hedge Forward Point Reclassification
      • Spot to Spot Reclassification
      • Gain
      • Loss
      • Gain (amortizing) - if amortizing the MTM
      • Loss (amortizing) - if amortizing the MTM
    • Add the following Balance Sheet accounts, with the Exclude from consolidated G/L position reports and Disallow use in G/L journals options cleared in each case:
      • OCI Balance (current)
      • OCI Balance (amortizing)
      • OCI Reclassification (B/S)
      • Deferred tax
      • Deferred tax (amortizing)
      • Deferred tax (spot to spot)
      • Deferred tax (forward points)
      • Hedge Forward Point Balance
      • Spot to Spot Balance
      • Revaluation Gain
      • Revaluation Loss
      • Revaluation Gain (amortizing) - if amortizing the MTM
      • Revaluation Loss (amortizing) - if amortizing the MTM
  5. On the Posting Rules page for Accounting Types inherited by the G/L Owner Entities:
    • On the Hedge Relationships tab, add or edit posting rules as required for the above accounts, ensuring the following settings on the BALANCE SHEET tab in the Posting Rules dialog box:
      • set Revaluation Gain to No Accounts, and
      • set Revaluation Loss to No Accounts.
  6. On the Posting Rules page for the G/L Owner Entities, add or edit posting rules as required for the above accounts, by either:
    • specifying one of the Accounting Types set up in Step 3 to be inherited, or...
    • ensuring the same settings as above on the BALANCE SHEET tab in the Posting Rules dialog box.

Futures Contracts

The Designated Risk associated with the contract for hedge accounting purposes depends on the type of Future. It is set to either:

  • Exchange Rate Risk - for a Commodity or Currency contract, or...
  • Interest Rate Risk - for an Interest Rate contract.

Upgrading to Quantum Hedge Accounting

When you first introduce Hedge Accounting functionality, part of the upgrade will include changing open deals to the new Accounting Rule settings. For details of how to select deals that mature after the start of hedge accounting, and retain before and after calculations, see Reapplying Cashflows and G/L Entries.

Exposures not previously reflected on the Balance Sheet may need to be entered. For example, new instruments and deals may be required for projected sales or purchases. See Actual Cash Flows in Hedge Accounting and Projected Cashflows in Hedge Accounting.
