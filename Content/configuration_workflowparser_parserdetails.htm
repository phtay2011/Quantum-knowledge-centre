Parser tab, Workflow Parser page

Use this tab to define and maintain the data columns in the input to be parsed.

For a new Parser record, the tab initially displays a Payload area as no columns have yet been defined.

To define the columns, click Generate Columns to open a dialog box. In the text box, enter or paste correctly formatted data using the specified Input Format; then click Generate. The dialog box closes and the Columns grid is now displayed.

To add a new input, click Add and enter the items listed below. To edit an existing input, select the row and select Edit in the grid menu, then change as required. To delete an input, click the x at the right of the row.

  • Name - a unique descriptor for the column.
  • Type - select Text, Date, Yes/No, Amount, or Number.
  • Default Value - a default value of the correct type. If you don’t set a default value, the system will set a null default (blank, zero, or No).

Use the Calculated Columns grid to define any new additional columns.

  • Name – a unique descriptor for the calculated column.
  • Type – select Text, Date, Yes/No, Amount, or Number.
  • Expression - use expressions to add more complex logic. For more details search for Expression Builder in Knowledge Center.

NOTE: Calculated column name must be unique. Column names and calculated column names cannot be the same.
