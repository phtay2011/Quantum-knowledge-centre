Configuring Table Views

When creating a report, use the Layout tab and Properties tab to change the results grid view. Click Apply (or Save) to see the effect of your changes.

A Table view presents the report in a scrollable grid.

Procedure Summaries

To configure a Table view

  1. Do one of the following:
    • for an existing Table view - select the name of the Table view in the name lookup and click the Configure link, or
    • for a new view - in the grid menu or the Configure View menu, select New Table.
  2. On the Layout tab's Columns lookup, select the data items for the columns you want to show in the view, in their display order.
  3. In the Sort lookup, select one or more data items to sort on, in order of priority. Each item's sort sequence is indicated ( for ascending, for descending). To reverse an item's sort sequence, click the item's button.

    To change the priority of your sort items, use drag-and-drop. To remove an item, click its x button or clear it in the Browse list.

  4. On the Filters tab, optionally add one or more filters as required. See Filtering the View's Displayed Results.
  5. (Optional for Quantum and Adaptiv only) On the Columns tab, you can change the default displayed names for columns in the view, and specify additional calculated columns. See Building an Expression (in Calculated Columns).
    • The fields in the Query Columns area are those selected on the Layout tab.
    • The fields available for selection in Calculated Columns expressions (and on the Layout tab) depend on the type of report being created.
  6. On the Colors tab, you can:
    • select the Columns you want to add background color to or select ALL for the whole row,
    • select the Background Color,
    • optionally add Conditions under which the color displays, where the condition uses any field or related field from the report, and/or
    • optionally add a Description that will appear when the user hovers over the cell.
  7. (Optional) On the Profile tab, you can summarize the report's result set into time buckets (periods) and display the total of all the amounts that occur in each time period. See Summarizing the Result Set Into Time Segments.
  8. On the Properties tab > Rounding area, select from the following:
    • Columns - the columns the rounding will apply to.
    • Rounding Method - type of rounding, based on the next digit after the least-significant displayed digit
      • 5/4 - round up when the next digit is 5 or more. Round down otherwise.
      • Up - always round up
      • Down - always round down (truncate)
    • Round Amounts - round the number to... Whole Amounts, Thousands, Millions, Billions, Embedded. Appears next to the grid heading. Select Embedded if you want to show embedded units, "k", "m", "b" etc , beside the result.
    • Decimals - set the number of digits to appear after the decimal point. Applies to all displayed currencies. The default is 0 (zero).

      TIP: Run the query to see the results with the rounding and decimals shown. Export the results and they appear with the rounding and decimals you set.

  9. (Optional) On the Properties tab, set the required Access Permission for the view, and click Save. See Saving a View Configuration.
  10. Click Apply to see the effects of your changes on the current results, if any.

To set up rounding with embedded units for a Pivot view or Table view

  1. In the grid menu, select New Pivot or New Table and then choose the Properties tab.
  2. Select the Column to round, the Rounding Method, and then in Round Amounts select Embedded.
  3. Choose the number of decimal places, remembering that percentages are presented as decimals.
  4. The export to Excel uses the raw numbers, but other exports use what is in the view.
