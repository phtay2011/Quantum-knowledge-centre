Hedge events

We cover the following types of hedge events.

System-defined events

During the life of a hedge relationship (HR), Quantum creates at least two basic events:

  • Designation (DESIGNATE HR on Deal Map) - the linking of a deal to a hedge relationship, done using the Linked Deals tab on the Hedge Relationship page.
  • Dedesignation (DEDESIGNATE HR on Deal Map) - the removal of a deal from a hedge relationship when that deal has been included in a position processing revaluation.

    Dedesignation creates a calculation point that the Position query uses for the next revaluation.

Other examples of system-defined events include TRANSITION and LINK DEAL ACTION.

Customized events

To deal with more complex situations, you can add customized events to a hedge relationship. The defaults for these customized events can be set up on the Events tab (Hedge Strategy page), and then can be modified for a specific hedge relationship on the Events tab of the hedge relationship deal itself. Setting up an event means identifying the event then specifying when it is to occur.

NOTE: The Events tab settings control when amounts are recognized in Profit and Loss, a significant feature of Hedge Accounting. To enable the use of generic rules, the timing can be calculated as an offset relative to an offset start of INCEPTION or MATURITY, or entered manually if ACTUAL DATE is selected.

  • Freeze until next user action - Select this option to halt Hedge Accounting following a user-defined event. No further entries will be posted, but nothing will be reversed unless the Hedge Relationship matures. Use the associated Freeze From and Freeze Date fields to specify the default timing for the freeze.
  • Set release date - Select this option and use the associated Release From and Release Date fields to specify the default timing of the release (and release transfer, if applicable).
  • Transfer to another entity at release date (available when Set release date is selected) - Select this option to transfer adjustment amounts to another entity following a user-defined event. Use the associated Transfer to entity and Release Date of Transfer fields to specify the entity and default timing of the transfer (the default timing of the release transfer is as specified for the release).
