XML Template Files for Other External Methods

In some cases - especially with SWIFT-style output - you may be able to use the supplied EFT Plug-In and modify the Seed XML Template files provided for SWIFT to create new sets of seed files for other types of EFT (other External Methods).

NOTE: Non-SWIFT-like and more complex EFTs will usually require new, specific plug ins and sets of XML Template files. First contact your FIS representative to check if XML Templates may be available for that Payment Method in a later release than yours, or are planned for an upcoming release.

To modify EFT Template Files for a new External Method

  1. Ensure that you:
    • contact your Quantum representative if you need any assistance,
    • always retain copies of the original Seed XML Template files supplied,
    • have the required expertise in maintaining XML code, and
    • co-ordinate your "live" testing plans and cut-over times with the external message service.
  2. Take copies of the current XML Framework Seed XML Template Configuration and Layout file for EFTs (in the AdvicePath folder configured for your site), and rename them for your new External Method.
  3. Edit the XML Layout file for the new External Method as required, and save it in the AdvicePath folder.

    CAUTION: NEVER edit a Data Template file.

  4. Edit the Layout element in the XML Template Configuration file for your new External Method to indicate the Layout file name and/or location, and save the Configuration file.
  5. Perform all other setup tasks for the new External Method, as required. See EFT Setup - Configuration Pages and EFT Setup - Static Data and Rules.
  6. (Optional) Contact your FIS Consultant for assistance in testing your proposed changes.
  7. Before fully implementing your new template files, test them thoroughly with "dummy" deal and settlement data.
