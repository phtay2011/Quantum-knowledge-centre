Closing G/L Periods - Key Points

CAUTION: All users should exit from posting application pages (such as deal entry and processing) before G/L Close is run. This is because such Quantum applications check for newly closed periods only when they start. If they are still running when a period is closed off, they remain able to post back-dated entries into the closed period until they are re-started.

General Ledger Close has two levels of close action:

  • Interim close
  • Full close

Closing a period does not prevent you from accessing it for reporting or query purposes.

Key Points

  • You do not need to perform a G/L Close after every accounting period.
  • If you select the latest of multiple open periods, all of those periods (including the selected one) will be closed.
  • G/Ls must be closed in date-sequential order.
  • A G/L close should not run across a financial year end if Year End postings are required The Year End Accounting Procedure closes Income Statement accounts to Retained Earnings, and performs a General Ledger Close after all entries have been posted.
  • For a Full close, the base currency equivalent amounts are calculated for each G/L entry. See Setting G/L Base Amounts.This is optional for an Interim close.
  • If under G/L Owner, you select ALL, the most recent G/L Full Close date and G/L Interim Close date is displayed.
  • If you select multiple G/L Owners or ALL, your selected action will be performed only for G/L Owners meeting the necessary condition in each case:

    Action

    Performed for G/L Owners only where...

    Full close

    Last Full Close date < New Close date for run

    Interim close

    Last Interim Close date < New Close date for run

    Full re-open

    Last Full Close date > New Close date for run

    Interim re-open

    Last Interim Close date > New Close date for run
    and
    Last Full Close date <= New Close date for run

  • You can optionally suppress Year End posting and/or account revaluation warning messages for Full Close or Interim Close runs. See Suppressing Warning Messages.
  • A closed period can later be re-opened by selecting the Full re-open or Interim re-open actions, as applicable. For more details, see Re-opening Closed Periods and these option descriptions under Completing the fields.
  • You can save and re-use run parameters. See Save and Re-use Run Parameters.

Interim Close

Select this action to close the ledger(s), allowing only those users that have the appropriate security permissions to post back into the interim closed period. This permission is set for individual users - see Adding Users.

  • If you do not have permission to post into an interim closed period, any backdated G/L entries are posted on the first calendar date of the open period.
  • If you do have permission to post into an interim closed period:
    • Any backdated G/L entries are posted back into the interim closed period.
    • Where backdated G/L entries are prior to the last full close, they are posted on the first calendar date after the last full close.

Example

In this example:

  • A G/L transaction dated 04-Feb-12 would be posted on:
    • 04-Feb-12 if you have rights to post into an interim close, but on...
    • 01-Mar-12 if you do not have these rights.

      NOTE: G/L entry is prior to the last interim close but after the last full close.

  • A G/L transaction dated 16-Jan-12 would be posted on:
    • 01-Feb-12 if you have rights to post into an interim close, but on...
    • 01-Mar-12 if you do not have these rights.

      NOTE: G/L entry is prior to the last interim close and prior to the last full close.

Full Close

This action closes the selected ledger(s), allowing no further transactions to be posted back into the ledger prior to the close date.

When a transaction is backdated into the closed period, the G/L posting is created on the first calendar date of the open period.

TIP: Where the New close date is after the last Interim close date, the Last interim close date is also updated to that new close date.
