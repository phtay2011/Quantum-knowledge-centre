Importing Counterparty Delivery Instructions

The Counterparty Delivery Instructions Importer supports the following QID-format fields.

Header fields

Field

Description

Mandatory

Values/Default

mode

Account Mode

No

BANK/STOCK (defaults to BANK)

cparty

Counterparty

Yes

Name as displayed in Counterparty Setup (may be ALL)

ccy

Currency

Yes

Must be a Currency (ALL is not valid)

acc_no

Account Number

Yes

Not validated by lookup

acc_name

Account Name

Yes

 

iban_acc_format

IBAN Account Format

No

Bank Mode only

cp_swift_data_
source_scheme

Counterparty SWIFT Data Source Scheme

No

Stock Mode only

cp_swift_unique_
ident

Counterparty SWIFT Unique ID

No

Stock Mode only

Examples

NOTE: When using these format examples, separate each line of input with a line break. Separate by pressing Enter at the end of each line.

Insert a new record (QID format)

EXENAME=CPDELACC
$DoMenu=mfnew
mode=BANK
cparty=JKL Singapore
ccy=AUD
acc_no=123456
acc_name=ABN Account
iban_acc_format=SG123456789012
bank_name=JKL FolksBank
vostro_int_bank_acc=78901
post_add=12TH Street|Downtown Avenue|USA
sort_code=AB-78-90
bic_code=JKLAUSD19
charges=BEN
payment_category=Standard Settlement Instruction
overseas_b=N
clear01=6524
instruct=Delivery Instruction
add_replace=A
banktobank=Bank to Bank Instruction
int_name=Intemediary Name
int_addr=345 Seventh Ave|Middletown|Zirconia
int_sort=INT-87-98
int_bic=INTBIC001
i_clear01=7412
$DoMenu=mfinsert
