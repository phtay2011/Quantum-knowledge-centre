Deal Details tab fields, Imported Hedge Relationships

The Hedge Relationships Deal Importer supports the following QID-format fields from the Deal Details tab (Hedge Relationships).

A setting of Y indicates that the corresponding check box in Hedge Relationships would be selected (on). N indicates that it would be cleared (off).

Field code

Field name

Example and Notes

deal_no

Deal Number

deal_no=129

A unique identifier assigned sequentially by Quantum when the deal is saved.

Validation occurs - for example, you cannot import a Deal Number that already exists in Quantum.

ticket_no

Ticket Number

ticket_no=B551115

Up to 20 characters; may be used as a reference for other systems, for example.

sectype

 

sectype=HR FAIR VALUE

The Hedge Strategy you specify here must already exist in Quantum. It supplies defaults and grouping for the Hedge Relationship you are importing.

You may modify some of these settings individually during importing.

entity

Entity

entity=FIS UK

Specify an Entity that is set up to perform Hedge Accounting.

deal_dt

Deal Date

deal_dt=15-Nov-2000

Specify the date on which the Hedge Relationship is transacted. The default is the input date (today's date).

This date is for your information only; the system does not consider it when performing a Revaluation Analysis or Position Inquiry. The date actually used for Hedge Accounting processing is the Inception Date, which is derived from the underlying (linked) deals.

mature_dt

Maturity Date

mature_dt=29-Oct-2001

Enter a date that can be used to collate sales or purchases by a certain date, for example.

This date is for your information only; the system does not consider it when performing a Revaluation Analysis or Position Inquiry. The date actually used for Hedge Accounting processing is the Derived Maturity Date, which is derived from the underlying (linked) deals.

hedgetype

Hedge Type

hedgetype=CASHFLOW

Defaults from the specified Hedge Strategy, enabling the system to process like items in a consistent way. Changing it here will change the way the Hedge Relationship is processed and accounted for.

Specify FAIR VALUE or CASHFLOW.

testing_posting_ccy

Testing And Posting Currency

testing_posting_ccy=USD

Specify a currency to calculate effectiveness and post standard and non standard hedge relationship entries.

The currency must be either the base currency of the Hedge Relationship Entity, or a currency present in any one of the underlying linked deals.

deal_link

Disable Deal Linking

deal_link=N

Specify Y to stop the inadvertent addition of deals to the Hedge Relationship.

Deals can still be removed from the Hedge Relationship. This occurs automatically if an underlying deal is closed out, for example.

nat_risk

See Designated Risk settings below

 

hedge_ddes

Hedge Dedesignation

hedge_ddes=A

Options are:

  • A for AMORTIZED
  • D for DEFERRED

amort_fwdpts

Forward Point Amortization

amort_fwdpts=Y

If you have a Cash Flow hedge and the Designated Risk is Exchange Rate or Overall Market, you must specify how you want the system to manage forward points. If:

  • N - no Amortization, so no accruals occur; or...
  • Y - amortization of forward points occurs from the Freeze Date through to the Dedesignation of the hedge.

spot_to_spot_adj

Perform Spot to Spot Adjustment

spot_to_spot_adj=Y

Specify Y if you have an underlying exposure that is subject to re-measurement at Spot rates. This function specifies the creation of a spot adjustment when the exposure is added to the balance sheet, and when it is re-measured. The adjustment offsets currency translation postings.

NOTE: If you specify Y, ensure all FX deals within the Hedge Relationship have their Revaluation Method set to NPV SEPARATE. Hedge Relationships that utilize the Spot to Spot Adjustment process cannot utilize the Automatic Allocation function.

Autoalloc

Enable Auto Allocation when Changing Linked Deals

autoalloc=N

Y is recommended, to ensure that any changes to a deal that is part of this Hedge Relationship maintain Automatic Allocation.

For example, if a deal is unwound and you specify:

  • Y - The allocation percentages of all remaining deals within the Hedge Relationship are automatically adjusted to maintain the system face value ratio.
  • N - The allocation percentages of all remaining deals within the Hedge Relationship remain at their present values. You cannot manually initiate Automatic Allocation to re-allocate the deal percentages.
