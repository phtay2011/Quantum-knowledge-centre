Counterparties Participating in a Facility

Use the two grids on the Facility page's Counterparties tab to record Counterparties associated with the Facility, and to view historical changes made to this data, as follows.

COUNTERPARTIES grid

Use this grid to record each Counterparty associated with the Facility, the role each plays, and the value of their participation.

You can define more than one set of participating Counterparties, with one grid row per Counterparty and a common Active From date for each.

Grid Menu Functions

Use the following grid menu functions to create and maintain details of participating Counterparties.

  • New - to add a new set of participants (from scratch): in the Initialize At dialog box, enter the fields listed below for the Principal Counterparty; then click OK (to add further Counterparties, use Assign To).
  • Assign To - to assign part or all of the selected Counterparty's participation to a new Counterparty: - in the Assign To dialog box select the new Counterparty and enter either their Percentage of the total amount OR their Amount; enter other data as required; then click OK. Your new Counterparty is added as a new row and the other one's participation is reduced accordingly.
  • Increase By - to increase the selected Counterparty's participation - in the Increase By dialog box enter either the changed Percentage of the total amount OR the Amount; enter other data as required; then click OK. The Counterparty's participation is changed accordingly.
  • Decrease By - to decrease the selected Counterparty's participation - in the Decrease By dialog box enter either the changed Percentage of the total amount OR the Amount; enter other data as required; then click OK. The Counterparty's participation is changed accordingly.
  • Copy All For Date - to copy a selected set of participating Counterparties in full - in the Copy From dialog box, enter a laterNew Active From Date and click OK. Your new set is added as new grid rows. Use the other grid menu functions to change participations.

The grid menu also the standard grid view functions (New, Configure, and Copy).

Field Descriptions

The following data items can be displayed in grid columns and/or function dialog boxes. You can enter and edit item values only in the applicable dialog boxes.

Active From - the date from which the entry row data becomes active.

Total Limit Amount (read-only) - the maximum value of the Facility, sourced from the Limits tab.

Total Amt Assigned (read-only) - the value that has been assigned to Counterparties.

Total % Assigned (read-only) - the percentage of the maximum value that has been assigned to Counterparties; generally 100 but less when not all of the agreed Facility has yet been distributed.

Counterparty* - a Counterparty that has a 'role' within the facility.

Principal* - Y or N, indicating whether or not the Counterparty is the Principal Counterparty.

Role* (optional) - a specified role (if any) for the Counterparty in the Facility; one previously set up on the Counterparty Role page.

Net* - Y or N, indicating whether or not cash flows are to be netted for the Counterparty within this Facility. See About Facility Netting.

Percentage - the percentage of the Total Amt Assigned that is assigned to the Counterparty; may be zero if the Counterparty does not have a transactional role. The total of these values (over one Active From date) must be 100.

Amount - The value of the Counterparty's participation.

Taxes (optional) - one or more transactional taxes (for example, Withholding taxes) that apply to the Counterparty's participation in the Facility. For more details, see Tax Maintenance.

NOTE: In the Assign To dialog box, the first group of these settings applies to the Counterparty you are assigning from; the second group to the one you are assigning to.

MOVEMENTS grid

NOTE: You cannot change any of this historical data. Only the basic grid navigation and view functions are available (paging through, searching,sorting on columns, and configuring the view to filter rows, change column layout, and so on).

This grid lists details of previous additions and changes made affecting the COUNTERPARTIES grid data. By default, all changes are listed.

All available columns are listed below in default order (left to right). Details are given only for items that differ from those described under COUNTERPARTIES grid above.

  • Active From - the applicable Active From date for the entry row that was changed.
  • Total Limit Amount
  • Total Amt Assigned
  • Total % Assigned
  • Counterparty
  • Action - the user action causing the change (menu function), or NEW TOTAL for a change to the total (made on the Limits tab).
  • Second Counterparty (ASSIGN TO action only) - the Counterparty who received the assignment.
