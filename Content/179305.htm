Including/Mapping to other Configuration files

Configuration files can specify other Configuration files to be read (included) statically or dynamically, by using the IncludeFile or MAP CONFIG parameters, as described below.

IncludeFile parameter

Description

Specifies another Configuration file whose parameters are to be read (included) in Bank Importer runs using the Configuration file containing the IncludeFile parameter.

When the Importer encounters this parameter in the current Configuration file, it immediately looks for and reads in the specified Configuration file. After the included file has been read, the Importer then continues to read the remainder of the current Configuration file.

This parameter can be used to maintain configuration parameters common to all processing in one Configuration file. Configuration files that implement a range of different processing scenarios can then include this file to load common configuration parameters.

Syntax

IncludeFile = filepath

Where filepath can be any direct or indirect file specification. The path must specify an actual file name and cannot refer to a directory name.

NOTE: An indirect file specification is relative to the location of the Configuration file containing the IncludeFile command.

Examples

IncludeFile = .\Config\Common\AccountFilter.ini

IncludeFile = C:\MyReadOnlyArea\Config\MyRules.ini

IncludeFile = \\FileServer\Quantum\Importers\AdditionalRules.ini

NOTE:  Instead of using an IncludeFile parameter to provide Transaction Rules (for data manipulation), you can use a TransactionRuleSet parameter after creating rules in (or uploading them into) Bank Transaction Rule Set records in Quantum.

MAP CONFIG Parameter

Description

Specifies another Configuration file whose parameters are to be conditionally applied in the Bank Importer run, based on the specified message type and/or sender. This enables you to apply different transformation rules for different message types (and possibly different senders also), in one run.

When the Importer encounters this parameter in the current Configuration file, it immediately looks for and reads in the specified Configuration file, whose parameters will only be applied to incoming messages of the specified message type, and/or from the specified sender; and they will override any in the current Configuration file or any other included file.

After the mapped file has been read, the Importer then continues to read the remainder of the current Configuration file.

NOTES:

  • The mapped file cannot be used to change log on details or anything to do with the initialization process. It should be used only to provide transformation options and specific parsing settings for the different message types (and possibly senders).
  • For Message Type (mType) BAI only, the Map Config has been extended to allow separate rules to be specified for Prior Day and Intraday using a new mode (PRIORDAY/INTRADAY) indicator. See example 4 below.

Syntax

MAP CONFIG { filepath,mType [,sender] }

Where:

  • filepath can be any direct or indirect file specification. The path must specify an actual file name and cannot refer to a directory name.

    NOTE: An indirect file specification is relative to the location of the Configuration file containing the MAP CONFIG parameter.

  • mType is the message type - for example, BAI, MT900, MT910, and so on.
  • sender is optional and gives a sender's BIC code.

Example 1

MAP CONFIG {C:\Users\BankImport\Rules900.ini, MT900}
MAP CONFIG {C:\Users\BankImport\Rules910.ini, MT910}

The parameters in Rules900.ini will be applied to MT900 messages, and those in Rules910.ini to MT910 messages.

Example 2

MAP CONFIG {C:\Users\BankImport\RulesBAI6.ini, BAI,WFBIUS6S}
MAP CONFIG {C:\Users\BankImport\RulesBAI7.ini, BAI,WFBIUS7T}

The parameters in RulesBAI6.ini will be applied to BAI messages from the sender whose BIC code is WFBIUS6S, and those in RulesBAI7.ini to BAI messages from the sender whose BIC code is WFBIUS7T.

Example 3

IncludeFile=C:\Users\BankImport\BaseRules.ini
MAP CONFIG {C:\Users\BankImport\Rules900.ini, MT900}

Parameter settings in BaseRules.ini will be applied except for MT900 messages, when the settings in Rules900.ini take precedence.

Example 4

BAI - explicit mode parameter for both Prior-day and Intra-day with a Sender:

MAP CONFIG {C:\Users\Echos BAI Import\EchosPriordayRules.ini, BAI, PRIORDAY}
MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY}
MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI,<Sender>, INTRADAY}

In this example the EchosIntradayRules will firstly apply a rule to a specified Sender and then for all other Senders will pick up the C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY rule.
