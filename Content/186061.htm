Fields (Workflow page XML)

The <Fields> element holds all fields that are modified.

Field (Workflow page XML)

The Field element defines all modifications to a single field.

Attribute

Values

Description

id

field id

The id of the field that is customized.

TIP: Use your browser's Developer Tools to target field ids.

title

text

The custom title for the field. This will be displayed for:

  • Field caption.
  • Basic validation messages.
  • Selection dialog title (if the field is a reference).
  • Column title in the browse list (when field is in a View).

NOTE: Validation messages provided by the back-end business logic may still refer to the old title.

defaultValue

text

The value the field is set to when a new item is created. The instrument is set first.

  • For Date fields use a relative date (for example, Today) or an ISO formatted date (that is, YYYY-MM-DD).
  • For Number and Amount fields use only a period (.) as a decimal point, with no other formatting (for example, 1000000.50).
  • For multi-select fields, use a semicolon (;) to separate multiple values.

disabled

false*

true

When set to true, the field is disabled. A disabled field displays its contents as read-only.

hidden

false*

true

When set to true, the field is hidden.

  • When all fields and grids on a tab are hidden, the tab is hidden.
  • When all fields and grids in a named panel are hidden, the panel title is hidden.

optionTitles

value=text;

For radio button groups and drop-down fields, list value=title pairs for each option. Separate the pairs with a semicolon (;).

includedValues

text;...

The valid values for the field.

Any values that don't match this list are removed as options for the field.

  • Values can contain wildcard characters: asterisk (*) and period(.) to represent any character and one character respectively.
  • Separate the values with a semicolon (;).

excludedValues

text;...

The values that are not allowed for the field.

The values from this list are removed as options for the field.

  • Values can contain wildcard characters: asterisk (*) and period(.) to represent any character and one character respectively.
  • Separate the values with a semicolon (;).

* Default value.

Example

<Field>
<Field id="Instrument" title="FX Template" hidden="false" disabled="false" defaultValue="FX FOrward" includedValues="FX*" excludedValues="3rd*"/>
<Field id="otherCurrency" hidden="false" defaultValue="CHF"/>
<Field id="otherCurrency_otherFaceValue" title="FIS Amount"/>
<Field id="DealDate" title="Dated"/>
<Field id="Domain_BuySell" title="High/Low" optionTitles="1=High;2=Low"/>
</Fields>
