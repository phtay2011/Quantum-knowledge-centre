Areas, Workflow Page

This topic covers the different grids seen in the Workflow Page module and describes how to use them.

Workflow grid (Workflow Page)

The Workflow grid contains a list of actions you want to occur either at certain times or associated with other actions.

In the grid menu select New to open the New Actions dialog box and add a new page-related action. Select Edit to change an existing action or Delete to remove one.

To add a new action, or edit an existing one

Add new actions in the order you want them to occur.

  1. Open Configuration > Workflow > Page.
  2. In the menu select New or Edit and the New Actions or Edit Actions dialog box opens.
  3. In Action Name click the lookup and drill down to choose the action under the various folders for the type of action: Create, Delete, Open, and so on.

    4. TIP: WorkflowIn Data Transfer Location records will show as Transfer actions; and any composite workflow actions will also be available.

  4. Enter Name - the display name.
  5. Enter Action Timing. Select when the action is to occur: After New(only available when the Set Context action is selected), After Load (from browse list), Before Create (from New), After Create, Before Update (from Edit), After Update, Before Delete, After Delete or After Menu( actions with this timing are invoked after you choose them from the menu).
  6. Select the Add Prerequisite link (optional). This opens Expression Builder to add a Yes/No (Boolean) condition under which this action will / will not occur. Choose a single item with a Yes/No data type and the system looks for whether this item is present or not present. If present, then the action occurs. Choose one or more items of any data type and create an expression. The system looks for whether the expression is true or false. If true, the action occurs.
  7. In the Inputs grid (all actions), enter values for the action and any required Expression.
  8. In the Outputs grid ("Create" actions only), enter the value identifying the new data created by the action and its associated Column name, for reporting purposes (for example, Deal Number when a new deal is created).

To rollback changes if an action fails

If an action in the Workflow grid fails, the system automatically:

  • stops any subsequent action, to prevent saving invalid data;
  • rolls back all database changes in the transaction; and
  • stores a failure message in Reporting > Message Log.

Examples

To create an action that will update a cash flow deal

In this example we will create a variant FX deal page and, for deal number 1234 any change to the deal amount will set off an action to update the Actual Cash Flow (AC) deal number 299.

  1. Open Configuration > Workflow > Page.
  2. Add a new page. Go ahead and set up the XML area for the FX deal.
  3. In the Workflow grid, select New to create a new action. The New Action dialog box opens.
  4. Set Action Name to Update Actual Cash Flow.
  5. Set Action Timing to After Update.
  6. Do not click Add Prerequisite. No prerequisite condition is needed in this example.
  7. In the Inputs grid, click Add and in the Select Inputs browse window, select:
    • Amount - the field for which any change in value initiates the action;
    • Comments - the Comment field in the other deal; and
    • Deal Number - the number of the other deal to be actioned upon (299 in this example).
  8. Click OK to return to the New Actions dialog box.
  9. In the Inputs grid, click Deal Number to open the Edit Setting dialog box and click the Add Expression link. In Expression Builder, enter "299" (the deal you want to update on the action) and click Done to close the Expression Builder, then click OK to close the Edit Setting dialog box.
  10. In the Inputs grid, click Amount and in the Edit Setting dialog box, click the Add Expression link. In Expression Builder, click the + button, and in the Browse window select Context > FaceValue then type + "80". Click Done, then click OK.
  11. In the Inputs grid, click Comments and in the Edit Setting dialog box, click the Add Expression link. In Expression Builder, type "Update action came from ", click the + button and in the Browse window select Context > Deal Number. Click Done then OK.

    The Inputs grid now looks like this:

  12. Finish creating the Workflow FX Deal page. In this example, the FX deal number is 1234.
  13. Open the FX deal page and access deal 1234. Change the Sell Amount from 2000 to 3000. Click Save, which performs the update. The following then happens:

    On the Actual Cash Flow page when Deal Number 299 is accessed, the Amount is now 3080 and the Comments field contains the text Update action came from 1234.

  14. In the Deal Audit query, you can view such actions.

NOTE: In a sequence of actions, if one fails, the rest of the actions do not complete.

To use the page to add a new day to your holidays table

  1. Open Configuration > Page.
  2. Add a new page. Go ahead and set up the XML area as required.
  3. In the Action grid, select New and the New Actions dialog box appears.
  4. Set Action Name to Update > Save Holiday Action. You can choose from these inputs: Currency, Location, Holiday Name, Observable, Start Date, Expiry Date, Fixed Date, Based on, Day, Day Name, Month, Day and Month.
  5. In Name, enter the name the user will see.
  6. Set Action Timing.
  7. Select Add Prerequisite if required.
  8. Select OK and continue to create the page.

To add a row to a custom table

This applies to a custom table (also called a workflow table) created in "Quantum_20xx.x_Database_Installation_Instructions.pdf".

  1. Open Configuration > Page.
  2. Add a new page. Go ahead and set up the XML area as required.
  3. In the Action grid, select New and the New Actions dialog box appears.
  4. Set Action Name to Create > Create [custom table name]. An action is available for each custom table.
  5. In Name, enter the name the user will see.
  6. Set Action Timing.
  7. Select Add Prerequisite if required.
  8. Select OK and continue to create the page.

The action creates a row identified by the NK fields (or PK fields if no NK fields). If there are no NK fields, the PK fields are output settings. No action is available for tables without PK or NK fields.

NOTE: You can also use actions to update and delete rows in a custom table.

To add an action to a custom table

This applies to a custom table (also called a workflow table).

  1. Open Configuration > Page.
  2. In Core Module, select one of your custom tables.
  3. In the Workflow grid, select New and the New Actions dialog box appears.
  4. Select the action's Name and Action Timing, and add expressions and inputs / outputs.
  5. Select OK and continue to create the page.

The actions include Create, Delete and Update.

NOTE: To create a custom table (workflow table), see "Quantum_20xx.x_Database_Installation_Instructions.pdf".

To create a page that filters by using a custom table

Ways to use a page to filter results include: to filter results from the counterparty; and to filter an instrument field for IO types.

  1. Create and save a basic workflow page.
  2. Open the page.
  3. Choose the required custom table as the module.
  4. Add a source to the page, for example, add an instrument source to the page that has settings that restrict it to the Interest Rate Option (IRO).
  5. Set the source attribute on the instrument field to point to the instrument source.
  6. (Optionally) Add validation, if required.
  7. Save.

Workflow action that calls on a deal’s action

You can set up a workflow page or workflow report to call on actions that are already in a particular type of deal. For example, the actions Fill or Mark as Filled that are already associated with FX Request deals.

In the Select Action Name dialog box there is a new Process folder, with sub-folders for each transaction type and supported static data item, including Variant pages, and for any Workflow reports that have multi-select actions.

The actions available include, for example:

  • the Dealing > FX actions Drawdown, Predeliver or Extend, Rate Fix, Rollback
  • the Dealing > FX Request actions Fill and Mark as Filled
  • deal customization actions for those deal types currently supporting them (Cross Currency and Interest Rate Swaps, Money Market and Securities)
  • Create, Update, and Delete actions for some deal types and Static Data (Currency Options, Cross Currency and Interest Rate Swaps, Money Market and Security deals, Counterparty, Entity, and User)
  • Workflow Templates (reports only), enabling users to apply a template to the report results when output to file

    • NOTE: All existing templates are available in the Download folder and the file can be downloaded in a browser, in either the output format set for the template if Convert to PDF is No or PDF format if Convert to PDF is Yes.

  • Workflow Components and Workflow Component Problems (reports only), enabling users to report on problems with any Workflow page, report, template, action, function, parser or event
  • multi-select actions configured on Workflow reports (using a toolbar button), which enable you to run the report with specified input values passed into the filters, and execute the action on all results from the report

NOTE: A deal can have more than one action associated with it, and actions appear in the list even if they are disabled - for example, a disabled XML menu action such as Save still appears in the list.

To set up a workflow action that calls on a deal action

Set up an action in a workflow report or workflow page that calls on an action already in a deal. This uses the Process folder in the Select Action Name dialog box.

  1. Open Configuration > Page.
  2. Select Core or Variant and enter the initial details for the page.
  3. In the Workflow menu, select New.
  4. When adding an Action Name, in the Process folder select the name of the page in Quantum that you want to call actions from. This Process folder includes all page names for core and variant pages in the allowed deals, for example, FX deals or FX Request deals.
  5. Set up the input for this action.
  6. Complete the set up.

Similar instructions apply to Configuration > Report.

To create a deal that is partially populated

You might regularly use a workflow page to create a certain type of deal, for example your FX deals. Or maybe you import certain deals using JSON. From 2020.2, you can populate fields as you create the deal using either of these methods. For example:

  • If all of your FX deals have the same Counterparty, then you might design a specific workflow page to both create the FX deal and partially populate the deal with the Counterparty name at the same time.
  • Or you might populate fields with the result of an expression, for example the sum of related deals, or the result of calculating a particular rate.

Prior to 2020.2, this functionality was only available when you saved a deal.

  1. Open Configuration > Page.
  2. Select Core or Variant and enter the initial details for the page.
  3. In Core Module, select the deal type, in this example an FX Deal.
  4. In the Workflow menu, select New.
  5. In Action Name find the Update folder and select "Set Context".
  6. In Action Timing select After New.
  7. Set up the input for this action. For example, set the counterparty to always be "[SelectTheCounterparty]".
  8. Complete the set up for the page.

Each time you use the workflow page to create a new FX deal, the deal will already contain the values you set up.

Similar instructions apply to Configuration > Report.

Validations grid (Workflow Page)

In the grid menu, select New to add a new condition, Edit to change an existing one, Delete to remove one. Add validations in the order you want them to occur.

The New Validation or Edit Validation dialog box has the following items.

  • Condition - Click the Add Condition link to open Expression Builder so you can enter any validation conditions.
  • Message - The message text to be displayed when the validation fails.
  • Type - Select whether the condition will Block users from saving the deal, or display a Confirmation message.
  • On Create - Select if the validation is to occur when creating the deal.
  • On Update - Select if the validation is to occur when updating the deal.

Calculated Fields grid (Workflow Page)

Use the Calculated Fields grid to add read-only fields that store some automatically calculated values. Define calculated fields for tables created in the Enterprise Workflow (EW) and for deals.

In the grid menu, select New to add a new calculated field, Edit to change an existing one, Delete to remove one.

The New Calculated Field or Edit Calculated Field dialog box has the following items:

  • Title - The caption to display above the field.
  • Display As - The display format: Text, Date, Number, Deal Number, Amount, FX Rate, Interest Rate, or Yes/No.
  • Before / After - Select whether to display the calculated field before or after the Field selected here.
  • Field - the field the calculated field will be displayed before or after.
  • Expression - Click the Add Expression link to open Expression Builder, so you can enter an expression defining the calculated field.

The grid lists the values you enter for each calculated field. The system formats the resulting value in fields according to data type - for example, it uses the site's format for date and number.

If the displayed value is blank, the system has not been able to calculate the expression.

Sources grid (Workflow)

Use the Sources grid to add other data sources that can provide data for use in actions specified in the Workflow grid.

In the grid menu, select New to add a new source, Edit to change an existing one, Delete to remove one.

The New Source or Edit Source dialog box has the following items.

  • Source - The data source used for the page. This includes internal sources such as Quantum and Analytics deals, reports, and static data; deal and settlement approval rules (in the Reporting folder); and database views (in the Database folder, with the prefix QVW_DATASOURCE_%).
  • Name - A name for the source (defaults to the Source value); useful to shorten the default name or when the same data source is included twice for comparison.
  • Columns - Selected columns from the data source that you want to be available in your target page, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level.
  • Settings - Settings to apply to the data, including mathematical operators and expressions for calculated columns.
    • Settings tab and grid - Settings to apply to the source data, including a mathematical operator and expression.

      TIP: You will get an error message if you have not included a required (key) data item in the Settings.

    • Columns tab, Group By grid (only when Aggregated is selected) - select one or more fields to group by (or none).
  • Joined To - Available if the source is joined with another data source.

Examples

To add an external data source from a Source folder to an expression

  1. In the Sources grid menu, select New and add the external data source. The New Source dialog box is displayed.
  2. Select All Data. The Columns and Settings tabs are then displayed.
  3. Select the Source: for example, Static Data > Counterparty.
  4. Edit the default Name if required.
  5. On the Columns tab, click Add, select the data items to be included, and click OK. The Columns grid lists the selected items.
  6. To change a default Alias, click on the Alias value in the grid, edit the Alias field (in the Edit Column dialog box), then click OK.
  7. On the Settings tab, click Add, select the data items to be set, and click OK. The Settings grid lists the selected items.
  8. To set an item value, select its grid row and select Edit in the grid menu; select the Operator; click Add Expression and use Expression Builder to enter its associated expression, then click OK.

    Items in the Sources grid then appear in the Sources folder in the Browse dialog box in the Workflow grid.

  9. In the Workflow grid, add an Action Name, then click Add Prerequisite. The Expression Builder dialog box opens.
  10. Select "+". The Sources folder appears in the list with the external data source inside.
  11. Continue to create the action.

To aggregate data

The New Source dialog box has All Data and Aggregated radio buttons.

  1. Select Aggregated and the Columns tab shows both:
    • a Group By grid - enabling you to select one or more fields to group by (or none), and
    • the Columns grid – enabling you to select fields to aggregate by.

    The available Aggregate By functions depend on the Data Type (as in pivots):

  2. You can aggregate within a source, allowing Actions against aggregated data.
  3. The data source returns the data aggregated by the aggregation functions and grouped by the fields.
  4. Use the aggregated results in joins with other data sources - that is, use only the Group By fields and selected columns to define a join with an other data source.

Select a Source and two tabs appear (as for a custom report): Columns and Settings.

XML Syntax area (Workflow Page)

The Workflow page's XML editor provides word completion for elements defined in the XML schema.

TIP: Press CTRL+SPACEBAR to display a list of possibilities within the editor.

The following can be defined in the XML editor:

  • Add lookup values and conditional defaults through scenarios. You can define fields using defaultValue and includedValues / excludedValues attributes. You can also define conditions, and the conditions can be partially or fully based on analysis codes. When the condition is met, the system applies the field settings under that condition. If the same field is matched under different scenarios, the first match is chosen per attribute. Since the includedValues and excludedValues attributes are both about the lookup values, these attributes are considered one definition. When a field only specifies one of them, it will use the default value for the attribute - "*" for include and "" for exclude.
  • Change field titles on a page. Changes to field titles are reflected in any basic validation messages that refer to the field and the title of the lookup dialog box (for reference fields).
  • Hide or disable fields on a page.
  • Restrict the valid values for a field using separators (";") and standard wildcards (that is, you can use "*" and "." to represent any character and one character respectively).
  • Move the Facility field.
  • Hide the main strip of tabs on a page or change the way the elements in the tab are displayed in the page (for example, move a field to a different tab).
  • Rename menu options or hide them.
  • Set your chosen value as the default value for an analysis code field in Quantum. Each new deal then has this default value in place, and the default persists even when you change the instrument. You can lock the default against editing and the field appears without a box.

  • Use includedValue or excludedValue to filter a named analysis code, for example:

    <AnalysisCode name="Named Analysis Code" includedValue-"e">

    In the analysis code field called "Named Analysis Code", the Select dialog box will only show the options starting with "e" or "E".

  • For analysis codes on two-legged deals such as Swaps:
    • include from="AnalysisCodes" to apply the code to both legs,
    • include from="AnalysisCodesRight" to apply the code to the right side only, or
    • do not include the from attribute to apply the code to the left side only.
  • If you select a custom table in Core Module when setting up a Customization page, the Fields in the XML area are limited to these attributes: id, title. disabled, hidden, defaultValue, includedValues, excludedValues, and source.
  • When setting up a workflow page created in the Enterprise Workflow (EW), you can use analysis codes as part of the XML Syntax area. Use within <Scenarios> </ Scenarios>, either as a stand-alone condition or as part of a larger condition. This applies to most modules in the Dealing and Pricing menus. We'll use the release notes to let you know about new modules as we add scenarios functionality to them.

Example of an FX Deal

<Page>
	<Scenarios>
	</Scenarios>
	<Fields>
		<Field id="Instrument" defaultValue="AGF-FXOUTRT" disabled="true" />
		<Field id="Facility" adjacentFieldId="Counterparty_ExternalCounterpartyName" adjacentPosition="before" />
	</Fields>
	<AnalysisCodes>
		<AnalysisCode name="Additional Conditions" fieldId="ContractRate" defaultValue="Agreement 1" />
	</AnalysisCodes>
<Actions>
</Actions>
</Page>
