Currency Translation Processing

To fully account for the effects of Currency Translation on foreign currency transactions, the following three-stage process should be carried out at the end of each accounting period.

  1. Run an Accounting Accrual page to post accruals up to the end of the accounting period.
  2. Run an Accounting > Position page to post revaluations on all foreign exchange deals.
  3. Run Currency Translation processing from an Accounting > Currency Translation page, to calculate and post both unrealized and realized currency translation gains and losses for non-FX deals in a single process.

About Currency Translation

The value of the reporting (base) currency of a foreign currency transaction changes over time due to fluctuations in the foreign currency exchange (FX) rate. Currency Translation is an accounting process that revalues the Balance Sheet foreign currency accounts into the reporting (base) currency of the Ledger.

A gain or loss will arise where foreign currency items are posted to the ledger at a different FX rate from the FX rate on revaluation. The resulting G/L entries will be posted to reflect the gain or loss in the reporting (base) currency.

  • The Quantum Currency Translation and Revaluation processes for FX Deals provide the necessary accounting entries in the G/L to support accounting standards for Foreign Currency Translation such as SFAS52 and AASB1012.
  • General Ledger entries associated with a Hedge Relationship are always in the reporting (base) currency of the G/L Owner of the Entity for that Hedge Relationship. As a result, the entries are not affected by the Quantum Currency Translation process.
  • IMPORTANT: First run using the Preview button, so you can check the G/L entries prior to posting them as database changes. When you are satisfied, then run using the Update button.
