EFT and Confirmation message formatting - New XML Layout file attributes

In the XML Layout files for EFT and Confirmation message formatting, new attributes enable you to exclude hyphens, colons, or LineFeeds in message tags, where required, to ensure messages will be processed successfully without any errors, for example, via ECHOS.

Implementation

Field structure elements in XML Layout files now support the attributes listed in the following table.

NOTE: These attributes may be used only in conjunction with multiline="yes".

Attribute

Required

Default

Description

colon

No

preserve

Defines how colons are handled in multi-line text:

  • collapse* - removes all leading colons of each line
  • preserve - does not remove any colons
  • remove - removes all colons inside the text

hyphen

No

preserve

Defines how hyphens are handled in multi-line text:

  • collapse* - removes all leading hyphens of each line
  • preserve - does not remove any hyphens
  • remove - removes all hyphens inside the text

linefeed

No

preserve

Defines how line feeds are handled in multi-line text:

  • preserve - does not remove any user-entered line feeds
  • remove - removes all user-entered line feeds

*TIP: With the hyphen and colon attributes' collapse setting, removal is after field content has been split into lines (according to the maxlength setting).

Example

In this example we set the following:

  • linefeed="remove"
  • colon="collapse"
  • hyphen="collapse"

<Field prefix="70" multiLine="yes" maxLine="4" maxLength="35" linefeed="remove" colon="collapse" space="preserve" hyphen="collapse" xpath="/eft/cashflows/cashflow/adhocDeliveryInstructions/deliveryNarrative/narrative"/>
