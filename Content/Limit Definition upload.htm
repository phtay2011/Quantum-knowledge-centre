Uploading Limits Definition data

You can automatically upload Limits Definition data, using the Quantum Static Data Import Service. Under this method, when a Limits Definition csv file is placed in a specific folder, that file will automatically be imported into Analytics.

Prerequisites

  • Create a folder named AnalyticsImports in your site directory under StaticImportation\Import.

    • The full pathname will be AvantGard Quantum\Site\SiteName\StaticImportation\Import\AnalyticsImports, where SiteName is replaced by the name of your site.

  • On the Operational Settings page under Active Services, ensure the Import static data option is selected.
  • The Static Data Import Service must be running.

To import a Limit Definition file using this method

Place the Limits Definition csv file into the AnalyticsImports folder.

The csv file must:

  • in the first line, have Path as the first column name, followed by all the Limits Definition page's Limit Details grid column names, in the same sequence as in the grid;
  • in the second line's first value, have a pathname that ensures the uploaded data goes to the correct Limits Definition, in the format ~\Limits\LDname, where LDname is the Limits Definition record Name in Quantum;
  • in the second line's subsequent comma-separated values, have valid settings for the corresponding fields; and
  • in any (optional) further lines, have the first value empty, and the subsequent values with further valid settings for the corresponding fields.

Example file

The following graphic shows the first few columns of a csv file to be imported into the Limits Definition record named Limits_Test (as the file would be displayed in Microsoft Excel).

Each line of the file will be imported and merged into the grid. If there is an existing row with the same Matching Criteria* values, that row is replaced by the imported line values. If not, a new row is added.

*The Matching Criteria are the first group of values from Counterparty through Term End. The remaining values define the limit to be applied when result set records meet the Matching Criteria for the Limit entry. See Limit Detail and Matching Criteria .
