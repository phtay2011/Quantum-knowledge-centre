Address Structure

There are two choices for how to set up addresses within Quantum:

  • unstructured address fields
  • structured address fields

Unstructured address fields

Use unstructured address fields if you want a simple solution and you don't want to integrate with 3rd party systems. An unstructured address field is one single field within which you type unstructured text.

Structured address fields

Example: Structured address - Static > Counterparty Bank Account:

Use structured address fields instead of unstructured address fields on any page that has an Address field. Set up structured addresses so that Quantum integrates with 3rd party systems such as SWIFT.

Structured addresses are particularly useful on Static pages, including Counterparty Bank Account, Entity, and Bank Account pages.

If you decide to use structured address fields, any new address field you create from that time will be a structured address field.

To set up structured address fields

  1. Open Configuration > Address Structure and select Activate. This does not commit you to any course of action.

  2. Arrange the fields, as required.

    1. Decide the order of fields. The Order determines the order for display, with "1" at the top.
    2. Select the Tag. This tag is a recognized ISO20022 database tag. The tag column is Read Only.
    3. Edit the Caption, if required. This caption appears within an empty field.
  3. Save the page.

    Any new address fields will use structured addresses and not unstructured addresses. Current unstructured address fields remain as they are.

To import/export/update/delete structured addresses using QXT

You can import, export, update, and delete the static data that makes up structured addresses from Quantum using QXT format. If data contains both structured and unstructured addresses, the system displays an error message and does not save the data.

NOTE: From Release 2020.1, the Static Data Transfer Spreadsheet's Upload and Download functions include the structured address field details in addition to the unstructured address field details for the following static data:

  • Entity
  • Counterparty (Street Address and Postal Address)
  • Bank Account
  • Counterparty Bank Account (Postal Address and Intermediary Bank Address)
  • Counterparty Stock Account

To deactivate and delete structured addresses

  1. Open Configuration > Address Structure.
  2. In the page menu, select DeActivate.
  3. Click Save. A message appears:

    "When deactivating address structure, it is recommended that existing structured address values are deleted. This will not impact any other part of the record, including the unstructured addresses."

  4. Do one of the following:

    • Select Delete exiting record and click OK to delete all structured address values. This removes any structures address values stored in your database, but will not impact the unstructured address values.
    • Deselect Delete existing records and click OK, and the structured address values remain in your database, but are hidden by the user interface. You can recover these hidden structured address values by using a workflow report, along with all other tags relating to address structure.

The system stores a record of the action in the audit log.

To migrate an existing system to use structured address fields

NOTE: Quantum maintains the existing address format over upgrade.

To use your existing unstructured address structure, no change is required.

To change to using structured addresses, consider the following:

Once you select Activate, you must enter all new address fields as structured addresses.

You do not have to change all existing address fields to structured addresses, but if you do, plan carefully. Depending on the size of your database, this may be a large task. Create a report to list which addresses are structured or unstructured (see below), and plan from there.

To create a list of all addresses

Use one of these methods to find out whether your addresses are structured or unstructured. You'll then be able to see the format of existing unstructured addresses.

  1. If you enabled the Address Structure page with the default settings, open Utility > Structured Address Migration.
  2. Run the query and view the existing addresses in your site, both structured and unstructured.

OR

  1. If you enabled the Address Structure page and then edited any of the setup in the configuration page, this Structured Address Migration page will need to be updated.
  2. Open Configuration > Report and create a copy of the existing Structured Address Migration report.
  3. Update the columns/actions to reflect your address configuration.

To migrate existing unstructured addresses to become structured addresses

CAUTION: Ensure that static data approval in data security is disabled while migrating unstructured addresses as not all unapproved static can be approved in bulk.

  1. Review all proposed structured data before starting. Use the utility Structured Address Migration, or create a Workflow Report page in the Enterprise Workflow (EW) based on the utility to suit the specific new address structure.
  2. Based on the common structures of those (unstructured) addresses, choose how to map one set of existing unstructured addresses to become structured addresses. This is determined by the selection of inputs to the "Save Structured Address" action.
  3. Migrate the set of addresses, then repeat for the next set of addresses.

For more details, see Structured Address Migration.

Structured Addresses in Confirmation and EFT messages

From Release 2020.1, all address fields in the XML Data File used for Confirmations and EFTs include structured address details.

For more details, see Confirmations and EFTs under Structured Address enhancements in the 2020.1 Release Notes.

Upgrading from one Quantum version to another

Quantum maintains the existing address formats over upgrade.

All changes are maintained in an Audit query.
