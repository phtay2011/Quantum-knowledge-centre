Enable a Query for Limit Checking

To include an Analytics query in the Check Limits and Pre-Deal Limit processes, configure that query as follows, making sure you are aware of the Restrictions listed below.

  1. On the Properties tab, select a Limit Definition and select the Use in Limit Check option, as highlighted in the following example.

    TIP: The Use in Limit Check option appears only when a specific Limit Definition is specified in the Limits field.

  2. On the Assumptions tab under Options, ensure that the Include all limits and Security Limit Mapping settings are as required.
  3. On the Columns tab, ensure there is a Calculated column called Utilisation.
  4. On the Limits tab, optionally change the default Limit entries inherited from the Limit definition you specified in Step 1 - in particular, the Limit Utilization Method in each case.

    NOTE: If the Weighted Average method is used, the Columns tab must also contain a column called Weight.

  5. Save your changes.
  6. RECOMMENDED: Stop and restart the Limits Server.

Restrictions

When configuring an Analytics query for Limit Checking, note that the following restrictions will apply when the Limits Server runs the query:

  • no Database Columns should be used in defining the Utilization calculation;
  • the Calculating functions available for Calculated Columns are restricted and should be used with caution - for example, functions such as WAV will NOT be executed;
  • query Filters are not used - to enforce filters, configure them using the Limit Query Filter Override page;
  • Mapping Profiles should not be used; and
  • Time Profiles should not be used (if they are, it will not generate an explicit error message, but the Utilization calculation will not be correct).

TIP: These restrictions apply because the Limits Server processes deals in memory, while standard query runs process deals in the database.
