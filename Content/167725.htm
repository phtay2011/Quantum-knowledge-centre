How CashXplorer Works

CashXplorer:

  • uses the Bank Import Service to import bank data and external data;
  • uses Bank Reconciliation to reconcile the confirmed (Prior-day) account balances;
  • uses Intraday Matching to provisionally match unconfirmed imported data for the current day;
  • integrates imported and reconciled (or Intraday-matched) balance and float data with:
    • other treasury activities available in Quantum, including forecasts, actual transactions, debt and investment activity, and FX transactions, and
    • other imported cash flow data (external cash flows) from non treasury-related sources;
  • retrieves, combines and stores the data according to the applicable Query Configuration;
  • automatically calculates projected end-of-day or end-of-period positions for all defined Horizons (time periods over which cash flows are forecast), and anticipated future cash concentration flows; and
  • stores the collated and generated results of each Query run as Data Versions that can be viewed, refreshed, or deleted at a later time.

Data Versions

A Data Version is a full set of results from a particular CashXplorer query run at a certain date and time. These results give a collection of cash flow data and balances, subject to the configured Query settings. Different Views can be applied to a Data Version to give subsets, summaries, and reformatted displays of the same Data Version result data (without having to re-run the query).

Each Data Version's history is recorded, so you can view the status of all data retrieval activity on that Data Version (on the View configuration's History tab). This includes when the Data Version was first created and any time the Data Version had an Incremental Data Refresh.

Provided you have the correct security access rights, you can delete a Data Version (or an entire query) at any time. (You can also use the History tab to have old or duplicate Data Versions deleted automatically.)

Notifications

Notifications are produced for the following CashXplorer processes:

  • Run Query,
  • Refresh Version, and
  • Delete Data Version.

For more details, see CashXplorer Notifications.

Views

User-defined Views are used to display, summarize, and/or filter the Data Versions in different ways. Each Query can have many Data Versions and many Views. A View is a collection of user-defined parameters describing how a Data Version will be displayed.

Each Query must have at least one View of each type. New Queries therefore have a Default View of each type (these show the full result set in each case).

The View heading's drop-down list shows all the views available for displaying the current query results (Data Version).

See also: Configuring CashXplorer Views
