Reporting View Enhancements

(First introduced in Release SP 6.7.2, December 2018.)

The following enhancements apply to all Analytics, Reporting, and Processing modules based on queries with configurable results views, including (but not limited to):

  • Settlements Processing,
  • Confirmations Processing,
  • EFTs Processing and Reprocessing, and
  • Deal, Settlement, and Static Data Approval Processing.
Background cell colors

In report Result Views, you can now define and apply background colors, subject to optional conditions, to:

  • Table view and Pivot view cells, and
  • Pivot view total columns and rows.

NOTE: The same color palette now also applies to Chart views.

Implementation

A Colors tab now appears in the Configure View panel when you are configuring a Table or Pivot view.

To apply background colors to a Table view:

  1. On the Colors tab, click Add.
  2. In the Columns lookup, select one or more view columns, or select All to color a row.
  3. (Optional) To apply a condition to the coloring, click the Add expression link and use the Expression Builder + button to select fields, operators, and formulas as required to specify the condition.
  4. Repeat Steps 1 to 4 for additional coloring as required.

    NOTE:Coloring is applied in the order specified (see Example 1 below).

  5. (Optional) To change the order in which coloring is applied, drag and drop rows as required (using the grab handles ).
  6. Click Apply.

NOTE: The procedure is similar for Pivot views, except that you can also select any Totals rows.

To export the pivot, including colors

  1. In the main page menu, select Export.
  2. Choose an export format for the report.

    If using Excel or PDF then the colors are exported too.

Custom Report Actions on Pivot views

In report result Pivot views, you can now define and apply custom actions.

Implementation

An Actions tab now appears in the Configure View panel when you are configuring a Pivot view. The actions you can select are as for a custom report - namely:

  • Commodity Swap Price Fix,
  • Create deal (Actual Cash Flow, FX, Money Market, Projected Cash Flow, or Security),
  • Open Dealing application (as above plus Commodity Swap),
  • Open FX Request or Money Market Request, or...
  • Open Analytics Deal Analyse.

When specifying an action, you can optionally change the displayed name, and specify default Settings values that will be populate specified fields in the relevant application.

When you apply your pivot view, each cell with a non-empty value will contain a link with a context menu that shows your configured action names. Clicking on a specific action will invoke the configured action, in the context of the given cell, which includes:

  • cell value,
  • sibling values,
  • rows,
  • columns, and
  • row and column filters.

On completion of the action, a status message will display indicating its success or failure, and the report results will be updated if the action affected them.

To apply custom actions to a Pivot view:

  1. In the Actions tab's grid menu, select New.
  2. In the New Action dialog box:
    1. Use the Action Name browse list to locate and select the action you want,
    2. Optionally edit the Name you want to appear in the context menu,
    3. (Optional) Click Add and in the dialog box select one or more view data items.
    4. (Optional) In the Settings grid, click a setting value and in the dialog box click the Add expression link and use the Expression Builder + button to select fields, operators, and formulas as required to specify the how to derive the setting value, then click OK.
    5. Repeat Step iv if required for other Settings.
    6. Click OK to close the New Action dialog box.
  3. Click Apply.
