Adding a Balance Statement

This topic covers adding an adjusting Balance Statement after a Prior Day or Intraday Matching run.

TIPS:

  • Any differences between the two types of Matching are indicated within the procedure and under Prior Day v Intraday Matching below.
  • This action uses the standard Quantum Projected Cash Flow page.
  • Unless explicitly stated, references below to menus and grids relate to those in the page you initiated the adjustment from.

To add a new Balance Statement

This function is not available if Resource Security applies and you are not Exempt or do not have Write access to the Bank Account.

  1. On the Matching page's Overview tab, select an account in the Accounts summary and, in the grid menu, select New Balance Statement.

    TIP: You can also select this function in the Matches grid menu on the Matches tab (the account has already been selected).

    The Add Balance Statement dialog box is then displayed, with all balances defaulting to "missing" (that is, not entered). The Balance Date defaults to the End Date for the current view, as set in the parameters.

  2. On the Balance Details tab, change the Balance Date if required. Then enter (and/or delete) one or more balances, as required.

    • You cannot specify a Balance Date before the latest Marked as Reconciled date for the account.
    • When you enter a zero in a balance, this is regarded as an actual zero amount (not a missing balance). The zero is displayed with the appropriate decimal places based on the bank account currency, and the zero amount remains visible when you select another field or control.
    • You can delete a balance (including a zero balance) by selecting the field and pressing the Delete key. This blanks out the field, resetting the balance to "missing" (or not entered).
    • No Float or Forward Available balances will be saved if no Available Balance is specified.
    • Float or Forward Available balances must be entered in a contiguous block starting from the 1 Day Float or 1 Day Balance. For example, you cannot enter a 3 Day value without also providing the 1 Day and 2 Day values.
    • Any 0 Day forward available balance is not validated against the collected balance, as it does not affect other forward available balances.
  3. (Optional) In the Statement Detail text box, enter text giving further information about the adjusting entry.
  4. Click OK.

Your entered balances (or floats) are then saved as a Bank Flow Balance Statement.

Missing (blank) balances are saved as NULL values.

How Multiple Statement Balances are Handled

The system uses the last entered (or imported) Statement Balance of the requested type, for the given day.

For example, a particular day may have two statements, with Statement 1 containing only a Ledger Balance, and Statement 2 containing only an Available Balance. Then:

  • if a Ledger Balance was requested for that day, Statement 1 would be used,
  • if an Available Balance was requested, Statement 2 would be used, or
  • if a Collected Balance was requested, none would be returned.

Prior Day v Intraday Matching

For matching purposes, adjusting Balance Statements apply only to the program in which you add them - that is, Balance Statements added in Prior Day Matching have no effect on Intraday Matching, and vice versa.

However, all Balance Statements related to an account are displayed in the Bank Statement dialog box when you select View Statements from the Accounts grid menu (Overview tab) or Matches grid menu (Matches tab).
