Importing FX Deals

The FX Deal Importer supports the following QID-format fields.

  • analyse

  • external_ref_id
  • external_ref_text

 

Notes:

  1. "Two-sided" FX Deals use a field name prefix to enable the two copies of each field to be uniquely identified and populated. See Back-to-back FX deals and FX Swap deals below.
  2. Do not use Carriage Return/Line Feed in the Comments field. For multi-line comments, use the pipe symbol | to represent a new display line. For example, the following import file content:

    comments=Multi-line comment:|Second Line|Third line.

    ...would appear in the Comments text box on the Money Market Deals form as:

    Multi-line comment:
    Second Line
    Third line.

  3. The fwd_rate field replaces the fwd_points field used in earlier versions of Quantum.
  4. These fields apply only to Non Deliverable Forward deals (NDFs).

Virtual fields Supported

The FX Deal Importer also supports the following virtual fields:

  • fx_type = moFXDEAL

(Normal FX Deal)
  • fx_type = moFX_Swap

(FX Swap)
  • fx_type = moFX_Back_to_Back

(Back to Back FX Deal)

Use these fields to set the program mode when using Tab-separated format (if using QID format, you can use the appropriate $DoMenu commands to change the mode).

This virtual field should be the first field specified for each FX Deal.

Back-to-back FX deals and FX Swap deals

Back-to-back FX deals and FX Swap deals use the following field name prefix mechanism to enable the two copies of each field to be uniquely identified and populated:

  • Prefix the names for fields on the left side of the FX Deal form with 0_ (note the underscore).

0_cparty = Barclay's Bank

  • Prefix the names for fields on the right side of the FX Deal form with 1_.

1_cparty = Bank of New Zealand

FX Swaps linked to Deal Sets

When importing the two legs of an FX Swap Deal that is linked to a Deal Set, ensure that either:

  • The deal_set_id value is specified in only one leg,

    -or-

  • The deal_set_id value is the same in both legs.

(If different values are specified in the two legs, the value from the last leg imported is used.)
