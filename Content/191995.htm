Statistics (Adaptiv)

System Administrators should use the Statistics page to create and maintain Statistics records that define the pre-processing methods to be applied on historical time series, which Price Factors to include, and which statistics to derive when you run a Model Calibration.

Statistics may be calculated using a series of stages, each of which can produce values for some or all of the available factors in the archive file. Each stage specifies a series of parameters which control how the returns will be calculated for these factors.

During each stage, some, none, or all of the available statistics may be calculated for each of these factors. Correlations may also be calculated for each pair of factors included in the stage. Each stage will produce all the requested statistics and correlations for the specified factor parameters (type, ID, Tenor point, and Method*) and these values will replace any that were calculated by a previous stage.

*CAUTION: If you change the return Method for a factor (from Log to Diff and vice versa), all the previous statistics will be lost, not just the ones being recalculated by the current stage.

There are two system-defined setups:

  • Lognormal - used for calibration of models used in VaR calculation, and
  • Normal - used for all other model setups.

Data descriptions

Code - a unique value identifying this Statistics record.

Name - a text description for this record.

Configuration - a text string of (at least) the following parameter settings, in the required syntax (see the example below).

NOTE For full details, refer to the Adaptiv Analytics Interfacing Guide or to your FIS Consultant.

  • Mode - New to clear any existing statistics before the calculation begins, or Append to preserve the existing statistics (if any)
  • Business_Days_In_Year -
  • Stages - one or more sets of stage parameters, all within square brackets, where each set contains the following parameter settings (also within square brackets):
    • Name - optional
    • Return_Methods - followed by one or more sets of parameters for Price Factors to be included in the stage, where each set contains the following parameter settings (also within square brackets):
      • Price Factor type - ForwardPrice, FXRate, InterestRate, and so on
      • Method - Log or Diff
      • IDs - All or Selected followed by a bracketed list of Price Factor IDs of the specified type
      • Tenor - null OR a text string to restrict the calculation to only those factors whose point matches this string exactly, OR an equals sign followed by a regular expression specifying the criteria for selecting a point
    • Statistics - All or Selected followed by a bracketed list of statistic names (Volatility, Drift, MenRevSpeed, LongRunMean, ReversionVol, Skewness, Kurtosis, Lambda1, Lambda2, Lambda3, Lambda4, and/or Beta)
    • Correlations - No or Yes to return the calculated correlations between every pair of price factors
    • Start_Date - optional
    • End_Date - optional
    • Horizon - time interval (for example 1d for 1 day)
    • Frequency - optional
    • Decay_Factor - a number
    • Diagnostics - No or Yes to save the rates stored in the market data archive and the calculated returns to a file named VarStatsCalc.csv in your My Documents folder.

Example

The Default Factor Groups record provided with your system has the following Configuration field content, with only one stage defined:

Mode=New,Business_Days_In_Year=260,Stages=[
[Name=,Return_Methods=[[DiscountRate,Log,All,,],[EquityPrice,Log,All,,],
[FuturesPrice,Log,All,,],[FxRate,Log,All,,],[FXVol,Log,All,,],
[ImpliedCorrelations,Log,All,,],[InterestRate,Diff,All,,],
[InterestRateVol,Log,None,,],[InterestYieldVol,Log,All,,]],
Statistics=All,Correlations=Yes,
Start_Date=,End_Date=,Horizon=1d,Frequency=,Decay_Factor=0.94,Diagnostics=No]]
