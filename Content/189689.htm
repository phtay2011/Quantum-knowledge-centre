Rate Set Deals for Multiple Dates

You can now specify a date range on the Rate Set Deals page in order to rate-set multiple deals at the same time.

History

Previously, you could only rate-set individual deals at a single rate-set date.

Implementation

Query

The Query area of the Rate Set Deals page includes the following new functionality.

  • The query configuration area of the Rate Set Deals page now includes radio buttons in place of drop-down lists, previously available, for selecting the rate-set action. This change, along with an additional field to accommodate the date range, enables you to rate-set fixed rate deals within a date range.
  • When you select Run Query and Rate Set in the page header, the system will use the corresponding rate from the curve to perform the rate-set for ALL the deals in the range.

View

When you rate-set across a date range, all deals whose rate-set dates lie between the specified range are displayed in the View. The View area of the Rate Set page includes the following new and enhanced functionality:

  • The rate-set date of the deal is displayed in the Rate Set Date column.
  • When you select multiple deals within a date range in the View and rate-set them using the Preview Rates option in the grid menu, ALL the selected deals are rate-set at the rate-set date of the deal with the most recent Rate Set Date in the selection (for example, in a selection, if the earliest Rate Set Date is 03-Jul-2017 and the latest Rate Set Date is 01-May-2017, all the selected deals will be rate-set at 01-May-2017).

    NOTE: When you preview the rates for multiple deals in a date range, the Save Latest Rates on Rate Set option is disabled in the Preview Rates dialog box.

  • When you select multiple deals within a date range in the View and rate-set them using the Perform Rate Set option in the grid menu, Quantum will use the corresponding rate from the curve to perform the rate-set for the selected deals.
  • When deals with multiple rate set dates are within the date range specified in the query, they will be shown in individual rows, one row for each rate-set date (in chronological order).

    NOTE: When you select any row of the deal with multiple rate-set dates, all the rate-set dates or rows of that deal will be selected (that is, you cannot select individual rate-set date (s) for a deal with multiple rate-sets.

Exclusions and Restrictions

  • The ability to forward rate-set In-Arrears floating rate deals in a date range is not included in this enhancement.
  • Following this update, the Rate Set Date values for existing queries will transferred to the Rate Set Start Date and the Rate Set End Date will default to Today. You will have to individually change the Rate Set Start Date to match the Rate Set End Date to view the original results.
  • When you preview the rates for a date range, the Save Latest Rates on Rate Set option is disabled in the Preview Rates dialog box.
