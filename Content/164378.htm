Index

Use the Index Rate page to create and maintain a Rate Series table specifically for Indexed Securities instruments. An Index Rate Series table provides the information needed to calculate the inflation index for an Indexed Bond.

NOTE:

  • You must have the required permissions to create, modify and delete an Index table.
  • Before creating a new Index Rate Series, you must first setup an indexed Rate Name, as this defaults the basis frequency for the Rate Series.
  • Before you can use an Index Rate Series with an Instrument using the INDEXED BOND formula, you must set up at least one Index Definition for that Rate Series, to specify how the CPI Inflation index is to be calculated from the Index values.

About the Index Rate Series grid

The Index Rate Series grid shows the index records for the Rate series, one record per Index date period, in date sequence.

These records show the indexation information that is published by a government or external entity, and which you maintain periodically in Quantum. The expected update and maintenance frequency is the same as the Index Rate Series frequency (the Default Basis, as set for the selected Rate Name).

TIP: The K Factor Instrument field applies only to Index Rate Series for Australian Capital Indexed Bonds using the CIB-RBA formula.

Each grid row shows one Index record, and contains the following items (in the respective columns).

Effective Date

The earliest date from which the Index record can be used in Index calculations.

  • Before this date, Quantum treats the Index value as if it does not exist.
  • The first index record's Effective Date is usually the Start Date for the Rate Series.

Effective Dates from the Index Rate Series table are not used in any calculations for Instruments that use the Indexed Bond Formula. The Effective Date for an Indexed Bond deal is calculated on deal save, using a combination of the available Index Rates and the Index Calculator Definition associated with the Instrument setup.

CAUTION: DO NOT change this value for past periods with Index values previously used in current and past deals.

Period Start Date

The start of the index period. The published Index value applies to the period starting from this date and ending on the day before the next Index record's Period Start Date.

TIP: When first setting up an Index Series, enter the Start Date of the first period.

Published Date

(Applicable only when the Use published date option is selected.)

The valid Published Date for each existing record (except the two latest-dated placeholder records), where the month is the one after the Period Start Date month in each case.

Timeband

A read-only label indicating the end month of the period; used as a sort key by Index Update.

Index

The value of the index, as published by the relevant government or external entity - for example, a Consumer Price Index (CPI) for use with a TIPS (Treasury Inflation-Protected Security) instrument. This index will be interpreted differently by the respective Indexed Security formulas.

CAUTION: Make sure that the index value is correct before you run Index Update. To correct an index series after Index Update has run, you will need to change the index value here, re-save all affected deals, and then run Index Update again. Quantum always uses the most recent figure to update an index.

Inflation Rate

The optional corresponding annual inflation rate for each Index value in the Rate Series.

Next Coupon Date

The coupon date that corresponds to the calculated K Factor (see below).

K Factor

Applicable only to Index Rate Series for Australian Capital Indexed Bonds using the CIB-RBA PERIODIC formula, which is valid only with a CPI index released on a Quarterly basis.

Thus calculating K Factors for a given Index Rate Series is only applicable for those with Default Basis set to Quarterly. A K Factor Instrument must also be specified (only those using the CIB-RBA formula are available for selection).

The K Factor is the calculated index ratio corresponding to the Nominal Value of the Principal. For example, if the Face Value of an Indexed Bond is 10,000,000 and the K-factor is 110.43, then the Nominal Value of the Principal is 11,043,000.

TIP: For information about the Capital Indexed Bond Formula, see CIB-RIBA Formulas.

Index Rate Series Procedures

On the Index Rate page:

To list Index Rate records

  1. Click the button.
  2. Optionally follow the Configure link to change browse list properties.

To add a new Index Rate record (from scratch)

  1. Click New.
  2. Select the predefined Index Rate Name.

    The Default Basis frequency defined for that Rate Name is displayed (read-only).

  3. If this Index Rate is for Australian Capital Indexed Bonds using the CIB-RIBA PERIODIC formula, select the K Factor Instrument to apply for K Factor calculation.
  4. Select Use full precision if you want to increase the limit on the number of decimal places users can enter for rate values in the current Index Rate series to 12 decimal places. Clear the option to restrict entry of rate values to 3 decimal places (the default).
  5. Select Use published date to enable entry of Published Date values into the Index Rate Series grid. You must then provide a valid Published Date for each existing record (except the two latest-dated placeholder records), where the month is the one after the Period Start Date in each case.
  6. In the Index Rate Series grid menu, select New.
  7. In the New Index Rate Series dialog box, enter an Effective Date, Period Start Date, Index value, and (if displayed) a valid Published Date* (except for), where the, then click OK.

    *NOTE:Published Date values do not apply to the last two placeholder entry rows; otherwise, their months must be one after the Period Start Date month in each case

    A entry row is added to the grid.

  8. Repeat Steps 6 and 7 to complete the Index Rate Series table.

    TIP: For similar entries, use the grid menu Copy function and edit the copied values.

  9. Save.

To add a new Index Rate record by copying an existing one

  1. Click to list available records.
  2. Select the Index Rate record you want to copy (use the < and > buttons to page through, or type in the Search box).
  3. In the page menu , select Copy.
  4. Select a different Index Rate Name.
  5. Proceed as in the procedure above (Steps 3 through 9).

To edit an Index Rate Series table

TIP: Changing the Index Rate Name field is NOT recommended. Create a new Index Rate record instead, for the Rate Name you want.

  1. Click to list available records.
  2. Select the Index Rate record you want to edit (use the < and > buttons to page through, or type in the Search box).
  3. Use the Index Rate Series grid menu functions to add, edit, delete, and copy entries in the grid, as required.
  4. Optionally select a new K-Factor Instrument for K-Factor calculation and change the Use full precision and Use published date option settings if required.
  5. Save your changes.

To delete an Index Rate record

  1. Click to list available records.
  2. Select the record you want to delete (use the < and > buttons to page through, or type in the Search box).
  3. In the page menu, select Delete.
  4. When prompted, confirm your action.
