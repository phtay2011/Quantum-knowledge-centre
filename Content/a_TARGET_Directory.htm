TARGET Directory

From 2020.3, we stopped using the TARGET2.txt file and included the addressee details in the EFT XML Framework. From 20.4.2, the Target table is seeded in Quantum.

Details of TARGET Addressee

In general, to set up addressee details in the EFT XML Framework:

  1. Edit the EFT layout and Includes records.
  2. Set up addressee details:
    1. In Quantum, set up the correct security and operational settings.
    2. Set up the workflow parser.
    3. Add the TARGET directory content into the table.
    4. Create a TARGET folder to use in the DTL.
    5. Set up the DTL.
  3. Run the import.
  4. Delete existing data and upload your next TARGET directory content.

NOTE: Upload the TARGET directory content regularly from your usual provider, as Quantum does not automatically upload this file. The new version of the TARGET directory content is published weekly.

Edit the EFT layout and Includes records

For more information about editing the layout-associated records, read XML Template Files (EFTs) for SWIFT, TARGET.

Set up addressee details

This one-off process sets up addressee details in TARGET Directory page.

  1. In Configuration > Users, set up the Module Security tab to give access to the Data Transfer Location module and the Workflow Parser module. And set up module security for the TARGET Directory page.
  2. In Configuration > Operational Settings, set up the operational settings of the database: in the Active Services area, enable Import static data and Process workflows.

  3. Open Configuration > Workflow Parser and set up a workflow parser, for example "TargetParser". The columns for the tab-delimited data used in the example, "TargetParser", are: Key = Line, Position = 1, Name = Line, Type = Text, and there is no default value. Add the data "LINE" in the Generate Columns dialog box.

    Save the parser as shown:

  4. Create a folder, "TARGET", under \Site\TEST\WorkflowIn\Import\TARGET

  5. Create a Data Transfer Location (DTL), for example, "TargetDTL" , where Parser TargetParser and Location =TARGET. In the Action grid, set up an action to create a TARGET directory. The inputs used in this example are: Participant BIC, Addressee, Account Holder, Institution Name, City Heading, National Sorting Code, Main Bic Flag, Type of Change, Date Valid From, Date Value, and Participation Type.

    An example of an expression associated with an input is "Participant Bic" with "Mid(LINE),1 ,11)".

    Field Formula
    Participant Bic Mid([LINE],1,11)
    Addressee Mid([LINE],12,11)
    Account Holder Mid([LINE],23,11)
    Institution Name Mid([LINE],34,105)
    City Heading Mid([LINE],139,35)
    National Sorting Code Mid([LINE],174,15)
    Main BIC Flag (Mid([LINE],189,1) <> "N")
    Type Of Change If(Mid([LINE],190,1)='A',1, If(Mid([LINE],190,1)='D',2, If(Mid([LINE],190,1)='M',3, If(Mid([LINE],190,1)='U',4, 0))))
    Date Valid From If(Mid([LINE],191,8) = '99991231', DateValue('47121231', "yyyyMMdd"),DateValue(Mid([LINE],191,8), "yyyyMMdd"))
    Date Valid Till If(Mid([LINE],199,8) = '99991231', DateValue('47121231', "yyyyMMdd"),DateValue(Mid([LINE],199,8), "yyyyMMdd"))
    Participation Type If(Mid([Line],207,2)='01',1, If(Mid([Line],207,2)='02',2, If(Mid([Line],207,2)='03',3, If(Mid([Line],207,2)='04',4, If(Mid([Line],207,2)='05',5, If(Mid([Line],207,2)='06',6, If(Mid([Line],207,2)='07',7, If(Mid([Line],207,2)='08',8, 0))))))))

  6. Start "AvantGard Static Data Import Service".

    Quantum will map data from this file to the respective fields in the EFT XML Framework.

  7. Copy the TARGET directory to the WorkflowIn > Import > TARGET folder. The service imports the file automatically once the file is in the folder, and stores to the "ewt_target" table.

Delete the existing table

There are two ways to delete the existing records from an existing TARGET table:

  • Create a DTL configured to delete the records in the table.
  • Use the Delete All action.

To create a DTL configured to delete the records in the table

  1. Create a new DTL page similar to the following:

    Location Type = FILE, Purpose = WorkflowIn, Location = <your location>, and Parser = <your TARGET parser>.

  2. Include an Action, such as Action Name = Delete Target Directory, where the action lists the Participant Bic, Type of Change etc. in your table.
  3. Save the DTL page and use it to delete the current TARGET directory content
  4. Place the new TARGET directory content in a location where the DTL page can access it. The DTL page set up to pull in the directory, for example, "TargetDTL" above, will pull in data from the new TARGET directory content.

The system imports the directory automatically once the directory is in the folder, and stores to the "ewt_target" table.

To use the Delete All action and reload the directory

Use a composite action to clear previous TARGET directory content before loading a new directory content.

  1. In the Configuration > Workflow page, in the Actions grid, select New.
  2. Enter a name for the action, in this example "Delete Target Directory".
  3. Select Scope = First. Using this will truncate the table.
  4. Enter a Prerequisite, if required.
  5. Add a new input and In the Select Action Name dialog box, Delete folder, select "Delete All".
  6. Click OK.
  7. Create a second action, in this example called "Create Target Directory". For this action, select the rows of the data table that contains the TARGET data.
  8. Save the page.

This replaces all data each time you run it. TARGET data is not static data.

The system checks for the deal's counterparty BIC code initially, to return an addressee. If the system doesn't find the BIC, it uses Our Bank BIC. If it finds no value, it returns an exception message.

Attributes associated with TARGET

From 2020.3, we stopped using the TARGET2.txt file and introduced some attributes, such as mapTable and targetDate, to support the TARGET functionality and return a unique addressee from a TARGET table.

"targetDate" has additional formatting logic that formats the given xpath value to an 11 character value, the same as the logic for the current SWIFT Target2 plug-in.

The process finds the fields Valid from and Valid till from the TARGET table, and ignores records with invalid effective date range (i.e. the cashflow date of the EFT not falling in the Valid from and Valid till date range).

For records with the same BIC and with multiple valid date ranges, the process selects the record with the earliest Valid till date, and ignores records with dates where Valid from is greater than Valid till.

For records with the same Valid till date, the process selects the record with the latest Valid from date.
