SWIFT Format Templates for Stock Flows

If you want to output SWIFT MT540/541/542/543 EFT messages for stock flows (see Note below), you must specify a template for each Stock account. You can set up different preferred stock flow message output format templates for use with different Stock accounts.

NOTE: Stock flows apply to Security deals based on instruments with the Allow stockflows option selected, and for Repo deals with underlying Security deals of that type.

  • MT540 and MT542 messages are for Receipt and Delivery of stock free of payment, respectively.
  • MT541 and MT543 messages are for Receipt and Delivery of stock against payment, respectively.

A SWIFT output format template specifies header details and SWIFT tags for SWIFT MT540 / 541 / 542 / 543 stock flow message output. It is stored in a text file in the AdvicePath for your Quantum site, with:

  • a file suffix of .QMT,
  • optional embedded comment lines (which start with a hyphen), and
  • one line for record header details, then one or more lines per SWIFT output tag, in output order.

    TIP: Some tag formats have options. For more details, see Tags with Optional Formatting.

A SWIFT EFT processing run will use the appropriate output format template to produce the actual message output tags (data fields), each with their corresponding data.

Two SWIFT output format template files are provided (in the configured AdvicePath for your site), as follows.

  • MT540_543_Mandatory_Fields.qmt - gives the minimum output required, and
  • MT540_543_Supported_Fields.qmt - gives the maximum output allowed.

Use these files as starting points for creating your own SWIFT output format templates. You can use a Text Editor program to edit either file, but you should save any changes under a new file name.

CAUTION: Changing the original files themselves negates their usefulness as references for creating your own templates.

If more than one SWIFT output format template is present, you must specify a default one for each Stock Account.

If no SWIFT output format template is found, the minimum (mandatory) output will be produced by the system.

For details of actual output file fields, contact your Quantum consultant.

Tag 35B - Stock Identification Options

The Tag 35B format specification in the MT54x message template files has several variants, enabling you to optionally:

  • Control the Stock identifiers that will appear in the output
  • Output different prefixes for CUSIP numbers (in the format /XX/, where XX represents any two alphabetic characters)

The optional parameters you can include in valid format variants are as follows. If included, the parameters must appear in the order indicated in the examples below, and the square bracket delimiters are required.

Formatting Notes

In the template format specification:

  • The square brackets are required.
  • If an alternative CUSIP abbreviation is specified, it must be specified as shown (two alphabetic characters within slashes).

Other points to note:

  • Blank values will not be output.
  • Not all three values can be blank, as one of them (the Primary Identifier) is mandatory.
  • If no options are specified (the default, as shown in Example 8 above), all non-blank values are output with ISIN and CUSIP prefixes applying.
  • Example 7 (where no two-character prefix is specified for CUSIP) drops the CUSIP prefix. This is different from the default (Example 8).

NOTE:

  • Tag 95 is a special "generic" tag that is replaced in the output by one of the others, depending on the variant and the actual field values in each case.
  • For full SWIFT compatibility, use each format variant only for the tags and message sequences indicated (as shown in the MT540_543_Supported_Fields.qmt file listing).
  • Use each variant once only - that is, in one Tag. For example, if you specify :95::ACCW, do not also specify :95P::ACCW or :95Q::ACCW.
  • Each format variant must immediately follow the tag designation.
