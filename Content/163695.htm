Entity Procedures (Quantum, non-group)

On the Entity page (in the Default perspective):

NOTES:

  • Actions are subject to your Module Security permissions, and Resource Security may also apply.
  • If the Replication Service is running, these records can also be accessed on the FIS Quantum Analytics Entity page, with most fields set as read-only.

To list Entities (including Groups)

  1. Click the button.
  2. Optionally follow the Configure link to change list properties - see Configure Browse list views.

To add a new Entity

  1. In the New menu, select Item.
  2. Enter a unique Name for the Entity, then enter the Legal Entity Identifier (if required), a Short Name and External Name.

    TIP: To include your new Entity in one or more groups, see Entity Group Procedures (Quantum).

  3. On the Address tab, enter the Contact person, Company Registration, Address, Phone, Telex, and Fax.
  4. On the Transactions tab:
    • Optionally enter one or more Taxes Applicable for the entity. For more details, see the following Edit procedure.
    • Under Calculate WTAX based on the Party, select the default Withholding Tax calculation method for the entity's Money Market, Securities, and Interest Rate Swap deals - either of the following, to default to the tax rate for the party Receiving interest or Paying interest. For more details, see the following Edit procedure.
    • Select the Country and Non Business Days Location.
    • Under Reporting Classification, select Exposure (Trading) or Cover (Hedging) to specify whether the transactions for this entity are Trading or Hedging, respectively. This setting is used to distinguish the entity's deals for exposure reporting of interest and currency positions.
    • Select Include Dealer in closeout matching criteria to include the Dealer as one of the criteria for closeouts for this entity. Changing this setting will not affect previously entered deals - the change (once saved) will come into effect the next time a deal is transacted under this entity.
    • Select Entity is a SWIFT Financial Institution to identify the entity as being a SWIFT financial institution for EFT processing purposes.
    • Select Exempt from Clearing to exclude the entity from having any of its trades cleared through a clearing house.
  5. On the Accounting tab, indicate whether deals made by the entity are subject to Hedge Accounting and, if so, specify the date from which Hedge Accounting should be applied, and how deferred tax should be treated for cash flow Hedge Relationships. See Entity General Ledger Options.
  6. On the Identifiers tab, optionally enter identifier information for the entity - for example, Clearing House codes and Matching System codes used in Settlements Control and Confirmation Matching. For more details, see the following Edit procedure.
  7. On the Analysis Codes tab, optionally link the entity to predefined Analysis Code values.
  8. Save your new Entity record.

To edit Entity details

  1. Click to list available records (or use the Search box).
  2. Select the record you want to edit.
  3. Edit the relevant fields. In particular:

    To add new Identifier fields:

    1. On the Identifiers tab, use the Select Identifier button to open a Browse window showing External Methods that have the Enable Identifiers option selected.
    2. Select one or more External (Payment) Methods. Use the search field at top right to filter the list, and use the < and > buttons to page through the list if required.
    3. Click OK to close the Browse window. Your selected External Methods are displayed in a two-column list.
    4. In the Identifier field for each Pay Method, enter the identifier value for this entity.

    To remove an Identifier:

    • On the Identifiers tab, use the x button to the right of the Identifier field in the Pay Method / Identifier list.

    To apply one or more new taxes to the Entity or Group:

    • On the Transactions tab under Calculate WTAX based on the Party, select the default Withholding Tax calculation method for the entity's Money Market, Securities, and Interest Rate Swap deals.
      • Select Receiving interest to default to the tax rate for the party who receives the interest. For borrowings, this is the Counterparty; for investments, it is the Entity.
      • Select Paying interest to default to the tax rate based on the party who pays the interest. For borrowings, this is the Entity; for investments, it is the Counterparty.

        This option controls only the defaulting of the tax rate used. This can be changed by the user at deal creation time. Also, if a tax is specified in Facility Maintenance, that takes precedence over the Entity tax for defaulting.

    • In Taxes Applicable, click to open the selection dialog box, select one or more taxes available for the entity (use the < and > buttons to page through the list if required), and click OK.

    To remove a tax:

    • In the Applicable Taxes box, click the x button for the tax (or click and deselect the tax in the selection dialog box).
  4. Save your changes.

To import Entity data

(Your User record must have the Can import data option selected.)

  1. Click to open the Browse window.
  2. In the Browse window menu, select Import.
  3. In the Import dialog box, browse for and select a File to Upload name and location.

    This file must contain Entity static data records, in a supported format (QXT, QID, or tab-formatted).

  4. Click OK.

    Entity static data records in the file are imported into the Quantum database provided the data is valid and you have the required security rights in each case.

    When the import operation is finished, you will receive a status notification.

To export Entity data

  1. Click to open the Browse window.
  2. Select the Entity and in the Browse window menu select Import or Export.
  3. In the Export dialog box, select the export format: Excel, CSV, XML, or PDF.
  4. Click OK.

    Data for the selected Entity is downloaded in the specified format, as indicated by the filename at the bottom left of the Entity page.

  5. (Optional) To open the downloaded file, click the filename (at the bottom left of the page). Or to locate the file on your computer, select Show in folder in the filename menu ^.

To delete an Entity record

  • You cannot delete a record if that affects transaction, settlement, or financial data, and you cannot delete a group if it has members.
  • Deleting removes the record completely from the database. To prevent use of the record but retain it in the database, deactivate the record instead.
  1. Click to list available records.
  2. Select the record you want to delete (use the < and > buttons to page through, or type in the Search box).
  3. In the menu, select Delete.
  4. When prompted, confirm your action.

To deactivate an Entity

  1. Click to list available records, or use the Search box.
  2. Select the record you want to deactivate.
  3. In the menu, select Deactivate. This action will not appear if the record is already deactivated.

To reactivate an Entity

  1. Click to list available records, or use the Search box.
  2. Select the record you want to reactivate.
  3. In the menu, select Reactivate. This action will not appear if the record is already activated.

To publish an Entity

TIP: All fully approved new or changed Entity records will be published automatically If the Business Structure option is selected (on the Operational Settings page under Publish).

  1. Click to list available records (or use the Search box).
  2. Select the record you want to publish.
  3. In the menu, select Publish Selected.

To publish all Entities (including groups)

  • In the page menu, select Publish All.
