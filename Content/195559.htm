Workflow

The Enterprise Workflow (EW) pages listed under the Configuration menu let you configure a huge variety of reports and templates, basing them on variants of existing pages. There are some common features involved with many of these pages including: sources, actions, functions, and a definition of what constitutes an invalid character.

Invalid characters for names

Do not use these characters in names that are used in workflow reports, pages, or templates:

  • "." - period / full stop
  • ] and [ - square brackets
  • } and { - curly brackets
  • / - forward slash
  • # - hash
  • ~ - tilde

While some fields may accept them in names, many will not and will throw a validation message if you try to save them.

Common feature - Sources

Use the Sources grid to select the sources of data in the Workflow page, report, or template page you are creating. Sources are sections of the database that you define by choosing:

  • the columns in the database that you want data from (Columns tab), and
  • the rows of the database that you want to include (Settings tab).

Common feature - Actions

Create, delete, convert, or manipulate data through buttons or menu items on target pages you are creating by adding actions via the Workflow grid (Workflow Page) or Actions grid (Workflow Report). Actions enable you to apply functionality that already exists in the software.

Common feature - Expression Builder and functions

Any time you use the Expression Builder dialog box you can choose from system-defined functions such as Sum, Count, DistinctCount, and so on.

Using these functions is the best way to add a high level of detail and specificity into the content of the cells in your results. You'll find Expression Builder opens in many places in Quantum, such as the Calculated Columns grid or the Validations grid.
