Importing Actual and Projected Cash Flows

The Actual and Projected Cash Flows Deal Importer supports the following QID-format fields:

  • amount
  • analyse
  • bank
  • append_to_deal
  • bankacc
  • ccy
  • ccy2
  
  • cflocode
  • cflow_dt
  • cflow_no
  
  • cflow_type
  • comments
  • cparty
  
  • cparty_type
  • dealer
  • deal_dt
  
  • deal_no
  • entity
  • external_ref_id
  
  • external_ref_text
  • face_value
  • fx_rate
  
  • grid
  • period
  • rate
  
  • sectype
  • strategy
  • ticket_no
  

This Deal Importer operates in either Actual Cashflows Mode or Projected Cashflows mode.

Additionally, G/L postings can be segregated into Bank flows and Broker flows. When using the QID format, you can use Commands to switch between these modes. See Actual and Projected Cash flow Commands.

Commands are not supported for Tab-separated format (row format) files, so you will need to use the following virtual fields to change modes.

Actual or Projected Mode

  • cflow_type = moactual

Actual cash flow
  • cflow_type = moprojected

Projected cash flow

Bank or Broker Mode

  • cparty_type = mobankmode

Bank cash flow
  • cparty_type = mobrokermode

Broker cash flow

NOTE: These virtual fields should be the first fields specified for each Deal.
