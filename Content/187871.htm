  • Projected Cash Flows,
  • Unit Deals,
  • Unit Holding Deals,
  • Hedge Relationship Deals,
  • Account Adjustments,
  • Cash Transfers, and
  • General Ledger Journals.

Resource Security Specifics by Deal Type

This topic summarizes the specific Resource Security effects that apply with the following deal types. (To view the secured Resources for all deal types, see Secured Resources by Deal Type above).

Actual and Projected Cash Flow Deals

Account Number and Bank (Counterparty) Resource Security is not considered when browsing and retrieving a deal. Users will still be able to view deal details and make changes, but will be prevented from changing any Resource-related field value they do not have Write access to.

When creating a broker cash flow (projected or actual), the Broker field is saved in the database as the Counterparty (even though it is displayed as Broker). The Broker database field represents the Counterparty of the Broker cash flow. For this reason, when the Counterparty Filtered Resources option is set to Enabled (on the Data Security page), the Broker cash flow browse will always be filtered by Broker, even if the Broker category option is selected (under Additional Counterparty Categories for Deal Browse Filtering on the Data Security page), is disabled in the Defaulting Rule.

Actual Cash Flow Deals only

If in Data Security, both the Bank Account and Apply to actual cash flows Filtered Resources options are selected, then Bank Account Resource Security is also applied to Actual Cash flow (AC) deals, and users will not be allowed to access AC deals for bank accounts they do not have access rights to. See Resource Security Options.

Letters of Credit

Resource Security on Issuer, Advisor, Confirmer, and Negotiator is not considered when browsing and retrieving a deal. Users will still be able to view deal details and make changes, but will be prevented from changing any of these Resource-related field values they do not have Write access to.

Account Adjustments and Cash Transfers

Resource-related fields are not considered when browsing and retrieving a deal. Users will still be able to view deal details and make changes, but will be prevented from changing any Resource-related field value they do not have Write access to.

General Ledger Journals

The G/L Journal deals available in the browse window depend on the User's access level to each G/L line item. The User must have Write access to all G/L row items in order to save or re-save the deal.

Entity Resource Security is not applied when a value is pasted into the field (but as above, the journal entry will be saved only if the User has Write access to all Entities in all G/L row items).
