Workflow Report Example 3

This example shows how to create a workflow report to see the status of general ledger (G/L) journals that you sent to a third party system (interfaced journals), and how to perform some actions on those journals. We've called this report the G/L Interface Journal Status report.

To create a G/L interface journal report

  1. On a Workflow Report page, click New.
  2. Enter a Name (less than 8 characters), Title, Url Name, Menu Area, and Menu Group for the report.
  3. Select the primary source. This is the data source used to generate the core report information:
    1. In the Sources grid menu, select New. In the New Source dialog box, click the Source drop-down and select Interfaced Journal Status.
    2. In the Columns tab, define the fields to display in the report, including External Link, External Reference, External Status, Interface Number, Last Updated, Message Code, Message Text, Post Set Number, and Status.
    3. In the Settings tab, define the query fields Interface Date and Status to filter the records; then click OK.
    4. Click OK to close the New Source dialog box.
  4. Configure the dates field that will use the date the journal was exported:
    1. Select “Interface Date” to open the Edit Settings dialog box.
    2. Select Operator = Equal to, Input = Filter, and Filter = Interface Date, and select OK.
    3. Select “Status” to open the Edit Settings dialog box.
    4. Select Operator = One of, Input = Filter, and Filter = Status, and select OK.
    5. Select OK to close the Edit Settings dialog box.
  5. Configure two filters that will be made available to the user when running the report:
    1. In the Filters grid menu, select New.
    2. In the New Source dialog box, in Name enter “Interface Date” or similar.
    3. Select Type = Date and a default value, if required, and select OK.
    4. Set up a second filter. In the Filters grid menu, select New. In the New Filters dialog box, in Name enter “Status”, or similar. Select Type = Lookup, Linked Source = Status, enable Allow multiple values, and add a default value, if required.
  6. Configure actions that you can make on the data from the Interfaced Journal Status data source, for example, if the export of a journal failed, resend the data through a report. See below for more examples.
  7. Click Save.
  8. Log off and on again to see your new report in the chosen menu.
  9. Open the report and configure the View to be the required format.
  10. Run the report.
  11. Add the report to your Dashboard.
