Setting Up Analysis Codes

Analysis Codes are data items that you can set up and use for categorizing and reporting on key Quantum static data, some instrument types, specific deal types, or across all deal types. For more details, see About Analysis Codes.

NOTE: To implement any Analysis Code setup changes in Analytics, you must run the Analysis Code Replication wizard.

To define a new Analysis Code

  1. In the Configuration menu, select Analysis Code. The Analysis Code page opens.
  2. Click New.
  3. Enter or select the following values.
    • Name - the mandatory unique name for the analysis code throughout the system.
    • Description (mandatory) - a text description of the Analysis code; up to 40 characters long.
    • Scope - the type of data records the analysis code applies to - one of the following:
      • Deals - all deal types. See Defining Analysis Code defaults for deals.

        NOTE: When Scope = Deals and Source of Values = Data Source, the Deals folder is available in Expression Builder. It contains fields that are common across all deal types such as Comment, DealDate, Entity…).

      • a specific deal type - Actual Cash flows, Bank Account Transfers, Commodity Swap Deals, Currency Options, and so on. See Defining Analysis Code defaults for deals.

        TIP: Selecting Money Market Deals covers both Money Market and Security deals.

      • Bank Accounts* - all bank, broker, and Money Market Fund accounts.
      • Commodity - all Commodity static data records.
      • Counterparty* - all counterparties.
      • Entity* - all entities.
      • Facility - Facilities and Dealing Lines.
      • Instrument Type* - a subset of instruments, namely: Money Market, Security, Hedge Strategy, Unit, and Unit Holding instruments.

        NOTE: Changes to Bank Account, Counterparty, Entity, or Instrument Analysis Codes may be subject to Static Data Approval (if the corresponding option is specified on the Data Security page).

    • Data type (available only for new records; read-only if a copied record) - select TEXT, NUMBER, AMOUNT, DATE, TIME, or DATETIME. If the Scope is Facility, then select TEXT.

      TIP: This setting cannot be changed after the record is saved.

    • Source of Values (available only when Data type is TEXT) - select one of the following to specify what text values users may enter in the field for this Analysis Code (on the applicable pages):
      • User Input - any text value, or
      • Predefined Values - any of the literal text values listed in the Predefined Values grid (see below).
    • Options:
      • Allow values other than predefined (available only when Data type is TEXT and Source of Values is Predefined Values) - select this option to allow users to enter free-format analysis code values (in addition to any predefined values); clear it to allow users to select from the list of predefined values only.
      • Warn when entered value is not predefined (available only when Allow values other than predefined is selected) -- select this option to request user confirmation before they can save a non-predefined value for this analysis code; clear it if the system should save such values without needing user confirmation.
      • Mandatory - select this option to force users to enter a value for this analysis code when entering the relevant type of data record; clear it to make analysis code entry optional.
      • Show on cash flow detail during bank reconciliation - select this option to enable users to select this analysis code as (1) a display column in the Cash Flows grid on the Flows tab of the Prior Day query page, and (2) a Cash Flow field to match with any of the generic Bank Flows fields In the Passes tab's Criteria grid on the Detail Match Script page.
    • Decimal places (available only when Data type is AMOUNT) - enter the number of decimal places for amounts entered under this Analysis Code.
    • Minimum and Maximum (available only when Data type is NUMBER or AMOUNT) - optionally enter a minimum and maximum to be enforced for values entered under this Analysis Code.
    • Predefined Values grid (available only when Source of Values is Predefined Values) - use the grid menu's New, Edit, and Delete functions to create and maintain the list of predefined text values.

      NOTE: If Allow values other than predefined is cleared, you must enter at least one predefined value.

  4. Click Save.

    Your new Analysis Code is now set up. The Analysis Code tab or area on the relevant application page will display the code field, labeled with the Name you entered in Step 3.

To edit a previously set up Analysis Code

  1. On the Analysis Code setup page, click and select the code you want from the list.

    TIP: Use the < and > buttons to page through the list, or the Search box to reduce the list. Click the Configure link to change list properties.

    The page fields are populated with the values for your selected Analysis Code.

  2. Enter or select new values as required (see the procedure above).

    TIP: Name and Data type cannot be changed on a saved record.

  3. Click Save.

    Your changes are implemented. Note that:

    • When you remove a Predefined Value, any occurrence of that value in affected data is replaced by NONE.
    • When you change the Scope - except when you expand coverage from a specific deal type to all Deals - it is equivalent to a delete and add. The Analysis Code is removed from all records and pages it previously applied to (including all backdated occurrences), and added to the data records and application pages it now applies to.

To delete a previously set up Analysis Code

  1. On the Analysis Code setup page, click and select the code you want from the list.

    TIP: Use the < and > buttons to page through the list, or the Search box to reduce the list. Click the Configure link to change list properties.

    The page fields are populated with the values for your selected Analysis Code.

  2. In the page menu, select Delete.
  3. Click OK to confirm.

    The Analysis Code is removed from all affected data records, including all backdated occurrences, and the field will no longer appear in the Analysis Codes tab or area of the relevant Quantum pages.

    NOTE: To implement any Analysis Code setup changes in Analytics, you must re-run the Analysis Code Replication wizard.

CAUTION: These settings allow a user to import a deal that has values that do not match the predefined values without warning.

To set up predefined values for regulatory reporting

We provide support for collateral tags in the workflow templates, for example, in the workflow templates "EMIR Regis-TR" , "EMIR DTCC" and "EMIR Regis SFT RA". This functionality allows you to report the compulsory Collateral tags with a new trade submission, for example, when using the regulatory reporting functionality in Quantum.

  1. To use this on deals, in Configuration > Analysis Codes create an analysis code named "Collateral Type".
  2. In the Predefined Values grid you can add "Fully" (FC),"OneWay" (OC), "Partially" (PC), or "Uncollateralized" (UC).
  3. If you see "U" in the output, this means Not Set.
