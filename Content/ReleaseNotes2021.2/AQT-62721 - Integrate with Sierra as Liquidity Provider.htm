Adding external Web API data as a source in Workflow Reports

Data Transfer Location â€“ WorkflowOut/In purpose

We've added a new Web API purpose called WorkflowOut/In on the Data Transfer Location page. This purpose handles sending data and receiving synchronous responses from external Web API services.

NOTE: The WorkflowOut/In purpose is available only when Location Type is set to WEB API.

Web API Authentication settings

WebAPISettings.config file is now replaced with DTL configuration. A new field called URL is introduced within the Details section where you can specify URL of external Web API. The URL field is required and validated against standard URL format.

We've added the Authentication section where you can define authentication details.

The following authentication types are available:

  • None

  • Basic - requires only username and password

  • Bearer - requires username, password, token name and Bearer token URL

Adding Web API as a data source in Workflow Report

Web API DTL is now added as a data source in WF Report.

Columns from the Web API source are exposed and available for selection. Selected columns will be displayed in Custom Report results. In the Join tab you can map Quantum columns to Web API source columns.

When you run a report that uses Web API source, the system sends Template data to the external Web API service and receives back an synchronous response. Parsed response data will be displayed in your Custom Report results.

Adding Web API as Workflow Action

In the Action tab > the Transfer folder you can now select DTLs with WorkflowOut/In purpose.

Web API action is using Template inputs to create an HTTP request and send it to an external Web API. The HTTP response is parsed and set to Web API Action outputs.

Monitoring of Web API requests/responses

We log Web API requests/responses in the monitoring table. This table is now exposed as database source.

You can create report and monitor all successful and failed Web API calls.
