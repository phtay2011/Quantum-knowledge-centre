FX Cover Counterparty Rule Entries

Use the Counterparty Rules grid on the FX Cover Request Rule page to specify:

  • one or more combinations of Counterparty and Projected Cash Flow deal criteria for applying the FX Cover Rule to, and
  • necessary data for the FX deal created when the rule is applied and the resulting FX Request approved.

The following prerequisites must first be set up:

  • Cash flow Instruments named Committed and Anticipated,
  • the FX Instrument to be used when creating the FX deal, and
  • Counterparty Analysis Codes for FX Cover Requests.

TIPS:

  • Where rule criteria overlap, the first matching criteria will apply.
  • Different percentages can be set up for each time band, in terms of either date range - for example, when a new percentage is to apply from a specific future date, specify two Deals From/To ranges.

Counterparty Rule Procedures

To add a new Counterparty Rule entry into the current FX Cover Rule

  1. In the Counterparty Rules grid menu, select New.

    TIP: If your new entry is similar to an existing one, select that grid row and use the Copy function instead. Then edit the record as in the next procedure.

  2. In the New Counterparty Rules dialog box:
    1. Select the Counterparty to whom the FX Cover Rule will be applied.
    2. In the next five fields, select or enter the Projected cash flow deal criteria for applying the rule to:
      • Cash flow Instrument - Committed or Anticipated
      • Exposure Date range (From and To) - select the applicable offsets from the cash flow's Value Date
      • Deals date range (From and To) - select the applicable dates to match against the cash flow creation date
    3. In the remaining fields, select or enter the necessary rule information for the FX deal created when the rule is applied and the resulting FX request approved:
      • FX Instrument
      • Dual Currency - clear this option for a single currency (obtained from the analysis code of the Counterparty); select for dual currency where the second (sell) currency is sourced from the ccy2 field of the Projected Cash flow.
      • Coverage % - the percentage of the Projected Cash flow amount giving the FX Deal amount (zero to 100)
      • Currency - the Buy currency
      • Internal Template file - the location and name of the FX Deal QID template file

        This determines which fields will be included from the cash flow when the FX Deal is created. The default is the hdgtempl.txt file supplied in the configured AppDataPath\FXCover\Templates location under your site root folder on the Application server. For more details, refer to the Quantum Installation Instructions or to your site administrator.

    4. Click OK.

      A grid row is added for the new Counterparty Rule entry.

  3. Repeat Step 2 to add any further rows, as required.
  4. Click Save.

To edit an existing Counterparty Rule entry

  1. Click a value in the entry grid row (or select the row and in the grid menu, select Edit).
  2. In the Edit dialog box change the required fields (see the previous procedure); then click OK.
  3. Click Save.

To remove an existing Counterparty Rule entry

  1. Select the entry grid row and in the grid menu, select Delete.
  2. In the dialog box click OK to confirm.

    The entry row is deleted.

  3. Click Save.
