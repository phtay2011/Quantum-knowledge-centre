Import General Ledger Account templates

(First introduced in Release SP 6.7.1, August 2018.)

You can now import General Ledger Account templates into Quantum Six, using the Static Data Transfer spreadsheet.

Implementation

In the Static Data Transfer spreadsheet, a GeneralLedgerAccountTemplates worksheet has been added, as shown in the example below.

To download G/L templates from the Quantum Six database into the spreadsheet:

  1. In the data control panel (on the right), optionally select one Accounting Type if you want to filter the templates on that Accounting Type.
  2. Click Download.

    The spreadsheet is populated with G/L templates from the Quantum Six database, one per row (for the Accounting Type if one is specified, otherwise all of them). The Status column contains Downloaded in each case.

To update existing G/L templates using the spreadsheet:

  1. First download templates from the database (see above).
  2. Make the necessary changes in the required spreadsheet rows.
  3. Select the rows with changes.
  4. In the data control panel, click Upload.

    Each selected G/L template will be updated in the database IF your changes pass validation.

    • If so, the Status column contains Updated in each case.
    • If not, the Status column contains Update Error and the Message column displays details of the error.

To add new G/L templates to the database:

  1. Optionally download templates from the database (see above).
  2. Create new spreadsheet rows for each new G/L template, entering new Template Name and Constant values, and valid values for (at least) Owner Name, Type, Currency, and Account Name.
  3. Select the new rows.
  4. In the data control panel, click Upload.

    New G/L template records will be created in the database IF your changes pass validation (for example, a Template Name or Constant already exists).

    • If so, the Status column contains Inserted in each case.
    • If not, the Status column contains Insert Error and the Message column displays details of the error.

To delete existing G/L templates from the spreadsheet:

  1. First download templates from the database (see above).
  2. Select the G/L template rows you want to delete.
  3. In the data control panel, click Delete.

    Each selected G/L template not in use (by generated accounts and/or by another template) will be deleted from the database and the Status column contains Deleted.

    G/L templates that are in use are not deleted. The Status column contains Delete Error and the Message column displays The item cannot be deleted because it is in use.

Prerequisites

The spreadsheet must have been set up by an administrator, for you to use. For more details, see the Static Data and Rates Transfer Setup Guide.

Reference

For more details, see the Static Data Transfer Spreadsheet User Guide.

Limitations

The Static Data Transfer spreadsheet is the only import method currently supported for G/L Account templates. Until further notice, formatted files are not supported for importing this type of data.

Only Accounting Type G/L templates may be transferred (NOT Entity G/L templates).
