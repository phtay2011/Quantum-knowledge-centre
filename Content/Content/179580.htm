Importing Hedge Relationships

The Hedge Relationships Deal Importer supports fields that relate to:

  • The Hedge Relationship as a whole - see Importer fields for Hedge Relationships
  • Linked deals - see Importer fields for linked deals.

Once your Import files are correctly set up, subsequent importing requires little or no intervention.

TIP: Before creating your first Import file, familiarize yourself with the logic of the Hedge Relationships form. In particular, note which settings are required before or with others.

The Hedge Relationship Deal Importer supports vertical QID format - see examples.

Order of fields

The order in which fields are given in the importation file is important.

In particular, any field that enables other fields on the Hedge Relationships form must be set to yes (Y) - or otherwise selected - in the imported file before the contents of those other fields are entered.

For example, rel_oci=Y must appear before tran_oci=Y in the Import file. This reflects the functionality of the Events tab on the Hedge Relationships form, where you must select the Release OCI check box before you can select the Transfer OCI check box.

NOTE: If a wrong combination would cause a warning message on the Hedge Relationships form, it will result in an entry to the Message Log when the file is imported.

Some faulty combinations of settings cause the entire record (all fields relating to the Hedge Relationship) to be rejected. For example, you cannot choose the Short-Cut Method for effectiveness calculations unless Nature of Risk is set to Overall Market Risk.

Conflicting information

The setting in a field should not contradict the setting in an earlier field. For example, if you switch on the Short-Cut Method and then specify Effectiveness parameters, the parameters will be rejected.

Message Log

If a field in the imported file is rejected for any of the above reasons, the Importer returns an Invalid field specified message to the Message Log, followed by details of the rejected field.

Optional fields

Fields that are optional on the Hedge Relationships form (for example, Ticket Number) are also optional during importation.
