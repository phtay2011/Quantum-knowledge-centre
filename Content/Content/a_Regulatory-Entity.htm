Regulatory Entity

Find a Regulatory Entity page in the Static menu, and use it to store regulatory reporting account information for the entity. For example, use this page to store the EMIR DTCC entity account information.

The system uses this page as a source in the workflow templates, to extract the applicable regulatory information related to the entity.

Use with the following regulatory reporting pages:

  • Rules > Regulatory Reporting
  • Reporting > Regulatory Reporting Status

Add this rule to a report in order to return reporting jurisdictions to the WorkflowOut folder.

The top section of the page contains these fields (natural keys):

  • Entity
  • Reporting Jurisdiction - (select only one of these values)
    • EMIR_REGIS
    • EMIR_DTCC
    • FCA_REGIS
    • FCA_DTCC
    • EMIR_REGIS_SFT

Corporate Sector can have multiple values for the same entity / counterparty and you enter the values in the relevant fields separated by a hyphen. Put in the codes directly in place of the fully expanded names.

For example, ‘L’ for ‘Alternative Investment Fund’, ‘C’ for ‘Credit Institution’, ‘1’ for ‘Agriculture, forestry and fishing’, and so on.

Use the Corporate Sub Sector field to describe additional sector classification details or to describe your corporate sector in greater detail.
