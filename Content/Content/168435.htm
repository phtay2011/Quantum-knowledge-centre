Audit

Use the Audit--Query page to review changes to "non-transactional" data by authorized users, including system administrators. Details include which data was changed, when, and by whom.

Included / excluded

The Audit query results cover changes to static data and most other non-transactional data, including Query configurations, Rates, Counterparty Delivery Instructions, Rules, Market Values, and so on. The Table Name browse window lists the relevant database tables.

NOTE: For Rules, the system only audits changes to the Counterparty Limit Rule header fields (Matching Criteria). This applies to the limits table only - not lmtdates, nor lmtdets.

EW: View changes to a page, report or dashboard

You can review / audit any changes made to a page (a Workflow Page page), a report (a Workflow Report page), or a dashboard (a Dashboard page) configured within the Enterprise Workflow (EW):

  • When someone configures a new page, report or dashboard, the system logs an Add action, visible in the Audit query.
  • When someone deletes an existing page, report, or dashboard, the system logs a Delete action, visible in the Audit query.
  • When someone amends a page, report, or dashboard, the system logs Before and After actions to the Audit query if the change affected the following fields: Title,Core Module (page only), Url Name, Menu Area and Menu Group. The system doesn't log XML content because it can be large, but you can view changes to the XML fields in the Audit Table.

The system also logs who made the change.

Run the audit on a custom table

This is an example of how to read the audit query results for a report that contains a custom table. Run the Audit query, targeted at the report that contains the custom table.

To read the table:

  • Add - someone added data
  • Update - someone changed data
  • Delete - someone deleted data
  • Table - descriptive name of the custom table
  • Table Name - name of the table in the database
  • Key Value -
    • natural key fields of the custom table, separated by commas if there are multiple fields, plus
    • natural key fields of the detail table (if natural key fields specified), or
    • all fields of the detail table (if no natural key fields specified).
  • Key Name - names for fields listed in Key Value in the database
  • Field - name of the field that was updated
  • Before - value before update
  • After - value after update
  • Date - date of this audit entry
  • Time - time of this audit entry
  • Actioned By - full name of the user who made the change
  • User Code - logon name of the user who made the change

Notes and Attachments in Static Data Audit Reporting

For Static Data applications the Audit report displays the following details for notes and attachments:

  • Table affected by the change to a note (that is, System Notes).

    - OR -

    Table affected by the change to an attachment (that is, Note Attachment).

  • The Key Value and Key Name for the note or attachment.

    NOTES:

    • As static data definitions are stored in multiple tables across the database, a unique identifier is required to uniquely identify the note or attachment with respect to the Static Data it is attached to. This unique identifier is provided by the combination of data displayed in the Key Value and Key Name columns.
    • The Key Value column displays the note_key value and additional data which will explicitly identify the static data to which the note or attachment is added (that is, Name, Account Number, Currency, and so on).
      At the same time, the Key Name column displays field or table names of that value in addition to sysnotes or note_attachment.
  • The field affected by the action (that is, Note Text for the text of the note ORNote Text Format for changes to the format of the text ORFile Name for changes to attachments).
  • The value of the field before and after the change.

    NOTES:

    • The content of the note is displayed in plain text in the grid. However, if the format is changed, the text is displayed in HTML format.
    • The entire content of the note is displayed in the grid and the scroll bars can be used to view longer notes.
    • The name of attachment is displayed for attachment.
  • Action performed on the note or attachment (that is, Add, Update, or Delete).
  • Date and time of the change.
  • The user who performed the action.
