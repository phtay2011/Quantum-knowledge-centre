Knocking In Barrier Options

A Barrier option can be knocked in when the value of the underlying exchange rate (the Spot rate) breaches the Barrier rate. When knocked in, the option is activated and becomes a normal Vanilla European option.

NOTE: You can reverse the status of a deal that has breached a Barrier rate - from Ready to Knock-In back to Unchanged Knock-In - by using the Clear Knock Status function. You may need to do this if there is a dispute regarding the deal, or where an incorrect rate was used to determine that the deal had breached the Barrier rate. See Clearing a Knock Status for more information.

There are three types of Barrier option that may be knocked in:

  • DOWN & IN

    A Barrier rate is set below the Spot rate for a Call option, and if the exchange rate of the underlying currency falls below the Barrier rate, the option may then be knocked in.

  • UP & IN

    A Barrier rate is set above the Spot rate for a Put option, and if the exchange rate of the underlying currency rises above the Barrier rate, the option may then be knocked in.

  • DOUBLE IN

    Both Lower and Upper Barrier rates are set, below and above the Spot rate, respectively (for a Call or Put option). If the exchange rate of the underlying currency breaches either Barrier rate, the option may then be knocked in.

To knock in selected Barrier options

NOTE: When this action has been performed for a particular date, the action can only be undone by using the Rollback Deal function.

  1. If the current result list does not show the deal/s you want, check the Filters values and that the selected view is Double Barriers or Single Barriers as applicable, or Default (to display all deals meeting the filtering criteria).

    TIP: Select the Single Barriers or Double Barriers view option to show only those options.

  2. If necessary, adjust the Filter values and re-run the query.
  3. In the Result list, locate and select the deal entries you want - which should have Single Barriers or Double Barriers in the Type column and a Status* of Ready to Knock In or Unchanged Knock In.

    *TIP: If the Results list does not have a Status column, use the view's Configure link and the Columns tab to add it.

  4. In the grid menu, select Knock In.

    The Knock In dialog box opens.

  5. In the dialog box, edit the Knock In Date and Current Market Rate if required. The default market rates are the FX Cross Rates defined in Exchange Rates for each applicable currency pair.
  6. Select a Dealer or accept the default, which is either the setting from Currency Option Instrument Types or yourself (the logged-on user - if your User record has the Dealer option selected).
  7. Optionally enter any text Comments for reference purposes.
  8. Click OK. The dialog box closes and:
    • the system attempts to Knock In the selected deals;
    • any selected deals that are not Knock IN Barrier options will be ignored,
    • a message will be displayed indicating the number of deals knocked in and any detected errors;
    • in the results list, successfully actioned deal Status values change to Knocked In and a second entry is added to the list in each case, with a Type value of Exercise Lapse and a Status of None; and
    • a Messages button appears if errors have been detected - that is, when you have attempted to Knock In an option after its Exercise Date, before its Deal Date, or where the Currency Rate does not breach a barrier level.
  9. If errors are indicated:
    1. Click Messages for more details. The Message Log query page opens, set to the Default query with Reporting Period set to Today and Module set to CurrencyOptionDealKnockIn.
    2. Click Run Query to view the error messages, which will show the Deal Number and a description of the error in each case.
