Customizing Accounting Rule Criteria

The Matching Criteria for Accounting Rules of each type may be customized. Such customizations:

  • may only be done by users with Customize rights for the Accounting Rule module;
  • may only be done on the Accounting Type - Master record;
  • are available for each transaction type except Bank Account Interest (AI);
  • will apply across all Accounting Rules for the transaction type;
  • if for any of the following transaction types - will apply only to that transaction type:
    • Currency Option (CO),
    • Exchange Traded Option (ETO),
    • Futures (FU),
    • Interest Rate Option (IO),
    • Money Market and Securities (SE), or
    • Unit Holding (UNH);
  • if for any of the remaining transaction types (except AI) as below - will also apply to all others in this group - namely to:
    • Actual Cash Flow (AC),
    • Commodity Swap (CMS),
    • Currency Swap (CS),
    • FRA,
    • FX,
    • Interest Rate Swap (IS), and
    • Repo (RA).

      TIP: This is because these transaction types have common available data for criteria selection, and it is good practice to keep the Matching Criteria consistent across rules where possible.

To customize Accounting Rule Matching Criteria

  1. On the Accounting Rules page, click the Browse button and select Accounting Type, then Master.

    TIP: The Customize function is available only for the Master rule.

  2. Open the tab for the transaction type you want to customize.

    TIP: You cannot customize Bank Account Interest Accounting Rules.

  3. In the grid menu, select Customize.

    The Customize Accounting Rule Condition dialog box is displayed.

  4. To add a new condition (criterion): in the menu select New; in the New dialog box select Name for the new data item; enter its Priority; then click OK.

    The new condition is added to the list, in order of its Priority (existing entries are moved down if necessary).

  5. To edit an existing condition: click the Name link (or select the row and select Edit in the menu); in the Edit dialog box select a different Name; and/or change the Priority; then click OK.

    The condition is amended in the list, and moved if its Priority has changed (existing entries are moved if necessary).

  6. To delete an existing condition: select the condition row and in the menu select Delete; in the Confirm Delete dialog box click OK. (Or click the Name link; in the Edit dialog box click Delete; then click OK to confirm.)
  7. When you have completed your changes: in the Customize Accounting Rule Condition dialog box, click OK.

    The dialog box closes. Your changes will appear in the Matching Criteria for all Accounting Rules of that transaction type (when editing or adding rule entries) and - in some cases - other transaction types also*.

    TIP: The applicable transaction types are indicated in the Customize Accounting Rule Condition dialog box in each case - see also list items E and F at the top of this topic).
