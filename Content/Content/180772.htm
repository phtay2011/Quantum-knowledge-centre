Bank Import File Formats

You can use the Quantum Bank Import Service to import bank flow and external cash flow data in the formats defined by the following parsers.

TIP: Follow a parser link for more details of that parser and its file format.

  • AFB120 - Standard defined by the French banking organization and standardization Committee
  • BAI* - American BAI
  • BANKFLOW - generic bank flow
  • camt.052 - Bank to Customer Account Report (intraday or prior day)
  • camt.053 - Bank To Customer Statement - prior day
  • CNAB - Brazilian Febraban
  • ECHOS* - statement parsing using ECHOS and IBM MQ without the need for statement files
  • EXTERNALCASHFLOW - generic format for external cash flows
  • FEDCIPS - Federal Reserve CIPS
  • FEDIFT - Federal Reserve Intraday format
  • FEDIRDR - Federal Reserve Previous Day format
  • MT900/MT910* - SWIFT Confirmation of Debit (MT900) /Credit (MT910)
  • MT940* - SWIFT Bank Statement (sent to a financial institution authorized by the account owner)
  • MT941 - SWIFT Account balance report
  • MT942* - SWIFT Interim transaction report
  • MT950* (uses MT940 parser) - SWIFT Bank Statement (sent to the account owner)

*NOTE: If you want to import Bank transaction data from ECHOS, you must set up the .ini configuration file to enable this. Received statements can be viewed using the ECHOS Message Management page. See also Importing Bank Data from ECHOS.
