Currency Rounding Override in Quantum

The ability to round currency amounts (UP, DOWN, or using the 5/4 method) is defined separately for each currency and then applied globally to all transactions entered in that currency.

Selecting the Allow Deal Override option allows you to override the default currency amount rounding method at deal entry time. On the deal page's Other Details tab, select the Currency Rounding Override method you are going to use for the deal. All downstream processes for the deal (confirmations, settlements, renegotiations, accruals, revaluations and limits) will then use that rounding method.

Restrictions

The following restrictions apply.

  • Money Market, Security, FRA, and Swap deals only.
  • The Allow Deal Override option applies to the method used to round currency amounts only. The number of decimal places an amount is rounded to cannot be changed at deal entry time.

    For example, you cannot set a currency rounding method to DOWN with zero decimal places, then override the method to 5/4 and round to 2 decimal places at deal entry time.

  • Interest Rate Swaps must use the same currency amount rounding method on both sides of the swap. If you override the currency amount rounding method at deal entry time, it will be changed on both sides of the swap.

    You can work around this restriction however, by replicating the deal using a Cross Currency Swap with both legs in the same currency but with different currency amount rounding methods.
