Data Security Changes

The Data Security page's new Security Approval area contains options for enabling the new approvals functionality for Data Security itself, Logon Configuration*, Operational Settings, and SMTP Settings, as highlighted below.

Logon Configuration covers the Logon Message, Logon Security, Password Filters, and LDAP Authentication pages.

These settings determine if approval is required or not for changes to the relevant pages in each case. The following dependency applies.

  • Data Security is automatically selected and set to read-only when any other Security Approval option is selected. This is to secure the Data Security page against users changing their own settings.

If any change is made to Security Approval options, a validation is applied on Save, to ensure that Data Security still has at least two Approvers. If not, a warning message will appear and the Save is canceled.

Both Approvers and non-Approvers may make changes on the Data Security page. After any changes to Security Approval settings are saved, the Data Security page settings will be read-only until all pending changes have been approved or rejected, a message is displayed indicating that status, and a Pending Changes button is provided.

Emails and/or notifications (with a link to the page) will be sent to Approvers who have approval rights for pending Data Security changes except that the user who made the change will not be included (because no-one is allowed to approve their own changes).

For non-approvers, the read-only Data Security page will display the Before changes settings. The Pending Changes button will open a dialog listing the pending changes for reviewing.

For Approvers, the read-only Data Security page will display the After changes settings. The Pending Changes button will open a dialog listing the pending changes, with Approve and Reject buttons available to approve or reject the changes.
