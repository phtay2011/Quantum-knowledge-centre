Account Template Properties

When you create or edit a G/L template, the following items display in the New or Edit Account Template dialog box, respectively.

After accounts have been generated from a template, key items become read-only (as indicated in each case).

Template Name (G/L Templates)

(Read-only if accounts have been generated from the template, or if Type is SYSTEM SUSPENSE.)

As templates are used in lookups in other parts of Quantum, it is recommended that you use a meaningful naming convention when defining your template names.

The template name must be unique across all Accounting types, and it cannot be changed after the template has been saved.

TIP: Initially, the name you enter in this field also populates the Account name field. If you change the template name before you save the record, the Account name will not reflect this change.

Type (G/L Templates)

(Read-only if accounts have been generated from the template, or if Type is set to SYSTEM SUSPENSE.)

Account Types are used to differentiate the kind of account created, group them for inquiry and reporting purposes, and enable the application of business rules. For example, whether currency translation can be enabled.

  • Off Balance Sheet - The system reports the account within the Off Balance Sheet category.
  • Balance Sheet - The system reports the account within the Balance Sheet category. Currency translation may be applied.
  • CCY Position - The system reports the account within the Balance Sheet category. This is a system account and can be used only for G/L components CCY Position, or Base CCY Position. These are located on the Balance Sheet tab in Posting Rules for G/L Journals. See G/L Journals (Posting Rules).

    NOTE: These accounts are used only for the Currency Translation process. You cannot manually post to these accounts.

  • User Suspense - User Suspense accounts are used where the system cannot create a G/L entry. For example:
    • A Posting rule does not exist.
    • A Posting rules exists, however Account Templates have not been selected.
    • The G/L account currency required is not defined on an Account Template.
    • The G/L account cannot or has not been created.

      NOTE: If User Suspense account templates are not defined, the system will post to a System Suspense account.

    • The system reports the account within the Balance Sheet category.
    • Currency translation may be applied to this account type.
    • Use is dependent on Allow postings to suspense being enabled on Ledger Properties.
    • An Accounting type may only contain one User Suspense template per currency. The Type lookup will not contain User Suspense if all currencies are already defined and the Currency lookup will not contain any currencies already defined.

      TIP: As the default currency is All, you may need to alter the Currency field on the template to enable selection of User Suspense.

  • System Suspense - System Suspense accounts are used where the system cannot create a G/L entry, and User Suspense accounts do not exist or cannot be created.
    • This is a system account template provided by Quantum.
    • The system reports the account within the Balance Sheet category.
    • The only modifiable setting is the Apply account level currency translation option (and the associated fields when it is set).
  • Profit and Loss - The system reports the account within the Profit and Loss category.
  • Retained Earnings - This Account type is used in the G/L End of year process.
    • The system reports the account within the Retained Earnings category.
    • You must have one per currency per Accounting Type for the End of year process.
    • You may have only one active Retained Earnings template at any one time.
Currency (G/L Templates)

(Read-only if Type is SYSTEM SUSPENSE.)

Select the currencies you want G/L accounts created in. By default, the ALL setting is selected. This means that all currencies are implicitly selected.

You can define specific currencies First remove the ALL setting by clearing the ALL check box in the lookup, then select individual currencies. These will appear in the bottom pane.

Account Code Format (G/L Templates)

(Read-only if accounts have been generated from the template, or if Type is SYSTEM SUSPENSE.)

TIP: Before you use this field for the first time, you should understand the concepts.

This field defines the format that will be used to construct your G/L account codes. It can be made up from a combination of text and system fields, as defined in the Account Code Format Detail settings. That format must give unique account codes across all Accounting types.

The default value for this field is <Constant>, indicating the value of the Constant field (also indicated in Field Name 1 in the Account Code Format Detail area).

Initially, the lookup contains only the value <Constant>. As you create new Account Code Format Detail combinations, these will appear in the list and can be selected in other templates (provided unique account codes will be generated - see above).

Constant (G/L Templates)

(Read-only if accounts have been generated from the template, or if Type is SYSTEM SUSPENSE.)

One or more alphanumeric characters to be optionally used in the name of G/L accounts created from this template. If a value is specified and Constant is included in the Account Code Format, that value will be included - along with other Account Code Format Detail field values - in all generated G/L accounts created from this template, is used in conjunction

By default, Constant initially populates Field Name 1 in the Account Code Format Detail area and (correspondingly) the <Constant> meta token populates the Account Code Format field.
