Deal (Approval)

Use the Deal Approval page to view and approve deals (and/or requests*) and their related action events.

  • By default, only actions that you (the logged-on user) may approve are returned and listed. Use different Query and View configurations to change the result set and listed items, respectively (see below).
  • Approver records and their associated Signature Lists define users' approval types and levels.
  • Deal Approval rules determine which actions may require approval (and by whom), and whether to allow or prevent any cash flow (Settlement) actions from occurring before those actions are approved.
  • The criteria established in Approval and Authorization Rules (Deal tab) determine which deals, deal types*, and actions may require approval (and by whom). Approver records and their associated Signature Lists define users' approval types and levels.

*NOTE: Throughout this section, the term deals refers to both actual deals and requests. Any Requests that require approval will - unless filtered out - be listed with their Deal Type showing as Request.

Use the query Filters fields and options to reduce or expand the returned result set. The default query filters are as follows:

  • Reporting Period start and end dates - to select only deals dated within that period,
  • Approval Status - select:
    • ALL - to include deals with any approval status,
    • Approved - to include only deals with all required approvals granted (or none required),
    • Rejected - to include only deals rejected by one or both approvers, or
    • Pending - to include only deals awaiting either first or second approval.
  • Include deals I cannot action* - Select this option to include deals that you, the logged-on Approver, are NOT allowed to action further. These will be deals you are specifically excluded from approving, or where do not have Approver rights to the deal entity.

    Clear this option to include only deals that you are allowed to action. This is the default setting.

  • Include deals I have approved* - Select this option to include deals that you, the logged-on Approver, have already approved, but still require a second approval.

    Clear this option to include only deals that you may approve at the current time (whether or not they require a second approval). This is the default setting.

  • Include deals I haven't approved but can reject* - Select this option to include deals that you, the logged-on Approver, have NOT approved, but are permitted to reject.

    Clear the option to exclude such deals. This is the default setting.

*TIP: These options cannot be changed unless you have the specific Deal Approval Module Security rights in each case.

You can also add the following query filters (by clicking the Query Configure link, clicking Add Filter, and selecting them):

  • Counterparty,
  • Dealer,
  • Deal Set Number,
  • Entity, and/or
  • Transaction Type.

Run a query to return and list deal actions based on your query Filters settings. Optionally change the results View (or configure a new one). Then in the result grid, either:

  • select one or more items (by clicking the check box in the first column of each item row) and click the applicable function button (Approve, Reject, or Reject with Note), or select the required function from the grid menu;

    or...

  • click on a linked value in a single item row and select the required function from the quick menu.

See Deal Approval Processing Functions.

TIPS:

  • Single deals and their related actions can also be approved from the appropriate deal page.
  • Projected cash flow deals never require approvals.
  • By default, users cannot change deals that have been fully approved. If you need specific users to have the ability to change fully approved deals, your system administrator must enable the system-wide option Allow change of fully approved deals by authorized users on the Data Security page, and assign the Settlements permission Allow EFT Amendments to each user and/or user group (on the Module Security tab in the required User and/or User Group records).
  • Approval and Authorization rules can be set up to allow or prevent any deal processing or cash flow (Settlement) actions from occurring before deal actions are approved (see the Allow Processing and Allow CF Action options under Settings, Deal Approval Rules).
  • Next Approver can be configured as a displayed Column or Filter in result Views. Its Yes or No values indicate or filter items currently pending your approval or not (for example, items pending first approval when you are the second Approver).
