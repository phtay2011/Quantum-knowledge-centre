About Paper Confirmations

Paper confirmation output can be:

  • printed - for faxing* or sending by post,
  • saved to a file (in a suitable format for printing),
  • e-mailed, and/or
  • a backup to Electronic Confirmations - see About the Associated Confirmation Method.

*NOTE: While direct output to fax is not supported, this functionality can in some cases be provided using third-party software on your mail server. In this scenario, an SMTP e-mail would be sent to your mail server, which would then convert that e-mail into a fax and send it to an appropriate recipient.

Printed Confirmations

Printed confirmations are usually posted or faxed to your counterparties. They may be the sole type of confirmation produced, or they may serve as a backup to e-mail and/or electronic confirmations. To produce them:

  • specify PRINT as a Print option for the appropriate Confirmation Rule records, and
  • use the Confirmations processing page to preview and send the printed confirmations, or to automatically print them.

    Quantum will send the confirmations to your printer, in the defined format. If any addressees are entered on the Additional Contact Information tab for the main Counterparty Contact, an extra confirmation will be printed for each one.

Confirmations Saved to File

Confirmations saved to file can optionally be printed later, attached to an e-mail, and so on. They may be the sole type of confirmation produced, or they may serve as a backup to printed, e-mailed, and/or electronic confirmations. To produce them:

  • specify FILE as a Print option for the appropriate Confirmation Rule records, and
  • use the Confirmations processing page to preview and save the print-formatted confirmations.

Quantum will save the confirmations, one per file, according to the applicable Confirmation Control record in each case - that is:

  • in the defined format,
  • to the specified location,
  • with a file name suffix determined by the specified Paper File Type (.DOC, .DOCX, .PDF, or .RTF), and
  • with file names as specified in the File Mask panel of the applicable Paper Confirmation Format record.

E-mail Confirmations

E-mail confirmations can be sent directly to your Counterparties. They may be the sole type of confirmation produced, or they may serve as a backup to printed, saved-to-file, and/or electronic confirmations. To produce them:

  • access to an e-mail system that supports SMTP messaging control is required;
  • in Counterparty Contacts:
    • enter the counterparty's e-mail address(es) as settlement defaults under SMTP Address, and
    • optionally use the Additional Contact Information tab to add details for any additional contact persons;
  • specify EMAIL as the Print option for the appropriate Confirmation Rule records; and
  • use the Confirmation Processing page to preview and send the SMTP confirmations, as you would for printed confirmations;

    Quantum will send the confirmations to your e-mail server as an RTF file, in the format defined in the applicable Paper Confirmation Format record.

    If the counterparty SMTP address information contains an e-mail address, your e-mail server will attach the RTF file to an e-mail message. If any e-mail addresses are entered on the Additional Contact Information tab for the main Counterparty Contact, an additional e-mail message (with the attachment) will be created for each one.

  • In addition to transaction type and deal number, the user can optionally set up the counterparty's external name to appear in the email subject. For more details, see Paper Confirmations - Counterparty in Email Subject.

E-mail Confirmations for Bank Account Transfers

You can generate e-mail Confirmations to both Entities (bank account owners) when a Cash Account Transfer deal is settled. This can also apply to intercompany transfers.

The required setup is similar to that for Print Confirmations, with the following differences:

  1. A Counterparty static data record must be set up for each Entity involved in Bank Account Transfers. To provide for intercompany transfers, ensure that the Bank Internal option is selected.
  2. In Counterparty Contact Rules, specify the contact details for each of these Entity/Counterparties, with Transaction Type set to AT (or ALL) and SMTP Address giving the required email address in each case.
  3. In Bank Accounts, for any accounts to be used in Bank Account Transfers:
    • On the Delivery Details tab, set the Counterparty to the one set up in Step 1 for the Entity owning that account.
    • If Intercompany transfers are required for the account, on the G/L Setup tab select the Enable Intercompany Defaults option.
  4. In Confirmation Rules, set up at least one Settlement record with Transaction Type set to Bank Account Transfers, and Print Method set to EMAIL.
  5. In Counterparty Bank Accounts, set up a record that is linked to a Counterparty Bank Account Defaulting Rule for each Entity/Counterparty.
  6. To provide for intercompany transfers, ensure that Intercompany Bank Account Default records exist for each G/L Owner A and G/L Owner A Entity combination, specifying Transaction Type as AT (or ALL) and giving the required bank account names.

For Bank Account Transfers where the above setup applies, email confirmations will be generated for each cash flow - that is, two for an "external" transfer, four for an Intercompany transfer.

Confirmation Layouts

Layout and content of printed, saved-to-file, or e-mailed confirmation output can be modified by creating different Paper Confirmation Format Rule records. You can then add or edit the appropriate Confirmation Rule records, assigning your new formats to the required events, transaction types, entities, and other selection criteria.
