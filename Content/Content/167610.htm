Securities

Use the Security Deal page to record and/or to edit discounted and bond-style securities. These transactions cover all physical deals, whether they are domestic or foreign currency, simple or structured flows.

The fields on the page vary, depending on the formula and timing of the Instrument you select, and on other key settings.

NOTE: Security deals were previously included in the Money Market deals user interface in classic Quantum.

External References

External Reference numbers and text appear as read-only values on the Deal Details tab, but only when:

  • you imported or input the deal data via an API, and
  • the system populated the External Reference fields or properties.

Deal Allocation

Deal Allocation is optional for Quantum Security deals, and enables you to link a master deal (the Source deal) to one or more automatically created additional deals of a similar type.

See Deal Allocation for Money Market and Security Deals.

Closeouts

When you input a Buy deal and a Sell deal based on the same tradeable instrument, closeout can occur. To determine which deals are to be closed out, Quantum matches deals using the Closeout Method specified on the underlying Securities instrument.

See Money Market and Securities Closeout Methods.

Hedge Relationship Deals

TIP:

  • To add and maintain any deals in a Hedge Relationship, use the Linked Deals tab on the Hedge Relationship page.
  • The Dealing page shows a Hedge Relationships tab if the current deal is a Linked Deal within one or more Hedge Relationships. If so, you can quickly access the Hedge Relationship from this tab by clicking on the HR Deal Number in the Allocations grid.

Default Deal Values

In most cases (as for other deal types), deal values will default to any specified for the selected Instrument.

However, for Security deals, you can optionally default the Issuer, Issue Date, Maturity Date, Coupon (%), and Currency from the selected Security Identifier record when applicable, overriding any defaults specified on the Instrument itself. This applies when the Use for Defaulting option is selected on the underlying Security Instrument's Issuer tab.

When the Use for Defaulting option is cleared or a Security Identifier is not specified, defaulting is from the Instrument values. You can clear these defaults and enter new values for the Security Deal. Additionally, you can apply the settings from a Security Identifier record by selecting one in the Security Identifier lookup, which lists only those Security Identifier records that match the deal's Currency, Issuer, Issue Date, Maturity Date, Coupon (%), Coupon Frequency, Margin, and Ex Interest Days values.

Overnight Index Deals (FRN)

A deal that uses an instrument with the formula timing FRN-CGL Periodic can use the Analyticsâ€™ Average Interest Rate Calculator.

To use the Average Interest Rate calculator in a deal

  1. Open Analytics > Rates > Average Interest Rate Calculator.

  2. Enter data into the following fields:

    Analytics Field Name Related Quantum Field Name
    Rate Name Basis + " " + Rate Type
    Currency Face Value Currency
    As At Date Today
    Start Date Previous Coupon Date
    End Date Settlement Date - 1
    Method Calculation Method
    Term & Term Type Basis Frequency
    Denominator Day Count (either 365 or 360)
    Holiday List Currency & Location or Rate-set Location if "separate rate-set location" is checked
    Suspension 0-99
    Lag 0-99
  3. Open Dealing > Securities.
  4. Select an instrument that has Formula Timing = FRN-CGL Periodic, andOvernight Index check box enabled.
  5. Fill in all fields. In particular, fill in:
    • Deal Details
      • Face Value (currency)
      • Basis
      • Basis Frequency
      • Rate Type
      • Previous Coupon Date
      • Next Coupon Date
      • Settlement Date
    • Other Details
      • Day Count
      • Calculation Method
  6. Quantum then automatically calls the Analytics Average Rate function to insert the average rate in the Average Rate field. Notice that the other fields also alter to relate to the average rate.
  7. Save the deal.
  8. If you want to override the average interest rate calculation from Analytics, then select the Override Average Rate checkbox. Select the check box and Coupon = Average Rate + Coupon Margin.

If Settlement Date is equivalent to Previous Coupon Date, then the system does not call Analyticsand Average Rate = 0.

To reset the deal after saving

  1. Open the deal.
  2. In the menu, select Reset.
  3. Edit any field and watch the system call to Analytics and calculate the new value for the average rate.
  4. Check the changes by opening the audit log.

Yield - CDI Periodic Deals

For Rate Convention = DAILY and Annual Margin is set to Base Rate or Margin, when you save the deal, as Quantum generates the Rate and Rateset items it calculates the effective base rate and the effective margin as the daily value and not an annualized value.

Deals convert rates to daily rates in certain circumstances. When:

  • Rate Convention = DAILY, and
  • Formula Timing = YIELD - CDI PERIODIC, and
  • Annual Margin is set to Base Rate or Margin,

we calculate:

  • Effective Base as a percentage, based off the day count convention used, either ACTUAL\360, ACTUAL\252BD, and
  • Effective Margin as a percentage, based off the day count convention used, either ACTUAL\360, ACTUAL\252BD.

In other words, when you save a deal, as Quantum generates the Rate and Rateset items it calculates the effective base rate and the effective margin.as the daily value instead of a annualized value.
