SWIFT Local Authentication Rules

NOTE: This module is new from Release 2020.3, and replaces the earlier system-wide Configuration module. For more details, see Upgrading from a pre-2020.3 release below.

Use the SWIFT Local Authentication Rule page to optionally apply the SWIFT SA-2 or HMAC-SHA256 Local Authentication Algorithm to Finastra (formerly Misys) Confirmation, SWIFT Confirmation, and SWIFT EFT output messages created by the system.

You can specify multiple rules, with different combinations of External Method and Entity, where ALL is a valid setting in each case. The most specific matching rule's algorithm will be applied, in conjunction with user-provided authentication keys. The resulting authentication code is appended to each Confirmation or EFT message being output.

If you want to apply only one set of LAU keys across the system, have only one rule with External Method and Entity both set to ALL.

If you do not want to use SWIFT Local Authentication, do not have any rule records (or none activated).

NOTES:

  • Only users with specific Module Security permissions may change the first and/or second part of an authentication key.
  • If enabled, Resource Security also applies for Entities.
  • The date and time of the last modification to these settings are indicated, along with the modifier's User ID. All changes are recorded in the audit log. Use Audit-Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.
  •  When you add new rules, or make any changes, they will not take effect until you next start an applicable Confirmations Export or EFT Interface application.

Upgrading from a pre-2020.3 release

After upgrading, your pre-existing LAU settings - if the Enable SWIFT local authentication option was currently selected - become a single rule record with External Method and Entity both set to ALL.

If the Enable SWIFT local authentication option was NOT currently selected, no rules will exist after upgrading.
