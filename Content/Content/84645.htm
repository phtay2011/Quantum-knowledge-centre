Email Settings

Alerts can be sent via the MAPI SMTP.

Use Analytics System Manager to specify the mail protocol and e-mail notification settings to be used, as detailed below.

Configure Mail Protocol settings

  1. Open the Analytics System Manager from the desktop Analytics folder or by executing AGRSystemManager.exe from within the GUI Folder.
  2. Click on the Email Settings icon.

    The Email Settings feature's Mail Protocol Settings tab is displayed, as in the example below.

  3. Define the following SMTP settings required for your site.

    Host

    Mail Server Host.

    The name of the machine that the mail server resides on.

    Alert emails are sent from the alert server, using this machine as the Host.

    Port

    Defaults to port 25.

    SSL Method

    Select from the available options:

    None

    SSL will not be used for encryption.

    Use SSL

    SSL will be for encryption.

     

    Sender Domain

    The domain name.

    From Address

    The email address that is to be displayed as the sender of alerts. (Optional).

    If left blank, this will default to the SMTP server default email address.

    Authentication Method

    Select from the available options:

    None

    No authentication is required at the SMTP mail server.

    Basic

    SMTP Login authentication type.
    Username & Password is required.

    Auto

    The SMTP server authentication will be automatically determined when sending a message.
    Username & Password are required.

     

    User Name

    The SMTP username if authentication is required.

    Password

    The SMTP password if authentication is required.

  4. If mail message alert are to be displayed in the Analytics monitor, select Display Mail Server Messages.
  5. To define email notifications by email for replication errors, see Configure E-mail Notification settings below.
  6. Select File > Save to commit the protocol settings. If you want to cancel the changes, select File > Exit and click Cancel on the dialog.

NOTE: Authentication of SMTP settings is completed when the alerts server attempts to send a message.

Configure E-Mail Notification settings

  1. Open the Analytics System Manager from the desktop Analytics folder or by executing AGRSystemManager.exe from within the GUI Folder.
  2. Click on the Email Settings icon and select the E-mail Notification tab, as in the example below.

  3. Define the following Notification settings required for your site.

    Enable Error Notification

    Select to send email notifications on replication failure. When:

    • selected, the remaining items on this tab are available and mandatory.
    • cleared, the remaining items on this tab are unavailable.

    Recipient's E-mail Address(s)
    (; separated)

    Enter one or more email addresses (semi-colon separated).

    Example:

    bob1@abc.com; sue1@xyz.com

    E-mail Header

    Enter a suitable header for the notification. This will be displayed in the Subject line of the email.

    Example:

    REPLICATION ERROR

    E-mail Delay (sec.)

    Enter the number of seconds the system should wh6> from when the error occurs until it sends the email. This allows for multiple error messages to be sent in the same email. The delay counter will reset each time an error occurs within the delay period specified.

  4. You will also need to define Mail Protocol Settings for the system to send emails. See Configure Mail Protocol settings above.
  5. Select File > Save to commit the email notification settings. If you want to cancel the changes, select File > Exit and click Cancel on the dialog.

When an error occurs in the Replication Manager on either static data or deal replication, the recipient(s) will be notified by email. The body of the email will display the same message as that displayed in the Import Server (Quantum Analytics Monitor) error message.
