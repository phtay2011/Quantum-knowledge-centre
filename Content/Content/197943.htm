Configuration parameters - camt only

These INI file configuration parameters apply only to the camt format.

Schema validation

The ISO20022 Technical Support Group specifies the schema for each message, fully describing each message set. The group provides an XSD file for each version of the message that can be downloaded and configured at your site.

No schema files are provided with your Quantum software. To use schema validation, you must download* the schema XSD file for the message version you are importing, and set the following parameters accordingly.

*NOTE: It is recommended that schema XSD files are NOT saved in the same location as the BKImport.ini file.

Quantum supports the latest camt schema version.

  • For later changes: depending on the frequency and type of change determined by the standards, the new or latest version of the schema will be supported after full in-house testing, which will be targeted for completion prior to maturity of the specified release date/cycle of the changes.

    The Quantum Release Notes will summarize the type of change.

ValidateDocument

Description

Specifies whether or not to validate the imported XML message document against the appropriate schema file.

When set to Yes:

  • a valid SchemaFileLocation parameter must be included,
  • all validation exceptions are logged in the Message Log and saved as a log file in the Log folder, and
  • a file that does not match schema rules will be EITHER:
    • rejected - if a ContinueOnSchemaValidationFailure parameter is absent or set to No, OR
    • processed further - if ContinueOnSchemaValidationFailure is set to Yes,

Syntax

ValidateDocument = Yes | No

Default

ValidateDocument = No

SchemaFileLocation

Description

(Mandatory if ValidateDocument is Yes, ignored if it is No or absent.)

Specifies the folder where the schema XSD files are located for use in validating an XML document.

It is recommended that schema XSD files are NOT saved in the same location as the BKImport.ini file.

Syntax

SchemaFileLocation=FolderPath

Example

SchemaFileLocation=C:\Projects\Camt.053\BankImportation\Schema\

ContinueOnSchemaValidationFailure

Description

(Ignored if ValidateDocument is No or absent, or if SchemaFileLocation is invalid or absent.)

Specifies whether or not imported XML messages that fail schema validation will be be processed. This is useful when importing messages that are not version 8 compliant or are of any lower version, as files can still be imported and statements successfully generated.

Syntax

ContinueOnSchemaValidationFailure = Yes | No

Default

ContinueOnSchemaValidationFailure = No

Data Validation

NOTE: The following parameters may be included in the INI file in addition to the standard Data Validation parameters.

ValidateUniqueMessageReference

Description

Specifies whether or not to check if Statements with the same Message Reference already exist in Quantum, excluding existing statements with a null Message Reference value.

When set to Yes, the following message is generated if validation fails: Statement message reference is not unique.

Syntax

ValidateUniqueMessageReference = Yes | No

Default

ValidateUniqueMessageReference = No

UniqueTransactionReferenceCheck

Description

Used to check if the Customer Reference in the imported Statement already exists in the system.

Specifically, when set to Yes, checks the trans_ref field for sub-field 7. If a duplicate customer reference is found, the the Incoming statement will be rejected. To effectively use this parameter, it is recommended to appropriately define the specific customer reference so that the check is always applied to the same reference.

Syntax

UniqueTransactionReferenceCheck = Yes | No

Default

UniqueTransactionReferenceCheck = No

Data Reference

CustomerReferenceXPath

Description

Specifies the xpath of any specific Ntry field in CAMT messages that contain the Customer Reference, starting relative to the Ntry field.

Syntax

CustomerReferenceXPath = .//prefix:relativexpath

The xpath string must:

  • start relative to the Ntry field, and
  • include the XML namespace prefix as defined by the XmlNamespacePrefix parameter (see below).

Default

CustomerReferenceXPath = .//ns:NtryDtls/ns:TxDtls/ns:Refs/ns:EndToEndId

XmlNamespacePrefix

Description

Specifies the XML namespace prefix used in any xpath strings; required for DOM navigation.

Syntax

XmlNamespacePrefix = prefix

Default

XmlNamespacePrefix = ns

Data Manipulation

ConcatenateTransactionAdditionalInformation

Description

Specifies whether or not the transaction additional informationis to be concatenated to form the Text (TX) field.

Syntax

ConcatenateTransactionAdditionalInformation = True | False

Default

ConcatenateTransactionAdditionalInformation = False

TransactionAdditionalInformationSeparator

Description

(Ignored if ConcatenateTransactionAdditionalInformation is set to False or is absent.)

Specifies the separator applied between strings when transaction additional information is concatenated to form the Text (TX) field.

Syntax

TransactionAdditionalInformationSeparator = separatorCharacter

Default

TransactionAdditionalInformationSeparator = -

(Hyphen)
