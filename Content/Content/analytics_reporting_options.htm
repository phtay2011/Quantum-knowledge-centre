Options area, Assumptions tab (Analytics Query)

Use this area to specify the following options for the report. The items below apply to all report types unless explicitly stated otherwise. Select:

Complete time profile - to include the Complete Time Profile in the results (which include a Time Profile). Clear this option to include only those Time Profile buckets that do have transactions assigned to them.

Include valuation margins - to apply valuation margins to deal valuations.

Project rateset (Position, Cash Flow) - to imply floating Ratesets from the yield curve where the Rateset has not been performed (for valuation purposes, the deal is then treated as though the Rateset has been performed).

Convert to currency group (Position, Cash Flow) - to convert all Currency Option deals to the Domestic Currency.

Convert to base currency (Position, Cash Flow) - to convert all the deals to the base currency as defined in the applicable Valuation Rule record.

Complete spectrum (Position) - to include all points from the Yield curve used in a spectrum query calculation in the results. This is relevant for spectrum queries that use a Yield curve consisting of cash, futures, and swap rates.

Apply GL netting (Position) - to include postings to the General Ledger.

Round bond pricing (Position) - to apply appropriate price per hundred (PPH) rounding to securities transactions as defined in Instrument Maintenance.

Report on scenario profit/loss (Position, Cash Flow) - to display only the Profit/Loss generated by the application of the scenarios.

Include all limits (Position, Cash Flow) - to include all Limits in the results. All limits will be displayed regardless of whether there is any utilization of the limit. Clear this option to include only those limits that have been utilized.

Use domestic currency valuation (Position) - to convert Currency Option deal valuations into their domestic currency.

Use RiskMetrics rates (Monte Carlo VaR, Parametric VaR) - to use Risk Metrics rates for valuation purposes (determining the PV of cash flows) where available. Otherwise, rates will be used according to the selected Valuation Control record.

Create rate files (Historical VaR, Monte Carlo VaR) - to create rate files and export them to the Reports folder, replacing any existing files.

  • For Historical VaR: HISTFXRATES.CSV (historical FX rates), HISTVOLRATES.CSV (Volatility), and HISTZCRATES.CSV (Zero Curve rates) and HISTZCMARGIN.CSV (Simulated Margins and Zero Curves built with those margins)
  • For Monte Carlo: MCCOVAR.CSV (Covariance of asset classes), MCMARGINS1.CSV (simulated rate details), and MCZCRATES.CSV (zero curves and rates)

Use repo instrument - to use the Repo instrument within the reporting Group By columns. Clear this option to use the deal instrument.

Include gamma adjustments (Parametric VaR) - to include a Gamma Adjustment in the VaR calculation for non-linear products (options). The Gamma Adjustment is a second-order adjustment used to allow for the non-linear option payoff with respect to market rates. It is calculated in terms of the derivative with respect to the underlying dollar price times the notional cash flows of the option.

Report at holding level (Position, Cash Flow) - to aggregate any included Unit Deals under their Holding. Clear this option to return the individual Unit Deals.

Security Limit Mapping - Select one of the following options for applying limits (allocating the risk) to Discount Security deals only: Counterparty until Settlement; Counterparty until after Settlement; Always Issuer; or Always Counterparty.
