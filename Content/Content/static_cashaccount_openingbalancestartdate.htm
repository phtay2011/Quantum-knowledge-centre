Opening Balance Start Date

Use this field to select the default opening balance required by the first user-defined Horizon for each account included in a CashXplorer Worksheet. This default can be overwritten in CashXplorer (Select Accounts). Parameters on the CashXplorer Worksheet itself specify the Start Date for the opening balance.

CashXplorer calculates the balances for subsequent horizons using this opening balance, together with the concentration flows, floats and cash flows.

In the lookup, select the opening balance to be used for the first user-defined Horizon - one of the following:

  • External Available Balance and Floats - the balance calculated by the bank that reflects all available cash flows and takes into consideration the float;
  • External Bank Value Date Balance - the balance calculated by the bank that reflects all available cash flows based on the bank-reported Value Date;
  • External Current Available Balance - the balance calculated by the bank that reflects the amount of funds in your account to which you have immediate access;
  • External Ledger Balance - the balance calculated by the bank that reflects all accounting entries regardless of float;
  • Internal Bank Value Date Balance - as the Internal Reconciled Balance (see below), but based on the bank-reported value date (instead of Value Date);
  • Internal Reconciled Balance - the balance calculated by Quantum that includes all the presented cash flows, or - if you are running Bank Reconciliation - the value of all balance-level and detail-level matched cash flows, and all balance-level and detail-level matched external cash flows; or
  • Internal Unreconciled Balance - the balance calculated by Quantum that includes all cash flows or - if you are running Bank Reconciliation - all cash flows plus all external cash flows. This is the default.

NOTE: If you specify an External (bank-calculated) balance, and a CashXplorer worksheet query cannot find that balance for a particular start date:

  • a Missing Statement alert icon displayed to the left of the affected amount in the CashXplorer results view,
  • the system will revert to the Fallback Opening Balance Start Date setting specified for the account,
  • an error message is displayed indicating the account and which fallback balance was used instead (use the CashXplorer Configure View History tab's Messages sub-tab to review all messages), and
  • you can find the requested statement date on the CashXplorer Configure View History tab's Accounts sub-tab.
