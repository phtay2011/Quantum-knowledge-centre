CashXplorer - Requested Opening Balance Date

The requested Opening Balance Statement Date for each account included in a CashXplorer results View is now displayed in the Data Version History.

Seeing exactly what date was requested will be particularly helpful in those situations where you believe the wrong balance has been used (specifically, where a statement cannot be found for the requested date, in which case the account's specified fallback balance is used instead).

Implementation

On the CashXplorer page's Configure View panel, the History tab now lists, in the Requested OB Statement Date grid column on its Accounts sub-tab, the Statement Date requested for providing the Opening Balance for each account, as highlighted in the following example.

IMPORTANT NOTE: When a requested Statement Date is NOT found:

  • a Missing Statement alert icon is displayed to the left of the affected Opening Balance in the results view (as for four accounts in the example below), and
  • an error message appears on the History tab's Messages sub-tab, indicating the account and which fallback balance was used instead (also shown below).

History

Previously, when a statement could not be found for a requested date, you could only make an assumption about the date the system was looking for. That date was available only by an analysis of your database.
