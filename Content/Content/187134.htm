SWIFT 2016 MT Release Standards

Quantum Six now supports the following changes required by the 2016 SWIFT MT Release Standards.

SWIFT EFT Messages

Tag 90A in MT 54x EFT Messages - Support for Negative Rates

A negative sign indicator (the character N preceding the digital part) may now be included in Field 90A of MT54x messages when the trade price is a negative rate, subject to the following conditions:

  • the Percentage Type Code is YIEL, and
  • the price must not be zero.

This applies to the following SWIFT EFT message types:

  • MT540 Receive Free
  • MT541 Receive Against Payment
  • MT542 Deliver Free
  • MT543 Deliver Against Payment

Tags 50A and 52A both allowed in MT910 Messages

SWIFT MT910 messages may now optionally contain both Field 50A (Ordering Customer) and 52A (Ordering Institution).

(It is still mandatory that one of these must be present, but the validation disallowing both has been removed.)

When both fields are present, 50A takes precedence.

This applies to SWIFT MT910 Confirmation of Direct Credit messages only.

SWIFT Confirmation Messages

Tag 37M in MT36x Confirmations - Support for Negative Rates

SWIFT MT36x Confirmation messages may now include Field 37M (in place of Field 37U) with a negative sign indicator (the character N preceding the digital part) when the trade price is a negative rate, subject to the following conditions:

  • the rate (expressed as a percentage) must not be zero,
  • the integer part of the rate must contain at least one digit, and
  • a decimal comma is mandatory, and is included in the maximum length (12 digits).

This applies to the following SWIFT Confirmation message types:

  • MT360 Single Currency Interest Rate Derivative Confirmation
  • MT361 Cross Currency Interest Rate Swap Confirmation

Tag 90A in MT518 Confirmations - Support for Negative Rates

A negative sign indicator (the character N preceding the digital part) may now be included in Field 90A of MT518 messages when the trade price is a negative rate, subject to the following conditions:

  • the Percentage Type Code is YIEL,
  • the price must not be zero

This applies to MT518 Market-Side Securities Trade SWIFT Confirmation messages only.

Bank Import File Changes

Tag 25P Account Identification - BIC Code Support

In SWIFT Bank Import messages Account Identification can now include a BIC Code when be Field 25P is present (in place of Field 25).

While the optional BIC Code must be a registered BIC, its presence in Account Identification does not alter processing. It is strictly based on the Account Number.

For example, either of the following are valid (but not both):

  • 25:123456

    -or-

  • 25P:123456
    PTSANZBB

This applies to the following Bank Import SWIFT message types:

  • MT900 Confirmation of Debit
  • MT910 Confirmation of Credit
  • MT940 Customer Statement Message
  • MT941 Balance Report
  • MT942 Interim Transaction Report

Tags 50A and 52A both allowed in MT910

MT910 messages in imported SWIFT Bank files may now optionally contain both Field 50A (Ordering Customer) and 52A (Ordering Institution).

(It is still mandatory that one of these must be present, but the validation disallowing both has been removed.)

When both fields are present:

  • 50A takes precedence and is used to populate the bank flow Counterparty value, and
  • only a single (Intraday) bank flow is generated.

This applies to SWIFT MT910 Confirmation of Direct Credit messages only.
