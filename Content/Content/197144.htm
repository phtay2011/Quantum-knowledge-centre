Generate Fees

Use the Process Fees page's Generate Fees function to calculate fee cash flows and (optionally) accounting entries for a specified date range, based on the current Fee Deal Position History table. You must do this before you can:

  • create Actual Cash flow deals for the generated fee amounts; and
  • post the generated fee accounting entries to the applicable G/Ls (if required).

CAUTION: Do this after a Get Base Details run for any date or period - otherwise you will get fees with zero amounts.

Procedure Summary

  1. On the Process Fees page, click Generate Fees.

    The Generate Fees dialog box opens.

  2. Select Apply Default Dates or select a specific Start Date and End Date of the range for which you want to generate fee cashflows and accounting entries.

    The default Start Date is the day following the last (latest) Generate Fees run, or (if no previous runs are detected) the current date. The default End Date is your specified Start Date or the current date, whichever is the later.

    The system validates (and, if necessary, adjusts) the Start Date for the specified Fee Deals to ensure that no dates are skipped from the last time the Fee Generator was run for those Fee Deals.

  3. Optionally select Calculate G/L Entries to generate (but not post) accounting entries (fee accruals, calculations, invoices, and payments) for the selected Fee Deal Schedules, over your specified date range.

    NOTE:Fee accruals apply over the number of days in your specified period.

    Clear this check box if you do not want to calculate any accounting entries in this run. For example, you may want to post G/Ls only at month end, so you could leave this option unselected until the last Generate run of each month.

  4. In the Fee Deals field, enter one or more Fee Deal numbers or ranges to generate only for those Fee Deal schedules, or accept the default of All to generate for all active Fee Deals.

    Use a dash (-)to specify a range and a comma (,)to separate distinct numbers or ranges, as shown in the following examples.

    Example

    Targeted Fee Deals

    45

    Only fee deal 45

    25,35,43

    Only fee deals 25, 35, and 43

    25,35-43

    Fee deal 25 and all fee deals between 35 and 43 inclusive

  5. Click OK.

    The system validates your settings, and if fee generation cannot proceed for any reason, a validation message is displayed. Correct your settings as required, and try again.

    NOTE:If you run Fee Generation in batch mode via Scheduler, this message is written to the message log and the run is terminated.

    Otherwise, fee generation proceeds to completion. You can now view the generated fee base amounts, cash flows, and accounting entries; make necessary status changes, and initiate further fee processing, as described in the following topics:

    • View generated Fees and Base amounts
    • View and edit Fee cash flows (per Fee Deal)
    • View, edit, and post Fee G/L entries (per Fee Deal)

TIP: To learn more about the generation process itself, see About the Generate Fees process below.
