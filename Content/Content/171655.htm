Price Entry Settings, Futures and ETO Instrument

Use the fields in the Price Entry area of the Futures Contract Details tab to indicate how prices for the Futures contract are quoted in the market. The system uses these settings to interpret prices.

Entry Type

This setting determines which fields appear in the rest of the Price Entry area.

  • If Pricing Formula is PER TICK, select either DECIMAL or FRACTIONAL;, as required.
  • For other Pricing Formula settings this is read-only and set to DECIMAL.

TIP: If you select FRACTIONAL, the system will also display the derived decimal prices in Revaluation Analysis.

Tick Move

(Available if Entry Type is DECIMAL.)

For both fixed and variable tick contracts, enter the price increment the tick value is based on - that is, how many percentage points of yield should be added to the deal price to calculate the variable tick value.

  • For Options, this number is also used to determine the variable tick value. It determines how many percentage points of yield should be added to the Strike Price to calculate the variable tick value.
  • This is usually 0.01%, and it usually represents the smallest movement of short-term Futures prices, although a Minimum Price Move can also be specified.
  • The system calculates the notional settlement value at the Strike Price, and at the Strike Price plus the number of percentage points of yield specified here. The difference between these two values is the variable tick value on which the Option premium is based. See also Variable tick value - Options formula.

Minimum Price Move

(Available if Entry Type is DECIMAL.)

Enter the smallest amount of change allowed in the price of the contract: a positive number of up to one whole number with up to 10 decimal places.

  • For a fixed tick Futures contract, you can specify both a tick size and a Minimum Price Movement, and they may differ. This means that you can enter a Futures price where the price increment is smaller than the specified tick size.
  • You can also specify a different tick size and Minimum Price Movement for an Option based on the Future. The Future Minimum Price Movement could be 0.01 for example, when the Option Minimum Price Movement is 0.005.
  • The system uses the Minimum Price Movement (not the tick size) to verify prices entered on the Futures and ETO Deals and Interest Rates pages. It validates mid-rates and interpolated rates calculated by the Interest Rates application against the Minimum Price Movement. If they fail validation, they are rounded according to the rule specified in the Rate Rounding field (see below).

Example: if you enter a Minimum Price Movement of 0.01 and the price is 99.1, the system interprets the .1 as 10 price increments.

See also Yield-based formulas.

Quantum does not currently support different minimum price movements for different price levels.

Tick Basis

(Available if Entry Type is FRACTIONAL.)

If a price is quoted as a whole number and a fraction, enter the denominator (basis) of the fraction here. When you enter a price, the number after the "point" (delimiter) must be between 0 and this number.

The system assumes a tick size of 1. You cannot enter part fractions such as 94-3 1/2 (94 and 3-and-a-half ticks).

Examples: if the contract trades in 1/32nds, enter 32 in this field. If the contract trades in 1/100ths, enter 100.

There are exceptions. For example, the tick size for a 5-year Treasury Note traded on the Chicago Board of Trade (CBOT) is ONE HALF of 1/32 of a point.You would enter the tick size for this contract (1/2 x 1/32 = 0.015625) in 64ths, as 1/64, rather than in a number and 1/2 32nds.

Value Per Tick

If Pricing Formula is PER TICK, enter the monetary value of one tick in the contract currency; otherwise this is read-only and set to VARIABLE.

The system rounds this value according to the rounding method specified for the contract currency.

Shift Decimal Place

(Available if Entry Type is DECIMAL.)

Specify how users will enter and view values that include decimals. This is the number of places an entered value's decimal point should be shifted by to give the actual value, and it applies to the Futures and ETO Deals and Interest Rates pages, and to the Revaluation Analysis report.

Enter a positive or negative number to shift the decimal point to the right or left, respectively. The default is 0 or no shift.

TIP: If you prefer to enter prices as percentage points (without entering a decimal point), set this to -2. You can then enter 50 points when the price of the Option is 0.5 (0.5% p.a.).

Rate Rounding

Select UP, DOWN, or NEAREST. The system validates mid-rates and interpolated rates calculated in the Interest Rates application against the Minimum Price Movement for the contract. If a rate fails validation, the rounding rule specified here is applied.

For example, the mid-rate derived from a Bid rate and an Offer rate during an intraday revaluation may not be an increment of the Minimum Price Movement. The system will use the specified rounding rule to round the rate and make it a suitable increment. See Futures Rates.
