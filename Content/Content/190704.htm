Revaluing Actual and Projected Cash Flows

A normal financial deal entered in Quantum is revalued according to the revaluation method selected for that Instrument Type in the Accounting Rules. However, no such revaluation records exist for Actual and Projected Cash Flows because these financial deals do not require revaluing unless they are part of a hedge relationship.

With Hedge Accounting, you can use Actual and Projected Cash Flows to enter a "non-financial deal" as either a Firm Commitment or a Forecasted transaction. To revalue it as part of a hedge relationship, the system requires the following additional settings on the Linked Deals tab of that hedge relationship.

A Forecasted transaction is one that is expected to occur, but for which there is no firm commitment. This means that when the event does occur, it will be at the prevailing market rates.

A Firm commitment is an agreement with an unrelated party that is binding on both parties and usually legally enforceable. It has the following characteristics:

The agreement specifies all significant terms, including the:

  • quantity to be exchanged,
  • timing of the transaction, and
  • Fixed price, expressed as a specified:
    • amount - in the Entity's functional currency or in a foreign currency,
    • interest rate, or...
    • effective yield.

The agreement includes a disincentive for non-performance, significant enough to make performance probable.

NOTE: You can revalue an Actual or Projected Cash Flow only as part of a hedge relationship. Where a hedge relationship contains an Actual Cash Flow deal for which the face value has changed, the revaluation process relies upon the deal images of the Cash Flow to determine the correct values based on the image dates. See Actual and Projected Cashflows in Hedge Accounting.

Revaluation Method

Select a revaluation method. The options are:

  • FORWARD,
  • FORWARD NPV,
  • IMPORTED,
  • NPV SEPARATE,
  • NPV SPOT, and
  • SPOT.

An Actual or Projected Cash Flow is treated as one side of an FX deal. For a description of the revaluation methods see Revaluing an FX Deal.

Revaluation Currency

By default, the revaluation is performed in the base currency of the entity for the hedge relationship. You can change the setting in the revaluation currency column to NON BASE CCY.

Forecast

By default, the value in this column is set to:

  • FIRM COMMITMENT for a fair value hedge, or...
  • FORECAST for a cash flow hedge.

For other Transaction Types, it is set to OTHER and disabled.

Interest Rate Name, Rate Type

You cannot save the hedge relationship without selecting a rate in these columns, as there are no default rates for Actual and Projected Cash Flows.
