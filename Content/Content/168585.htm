Rate Type - Static

Use the Rate Type page to maintain unique Rate Types. Quantum uses Rate Types to distinguish between different uses for a single Rate Name.

For example, if you set up a single Rate Name for the BBSW rate, Quantum can apply it differently for Risk and for FRN Market Margins. The creation of the two Rate Types Risk and FRN Market Margins then enables Quantum to distinguish between BBSW-Risk and BBSW-FRN Market Margins.

More about rates

NOTES:

  • Quantum is shipped with a default Rate Type of Live selected as the default.
  • When you set up a rate record (for example, in the Interest Rates, Exchange Rates, or Volatility Rates programs) you link a Rate Name to a Rate Type, Currency, and Date. This combination uniquely identifies the rate record.
  • Similarly, when you set up a Unit Price record (in Analytics), you link a Security Identifier (identifying the Unit Security issue) to a Rate Type and Price Date.
  • When Rates-related static data is imported from Quantum into Analytics, the Replication Manager creates one Analytics Rate Name record for each combination of Quantum Rate Name and Rate Type, with the Analytics record's Code value set to the concatenated Quantum Rate Name and Rate Type values in each case.

    For example, if you have three Quantum Rate Names (A, B, and C, say) and two Rate Types (X and Y), then six AnalyticsRate Name records will be created (with Code values AX, AY, BX, BY, CX, and CY).
