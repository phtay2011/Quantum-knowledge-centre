How the Matching and Actioning Settings Work

This topic explains how the matching (External Method), Actioning, and other related settings in Confirmation Rule record entries for Deal Actions affect the run-time processing and workflow of applicable Quantum cash flows.

NOTE: These settings apply to paper and/or electronic Confirmation Rule entries on the Deal tab only.

External Method

This setting specifies the type of matching for Confirmations of applicable deal actions: select an External Method*, or MANUAL, or NONE.

*NOTE: Only External Methods whose Method Type is set to Confirmation (in External Methods) are available for selection.

If you select:

  • an External Method:
    • confirmation messages for applicable deal actions will be processed by the Confirmation Delivery Service (including forwarding to the appropriate External Matching system);
    • supported message types will be formatted according to the appropriate Electronic Confirmation Format Rule;
    • response message received will be processed by the Confirmation Matching Service (response messages for SWIFT and Finastra confirmations are supported);
    • the matching of the confirmation can optionally update the flow status to ACTIONED in Settlements (see Actioning below);
    • you must specify the following settings:
      • Require Approval - defines whether or not the deal requires full approval before a confirmation is generated,
      • Electronic Output Subfolder - the output file location on the application server,
      • Electronic File Extension, and
      • Print Paper Confirmation - allows you to optionally generate a PAPER confirmation in addition to the Electronic one, or if the Electronic one fails.
  • PAPER:
    • confirmation messages for applicable deal actions will be processed by the Confirmation Delivery Service;
    • confirmations will be formatted according to the appropriate Paper Confirmation Format Rule;
    • the matching of the confirmation can optionally update the flow status to ACTIONED in Settlements (see Actioning below);
    • you must specify the following settings:
      • Require Approval - defines whether or not the deal requires full approval before a confirmation is generated,
      • Paper Print - defines the type/s of output: a physical PRINT, EMAIL to Counterparty Contact, and/or saved to FILE.
      • File Subfolder (if the confirmation is to be saved to file) - the file location on the application server, and
      • Paper File Type (if the confirmation is to be e-mailed or saved to file) - DOC, DOCX, PDF, or RTF.
  • NONE (meaning not applicable):
    • confirmation messages for the applicable deal actions are not required, and
    • UNLESS you select Include all deal actions, they will not appear in the Confirmation Processing page for matching.

Actioning

This setting relates to the actioning of deal cash flows. It determines how and when cash flows will be actioned following deal approval and/or confirmation matching, as and when applicable.

Deal approval can be made directly on the deal or from the Deal Approvals Processing page. Also, one or two approvals may be required, as specified in the applicable Approval and Authorization Rule record.

Auto Action

Applicable cash flows will (or can) be automatically actioned regardless of the Confirmation Status, as follows.

If deal approval is...

The cash flow will (or can) be actioned...

Not required

Automatically, when the deal is saved

Required, and Allow CF Action is set to YES (in Approval and Authorization Rule)

Automatically, when the deal is saved

Required, and Allow CF Action is set to NO

Automatically, following full deal approval

Manual Action

Applicable cash flows can be manually actioned in Settlements Processing, regardless of the Confirmation State, as follows.

If deal approval is...

The cash flow can be manually actioned...

Not required

Any time after the deal is saved

Required, and Allow CF Action is set to YES (in Approval and Authorization Rule)

Before or after full deal approval

Required, and Allow CF Action is set to NO

Only after full deal approval

Any SWIFT Payment messages that may be output will be validated when the user manually actions the cash flow on the Settlements Processing page.

Match & Auto Action

(Not available if the External Method column is set to NONE.)

Applicable cash flows will be automatically actioned when the Confirmation message status changes to Manually Matched1 or Electronically Matched2, as follows.

Any SWIFT Payment messages to be output will be validated at the EFT Processing stage.

If deal approval is...

The cash flow will (or can) be actioned...

Not required

Automatically, when the Confirmation message status changes as above

Required, and Allow CF Action is set to YES (in Approval and Authorization Rule)

Automatically, when the Confirmation message status changes as above, OR

Manually if required, before or after full deal approval.

Required, and Allow CF Action is set to NO

Automatically following full deal approval only, after the Confirmation message status changes as above

1

A Manually Matched status is set when you manually apply the Match function to selected Confirmations on the Confirmation Processing page.

  • If the Matching setting is MANUAL, this is the only Matched status that will occur.
  • If the Matching setting is an External Method, this status will NOT override an Electronically Matched status.

2

An Electronically Matched status is set by the relevant External Confirmation Matching system (via its interface).

  • The Matching column setting determines which system is used.
  • An Electronically Matched status will override a Manually Matched status if the Message Lifecycle state has not changed in the meantime.

CAUTION: This setting automatically actions all future cash flows of the same type, and is therefore unsuitable for transactions that may require further actions (for example, Pending cash flows for Repo deals and Yield on Statement Money Market deals).

Match & Manual Action

(Not available if the External Method column is set to NONE.)

Applicable cash flows can be manually actioned in Settlements Processing, after the Confirmation message status changes to Manually Matched or Electronically Matched, as follows.

If deal approval is...

The cash flow can be manually actioned...

Not required

After the Confirmation message status changes as above

Required, and Allow CF Action is set to YES (in Approval and Authorization Rule)

After the Confirmation message status changes as above, and

Before or after full deal approval

Required, and Allow CF Action is set to NO

After the Confirmation message status changes as above, and

Following full deal approval only

Any SWIFT Payment messages that may be output will be validated when the user 'Actions' the cash flow manually on the Settlements Processing page.

Cash Flow Netting

Netting occurs prior to actioning the applicable deal flows. Therefore if you want to complete Netting using Auto or Manual Netting, set Actioning to either Manual Action or Match & Manual Action in the applicable Confirmation Rule record entry.

The Confirmation state is irrelevant, as netting is a payment function - it has nothing to do with confirmation.

A limit on the maximum number of cash flows that may be netted could apply, depending on the NetItemsMax setting in the Web.AppSettings.config file.
