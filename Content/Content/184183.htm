Replicate Fees on Unit Deals to Analytics

Any fee items (up to three) on a Quantum Unit Deal are now mapped to Analytics as individual Cash Flow deals. Each of these Cash Flow deals will:

  • use an Analytics Cash Flow instrument replicated* from the applicable Quantum Fee Name record,
  • share the same Analytics Deal Number as the associated Unit Deal,
  • have their own unique Deal Identifier, and
  • store the Deal ID of the associated Unit Deal for reporting purposes.

Analytics Cash Flow Instrument page

Quantum Fee Name records will be replicated as Analytics Cash Flow instruments, with the instrument's Code and Name set to the Quantum Fee Code and Fee Name. All other instrument details will either remain unpopulated or be set to default values.

These instruments will then be used any the fee cash flows replicated with each Unit deal.

Analytics Cash Flow Pricing page

When an Quantum Unit Deal is replicated to Analytics, individual Analytics Cash Flow deal records are now created for each of its Fee items. When such a record is recalled on the Cash Flow Pricing page, the following settings will apply:

  • Instrument is set to the Analytics Cash Flow Instrument name for the corresponding replicated Fee Name record (see above),
  • Deal Date and Start Date are set to the Deal Date on the Unit Deal,
  • Maturity Date is set to the Settle date on the Fee item,
  • Frequency defaults to Maturity,
  • Business Day Convention defaults to Following,
  • Pay/Receive, Currency, Amount, and Counterparty are set to the equivalent settings on the Fee item, and
  • all other properties are set from the equivalent settings on the Unit deal.

TIP: Use the Deal Number and Instrument to identify fee item when browsing for deals in the pricing pages. The associated Unit Deal ID is not displayed on the Pricing page.

Analytics Reporting

When running the following queries in Analytics, any Quantum Unit Deal fee items that have been replicated as Cash Flow deals will be handled as follows.

  • All query types - fee Cash Flow deals are available for selection (or exclusion) through the query filters definition and will be included in the query output.
  • Bank Balance or Entity Balance query - fee amounts will be included in the appropriate account or Entity balance, respectively.
  • Position or Cash Flow query - if the Report at Holding Level option in the query Assumptions is:
    • cleared - the Unit Deal ID will be available as a Database Column for the Cash Flow transaction type, and can be used to link the Unit deal with the fee Cash Flow deal/s.
    • selected - only the Holding deal is available for reporting, along with aggregated values for all units within the holding. The Unit Deal ID column, mentioned above, can be used to drill down to the Unit Holding Deal ID Database Column. The Unit Holding Deal ID can be used to link the Unit Holding deal with the fee Cash Flow deal(s).

Analytics Reconcile Utility

Reconcile queries will include any Analytics fee Cash Flow deals linked to a single Unit Deal when:

  • the Unit Holding and Include cash flows options are selected - the fee cash flow deal items will be included as part of the linked Holding, and will be reconciled against the fee items on each of the Quantum Units under the Holding;
  • the Bank account balances and the associated Bank Account option is selected - account balances will include the fee cash flows; and
  • the Bank account balances and the associated Deal option is selected - Unit deal balances will include the fee cash flow amounts.

History

Previously, fee items in a Quantum Unit Deal were not replicated in Analytics, and subsequently the fee amounts were not included in any Bank Account Balances.
