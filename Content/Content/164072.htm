Index Definition Procedures

On the Index Definition page:

To list Index Definition records

  1. Click the button.
  2. Optionally follow the Configure link to change list properties - see Configure Browse list views.

To add a new Index Definition record [Default perspective]

  1. Click New.
  2. Enter a new unique Name for the Index Calculator Definition.
  3. Enter a Short Name and Description. These do not have to be unique.

    NOTE: Currently, the only Index Formula option is TIPS, which is suitable for use with all supported index-linked bonds in Quantum that use Index Definitions. With this formula, the Index Frequency, Reference Column, and Filter Rule fields are preset and cannot be changed.

  4. In Day Count, select the Business Days Convention to be used for interpolating Base Index and Index Ratios.
  5. (Optional) Change the Offset 1 and/or Offset 2 values if necessary; zero or a negative integer from -20 to -1 in each case.

    These are the two index table offsets used to find the reference index values used to calculate the Index value for a specific date (looking backwards from that date): Ref_CPIN and Ref_CPIM, respectively.

  6. (Optional) Change the Reference Index rounding method and Places value if required.
  7. (Optional) Change the Index Ratio rounding method and Places value if required.
  8. Save the Index Calculator Definition.

This index calculator definition is now available for selection when setting up an index-linked bond Security Instrument.

NOTE: These records can also be viewed (but not edited) in the FIS Quantum Analytics perspective if the Replication Service is running.

To add a new Index Definition record [Analytics perspective]

  1. Click New.
  2. Enter a new unique Code for the Index Definition.
  3. Enter a Name.
  4. Accept the default Index Formula and Index Frequency settings (TIPS and MONTHLY).

    NOTE: Currently, these are the only supported options.

  5. In Day Count, select the Business Days Convention to be used for interpolating Base Index and Index Ratios.
  6. (Optional) Change the Ref_CPIN and Ref_CPIM values if necessary; zero or a negative integer from -20 to -1 in each case.

    These are the two index table offsets used to find the reference index values used to calculate the Index value for a specific date (looking backwards from that date).

  7. (Optional) Change the Reference Index rounding method and Places value if required.
  8. (Optional) Change the Index Ratio rounding method and Places value if required.
  9. Save the Index Calculator Definition.

This index calculator definition is now available for selection when setting up an index-linked bond Security Instrument.

NOTE: These records can also be viewed (but not edited) in the Analytics perspective if the Replication Service is running.

To edit an Index Definition record

On the Analytics perspective page, you cannot edit a record sourced from Quantum (via the Replication Service).

  1. Click to list available records, or use the Search box.
  2. Select the record you want to edit.
  3. Edit the relevant fields.
  4. Save your changes.

To delete an Index Definition record

NOTE: You cannot delete a record if:

  • transaction, settlement, or financial data would be affected, or
  • the record is sourced from Quantum and you are on the Analytics perspective page.

TIP: This procedure deletes the record completely from the database. To prevent use of the record but retain it in the Quantum database If you are on the Default perspective page, deactivate it instead (see below).

  1. Click to list available records.
  2. Select the record you want to delete (use the < and > buttons to page through, or type in the Search box).
  3. In the menu, select Delete.
  4. When prompted, confirm your action.

To deactivate an Index Definition record

(Default perspective only)

  1. Click to list available records, or use the Search box.
  2. Select the record you want to deactivate.
  3. In the menu, select Deactivate. This action will not appear if the record is already deactivated.

To reactivate an Index Definition record

(Default perspective only)

  1. Click to list available records, or use the Search box.
  2. Select the record you want to reactivate.
  3. In the menu, select Reactivate. This action will not appear if the record is already activated.
