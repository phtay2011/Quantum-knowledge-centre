Money Market & Securities Accounting Rules

Use the Money Market & Securities tab to create and maintain accounting rules for Money Market and Securities deals.

There must be at least one Accounting Rule entry that matches each type of these deals that your organization uses. If more than one, the most specific matching rule entry takes precedence. For example, entries for Instrument groups take precedence over ALL Instrument Name entries (labeled For any ... in the rules grid). Individual instruments that are members of a group will apply settings defined for that instrument before those for the group, before those for an ALL entry.

You can add, edit, and delete specific rule entries using the tab's grid menu, as follows.

NOTE: You cannot add, edit, nor delete inherited entries.

To add a new rule entry

  1. In the Money Market & Securities tab's grid menu, select New (or select an entry row and in the grid menu, select Copy).
  2. In the dialog box under Matching Criteria select:
    • the Money Market Type (Standard or Embedded),
    • one or more Instrument Name and Counterparty Name values or ALL in each case, and
    • any other customized criteria that may apply to this transaction type.

      NOTE: You cannot specify a combination of criteria already covered by an existing entry.

  3. Under General Settings:
    • set Balance Sheet Timing to the date the G/L and Revaluation accounting entries will first appear on the balance sheet: DEAL DATE or SETTLE DATE, and
    • set Principal Accounting to specify how to present the Principal amount in the Balance Sheet for Discounted and Bond Style instruments: FV (Face Value), FVC* (Face Value Contingent), or NV (Net Value).

      *FVC applies only when Balance Sheet Timing is SETTLE DATE and Apply from deal date to settlement date is selected (in Step 4).

  4. If Balance Sheet Timing is SETTLE DATE, then under Contingent Accounting Settings:
    • optionally select Apply from deal date to settlement date to post contingent accounting entries between those dates, and set Deal to Settlement Amount to FACE VALUE, and/or...
    • if the Instrument Name includes any with Formula Timing set to YIELD PERIODIC or YIELD ON MATURITY: optionally select Apply from settlement date to maturity date to post contingent accounting entries between those dates, and set Settlement to Maturity Amount to FACE VALUE PLUS INTEREST.
  5. Under Accrual Settings select any of the following options to post the relevant accruals to Profit & Loss:
    • Accrue interest income/expense,
    • Accrue capital premium/discount,
    • Accrue adhoc amount,
    • Accrue face value index, and/or...
    • Accrue issue fee (applies only when Accrual Method is AMORTIZED COST).
  6. If you selected Accrue interest income/expense in Step 5, then select ACCRUE or AMORTIZE to specify whether to amortize or accrue the ad hoc interest, and select the associated Interest Translation date:
    • POSTING DATE CCY TRANSLATION - to use the relevant FX Rate on the day the accrual entries are posted, or...
    • CASHFLOW DATE CCY TRANSLATION - to use the FX Rate on the day the cash flow to be accrued/amortized is posted. Where the cash flow being accrued is in the future, the posting date should be used.
  7. If the Instrument Name includes any with embedded options (Callable Bonds) AND you selected Accrue capital premium / discount in Step 5, select the Capital Accrual Period:
    • FULL (the default) - to accrue or amortize the capital over the entire term of the bond deal (from Deal Date to Maturity Date), or...
    • FIRST EXERCISE - to accrue or amortize the capital over the term of the embedded option, from Deal date to First Exercise Date.

      TIP: In both cases, the actual dates used are those entered for the specific deal.

  8. If you selected Accrue capital premium / discount in Step 5, select the Capital Translation date (the options are as for Step 6).
  9. If you selected Accrue adhoc amount in Step 4, then select the associated Adhoc Amount Translation date (the options are as for Step 6, but are prefixed ACCRUE and AMORTIZE, respectively).
  10. If you selected Accrue face value index in Step 5, select the Face Value Index Translation date(the options are as for Step 6).
  11. If you selected Accrue issue fee in Step 5, select the Issue Fee Translation date(the options are as for Step 6).
  12. Select the Accrual Method to calculate the accrual of Interest or Capital amounts: AMORTIZED COST, STRAIGHT LINE, YIELD CUM, or YIELD EX.
  13. Select the Interest Accrual Day Count option: USE NUMBER OF DAYS IN PERIOD or USE DEAL INCOME DAYS BASIS.
  14. If you selected Accrue capital premium / discount in Step 5, select the Capital Accrual Day Count option: USE NUMBER OF DAYS IN PERIOD or USE DEAL INCOME DAYS BASIS.
  15. If you selected AMORTIZED COST in Step 12, then under Amortized Cost Settings:
    • optionally select Apply Accounting Calendar and select the Accounting Calendar to use that calendar's dates for calculating amortization amounts (otherwise the deal's Coupon dates will be used), and/or...
    • select the On Closeout or Unwind option for handling partial closeouts and unwinds: APPORTION EXISTING AMORTIZATION SCHEDULE or CALCULATE NEW AMORTIZATION SCHEDULE.
  16. Under Revaluation Settings, select Revalue money market deals* if you want to create revaluation postings for those deals only, and as required, select:
    • Money Market Reval Method (BILLS, BILLS NET, PROJECTED, PROJECTED NET, or IMPORTED),
    • Posting Method (REALIZED or UNREALIZED),
    • Interest Rate* and Interest Rate Type*, and
    • Add margin to base rates* and Margin Rate Type*.

      * Mandatory settings when Hedge Accounting applies.

  17. Under Revaluation Settings, select Revalue security deals* if you want to create revaluation postings for those deals only, and as required, select:
    • Security Reval Method (as for Step 15 plus FRN-COUPON, MARKET CUM, or MARKET EX),
    • Posting Method (REALIZED or UNREALIZED),
    • Interest Rate* and Interest Rate Type*, and
    • Add margin to base rates* and Margin Rate Type.

      * Mandatory settings when Hedge Accounting applies.

  18. If you selected Accrue interest income/expense in Step 5, then select the associated Adhoc Interest in Book Value option (INCLUDE or EXCLUDE).
  19. If you selected Accrue adhoc amount in Step 5, then select the associated Adhoc Amounts in Book in Value option (INCLUDE or EXCLUDE).
  20. Under Translation Settings, optionally select Apply deal level currency translation to apply deal-level translation to the specified ledger.
  21. If you selected Apply deal level currency translation in the previous step, optionally select Summarize unrealized postings on principal repayments to apply summary (Net) posting of unrealized deal level G/L entries for multiple principal repayments on a deal.
  22. Click Save. The new rule entry is added to the grid.

To edit an existing rule entry

  1. Click an entry value in the grid (or select the entry and in the grid menu, select Edit).
  2. In the Edit window, edit the settings as required (see above), and click OK.
  3. Save your changes.

To delete an existing entry

  1. Select the entry in the grid and in the grid menu, select Delete.
  2. In the dialog box, click OK to confirm.
