Events tab fields, Imported Hedge Relationships

The Hedge Relationships Deal Importer supports the following QID-format fields, from the Events tab.

A setting of Y indicates that the corresponding check box in Hedge Relationships would be selected (on). N indicates that it would be cleared (off).

NOTE: Values that must be calculated by Quantum are not imported.

Many of these fields are dependent on others, and it is important that you include them in the correct order. Others share a setting with another field, and you cannot access them directly. For example, Offset Start for Release Transfer OCI/MTM defaults from Offset Start for Release OCI/MTM.

NOTE: These settings default from the Events tab of Hedge Strategy. Changing them in this way will cause calculations for the deal to differ from those for other deals based on the same Hedge Strategy.

Field code

Field name

Example and Notes

frez_oci

Freeze until next user action

frez_oci=N

Specify Y to halt (freeze) Hedge Accounting for this Hedge Relationship, following the user-defined event. No further entries will be posted, and nothing will be reversed unless the Hedge Relationship matures.

You can edit the timing of the freeze if necessary.

Quantum calculates an actual freeze date.

fr_os_st

Freeze From

(Freeze until next user action)

fr_os_st=INCEPTION

The timing of an action can be specified using an ACTUAL DATE, or be calculated (by the System) relative to the event that triggers it. Where it is to be calculated, specify MATURITY or INCEPTION to change the starting point for the calculation. The offset is specified relative to this.

fr_ofset

Freeze Offset

(Freeze until next user action)

fr_ofset=2 MONTHS

Where the timing of an action is calculated relative to the event that triggers it, specify any offset relative to MATURITY or INCEPTION (as specified for fr_os_st).

Select a unit of time and enter the required number in front of it. Use the same unit of time for all Release and Transfer Offsets. The Offset selected for the release of OCI/MTM is automatically applied to its transfer.

Quantum will not adjust the result of this calculation for Business Days Conventions. The Hedge Relationship will do that.

rel_oci

Set release date

rel_oci=Y

Specify Y to release the following amounts to Profit and Loss following the user-defined event:

  • For a Cashflow hedge, all Other Comprehensive Income (OCI) adjustments for the Hedge Relationship
  • For a Fair Value hedge, all mark-to-market adjustments for the Hedge Relationship

You can also edit the timing of the release if necessary.

Quantum calculates an actual release date.

re_os_st

Release From

(Set release date)

re_os_st=MATURITY

The timing of an action can be specified using an ACTUAL DATE, or be calculated (by the System) relative to the event that triggers it. Where it is to be calculated, specify MATURITY or INCEPTION to change the starting point for the calculation. The offset is specified relative to this.

The Offset Start selected for the release of OCI/MTM is automatically applied to related transfers and release transfers.

re_ofset

Release Offset

(Set release date)

re_ofset=1 WEEK

Where the timing of an action is calculated relative to the event that triggers it, specify any offset relative to MATURITY or INCEPTION.

Select a unit of time and enter the required number in front of it. Use the same unit of time for all Release and Transfer Offsets. The Offset selected for the release of OCI/MTM is automatically applied to its transfer.

Quantum will not adjust the result of this calculation for Business Days Conventions. The Hedge Relationship will do that.

tran_oci

Transfer to another entity at release date

 

tran_oci=Y

The group of Transfer OCI/MTM settings controls postings between different General Ledgers in Quantum. The fields identify when adjustments are released by the Entity and transferred to another Entity, and when those transferred adjustments are released to Profit and Loss.

Specify Y to transfer the balance of the adjustments on a Hedge Relationship from one Entity to another ,following the user-defined event.

You can edit the Entity field if necessary, as well as the timing of the release of transferred adjustments.

Quantum calculates an actual transfer date.

Transfer - Offset Start and Offset

OCI or MTM adjustments must be released by one Entity before they can be transferred to another. Consequently, the Offset Start and Offset for the transfer default from the Release fields above. If none were set, these fields remain empty, and you cannot change them here.

tran_ent

Transfer To Entity

(Transfer to another entity at release date)

tran_ent=FIS UK

Specify an Entity where the accounting is to take place (one set up for Hedge Accounting).

Quantum will use the same Entity for the release of the transferred adjustments. This obviously cannot be the same as the Entity on the Deal Details tab for the Hedge Relationship.

tr_r_ofset

Transfer Offset

(Transfer to another entity at release date)

tr_r_ofset=4 DAYS

Specify when any transferred adjustments will be released to Profit and Loss, relative to an Offset Start of INCEPTION or MATURITY. This date must be later than the date of the transfer. Select a unit of time and enter the required number in front of it. Use the same unit of time for all Release and Transfer Offsets.

Quantum applies the Offset Start selected for the release to both the transfer and the release transfer. It calculates (or allows you to enter) an actual release transfer date in the field on the right.

The Entity for the release of the transferred adjustments is always the same as the Entity named for the transfer. You cannot change this here.

maturity_amortize

Amortize from maturity date to release date

maturity_amortize=Y
