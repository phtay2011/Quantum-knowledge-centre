Password Reset Process

(First introduced in Release 6.3.1 SP, August 2016.)

Password control has been improved to address previous gaps in security and provide a more secure password reset process.

Implementation

User Page

A new Email Address field has been added to record a valid email address for transmission of credentials and other security-related information between the system and user.

Previous functionality has been changed as follows:

  • New users are emailed temporary credentials and instructions to set up their account at the email address provided in the Email Address field.
  • The password reset function in the page menu (Reset Questions and Password) will now use the email address provided in the mandatory Email Address field to send temporary credentials and other information directly to the user.

  • With Approvals enabled for Users and Groups, the following rules are now enforced.
    • New users (including their email address) are subject to approval via the Unapproved User page by one or more Static Data approvers with the necessary permissions.
    • A user with the necessary administrative permissions cannot create a new user AND approve the new user using the Unapproved User page; another user (or two other users, when the approval Level is set to Dual) with the necessary permissions must approve the new user.

Implementation Notes

Change of Responsibilities

  • By sending an email to the user to change the password, the responsibility of setting a password now lies with the user. The administrator will never get to see the user's password, making the approach more secure.
  • The responsibility of choosing an email address for a user has now shifted from the user to the administrator. This gives an organization more control over where outgoing emails are sent.

Email Address Field

  • The Email Address field is required to be filled in for new users or when an existing user is saved.
  • When the SMTP settings are configured with a default email format, the Email Address field is populated when:
    • the Full Name field is entered for new users, or
    • an existing user record without an email address is opened.

Setting an Email Address for Existing Users

  • The Password Reset feature will work only if an email address has been set up for the user.
  • An administrator will need to go through all existing users and set an email address.
  • If existing users have email addresses that are derived from the Full Name field, do the following:
    1. configure the SMTP settings with the default email format and domain, and
    2. open each user, verify the email has defaulted correctly, and save.

  • If the existing user email addresses cannot be derived from the Full Name field, do the following:
    1. configure the SMTP settings to have no default email format applied, and
    2. open each user, enter the email address manually, and save.
