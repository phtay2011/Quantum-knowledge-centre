EFT Setup - Static Data and Rules

NOTE: Default perspective only.

External Method

An External Method record must be present for each type of EFT output, as follows:

For SWIFT

  • Name, Short Name, and External Name are set to SWIFT,
  • Method Type is set to Payment,
  • Export Source is set to SWIFT (the name of the External Method Source),
  • EFT Plug-In / Template is blank or set to SWIFT, and
  • Enable Identifiers is selected.

For TARGET - 2020.3 onwards

From 2020.3, we stop using the TARGET2.txt file and include the addressee details in the EFT XML Framework. You can define an external method for TARGET EFT payments.

For more information about TARGET, read Details of TARGET Addressee.

For Other EFTs

  • Name is set to a unique value, meaningful for the EFT type,
  • Method Type is set to Payment,
  • Export Source is set to the name of the corresponding External Method Source record,
  • EFT Plug-In / Template is set to the name of the required EFT Plug-In, and
  • other settings are as required (similarly to those for the External Methods above).

NOTE: Other Payment Methods may require development of special EFT Plug-Ins.

Entity

To populate the Basic Header Block with an Entity Identifier for the Bank Account Owner (the sender of the EFT) appropriate for the External Method (if required):

  1. On the Entity record, create an identifier with the appropriate name - for example, SWIFT, BIC code, or EDIFACT.
  2. Populate that field with the value for that entity (normally supplied by the relevant authority).

Counterparty

To populate the Basic Header Block with an Identifier for the Counterparty receiving bank (the recipient of the EFT) and/or the Intermediary bank (if required):

  1. On each applicable Counterparty record, create an identifier with the appropriate name - for example, SWIFT, BIC code*, or EDIFACT.
  2. Populate that field with the value for that counterparty (normally supplied by the relevant authority).

*NOTE: If using the TARGET2 External Method, you can optionally change the default BIC Code to that of "Our" (the Deal Entity's) Bank Account - see External Method Sources under EFT Setup - Configuration Pages.

Counterparty Bank Account

(Optional with the SWIFT External Method.)

To suppress EFT output for a Counterparty account:

  • Set Payment Category to EFT Not Required.

To output SWIFT MT900 and/or MT910 Advice messages instead of standard SWIFT messages (MT101, MT103, 103+, 200, 202, 203, and 210):

  • Set Payment Category to Debit / Credit Advice.

Bank Account Defaulting Rules

Bank Account Defaulting Rule records must cover all required Matching Criteria (Entity, Transaction Type, Instrument, Counterparty, and so on), with Instructions entries for each Bank Account using EFTs specifying the required EFT Default Payment Method (the External Method name) and the default Entity Identifier if required.

Bank Account

For each Bank Account from which EFT payments will be made (and receipts credited to), ensure that:

  • the correct Bank Account Defaulting Rule will be applied, and
  • the Allowable Payment Methods include the required EFT External Method.

NOTE: The Default Payment Method specified in the Bank Account record applies only to Bank Account Transfers. For other transaction types, the default payment method is the one specified in the applicable Bank Account Defaulting Rule record.

Stock Account Defaulting Rules

(Optional with the SWIFT External Method.)

If SWIFT MT540-543 EFT stock flow messages are to be output for any Stock Accounts, ensure that Stock Account Defaulting Rule records are present that:

  • cover all required Matching Criteria (Entity, Transaction Type, Instrument, Counterparty/Broker, and so on)
  • have Instructions entries for each Custodian and Stock Account number using EFTs that specify the Default Payment Method as SWIFT, and the Entity Identifier if required (Default SWIFT MT Format is optional here).

Stock Account

(Optional with the SWIFT External Method.)

For each Stock Account for which SWIFT MT540-543 EFT stock flow messages are to be output, ensure that:

  • the correct Stock Account Defaulting Rule will be applied,
  • Allowable SWIFT MT Formats include the supplied MT540_543_Supported_Fields.qmt file and/or any modified versions and additional files,
  • Default SWIFT MT Format is set to the supplied MT540_543_Mandatory_Fields.qmt file OR a modified version if required, and
  • Default Payment Method is set to SWIFT.
