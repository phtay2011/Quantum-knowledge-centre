About XTC Configuration XML Data records (Confirmations)

The XTC Configuration XML Data record* for each Confirmation External Method::

  • is accessible only via the XML Data page*,
  • has Type set to XTC Configuration (corresponding to the earlier filename suffix*),
  • contains specific XML code (see Structure below),,
  • gives the names for the Data Template, Layout, and Stylesheet XML Data records used for processing a particular Confirmation message type,
  • can be attached to a default Electronic Confirmation Format Rule for a given Deal Action, and
  • can use, or be based on, the seed XML Data record for the message type and named SeedXMLConfig_MTnnn, where nnn is the message type - for example SeedXMLConfig_MT300, and so on.

    • CAUTION: If you edit the system-supplied seed record, your changes will be lost on any upgrade of your database. Always edit copied and renamed records of this type.

*NOTE: Prior to Release 2020.2, this data was available only in a separate XML-formatted file with the filename suffix .xtc.

Structure

<!--Confirmations Configuration for MT...--> <Configurations>
  <Configuration>
    <Messages>
      <Message>
        <Output/>
        <Input>
          <DataTemplate name="..."/>
          <Layout name="..."/>
          <Stylesheet name="..."/>
        </Input>
      </Message>
    </Messages>
  </Configuration>
</Configurations>

Where:

Configuration

Only one Configuration element is allowed.

No attributes are supported.

Message

Only one Message element is allowed per Configuration element.

No attributes are supported.

Output

Only one Output element is allowed per Message element.

Attributes are:

  • name: Output file name (optional)
  • path: Output file directory path (optional)
  • extension: Output file extension (optional)
  • separator: Where an output file name is specified, this provides the separator to be used between file name and the <actions.action_no> extension (optional)

NOTE:

  • <actions.action_no> will be used in all output file names to help ensure uniqueness. Where more than one message is produced from the one action (for example, Cancel/New Confirmations), then one of the messages will have a 001 extension by default.
  • Where any of the above attributes are not specified, the output path and filename default to applicable settings in Confirmation Rules, Confirmation Processing, and/or External Method Sources.

Input

Each Message element must have a DataTemplate, Layout, and Stylesheet element.

Each of these elements are XML-based, and can be either a linkage to an XML Data record, or contain the required XML content.

Where it is a linkage to an XML Data record, each element must contain:

  • name: the name of the relevant XML Data record (mandatory)

Example (MT300 Confirmations)

<?xml version="1.0" encoding="UTF-8"?>
	<!--Confirmations Configuration File for MT300-->        
<Configurations>        
  <Configuration>        
    <Messages>        
      <Message>        
        <Output/>
        <Input>        
          <DataTemplate name="SeedXMLDataTemp_MT300.xml"/>        
          <Layout name="SeedXMLLayout_MT300.xml"/>        
          <Stylesheet name="SeedXMLStylesheet_SWIFTFIN.xsl"/>        
        </Input>        
      </Message>        
    </Messages>        
  </Configuration>        
</Configurations>
