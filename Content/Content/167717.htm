Balance Match Scripts

Use the Balance Match Script page to view and maintain balance-level match scripts.

Changes to all Match Scripts and Script Chains are recorded and can be viewed using the Audit Reporting page.

Balance-level scripts:

  • apply only to Prior Day Matching (not Intraday), and
  • enable you to automatically match, over a specified period, the sum of the cash flows (including external cash flows) against the difference between the Closing and Opening Ledger OR Available Balances* on the bank flows side. This is termed matching at the Balance level, or Balance-level matching.

*TIP: To select which balance applies, select either Ledger Balance or Available Balance under Calculate Balance using on the Details tab of the Edit Balance Match Script page (however, Bank flow balance adjustments always apply to the Ledger Balance).

You cannot match at the Balance level if:

  • the specified period contains any items already matched at the balance level, or
  • there is no Closing Balance for the End Date of your chosen reconciliation period.

When a Balance-level script is run:

  • Individual bank flows are not summed and do not affect the Balance match process at all, therefore you can reconcile (at the balance level only) without loading any bank flows - provided you do load the relevant statement balances.

    NOTE: However, if individual bank flows are loaded, your custom Import service can be set up to check that the bank flows total matches the relevant statement closing balance. For more details, see Importing Cash Management Data.

  • Bank flow balance adjustments are treated the same as imported Ledger balances,.
  • If the Include All Data option is selected in the script, the Opening Balance is assumed to be zero

If the script criteria are satisfied:

  • if a tolerance is specified in the script, a single balancing adjustment is made if necessary (with either the current date or Bank Statement Balance Close Date, as specified on the Balance Match Script page's Auto Adjustment tab;
  • the following contributing items are stamped with a unique Balance Match number:
    • all bank flows, cash flows, and external cash flows within the period being balance-matched,
    • the statement containing the Closing Balance at the specified end date, and
    • the statement (if any) containing the Opening Balance at the specified start date.

A statement may therefore have two Balance Match fields: one for the opening balance and one for the closing balance.

About the Opening Balance Calculation

For balance-level matching, the Opening Balance (for the start date specified in the script) is calculated as follows.

The opening balance is assumed to be zero if:

  • the Include All Data option is selected in the script, or
  • no closing balance exists prior to the start date, and no Bank Flow Balance adjustment is present.

Otherwise, the opening balance is set to the latest Closing Ledger Balance held for the account, prior to the start date specified in the script. This process does not use the business day convention.

NOTE: You can optionally validate that the Opening balance equals the previous Closing balance at importation time (if selected in the Import service).
