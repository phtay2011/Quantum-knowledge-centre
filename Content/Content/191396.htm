Capitalizing Interest Items on Money Market Deals

This type of customization is valid only on a Money Market or Security deal using one of the following YIELD Formula/Timings:

  • YIELD ON STATEMENT,
  • YIELD PERIODIC,
  • YIELD ROLLING,
  • YIELD CDI ROLLING, or
  • YIELD COMPOUND ROLLING.

Two methods apply - one for YIELD PERIODIC, one for the other Timings, as described below.

For all Timings except PERIODIC

Use the bulk Insert or Replace Forward function, adding CAPITALISE INTEREST items or changing INTEREST items to CAPITALISE INTEREST as required and where valid.

For more details, see:

  • Bulk Replace and Insert for Deal Map Items
  • Yield Formulas - Customization Restrictions

For PERIODIC Timing only

To capitalize Interest items on a new (un-customized) saved deal:

(You must have Customize Deal Module Security permission for the Dealing module.)

  1. In the page menu, select Capitalize Interest.

    The Capitalize Interest dialog box opens.

  2. To capitalize Interest items, select the check box at the left of the required rows.
  3. To cancel capitalization for an interest item, clear the check box for that row.
  4. Click OK to return to the deal page.
  5. Click Save.

    Your changes will be displayed on the Deal Map tab, with CAPITALISE INTEREST and INTEREST (ACC) items displayed for any capitalized interest items. Also:

    • the G/L Entries tab will show the required Accruals postings (if any), and
    • (if there no other changes) the deals will be badged as Customized-Minor.

To change Interest items on an already customized deal:

(You must have Update Customized Deals Module Security permission for the Dealing module.)

The procedure is as above EXCEPT THAT:

  • If you amend a Deal Map item that has already been Actioned, or add an item that impacts items that have already been Actioned, you may have to manually reprocess those items.

    This is indicated by the Flow Lifecycle value, which you can display as a grid column on the Cash Flows tab, in Settlements and Confirmations results grids, and use as a filter in deal or flow queries or result views (for more details, see Flow Lifecycles, States, and Phases).

See also: Yield Formulas - Customization Restrictions.
