Workflow Parser

Use the Workflow Parser page to define a parser that can convert incoming data to a data source-like structure so that a workflow action can act on the data - for example, to create an Actual Cash Flow deal database record based on a fee invoice received from a regulatory authority.

The parser is a tool for importing data into Quantum. It is a part of Workflow, and it can be used with Data Transfer Location (DTL) when:

  • the system-wide Process Workflows option is selected on the Operational Settings page,
  • the Static Data Import Service is running,
  • you create a location with a Purpose of “WorkflowIn” and define a parse,
  • you configure the action to execute for each occurrence found in the file, and
  • incoming text input is found - for example, a file is found in the WorkflowIn\Import folder at the Location specified in the DTL record.

NOTE: You cannot modify nor delete an in-use parser - that is, one specified in a Data Transfer Location (DTL) record.

To set up a parser, provide it with an example of the sample data you want it to process. Set the sample data first and then use Generate Columns… to paste in the same text. Click Generate and the system fetches the possible columns from the sample data. Use the Preview tab to view how data given in the sample will be parsed.

Adjust Columns by changing their displayed name or data type, or by removing them from the output of the parser. You cannot create new columns; you may select only those provided from the sample. See Data items for more details.

Exporting and Importing Parsers

Parsers can be exported in any of several formats (CSV, Excel, JSON, PDF, and XML where valid). The Workflow Parser page's browse list menu includes an Export option that will export the parsers displayed in the browse list to a single file in your selected format. Any Parser sample data is included.

JSON-formatted file from parser pages can be imported for the following functions: create, update, delete. Parsers can also be imported via the Static Data Import Service.

Data items

Name - the internal name that the system uses to recognize the parser record.

Input Format - select Delimited, Text or XML.

Delimiter (available when Input Format is Delimited) - select Comma, Pipe, Semicolon, or Tab.

Description - a short description of the use of the parser.

Data has a header row (available when Input Format is Delimited) - by default, this check box is checked. To parse a delimited file without a header row, deselect the check box. Once the columns are generated, the check box becomes read-only.

Data Starts at Line (available when Input Format is Delimited, or Text) - specify the line number from which data starts, so that you can distinguish and parse the system header and data appropriately. By default, the value is set to 1. Click 1 to open the Expression Builder window and change the value. Once the columns are generated, this field becomes non-editable.

Data Ends at Line (available when Input Format is Delimited, or Text) - specify the line number from which data ends, so that you can distinguish and parse the system footer and data appropriately. By default, the value is the following expression - Count(Lines([Raw Data])). Click it to open the Expression Builder window and change the value. Once the columns are generated, this field becomes non-editable.

Data has multiple items (available when Input Format is set to Text) - by default, this check box is deselected. When you check the check box, the Item Separator field is shown.

Item Separator parses the data. Based on the item separator value specified here, data set is available in the next row. If the value specified in the Item Separator field is not present in the Sample data, the entire data is considered as one item.

NOTE: Newline can be represented in the Item Separator field by using any of the following characters:

  • \r\n
  • \n
  • \r

Parser tab, Payload area (when no Columns grid is displayed) - click Generate Columns to open a dialog box; in the text box enter or paste correctly formatted sample data, using the specified Input Format; then click Generate. The dialog box closes and a Columns grid is now displayed.

Parser tab, Columns grid (available when Input Format is set to Delimited or XML) - to add an ignored column click Add, and when all columns are added, the Select Columns lookup table will be empty. To edit an existing column, select the row and select Edit in the grid menu, then change as required. To delete a column, click the x at the right of the row.

  • Payload Name (read-only) - the unique descriptor for the column.
  • Display Name - the (editable) displayed name for the column; defaults to the Payload Name.
  • Data Type - select Text, Date, Yes/No, Amount, or Number.
  • Default Value - a default value of the correct type. If you don’t set a default value, the system sets the following default values for different data types: Blank for Text or Date, 0 for Amount or Number, and No for Yes/No.

Parser tab, Calculated Columns grid - to enhance the parser calculated columns can be defined. These columns are expressions based on the parsed columns. The calculated columns can be used to transform or enhance the incoming data (e.g. Left([TicketNumber],10)).

To add a new calculated column click New, and enter the items listed. To edit an existing input, select the row and select Edit in the grid menu, then change as required. To delete an input, click the x at the right of the row. To copy an existing input, select the row and select Copy in the grid menu, add a new name for the copied calculated column, and select OK.

  • Name – a unique descriptor for the calculated column.
  • Type – select Text, Date, Yes/No, Amount, or Number.
  • Expression - use expressions to add more complex logic. For more details search for Expression Builder in Knowledge Center.

Configured calculated columns are available in DTL > Action’s Build Expression > Context, so you can assign them to action inputs.

In the Calculated Columns grid, you can use the following items (available when Input Format is Delimited or Text):

  • Raw Data – the entire data in the sample file.
  • Before Data – the header of Raw Data.
  • Data – the data between Before Data and After Data.
  • After Data – the footer of Raw Data.
  • Data Item (available only when Input Format is set to Text) - the data between separators. Combining Data Items creates the Data.

These items are available in Expression Builder > Context > Data Fields.

You can parse a file’s header (Before Data) and footer (After Data) by adding an expression in the calculated column.

Delimited Data and Sample Data are available in Expression Builder > Context > Data Fields > Data / Raw Data.

File Name can also be parsed. The File Name field is available in DTL > Action’s Build Expression > Context, for a selected parser.

Preview tab, Sample Data box (read-only) - initially displays the sample data you entered in the Generate Column dialog box. To change, select Bulk Replace in the page menu and in the Bulk Replace dialog box, either edit directly, or copy in correctly formatted data.

Preview tab, Parsed Columns grid - displays the parsed output for the Sample Data (if any), with the data item names as the column and column headings and each data line as a grid row with the parsed values. If the sample data does not have any values, default values will be shown.

If an error occurs in parsing the input, an error message appears.
