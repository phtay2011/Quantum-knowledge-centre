Holidays tab field descriptions

These fields apply to individual holiday entries (grid rows) on the Holidays tab, for the currently selected Business Day calendar Rule record. To add a new holiday entry or edit an existing one, see Maintaining Holiday Rule Entries.

Holiday Name

The name identifying the holiday.

Observable

(Mandatory.)

Select one of the following:

  • Ignored - to never observe this day itself as a holiday, but to enable other holidays to reference it in theirBased On setting.
  • Anniversary Rule - to observe the holiday on:
    • the following Monday if the actual day is a Friday, Saturday, or Sunday,
    • the actual day itself if it is a Monday, otherwise...
    • the preceding Monday (unless those who gazette holidays decide on some other day).
  • Monday Following - to observe the holiday on the following Monday (unless the calculated date falls on a Monday).
  • Actual Observed - to observe the holiday on the actual day, whatever the day of the week.
  • Not Weekend - to observe the holiday on the actual day unless it is a Saturday or Sunday, when it is observed on the following Monday - for example:
    • Christmas Day should be defined as 25 December, Not Weekend, and
    • Boxing Day should be defined as 1 Day after Christmas Day, Not Weekend.
  • Monday to Saturday - to observe the holiday on the actual day unless it is a Sunday, when is observed on the following Monday.
  • Modified Weekend - to observe the holiday on the closest possible business day - that is, on the previous Friday if the actual day is a Saturday, on the following Monday if it is a Sunday, otherwise on the actual day.

Start Date and Expiry Date

(Invalid if a Fixed Date is specified; otherwise optional.)

The dates between which the holiday entry applies. Outside that period, the entry does not apply.

  • When entered, Start Date must precede Expiry Date.
  • Blank values indicate no restricted period.

Fixed Date

(Mandatory when the holiday cannot be deduced from other field values; specifically when neitherDay nor Day and Month specified.).)

The actual date the holiday will be observed - for example, the system provides Fixed Dates for Good Friday up to 2050.

TIP: Use Day and Month instead of repeating the same Fixed Date in every year.

Based On

(Optional; not required if a Fixed Date is specified.)

Select an existing Holiday Name to be used in conjunction with Day (see below) to specify a holiday that is a set period of time after (or before) that holiday - for example, Easter Monday is based on Good Friday plus three days.

Day

(Mandatory when neitherFixed Date nor Day and Month specified.)

This value's meaning varies depending on other field values. When Day is used in conjunction with:

  • Based On and Day Name - the Day value gives the ordinal number of the specified Day Name relative to the Based On holiday date. For example, if Based On is New Years Day, Day Name is Friday, and Day is:
    • a small positive integer - the holiday will occur that many Fridays after the Based On holiday date (therefore if Day is 1,this holiday will be observed on the Friday following New Years Day),
    • zero - the holiday will occur EITHER on the Based On holiday date if that is a Friday, otherwise on the Friday before that date, or
    • a small negative integer - the holiday will occur EITHER that many Fridays before the Based On holiday date if that is a Friday, otherwise that many plus one Friday before that date (therefore if Day is 1,this holiday will be observed on EITHER the Friday before New Years Day if that is a Friday, otherwise two Fridays before it).
  • Based On only (with no Day Name value):
    • a positive integer gives the number of days the holiday will occur after the Based On holiday - for example, Easter Monday is 3 Days after Good Friday),
    • a negative integer gives the number of days the holiday will occur before the Based On holiday, and
    • zero indicates the same day.
  • Day Name and Month (with no Based On value):
    • a positive integer gives the ordinal number for that day in the month (for example, the 3rd Monday in October).
    • zero or a negative integer have no meaning.

Enter 0 if you do not want to apply an offset to the holiday date selected.

NOTE: The system will display a warning if you do not enter a value in the Day field.

Day Name

(Optional.)

When this value is used in conjunction with:

  • Day and Month (with no Based On value) - Day gives the ordinal number for that Day Name in the month (for example, the 3rd Monday in October).
  • Based On and Day - Day gives the ordinal number of the specified Day Namerelative to the Based On holiday date (see Day above for examples).

Month

(Optional.)

Use this value in conjunction with Day and Day Name, where Day gives the ordinal number for that Day Name in the Month (for example, the 3rd Monday in October).

Day and Month

(Mandatory when neitherFixed Date nor Day specified; otherwise invalid.)

Enter the actual day and month for a holiday occurring on the same day and month each year, with Observable set as required for any adjustment - for example, enter 25 December for Christmas Day, with Observable set to Not Weekend.
