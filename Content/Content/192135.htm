Preparing Quantum to Use an External Matching System for Deal Confirmations

You must define the external method sources, and the external methods before you can prepare Quantum to use an external system for matching deal confirmations.

Refer to documentation from the appropriate matching service for information about file naming standards and other requirements (for example, Finastra requires a registered User ID and password).

Specify the Entity Identifiers for the External Matching System

  • For each applicable entity, on the Identifiers tab specify the required Entity Identifier/s for the External Matching System (normally allocated by the matching service; note that Finastra requires two). If the entity does not exist, then create the entity record.

Select the Counterparty Identifiers for the External Matching System

  • For each applicable counterparty, on the Identifiers tab specify the required identifier for the External Matching System (normally allocated by the matching service). If the counterparty does not exist, create the counterparty record.

(Optional) Apply SWIFT Local Authentication

  • Set the option and keys to apply the SWIFT Local Authentication algorithm (SA-2 or HMAC-SHA256) to your SWIFT and/or Finastra (Misys) confirmation output messages (as well as to SWIFT EFT output).

Select the type of transactions that require external matching

  • In Confirmation Rules, select the types of transactions that require external matching on the Deal tab. Select an appropriate Payment Method in the Matching column.

    NOTES:

    • Do not use the Settlement or Deal Ticket tab. Quantum does not support external matching for settlement (cash flow) or Deal Ticket confirmations at present.
    • Special Location Codes, Rate Names, and Analysis Codes are required for some Deal Confirmations with SWIFT Accord. These must be initially set up.
    • If you are using SWIFT Accord with XML Confirmation templates, you must also set up defaults for these templates.

Quantum is now ready to send data files to an external matching system, upload responses from these systems, automatically match responses against Confirmations you have sent, and change the deal status accordingly.
