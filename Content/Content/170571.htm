Netting Pool Administration Procedures

On the Netting Pool page:

To select an existing Netting Pool

  1. Click .

    The first browse page of netting pools is displayed.

  2. Use the > and < buttons to page forward or back through larger lists, or...

    Start typing the Pool id in the Search field to reduce the list to those matching your entered value.

    TIP: You can also click the Configure link in the Browse window and filter which pools are displayed.

  3. When the pool you want is located, select it.

To add a new Netting Pool

  1. Click New.
  2. Enter a unique Pool identifier and the following Details area values (all are required):
    • Name - a descriptive name for the pool, displayed in lookup lists and can be used for reporting purposes
    • Top Entity - the code of the entity that will act as administrator of the pool; also the default Counterparty on a Netting Request or Netting Offer
    • Trans Type - the type of intercompany invoice to be netted in the pool: PAYABLE or RECEIVABLE
    • Common Currency - the common currency to which all payable/receivable flows are converted before netting; also the Rate Currency against which all other session rates are quoted

      For example, if the Rate Currency is EUR, then all session rates will have EUR as either CC1 or CC2.

    • Instrument - an existing Quantum instrument applying to the pool; ensures that the pool will include transactions based on only that instrument.
  3. Click Save.
  4. On the Phases tab, click on each Status value; in the dialog box select or clear the Can Add, Can Mod, and Can Del options, to specify whether or not pool participants may add, edit, and delete transactions when any session is in that status, respectively; then click OK. Repeat for each status row.
  5. On the Sessions tab, add at least one session for the pool: select New from the tab menu; in the dialog box enter a Name for the session, enter the following details for the session, and click OK (repeat to add further sessions as required).
    • Information Date - when netting participants will be advised of the provisional amounts they can expect to pay or receive
    • Transaction Date - when rates are set
    • Settlement Date - when transactions are settled
    • Notification Date - when participants should have entered all flows to be included in the session
    • Incl Disputed - select this option to include netting transactions with a state of DISPUTED in the netting calculation; clear it to exclude them
    • NOTES - optional text comments or information about the session

      TIP: Only basic session details are provided here. For all session details, see the Netting session page.

  6. Click Save.

To edit Pool details

NOTE: You cannot change the Pool id. Instead, Copy the pool and give it the new id.

  1. Select the pool from the list.
  2. Change the Details area field values by direct entry, if required.
  3. In the Phases tab menu (if required), select Edit; in the dialog box change selections as required; then click OK.
  4. On the Sessions tab (as required):
    • To edit an existing session - click on the Session name; in the dialog box change values as required (see above); then click OK.

      TIP: Only basic session details are provided here. For all session details, see the Netting Session page.

    • To add a new session - select New from the tab menu; in the dialog box enter a Name for the session, select a Status, enter the dates for the session (see above), then click OK.
    • To delete a session (not available if there are any invoices, offers, or requests still tied to the session) - select the session row; in the tab menu, select Delete; then confirm.

To delete a Pool completely

(Not available if there are any invoices, offers, or requests still tied to any session for the pool)

  1. Select the pool from the list.
  2. In the page menu, select Delete.
  3. Confirm the deletion.
