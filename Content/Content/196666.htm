Configure WebLogic Server

In the WebLogic Server Console, perform the following steps.

TIP: The names used below are recommendations only. Replace with your own names as and if required.

Create a JMS Server

  1. Under Services -> Messaging -> JMS Servers, select New.
  2. Enter the following settings:
    • Name: QuantumJMSServer
    • Persistent Store: None
    • Target: myserver (or choose an available server)
  3. Press Finish.

    Your JMS Server should now be visible in the list with Health OK.

Create a JMS Module

  1. Under Services -> Messaging -> JMS Modules, select New.
  2. Enter the following settings:
    • Name: QuantumJMSModule
    • other options: leave empty
    • Targets: check myserver (or choose the same as the JMS server)
  3. Press Next.
  4. Leave the option Would you like to add resources to this JMS system module unchecked.
  5. Press Finish.

Create a SubDeployment

The subdeployment in this example is used to target the following Connection Factory and JMS queues to the JMS Server you created above.

  1. Under Services -> Messaging -> JMS Modules, select QuantumJMSModule.
  2. Select the Subdeployment tab and press New.
  3. Set Subdeployment name to QuantumSubdeployment and press Next.
  4. Set Target to QuantumJMSServer and press Finish.

Create a Connection Factory

  1. Under Services -> Messaging -> JMS Modules, select QuantumJMSModule and press New.
  2. Select Connection Factory and press Next.
  3. Enter the following settings:
    • Name: QuantumConnectionFactory
    • JNDI Name: jms/QuantumConnectionFactory
    • other values: leave at their default values
  4. Press Next.
  5. On the Targets page, select the Advanced Targetting button and select QuantumSubdeploymet.
  6. Press Finish.

Create JMS Queues

  1. Under Services -> Messaging -> JMS Modules, select QuantumJMSModule and press New.
  2. Select Queue and press Next.
  3. Enter the following settings for the first queue - for example:
    • Name: WLDealImport
    • JNDI Name: jms/WLDealImport
    • Template: None
    • Subdeployments: QuantumSubdeployment
  4. Press Finish.
  5. Repeat the previous steps to create the following queues for each incoming and outgoing Purpose, as required.

    Name

    JNDI Name

    DealImport

    jms.DealImport

    BankImport

    jms.BankImport

    StaticImport

    jms.StaticImport

    Confirmation

    jms.Confirmation

    Eft

    jms.Eft

    GLInterface

    jms.GLInterface

    ConfirmationDelivery

    jms.ConfirmationDelivery

    The JMS queues you create can now be accessed from Quantum using their JNDI names.
