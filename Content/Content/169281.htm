Zero Curve (Analytics)

Use the Zero Curve page to define and modify the definition of Zero Curves.

  • Use the Rate Definition area to view and maintain the defining values for a Zero Curve record.
  • Use the Settings area to view and maintain the basic calculation parameter settings for the current Zero Curve record,
  • If Apply dual curve bootstrapping is selected - use the Dual Curve Settings area to view and maintain the parameter settings for the second curve, and
  • Use the Rates grid to enter and/or change individual rate entries (components) of the current Zero Curve.

NOTE:

  • The Rate Definition combination of Name, Currency, and Effective Date settings identify a unique Zero Curve record.
  • A Rate Name Record must be created before you can select a Name.

Rate Definition area

Use this area to view and maintain the defining values for an Analytics Zero Curve record.

To list Zero Curve records - click the browse button to open the browse window. (Click the Configure link to change browse list properties.)

To add a new (empty) Zero Curve record - click New; select a new combination of Name, Currency, and Effective Date (see below); in the Settings area select the required values; when applicable, select the appropriate Dual Curve Settings; in the Rates grid add individual rate entries for the curve components; then click Save.

TIP: You can also add a new record by selecting an existing one, selecting Copy in the grid menu, changing the values as required, and clicking Save.

To view and edit another Zero Curve record - in the main browse list, select the record you want; edit the Settings and Dual Curve Settings values if required; add, edit, and/or remove individual component rate entries (grid rows) as required; then click Save.

To delete a complete Zero Curve Rate record - in the main browse list, select the record you want; in the site menu, select Delete; then click OK to confirm.

Field Descriptions

Name - the name of the required rate record (the Code value of an existing Analytics Rate Name record).

Currency - the currency for which the Zero Curve rates apply.

Effective Date - the date from which the Zero Curve rates will be applied (up until the next Effective Date - if any - for the same Name and Currency).

Settings area

Use this area to view and maintain the basic calculation parameter settings for the current Analytics Zero Curve record.

Field Descriptions

Bootstrap Frequency - select Quarterly, Semi Annual, Annual, or Natural.

TIP: If Apply Dual Curve bootstrapping is selected, it is recommended (but not mandatory) that you set this to Natural.

Interpolation - select the Interpolation method: Linear or Logarithmic. See Interpolation Methods (Analytics Zero Curve)

Model - select the Zero Curve model - Linear Par, Quadratic, Bond, or Cubic Spline.

Apply dual curve bootstrapping - select this option to apply Dual Curve Bootstrapping to the Zero Curve and complete the fields in the Dual Curve Settings area and Rates grid.

See also: Dual Curve Bootstrapping - Basis Swaps.

Dual Curve Settings area

(Available only when Apply dual curve bootstrapping is selected.)

Use this area to view and maintain the second curve's parameter settings for the current Analytics Zero Curve record.

TIP: Dual Curve Bootstrapping enables you to define and utilize two distinct curves when bootstrapping the forward curve, allowing separate projecting and discounting rate and margin curves to be applied in Swap valuations. This provides more accurate and internally consistent valuations when different curves are being used. For more details, see Dual Curve Bootstrapping - Basis Swaps.

Field Descriptions

Frequency - the tenor of the floating index, and the frequency at which the floating side Forward Rates will be found: Monthly, Quarterly, Semi-Annual, or Annual. The default is the fixed Swap Par rate Frequency.

Day Count - the floating index day count convention to be applied on the floating leg (in the bootstrap calculation). The default is the fixed Swap Par rate convention.

Discount Curve - the Rate Name for the Discount Curve to be used within the dual bootstrapping calculation.(one set up as an Analytics Rate Name).

While this will usually be set to the OIS curve used for discounting Swap cash flows, you can select any other Zero Curve definition that uses traditional single curve models (those with Dual Curve set to Yes will not be available for selection).

Rates grid

Use the Rates grid to enter and/or change individual entries (components) of the current Zero Curve.

To add a new Rates component to the current Zero Curve record - in the Rates grid menu, select New and in the dialog box:

  1. Select the Rate Name, Zero Rate Type (Yield, FX, Security, Futures, or FRA), Start Offset units, and Start Offset Type.
  2. If Zero Rate Type is FX, select the zero curve Base Rate name and Base Currency to be used in the interest rate parity calculation.
  3. If Zero Rate Type is Futures, select the Futures Exchange and Futures Contract.
  4. Click OK.

*TIP: You can also add a new row by selecting an existing one, selecting Copy in the grid menu, changing the values as required, and clicking Save.

To edit a rate component of the current Zero Curve record:

  1. Click a cell value in the entry's grid row (or select the row you want to edit and in the grid menu, select Edit).
  2. In the Edit Rate dialog box, change the values as required (see Field Descriptions below).
  3. Click OK. Your changes are shown in the Rates grid.

To remove a rate component from the current Zero Curve record:

  1. Select the grid row you want to remove.
  2. In the grid menu, select Delete.
  3. In the Confirm Delete dialog box, click OK to confirm. The term definition is removed from the Rates table.

Field Descriptions

Rate Name - the Rate Name of the underlying market rate.

Zero Rate Type - the Zero Rate Type: Yield, FX, Security, Futures, or FRA*.

*NOTE: An FRA strip cannot be the first component curve, and a Yield curve must be available in the Zero Curve definition with a Start Offset prior to the FRA strip's.

Start Offset - specifies where the underlying market rate starts in the blended Zero Curve, as follows:

  • in the lookup on the right, select one of Overnight, Futures Contract, or a time unit (Days, Weeks, Months, or Years) and
  • if a time unit - enter a positive integer in the field on the left to give the number of those units.

    For example, for:

    • Cash Rates - Overnight,
    • FRA Rates - 4 Months, or for
    • Swap Rates - 1 Year.

Base Rate (available when Zero Rate Type is FX - that is, for curves derived from FX Forward rates) - select the base currency Zero Curve Rate Name to be used in the interest rate parity calculation.

Base Currency (available when Zero Rate Type is FX) - select the base currency.

Futures Exchange (available when Zero Rate Type is Futures) - select the exchange.

Futures Contract (available when Zero Rate Type is Futures) - select the Futures Contract.
