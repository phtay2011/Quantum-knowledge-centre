Defining Workflow incoming data (DTL)

Set up a Data Transfer Location (DTL) record with a WorkflowIn Purpose to manage incoming data. When this is set, a Parser can be chosen to parse the payload into a data source on which workflow actions can be configured.

When a Transfer action specifying that DTL is performed on a report created via the Workflow Report page, the Static Data Import Service monitors the specified location and when a file is found, the system will use the Parser specified in that record to parse the file, perform the Action specified in the DTL record (using the parsed data), then move the file from the WorkflowIn\Import folder to the WorkflowIn\Processed folder (or if there is no associated DTL record or a Parser error, the system moves the file to the WorkflowIn\Error folder instead).

The parsed data is then available as a data source in other Workflow pages - for example: user-designed variant pages, other Workflow reports, and Workflow templates.

Prerequisites

Before you can import Workflow data files, ensure that:

  • the Process workflows option is selected on the Operational Settings page (under Active Services);
  • the configuration file settings are correct on your Application Server;
  • at least one parser is available - one configured using the Workflow Parser page (or - rarely, and for internal use only - a System parser);
  • the incoming files - correctly formatted for the parser to process - will be placed in the \WorkflowIn\Import folder'
  • a Data Transfer Location (DTL) page has been set up with Location Type set to FILE, Purpose set to WorkflowIn, Parser set to the system Parser name, and the Actions grid details the action performed by the system on the incoming data;
  • a Transfer action specifying that DTL is performed on a report created via the Workflow Report page; and
  • you have the required Module Security permissions.

To set up a new WorkflowIn DTL record:

  1. On the Data Transfer Location page, click New.
  2. Enter a unique Name to identify this DTL record.
  3. Set Location Type to FILE.
  4. Set Purpose to WorkflowIn (not available if no Parsers are available - see Prerequisites above).
  5. In Location, optionally enter a subfolder name if required.
  6. Set Parser to the appropriate parser - either a System one installed in the Web Server's QuantumWeb/bin folder, or one configured using the Workflow Parser page.

    7. The Action grid is then displayed.

  7. In the grid menu, select New.
  8. In the New Action dialog box, select an action name in the browse list (under Create, Delete, Merge, Process, Transfer, or Update). For example, select Create > Create Actual Cash Flow.

    You are returned to the New Action dialog box, and an empty Inputs grid is displayed.

  9. Click Add.
  10. In the Select Inputs dialog box, select the at least the mandatory input fields. These depend on your selected Action and the type of data your action applies to.

    You are returned to the New Action dialog box, with the Inputs grid now populated with your selected input fields, with the Expression column showing the field names as default settings for non-text data fields.

  11. To enter or edit an Expression, click the Input name.
  12. In the Edit Setting dialog box, click the [Expression] or Add Expression link.
  13. In the Expression Builder: to enter a text value, type the text within double quotes. For a data item, you can type or edit directly in the text box, or click the + button and select a Function, Operator, or - under Context - a parsed data column.
  14. Continue building the expression as required. Click Validate to check it and edit if required. Click Done.
  15. In the Edit Setting dialog box, click OK.
  16. Repeat as required until all Inputs grid rows have valid expressions, then click OK.

    17. NOTE: You may create only ONE Action.

  17. Click Save.
