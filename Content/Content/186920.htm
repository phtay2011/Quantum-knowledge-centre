Custom Queries and Reports

Queries can now be controlled and secured using permissions in Module Security and Resource Security for the following pages:

  • All Reporting pages,
  • Confirmations Processing,
  • Settlements Processing,
  • Deal Approval, and
  • Settlement Approval.

Implementation

Data Security page

On the Data Security page a new Query filter has been added under Filtered Resources, which must be enabled before individual queries can be secured on the User and User Group pages.

When you enable Query resource filtering for Resource Security:

  • you can control user or user group access to public queries and the actions they can perform on associated pages (for example, Run, Configure, Save, Delete, and so on),
  • Module Security and Resource Security work together to allow or restrict actions within queries and reports, and
  • it applies to all Reporting menu queries and the following Processing and Accounting menu queries:
    • Confirmations and Settlements,
    • Deal Approval and Settlement Approval,
    • Account Interest and Account Statements,
    • Accruals,
    • Position,
    • Futures Margin, and
    • Currency Translation.

Resource Security

On the User and User Group pages, queries for the relevant pages can be configured with standard Resource Security permissions.

The Resource Security permissions work as follows for queries:

  • No Access - The query will not be available for selection.
  • View - The query can be selected and previous runs can be opened and displayed.
  • View/Update - The query can be selected, previous runs can be opened and displayed, and the query can be run.

NOTE:

  • The Resource Security permissions can be applied for a user or user group for ALL public queries in ALL modules (that is, Default access), for ALL public queries in individual modules, or for individual public queries. Access settings at specific levels take precedence over those at a general level.
  • Confirmations, Settlements, Deal Approval, and Settlement Approval only have the No Access and View / Update permissions for selection.

Module Security

On the User and User Group pages, the modules that support queries can be configured with following permissions:

Configure Public Queries and Views

When enabled, allows the user to do the following:

  • create, configure, save, copy, and delete public queries;
  • keep and delete public reports created by other users; and
  • select Public in the Access Permission area of the query's Properties tab.
    (This option is selected and read-only by default when the user does not have Configure Private Queries permission).

NOTE:

  • When you enable Query resource filtering for Resource Security, Resource Security and Module Security work together to allow or restrict actions within queries and reports.
  • If a user does not have either Configure Private Queries or Configure Public Queries permission for the module, they will not be able to configure any queries.

Configure Private Queries

When enabled, allows the user to do the following:

  • create, configure, save, copy, and delete private queries; and
  • select Private in the Access Permission area of the query's Properties tab.
    (This option is selected and read-only by default when the user does not have the Configure Public Queries and Views permission).

NOTES:

  • When you enable Query resource filtering for Resource Security, Resource Security and Module Security work together to allow or restrict actions within queries and reports.
  • If a user does not have either Configure Private Queries or Configure Public Queries permission for the module, they will not be able to configure any queries.

Effect of Resource and Module Security on Query Actions

Module Security and Resource Security work in conjunction to allow or restrict actions on reporting and processing pages.

Reporting and Processing Pages

  • Queries and their previous runs are now selected from a single browse list titled Select Query or Run.
  • The browse list displays all queries the current user has access to.
  • A folder icon is displayed next to queries that have previous runs available, indicating that users can drill-down and select the appropriate run for these.
  • Reports of scheduled queries, which are set to run for latest, can be selected from the browse list. These reports are prefixed with Latest: in the Summary column of the browse list, and are automatically refreshed with their latest run when it becomes available.

Impact of Query Resource Security Filtering on Analytics

Security set up against the Query resource in Quantum does not affect Analytics queries.

Implementation Notes

Following this update, when Query filtering is enabled in Resource Security for the first time, ALL users and groups are assigned No Access permission for ALL public queries and relevant pages by default. An administrator will need to manually configure the necessary permissions for users and groups for the individual public queries and relevant pages.
