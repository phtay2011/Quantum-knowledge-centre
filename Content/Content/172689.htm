Standard Bond Instruments

To set up Standard Bond instruments in Quantum, use the Securities Instrument page*. On the Details tab, set Formula Timing to BOND PERIODIC. This updates the page display to display the applicable tabs and items for the definition of standard bond instruments and their variations.

The definition of Standard Bonds requires the entry of certain mandatory information. Optional details can also be entered, some of which define variations of Standard Bonds such as Odd Coupon Periods, Callable Bonds, Japanese Government Securities, Dual Currency Bonds and Government Bonds (using seed templates).

*NOTE: You could use the Money Market Instrument and Dealing pages to specify this type of deal, but those pages have no settings related to tradability - for example an Issuer, Security Identification, and so on.

A general summary of Bond instrument settings is given below, in display order (for more details, follow one of the above links).

TIPS:

  • References to deals here cover any deal or deal leg that may specify the Securities* instrument - namely, a Securities deal, Bond Interest Rate Option deal, or Repo deal Security leg.
  • If the instrument is non-modifiable, values defaulted from the instrument cannot be changed at deal entry time.

Header area

  • Govt Bond Template - optionally select a Government Bond template to base your instrument on. If you do, the default information for that Government Bond is displayed automatically. See Government Bond Templates for the complete list of templates available and their corresponding default values.

Details tab

  • Include embedded options - clear this option.
  • Show Broker - select to display a Broker field for deal entry.
  • Broker Required - if Show Broker is selected, select this option to make entry of a broker mandatory at deal entry.
  • Show IRR - select to display the internal rate of return at deal save.

Dates tab

Dates & Frequencies

  • New Issue - select if the instrument is for a new bond issue.
  • Issue Date - if New Issue is selected, enter the Issue date.
  • Days to Settlement - enter the number of days between the Deal Date and the Settlement Date to apply at deal entry.

Odd Period Definition

Use these fields to define a first and/or last coupon period that is shorter or longer than normal. In particular, when both Coupon Day Count and Income Day Count are set to ACTUAL/ACTUAL, you can optionally select the Treat Odd Periods as Regular option to specify that odd (irregular length) periods are to be treated as Regular periods for coupon, accrued interest, and yield calculations. See Odd Coupon Period definition .

Pricing tab

Coupon & Price Type

  • No Coupon on Maturity - select if there is no coupon payment calculation on the bond at maturity.

Issuer tab

Issue Details

  • Issuer - select the bond issuer.
  • Issuer Required - select if you want entry of a valid Issuer/Acceptor to be mandatory for deal entry.
  • Total Issue - enter the total size of the issue.

Security Identifier

  • Security Identifier - select the Security Identifier record, to link the unique identification numbers (such as CUSIP and/or ISIN) and other issue details of the bond.
  • Display security identifier - If selected, a Security Identifier field will be displayed for deal entry. Its value will default to any value you enter here.
  • Required at deal save - optionally select to make selection of a Security Identifier mandatory for deal entry.
  • Use for Defaulting - select if you want to use default deal values from the selected Security Identifier record where applicable, overriding the defaults specified on the instrument. This applies only to the Issuer, Issue Date, Maturity Date, Coupon (%), and Currency default settings.

    Clear this option to default these values from the instrument.

Issuing and Paying Agent

  • Issuing and Paying Agent - optionally select a default Issuing and Paying Agent (IPA) if applicable.
  • Display IPA - select if you want to specify an IPA for deal entry (whether or not you specify an IPA here).
  • Required at deal save - optionally select to make selection of an IPA mandatory for deal entry.
  • All cash flows with IPA - select if all settlement and post-settlement cash flows are to be with the IPA.

    Clear this option if only post-settlement cash flows are to be with the IPA, while settlement cash flows are with the Counterparty.

    NOTE: If an IPA is not specified when you save or exercise a deal based on this Instrument, the standard scenario applies (settlement cash flows with the Counterparty, post-settlement cash flows with the Issuer).

  • Default Counterparty to IPA - select if you want the deal Counterparty to default to the IPA.

Other Details tab

Default Settings

  • Currency - select a default deal currency.
  • Dealer - select a default dealer (in your organization) to be recorded against deals.
  • Relationship - select a default relationship for deals.
  • Location - select a default location to determine non business days for deals in this bond.

Tax Settings

  • Tax Exempts - optionally select any taxes deals will be exempt from.
  • Calculate WHT on accrued income - select to calculate Withholding Tax on accrued interest (Purchased Income).
  • Calculate WHT on capital discount or premium - select to calculate Withholding Tax on Capital Discount or Premium.

Analysis codes tab

Select any codes for reporting and/or analysis.
