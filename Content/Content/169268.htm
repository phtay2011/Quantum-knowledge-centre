Security Rates (Analytics)

Use the Security Rate page to view and maintain rates for Securities in Analytics. Use the:

  • Rate Definition area - to view and maintain the defining values for an Analytics Security Rate record,
  • Rate Dates area - to view and maintain different sets of rates with different effective dates, for the current Security Rate record, and
  • Rates grid - to enter and/or change individual rate values for the current set of rates.

NOTE:

  • The Rate Definition combination of Name and Currency identify a unique Security Rate record.
  • An Analytics Rate Name Record must be created before you can select a Rate Definition Name.
  • Each Rate record may have one or more effective dates, each with a set of rate values. The number of these dates is indicated in the badge at top right of the page and the Date value shows the currently selected effective date for the set of rates displayed in the grid.

Missing Rates

(New functionality introduced in Release 6.7.2.)

For Security deals, when a missing rate is:

  • found within the configured lookback period* - that rate is applied and a message is output informing the user that a historical rate has been used, or if...
  • not found within the lookback period* - the rate is interpolated from the other Securities on the latest applicable rate record and a message is output informing the user that the rate could not be found and an interpolated price has been used.

*TIP: The default lookback period is five calendar days from the rate date. This default can be overridden by changing a configuration setting. (For more details, see Configuring the Analytics Server in the Quantum Installation instructions document).

Rate Definition area

Use this area to view and maintain the defining values for an Analytics Security Rate record.

To list Rate records - click the page browse button (the top one) to open the browse window. (Click the Configure link to change browse list properties.)

To add a new (empty) Rate record - in the page menu, select New; in the New Rate Definition dialog box enter new Rate Definition values, enter the required settings (see Field Descriptions below); Then click Save, and:

  • use the Rates Date area to specify an effective Date for a set of rates for the Rate record, then
  • use the Rates grid to enter and/or change individual rate term values to create the set of rates for that effective Date.

Each Rate record may have one or more effective dates, each with a set of rate values. The number of these dates is indicated in the badge at top right of the page and the Date value shows the currently selected effective date for the set of rates displayed in the grid.

To view and edit another Rates record - in the main browse list, select the Rates record you want and add, edit, and/or remove complete rate sets and their individual rate entries (grid rows) as required.

To delete a Rate record - in the main browse list, select the Rates record you want to remove; in the Rates menu, select Delete; then click OK to confirm.

Field Descriptions

Name - the name of the required rate record (the Code value of an existing Analytics Rate Name record).

Currency - the currency for which the Security rates apply.

Rate Type - Yield, Capital Price, or Cash Price.

Rate Dates

Each Rate record may have one or more effective dates, each with a set of rate values.

The Date value shows the currently selected effective date for the set of rates displayed in the grid below. This is the date those rates apply from (up until the next effective date, if any). The default for a new set of rates is Today's date and time.

TIP: The number of these dates is indicated in the badge at top right of the page.

To add a new set of rates (with a new effective Date) to the current Rate record:

  1. In the Rates menu (the middle menu), select New.
  2. In the New Rate Date dialog box, enter the new effective Date.
  3. Click Save.
  4. Add individual Rate term entries to the Rates grid as required.

TIP: You can also add a new set of rates by selecting an existing effective Date set, selecting Copy Rates in the Rates menu, changing the Date and clicking Save, then adding, editing, and/or removing individual rate entries as required.

To view and edit another set of rates in the current Rates record:

  1. Click the Rates browse button (the lower one) and select the effective Date for the rates set you want.
  2. Add, edit, and/or remove single rate term entries in the grid as required.

To remove a set of rates from the current Rates record:

  1. Click the Rates browse button (the lower one) and select the effective Date for the rates set you want to remove.
  2. In the Rates menu, select Delete.
  3. In the Confirm Delete dialog box,click OK to confirm.

    The rate set for that effective Date is removed from the current Rates record.

Rates grid

Use the Rates grid to enter and/or change individual rate values in the set of rates for the currently selected effective Date (for the current Rate record).

To add a single rate term entry to the current set of rates:

  1. In the grid menu, select New*.
  2. In the New Rate dialog box, enter a new Term and Term Type, and enter the remaining values as required (see below).
  3. Click Save.

    The new rate entry is added to the Rates grid.

*TIP: You can also add a new row by selecting an existing one, selecting Copy in the grid menu, changing the values as required, and clicking Save.

To edit a rate term entry in the current set of rates:

  1. Click a cell value in the entry's grid row (or select the row you want to edit and in the grid menu, select Edit).
  2. In the Edit Rate dialog box, change the values as required (see below).
  3. Click Save. Your changes are shown in the Rates grid.

To remove a rate term entry from the current set of rates:

  1. Select the grid row you want to remove.
  2. In the grid menu, select Delete.
  3. In the Confirm Delete dialog box, click OK to confirm. The term definition is removed from the Rates grid.

Field Descriptions

Instrument - the Security instrument.

Security (optional) - the Security identification, or Null (or blank).

Bid - the bid rate.

Offer - the offer rate.
