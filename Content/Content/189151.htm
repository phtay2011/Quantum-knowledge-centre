Instructions Settings, Broker Account Defaulting Rule

These items define the Instructions that will be applied when a broker cash flow meets the Matching Criteria for the applicable rule entry.

The instructions that will be used are those with the applicable Effective Date for the cash flow date.

Effective Date - The date a set of instructions is active from.

  • If more than one set of instructions is defined, each set must have a different Effective Date.
  • The instructions that will be used have the nearest Effective Date on or before the cash flow date - that is, where the cash flow date is greater than or equal to the Effective Date, but less than any later Effective Dates.

    TIP: DO NOT change the Effective Date if you are changing any other settings in that instruction. Instead, add a new set of instructions with the new Effective Date. To do this, select New from the grid menu and enter the new instruction values.

Bank / Broker - The broker to use for fee and commission payments when this account instruction is applied.

NOTE: If both a Clearing Broker and Executing Broker are specified on a deal and your Broker Account Rules specify different accounts for those brokers (in the Matching Criteria), the system posts the broker flows:

  • for the Exchange Fee and Clearing Fee - to the account of the Clearing Broker, and
  • for the Commission - to the account of the Executing Broker.

Account Number - The broker's bank account number to use when this account instruction is applied.

  • The Bank / Broker specified for this instruction determines which account numbers are available for selection here.
  • When you select an account number, the Account Name and Account Currency read-only fields are populated for this instruction.

Default Payment Method - The payment method to be used for cash flows when this account instruction is applied.

TIP: If you do not specify a payment method, the default payment method associated with the selected account applies (if there is one).

Entity Identifier - The Entity identifier value for the specified Default Payment Methodif an identifier is specified for that Payment Method (for example, for a Clearing House). You can select only from Entities valid for the payment method (if any).
