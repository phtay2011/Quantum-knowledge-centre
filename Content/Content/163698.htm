Entity General Ledger Options

Use the Accounting tab to indicate whether deals made by the entity are subject to Hedge Accounting and, if so, to specify the date from which Hedge Accounting should be applied, and how deferred tax should be treated for cash flow Hedge Relationships.

For a group-level Entity that is a G/L Owner, you can also specify the default Entity for posting.

NOTE: New entities are, by default, non-posting. To change a non-posting Entity to a G/L Owner or (if it is in a group) to one that posts to a ledger owned by another Entity in the group, use the General ledger Properties page's New Ledger or Start Posting functions, respectively.

Specify the following General Ledger options for an Entity.

Enable Hedge Accounting for this Entity

(Not available for Group Entities.)

Use this check box to enable Hedge Accounting for the entity.

Prerequisites: Your system must be licensed for Hedge Accounting, and it must be enabled system-wide (on the Accounting tab of System Defaults).

This check box is available only for a transaction-level entity that is either a G/L Owner, or posts to the G/L of another Entity. See Accounting Setup.

NOTES:

  • Hedge Accounting can only be applied to an Entity if it posts to a General Ledger.
  • You cannot enable Hedge Accounting for a group-level Entity.

Only Entities that have Hedge Accounting enabled:

  • are available for selection in a Hedge Relationship
  • can link a deal to a Hedge Relationship at deal-save time
  • are eligible for Hedge Accounting processing

NOTE You cannot clear this option if there are Hedge Relationships against the Entity.

Transition Date

(Available only when Enable Hedge Accounting for this Entity is selected.)

When you first switch on Hedge Accounting for this Entity, use the calendar to edit this date, which is the date on which processing will start for the Entity. Because a different transition date can be specified for each Entity, you can have multiple transition dates within a single system.

This enables you to link deals beforehand, but only have them processed on the date specified. Hedge Relationships entered in this way are effective from the Transition Date, not the Designation Date.

NOTE You cannot change the Transition Date while there are Hedge Relationships in the system.

Apply tax to OCI

(Available only when Enable Hedge Accounting for this Entity is selected.)

Select this check box to apply deferred tax, at the rate entered in the OCI Tax Rate field, to Other Comprehensive Income (OCI) for all cash flow Hedge Relationships recorded for this Entity.

When the OCI is calculated for cash flow Hedge Relationships in Position Reporting or Position Processing, the balance sheet OCI posting will be split into two separate entries: OCI (current) and Deferred tax.

OCI Tax Rate

(Available only when Apply tax to OCI is selected.)

Enter the tax rate to apply to the OCI for all cash flow Hedge Relationships for this entity. Valid tax rates are greater than 0% and less than or equal to 100%.

Changing the tax rate causes Position Reporting or Position Processing to apply the new rate regardless of the date it was changed. This means that if you change the rate today and run a backdated Position query, the new rate applies to the postings.

Default Posting Entity

(Available only with a group-level Entity.)

If the Entity owns a General Ledger, select the default Entity the system will use to post the General Ledger entries that it generates for this G/L Owner.
