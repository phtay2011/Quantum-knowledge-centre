Futures Final Trading Days - Exception to Primary Rule

(First introduced for Default perspective in Release 6.6.1 SP, March 2018. Extended for Analytics in Release 6.7.0.)

When you define a Futures Final Trading Days rule, you can now specify one date as an exception to the Primary rule.

For example, some Futures Contracts have final trading days as the last day of each month EXCEPT for December's contract, which expires before Christmas - that is, on 24 December or the previous business day.

Implementation - Quantum

(First introduced in Release 6.6.1 SP, March 2018.)

On the Futures Final Trading Days page (Default perspective), you can now select EXCEPTION in the Apply 2nd Rule field. When you do, Day, Month, and Business Rule fields are displayed next to the Apply 2nd Rule field (as highlighted in the following example). Use those fields to specify a Business Rule that will override the Primary Rule for the specified day and month.

If the example rule above was applied (via the Futures or ETO Instrument Date settings), the Futures or ETO Dealing page's Deal Map and Cash Flows tabs would, upon December's contract selection, reflect the rule exception - that is, the Effective/Action Date of Expiry and Fee components would be dated 24 December if that was a business day, otherwise the preceding business day.

Implementation - Analytics

(First introduced in Release 6.7.0, May 2018.)

On the Futures Final Trading Days page (Analytics perspective), you can also now select EXCEPTION in the 2nd Rule Applied field. When you do, Month, Day, and 2nd Business Day fields are displayed next to the 2nd Rule Applied field (as highlighted in the following example). Use those fields to specify a Business Rule that will override the Primary Rule for the specified day and month (as for Quantum - see above).

Replication

Quantum records with Apply 2nd Rule set to EXCEPTION will be successfully replicated into Analytics (2nd Rule Applied set to EXCEPTION and the Day, Month, and 2nd Business Day values set to the corresponding Quantum values).

History

Previously:

  • from Release 6.6.1 - an EXCEPTION setting was supported for the Default perspective only, and Quantum records with that selected would fail replication into Analytics, and
  • prior to Release 6.6.1 - an EXCEPTION setting was not supported for either perspective.
