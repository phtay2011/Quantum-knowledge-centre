Hedging the intrinsic value of Currency Options

Quantum recognizes the change in the time value of an option when only its intrinsic value is designated in a hedge relationship as follows.

  • For transaction related hedged items, the change in fair value is accumulated in a separate component of Equity and reclassified to profit and loss.
  • For time period related hedged items, the change in fair value is accumulated in a separate component of Equity and amortized on a rational basis over the term of the hedge relationship.

Designated Risk: Overall Market Risk verses Currency Option - Intrinsic

When an option is designated in its entirety (that is, at the Market Value) as the hedge of a non-option item, hedge effectiveness is determined by comparing the fair value of the entire option with the change of value of the hedged item. Changes to the time value portion of the fair value result in ineffectiveness and, in some cases, the entity could be forced to discontinue hedge accounting due to ineffectiveness resulting from these changes to the time value.

When only the intrinsic value of the option is designated in a hedge relationship with Designated Risk equal Overall Market Risk, changes to the time value portion of the fair value have to be accounted in profit or loss, resulting in potentially significant profit or loss volatility.

When the intrinsic value is designated with Designated Risk equal Currency Option - Intrinsic, the fair value change in the time value must be accumulated in OCI and recognized in profit and loss, either over the period of the hedge (for time period related hedges) or when the hedged transaction affects profit and loss (for transaction related hedges).

Hedging procedures

Create the Hedge Strategy page

Create a valid Hedge Strategy page with the following options:

  1. A Fair Value Hedge or Cash Flow Hedge for the Hedge Type; CURRENCY OPTION - INTRINSIC for the Designated Risk (Details tab).
  2. If required, set a release date for the adjustment amounts (Events tab).

Create the Hedges, Hypotheticals, and Exposures

Create the following deals in Quantum according to the requirements of your hedge:

  1. Hedge: Create one or more Currency Option deals that represent the terms of the hedge.
  2. Hypothetical*: For each Hedge deal created in Step 1, create a Currency Option deal that counteracts the purpose of the hedge - that is, Hedge (Buy) / Hypothetical (Sell) - but matches the terms of the hedge exactly.

    * NOTE: It is possible to account for the time value component of the option only to the extent that it relates to the hedged item. When the terms of the hedge do not match at inception, the Aligned Time Value is determined using the Hypothetical, which represents the hedged risk.

  3. Exposure: Create one or more Actual Cash Flow deals to designate the risk being hedged.

Create the Hedge Relationship

Create a hedge relationship with the following options:

  1. Select the Hedge Strategy page with CURRENCY OPTION - INTRINSIC as the Designated Risk, created above.
  2. Select the Maturity Date, and ensure that the Designated Risk is set to CURRENCY OPTION - INTRINSIC (Deal Details tab).
  3. For time-period related hedges, select Amortize hedging costs (time value) (Deal Details tab).
  4. Add the Hedges, Hypotheticals, and Exposures created above to the hedge relationship (Linked Deals tab).

NOTE: When the Currency Option deal being hedged is exercised, the FX deal that results from the exercise is automatically added to the hedge relationship, and the Derived Maturity Date of the hedge relationship is adjusted according to the remaining Hedges and Hypotheticals.

IMPORTANT: Once a Currency Option - Intrinsic Hedge Relationship has been designated and postings have been started, it is currently not possible to add additional deals into the Hedge Relationship.

Create the Posting Rules

For the entity, select the following accounts for the Posting Rule for hedge relationships:

  • OCI Balance (Hedging Costs)
  • Deferred Tax (Hedging Costs)
  • Hedging Costs Reclass
