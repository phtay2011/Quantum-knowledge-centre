Reapply Cash Flows and Accounting Rules page

CAUTION: Before using this utility, consult with your FIS Consultant.

Use this page to update existing transactions (including deal cash flows) after changes to Accounting Setup and/or Bank Account defaulting rules.

TIP: This may be required because postings and cash flows dated before such changes continue to use the defaults that applied when those transactions were first saved (only deals saved after the changes use the new settings).

Procedure Summary

TIP: As you specify your selection criteria, the Items to be Processed counts of the applicable Deals, Cash Flows, and G/L Entries are updated.

  1. Under Filters, select By transaction details or By deal number.
  2. If you selected By transaction details:
    1. select an Entity, Transaction Type, and Instrument (ALL is valid in each case),
    2. optionally select Filter by deal maturity date and enter a Deals maturing after date if you want to apply changed Accounting settings to deals maturing after that date,
    3. under Reapply, select Apply any 'Balance sheet timing' changes to reapply the Balance Sheet Timing for existing deals meeting the selection criteria, then
    4. go to Step 4.
  3. If you selected By deal number (in Step 1), enter one or more Deal Numbers separated by commas.
  4. Optionally select Include non posting entities to cover deals involving such entities.
  5. Under Reapply, select either Cash flows and G/Ls or G/Ls only and, If the former, optionally select Our bank account defaults (nostro) and/or Counterparty bank account defaults (vostro) to reapply cash flows with your and/or the Counterparty's latest bank account defaults.
  6. Under Options, select either Delete existing G/Ls* or Reverse existing G/Ls.

    *CAUTION: The Delete Existing G/Ls setting is not recommended on live databases as it may cause serious problems with your data.

  7. Optionally select Suppress warnings when applying new defaults to stop display of such messages that may arise during processing.
  8. Optionally select Post to suspense if no G/L defaults exist to post entries to the Suspense account when no matching G/L defaults can be found.
  9. Click Reapply.

Quantum reverses or deletes existing G/L entries and reapplies them using the changed defaults. If they are selected, cash flows will also be re-applied to reflect changes to Bank and/or Counterparty Account defaults (without affecting customized cash flows or closeouts on deals).

NOTE: If G/Ls are reapplied then you must rerun accruals, revaluations, and currency translation, as all existing G/L entries will have been removed or reversed.
