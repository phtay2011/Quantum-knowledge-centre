Fee 3 area (Unit Fee Defaulting Rule)

Use this area to specify details of the (optional) third fee incurred by Unit deals this Fee defaulting rule record applies to.

You can specify from one to three different fees to be applied. To delete the last fee: in the Page menu, select Delete Last.

Fee Code - select the code for the applicable Quantum Fee Name record to be associated with this fee (as set up on the Fee Name page).

PAY or RECEIVE - select whether the fee amount is to be paid or received. Pay is the default.

Type - select Fixed, 1 Tier, 2 Tiers, or 3 Tiers.

Timing - select when the fee will become due for payment: ON DEAL DATE or ON SETTLEMENT DATE.

Currency - select the fee currency.

Fee Counterparty - select the Counterparty for Settlement of the fee.

Method (applies only when Type is Fixed) - select whether the fee Amount applies PER DEAL or PER UNIT.

Amount (applies only when Type is Fixed) - enter the flat fee amount to be applied to each deal or unit, depending on the Method.

Gross Settlement Amount (applies only when Type is NOTFixed) - enter the limiting amount for applying the relevant tier or tiers, where:

  • only ONE of these fields appears when Type is 1 Tier or 2 Tiers, but
  • TWO of these fields appear when Type is 3 Tiers: the first delimits tiers 1 and 2; the second delimits tiers 2 and 3.

Up To Fee in % (applies only when Type is NOTFixed) - the percentage of the Gross Settlement amount giving the fee amount for the applicable tier, where:

  • only ONE of these fields appears when Type is 1 Tier or 2 Tiers, giving the percentage for tier 1, but
  • TWO of these fields appear when Type is 3 Tiers: the first giving the percentage for tier 1; the second the percentage for tier 2.

Over Fee in % (applies only when Type is 2 Tiers or 3 Tiers) - the percentage of the Gross Settlement amount giving the fee amount for the highest tier.
