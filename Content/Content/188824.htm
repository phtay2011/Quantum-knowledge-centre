Relationship to G/L Account Templates

G/L Account Templates contain the information required to generate a physical G/L account. G/L entries are then posted to the physical accounts created from the templates.

The G/L account templates you require depend on the accounting rules you set up. For example, if you have accounting rules that accrue interest you will need G/L account templates to post the accruals to.

As part of their definition, each posting and inter-company rule specifies the G/L account template on which they want to base generated G/L accounts.

NOTE: Accounting rules do not use G/L account templates.

Posting rules

G/L account templates are attached to the G/L component tags on posting rules. Quantum uses the relevant deal details and the G/L account template to determine the physical account to which the G/L entry is posted.

Determining which posting rules and G/L account templates are required

The posting rules and G/L account templates required depend on the accounting rules you set up. For example, you may have a ledger consisting of trades held to maturity, which have accounting rules set up to:

  • accrue all income/expense and amortize premium/discount over the life of the deal, and to...
  • not mark to market.

G/L account templates would be needed for the accruals, but none would be required for mark to market revaluations. Posting rules for the ledger would include the accrual G/L account templates, but would not include any for revaluations.

Based on the accounting rule, revaluations are never performed on the deals so no postings are required to the revaluation accounts. This means no G/L account templates are required as there is no need to derive accounts to post the revaluations to.

See Illustrating the relationship between accounting rules, posting rules and G/L account templates.

What happens when a deal is saved

When a deal is saved, among its various processes:

  1. Quantum checks for posting rule conditions applicable to the details of the deal.
  2. Quantum then determines the posting treatment for each G/L entry that comprises the posting rule.

    For example, the posting treatment for an FX deal involving CP Citibank, London might be to post Forward Points Receivable to the G/L accounts generated from the Forward Points Receivable G/L account template.

The actual G/L account codes created depend on the format defined in the G/L account template. The posting rule determines the value of the format.

Inter-company rules

G/L account templates are attached to the entities that comprise an inter-company rule. Quantum uses the relevant deal details and the G/L account template to determine the physical account to which the inter-company G/L entry is posted.

Determining which G/L account templates are required

The G/L account templates that are available for use within an inter-company rule depend on the entity/rule relationship. For example, if the rule applies to the USD currency, then only active G/L account templates with a currency of ALL or USD are available. Similarly, only those G/L account templates relevant to the entity are available.

Illustrating the relationship between accounting rules, posting rules and G/L account templates

This topic provides an illustration of the relationship between accounting rules, posting rules, and G/L account templates. It illustrates the different ways in which you can use posting rules and G/L account templates to achieve your business goals.

Background

You want all your Bonds to follow the same accounting treatment. To allow this, you create one accounting rule for the group of Bonds.

NOTE: You can set up posting rules at the Master accounting type level that cover all ledgers. In this case you may need to include Revaluation G/L account templates for other ledgers. They will only be used where required.

Scenario One

You want to post the coupon to different G/L accounts, based on the Counterparty with whom the Bond is dealt.

Posting rule option A

Set up a separate posting rule for each Counterparty.

Posting rule option B

  1. Set up one posting rule to cover all Counterparties.
  2. Create a G/L account template for the coupon component that includes the Counterparty name as part of the G/L account code.
  3. Attach the G/L account template to the posting rule for the coupon component.

When the component is used in the posting rule Quantum determines the separate G/L accounts required based on the Counterparty.

Scenario Two

You want to post all the entries to different G/L accounts for a specific Instrument analysis code value only.

Posting rule option A

  1. Create a set of specific G/L account templates for the Instrument analysis code.
  2. Create a set of generic G/L account templates
  3. Set up a separate posting rule for the specific Instrument analysis code, and attach the specific G/L account templates
  4. Set up a generic posting rule for all other Instrument analysis codes.

Posting rule option B

  1. Create one G/L account template that includes the Instrument analysis code in the make up of the G/L account code
  2. Set up one posting rule to cover all Instrument analysis code values
