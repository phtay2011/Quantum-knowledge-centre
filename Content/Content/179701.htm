Regular Expression Functions

Regular expression functions allow complex searches of text data to be carried out.

Functions

RXTEST(expression, pattern[, ignorecase[, multiline]])

Returns true if a match is found for the specified regular expression.

RXMATCH(expression, pattern[, ignorecase[, multiline [,multimatch [,delimiter]]]])

Returns the text matched by the specified regular expression. If multimatch is true then all matches are returned, delimited by the specified delimiter.

RXREPLACE(expression, pattern ,replace [, ignorecase[, multiline [,multimatch ]]])

Returns modified text where the text matched by the specified regular expression is replaced by the replace text. If multimatch is true then all occurrences are replaced.

Common Arguments

Argument

Description

expression

The text expression to be tested, matched, or replaced.

pattern

The regular expression pattern to use for testing, matching, or replacing.

replace

The text expression used to replace matched text.

ignorecase

If true, case is ignored when scanning for the pattern.

The default is true.

multiline

If true, scanning will continue over multiple lines.

The default is false.

multimatch

If true, every matched occurrence is returned or replaced. If false, only the first occurrence is processed.

The default is false.

Regular Expression Pattern-Matching Syntax

The following table describes the special characters and sequences that can be used in patterns for regular expressions, and gives examples.

Character

Description

\

Marks the next character as either a special character or a literal. For example:

  • \n matches a New Line character.
  • \\ matches a backslash \.
  • \( matches a left bracket (.

^

Matches the beginning of input.

$

Matches the end of input.

*

Matches the preceding character zero or more times. For example, zo* matches z , zo, zoo, and so on.

+

Matches the preceding character one or more times. For example, zo+ matches zo and zoo (but not z).

?

Matches the preceding character zero or one time. For example, zo? matches z and zo (but not zoo).

.

Matches any single character except a New Line character.

(pattern)

Matches the pattern and remembers the match. The matched sub-string can be referenced in a replace pattern - see /num below.

x|y

Matches either x or y. For example:

  • z|wood matches z or wood.
  • (z|w)oo matches zoo or woo.

{n}

Matches exactly n times, where n is a non-negative integer. For example, o{2} matches the first two o's in foooood, but does not match the o in Bob.

{n,}

Matches at least n times, where n is a non-negative integer. For example, o{2,} matches all the o's in foooood, but does not match the o in Bob.

TIP:o{1,} is equivalent to o+. o{0,} is equivalent to o*.

{n,m}

Matches at least n and at most m times, where m and n are non-negative integers. For example, o{1,3} matches the one o in Bob and the first three o's in fooooood.

TIP:o{0,1} is equivalent to o?.

[xyz]

Matches any one of the enclosed characters. For example, [abc] matches the a in plain and the c in lock.

[^xyz]

Matches any character not enclosed. For example, [^abc] matches the p in plain, but does not match cab.

[x-z]

Matches any character in the specified range. For example, [a-z] matches any lower-case alphabetic character in the range a through z.

[^x-z]

Matches any character not in the specified range. For example, [m-z] matches any lower-case character not in the range m through z.

\b

Matches a word boundary (the position between a word and a following space). For example, er\b matches the er in never, but not the er in verb.

\B

Matches a non-word boundary.For example, ear\B matches the ear in early, but not the ear in appear.

\d

Matches a digit character. Equivalent to [0-9].

\D

Matches a non-digit character. Equivalent to [^0-9].

\f

Matches a Form-feed character.

\n

Matches a New Line character.

\r

Matches a Carriage Return character.

\s

Matches any white space including space, tab, Form-feed, and so on. Equivalent to [ \f\n\r\t\v].

\S

Matches any non-white-space character. Equivalent to [^ \f\n\r\t\v].

\t

Matches a tab character.

\v

Matches a vertical tab character.

\w

Matches any alphanumeric character, including underscore. Equivalent to [A-Za-z0-9_].

\W

Matches any non-alphanumeric character. Equivalent to [^A-Za-z0-9_].

\num

Matches a remembered match referenced by num, where num is a positive integer. For example, (.)\1 matches two consecutive identical characters.

\n

Matches n, where n is an octal escape value of one, two, or three digits. This allows ASCII codes to be used in regular expressions. For example, \11 and \011 both match a tab character.

  • \0011 is equivalent to \001 followed by a 1.
  • Octal escape values must not exceed 256. If they do, only the first two digits comprise the expression.

\xn

Matches n, where n is a two-digit hexadecimal escape value. This also allows ASCII codes to be used in regular expressions. For example, \x41 matches A.

TIP:\x041 is equivalent to \x04 followed by a 1.

Reference
