Dashboard XML Syntax

The XML elements completely define the composition of the dashboard page.

The Dashboard page's XML editor provides word completion for elements defined in the XML schema.

TIP: Press CTRL+SPACEBAR to display a list of possibilities within the editor.

Page (Dashboard XML)

The page element holds all definitions for the target dashboard. It can contain the following elements:

  • panel
  • message
  • external
  • report
  • browse

Panel (Dashboard XML)

The panel element can be used to group other dashboard components. It can contain the following elements:

  • panel
  • message
  • external
  • report

TIP: Panels can be nested to structure the layout.

Attribute

Values

Description

title

text

The optional title of the panel.

columns

1, 2, 3, 4, 5, 6*

The number of columns the element spans.

* Default value.

Example

<Page>
<Panel title="First Panel" columns="3">
</Page>

Message (Dashboard XML)

The message element defines text that is displayed on the dashboard.

TIP: Text specified can contain HTML.

Attribute

Values

Description

type

warning

info

Display the message using the system's standard presentation formats as follows:

  • warning - a warning message.
  • info - an informational message.

columns

1, 2, 3, 4, 5, 6*

The number of columns the element spans.

* Default value.

Example

<Page>
<Message type="info" columns="3">This is a message with a <a href="https://google.com">link</a></Message>
</page>

External (Dashboard XML)

The external element defines an external site for display in the dashboard.

NOTE: External websites need to be whitelisted in the Web.AppSettings.config file's AllowedDashboardDomains key first, for reasons of security.

Attribute

Values

Description

url

text

The full URL of the external site.

NOTE: The external site is included in the dashboard through an iframe. Websites which do not permit display within an iframe will not render in the dashboard.

min-height

number

The minimum height of the element (in number of pixels).

columns

1, 2, 3, 4, 5, 6*

The number of columns the element spans.

* Default value.

Example

<Page>
<External url="https://google.com" columns="3" />
</page>

Report (Dashboard XML)

The report element defines a report module to display.

Attribute

Values

Description

pagesize

number

Number of rows displayed per page in the main grid.

columns

1, 2, 3, 4, 5, 6*

The number of columns the element spans.

query

number

The query id.

url

text

The URL of the reporting page (that is, the part after quantum/).

NOTE: This can include the query string of the URL (the part after ?).

view

text

The view id.

after

number

Period before a new run is executed (in seconds).

latest

false*

true

When set to true, the report will display the latest report generated by Scheduler.

autorun

false*

true

When set to true, the report will run automatically when the page loads.

live

false*

true

When set to true, the report will run Live.

* Default value.

Example

<Page>
<Report url="/straightthroughprocessing/deal-approval" query="-1" pagesize="10" autorun="true" columns="3" />
</Page>

Example (Charts)

<Page>
<Report url="/reporting/cash-flows" query="1797" view="DataView2127" pagesize="3" autorun="true" />
</Page>

Example (Autorun)

Autorun in the context of the Dashboard means to automatically run the query only once. In order to run the query periodically, add after="{interval in seconds}". For example:

<Report url="/straightthroughprocessing/deal-approval/" query="Default" view="Default" autorun="true" after="5" />

We recommend removing live="true" from the configuration, because live and autorun shouldn't be enabled at the same time. If both are enabled, the system will only perform autorun.

Browse (Dashboard XML)

The browse element defines the page records to be displayed in the dashboard from browse lists for applicable pages (that is, Dealing†, Pricing, Instruments, Rates, Static Data, and Rules).

† Includes Dealing page variants.

NOTE: The <Browse> element does not currently support auto-run or live updating. You will have to refresh the dashboard page to see the latest data.

Attribute

Values

Description

url

text

Number of rows displayed per page in the main grid.

view

text

The view id.

columns

1, 2, 3, 4, 5, 6*

The number of columns the element spans.

pagesize

number

Number of rows displayed per page in the main grid.

* Default value.

Example

The syntax below will display a grid containing three Cross Currency Swap deals per page, that are returned by the View named "open".

<Page>
<Browse url="/dealing/cross-currency-swap" view="open" pagesize="3" columns="3" />
</Page>
