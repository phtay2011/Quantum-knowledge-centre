New – A quantitative inception date test

For non-shortcut hedges, Quantum Six now enables you to perform an initial prospective quantitative test using information as it existed at the hedge inception date. This test supports FASB guidance, paragraph 815-20-25-3.

New functionality

To set both prospective test timing fields:

  1. In the Hedge Strategy page (Accounting menu), set up a strategy and use the Effectiveness tab to set up inception test.
  2. Save the Hedge Strategy.
  3. Then open the Hedge Relationship, Effectiveness tab.

    Prospective Test Timing populates automatically with the same information.

To set up the test and view the results:

  1. In Prospective Test Timing, check (or select) Inception Dates.
  2. In Prospective Test Method, select Cum Dollar Offset, Per Dollar Offset, or Regression.
  3. Select the Range for the test.
  4. Save.

    A quantitative Inception Date Test runs, and the system stores the results in two new tables.

New fields

PT Timing is a new field in HR. The field includes:

  • INCEPTION DATE (default) - make a single run of the test as at Inception Date of Hedge Relationship (HR).
    • Use for first revaluation run of HR.
    • Don’t post G/Ls.
    • Calculate G/Ls for Position Date of revaluation run. When running Position for subsequent valuation dates, don’t calculate another test.
  • INCEPTION DATES AND POSITION DATES - runs a test at Inception Date, and subsequent tests at each Position Date. Calculate test results for all Position Date runs.
  • POSITION DATES - makes a Position Date run.

Auditing

For all test methods, the system creates a quantitative Inception Date Test, with the results stored in two new tables.

For auditing purposes, you can view effectiveness results on the dealing page for the Hedge by opening the Effectiveness tab.

You can reconcile the test with the results from running Deal Revaluation with the Position Date = Inception Date.
