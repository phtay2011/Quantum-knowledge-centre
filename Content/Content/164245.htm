Hedge strategy procedures

TIP: A Hedge Strategy page is effectively an Instrument on which hedge relationship deals may be based, and this is reflected in use of the term Instrument in these procedures.

On the Hedge Strategy page:

To list Hedge Strategy pages (including groups)

  1. Click the button.
  2. Optionally follow the Configure link to change list properties - see Configure Browse list views.

To add a new Hedge Strategy page (transaction-level)

  1. In the New menu, select Item.
  2. Enter a unique Name for the Hedge Strategypage, then enter a Short Name and Description.
  3. Optionally select an existing Instrument Grouping or leave as NONE (leave GroupTransactor set to Transaction).
  4. On the Details tab:
    • In Hedge/Exposure, select the hedge type: Fair Value Hedge or Cash Flow Hedge.
    • Select the Designated Risk being hedged: INTEREST RATE RISK, EXCHANGE RATE RISK, OVERALL MARKET RISK, CURRENCY OPTION - INTRINSIC, or FOREIGN EXCHANGE - SPOT.

      Only EXCHANGE RATE RISK is valid when Enable Recognition Reclassifications is selected.

    • (Optional for cash flow hedges with EXCHANGE RATE RISK only) To specify a Dual Purpose hedge, select Enable Recognition Reclassifications and:
      • Select Include Forward Points if Forward Point reclassifications are to be generated from Recognition Date.
      • Select One Off Reclassification at Recognition Date if that type of reclassifications is to be generated.
    • (FAIR VALUE hedge only) In Posting based on select whether you want postings to the General Ledger based on the EXPOSURE or HEDGE deals.
    • In Hedge Dedesignation select AMORTIZED or DEFERRED to specify how you want Quantum to treat other comprehensive income (CASHFLOW Hedges) or MTM Revaluation gain/loss (FAIR VALUE Hedges) when an Exposure linked deal is removed from a hedge relationship.

      NOTES:

      • Only DEFERRED is valid when Enable Recognition Reclassifications is selected.
      • Not available (set to DEFERRED) when Designated Risk is Currency Option - Intrinsic or Foreign Exchange - Spot.
    • (Optional) If required for a cash flow hedge, in Default OCI Release Account select the default account for the reclassification of OCI.
    • Select whether the Maturity Gain/Loss is to be REALIZED or UNREALIZED.

      Only REALIZED is valid when Enable Recognition Reclassifications is selected.

  5. On the Effectiveness tab, choose the effectiveness calculation method and enter effectiveness target values.
  6. On the Documentation tab, record the documentation details of, for example, your hedge objective, risk management policy, or effectiveness measurement rules.
  7. (Optional) On the Events tab, define any customized events you want to add to a hedge relationship.
  8. (Optional) On the Analysis Codes tab, select analysis code values for reporting purposes.
  9. Save the Hedge Strategy page.

You have defined a Hedge Strategy page according to the settings required. You can apply this strategy to multiple hedge relationships that share the properties and information it provides.

To add a new Instrument Group

  1. In the New menu, select Group.
  2. Enter a unique Name for the group, then enter a Short Name and Description.
  3. To include your new group within another group, select an existing Instrument Grouping. Otherwise, leave as NONE.
  4. To add instruments to the group, click Add Member, select an instrument from the Browse dialog box, and click OK.

    TIP: You can also add an instrument to a group when editing that instrument by setting Instrument Grouping.

  5. Save your new group.

To edit Instrument details

NOTE: If you change the values for an instrument, any existing deal specifying that instrument will NOT be updated with those changes UNLESS you later change anything in that deal that will cause a recalculation. In such cases, you will be prompted to confirm that the recalculation should use the new details.

  1. Click to list available instruments, including groups.
  2. Select the instrument you want to edit (use the < and > buttons to page through, or type in the Search box).
  3. Edit the relevant fields and Save.

    NOTE: You cannot change the key Name field. Create a new instrument with the new name you want and, optionally, delete the wrongly-named instrument record.

To edit Instrument Group details

  1. Click to list available instruments, including groups.
  2. Select the group you want to edit.
  3. Edit Short Name, Description and Instrument Grouping if required.

    NOTE: You cannot change the key Name field. Create a new instrument with the new name you want and, optionally, delete the wrongly-named instrument record.

  4. To add new instruments to the group, click Add Member, select an instrument from the Browse window, and click OK.
  5. To remove an instrument from the group, click the x button to the right of that instrument in the list of members.

    TIP: You can also remove group members by deselecting them in the Browse window. Or, when editing an instrument, reset Instrument Grouping to NONE.

  6. Save your changes.

To delete an instrument or group

  • You cannot delete a record if that affects transaction, settlement, or financial data, and you cannot delete a group if it has members.
  • Deleting removes the record completely from the database. To prevent use of the record but retain it in the database, deactivate the record instead.
  1. Click to list available records.
  2. Select the record you want to delete (use the < and > buttons to page through, or type in the Search box).
  3. In the menu, select Delete.
  4. When prompted, confirm your action.

To deactivate an Instrument or Group

TIP: You cannot deactivate an Instrument that is in active use.

  1. Click to list available records, or use the Search box.
  2. Select the record you want to deactivate.
  3. In the menu, select Deactivate. This action will not appear if the record is already deactivated.

To reactivate an Instrument or Group

  1. Click to list available records, or use the Search box.
  2. Select the record you want to reactivate.
  3. In the menu, select Reactivate. This action will not appear if the record is already activated.
