Sample set for Regression

Like the target measurements, the sample set is defined by parameters on the Effectiveness tab of the Hedge Relationship page, as follows.

Sample style

This setting influences the size of the sample. Select one of:

  • FIXED START DATE - to fix the date of the first point in the regression sample. As the position date moves forward, new points are added to the existing sample. The size of the sample grows over time, but has a minimum of the Initial Sample Size value.
  • ROLLING - to move the date of the first point in the regression sample relative to the latest position date. The size of the sample is fixed at one more than the Initial Sample Size value.
  • DECLINING MATURITY - the term to maturity of the instrument valuations is declining between the respective effectiveness testing data points. The purpose is to include the run-down of any time value difference between the hedge Instrument or hedged item (hypothetical derivative) within the overall effectiveness result.

Interval

The Interval specifies the period between revaluations, which determines their frequency. Generally, you would match this period to the life of the intended hedge relationship. For a 10-year hedge relationship, for example, you would look back over a 10-year period.

For Prospective Effectiveness testing only, you can optionally select an Interval of ACCOUNTING PERIODS. Regression calculations take place on the dates loaded as accounting periods. (Accounting periods are defined by the G/L Owner associated with the Entity field for the hedge relationship.) When a position date does not coincide with an accounting period, the last interval will be shorter and will end on the position date.

For both types of Effectiveness testing, you can enter a number from 1 to 999 and select a unit of time in the adjacent drop-down list. Regression calculations take place at this interval, on the anniversary of the Inception date of the hedge relationship. If the Position Date does not coincide with an anniversary, the last interval will be shorter and will end on the Position Date.

For the sample, the system will always use the specified Initial Sample Size. If you select an interval that does not support this, a warning message appears.

Maturity

When the defined cycling steps past the Maturity Date of the Hedge Relationship, or the End Date of the earliest linked deal, then we will reset the position date back to the start date. In other words,

If Position Date +N interval cycles is greater than or equal to Min (HR Maturity Date, Linked Deal Maturity Date), then the system resets the position date to the Position Date.

When the Position Date cycles back to the Start Date, the cumulative change will reset to zero and the system will not calculate movement for this date. The system excludes reset points from the observation number and from regression calculation.

Initial sample size

The Initial Sample Size specifies the number of revaluations used at the outset, before any post-inception calculations are made.

Specify the size of the required sample. Enter a whole number between 3 and 999 (30 might be considered a reasonable sample size).

NOTES:

  • Appropriate rate records must exist in order to cover the selected sample.
  • The system requires one more than this number of dates to calculate the regression statistics. Where there are fewer actual data points, the system generates historic data points by using an image of the deal at inception and historic rates.
  • If the associated Sample Style is ROLLING, the start date changes but the overall sample size does not. Regression calculations are always based on the number of dates identified here.
  • If the associated Sample Style is FIXED START DATE, the start date does not change but the overall sample size will increase - once there are enough data points to generate more than the Initial Sample Size value.
