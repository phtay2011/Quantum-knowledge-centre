QID Import File Commands

NOTE: The commands described in this section are available only when using the Quantum Import Data (QID) format.

Save, Insert, Modify Commands, Import Files

One of these commands must be included in the import file at the end of each record to save the record before starting the next one.

  • $DoMenu = mfSave

Generic save (inserts or modifies as appropriate)
  • $DoMenu = mfInsert

Save a newly created record
  • $DoMenu = mfModify

Save an existing record after modification

It is recommended that you use Insert or Modify as appropriate, rather than Save. This provides a control for the following situations:

  • If you are importing new data into the Counterparty table (for example) and use a Counterparty name that already exists, the save is correctly avoidedonly when you use the Insert command, preventing inadvertent changes to live records.
  • When using the Importer to modify existing data, the Modify command will correctly fail if the record does not already exist (using Save would result in a new record being added incorrectly).

Example

Commands to modify an existing Deal Set:

EXEName=DEALSET
$DoMenu=<set the mode>
deal_set_id=<Deal Set No.>
<items that you wish to modify>
$DoMenu = mfModify

NOTE: To recall records to modify or delete, use the $SetKey command to define one or more key fields.

Actual and Projected Cash Flow Commands, Import Files

These commands are used to switch between modes in the Actual and Projected Cashflows program.

  • $DoMenu = moactual

Actual cashflow
  • $DoMenu = moprojected

Projected cashflow

TIP: This can also be achieved by specifying the same value for the cflow_type virtual field - see Importing Actual and Projected Cashflows.

Cashflows can be Bank or Broker cashflows:

  • $DoMenu = mobankmode

Bank cashflow
  • $DoMenu = mobrokermode

Broker cashflow

TIP: This can also be achieved by specifying the same value for the cparty_type virtual field - see Importing Actual and Projected Cashflows.

Cashflow Status Check Off / On Commands, Import Files

NOTE: These commands are available for Money Market Deals only.

When you delete a deal, Quantum checks the settlement status of the cashflows on that deal, and issues a warning message if any cashflows have been settled. This warning message can interfere with processing deal deletions using the Importer.

To prevent the cashflow settlement status check from occurring, use the following command:

$DoMenu = cfStatusCheckOff

At the end of processing, switch status checking back on by issuing this message:

$DoMenu=cfStatusCheckOn

Deal Set Commands, Import Files

Specifying the Deal Set Import mode

Before you import a Deal Set, you need to specify the Deal Set mode: either Forecast or Standard.

NOTE: If you do not specify the mode, the Deal Set is imported using the current mode set for the program.

Use one of the following commands.

  • $DoMenu=mncForecast

Forecast mode
  • $DoMenu=mncDealSet

Standard mode

Modifying and deleting Deal Sets

Use the Modify and Delete Import file commands.

Importing Deals belonging to a Deal Set

You import deals that are linked to a Deal Set in the same way that you import individual deals, but with additional fields. See Importing deals that belong to a Deal Set.

Delete Command, Import Files

To delete a deal

  • Specify the deal name, immediately followed by the mfDelete command, as shown in this example:

    deal_no = 12345

    $DoMenu = mfDelete

To delete a Deal Set

  • Use the following method:

    EXEName=DEALSET

    $DoMenu=<set the mode>

    deal_set_id=<Deal Set No.>

    $DoMenu = mfDelete

NOTE: To recall records to modify or delete, use the $SetKey command to define one or more key fields.

Exit Command, Import Files

Use the Exit command at the end of the Import file to terminate the Quantum program.

$DoMenu = mfExit

This is advised for security purposes.

FX Deal Commands, Import Files

  • $DoMenu = moFXDEAL

Normal FX Deal mode
  • $DoMenu = moFX_Swap

FX Swap mode
  • $DoMenu = moFX_Back_to_Back

Back to Back FX Deal mode

TIP: This can also be achieved by specifying the same value for the fx_type virtual field - see Importing FX Deals.)

Linked Deal Commands, Import Files

You can link a deal to a Deal Set, a Hedge Relationship, or both.

Deal Sets

Use the same commands to import a Deal Set as you do to import individual deals, but also specify the Deal Set mode. See Importing Deal Sets.

Similarly, use the same commands to import a deal that is linked to a Deal Set, but with additional field information. See Importing deals that belong to a Deal Set.

Hedge Relationships

Use the same commands to import a Hedge Relationship as you do to import other deals. See Save, Insert, Modify. The provided fields identify the Hedge Relationship, and then the required action is specified.

However, deals linked to a Hedge Relationship require additional commands. Within a Hedge Relationship, each individual linked deal must also be identified, with the required action for that linked deal specified in the line that follows.

TIP: Import commands for linked deals all include the word Detail - for example, mfAddDetail, mfModifyDetail, and mfDeleteDetail.

Default

The default command is mfAddDetail, which is assumed if no command is provided after hrlinks=.

Working only with linked deals

You will need to provide enough details (Deal Number, Transaction Type, and Start Date) to identify which Hedge Relationship the deals are linked to, and then provide details of the changes that are to be made to the linked deals.

User Default Commands, Import Files

The importing process can be simplified if you use the first record to set default values for some fields. Each subsequent record then needs only entries for fields not covered by the defaults.

These commands equate to using the menu options Save Defaults, Enable Defaults, and Disable Defaults to speed up manual data entry.

To use this facility, set field values as normal and then issue the Save Defaults menu command. During the importation process it is possible to subsequently enable and disable these defaults by using the following syntax.

  • $DoMenu = muSaveDefaults

Save and Enable user specified defaults.
  • $DoMenu = muDisableDefaults

Disable user specified defaults.
  • $DoMenu = muEnableDefaults

Enable user specified defaults.

User-Definable Key Fields, Import Files

Another feature of the Importer is the ability to specify user-definable key fields, which can be used to recall records to modify or delete.

CAUTION: Using a key field other than the Deal Number to identify a deal you want to modify carries the risk of updating the wrong deal. If more than one deal in your database has the same key value, the system will update the first deal it locates with that value. You must take that risk into consideration before use this feature.

To define one or more key fields

  • Use this command:

    $SetKey = fieldname1[,fieldname2…]

    Where fieldname1 is the name of the first field you wish to use as a key, and so on.

For example:

  • To use the ticket number as a key field (see CAUTION above):

    $SetKey = ticket_no

  • To specify the ticket number and dealer:

    $SetKey = ticket_no,dealer

You can also specify an analysis set as a user defined key, as follows:

$SetKey = analyse|SetName

Where SetName is the analysis set you want as a user-defined key (see CAUTION above).

User-defined key fields can only be fields on the main table for a given program. For dealing programs, this is the deals table, and key fields could not be specified as fields or analysis sets from the mmdeals table, for example.

When you have defined the key field, it remains in use for the remainder of the import file or until it is changed or cleared.

To use a key that you have defined

  • Specify a value for the key field.

    The system will search for a matching record, and:

    • if only one matching record, that record will be loaded,
    • If more than one matching record, the first one will be loaded (see CAUTION above), or
    • if there are no matching records, no record will be loaded.

Example

$SetKey = ticket_no
ticket_no = AX100
face_value = 1000000
$DoMenu = mfModify

This example will EITHER:

  • change the face value to 1,000,000 on the first deal record whose Ticket Number is AX100 (see CAUTION above), OR

do nothing if no deal records are found with a Ticket Number of AX100.
