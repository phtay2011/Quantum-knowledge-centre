Implementing SWIFT Authentication

NOTE: Only users with specific Module Security permissions may change the first and/or second part of an authentication key.

Use the SWIFT Local Authentication Rule page to view current settings, enter a new rule, or edit an existing one. For each rule:

  • specify the Matching Criteria - select one External Method (or ALL) and one Entity or Entity group (or ALL);
  • select either the SA-2 or HMAC-SHA256 Algorithm Type, and
  • enter one or both parts of the key (requires a specific Module Security permission for each).

TIP: You should change your authentication key on a regular basis - for example monthly. Check with your External Matching or payments service provider.

If a key is to be changed:

  • If any users are processing Electronic Confirmations or EFTs, ask for those processing runs to be closed down. Also ensure no users will generate any further Electronic Confirmation or EFT output until you have finished changing the keys.

    If you do not, the wrong key could be applied, resulting in the rejection of that message by the applicable External Matching or payments service.

  • It is your administrator's responsibility to synchronize:
    • changes to each key part, when made by different users; and
    • key changes in Quantum with identical changes in the receiving External Matching or payments service interface.
  • You must be aware that the system cannot ensure that the correct key parts are entered.

    TIP: You can check this after each key change by sending a dummy message to ensure that the key is accepted at the external destination, before attempting to generate live messages.

When using the SWIFT Local Authentication function:

  • You cannot access the page unless you have at least the Inquire Module Security permission for SWIFT Local Authentication. If you do, only items you have rights to change (if any) can be edited.
  • You cannot change a key unless you have the specific Module Security permission (Change first part of key or Change second part of key).
  • The two parts of the key can be entered and saved separately.
  • Any changes do not take effect until the next applicable Confirmations or EFT processing run starts.
