General Syntax

FOR ( ACCOUNT number | ACCOUNTSrange) | ALL ACCOUNTS ON STATEMENT | TRANSACTION {
IronPython Code
}

Where:

  • The FORAccount Selection clause is the same as with legacy data manipulation syntax - namely:
    • number is an account number string. If the account number has hyphens in it, then the account number must be enclosed in quotes - for example "123456-8976".
    • range specifies the first and last account number in a range, in the following format:

      account numberTOaccount number

  • ON STATEMENT causes Bank Importer to apply the IronPython code at a bank statement level to all accounts that match the given account selection criteria. The current statement data is exposed to the IronPython code via the Statement object.
  • ON TRANSACTION causes Bank Importer to apply the Iron Python Code at a transaction level to all accounts that match the given account selection criteria. The current transaction data is exposed to the Iron Python code via the Transaction object and the current statement data via the Statement object.
  • Iron Python Code represents lines of valid Iron Python code.
  • The brace characters { and } enclosing the Iron Python code are required.

Example of IronPython Scripting

The following example converts the 583 balance to a Debit transaction.

FOR ALL ACCOUNTS ON STATEMENT {
if source.SourceType == "BAI":
if source.Balances.HasBalance("583"):
bal = source.Balances.Balance("583")
if bal.Amount != 0:
t = statement.NewTransaction()
t.Amount = bal.Amount
t.ValueDate = statement.ClosingBalanceDate
t.TransactionType = "583"
t.DebitCreditMark = "D"
}

Important Notes

When creating your IronPython script note the following general rules.

Indentation

Indentation is critical in IronPython as it determines the scope of the statements. Adding another indentation level is like adding a pair of {} around the code at that level. This means that variables may not be available anymore at a different indentation level.

Case sensitivity

IronPython is case-sensitive. Quantum Six provides three variables, statement, transaction, and source, which must be in LOWER CASE* form.

*NOTE: In the legacy syntax, these variables use an initial capital letter.

Whitespace

Quantum Six Bank Importer removes whitespace from the beginning of the first statement.

For example:

This...

Becomes...

Ccy = statement.Currency
if Ccy == "CNY":
statement.Currency = "CNH"

Ccy = statement.Currency
if Ccy == "CNY":
statement.Currency = "CNH"

This will result in an "unexpected indent" exception.

Therefore, you should always begin your code without any indents.

Ccy = statement.Currency
if Ccy == "CNY":
statement.Currency = "CNH"

OR

if statement Currency == "CNY":
statement.Currency = "CNH"

Miscellaneous

  • Use + for string concatenation.
  • Variable declaration is not required in IronPython.
  • All legacy syntax conditional constructs will work in IronPython; for example:

    FOR ALL ACCOUNTS WHERE CCY="CNY" SET CCY="CNH"
