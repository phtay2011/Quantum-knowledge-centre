Analytics - Uploading Limits Definition data

Using the Quantum Static Data Import Service, you now have another method to upload Limits Definitions data into Analytics. Under this new method, when you place a Limits Definition csv file in a specific folder, the system automatically imports that file into Analytics.

Prerequisites

  • Create a folder named AnalyticsImports in your site directory under StaticImportation\Import.

    The full path name will be AvantGard Quantum\Site\SiteName\StaticImportation\Import\AnalyticsImports, where SiteName is replaced by the name of your site.

  • On the Operational Settings page under Active Services, select the Import static data option.
  • The Static Data Import Service must be running.

Importing a Limits Definition file using this method

Place the Limits Definition csv file into the AnalyticsImports folder.

The csv file must:

  • in the first line, have Path as the first column name, followed by all the Limit Definition page's Limit Details grid column names, in the same sequence as in the grid;
  • in the second line's first value, have a pathname that ensures the uploaded data goes to the correct Limits Definition, in the format ~\Limits\LDname, where LDname is the Limit Definition record Name in Quantum;
  • in the second line's subsequent comma-separated values, have valid settings for the corresponding fields; and
  • in any (optional) further lines, have the first value empty, and the subsequent values with further valid settings for the corresponding fields.

Example file

The following graphic shows the first few columns of a csv file to be imported into the Limits Definition record named Limits_Test (as the file would be displayed in Microsoft Excel).

The system imports and merges each line of the file into the grid. If there is an existing row with the same Matching Criteria* values, the system replaces that row by the imported line values. If not, it adds a new row.

*The Matching Criteria are the first group of values from Counterparty through Term End. The remaining values define the limit to be applied when result set records meet the Matching Criteria for the Limit entry.
