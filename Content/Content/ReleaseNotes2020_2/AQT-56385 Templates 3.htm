Templates page enhancements

In this release we have made various small changes to Enterprise Workflow (EW) Templates page, found in Configuration > Templates.

For more information about workflow templates, search for the Knowledge Center topic "Template".

Data Transfer Location – Workflow In/Out

On the Data Transfer Location (DTL) page, when you set the following, this provides an action with settings for each Input on the named template, similar to how FILE and QUEUE work. Set:

  • Location Type to WEB API, and
  • Purpose to WorkflowOut.

For more information, search topics in the Knowledge Center:

  • "Template"
  • "Data Transfer Location (DTL)"
  • "Configure API Connections"

First and Last item in loop

You can now identify the first and last item in a loop of template output.

Whitespace and newline handling altered

Quantum now handles whitespace and newline in the following consistent manner..

For a line that contains #each, /each, #if, /if, #unless or /unless:

  • Don't output the line if the rest of the line only consists of whitespace.

  • Output the line if the rest of the line has any non-whitespace characters

NOTE: This may affect existing templates.

We have changed whitespace handling, so Template pages written prior to 2020.2 may now output less whitespace and fewer line feeds. Please review existing Template pages if this is critical to your output.

True/False is now Yes/No

Any Boolean value that goes into the output and in the preview area now has a value of Yes or No and not True/False.

Template implicitly applied to each row

In 2020.1, if you don’t specify an #each statement, the system applies the template XML only to the first row of the data. We have resolved this for templates that don’t have #each statements by applying the whole template to each row. However:

  • You can use @index to indicate the sequence number of the rows being processed by the template.
  • You cannot use @index outside of any explicit #each statement.
  • You can use @first and @last with an implicit #each block.

Empty string allowed for text

You can now use an expression in a text input where the string is blank (empty string).
