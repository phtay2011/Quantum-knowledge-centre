User Maintenance and Resource Security

When you save a change to a User or User Groups that affects current Resource Security permissions, Resource Security will automatically recalculate the permissions. This includes additions, deactivations, reactivations, group changes and deletions.

This ensures Resource Security automatically maintains any changes in Individual Users/Groups program without your having to access Resource Security.

Adding a User

The permissions of new Users will be inherited from their User Group or Groups. If the new User is not in any User Group the default permission for the User will be No Access.

Adding a User Group

When you add a new User Group, the default Resource Security permissions for the new User Group will be No Access.

Modifying a User Group

Whenever a User is moved to or removed from a User Group, the User will lose all permissions that were applied directly. If they are moved to another User Group their permissions will be applied using inheritance.

Deleting a User Group

Before you can delete a User Group you must remove all members of the Group. See Delete a User from a User Group. When a User Group is deleted, no permissions are required and there will be no reference to it in Resource Security.

Deactivating a User or User Group

The permissions that a User or User Group had when deactivated will remain constant while deactivated. This provides a historical audit of the User's last permissions. A deactivated User cannot logon, and it is not possible to update permissions while the User or User Group is deactivated.

Any active Users in a deactivated User Group will not inherit permissions from the deactivated User Group. The Users will lose their direct permissions and have their permissions recalculated based on inheritance when the deactivated User Group is saved.

Reactivating a User or User Group

When a User is reactivated in Individual Users/Groups program, the permissions will be applied using inheritance. Any direct permissions that a User had when deactivated will be replaced by inheritance.

When a User Group is reactivated the permissions it had before it was deactivated will be retained. The members of the User Group will inherit the permissions of the group. Any direct permissions that a User had are replaced.
