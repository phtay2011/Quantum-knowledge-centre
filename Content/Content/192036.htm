FX Cover Setup

These are the configuration options and prerequisites for implementing FX Cover.

Auxiliary Files

The following auxiliary files are provided in your Site's AppData folder.

  • hdgfxmap.txt - Maps Projected Cash Flow data to FX Cover Request data
  • hdgtempl.txt - Template file giving format for exported FX Deal QID files; maps FX Cover Request data to FX Deal data

If you have special requirements, these files can be modified to cater for these. For example, the Comments text box (blank by default) could be set to FX Deal Created from FX Request n (where n is the Request number).

CAUTION: Extreme care is needed when changing these files, and you should always keep backups of the original files supplied with Quantum. It is strongly recommended that any customization is done by,or under the supervision of, your FIS Consultant.The following must be set up in Quantum for FX Cover processing.

Dealer

A special Dealer user named AUTOMATED.

Analysis Codes

Ensure that the following Analysis Codes are available on the Analysis Code page.

Projected Cash Flow Analysis Codes

A Projected Cash Flow analysis code set named Exposure Approval must be set up with the Pending, Approved, and Rejected codes for the FX Cover Processor to work. Only Projected Cashflows with Exposure Approval set to Approved will be picked up by the FX Cover Processor.

Counterparty Analysis Codes

A Counterparty analysis code set named Domestic Currency must be set up with a code for each currency needed by Counterparties using the FX Cover process. This is referenced in FX Cover Rule Set page.

Instruments

Cash Flow Instruments

Two Cash Flow Instruments named Committed and Anticipated are required for attaching FX Cover rules. These are selected in the FX Cover Rule Set page when defining the rules.

FX Instruments

One or more FX Instruments must be provided for use when the FX Deal is created from FX Requests. These Instruments are then configured in the FX Cover Rule Set page.
