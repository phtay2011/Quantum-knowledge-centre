Bulk Processing Currency Options

On the Currency Options Processing page, you can now Lapse, Knock In, or Knock Out multiple deals.

Implementation

In the Currency Options Processing page's Results view, a selection check box has been added that enables you to multi-select deals and then select the Lapse, Knock In, or Knock Out* action in the grid menu. In each case, a dialog box is displayed, showing:

  • Date for actioning all applicable deals - defaults to the current date,
  • Dealer for all applicable deals - defaults to the logged-on Dealer user,
  • Current Market Rates for each applicable currency pair - these default to the current exchange rates in each case,
  • Comments - explanatory text for all applicable deals, and
  • OK and Cancel buttons.

* The Knock Out dialog box also shows the Rebate Date for payment of any rebate. (If only one deal has been selected, the Rebate Amount for that deal is also shown).

For a Lapse action, when you click OK on the dialog box, it closes and:

  • the system will attempt to Lapse the selected deals;
  • a message will be displayed indicating the number of deals lapsed and any detected errors;
  • in the results list, successfully actioned deal Status values change to Lapsed; and
  • a Messages button appears if errors have been detected - that is, when you have attempted to Lapse an option after its Exercise Date or before its Deal Date.

    Clicking this button will open the Message Log query page set to the Default query, with the Reporting Period set to Today and Module set to CurrencyOptionDealLapse. Click Run Query to view the error messages, which will show the Deal Number and a description of the error in each case.

For a Knock In action, when you click OK on the dialog box, it closes and:

  • the system will attempt to Knock In any eligible Barrier options among the selected deals;
  • any selected deals that are not Knock IN Barrier options will be ignored,
  • a message will be displayed indicating the number of deals knocked in and any detected errors;
  • in the results list, successfully actioned deal Status values change to Knocked In and a second entry is added to the list in each case, with a Type value of Exercise Lapse and a Status of None; and
  • a Messages button appears when errors are detected - that is, when you have attempted to Knock In an option after its Exercise Date, before its Deal Date, or where the Currency Rate does not breach a barrier level.

    Clicking this button will open the Message Log query page set to the Default query, with the Reporting Period set to Today and Module set to CurrencyOptionDealKnockIn. Click Run Query to view the error messages, which will show the Deal Number and a description of the error in each case.

For a Knock Out action, when you click OK on the dialog box, it closes and:

  • the system will attempt to Knock Out any eligible Barrier options among the selected deals;
  • any selected deals that are not Knock OUT Barrier options will be ignored,
  • a message will be displayed indicating the number of deals knocked out and any detected errors; and
  • in the results list, successfully actioned deal Status values change to Knocked Out; and
  • a Messages button appears when errors are detected - that is, when you have attempted to Knock Out an option after its Exercise Date, before its Deal Date, or where the Currency Rate does not breach a barrier level.

    Clicking this button will open the Message Log query page set to the Default query, with the Reporting Period set to Today and Module set to CurrencyOptionDealKnockOut. Click Run Query to view the error messages, which will show the Deal Number and a description of the error in each case.

Restrictions

Only the above processing actions support multi-select (other actions become unavailable when you select more than one deal).

History

Previously, on the Currency Options Diary page, you could not multi-select deals in the results view, for any processing action.
