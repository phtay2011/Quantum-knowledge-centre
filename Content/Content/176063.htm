About CLS in Quantum

CLS (Continuous Linked Settlement) is a unique process for Foreign Exchange and Currency Options. It:

  • enables cross-border currency transactions to be settled intra-day,
  • eliminates settlement risk and substantially reduces operational risk in FX markets (because the CLS Service Provider carries the risk), and
  • does not involve the sending of any EFT messages.

    Settlement takes place on a PvP* (Payment-versus-Payment) basis and amounts are automatically debited or credited to the Bank Account held with the CLS Service Provider following the matching of the MT304 CLS Deal Confirmation message, which is sent to the CLS Service provider (not to the Deal Counterparty, as for a non-CLS MT300 Deal Confirmation).

    TIP: PvP is a simultaneous exchange of values between settlement parties.

This section covers CLS topics common to both Quantum FX deals and Currency Options.

Who is involved in CLS?

The CLS Group describes the parties as follows:

  • Settlement Members

    "Settlement Members are also shareholders of CLS Group. Settlement Members must meet certain minimum standards for membership in CLS Bank. Settlement Members hold a direct account with CLS Bank to undertake settlement."

  • User Members

    "User Members do not have an account with CLS Bank, but are sponsored by a Settlement Member, which acts on their behalf and assists them in liquidity management and processes their payments to CLS Bank. A User Member maintains confidentiality in trading information by submitting payment instructions directly to CLS Bank."

  • Third parties

    "Third parties have no direct relationship with CLS Bank, but either a Settlement Member or User Member submit instructions and process payments on their behalf."

The CLS participation level roles in Quantum are illustrated and summarized below.

Dealing Entity - a third-party participant using a Bank Account that is a CLS Service Provider (either a CLS Settlement Member or a CLS User Member)

Deal Counterparty - a third-party participant that is also capable of CLS Settlement

Settlement Bank (Nostro Agent) - either a CLS Settlement Member or a CLS User Member (referred to as a CLS Service Provider in Quantum)

Deal for CLS Settlement - applies when EITHER:

  • both currencies on the deal are CLS-eligible and the default Bank and Counterparty Accounts are made CLS-eligible - that is, each account is in a CLS-eligible currency and its CLS Bank Clearing field is set to YES; OR
  • for an FX deal, the Is CLS property setting is Yes. See CLS workflow enhancements for more details.

NOTE The fact that a currency is supported by CLS does not necessarily mean that a CLS third party is able to settle that particular currency using CLS.

Output messages for CLS and non-CLS deals

CLS settlement

MT304 Confirmation messages for CLS FX deals are sent to the CLS Service Provider (not to the Deal Counterparty).

The system does not create EFT messages - the amounts are automatically debited or credited from the Bank Account.

Standard settlement

The system sends confirmations on deals not settled via CLS to the Deal Counterparty, and sends EFTs to the Settlement Bank to arrange for payments.

Defaults for CLS and Bank/Counterparty Accounts

Bank Account Defaulting Rules

TIP: Bank Account Defaults are mandatory. If no defaulting rule is found for a deal, it cannot be saved.

For FX deals

For each applicable Counterparty/Instrument/Currency combination:

  • Create one Bank Account Defaulting rule for CLS, setting the Transaction Type to FX Deals only and selecting the CLS option. Add an Instruction entry with the required Effective Date, Bank, CLS-eligible Account Number, and so on.

    NOTE: The CLS rule will be applied only when an applicable Cut-Off Time Rule is found for the deal and the Cut-Off Time has not been passed (the Is CLS value is set to Yes). See CLS workflow enhancements.

  • Either select an existing rule that covers the applicable combination for Standard Settlement, or create a new rule with the Standard Settlement option either selected or not shown. The rule must have an Instruction entry with the required Effective Date, Bank, Account Number, and so on.

For Currency Options

Depending on whether or not a CLS Bank Account is used only for CLS deals, the desired Bank Account defaulting rule can be selected at an instrument level.

If you are using different Bank Accounts, then provided you are using separate Instruments solely for CLS deals:

  • set up a Bank Account Defaulting Rule specifying the CLS Instrument and the CLS Bank Account, and
  • set up another Bank Account Defaulting Rule specifying the non-CLS Instrument and the non-CLS Bank Account.

If you are not using different Bank Accounts, you should still set up two Bank Account Defaulting Rules: one specifying the CLS Instrument and the other the non-CLS Instrument.

Counterparty Account Defaulting Rules

TIP: Counterparty Account Defaults are not mandatory in order to save a deal. If no default is present, the deal is not CLS-eligible.

For FX deals

For each applicable Counterparty/Instrument/Currency combination:

  • Create one Counterparty Bank Account Defaulting rule for CLS, setting the Transaction Type to FX Deals only and selecting the CLS option. Add an Instruction entry with the required Effective Date, Bank, CLS-eligible Account Number, and set Payment Category to EFT NOT REQUIRED.

    NOTE: The CLS rule will be applied only when an applicable Cut-Off Time Rule is found for the deal and the Cut-Off Time has not been passed (the Is CLS value is set to Yes). See CLS workflow enhancements.

  • Either select an existing rule that covers the applicable combination for Standard Settlement, or create a new rule with the Standard Settlement option either selected or not shown. The rule must have an Instruction entry with the required Effective Date, Account Number, and Payment Category set to STANDARD SETTLEMENT INSTRUCTION.

For Currency Options

If you are using separate Instruments solely for CLS deals (recommended):

  • For the CLS Instrument, set up a Counterparty Account Defaulting Rule with a Payment Category of EFT Not Required.
  • For the non-CLS Instrument, set up a Counterparty Account Defaulting Rule with a Payment Category of Standard Settlement Instructions or Debit/Credit Advice.

CLS and Confirmations

Operational Settings

Under Active Services, both Process Confirmations and Match Confirmations must be selected.

Confirmation Rules

You do not need to define separate Confirmation Rule records for CLS and non-CLS deals, as the External Method setting can be the same for both CLS and non-CLS deal actions (one of SWIFT Confirmation, Electronic, or Misys (for Finastra)). The system creates the SWIFT Confirmation message output in both cases; it is the applicable Electronic Confirmation Format Rule record that determines the message type, as below.

Electronic Confirmation Format Rules

TIP: You may be using modified XML Configuration and Template files, or the provided Seed files. See Supplied Confirmation XML Template Files.

For Currency Options

You need Electronic Confirmation Format rules covering:

  • each non-CLS Instrument and each (non-CLS) Counterparty (or ALL), and
  • each CLS Instrument (or group) and each (CLS) Counterparty (or ALL).

In each case:

  • set Transaction Type to COE (Exotic) or COV (Vanilla) as required,
  • select the Entity and Relationship (where ALL is valid in each case),
  • in the Settings area, set the Exercise action to either:
    • the MT300 Configuration file for non-CLS instruments, or
    • the MT304 Configuration file for CLS instruments, and

      NOTE: A PHYSICAL Exercise action will trigger creation of an FX deal, and CLS Settlement may then come into play for that FX deal.

  • set all other actions to either:
    • the MT305 file for the COV Transaction Type, or
    • the MT306 file for the COE Transaction Type.

For FX deals

You can use the same Electronic Confirmation Format rules to cover both CLS and Standard Settlement, because the same set of XML Template files caters for either MT300 or MT304 Confirmation message output, depending on the Is CLS value for each FX deal.

To do this, specify the SeedXMLConfig_MT300_MT304_CLS.xtc XML Template Configuration file in your FX deal Electronic Confirmation Template Defaulting Rules, which will generate an MT300 when Is CLS is No, and an MT304 when Is CLS is Yes. This Configuration file specifies the Layout file that is required (SeedXMLLayout_MT300_MT304_CLS.xml).

Confirmation Matching

There is no difference in the matching of CLS and non-CLS deal Confirmations. This can be done electronically via the relevant matching service, or manually.
