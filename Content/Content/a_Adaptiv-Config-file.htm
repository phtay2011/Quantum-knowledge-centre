Query settings in AAI.AAIS.exe.config file

NOTE: From Release 2020.4, you can use the Calculation Properties page to define multiple sets of any Calculation Property values, which you can then specify in applicable Assumptions Definitions (as the default in each case), and - at the query level -on the Assumptions tab for the current query run. Those settings will then override any settings of the specified properties in the AAI.AAIS.exe.config file.

You may find the following property settings in the AAI.AAIS.exe.config file useful for adapting your site's Adaptiv CVA, PFE, or Historical VaR query calculations and results across the system.

(There are other properties and settings in this file. For more details, refer to the AA Studio documentation or help.)

BaseTimeGrid and ScenarioTimeGrid

(CVA or PFE queries only.)

Use these two properties to specify the Base Time and Scenario Time grid points you prefer, in relative date (tenor) format.

For example:

  • <add key="BaseTimeGrid" value="0d 1d 1w 1m 2m 3m 6m 1y"/>
  • <add key="ScenarioTimeGrid" value="0d(1w) 1y(1m) 10y(3m)"/>

Where:

  • 0d 1d 1w 1m 3m 6m 1y means forward date grid points will be generated for zero days, 1 day, 1 week, 1 month, 3 months, 6 months, 1 year, and after that for every 6 months (the difference between the last two terms); and
  • 0d(1w) 1y(1m) 10y(3m) means forward date grid points will be generated weekly for 1 year, monthly up to 10 years, then quarterly after that.

    • TIP: This is the default.

FirstToDefaultAdjustment

(CVA or PFE queries only.)

The default setting of the AA Studio Calculate dialog value for First_To_Default_Adjustment is None, meaning that no adjustment is applied.

If you want to apply the Bilateral setting instead, you must include the following property setting in the AAI.AAIS.exe.config file:

  • <add key="FirstToDefaultAdjustment" value="Bilateral" />

Other values are also valid - for more details, refer to the AA Studio documentation or help.

ReturnMethods

(Historical VaR queries only.)

Add this property when you want to change the default settings of the AA Studio Calculate dialog value for Price Factor Return Methods.

The settings available for each Price Factor are:

  • None
  • Historical
  • Absolute
  • Absolute Cumulative
  • Proportional
  • Proportional Cumulative
  • Rebootstrap
  • Pass Though Absolute (Adaptiv Studio has a spelling mistake so this option also does)
  • Pass Through Proportional
  • Pull-to-One
  • Vol Weighted Fractional Return
  • Vol Weighted Absolute Return
  • Vol Weighted Log Return

For example, to configure InflationRate and InterestRate Price Factor Return Methods to Absolute and Historical respectively, and leave the remaining Return Methods at the default of Proportional, include the following property setting in the AAI.AAIS.exe.config file:

  • <add key="ReturnMethods" value="InflationRate=Absolute;InterestRate=Historical" />

ScenariosPath

(CVA queries only.)

Add this configuration option to define your Deterministic Scenarios from file, by specifying the file pathname. For example:

<add key="ScenariosPath" value="C:\DetScens\DSFactorArchive.xml"/>

If you include the above line, the system will pass this setting to Adaptiv through the integration layer and set the Calculation properties on the CVA query as follows:

  • Deterministic Scenario is set to FromFile and
  • Scenario Path is set to C:\DetScens\DSFactorArchive.xml.

If the scenario path line is commented out or absent, or the path setting is invalid, Deterministic Scenarios will be set to Off.
