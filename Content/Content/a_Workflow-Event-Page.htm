Workflow Event

Use the Workflow Event page to define actions to be performed when system events are triggered. The currently supported events - selectable in the Name field - are:

  • Confirmation Match,
  • Counterparty Approval,
  • Entity Approval, and
  • Strategy Approval.

The Workflow Event page has the following grids.

  • Actions - the action or actions to be triggered. These actions are independent. A prerequisite condition must hold true for the action to execute.
  • Sources - can be used in the expressions.

    TIP: You will get an error message if you have not included a required (key) data item in the Settings for the data source.

  • the counterparty in the Approval event is part of the Deal External category, and
  • the counterparty in the Approval event is also part of the Bank Internal category.

Importing and Exporting Workflow Events

You can export events using a JSON file format. The workflow eventâ€™s browse menu includes Import and Export options. The Import option is shown if the Can import data option is checked for the appropriate user in Configuration > User. The Update action for JSON import needs all inputs specified even if the change is in limited number of inputs. If all inputs are not mentioned in the JSON file, it would simply replace the ones that are mentioned and delete the ones which are not in the JSON file.

You can import events in QXT and JSON formats via the Utility > Static Data Import Status page as well.

NOTE: QXT is no longer available as an export option, however the import action supports both QXT and JSON formats.

Success or failure message for the import is shown at the top of the page.
