Importing Bank Account Rules

The Account Defaults Importer supports the following QID/QXT-format fields.

Examples

NOTE: When using these format examples, separate each line of input with a line break. Separate by pressing Enter at the end of each line.

Insert a new record

(QID Vertical format)

EXEName=ACCDEF
*
*Quantum vertical format
*
$DoMenu=mfnew
mode=Bank
entity=Our Posting Entity
trans_type=ALL
sectype=ALL
cparty=ALL
relationship=ALL
cflow_type=ALL
ccy=NZD
pay_rec=RECEIPTS
effect_dt=21-May-2010
bank=Bank of New Zealand
bankacc=DFT1
pay_method=SWIFT LB
entity_id=aaasdfg
$Domenu=mfinsert

(QID format)

EXENAME=ACCDEF
$DoMenu=mfnew
mode=BANK
entity=THE Corporation
trans_type=SE
sectype=Yield Statement
cparty=SWIFTCNF Counterparty
relationship=NONE
exchange=ALL
cflow_type=ALL
ccy=AUD
pay_rec=BOTH
effect_dt=21-Jan-2005
bank=XYZ Bank
acc_no=66663333
def_method=SWIFT
entity_ident=98306072
def_mt_format=
$DoMenu=mfinsert

(QXT format)

@BankAccountRule
$new
Entity = Owner - Level 1
TransactionType = Money Market
Instrument = Simple Int - Float Pre
Counterparty = ALL
Facility = ALL
CashFlowType = ALL
Currency = AUD
PaymentsReceipts = Both
Exchange = ALL
$$ADD Instructions#EffectiveDate = 00-Jan-1990;Bank = CITIBANK;AccountNumber = 444 333 222 111;DefaultPaymentMethod = Statement Cheque Run;AccountName = CITIBANK AUD Statement Chequ Run acc;AccountCurrency = AUD
$Insert
