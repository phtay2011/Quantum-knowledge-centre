Adding a Bank Flow Adjustment

This topic covers adding an adjusting Bank Flow after a Prior Day or Intraday Matching run.

TIP: Adjusting bank flows added in Prior Day Matching have no effect on, and cannot be seen in Intraday Matching, and vice versa.

To add an adjusting Bank Flow

This function is not available if Resource Security applies and you are not Exempt or do not have Write access to the Bank Account.

  1. On the Matching page's Overview tab, select an account in the Accounts summary.

    The Flows and Matches tabs are then displayed.

  2. (Optional) On the Flows tab, select the relevant item or items in the BANK FLOWS grid.
  3. Select New from the BANK FLOWS grid menu.

    The New Bank Flow dialog box opens.

  4. Enter the Value Date, Bank Value Date, and Amount for the adjusting bank flow you are creating. Enter the Reference Information values: Deal Number, Net Number, Check Number, Transaction Type, and so on as required (see Data items, below).
  5. In the Extra Information text box, optionally enter text giving further information about the adjusting entry.
  6. Click OK.

    Your adjusting bank flow is then added, with the specified details, and the dialog box is closed. No cash flows nor G/L entries are created.

  7. (Optional) To match the the adjusting entry to selected items, select it and those items (if not already selected in Step 2), and in the Match button drop-down list, select one of Match, Probable Match, or Possible, Match as required.

Data items

Value Date

The Quantum Value Date for the adjusting bank flow you are creating. The default date is:

  • for a detail-level adjustment - the common Value Date of your selected bank flow or cash flow entries (if any, and if they all have one date in common), or
  • in all other cases - the current reconciliation period end date.

To specify another date, use the calendar lookup, or directly type a valid date.

Bank Value Date

The Bank's Value Date for the adjusting bank flow. The default date is:

  • for a detail-level adjustment - the common Bank Value Date of your selected bank flow or cash flow entries (if any, and if they all have one date in common), or
  • in all other cases - the current reconciliation period end date.

To specify another date, use the calendar lookup, or directly type a valid date.

Amount

The amount of the adjusting bank flow in the currency of the bank account (indicated above the field). The default is the adjustment amount required to match your selected items (if any).

You can enter another amount if desired. This means more adjusting entries will be needed to reconcile the relevant amounts (either manually, or by the Auto Match process).

Bank Account area

These read-only fields show the Account Number, Bank, and Account Name, for information purposes only.

Deal Number

Type a Deal Number for the adjusting bank flow.

Nett Number

Where applicable, type a Net Number for the adjusting bank flow.

Check Number

An optional check number for the adjusting bank flow.

Transaction Type

Enter a value for, or use the lookup to select, the Transaction Type for the adjusting Bankflow you are creating.

Cashflow Type

Enter a Cashflow Type for the adjusting bank flow.

Instrument*

Optionally select a Bankflow Instrument for reference purposes, if required.

You can either select from a lookup of Bankflow Instruments defined in Quantum or type in any text value. However, if your value is not a valid Bank Flow Instrument in Quantum, this bank flow's Instrument will be displayed as None in CashXplorer.

Counterparty*

Optionally select a Counterparty for reference purposes, if required.

Ticket Number

Optional deal ticket information - for example, a transaction reference for interfacing to other deal capture systems.

Entity*

Optionally select an Entity for reference purposes, if required.

Cash Flow Instrument*

Optionally select an Actual Cash Flow Instrument for reference purposes, if required.

*NOTE: When Resource Security applies, available field values will be filtered based on your allowed access to them.
