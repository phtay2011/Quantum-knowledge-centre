XML Confirmation and EFT formatting enhancements

We have further enhanced the XML Framework used for confirmations and EFTs in the following areas.

Data Template file (EFT)

In the EFT Data Template file, a new paymentDetails data section is now available.

In theory, it replaces deliveryInstructions and adhocDeliveryInstructions where only one or the other section is populated depending on whether the counterparty delivery instructions have been modified using Adhoc.

paymentDetails is populated with Adhoc details should they be in play, otherwise with deliveryInstructions.

Character Formatting Attributes

Number formatting if not using ‘NUMBER_SWIFT’ has been enhanced by adding the following number-formatting variables.
For example, these three bullets show formatting the value -10,000,245.4580

  • Optional to show negative indicator for absolute values
    FormatType NUMBER_X,,none,4  Result = 10999245.4580
  • Support for ‘none’ as a thousand separator (previously this had to be either comma or decimal)
    FormatType NUMBER_X,,none,0  Result = 10999245
  • Support for specific decimal separator; either comma or decimal
    FormatType NUMBER_,,none,2,comma  Result = 10999245,46
