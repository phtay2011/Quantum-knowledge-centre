Deal Allocation Defaulting Rule page

Use this page to create and maintain named deal allocation defaulting rule (template) records for optional use with Money Market, Security, Currency Swap, or Interest Rate Swap deals in Quantum.

Deal Allocation enables you to link a master deal (the Source deal) to one or more additional deals of a similar type. These deals are created by the system when you commit the Deal Allocation action (on the Dealing page), after selecting a defaulting rule Template and changing the defaults that template supplies, as required.

To list records - Click to open the browse window. (Click the Configure link to change browse list properties.)

To add a new rule record:

  1. Click New.
  2. Enter a unique Template Name and a text Description.
  3. Select the Transaction Type - Currency Swaps, Interest Rate Swaps, or Money Market (which also covers Securities).
  4. Under Settings:
    • set Allocation not equal to Source, Reversal not equal to Source, and Allocation not equal to Reversal to None, Warn, or Error in each case; and
    • optionally select Copy Analysis Codes, Copy Strategy Codes, and/or Copy Deal Comments to copy the Source deal's relevant codes and/or comments to other deals within the set.
  5. Edit and/or add allocation entries as required (see below).
  6. Click Save.

To change a record's details - select it in the browse list, optionally change key values if required; add, edit, and/or delete Allocation entries (grid rows) as follows, and click Save.

NOTE: If a Rule template is changed after a deal allocation set has been created, the changes will affect only any new deals created for that set (not the existing deals).

  • To add a new allocation entry - in the grid menu, select New; in the dialog box select a Role (ALLOCATION or REVERSAL), select DEFAULT, SOURCE, or a specific Counterparty, Entity, Instrument, and (optional) Relationship, enter optional Comment text, a Long Percentage, and either one (Money Market/Security) or two (Swaps, Pay and Receive) Margin values; then click OK.
  • To edit an allocation entry - click a data value in the grid row; in the Edit window, make the required changes; then click OK.
  • To delete an allocation entry - select the grid row; in the grid menu select Delete; then confirm.

To delete a rule record in full (not recommended; Deactivate instead) - Select it in the browse list; in the page menu, select Delete; then confirm.

To deactivate a record - Click , select the record, and in the page menu select DeActivate. You can later reactivate it in the same way.
