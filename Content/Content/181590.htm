Scheduler Setup for Quantum Six

Company Setup

In Scheduler Administration, create a Company of the same name as the Quantum Six Site that users log on to, as in the example below.

About the Default Company

For existing installations where Jobs have already been assigned against the Default Company, the Default Company Name can be upgraded to match the Quantum Six Site name by running the following on the Scheduler database (replacing <SITE SECTION> with the Quantum Six Site name):

update schcompany set Name = '<SITE SECTION>' where name = '(Default)'

NOTE:

  • If the Default Company was used for more than one Site, then jobs for the secondary Site Sections should be removed.
  • Within a job, any tasks for a different Quantum Six Site will be identified with a warning message.
  • Once a company has been defined for the Site, the Default Company should be deleted.

About Time Zones

Scheduler has its own concept of time zones, separate to Quantum Six.

The Time Zone defined for the Company and each Job should match the time zone of the machine hosting Scheduler. Each reported time is transferred directly to the Quantum Six client without further adjustment.

Category, Job, and Task Setup

Within Scheduler, each Job is assigned against a Category.

In Scheduler Administration, add in a Job Category that is associated with the Company you have created, as in the example below. The Category Name must not exceed eight characters, as it represents the Quantum Six Module (program) name.

Create one or more Jobs for the Company and Category you have created and add as many Tasks as required to each Job, as in the example below.

TIP: The Time Zone defined for each Job should match the time zone of the machine hosting Scheduler. See About Time Zones above.

For Analytics, Tasks can be set up for:

  • Replication Manager
  • Reconcile
  • Deal Importer / Deal Remover
  • Reporting
  • Market Value Importations (MVI)

For Quantum, Tasks can be set up for:

  • Processing
  • Reporting
  • Static Data
