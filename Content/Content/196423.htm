Get Base Details (Process Fees)

Use the Process Fees page's Get Base Details function to create the base data for calculating fees - that is, the data that will populate the Fee Deal Position History database table. You must do this before you can:

  • test any Fee Base Templates and Formulas against the latest base data for fees, and
  • generate and process fee cash flows and accounting entries based on that data and for the date you specify.

RECOMMENDATION: Run this function daily, as the deal value information in the Fee Position History table is maintained on a daily basis.

Procedure Summary

  1. On the Process Fees page, click Get Base Details.

    The Generate Base Deals dialog box opens.

  2. Select the Position Report and Accruals Report. These are the respective Query configurations set up for Fees processing - see Prerequisites for Processing Fees.
  3. Enter the As At Date to be used for the Position query and Accrual query runs. The default is the current date.
  4. Enter the Start Date of the range over which the Position History is to be generated. The default is the next immediate date for which the Position History table requires updating.

    CAUTION: Normally, you should ensure that no dates are skipped between this date and the End Date from the last Get Base Details run (if any).

  5. Enter the End Date of the range over which the Position History is to be generated. The default is your specified Start Date or the current date, whichever is the later.
  6. Click OK.

Results

Your specified Position query is run, generating Position report output that the system uses to create new records in the Position History table and to update any existing records that have changed since the previous Position History run (if any).

Your specified Accrual query is run, generating accrual report output that the system uses to update interest values in existing records in the Position History table (any interest value calculated for a deal that does not have an existing record in the table is ignored).

The updated Position History table records are displayed as rows in the Base Deals grid. The records contain the following extracted Quantum deal data for calculating the applicable fees as at the date you specify:

  • deal static data (Counterparty, Instrument, Currency, Entity, Relationship, and so on);
  • deal values (for Position fees - Face, Settlement, Capital, Security, and Interest value);
  • deal events (for Transaction fees - Deal Date, Settlement Date, and Maturity Date); and/or
  • Facility/Dealing Line values (for Facility fees - Utilized, Unutilized, and Notional values).

The Fee deal records active for your specified As At Date are displayed on the Fee Deals tab.
