Areas, Workflow Action page

This topic covers the different grids seen in the Workflow Action module and describes how to use them.

Name - user-defined name

Folder - there are a folders already set up for you to place the actions in: Create, Delete, Merge, Upgrade, etc. Create new folders and sub folders by entering NewFolderName, SubFoldername with the comma in between.

Success Message - When the system completes a workflow action, it returns the Success Message text. Any errors come from the embedded actions. When any of the actions fail, the system rolls back the whole workflow action.

For some examples of using workflow actions, read Workflow Action: Examples.

Inputs / Outputs (Workflow Action)

The Inputs grid defines what can come into the workflow action. These inputs form the "context" of what can go into the embedded actions.

Set a Name and a Type (for example, text, date, Yes/No, amount, or number) and choose the default value.

The Outputs grid defines what format the workflow action ultimately returns. Outputs are also part of the "context" and you can then pass them into subsequent actions.

Set a Name and a Type (for example, text, date, Yes/No, amount, or number). Select Recommended or enter a default value.

To add an input or output

In the grid menu, select New to add a new input / output, Edit to change an existing one, Delete to remove one.

Aggregated Inputs

Aggregated inputs can be defined on the data set that goes into a workflow action. For example, the total number of row, or the sum of all positive amounts.

To add an aggregated input

  1. In the grid menu, select New to add a new aggregated input, Edit to change an existing one, Delete to remove one.

  2. Set a Name and an Aggregation (for example, count, sum, max, min, mean, or distinct count)

    1. Aggregation Type Type of lookup fields
      Sum Number, Amount
      Mean Amount
      Min Number, Amount, Date, Date Time, Text, Yes/No
      Max Number, Amount, Date, Date Time, Text, Yes/No
      Count Distinct Number, Amount, Date, Date Time, Text, Yes/No

  3. Add a matching criteria using Expression builder.

Calculated Outputs (Workflow Action)

This grid defines outputs that are based on expressions. The expressions include reference to the sources, inputs, and outputs. The system evaluates the expressions when the workflow action occurs.

To add a calculated output

  1. In the grid menu, select New to add a new calculated output, Edit to change an existing one, Delete to remove one.
  2. Set a Name and a Type (for example, text, date, Yes/No, amount, or number).
  3. Add an expression using Expression Builder.

Validations (Workflow Action)

The Validations grid applies to inputs. The system must check and pass all validations before it triggers the first action. For example, set up a validation to test whether a current user is in a particular user group before deleting.

  • Condition - Click the Add Condition link to open Expression Builder so you can enter any validation conditions.
  • Message - The message text to be displayed when the validation fails.
  • Type - Block makes the system skip that particular record and process the other remaining rows. Terminate makes the system terminate all actions and process none of the rows from the workflow report.

To add a validation

  1. In the grid menu, select New to add a new condition, Edit to change an existing one, or Delete to remove one.
  2. Add validations in the order you want them to occur.

Actions (Workflow Action)

The Actions grid lists the actions to be triggered, in order. A prerequisite condition must be "true" for the system to execute the action. The action applies to all lines in a data set, or just the first line, or just the last line.

To add a (general) action

  1. Open Configuration > [type of workflow page].
  2. In the menu select New and theNew Actions dialog box appears.
  3. In Action Name click the lookup. The top level folders are organized as follows:
    • Create - the action creates a new deal of your selected type without opening a dealing page, or a Commodity Swap Price Fix.
    • Delete - make a button or menu item that will delete a deal.
    • Merge - merge sets of data.
    • Open > Analytics > Deal Analyse - to make the action open an Analytics Deal Analyse page; or Open > Dealing - a Quantum deal or request type to make the action open a new page for your selected deal or request type.
    • Transfer - choose a data transfer location (DTL). Populated with locations that have a template associated with them.
    • Update - the action updates a deal.
      • Select Update > Bulk Update to use the calculated column as a Column (data source).
      • Select Update > Save Holiday to use the report to add a new day to your holidays table. The inputs include: Currency, Location, Holiday Name, Observable, Start Date, Expiry Date, Fixed Date, Based on, Day, Day Name, Month, Day and Month.
      • Select Update > Update [custom table name] to use the report to update data in a row in a custom table. You can also use actions to update and delete rows in a custom table.
  4. Define the Inputs.
  5. Set up any Prerequisites.
  6. Save.

Sources (Workflow)

The Sources grid specifies the source data used for input to the action or actions to be performed when the event occurs (subject to any specified Settings). The grid has the following columns:

  • Source - the data source used for the page. Includes internal data sources such as Quantum and Analytics deals, reports, and static data; deal and settlement approval rules (in the Reporting folder); and database views (in the Database folder, with the prefix QVW_DATASOURCE_%). Also includes Workflow Templates.
  • Name - a name for the source. Defaults to the Source value. Used to:
    • shorten the source name, or
    • include the same data source twice so you can compare the source under different conditions.
  • Columns - selected columns from the data source to be available in your target's result set, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level. Formats available within the selected column include:
    • Amount - the length varies with the Currency selected (default 2 dp),
    • FX Rate - 8 dp,
    • Interest Rate - 12 dp,
    • Number - an integer,
    • Deal Number - a link to the deal,
    • Date -your date format, and
    • Text.
  • Settings - Settings to apply to the data, including mathematical operators and expressions for calculated columns. Use these settings to reduce the target's result set. Either fixed values or linked to query filters on the target page.

    • TIP: You will get an error message if you have not included a required (key) data item in the Settings.

  • Joined To - Available if the source is joined with another data source.

To add a source

Select New to open the New Source dialog box.
