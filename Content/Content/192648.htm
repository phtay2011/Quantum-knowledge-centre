Commodity Swap Deals

See also: Commodities in Quantum Six

Use the Commodity Swap Deal page to create and maintain Commodity Swap contracts.

Commodity Swap deals are implemented similarly to other types of Swap deal in Quantum, with the added ability to price them using Analytics market valuations. Quantum Six enables you to create and maintain two types of Commodity Swap:

  • Fixed for Floating - guarantees the buyer a fixed price over the trading period, while the seller is hedged against a decrease in the commodity's cost over the same period; and
  • Floating for Floating - involves the simultaneous purchase of one commodity and the sale of the same or a similar commodity, the price being established using a spread (the difference between the spot price and the future price). Floating for Floating swaps are usually used to limit risk associated with two indexes fluctuating in value.

A Commodity Swap deal:

  • is a financial deal that moves the risk of price fluctuation from your organization (Entity) to a Counterparty;
  • does not involve the buying or selling of the actual commodity (the physical deal*);
  • locks in a fixed price at an earlier date, eliminating price uncertainty between then and the date the commodity is required; and
  • will, on the future date, have a Settlement Amount of the difference between the actual price then and the locked-in fixed price, multiplied by the agreed quantity.

NOTE: The Commodity Swap deal does not track the physical deal (the actual delivery of the commodity). That would typically be entered independently, as an Actual Cash Flow deal.

For more details, see Commodity dealing workflow - Example.

To fix the price

  • To manually fix the price for the current deal on the Commodity Swap Deal page: select Price Fix from the page menu.

    (You can also navigate from the Settlements page to a selected deal page's Price Fix window.)

  • To fix the price in bulk for selected deals using either system-calculated prices, or a value you specify: use the Commodity Price Fix page.

NOTE:

  • For a Fixed for Floating deal, the Price Fix function always applies to the next unpriced period in sequence. If there are no unpriced periods, the function is not available.
  • For a Floating for Floating deal, the Price Fix function applies to the single trade period (if currently unpriced). Each side (Near or Far) is separately priced.

To add a single period for the current deal

NOTE:The deal must be open (the Maturity Date has not passed).

  1. In the Deal Details tab's Periods area, click New.
  2. Enter the Start Date, End Date, Quantity, Fix Date, and Payment Date for the period.

To create one or more periods for the current Fixed for Floating deal

  1. In the Deal Details tab's Periods area:
    • Select New and add as many periods as you need, entering the Start Date, End Date, Quantity, Fix Date, and Payment Date for each period, or...
    • Select Generate Strip, entering the number of periods across the deal, the quantity of the commodity, the frequency of strips (Monthly, Quarterly, Annually), the signal to start cycling (Calendar Month, Day of Month, Offset), and other relevant fields.

For a Fixed for Floating deal - you can create one period at a time, or a regular group of periods (a Strip).

For a Floating for Floating deal - you can create only the one trading period.

To replace all data for periods*

  1. Open the deal from the Dealing menu.
  2. Open the Deal Details tab.
  3. In the Periods grid menu, select Bulk > Replace. The Bulk Replace dialog box opens with all data highlighted.

  4. Use standard keyboard actions to replace all or some data:
    • Click in the dialog box – to stop highlighting
    • Ctrl + C - to copy the highlighted data
    • Ctrl + V – to paste new data / replace highlighted data
  5. Use the Quantum format for dates and numbers:
    • 1,000.00 (comma, point)
    • 06/14/2019 (MM/dd/yyyy)
  6. Select Replace to save the new data.
  7. View the updated data. The system sorts the periods chronologically by start date or payment date (fixed exposure).

* This applies to Fixed for Floating deals only.

To generate strips using Day of Month

The Day of Month rule enables you to set two dates, within the specified Frequency, at which the period will start and end every time.

  1. Open Dealing > Commodity Swap.
  2. Select New to create a new deal.
  3. In the Deal Details tab, find the Periods menu.
  4. Select Generate Strip and for Start Date Cycling select Day of Month.
  5. Complete any other fields, as required, and click Generate.

To generate strips using a supplied Offset

Offset rules enable you to easily create multiple strips or "periods" based around common patterns.

  1. Open Dealing > Commodity Swap.
  2. Select New to create a new deal.
  3. In the Deal Details tab, find the Periods menu.
  4. Select Generate Strip and for Start Date Cycling select Offset.
  5. Complete any other fields, as required, and click Generate.

For example, choosing the THIRD WEDNESDAY rule sets the start date to the Monday before the third Wednesday of the month, and the end date to the Monday before the third Wednesday of the final / end month. The pattern repeats at a length of time after the start date, as specified in Frequency.
