How to Report Trades

This page describes how to use Regulatory Reporting workflow for reporting trades.

Reporting a trade

  1. Go to Processing > Regulatory Reporting > Report Deal Actions and run the REGREP Workflow report.

  2. In the Reporting Jurisdiction filter select the jurisdiction to which you want to report and click Run Query.

    NOTE: Deal Number is an optional filter. If you leave it empty it doesn't perform any filtering on deal numbers and returns all the eligible trades.

The report has two sources, Reportable Deal Actions and Regulatory Reporting Rule which work together in bringing the trades correctly. The Reportable Deal Actions source internally works with Regulatory Reporting Status table which registers all the reporting activities on the trades, Jurisdiction wise. The report has one workflow action Process Deal Actions configured which takes care of generating reporting files for the eligible trades.

Regulatory reporting workflow system can populate IRS/CCS amortizations in the Reportable Deal Actions base source. If the templates are updated for handling this category of reporting, this data can be used for reporting Amortized MX files to the EMIR and FCA repositories.

  1. Select the deals that you want to process and click Process Deal Actions. This generates the outbound trade files in the folders defined in the respective Data Transfer Locations for the template actions.

  2. When the repository receives the response file, they are placed under the appropriate folders defined in the Parser DTLs:

    • For EMIR_DTCC: WorkflowIn\Import\EMIR DTCC Response

    • For EMIR_REGIS: WorkflowIn\Import\EMIR REGIS Response

    • For FCA_DTCC: WorkflowIn\Import\FCA DTCC Response

    • For FCA_REGIS: WorkflowIn\Import\FCA REGIS Response

    • For EMIR_REGIS_SFT_RA: WorkflowIn\Import\EMIR REGIS SFT Response
