File Name Mask (Paper Confirmation Format Rule)

Use this area to specify, for print-formatted Confirmations being saved to file, the structure and content of the file names that will be used when this rule applies (as defined in the Matching Criteria fields).

TIP: Confirmations are written to file when the Print option is set to File in the applicable Confirmation Rule record entry.

  • The file name suffix and file location are determined by the File Type and File Path settings in that rule entry, respectively.
  • Valid file types are DOC, DOCX, PDF, or RTF.

Default

Select this option to use the standard file naming based on Transaction Type, Deal Number, date, and time. For example, for a Confirmation of FX deal number 789 created on December 23, 2016 at 14:55:48, in PDF format, the file name would be:

        FX78920161223145548.pdf

  • Select User Defined to specify your own

User Defined

Select this option to specify your own file naming convention, using the text box, using the associated text box.

Enter text and/or one or more of the available placeholders:

  • [Transaction Type]
  • [Deal No]
  • [Date]
  • [Time]

NOTE: These are the only valid placeholders for File Name Mask.

CAUTION: If you do not include [Deal No], [Date], and/or [Time], you could get duplicated file names, with earlier files overwritten.

For example, for FX Deals, you might specify the following File Name Mask text:

        Deal[Deal No]_DT[Date]_T[Time]

For a print-image file confirmation of FX deal number 789 created on December 23, 2016 at 14:55:48, in RTF format, the file name would then be:

        Deal789_DT20161223_T145548.rtf
