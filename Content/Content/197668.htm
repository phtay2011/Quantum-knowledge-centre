Using Overnight Index Periodic - Dealing

OVERNIGHT INDEX PERIODIC is a formula timing related to some swap instruments that are Overnight Indexed Swaps (OIS), for example Interest Rate Swaps (IRS) and Cross Currency Swaps (CCS). The same functionality appears in modules such as Money Market instruments and Securities instruments when using Formula Timing = Yield Periodic along with selecting the Overnight Index check box. The deals based on these instruments also have many of the fields associated with OIS.

This section describes how to create and use a Quantum IRS or CCS instrument, deal, and accounting rule with the overnight index functionality, but also applies, in general, to other types of deal.

To set up overnight index functionality

  1. Open Instrument > Interest Rate Swap and use the Pay and/or Receive Details tabs to set up an IRS or CCS instrument that uses the new field, Payment Date Offset. This is the number of business days that the existing action date will move forward by.
  2. Select Formula Timing = OVERNIGHT INDEX PERIODIC.

  3. Enter a Pay Frequency. Notice that the system populates the Pay Frequency and Rateset Frequency with the same value.
  4. (Optionally) Select the Rate Type.
  5. (Optionally) Select a Lag. The lag shifts the rate observed.
  6. In the Rounding area in the Pay Pricing tab, set the rounding for the number of decimal places for rates and margins.

  7. Open the Deal Details tab in a Dealing > Interest Rate Swap page, view the Pay or Receive areas, and select Formula Timing = OVERNIGHT INDEX PERIODIC. This is always Floating. For the Fixed leg, you can use Formula Timing = Yield Periodic Fixed, which also has Payment Date Offset available.
  8. Use Payment Date Offset to set the number of business days that the existing action date needs to move forward by.

  9. View the deal map when finished. The representation in the deal map is very similar to when Formula Timing = Yield Periodic. The system allows you to set bulk (batch) OIS rates based on the Action Date at the end of the calculation period.
  10. Use the page menu option Unwind to unwind the deal. The Unwind dialog box now shows the compounded rate up to the unwind date for the OIS legs. You can partially or fully unwind.
  11. Use the Instrument Details tab in the IRS deal to add the new field: Rate Type.
  12. Set up a rule in the Static > Accounting Rule page. Open the Interest Rate Swaps tab, and select New to open the New Accounting Rules dialog box. Select the Instrument Name of an OIS instrument and complete the set up.

    Quantum uses Analytics to calculate the overnight index periodic rates, so the results have a new column "Calculated Rates". See more: Rate Set Deals (in bulk)

  13. (Optionally) Export an OIS deal by generating a QXT file. The ability to import a QXT file is current functionality, which we extended in v6.8.2 to give the ability to import multiple deals simultaneously via QXT.

    You can also delete one or many deals using a QXT file.

  14. (Optionally) Accrue the Interest Rate Swap deal using the OVERNIGHT INDEX PERIODIC formula timing. The fields are similar to those used in the deal and instrument.
  15. (Optionally) Open Reporting > Deal Revaluation to revalue an IRS deal that uses the OVERNIGHT INDEX PERIODIC formula timing. Also applies to a Cash Flow Hedge Relationship deal with an Overnight Index Swap deal linked to it.
    • If the rate is not set, then the system uses the compounded rate for the current interest period. The system calls Analytics for this compounded rate.
    • The book value used in the revaluation is accrued interest within the current interest period.
    • You can see the book view and the market value of the deal in this report. If your site doesn't use Analytics, Quantum can still create satisfactory data.

      See more: Payment Date Offset, Interest Rate Swap

NOTE: The Payment Date Offset is a financial field and so is locked. To change the field after it has been saved, use Reset. Using Reset:

  • does not affect the effective date, and
  • represents the number of business days, based on currency and location setting, that the existing action date needs to move forward by. The action date is the date returned after the system applies the payment date business day convention.

NOTE: The system adjusts the Action Date of the deal map items - interest, repay notional, and repay principal - forward, based on the offset calculation. The action date can be after the deal's Maturity Date.

How does Overnight Index affect other tasks?

When both Payment Date Offset and Suspension are non-zero in a deal with Formula Timing = YIELD PERIODIC and with Overnight Index selected, expect the following behavior:

Bulk Rate-set

  • When the system calls Analytics for the average rate, the system only passes the Suspension value to Analytics for the final rate-set. All other rate-sets for that deal have a suspension value of 0.

Unwind

  • The system takes account of whether the unwind effective date is before or after the penultimate interest effective date when calling Analytics for the average rate. It does not call for a rate when the effective date is on the interest effective date (current functionality).

Revaluation

  • The system takes account of whether the position date is before or after the penultimate interest effective date when calling Analytics for the average rate for book value. It does not call for a rate when the effective date is on the interest effective date (current functionality).

MM deal

  • Payment Date Offset applies to the INTEREST Action Date and moves the date forward for all INTEREST rates, except the final one.
  • Suspension - applies to the RATE SET Action Date and moves the date backwards only for the final one.

When using only a single period / payment on the deal, then the system applies both Payment Date Offset and Suspension to the single period / payment.

Arithmetic average

For Securities deals and for Money Market deals when using FRN-CGL PERIODIC with Overnight Index check box enabled, when choosing Calculation Method = ARITHMETIC AVERAGE (Calendar), Quantum passes the Coupon Margin to Analytics, and returns the "all in" rate. Then finishes the calculation with Coupon = Average Rate + Coupon Margin.

History

From 2019.4 (v6.8.2) and 2020.1 we introduced a new field, Payment Date Offset, available in IRS and CCS instruments, deals, and accounting rules.

From 2020.2, this field was available in other non-swap instruments, such as Money Market instruments. The field becomes available when you enable the Overnight Index checkbox. We will steadily roll out the functionality to other deals. New fields also include Rate Type and Lag.

In 2020.3, we introduced Suspension and rolled out similar functionality to Securities instruments.

In 2020.4, we introduced a new Calculation Method = ARITHMETIC AVERAGE (Calendar), available in Securities deals and also available in Money Market deals when using FRN-CGL PERIODIC with Overnight Index check box enabled. We've relabeled the option ARITHMETIC AVERAGE to be ARITHMETIC AVERAGE (Business). Existing deals will update automatically to "Business".
