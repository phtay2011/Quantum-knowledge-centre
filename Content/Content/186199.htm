Filling and Resetting Money Market Requests

The following procedures cover Money Market Request Fill-related actions available on the Money Market Request page.

These actions are enabled for a saved Request when:

  • the logged in user has Fill rights to the Money Market Request page in Module Security, and
  • the Request does not require approval, or
  • the Request is fully approved, or
  • the request is NOT fully approved but the Allow Processing option is set to YES in the applicable Deal Approval Rule.

Request Fill and Mark as Filled actions are not currently subject to Approvals, although you can set up Approval Rules for the actual deals that will be created by the Fill action. Approvals do not apply to the FX deal linked by the Mark as Filled action, as no financial change has been made.See Requests and Approvals.

To create a new Money Market Deal to fill a Request

  1. Select the Request you want to fill.
  2. In the page menu, select Fill.
  3. In the New Money Market Deal dialog box that appears, enter additional data values as required for the actual deal - for example, Counterparty, Entity, and Dealer must be set.
  4. Click Save.

    If validation is successful:

    • the new actual Money Market deal will be saved in the system (it is badged Fills request mmm on the Deal page, where mmm is the Request deal number),
    • the Deal dialog box is closed,
    • on the Request page, the Fill and Mark as Filled actions are disabled in the menu,
    • the Request is badged Filled by nnn, where nnn is the new actual deal number, and
    • the date/time of the Fill action is saved with the Request, and may be included in the Request browse view as a column or used as a filter.

To fill a Request with an existing Money Market Deal

  1. Select the Request you want to fill.
  2. In the page menu, select Mark as Filled.

    The Mark Request as Filled dialog box is displayed, with Mark as Filled and Cancel buttons and an Available Deals grid in which you can select an existing Money Market deal from those deals that:

    • have not been not deleted,
    • are not yet linked to a Request,
    • have a Maturity/Value Date no earlier than the current date, and
    • are consistent with your Resource Security permissions.
  3. In the dialog box's Available Deals grid, select the existing Money Market deal you want to link, then click Mark as Filled.

    When the action is completed:

    • the linked actual Money Market deal is badged Fills request mmm on the Deal page, where mmm is the Request deal number),
    • the dialog box is closed,
    • on the Request page, the Fill and Mark as Filled actions are disabled in the menu,
    • the Request is badged Filled by nnn, where nnn is the new actual deal number, and
    • the date/time of the Fill action is saved with the Request, and may be included in the Request browse view as a column or used as a filter.

To reset a Filled Money Market Request

TIPS:

  • The Reset action is enabled for a fully approved and filled Request when you have Change rights to the Money Market Request page in Module Security.
  • Use a Reset only when you need to change an existing filled Request to correct errors or to add additional details.
  • You can also delete a Request filled in error, removing it from any further processing or reporting - see under Maintaining FX Requests.
  1. Select the request you want to reset.
  2. In the page menu, select Reset.

    The Reset Request as Filled dialog box is displayed, with OK and Cancel buttons and the following message:

    The request has been filled. Resetting the request will
    disconnect the deal from the request.

    Do you want to continue?

  3. Click OK:

    When the action is completed:

    • details of the fill (deal number and fill date) are removed from the Request and the previously linked deal, and
    • changes may be made to the Request details and saved.

      NOTE: Changes will be subject to any applicable Approval Rule, but the Reset action itself will NOT trigger a CHANGE approval where a rule exists, as the clearing of the link details alone is not a financial change.
