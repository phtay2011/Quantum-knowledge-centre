Areas, Workflow Event Page

This topic covers the different grids seen in the Workflow Event page and describes how to use them.

Actions Grid (Workflow Event)

The Actions grid shows the list of actions you have configured the Event page to perform when the specified system event (as selected in the Name field) occurs.

To add a (general) action

  1. Open Configuration > [type of workflow page].
  2. In the menu select New and theNew Actions dialog box appears.
  3. In Action Name click the lookup. The top level folders are organized as follows:
    • Create - the action creates a new deal of your selected type without opening a dealing page, or a Commodity Swap Price Fix.
    • Delete - make a button or menu item that will delete a deal.
    • Merge - merge sets of data.
    • Open > Analytics > Deal Analyse - to make the action open an Analytics Deal Analyse page; or Open > Dealing - a Quantum deal or request type to make the action open a new page for your selected deal or request type.
    • Transfer - choose a data transfer location (DTL). Populated with locations that have a template associated with them.
    • Update - the action updates a deal.
      • Select Update > Bulk Update to use the calculated column as a Column (data source).
      • Select Update > Save Holiday to use the report to add a new day to your holidays table. The inputs include: Currency, Location, Holiday Name, Observable, Start Date, Expiry Date, Fixed Date, Based on, Day, Day Name, Month, Day and Month.
      • Select Update > Update [custom table name] to use the report to update data in a row in a custom table. You can also use actions to update and delete rows in a custom table.
  4. Define the Inputs.
  5. Set up any Prerequisites.
  6. Save.

Sources Grid (Workflow)

The Sources grid specifies the source data used for input to the action or actions to be performed when the event occurs (subject to any specified Settings). The grid has the following columns:

  • Source - the data source used for the page. Includes internal data sources such as Quantum and Analytics deals, reports, and static data; deal and settlement approval rules (in the Reporting folder); and database views (in the Database folder, with the prefix QVW_DATASOURCE_%). Also includes Workflow Templates.
  • Name - a name for the source. Defaults to the Source value. Used to:
    • shorten the source name, or
    • include the same data source twice so you can compare the source under different conditions.
  • Columns - selected columns from the data source to be available in your target's result set, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level. Formats available within the selected column include:
    • Amount - the length varies with the Currency selected (default 2 dp),
    • FX Rate - 8 dp,
    • Interest Rate - 12 dp,
    • Number - an integer,
    • Deal Number - a link to the deal,
    • Date -your date format, and
    • Text.
  • Settings - Settings to apply to the data, including mathematical operators and expressions for calculated columns. Use these settings to reduce the target's result set. Either fixed values or linked to query filters on the target page.

    • TIP: You will get an error message if you have not included a required (key) data item in the Settings.

  • Joined To - Available if the source is joined with another data source.

New Source dialog box

Use the New Source dialog box to add, aggregate, or join sources, and to edit columns and settings.
