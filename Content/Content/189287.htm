MM & Sec Man Rules

Use the Money Market & Securities tab to define the management reporting revaluation rules and methods Money Market and Securities deals.

Matching Criteria (Money Market and Securities Management Rule)
  • Instrument - Defines the instruments to which these rules apply.

    The rules are established by row in the grid, and any number of instruments of this type may be recorded to define the applicable and appropriate treatment.

    TIP: Create a rule for ALL instrument types, and then a rule for each instrument type that requires different treatment.

  • Entity - Defines the entities to which these rules apply.

    TIP: Create a rule for ALL entities for the specified Instrument type, and then a rule for each entity or entity group within the entity structure that requires different treatment.

Settings (Money Market & Securities Management Rule)
  • Book Value - The Book Value that will be provided in position processing. In order to generate a revaluation gain or loss, it is necessary to determine the market value. Because Quantum can determine a number of different book values to which the market value may be compared, it is necessary to specify the basis of the Book Value.

    NOTE: Adhoc amounts/interest are always excluded from the Book Value when running the Accounting > Position page or the Reporting > Deal Revaluation in using management rules.

    • Adjusted - The book value is a theoretical value, which is calculated using the deal value and rate, with the settlement date as the as at date.
    • Posted - The actual value in the General Ledger that is represented by the entries posted to the General Ledger at the as at date. It represents the net of the principal, interest/coupon, ad-hoc interest and capital premium/discount entries.
    • Purchase - The settlement amount of the deal as recorded on the settlement date.
  • Accrual Method - Used to define the basis on which amounts that are to be accrued are calculated.
    • Yield - The amount to be accrued is calculated by repricing the instrument on each accrual date, using the dealt discount or yield rate.

      The amount to accrue is the difference between the last repriced amount (the last accrual date amount) and the most recent repriced amount.

      This results in a different daily amount for each period. By repricing the instrument, the accrual is simply the result of the reduced number of days to maturity date.

    • Straight Line - If the deal's Accrual Method is Amortized Cost, the system will calculate the capital accrual amount (and Book Value) using that method (regardless of this setting).

      Otherwise, the total amount to be accrued is divided by the number of days in the total period, and multiplied by the number of days in the current accrual period - that is, equal amounts are accrued daily.

  • Interest Rate - For instrument type/s defined in the Instrument field, the interest rate defines the name of the Interest Rate Table used to source the revaluation rates/prices for the revaluation of a deal meeting the selection parameters.
    The rate type must be known in conjunction with the interest rate to locate an actual rate table and the rates to NPV of any accrual amounts referred to above.
    The rate table to source the FX rates to revalue the currency component is defined in the CCY Translation Rate Type field in the Management Control header. (FX rates do not have a rate type).
    If you are using a User Input Zero Curve or User Input Discount Factor, select the name used for the curve on the Zero Definition form.
    If you select a User Input Zero Curve or User Input Discount Factor for an instrument type that is not compatible with that type of curve, an error message is displayed when you perform a revaluation.
  • Rate Type - Defines the type of rate table used to source the revaluation yield rates for the revaluation of a deal meeting the selection parameters for instrument type/s defined in the instrument column.
    The rate type must be used in conjunction with the interest rate to locate a rate table.
    If you are using a User Input Zero Curve or User Input Discount Factor, select the rate type used on the Yields Rate or Zero Definition form.
  • Revaluation Method - Determines the method by which deals are to be revalued for Money Market instrument types defined in the Instrumentfield. Select from the following:
    • Bills
    • No
    • Projected
    • Bills Net
    • Projected Net
    • Imported
  • Security Revals - Determines the method by which deals are to be revalued for Money Market Security instrument types defined in the Instrumentfield. Select from the following:
    • Bills
    • Bills Net
    • FRN-Coupon (FRN Securities only)
    • Imported*
    • Market (Securities only)
    • Market Cum (Securities only)
    • No
    • Projected
    • Projected Net
  • Embedded Revals - Determines the method by which deals are to be revalued for Money Market instruments with embedded options (Callable Bonds) defined in the Instrument column. Select from:
    • IMPORTED
    • MARKET
    • MARKET CUM
    • NO
    • BLACK-DERMAN-TOY
    • BLACK-KARASINSKI
    • HO-LEE
    • HULL WHITE
  • Margin Type - To use a margin over the base rate to revalue deals meeting the selection criteria, specify the Margin Type table that holds the applicable margin rates.
