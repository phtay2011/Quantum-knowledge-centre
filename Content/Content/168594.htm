Crystal Mapping

System Administrators or Database Administrators can use the Crystal Mapping page to attach standard or user-created custom Crystal Reports to a Quantum application module.

TIP: The module name refers to the underlying program an application page interfaces to.

In some cases, more than one report can be mapped to a module. For example:

  • The CONFFMTP module (Paper Confirmation Format Rules) has different reports for each deal type and action.
  • The BNKACENQ module (Bank Account Statements reporting) can produce Bank Account Balances, Stock Account Balances, statements, and so on.

Linking a Crystal Report

Whether you wish to attach a standard Quantum report or a user-created Crystal Report format to a Quantum application (module), the process is the same. Open the Crystal Mapping page and fill in the details as explained below.

TIP: If you do not have the required permissions, contact your system or database administrator.

  1. Click New.
  2. Enter a unique and useful View Name (this name will show in lookup lists when selecting).
  3. Select the Module to link to from the drop-down list, or type the name in UPPER case.
  4. Select a Crystal Report name from the drop-down list of those in the configured ReportPath directory for your site. Report formats must be in this directory to work within Quantum.

    NOTE: If Resource Security is enabled, this name should be unique when used for Static Data reports.

    Examples:

    Application

    Module

    Crystal Report

    Paper Confirmation Format Rules

    CONFFMTP

    Select the appropriate report for the deal type and action - for example CO Deal, FRA Delete, FX RateFix, CMS Deal, and so on.

    Bank Account Reporting

    BNKACENQ

    BNKACRPT for Bank Account Balance reports

    STKACRPT for Stock Account Balance reports

    STMTRUN Bank Account Statement reports

    STMTRNST Stock Account Statement reports

    Counterparty

    CPARTY

    CPARTY for approved counterparties

    CPARTYU for unapproved counterparties

    Hedge Accounting

    HRWBENCH

    HRExposures for Exposures available to hedge

    HRDetails for Hedge Relationship details

    HRIneffectiveness for HR ineffective deals

    Users and User Group

    CTUSR

    CTGRPS

    Users and UserGroups for all user and user group records in the system

    UsersAudit and UserGroupsAudit for all user and user group records in the system including a history of any changes that were made to individual records

    NOTE: The file names for custom Crystal Reports for Users or User Group modules must begin with "user"; for example, UsersActive.rpt.

  5. Where required, select the actual Report Table name where report results are put into more than one table (for example, Position Processing creates both a posnrpt table and a pairrpt table).
  6. Add a text Description for the report mapping record.
  7. Click Save.

The report format has now been linked to the application module.

 

Changes to Crystal Mapping are audited. Use Audit-Query to list details of records added, changed, and deleted.
