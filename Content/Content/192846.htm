Report

The different tabs seen in a Configuration > Report page include:

  • Sources tab
  • Filters tab
  • Calculated Columns tab
  • Actions tab
  • Structures tab

Use the Workflow Report page to create and maintain site-specific reporting pages, which can combine data from multiple Quantum Six data sources, and which can be placed anywhere in the Quantum menu structure. To the end user, these reporting pages will look and behave like any other core Quantum reporting page.

Currently supported sources of data include:

  • other Quantum reports,
  • Analytics reports,
  • Quantum deals,
  • Quantum and Analytics static data ,
  • Prior Day and Intraday bank balances from CashXplorer,
  • Analytics custom tables, and/or
  • structures (data in spreadsheet form).

TIP: Don't confuse the Workflow Reportpage with the Workflow Page page, which can be used only to modify the user interface of core dealing-related pages.

An example of the Workflow Report page is shown below.

Use the Name field to enter a unique module name for the target page (used in Module Security) and the Title field for the page title. The Url Name gives the last part of the page URL.

Use the Menu Area and Menu Group fields to enter the area in the site menu where the target page will appear (Analytics, Processing, Reporting, and so on) and the menu group under which it will appear (also used in Module Security).

The Sources grid will have one or more entry rows, showing information about the data source.

The Filters grid shows data items to be displayed as Filters fields on the target reporting page, and which can also be used by the data source settings to reduce the result set.

The Calculated Columns grid enables you to set up columns based on expressions and which will appear in the results view of the target report..

The Actions grid shows optional additional actions made available in your target report's results view (as quick menu items or toolbar buttons). Defining actions for a report changes the reporting page into a processing page. Permissions apply to actions.

The Structures grid shows optional structures that enable data in spreadsheet form to be included as data sources. The target report end-user can edit such data directly or via a spreadsheet application (as with Bulk Replace and Insert for Deal Map Items).

NOTE: To see a new target page in the site menu, you must refresh the page (press F5) or log out and in again.

Create a report in the EW

Learn how to create a report in the Enterprise Workflow (EW).

Use the report

To add, edit, or delete columns or filters

The following happens to the query, any runs of the query, and the user-defined views for a query when you add, edit, or delete columns (Columns tab), calculated columns, or filters (Filters tab). This applies to an existing workflow report in the View area:

  • Add a column or filter to an existing report, and the system keeps previous user-defined views and queries, but deletes runs.
  • Delete a column or filter from an existing report, and the system removes the column or filter from previous queries, deletes runs, and keeps the queries. The system deletes any user-defined views that used the column or filter.

The system displays a warning message before these actions similar to the following:

“Adding a column to the definition will delete all previous runs on the workflow report.”

To view recommended inputs for an action

When creating workflow reports or pages in the Enterprise Workflow (EW), some of the actions you add have more obvious inputs than others. Quantum shows you the inputs we recommend for a particular action.

For example, in the New Actions dialog box, for the Delete Deal action you are shown the (mandatory) inputs Deal Number and Reason. You must then define the Expression.

You can add inputs, and you can delete non-mandatory inputs.

To alter a custom table

You can update custom tables by using a workflow report or a workflow page to add an action that targets the table. You can also create new rows or delete rows in this way.

For more information about altering rows on custom tables, see Workflow grid (Page) or Actions grid (Report).

For more information about creating new custom tables, see "Quantum_20xx.x_Database_Installation_Instructions.pdf".

Example 1

To find out which rules apply to a deal, ordered by priority

  1. Open the Configuration menu and create a workflow report.
  2. Enter the basic fields.
  3. In the Sources grid, open the New Source dialog box, and use the Source field's browse list to select the Reporting folder and select Deal Approval Rules.
  4. In the Columns area, add columns
  5. In the Settings area, add settings to filter by.
    • If using the deal number as the filter, create a report by selecting Deal Number as a filter.
    • If finding out what rules would apply to this deal for a different amount, select Deal Number and Amount as filters. This will find what rules would apply to the deal for the specified amount.
    • If finding out what rules apply when you do an action, for example, Delete, select Deal Number and Action. This will find what rules would apply to this deal for the specified action.
    • You don't have to include Deal Number as a filter, instead specify other inputs to determine what rules would apply to a hypothetical deal.
  6. In the Filters grid, add a Filter.
  7. Join the filter to a particular Setting.
  8. Open the new report.
  9. Run the query for the report.
  10. The View grid contains all rules that match the query, listed in order of priority from the highest priority rule to the lowest. The priority for a given rule is specified in the Priority column.

NOTE: When specifying a deal/flow number, the View grid displays the rules as they would apply to the deal/flow on the day you run this query, not on the day the deal/flow was initially created.

FIS Scheduler Support

You can set up your workflow reports to run in Scheduler, using the Reporting task provider. For more details, see the Quantum Scheduler Supplement.

View changes to the report

You can review / audit any changes made to a report configured within the Enterprise Workflow (EW):

  • When someone adds a new report, the system logs an Add action, visible in the Audit query.
  • When someone deletes an existing report, the system logs a Delete action, visible in the Audit query.
  • When someone amends a report, the system logs Before and After actions to the Audit query if the change affected the following fields: Title,Url Name, Menu Area and Menu Group. The system doesn't log XML content, but you can view changes to the XML field in the Audit Table.

The system also logs who made the change.

Importing and Exporting Workflow Reports

You can export reports using a JSON file format. The workflow report's browse menu includes Import and Export options. The Import option is shown if the Can import data option is checked for the appropriate user in Configuration > User.

You can import reports in QXT and JSON formats via the Utility > Static Data Import Status page as well.

NOTE: QXT is no longer available as an export option, however the import action supports both QXT and JSON formats.

Success or failure message for the import is shown at the top of the page.
