Static Data Approval

There are currently two methods for static data approval:

  • Unapproved static data - see Unapproved Static Data
  • Entity and Instrument static data - see Static Data Approvals

The latter is new development available from v6.7.1- in future releases it is to replace the existing static data approval functionality.

General Information - Unapproved Static Data

For Static Data items requiring Static Data Approval, there is a separate page for unapproved records. This can apply to Approver, Bank Account, Broker Account, Stock Account, Counterparty, Counterparty Bank Account, Counterparty Stock Account, Facility, and/or Money Market Fund records.

The applicable Unapproved pages can be accessed either from the Static menu, or by following a Pending Changes badge link on the main application page. This badge is displayed after a new or amended record is added, as shown in the following example of a Bank Account page.

General Information - Static Data Approvals

The Static Data Approval page is available from the Processing menu.

As of v6.7.1 it applies to the following static data types/modules:

  • Bank Flow Instrument
  • Cash Flow Instrument
  • Cash Transfer Instrument
  • Commodity Swap Instrument
  • Currency Option Instrument
  • Deal Set Instrument
  • Entity
  • FRA Instrument
  • Future Contract Instrument
  • FX Instrument
  • Interest Rate Option Instrument
  • Letter of Credit Instrument
  • Money Market Instrument
  • Repo Instrument
  • Security Instrument
  • Stock Transfer Instrument

The Static Data approval page matches the Processing menu item's Deal approval and Settlement approval pages, and provides a configurable list of static data items which are awaiting either Approve or Reject actions. These items appear in the list as a "Pending" state because they may have recently been created as new, updated, or deleted.
