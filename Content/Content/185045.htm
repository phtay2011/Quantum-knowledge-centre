Configuring Pivot Views

When creating a report, use the Layout tab in a Pivot view configuration to define grouping of column and row data, aggregation, sub-totals or grand totals, drill-down levels, and other options for displaying the results of a query.

Additional Public or Private user-defined Pivot views can be added by selecting New Pivot in the Configure View menu.

An example Layout tab for a Pivot view configuration is given below.

Procedure Summaries

To configure a Pivot view

  1. Do one of the following:
    • for an existing Pivot view - select the name of the Pivot view in the name lookup and click the Configure link, or
    • for a new view - in the grid menu or the Configure View menu, select New Pivot.
  2. On the Layout tab, optionally select any Columns - group columns by fields you want to group the columns by (optional), and for each of those fields:
    • select or clear the Sub Total, End Total, and/or Hidden options (see Configure Displayed Totals in Pivots),
    • select the drill-down Level (see "Notes on Levels" below), and
    • if required, select the display Format.
  3. Select one or moreRows - group rows by fields you want to group the rows by and, for each one, select or clear the options for each of those fields (as in Step 2 for Columns).
  4. Optionally select any Values - aggregate on fields whose data you want to aggregate (if any), and for each of those fields:
    • select the Aggregate By and Total By options (see Configure Displayed Totals in Pivots);
    • select the drill-down Level (see "Notes on Levels" below), and
    • if required, select the display Format.
  5. Optionally select Show grand total row and/or Show grand total column to display a grand total row or column, respectively. For details, see Configure Displayed Totals in Pivots.
  6. Optionally select Split view by level 0 groups to split the the pivot into separate views. For details, see Configure Split Views in Pivots.
  7. Optionally select Drill down to table view to enable drill-down on aggregated values and select the Table View Columns and Sort fields. For details, see Drill Down to Table View in Pivots.
  8. On the Filters tab, optionally add one or more View filters as required. See Filtering the View's Displayed Results.
  9. (Optional for Quantum and Adaptiv only) On the Columns tab, you can change the default displayed names for columns in the view, and specify additional calculated columns. See Building an Expression (in Calculated Columns).
    • The fields in the Query Columns area are those selected on the Layout tab.
    • The fields available for selection in Calculated Columns expressions (and on the Layout tab) depend on the type of report being created.
  10. On the Colors tab, you can:
    • select the Columns you want to add background color to or select ALL for the whole row;
    • select the Background Color;
    • optionally select the color for rows that show totals;
    • optionally add Conditions under which the color displays, where the condition only uses fields in drill-down level 0 of the pivot (row, column, or value);
    • optionally add a Description that will appear when the user hovers over the cell.

      NOTE: The Totals area appears when you select Sub Total or End Total in the Layout tab for the current view.

  11. (Optional) On the Profile tab, you can summarize the report's result set into time buckets (periods) and display the total of all the amounts that occur in each time period. See Summarizing the Result Set Into Time Segments.
  12. (Optional) On the actions tab, you can define and apply custom actions (as for a custom report). See Add Custom Actions in Pivot views.
  13. On the Properties tab > Rounding area, select from the following:
    • Columns - the columns the rounding will apply to.
    • Rounding Method - type of rounding, based on the next digit after the least-significant displayed digit
      • 5/4 - round up when the next digit is 5 or more. Round down otherwise.
      • Up - always round up
      • Down - always round down (truncate)
    • Round Amounts - round the number to... Whole Amounts, Thousands, Millions, Billions, Embedded. Appears next to the grid heading. Select Embedded if you want to show embedded units, "k", "m", "b" etc , beside the result.
    • Decimals - set the number of digits to appear after the decimal point. Applies to all displayed currencies. The default is 0 (zero).

      TIP: Run the query to see the results with the rounding and decimals shown. Export the results and they appear with the rounding and decimals you set.

  14. (Optional) On the Properties tab, set the required Access Permission for the view, and click Save. See Saving a View Configuration.
  15. Click Apply to see the effects of your changes on the current results, if any.

To set up rounding with embedded units for a pivot or table

  1. In the View grid menu, select New Pivot or New Table and then choose the Properties tab.
  2. Select the Column to round, the Rounding Method, and then in Round Amounts select Embedded.
  3. Choose the number of decimal places, remembering that percentages are presented as decimals.
  4. The export to Excel uses the raw numbers, but other exports use what is in the View.

Notes on Levels

  • By default, a pivot displays the Level 0 fields.
  • When there is a next level configured, the pivot will display hyperlinks that can be used to drill down to that level.
  • Navigating to the next level will make all fields up to that level visible.
  • A breadcrumb trail appears at the top of the page from Level 1 onwards to show the context of the drill-down.
  • Click a higher level in the breadcrumb trail to return to that level.
  • You can use the assigned levels to split the the pivot into separate views.
