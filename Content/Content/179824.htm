Supporting Attributes, Layout XML Data record

This section covers some of the important supporting attributes in the Layout XML Data record.

Data Selection (xpath), Layout XML Data record

xpath is the most important construction element in the Layout record. Its primary use is to specify where the data should come from, using W3C standard XPath language.

Some simple arithmetic and date functions may also be included.

Structure

<Field â€¦ xpath="...".../>

Supported Functions

  • Abs(): Finds the absolute value of a data item
  • DateDiff(<dt1>, <dt2>): Finds the number of days between two dates
  • DateValue(<dt1>): Converts the date into an Integer so that comparison between dates is allowed
  • ends-with(): Returns true if the first argument string ends with the second argument string, otherwise returns false
  • Matches(): Finds the specified input string for all occurrences of a regular expression
  • Round(<num>, <number of dp>): Rounds the number to a number of decimal places
  • Sgn(): Returns an integer to reflect the sign of a number.
  • starts-with(): Returns true if the first argument string starts with the second argument string, otherwise returns false
  • substring-after(): Returns part of the string value for the xpath after the item specified
  • substring-before(): Returns part of the string value for the xpath before the item specified
  • Ucase(): Converts the string value of xpath into upper case
  • +, -, /, *: Normal arithmetic functions to add, subtract, divide and multiply

    NOTE: When using + to add numeric fields, specify eval(#...#). Otherwise + performs string concatenation.

Examples

<Field xpath="/confirmation/action/eventNo" .../>

<Field xpath="#/confirmation/.../price# - #/confirmation/.../discount#" .../>

<Field xpath="Abs(#/confirmation/.../price#)" .../>

<Field xpath="DateDiff(#/confirmation/.../startDate#,#/confirmation/.../matureDate#)" .../>

<Field xpath="Abs(DateDiff(#/confirmation/.../mateureDate#,#/confirmation/.../startDate#))" .../>

<Field xpath="substring-before(#/confirmation/.../accountNumber#, '[')" .../>

processMethod attribute, Layout XML Data record

Normally, values of each Field or FieldItem element are extracted by using an xpath element in the XML Layout record. The xpath element specifies where the data is located in the data source, and may appear several times. In such cases, the processMethod attribute provides special operations related to the xpath elements (see count and repeat below).

Structure

Field ... processMethod="..." .../>

Values

  • count: Counts the number of times the element or attribute specified in the xpath element appears in the data source, and assigns the result to the Field or FieldItem element.
  • repeat: Iterates through the xpath of elements or attributes from the data source and takes the contents into the Field or FieldItem element, treating it as though the Field or FieldItem element itself is repeated.
  • repeat_asc: As repeat, but also sorts the resulting contents in ascending order.
  • repeat_desc: As repeat, but also sorts the resulting contents in descending order.

Example 1

If there are four pay cashflows in the data source, this example gives a result of 4.

Field id="18A" prefix="18A" mandatory="yes" processMethod="count" xpath="/confirmation/action/deal/cashflows/cashflow[@context='pay']"/>

Example 2

If there are four pay cashflows in the data source with the dates shown below, the following example gives the following results:

:30F:20050107
:30F:20050407
:30F:20050707
:30F:20051007

Field id="30F" prefix="30F" mandatory="yes" processMethod="repeat" xpath="/confirmation/action/deal/cashflows/cashflow[@context='pay']/date""/>

Mapping attributes, Layout XML Data record

Use mapping when a particular value obtained from the database needs to be mapped to a value that is more useful to the recipient of the message.

Structure

<Field ... xpath="..." map="...">
  <MappingDetail method="..." replace="..." space="..."/>
</Field>
...
...
<MappingCategories>
  <MappingCategory category="...">
    <MappingGroup>
      <MapFrom>
        <value>...</value>
        ...
      </MapFrom>
      <MapTo>...</MapTo>
    </MappingGroup>
    ...
  </MappingCategory>
  ...
</MappingCategories>

Rules

When a mapping needs to be performed, use the xpath and map attributes within a Field or FieldItem element to give the location of the "source" value to be mapped and the name of the MappingCategory to be used, respectively. If your mapping requires non-default behavior - for example, partial matching - use a MappingDetail element under the Field or FieldItem element to specify more detailed instructions.

The MappingCategories element (at the end of the XML Layout file) contains one or more MappingCategory elements, with their category attribute giving the target name referred to by the map attribute within any Field or FieldItem element (see above).

During mapping, the program will:

  • search within the target mapping category's MappingGroup elements for a value within a MapFrom element that matches the source value,
  • apply the default matching method (direct), unless contain is explicitly specified (see the MappingDetail element's method attribute below),
  • if a match is found, return the MapTo value within that group, then replace (in the output message) the source value with that returned value, EITHER in full and retaining spaces (the defaults) OR applying any explicitly specified replace and/or space attribute settings (within a MappingDetail element), or
  • if a match is NOT found, and the matching method is:
    • contain - the source value is retained, or
    • direct (the default) - the source value is replaced by an empty character string*.

      *IMPORTANT: If you do not want specific values to be removed when using the direct method, include a MappingGroup element for each such value, specifying the same value within both its MapFrom and MapTo elements.

      See Example 1 below.

Attributes

MappingDetail:

Attribute

Required?

Default

Description

method

No

direct

Defines how the source field data will be mapped for the category; possible values are:

direct

  • if the field data fully matches a value in a MapFrom section, the MapTo value will replace the data in full, AND
  • if the field data does not match any MapFrom value, that field data will be replaced with an empty string.

contain

  • if the field data contains a value in a MapFrom section, the MapTo value will replace the data in full or part (according to the replace setting), AND
  • if the field data does not match any MapFrom value, that field data will be retained.

replace

No

all

(Applies only when method = contain.)

Defines how much of the source field data is to be replaced by the MapTo value; possible values are:

  • all - replaces all the field data,
  • whole - replaces only the whole word containing the matching value, or
  • partial - replaces only the matching characters.

space

No

preserve

Defines how spaces are handled; possible values are:

  • collapse - removes all leading and trailing spaces and replaces multiple spaces with one,
  • preserve - does not remove any spaces, or
  • remove - removes all spaces.

Where spaces are required to fill an entire field, the special attribute xml:space="preserve" must be used.

MappingCategories:

  • category: the unique identifier for the MappingCategory that can be referenced by a Field or FieldItem element's map attribute.

<!-- The next line represents the actual source value to be mapped -->
xxxxx
</FieldItem>

    <MappingGroup>
      <MapFrom>
        <value>FedWire</value>
        <value>FW</value>
      </MapFrom>
      <MapTo>FW</MapTo>

Character Formatting Attributes, XML Layout File

Use the formatType attribute to format date, number, and string values. formatType is an optional attribute for Field and FieldItem elements.

Structure

<Field ... formatType="...".../>

Rules

The value specified in formatType instructs the program in how to format the value of its belonging element. Specific rules and conventions need to be followed for each value, as described below.

For dates, use the convention DATE_format, where format is a valid combination of Ys, Ms and Ds (for year, month and day respectively) and separator characters.

String manipulation

To manipulate a string, use one of the following syntaxes:

  • STRING_[UPPER|LOWER] [charIndexes]

Where:

  • UPPER or LOWER are optional, and force the result to be all upper or lower case, respectively. For example, SWIFT requires all characters to be in upper case, but static data in Quantum may not necessarily be in upper case, and therefore needs to be converted.
  • charIndexes is a list of numbers and/or number groups that correspond to the index (position) of characters inside the string value of the current Field or FieldItem element. The relevant characters are extracted to form a new string value for the current Field or FieldItem element.

    A hyphen or dash between two index values specifies an inclusive group of consecutive numbers, and individual indexes or groups must separated by a comma.

Validation attributes, Layout XML Data record

Validation messages are used for reporting purposes and to advise the user that something is incorrect. The objectives are to submit the appropriate message to the message log and to determine if the problem is serious enough to stop processing.

TIP: Most messages will be provided through the use of an Include.xml file containing a series of messages. See Include element for further information.

Structure

Rules

The Validations element defines actual message content and should be located at the end of the XML Layout file. It contains one or more Validation elements and/or one or more Include elements.

Each Validation element (within either a Validations element or a referenced Includes record):

  • Has an id, type, and title attributes (see Attributes of the Validation element below)
  • Is uniquely identified by its id attribute value
  • Defines a single validation message's content
  • May contain:
    • Text
    • field elements - replaced in the output message by the actual field value
    • SelectCase elements containing Case (and Else) elements - replaced in the output message by the value specified for the case that applies at the time

The validationID attribute value can be used to refer to the actual message defined in the Validation element whose id attribute matches that value (EFTs) or located at the end of the Layout XML Data record (Confirmations).

For all elements that support validationID (except Message):

  • If any of the following occur:
    • mandatory = "yes" and no values are obtained
    • mandatory = "loose_conditional" and the condition is evaluated as True and no values are obtained
    • mandatory = "strict_conditional"and the condition is evaluated as Trueand no values are obtained
  • Then the system will:
    • Log one of the following messages:
      • The message referred to by validationID
      • A "No message content found" message if validationID is not found or has invalid attributes
      • A "Mandatory field has no value" default message if validationID is not included.

    -and-

    • Stop the current message process.

For the Message element, the following also applies:

  • If mandatory = "strict_conditional"and the condition is evaluated as False...
  • Then the program will:
    • Log only the message referred to by validationID (no message if validationID is not found or not included)

    -and-

    • Stop the current message process.

For all other elements, no message will be logged, nor should the process be stopped.

mapTable Attribute, Layout XML Data record

Maps an input value, specified in the Layout XML Data record, to a Quantum table field, and returns the required value in the EFT message from the table when specified in the EFT layout for a given input data defined in the xpath. The system logs when it finds duplicate records or when it finds no value.

mapTable=(EWTable, map From, map To)

Where:

  • EWTable is the Enterprise Workflow table name
  • mapFrom is the field in the table you map from
  • mapTo is the field in the table you map to
