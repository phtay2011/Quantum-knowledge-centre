Instructions Settings, Stock Account Defaulting Rule

These items define the Instructions that will be applied when a stock flow meets the Matching Criteria for the applicable rule entry.

The instructions that will be used are those with the applicable Effective Date for the stock flow date.

Effective Date - The date a set of instructions is active from.

  • If more than one set of instructions is defined, each set must have a different Effective Date.
  • The instructions that will be used have the nearest Effective Date on or before the stock flow date - that is, where the stock flow date is greater than or equal to the Effective Date, but less than any later Effective Dates.

    TIP: DO NOT change the Effective Date if you are changing any other settings in that instruction. Instead, add a new set of instructions with the new Effective Date. To do this, select New from the grid menu and enter the new instruction values.

Bank (Custodian) - The custodian to use when this stock account instruction is applied.

TIP: Only Counterparties with the Custodian category selected apply,and the Currency specified in the Matching Criteria area further determines what is available for selection here.

Account Number - The custodian stock account number to use when this account instruction is applied.

  • The custodian and currency specified for this instruction determines which account numbers are available for selection here.
  • The Account Number should not include the Sort or BIC codes.
  • When you select an account number, the Account Name and Account Currency are indicated for this instruction.

SWIFT DSS (read-only) - shows the optional SWIFT Data Source Schema code value (if any), as specified in Counterparty Delivery Instructions. This code will be output in stock flow messages for the counterparty when this instruction is applied and:

  • the Payment Method for the corresponding cash flow is SWIFT,
  • the counterparty Stock account has a Unique Identifier value that applies for this code, and
  • the counterparty Stock account's specified SWIFT MT Format Template indicates that this Data Source Scheme code and the Unique Identifier are to be output.

Unique Identifier (read-only) - shows the optional Unique Identifier value for the Counterparty's Custodian, as specified in Counterparty Delivery Instructions (and as usually supplied by the Custodian).

Default Payment Method (read-only) - the payment method to be used for cash flows when this account instruction is applied.

TIP: If you do not specify a payment method, the default payment method associated with the selected account applies (if there is one).

Entity Identifier - the Entity identifier value for the specified Default Payment Methodif an identifier is specified for that Payment Method (for example, for a Clearing House). You can select only from Entities valid for the payment method (if any).

Default SWIFT MT Format (optional) - the default SWIFT MT Format Template for the Custodian Stock account when this instruction is applied to a stock flow. A SWIFT MT Format template specifies the output format of stock flow data for SWIFT EFT output.

  • You can select only from SWIFT output format templates found in the Allowable SWIFT MT Formats list specified for the Custodian.
  • This value may be superseded for individual stock flows in Settlements Processing.

Link to settlement cash flows - Use this option to specify whether or not cash flows are to be linked to stock flows at deal save time when this defaulting rule applies for that deal. The setting for each Instruction grid row is indicated in the Linked to Cash column in the Instructions grid as Yes or No, respectively.

When this option is selected (the default), cash flows will be linked to stock flows where applicable* at deal save time when this defaulting rule applies for that deal, and the stock flow Payment Type is set to AGAINST LINKED PAYMENT. When the flows are listed in the Settlements page's results grid:

  • both flows are badged Stock,
  • they will be actioned and further processed together as a linked set, and
  • when they are actioned, associated EFT messages are output (indicated by the sameEFT External Message Reference value).

When the option is cleared, stock flows and cash flows that would otherwise be linked* are created as standalone flows at deal save time when this defaulting rule applies for that deal, and the stock flow Payment Type set to FREE OF PAYMENT. When the flows are listed in the Settlements page's results grid:

  • the flows are separately badged Cash and Stock,
  • they will be actioned and further processed independently, and
  • when actioned, separate EFT messages are output (indicated by differentEFT External Message Reference values).

*TIP: Some stock flows never have associated cash flows - for example, in Stock Transfers, Unit Holding Splits, and so on.
