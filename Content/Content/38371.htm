FX Deal Pre-Delivery or Extension

Use the Predeliver or Extend function to:

  • display a summary of any pre-deliveries or extensions already recorded for the current FX deal, with a default final entry showing the Residual (outstanding) balance left on the deal;
  • settle part or all of the deal after the Value Date of the original deal (Extension); and/or
  • settle part or all of the deal before the Value Date of the original deal (Pre-delivery, similar to drawing down a deal using the Time Option feature).

TIP: You can also Pre-deliver or Extend an FX Deal in Settlement Processing (when you select a single cash flow for a suitable FX deal).

The system manages any residual amounts as usual, on the Value Date of the original FX deal.

The following conditions and restrictions apply to Pre-deliveries and Extensions.

  • You can pre-deliver or extend any transaction that has not yet been actioned.
  • You cannot pre-deliver or extend an FX Time Option.
  • You can pre-deliver or extend a single FX deal any number of times, provided you do not exceed the value of the original deal.
  • You can pre-deliver or extend an FX deal that is in a Back-to-Back or Swap relationship while that relationship exists; however you will need to modify the other deal in the relationship also or the Back-to-Back or Swap deal will be unsynchronized.
  • Split Settlement dates cannot be changed for Pre-deliveries and Extensions.

Procedure Summary

  1. In the page menu, select Predeliver or Extend.

    The New Predeliver or Extend Transaction dialog box opens, displaying a summary of any pre-deliveries or extensions already recorded for the deal, with a default final entry showing the Residual (outstanding) balance left on the deal.

    TIP: Only the default entry shows on a new deal.

  2. To add a new Pre-delivery or Extension: select an existing entry in the Transactions grid and in the grid menu, select Edit (or click on any cell value).

    The Edit Transactions dialog box is displayed.

  3. Select BUY or SELL, enter the relevant Amount, New Value Date, and New Rate, and click Save.

    The system compares the New Value Date with the Value Date of the original deal to determine whether the new transaction is a Pre-delivery or an Extension. Forward points are calculated from the difference between the old and new Contract Price.

    The new Extension or Pre-delivery transaction is recorded with the deal and any subsequent Deal Map items are adjusted as required.

  4. Save the deal.

IMPORTANT:

The Non Business Day Settlement rule for the deal can affect the pre-delivery or extension when the specified New Value Date is a non business day in either currency. If the rule is:

  • Not Allowed - the pre-delivery or extension will not occur,
  • Asymmetric Cashflows - the pre-delivery or extension will occur and the cash flow is moved to the next valid business day for the currency, or
  • Symmetric Cashflows - the pre-delivery or extension will occur and the cash flow remains on that date (whether a valid business day in both currencies or not).

Position Reporting

After running a Position query to report on FX deals as at a particular date, you can view the results using either:

  • a deal-level View format that will show only a summary of the events, or...
  • a detail-level View format that will show details of each FX Pre-delivery or Extension for the period covered.

Detail-level information is available whether you choose Management or Accounting Reporting Control.

When an FX Deal is partially pre-delivered for example, a detail-level View shows both the pre-delivered amount and the balance remaining on the original forward date. If the View format includes the Forward Rate field, a rate is shown for the Pre-delivery. This is the rate current at that time, not that of the original contract.

NOTE: Position query results show only events meeting the Query configuration settings. You can run Deal Details or a Revaluation Analysis to see all events on a particular FX deal.
