Currency Swap Accounting Rules

Use the Currency Swaps tab to create and maintain accounting rules for Cross Currency Swaps.

There must be at least one Accounting Rule entry that matches each type of these deals that your organization uses. If more than one, the most specific matching rule entry takes precedence. For example, entries for Instrument groups take precedence over ALL Instrument Name entries (labeled For any ... in the rules grid). Individual instruments that are members of a group will apply settings defined for that instrument before those for the group, before those for an ALL entry.

You can add, edit, and delete specific rule entries using the tab's grid menu, as follows.

NOTE: You cannot add, edit, nor delete inherited entries.

To add a new rule entry

  1. In the Currency Swaps tab's grid menu, select New (or select an entry row and in the grid menu, select Copy).
  2. In the dialog box under Matching Criteria select one or more Instrument Name, Counterparty Name, and Strategy Name values2,3 or ALL in each case, and/or any other customized criteria that may apply for this transaction type.

    NOTE: You cannot specify a combination of criteria already covered by an existing entry.

  3. Under Contingent Accounting Settings optionally select Apply from deal date to expiry date to post contingent Settlement receive and pay amounts calculated from Deal date to Exercise/Lapse/Knock-out date, and set Amount to SETTLEMENT.
  4. Under Accrual Settings select any of the following options to post the relevant accruals to Profit & Loss (for both swap legs):
    • Accrue interest income/expense,
    • Accrue capital premium/discount, and/or
    • Accrue adhoc amount.
  5. If you selected Accrue interest income/expense in Step 4, then select the associated Interest Translation date:
    • POSTING DATE CCY TRANSLATION - to use the relevant FX Rate on the day the accrual entries are posted, or
    • CASHFLOW DATE CCY TRANSLATION - to use the FX Rate on the day the cash flow to be accrued/amortized is posted. Where the cash flow being accrued is in the future, the posting date should be used.

      TIP: The accrual postings created by End of Day Accrual Processing are translated to base using the Accrual Rate Type setting for the relevant General Ledger record.

  6. If you selected Accrue capital premium/discount in Step 4, then select the associated Capital Translation date (the options are as for Step 5).
  7. If you selected Accrue adhoc amount in Step 4, then select the associated Adhoc Amount Translation date (the options are as for Step 5, but are prefixed ACCRUE and AMORTIZE, respectively).
  8. Select the Accrual Method to calculate the accrual of Interest or Capital amounts: Straight line or Yield.
  9. Select the Accrual Day Count option: USE NUMBER OF DAYS IN PERIOD or USE DEAL INCOME DAYS BASIS.
  10. Under Revaluation Settings, optionally select Revalue deals* if you want to create revaluation postings, and as required, select:
    • Revaluation Method (BILLS, BILLS NET, PROJECTED, PROJECTED NET),
    • Add margin to base rates* and Margin Rate Type*,
    • Revaluation Currency (SEPARATE or DOMINANT ONLY),
    • Posting Method (REALIZED or UNREALIZED), and
    • Interest Rate* and Interest Rate Type*.

      * Mandatory settings when Hedge Accounting applies.

  11. If you selected Accrue interest income/expense in Step 4, then select the associated Adhoc Interest in Book Value option (INCLUDE or EXCLUDE).
  12. If you selected Accrue adhoc amount in Step 4, then select the associated Adhoc Amounts in Book in Value option (INCLUDE or EXCLUDE).
  13. Under Translation Settings, optionally select Apply deal level currency translation to apply deal-level translation to the specified ledger.
  14. If you selected Apply deal level currency translation in the previous step, optionally select Summarize unrealized postings on principal repayments to apply summary (Net) posting of unrealized deal level G/L entries for multiple principal repayments on a deal.
  15. Click Save. The new rule entry is added to the grid.

To edit an existing rule entry

  1. Click an entry value in the grid (or select the entry and in the grid menu, select Edit).
  2. In the Edit window, edit the settings as required (see above), and click OK.
  3. Save your changes.

To delete an existing entry

  1. Select the entry in the grid and in the grid menu, select Delete.
  2. In the dialog box, click OK to confirm.
