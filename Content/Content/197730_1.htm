Expression Builder - New and changed functions

Functions are listed in alphabetical order.

DateValue function in Expression Builder

When using Expression Builder - for example when configuring Quantum report result views - you can now include a function to convert a text string into a date, as follows.

Description

Converts a date from a text string in the specified format to a date value

If the system cannot resolve the date, it returns a blank value

Format

DateValue(dateAsString, dateFormat)

Arguments

dateAsString - a string representation of the date (a text field or a literal value)

dateFormat (optional) - the format of the date in the string. If not specified, the user's default date format applies.

Examples

DateValue("31/10/2019", "dd/mm/yyyy")

DateValue("19Oct31", "yyMMMdd")

The result in both cases is a date field with the value for October 31, 2019.

Hour

Description

Returns the hour number from the date value. The value ranges from 0 to 23.

Format

Hour(date)

Arguments

date - use the current value from a date field. In datetime format

Example

Hour(3 Dec 2019 15:36:28)

Returns the hours value from the date field - 15

Item

Description

Returns the item at the specified position in the list

Format

Item(position, list)

Arguments

position - the specified position in the list of the item to return. Returns null when the position is outside the boundaries of the list. Is 1-based, i.e. Item(list, 1) is the first item in the list.

list - the comma-separated, ordered list that contains the items

Example

Item(1, {"hello","there"})

Returns hello

IsBlank

Description

Returns Yes if the given field is blank; No if not

Format

IsBlank(field)

Arguments

field - the field to be checked

Example

NA

Join

This function has been changed, with a new format added that includes a user-defined separator. The original format is retained, so that any existing use is unaffected.

Description

Returns a single text string of joined values delimited by a specified separator.

Format

Join({values},separator)

Arguments

{values} - the text strings and/or numbers to be joined, in correct order, comma-separated, within curly brackets

separator - user-defined separator

Examples

Join({[Comment], "Dare", "To", "Dream" }, " ")

If Comment is Always, the result is Always Dare To Dream with the separator being a space.

Join({[Currency],[Entity],[Instrument]},"-")

If Currency is AUD, Entity is OurCorp, and Instrument is AUD LT Borrow, the result is:

AUD-OurCorp-AUD LT Borrow

Additional Notes

  • The number of arguments is unlimited.
  • Arguments can be any other expression, field, or value.
  • A Join function is not allowed within an If function.

Split

Description

Splits the text at the separator and returns a list of text fragments.

Format

Split(text, separator)

  • Returns a list with one text (original text) if the delimiter cannot be found in the text
  • Returns a list of characters if the separator is not specified or blank
  • The grid displays the list output as newline separated values in one cell.

Arguments

text- the section of text to look through

separator - the type of separator to look for and split at

Example

Split("this is a series of words", " ")

Splits the phrase at the first space (" ") and returns "this"
