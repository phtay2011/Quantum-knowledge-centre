Implementing Fees on Quantum deals

This section summarizes the methods for implementing fees on the following types of Quantum deal.

Fees on Unit and Unit Holding deals

Unit Fee Defaulting Rule records can be defined for combinations of Instrument*, Trading Style, Exchange*, Broker*, Security ID*, and Deal Actions.

TIP:ALL may be specified for these.

For each rule record, up to four fees can be specified, with each fee specifying:

  • a Fee Code (from an existing Fee Name record),
  • either PAY or RECEIVE;
  • a Type (Fixed, 1 Tier, 2 Tiers, or 3 Tiers);
  • Timing, Currency, and Fee Counterparty; and
  • other settings depending on the Type.

Fee details default to those specified in the best-matching Unit Fees Defaulting record (if any) for:

  • Unit deals,
  • PIK Dividends (Payment in Kind) on Unit Holdings, and
  • Transfers on Unit Holdings.

Fees on Futures and ETO deals

Exchange Fee records can be defined for combinations of Exchange, Contract*, Executing Broker*, and Contract Type*.

TIP:ALL may be specified for these.

Each Exchange Fee record specifies the fee Currency and details for one or more of the following fee types:

  • Exchange,
  • Clearing, and
  • Commission.

Fees on Futures or ETO deals will be applied according to the best-matching Exchange Fee record.

Fees on Repo deals

Fees are an integral part of Repo deals for the following Repo formats:

  • Fee Based Borrowing,
  • Fee Based Lending,
  • Security Borrowing, and
  • Security Lending.

Fee defaults may be specified on the underlying Repo instruments in each case.

TIP: Fee Names do not apply.

Fees on Money Market, Securities, and Interest Rate Swap deals

A more advanced fees implementation applies for these deal types, which use the Fees dealing, Process Fees, and related pages.

The main tasks for implementing fees in Quantum for these deal types are:

  1. Set up Quantum Static Data and instruments for Fees.

    Before you can define fee deal parameters and scheduled dates, you must specify a Fee Name for each fee to be implemented, and an Actual Cash Flow instrument for each type of fee cash flow to be generated.

    Before you can define Fee Base Templates for Facility fees, you must specify at least one Facility or Dealing Line with appropriate settings - see Specific requirements for Facility fees.

  2. Set up Fee Base Templates and Formulas.

    Before you can save a Fee deal definition, you must create at least one Fee Base Template or Fee Formula, as required. You can access these via the Instruments menu, or from the Fees Dealing page menu.

  3. Create or modify Fee deal definitions.

    NOTE: A "Fee deal" is not like other deals in Quantum. It is more like a template, providing settings and scheduled dates that enable generation of fee cash flows and G/L entries (via the Process Fees page). Deal approval does not apply to Fee deal definitions.

    Use the Fees dealing page to:

    • assign Base Templates or Formulas to the Fee deal,
    • specify static data for generated fee cash flows,
    • define Fee estimation parameters (when applying True Ups),
    • schedule dates for fee calculation, payment, invoicing (for True Ups), and base amount resets, and
    • specify G/L codes for Fees journal entries (Quantum Posting Rules do not apply).
  4. Create Position and Accrual queries for Fees.

    Before you can process fees, you will need:

    • a Position query - to provide the fee position history, giving deal values, events, and static information for Fee base amounts as at the required date; and
    • an Accrual query - to provide the Interest values for Position fees as at the required dates, enabling generation of G/L entries for fees accruals.
  5. Create or update Fee Position History and base amounts.

    On a regular basis (daily is recommended), use the Process Fees page's Get Base Details function to retrieve deal data for calculating all fees as at the date you specify (depending on the applicable deals, Fee types, and Fee deal definitions). This data populates the Fee Position History database table.

  6. Evaluate (Test) a Fee Base Template or deal filter values against the current Fee Position History table.

    Use the Process Fees page's Evaluate settings and Run Query function to view - within the current Fee Position History table - applicable base deals, total base amount, and total fees when you apply either a specified Fee Base Template or specified Filters values.

  7. Generate Fee cash flows and G/L entries.

    Use the Process Fees page's Generate Fees function to:

    • calculate the Fee Base Amounts, cash flows, and accounting entries for a specified Fee deal definition, or All active Fee deals, within a date range; and
    • review the calculated results, including marking items as approved for processing or suppressing them.)
  8. Process the generated Fee cash flows.

    Use the Process Fees page's Process Cash Flows function to process selected fee cash flows.

  9. Post the generated Fee G/L entries.

    Use the Process Fees page's Post Selected G/Ls function to post selected fee G/L entries.
