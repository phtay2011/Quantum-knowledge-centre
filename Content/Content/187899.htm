Resource Security and Other Applications

This topic summarizes the specific Resource Security effects that apply with the following application pages. (To view the secured Resources for all deal types, see Secured Resources by Deal Type.)

Counterparty Bank Account

Resource Security is not applied to the Bank field on both the Bank Details tab, and Intermediary Bank tab.

Counterparty Stock Account

Resource Security is not applied to the Custodian field on the Custodian Details tab.

Confirmations Processing

Individual query access can be secured against Users if the Query Filtered Resources option is set to Enabled (on the Data Security page). See Applying Resource Security to Queries and Reports.

Users may export and print Confirmations for actions on deals for Counterparties and Entities they have at least Read access to.

Deal Approvals

Individual query access can be secured against Users if the Query Filtered Resources option is set to Enabled (on the Data Security page). See Applying Resource Security to Queries and Reports.

When running a query with the Include deals I cannot action option selected, the result set will include cash flows for Resources the User has eitherRead or Write access to.

Users may approve only actions for deals they have Write access to, and which meet the applicable Approver, Signature List, and Deal Approval Rule criteria.

Settlement Approvals

Individual query access can be secured against Users if the Query Filtered Resources option is set to Enabled (on the Data Security page). See Applying Resource Security to Queries and Reports.

When running a query with the Include flows I cannot action option selected, the result set will include cash flows for Resources the User has eitherRead or Write access to.

Users may approve only cash flows they have Write access to, and which meet the applicable Approver, Signature List, and Settlement Approval Rule criteria.

Settlement Processing

Individual query access can be secured against Users if the Query Filtered Resources option is set to Enabled (on the Data Security page). See Applying Resource Security to Queries and Reports.

The results grid lists only those cash flows and Diary items for Resource-related data the User has Write access to.

Use of the Approve function (on items pending approval) is subject to Settlement Approval Resource Security (see above).

General Ledger Close

If a User has Write access to the G/L Owner, then all its sub-entities will be processed irrespective of the User's access level to those sub-entities.

General Ledger Year End

Only the G/L Owner Entity is securable.

General Ledger Interface and Interface Mapping

Only the G/L Owner Entity is securable.
