Bank Details tab (Settlement; Counterparty Instructions)

In Account mode, you can change the default values in these fields.

In Adhoc Instruction or Third Party mode, use these fields to enter new details.

See also: Changing Counterparty Instructions.

This tab records the actual details of the delivery instruction. If the fields are insufficient in size or format, you may use free-format instructions in the Delivery Narrative text box (Beneficiary Details tab) instead of (or in addition to) these details.

Bank

This is the name of the bank where the Counterparty account is held. You can either select from the drop down list of counterparties with a Bank category, or enter a new free-format name as required. When you select a name from the list, related fields on this tab are repopulated accordingly.

Entering a new name here does not create a new Counterparty record (nor do you have to create one in Counterparty Setup).

NOTE: If an Intermediary Bank is specified, you must enter a value in this field. However, with the SWIFT payment method, you may enter N/A provided that you enter a BIC Code.

BIC/Swift Code

The Bank Identification Code or SWIFT Code for the delivery bank.

Sort Code

Enter the prefix sort code digits for this bank.

Clearing Fields

This area displays the Clearing Fields defined on the Bank's Counterparty record (if any).

Optional Intermediary Bank Clearing Codes may be used with some currencies. Custom codes may also be set up. For more details, see Currency Setup - Bank Clearing tab.

Bank Clearing Codes entered here will be included in some Quantum cash flow output (for example, SWIFT Confirmations and EFTs).

CLS (Continuous Linked Settlement)

A CLS Bank Clearing field displays if one is defined for the currency (that is, the currency is CLS-eligible).

To make this Counterparty Account CLS-eligible, set the CLS Bank Clearing field to YES. For more details, see CLS in FIS Quantum.
