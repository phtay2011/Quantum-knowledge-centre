Double Barrier Options

At-expiry, double barrier options are priced similarly to single barrier options, by forming a replicating portfolio that will yield exactly the same cash flow as the barrier option in all possible states. The value of the barrier option is determined by summing the known prices for the portfolio component option (Vanilla cap/floor and European digital options).

This section shows the composition of the various replicating portfolios and proves that they yield the same results for two types of double barrier option.

  • Replicating Portfolios for At-Expiry Double Barrier Options
  • Proof for Down & In / Up & In Cap
  • Proof for Down & Out / Up & Out Floor

Replicating Portfolios for At-Expiry Double Barrier Options

This topic shows the combinations of digital options and Vanilla caps and floors that replicate the payoff structure for the different types of double barrier option. The sum of the values in each combination can therefore can be used to value the barrier option at any time prior to maturity.

In some cases a restriction applies between the option's barrier levels (LB and UB) and Exercise price (X), to either ensure that a payoff is feasible or to ensure that the barrier option is distinct from an equivalent Vanilla cap or floor.

TIP:RT = Reference Rate at maturity.

Down & In / Up & In Cap

The Replicating Portfolio consists of:

  1. A Put Digital option with:
    • Position = Long
    • Strike = LB
    • Payoff = max[LB-X,0], where UB > LB > X
  2. A Vanilla Floor option with:
    • Position = Short
    • Strike = LB
    • Payoff = - (max[LB-RT,0]), where flow is negative for a sale
  3. A Vanilla Floor option with:
    • Position = Long
    • Strike = X
    • Payoff = max[X-RT,0]
  4. A Vanilla Cap option with:
    • Position = Long
    • Strike = UB
    • Payoff = max[RT-UB,0]
  5. A Call Digital option with:
    • Position = Long
    • Strike = UB
    • Payoff = max[UB-X,0]

Down & Out / Up & Out Cap

The Replicating Portfolio consists of:

  1. A Vanilla Cap option with:
    • Position = Long
    • Strike = LB
    • Payoff = (max[RT-LB,0]), where UB > LB > X
  2. A Call Digital option with:
    • Position = Long
    • Strike = LB
    • Payoff = max[LB-X,0]
  3. A Vanilla Cap option with:
    • Position = Short
    • Strike = UB
    • Payoff = -max[RT-UB,0], where flow is negative for a sale
  4. A Call Digital option with:
    • Position = Short
    • Strike = UB
    • Payoff = -max[UB-X,0], where flow is negative for a sale

Down & In / Up & In Floor

The Replicating Portfolio consists of:

  1. A Put Digital option with:
    • Position = Long
    • Strike = LB
    • Payoff = max[X-LB,0], where X > UB > LB
  2. A Vanilla Floor option with:
    • Position = Long
    • Strike = LB
    • Payoff = max[LB-RT,0]
  3. A Call Digital option with:
    • Position = Long
    • Strike = UB
    • Payoff = max[X-UB,0]
  4. A Vanilla Cap option with:
    • Position = Short
    • Strike = UB
    • Payoff = -max[RT-UB,0], where flow is negative for a sale
  5. A Vanilla Cap option with:
    • Position = Long
    • Strike = X
    • Payoff = -max[RT-X,0]

Down & Out / Up & Out Floor

NOTE: Unlike a Down & In / Up & In Cap, the replicating portfolio is not just a combination of the components used to replicate a Down & Out single barrier floor and those for an Up & Out single barrier floor. Rather, the portfolio consists of a long and short position in an Up & Out floor, one struck at the upper boundary UB and the other struck at the lower boundary level LB.

The Replicating Portfolio consists of:

  1. A Put Digital option with:
    • Position = Long
    • Strike = UB
    • Payoff = max[X-UB,0], where X > UB > LB
  2. A Vanilla Floor option with:
    • Position = Long
    • Strike = UB
    • Payoff = max[UB-RT,0]
  3. A Put Digital option with:
    • Position = Short
    • Strike = LB
    • Payoff = -max[X-LB,0], where flow is negative for a sale
  4. A Vanilla Floor option with:
    • Position = Short
    • Strike = LB
    • Payoff = max[LB-RT,0], where flow is negative for a sale

*NOTE: Four at-expiry options are effectively the same as the single barrier option indicated above in each case, and as shown in the example below. Similar arguments apply in the other three cases, and given this redundancy, these double barrier options are not supported.

For details of the relevant options, see Replicating Portfolios for At-Expiry Single Barrier Options.

Example: Down & In / Up & Out Cap

This option will have a positive payoff only if the reference interest rate R falls between the lower boundary LB and the exercise price X. The upper boundary UB is irrelevant, so this option is equivalent to a Down & In Single barrier cap.

Proof for Down & In / Up & In Cap

The following diagram represents the payoff (CF) for a Down & In / Up & In Double Barrier Cap option, where the shaded area indicates a zero payoff.

The reference interest rate at the maturity of each caplet can fall into one of the four zones numbered above. The payoff from the replicating portfolio exactly matches the payoff for all potential values of the reference rate at maturity RT (corresponding to the four zones in the diagram), as shown below.

  1. When RT > UB, Payoff = RT -X.

    The sum of the replicating portfolio elements is the same:

    • Put Digital option (Strike LB): Payoff = max[LB-X,0] = 0
    • Vanilla Floor option (Strike LB): Payoff = -(max[LB-RT,0]) = 0
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = 0
    • Vanilla Cap option (Strike UB): Payoff = (max[RT-UB,0]) = RT-UB
    • Call Digital option (Strike UB): Payoff = max[UB-X,0] = UB-X
    • Net Payoff (Sum) = (RT-UB) + (UB-XB) = RT - X
  2. When UB > RT > LB, Payoff = 0.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike LB): Payoff = max[LB-X,0] = 0
    • Vanilla Floor option (Strike LB): Payoff = -(max[LB-RT,0]) = 0
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = 0
    • Vanilla Cap option (Strike UB): Payoff = (max[RT-UB,0]) = 0
    • Call Digital option (Strike UB): Payoff = max[UB-X,0] = 0
    • Net Payoff (Sum) = 0
  3. When LB> RT > X Payoff = 0.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike LB): Payoff = max[LB-X,0] = LB-X
    • Vanilla Floor option (Strike LB): Payoff = -(max[LB-RT,0]) = -(LB-RT)
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = 0
    • Vanilla Cap option (Strike UB): Payoff = (max[RT-UB,0]) = 0
    • Call Digital option (Strike UB): Payoff = max[UB-X,0] = 0
    • Net Payoff (Sum) = (LB-X) - (LB-RT) = RT - X
  4. When RT < X, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike LB): Payoff = max[LB-X,0] = LB-X
    • Vanilla Floor option (Strike LB): Payoff = -(max[LB-RT,0]) = -(LB-RT)
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = X-RT
    • Vanilla Cap option (Strike UB): Payoff = (max[RT-UB,0]) = 0
    • Call Digital option (Strike UB): Payoff = max[UB-X,0] = 0
    • Net Payoff (Sum) = (LB-X) - (LB-RT) + X-RT = 0

Proof for Down & Out / Up & Out Floor

The following diagram represents the payoff (CF) for a Down & Out / Up & Out Double Barrier Floor option, where the shaded areas indicate a zero payoff.

The reference interest rate at the maturity of each caplet can fall into one of the four zones numbered above. The payoff from the replicating portfolio exactly matches the payoff for all potential values of the reference rate at maturity RT (corresponding to the four zones in the diagram), as shown below.

  1. When RT > X, Payoff = 0.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike UB): Payoff = max[X-UB,0] = 0
    • Vanilla Floor option (Strike UB): Payoff = max[UB-RT,0] = 0
    • Put Digital option (Strike LB): Payoff = -max[X-LB,0] = 0
    • Vanilla Floor option (Strike LB): Payoff = -max[LB-RT,0] = 0
    • Net Payoff (Sum) = 0
  2. When UB > RT > UB, Payoff = 0.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike UB): Payoff = max[X-UB,0] = 0
    • Vanilla Floor option (Strike UB): Payoff = max[UB-RT,0] = 0
    • Put Digital option (Strike LB): Payoff = -max[X-LB,0] = 0
    • Vanilla Floor option (Strike LB): Payoff = -max[LB-RT,0] = 0
    • Net Payoff (Sum) = 0
  3. When UB> RT > LB Payoff = X-RT.

    The sum of the replicating portfolio elements is the same:

    • Put Digital option (Strike UB): Payoff = max[X-UB,0] = X-UB
    • Vanilla Floor option (Strike UB): Payoff = max[UB-RT,0] = UB-RT
    • Put Digital option (Strike LB): Payoff = -max[X-LB,0] = 0
    • Vanilla Floor option (Strike LB): Payoff = -max[LB-RT,0] = 0
    • Net Payoff (Sum) = (X-UB) + (UB-RT) = X-RT
  4. When RT < LB, Payoff = zero.

    The sum of the replicating portfolio elements is also zero:

    • Put Digital option (Strike UB): Payoff = max[X-UB,0] = X-UB
    • Vanilla Floor option (Strike UB): Payoff = max[UB-RT,0] = UB-RT
    • Put Digital option (Strike LB): Payoff = -max[X-LB,0] = -(X-LB)
    • Vanilla Floor option (Strike LB): Payoff = -max[LB-RT,0] = -(LB-RT)
    • Net Payoff (Sum) = (X-UB) + (UB-RT) - (X-LB) - (LB-RT)= 0
    • Put Digital option (Strike LB): Payoff = max[LB-X,0] = LB-X
    • Vanilla Floor option (Strike LB): Payoff = -(max[LB-RT,0]) = -(LB-RT)
    • Vanilla Floor option (Strike X): Payoff = max[X-RT,0] = X-RT
    • Vanilla Cap option (Strike UB): Payoff = (max[RT-UB,0]) = 0
    • Call Digital option (Strike UB): Payoff = max[UB-X,0] = 0
    • Net Payoff (Sum) = (LB-X) - (LB-RT) + X-RT = 0
