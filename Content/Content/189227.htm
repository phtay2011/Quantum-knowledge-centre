Setting Up Rules

Part of the accounting set up procedure is to define rules for:

  • accounting processes applied to transactions (see Accounting Rules),
  • accounts to which G/L entries are posted (see Posting Rules), and
  • accounts to which inter-company deal transactions are posted (see Inter-Company transactions).

NOTE: The procedure for setting up inter-company rules is distinctly different from that for setting up accounting and posting rules.

Combined, these rules determine how specific transactions are accounted for in Quantum.

Accounting and Posting Rules

Accounting and Posting Rules are set up in Accounting Rules and Posting Rules. These rules can be defined at any Accounting Type or Entity level.

NOTE:

  • You can define accounting and posting rules at the group entity level even when the entity is not associated with a ledger. The rules apply to any transactor entities that form part of the group, and either own a ledger or post to another entity's ledger.
  • To define an accounting rule or posting rule for a transactor entity, the entity must post to or own a ledger.

The level at which you define the rule determines whether the rule applies to lower level accounting types or entities. For further information, see Relationship to accounting types and entities.

Setting up Accounting/Posting Rules

To help you determine your accounting structure and requirements, we recommend you use a diagram of your Business Structure, with group and transactor entities identified.

  1. Decide what rules you want to apply to all ledgers, regardless of accounting type or entity. Create these at the Master accounting type level.
  2. Decide what rules you want to apply to all ledgers of a specific accounting type, regardless of entity. Create these at the specific accounting type level.
  3. For each ledger, decide what rules you want to apply to the ledger and all the entities that post into it.

    NOTE: Where the Entity that owns the ledger is a group, and it has child entities that have their own ledgers, then if the child entity's ledger is associated with:

    • the same accounting type as the parent ledger - any rules set up for the parent ledger will be inherited by the child ledger but if...
    • a different accountingtype to the parent ledger - the rules set up for the parent ledger will not be inherited by the child ledger.
  4. For each group entity that does not own a ledger, decide what rules you want to apply to the children of that entity. These rules will be applied irrespective of whether the child entities own their own ledger or post into a parent ledger.
  5. Apply any transactor entity specific posting rules at the specific entity level.

TIP: It is more efficient to set up your rules using groups rather than individual items.
