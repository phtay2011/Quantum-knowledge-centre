Database synonyms in Analytics

Introduction

Synonyms are created in the Analytics database as a way to access specific Quantum tables for Analytics database views.

There are no additional steps required to create synonyms. The system creates the synonyms during the database upgrade process.

These actions happen while you configure the Quantum site:

  • the system uses the Quantum Configuration utility to configure, validate, and fix synonyms as required on the site, and
  • the system upgrades the database using the Quantum Database Maintenance utility. This creates the synonyms needed on the site.

Use the information below to troubleshoot the process in case of issues.

Quantum Configuration utility and synonyms

The Quantum Configuration utility configures, validates, and fixes the database synonyms for all configured sites every time the utility runs. If the site is already at the correct version and you make any changes to the site configuration, the utility validates and fixes the synonyms for the site automatically.

For information about warnings/errors in logs related to synonyms, see Troubleshooting and fixing synonyms.

Quantum Database Maintenance utility and synonyms

The Quantum Database Maintenance utility validates and fixes database synonyms before recreating Analytics views during the database upgrade process.

For more information about an upgrade that fails and has errors for synonyms, see Troubleshooting and fixing synonyms.

Limitations on user-created synonyms

If you create your own synonyms for different purposes, DO NOT use the suffix "_syn". The system validates any synonyms with suffix "_syn" during either the site configuration, using the Quantum Configuration utility, and/or the database upgrade, using the Quantum Database Maintenance utility. If you use the suffix "_syn", the system returns a descriptive error in the logs during either the configuration or database upgrade steps.

For more information, see Descriptive errors.

Sites with combined databases

When a site is configured to use the same Quantum and Analytics databases (i.e. Quantum database schema and Analytics database schema are in same physical database):

  • No synonyms are created in Analytics database by the Quantum Configuration or the Quantum Database Maintenance utility.
  • Synonyms are not required.
  • Analytics views access the Quantum tables directly as they are in the same schema.

Sites with separate databases

When a site is configured with the Quantum database schema and Analytics database schema in separate physical databases:

  • The Quantum Configuration utility or the Quantum Database Maintenance utility create synonyms for specific tables in the Analytics database.
  • The naming convention used by synonyms is the Quantum table name.
  • The Analytics views use synonyms to access Quantum tables, as required.
  • Valid synonyms must exist in order to use Quantum or the Logon with fail with an error.

Synonyms need fixing after certain actions

After certain actions, the system must validate and fix synonyms. Please run the Quantum Configuration utility after you have:

  • moved the Quantum and/or Analytics database to a different database server, or
  • replaced a Quantum and/or Analytics database for a preconfigured site without renaming the database.

Run either the Quantum Configuration utility or the Quantum Database Maintenance utility after you have:

  • renamed a Quantum and/or Analytics database for a preconfigured site.

If you see any errors that refer to synonyms during either process, see Troubleshooting and fixing synonyms for more information.

Troubleshooting and fixing synonyms

Read this section if you see errors relating to synonyms when running the Quantum Configuration utility or the Quantum Database Maintenance utility.

Descriptive errors

Descriptive errors appear in the logs near the end of the utility wizards. If the error is descriptive and mentions a specific manual action to do in the Analytics database to fix the problem, then this is a known scenario. The automated process cannot fix a problem in the database because the database objects are user-owned. Please take the actions described and ask for help from your consultant, as required.

General / non-descriptive errors

If the error is not descriptive enough, please talk to your consultant.

Fixing synonyms using SQL scripts

The consultants can use the scripts installed with Quantum in the "~\ Installed\Bin\Utils6.8\" directory to fix the synonyms. However, it is recommended to use the Quantum Configuration utility or the Quantum Database Maintenance utility to fix any problems, when possible.

Follow the instructions that are inline in the script files.

For an SQL Server:

  • qsn_analytics_fix_synonyms_separate_databases.sql

For Oracle:

  • qsn_analytics_fix_synonyms_separate_databases_step_1.ora
  • qsn_analytics_fix_synonyms_separate_databases_step_2.ora
