Maturity Profile

Use the Maturity Profile--Query page to present market valuation information in profile form, based upon transaction Maturity Dates.

NOTE: If imported market values are used, it may not always be possible to calculate a result. See Imported Market Values and the Maturity Profile query below.

Transaction types included

  • Currency Options - If the revaluation method is Imported, the deal value will be shown as zero when this inquiry is run. Quantum can only import an overall market value, not separate values for the components.
  • Interest Rate Swaps and Currency Swaps - If the revaluation method is Imported Separate, Quantum uses the Projected method to prevent requesting a Market Value it will not use (but needs for NPV calculations).
  • Money Market Deals and Forward Rate Agreements (FRAs) - If the revaluation method is Imported, Quantum uses the Projected method to prevent requesting a Market Value it will not use (but needs for NPV calculations).

Imported Market Values and the Maturity Profile query

If you use Imported Market Values for this query, it may not always be possible to calculate a result.

Market values are imported into Quantum at a deal level not the cash flow level required for this query, it may be necessary to edit the parameters to obtain a result.

NOTE: The As At date is the date of revaluation.

The following transaction types are processed by this query:

Currency Options

If a Currency Option has a revaluation method of Imported, its value will be shown as zero when this query is run. For Currency Options, Quantum can only import an overall market value, not separate values for the components of this type of deal.

Interest Rate Swaps

If an Interest Rate Swap has a revaluation method of Imported Separate, Quantum uses the Projected method to prevent Quantum from requesting a Market Value it will not use (but needs for NPV calculations).

Currency Swaps

If a Currency Swap has a revaluation method of Imported Separate, Quantum uses the Projected method to prevent Quantum from requesting a Market Value it will not use (but needs for NPV calculations).

Money Market deals

If a Money Market deal has a revaluation method of Imported, Quantum uses the Projected method to prevent Quantum from requesting a Market Value it will not use (but needs for NPV calculations).

FRAs

If an FRA deal has a revaluation method of Imported, Quantum uses the Projected method to prevent Quantum from requesting a Market Value it will not use (but needs for NPV calculations).
