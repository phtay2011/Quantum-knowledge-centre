Revaluation Currency (FX Accounting Rule)

(Available only when Revalue deals is selected AND Revaluation Method is FORWARD, FORWARD NPV, IMPORTED,NPV SPOT, or SPOT*.)

Select the currency used to perform revaluation calculations and posting revaluation gain/loss - one of:

  • BASE - to use the ledger's Base currency IF one leg of the deal is in that currency, OTHERWISE the Dominant currency (translation to the Base currency will be made before posting the gain/loss),
  • DOMINANT - to use the the Dominant currency, or
  • NON-BASE - to use the other leg's currency IF one leg of the deal is in the ledger's Base Currency, OTHERWISE the non-Dominant currency.

If Revaluation Method is IMPORTED:

  • the market value of the deal is always imported, in the currency defined here (see Revaluing an FX Deal using an imported market value),
  • if one FX Deal is linked to multiple Hedge Relationships, a different market value can be imported for each Hedge Relationship, and
  • the Book Value calculation depends on any Accruals Settings - see Book value, FX Deals with imported market values.

TIPS:

  • Forward points will be considered separately from Spot value when Accrue Forward Points is selected, depending on the associated Accrual Settings.
  • The Deal Revaluation page will indicate which currencies were used to perform individual deal revaluation calculations, and which were used to post the results of the revaluation.

*NOTE: This field is not displayed for other Revaluation Method settings. instead, each leg of the deal is revalued in its own currency, and for:

  • IMPORTED SEP - Quantum imports a separate market value for each leg and calculates a separate book value (the amount of the leg, net of its last revaluation amount) - see Book Value, FX Deals with Imported Market Values.
  • NPV SEPARATE or NPV SEPARATE ALT - see also FX Book Value, NPV Separate accounting.
