Adaptiv Queries

Queries are used to determine how you want the data you have previously defined in the Definition Elements to be displayed. Before you can create and run a query, you need to have defined at least the mandatory Definition elements for the query.

Queries may reside at the root level or within folders and must have a unique name. You can create a new query using the page menu. You can move a query into a folder by clicking the folder's link and then clicking Add in the grid menu.

Naming Conventions

To help users to locate queries, adopt a naming convention that reflects these categories - preferably for all queries and folders across all users.

For example, the names of queries can (and should) include an indication of the type, so that:

  • Base Valuation query names could include the abbreviation BaseVal or BV,
  • Cash Flow query names could include Cash or CF, and
  • Historical VaR query names could include HV, and so on.

This would also apply to folders that contain queries of one type only.

Workflow

The typical workflow for setting up and running an Adaptiv query is as follows.

  1. Set up the required Adaptiv Static Data (Cap/Floor Tenor Mapping, Price Index, and so on).
  2. Configure any required Adaptiv-specific settings (VaR Rates Definition, Calibration Parameters, and so on).
  3. Configure the Query, specifying Defaults and Properties, including all the query definitions required. These determine the data reported on and the full result set.
  4. Optionally configure one or more report display formats or Views (Table, Pivot, Chart, or Crystal view types, with filters, rows, and columns as required), enabling users to view the same result data in different ways without having to re-run the query.

    TIP: A Default view is always provided. This is a Table view of the full result set.

  5. Run Utilities as and when required (Market Data Archive, Model Calibration, Correlation Feed, and so on).
  6. Run the query, to produce the full result set.
  7. Download the results to an AAJ file for input to the the Adaptiv Analytics engine, which calculates and returns additional data to your query result set.
  8. Optionally apply a new View (for example a Pivot) specifically configured to display the additional data as required.
