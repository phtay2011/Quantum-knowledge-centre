Match Passes

When a Match pass type is run in Prior Day or Intraday Matching:

  1. The applicable Match Status is the detail-level status for the run's Matching type (Prior Day or Intraday).
  2. The system searches for bank flow and cash flow items that:
    • satisfy the defined matching criteria for the pass, which specify one or more bank flow fields, the corresponding cash flow fields to compare with in each case, and (for numeric values) the maximum allowed difference (which can be zero), and
    • have the applicable Match Status set to Unmatched.

      TIP: To include items previously matched at the Matched, Probable, and/or Possible levels (for all passes of the script), select the corresponding Reset items set to... option/s as required.

  3. When such items are found:
    • if necessary, adjustments are made as specified (see below), and
    • the applicable Match Status and a unique detail-level Match ID are assigned to those items.

TIP: Items matched by one pass will be ignored in subsequent passes.

Adjustments

For a Match pass, an adjustment is required when:

  • Match Status is Matched or Probable, AND
  • the Criteria grid contains an Amount entry with a tolerance amount or percentage.

To specify the adjustment settings:

  1. In Adjust Using, select the type of adjustment - one of:
    • ACTUAL CASHFLOW - to create a new Actual Cash Flow with Bank Value Date set to that of the matched bank flow, adjusted if necessary according to the Non-Business Days setting for the specified Instrument,
    • BANK ACCOUNT ADJUSTMENT - to create an adjusting Bank Account cash flow, or
    • BANKFLOW ADJUSTMENT - to add an adjusting bank flow (does not create any cash flow or G/L entries).
  2. Optionally select the Copy into Comments Field option to automatically copy the More Information field contents from the matched Bank Flow item into the Comments field of the created adjustment entry (whatever its type). Clear the option to exclude this information.
  3. If Adjust Using is ACTUAL CASHFLOW or BANK ACCOUNT ADJUSTMENT, optionally select the Also adjust for external cash flows option to prevent an automatic Detail-level adjustment entry if an external (imported) cash flow is involved. This prevents the Treasury position being affected by automatic adjustments made when matching external (non-Quantum) cash flows.

    Select this option to allow an automatic Detail-level adjustment entry whether or not an external cash flow is involved. The Treasury position will then be affected by any deal adjustments created when matching external (as well as Quantum) cash flows.

    NOTE: In most cases where external cash flows do not exactly match, you should notify the organization or system from which they were imported.

  4. If Adjust Using is ACTUAL CASHFLOW, select any of the Set From Bankflow Record options to automatically (when the Pass is run) copy those values - provided they are valid† - from the applicable bank flow into the corresponding field of the cash flow adjustment, superseding any value in the corresponding Adjustment Detail field.

    Clear an option to prevent such copying. The corresponding field in the cash flow adjustment will be set to the corresponding Adjustment Detail field value if populated or, if not, to the relevant default - for example, Entity would be set to the bank account's Default Transaction Entity.

    † NOTE: The bank flow Ticket Number, Instrument, and Entity fields must be populated by the Bank Statement Import process, using Data Manipulation commands (for example, as supplied via Transaction Rule Set records).

    If not all mandatory fields are valid for an adjustment, no adjustment will be created and no match will be made.

  5. In the Adjustment Detail area, optionally select static data values* for the adjustments generated when the script pass runs. The available data fields vary depending on the selected adjustment type (in Adjust Using), as indicated below.

    *IMPORTANT NOTE: The starred settings below (Ticket Number through Counterparty) will be superseded for an ACTUAL CASHFLOW adjustment if the corresponding Set From Bank Flow Record options are selected. (See also Note above.)

    • *Ticket Number (not applicable for Bankflow Adjustments) - optional transaction ticket information - for example, a reference for interfacing to external systems.
    • *Instrument (not applicable for Bank Account Adjustments, optional for Bankflow Adjustments, mandatory otherwise) - select an instrument the transaction is based on. Only instruments defined for the adjustment type are available.
    • *Entity (not applicable for Bank Account Adjustments) - the entity (legal party) on your side of the transaction.
    • *Counterparty (not applicable for Bankflow Adjustments) - the counterparty you are transacting with. The drop-down list includes all transaction-level Counterparties with Category set to Deal external or Deal internal in Counterparty Setup.
    • Dealer (not applicable for Bank Account Adjustments and Bankflow Adjustments, mandatory otherwise) - select a dealer; defaults to the current user if they are set up as a Dealer in User Maintenance.
    • Strategy (Actual or Projected Cash Flows only) - optionally select, for the adjusting deal, a Strategy group.
    • Comments (not applicable for Bankflow Adjustments) - optional text giving further information about the adjustment.
  6. Save your changes.
