Auto-Update a Query

Query results may be updated based on system events, time intervals, or execution of a scheduled report run.

Update Modes

Automatic updates are supported in either Auto-run or Update Live mode.

When to Use Each Update Mode

  • Use Update Live when possible because it is the most efficient approach.
  • Use Auto-run when:
    • Update Live is not available on the page, or
    • the report depends on data not covered by the change events.

Starting Automatic Updates

To run a query with automatic updates do the following:

  1. In the Run Query button's drop-down menu, select Auto-run.

    TIP: If the Expiry Strategy option is Keep a number of reports (Query Configure/Properties tab), the specified number of completed runs will be kept; otherwise, only the last two completed runs will be kept.

  2. In the Auto-run Query dialog box, select a run option - one of:
    • Auto-run after a pause (and enter the Pause time and units),
    • Auto-run immediately, or
    • Update live on change in the system.

      TIP: If any of these options have been used before on the page, the system shows the last three of these in the Run Query button's drop-down menu.

  3. Click OK.

NOTE: The Auto-runbadge appears next to the query name when it has been configured to auto-run, and when it is updating automatically (Live, Latest, or Auto-run), a Pause button appears next to the Run Query button.

Pausing and resuming Automatic Updates

When a query is updating automatically (Live, Latest, or Auto-run), a Pause button appears next to the Run Query button.

  • Click the Pause button to pause the automatic update. A Resume button is now displayed in its place.
  • Click Resume to have the system update the query automatically again.

Events that Stop Automatic Updates

Automatic updates for a page will stop when you:

  • navigate away from the page,
  • run the query any other way,
  • change the query,
  • create a new query, or
  • select a previous run.
