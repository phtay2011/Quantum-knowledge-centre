Structured Address enhancements

The following enhancements relate to addresses in this version.

Confirmations and EFTs

All address fields in the XML Data File used for Confirmations and EFTs now include structured address details. This is in addition to the existing unstructured address details, so this change does not impact existing EFTs and Confirmations.

The address field names mirror the names used in ISO20022 messages. To see some examples of these, look in the MT messages, where SWIFT Standards now enforce a somewhat structured format.

These XML Layout Files include examples of structured addresses:

  • SeedXMLLayout_MT300.xml
  • SeedXMLLayout_MT300_Misys.xml
  • SeedXMLLayout_MT300_MT304_CLS.xml
  • SeedXMLLayout_MT304.xml
  • SeedXMLLayout_MT304_Misys.xml

TIP: If you don't want the extra lines in the structured addresses, remove separator = "line_break".

Deleting existing structured addresses upon deactivation

You can now optionally delete all existing structured address records in the database when deactivating Configuration > Address Structure. This allows you to start fresh with a different address structure.

Previously, any migrated data remained in the database despite structured addresses not being active.

Static Data Import/Export

The Static Data Transfer Spreadsheet's Upload and Download functions now include the structured address field details in addition to the unstructured address field details for the following static data:

  • Entity
  • Counterparty (Street Address and Postal Address)
  • Bank Account
  • Counterparty Bank Account (Postal Address and Intermediary Bank Address)
  • Counterparty Stock Account

The structured address fields will be included when you generate QXT file details for selected spreadsheet rows (using CTRL+Shift+E). Use the Static Data Import Status utility to upload structured addresses that are in the QXT file.

See Address Structure and Using the Static Data Transfer Spreadsheet
