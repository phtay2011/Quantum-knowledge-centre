External Deal Import CSV file format

The first header line of the CSV file contains the column name properties (separated by commas) to be set on each deal.

The second and subsequent detail lines contain the deal information that is to be imported into Analytics.

Define a special column property name changetype to state whether a deal is being added, updated, or deleted.

Example

Line 1: dealno,Changetype,transtype,instrument,borrowlend,ccyid,amount....etc

Line 2: Ext001,Added,MM,11AM Borrowing, Borow,AUD,20000000....etc

The CSV file can contain column name properties for all the deal field names in the Analytics database. Lookup field values and codes are also accepted input.

MANDATORY FIELDS:

  • transtype (Transaction Type).
  • Any mandatory fields for the particular transaction type.

TIP: Use either of the following tools to view the full list of field names across all transaction types.

  • Analytics Database Maintenance utility (AGRDBMnt.exe). This utility is located in C:\Program Files (x86)\SunGard\AvantGard Quantum\Bin\Risk\DBTools on a default installation.
  • Analytics Deal Viewer (AGRDealView.exe). This utility is located in C:\Program Files (x86)\SunGard\AvantGard Quantum\Bin\Risk\Tools on a default installation.

NOTE For Swap transactions, prefix the fields on the deal table with either 'pay' or 'receive'.

Use the following Header rows* in Deal Import csv files.

* Only transtype is mandatory and additional fields may be specified as mentioned above.

dealno

Required to link information between the deal side and cash flow data.

  • If the specified dealno value is already in the Analytics database, the deal is loaded and amended.
  • EXT is appended to the specified dealno value to distinguish external deals from Quantum deals.

Alpha numeric, up to 20 characters.

transtype

Determines the Analytics Pricing module the deal and its cash flows will be saved under.

The available options are:
Bond, BO, IO (Cap\Floor), CF, CS, CO, Discount, FR, FRN, FU, FXForward, FXParForward, IS, MM, SO

changetype

Values: Added, Modified, Deleted.

Mandatory if deals are being deleted (if so, dealno must also be populated).

If the column parameter is not defined, it will default to:

  • Added - if the deal does not exist or there is no dealno specified, or
  • Modified - if the deal already exists in the database.

linktype

(field refers to FXForward transaction types only)

If this is Back to Back or Swap, the system assumes that the next line in the csv file will be the other side of the deal.

collar

( Currency Option, Cap\Floor, and Swaption transaction types only)

If this is True or Yes, the system assumes that the next line in the csv file will be the other side of a collar.

sourcereference

Used for grouping of deal or cash flow items, for reporting purposes and in Deal Remover (to remove only deals with the specified sourcereference).

analysis codes

Enables analysis codes to be linked to each deal and/or cash flow.

ID or name of columns

The deal save process allows both instrument and instrumentid to default the instrument type to a deal (this is true for all fields that end in ID).
