Average Interest Rate Function in Analytics

(First introduced in Release 6.4.1 SP, March 2017.)

You can now use the new AverageInterestRate function in a new or existing Calculated Column in any type of Analytics query.

This function can be used to calculate a day-weighted average interest rate across a period (Arithmetic method) or a compounded return - for example, the return on EONIA for a period (Geometric method). A compounded return is very useful for performance measurement purposes - for example, to benchmark a portfolio of short-term assets or liabilities.

The function calculates the average of a given interest rate over a specified period, and has the following syntax:

AverageInterestRate(ccy, rate_name, start_date, end_date, as_at_date [, term, term_type [, method ] ])


... where:

  • ccy is the currency linked to the Yield Rate to be averaged;
  • rate_name is a Yield Rate name which, together with ccy, determines the Yield Rate to be averaged;
  • start_date gives the start of the averaging period (the first rate record used in the calculation will have a Rate Date equal to or immediately preceding this date);
  • end_date gives the end of the averaging period (the last rate record used in the calculation will have a Rate Date equal to or immediately preceding this date);
  • as_at_date is the date used to select the appropriate data images, enabling users to recreate results from historical runs; should be consistent with the same argument within other Calculated Column functions; and
  • term is:
    • zero or a positive integer that together with term_type gives the point on the Yield Rate curve for selecting the rate to be averaged*,
    • optional (see term_type below);

      *TIP: The applicable mid rate is used. This is the average of the Bid and Offer values in the Rates grid row with the matching Term and Term Type settings. If there is no matching row, an error will be reported.

  • term_type is:
    • Overnight (the default), Tom Next, or a time unit (Days, Weeks, Months, or Years) that together with term gives the point on the Yield Rate curve for selecting the rate to be averaged,
    • required if term is present,
    • not required (and must be absent) if term is absent;
  • method determines the averaging method used and is:
    • G or Geometric (the default) for the geometric mean (for compounding rates), or
    • A or Arithmetic for the arithmetic mean (the sum of the rates divided by the number of rates),
    • optional if term and term_type are present,
    • not required (and must be absent) if term and term_type are absent.

Calculation Notes

  • Where Bid and Offer rates are available for a particular Term and Term Type, Ri will be the mid rate or average of the two values
  • Where a rate record is missing for a date within the range (for example due to a non-business day), the preceding day's rate will be used. Where there is no available preceding rate, the first available rate will be back-filled.
  • The Day Count for the averaging calculation is taken from the Rate definition.

Example 1

AverageInterestRate("EUR","EURIBOR",QueryStartDate,QueryEndDate,QueryAsAtDate)

Gives the geometric (compounded) average EURIBOR Yield rate for the EUR currency, using the OVERNIGHT rate curve points. The rates will be taken between the query's specified Start Date and End Date, and the query's As At Date applies.

NOTE: The term, term_type, and method parameter values have not been specified in this example, so that the Overnight and Geometric default settings apply.

Example 2

AverageInterestRate ("AUD","BBSW",QueryStartDate,QueryEndDate,QueryAsAtDate,
"3","MONTHS","A")

Gives the arithmetic mean BBSW Yield rate for the AUD currency, using the 3 MONTHS rate curve points. The rates will be taken between the query's specified Start Date and End Date, and the query's As At Date applies.
