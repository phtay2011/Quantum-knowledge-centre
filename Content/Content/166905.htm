Deals Eligible for WAC Processing

Money Market deals meeting the following criteria can be processed by the Weighted Average Closeout (WAC) Processor.

  • Formula Type (from the underlying Instrument) is one of:
    • BOND PERIODIC
    • BOND ROLLING (see BOND ROLLING Restrictions (WAC) for more details)
    • CERTIFICATE DEPOSIT PERIODIC
    • DISCOUNT PREPAID
    • FRN-CGL PERIODIC (see FRN-CGL Ratesets and the WAC Processor for more details)
    • SIMPLE DISCOUNT PREPAID
    • ZERO COUPON DISC ON MATURITY
  • The deal has not been customized at level two - see Customized deals and the WAC Processor for more information
  • Accrual Method in the Accounting Rule (even if selecting not to accrue) is one of:
    • AMORTIZED COST
    • STRAIGHT LINE
    • YIELD CUM
    • YIELD EX
BOND ROLLING Deal Restrictions (WAC)

For WAC Processing, the following restrictions apply to deals using the BOND-ROLLING formula:

  • Rolling the deal is not supported.

    CAUTION: Currently, the system will not prevent you from rolling the WAC deal, nor any of the underlying deals, but the WAC Processor cannot process them.

  • If you add a new deal to an existing portfolio, its Previous Coupon Date defaults to the Settlement Date. You must ensure that it is is set to the Previous Coupon Date on all existing BOND ROLLING deals in that portfolio.
FRN-CGL Rate Sets and WAC Processing

Ratesets of FRN-CGL deals to be processed by the Weighted Average Closeout (WAC) Processor will need to be performed on both the actual deal and the notional deal.

On the actual deal, the rate sets are required so that the interest cash flows are correct.

On the notional deal, the rate sets are required so that the accruals for the interest cash flows are correct.

NOTES:

  • Rate sets performed on the actual deal will be transferred to the notional deal when the WAC Processor is run. These will overwrite rate sets performed on the notional deal if they are not identical.
  • If both actual and notional deals are rateset manually using Automatic Rates Fill Down and they are rate set with different rates, the WAC Processor will only transfer the Actioned ratesets and will not update later rate set periods with the new rate.
  • Rate set performed on the notional deal will not be transferred to new subsequent notional deals. The new notional deal will take its rate sets from the actual deal being processed.
Customized Deals and WAC Processing

With respect to the use of the Weighted Average Closeout (WAC) Processor program, there are two levels of customization of deals in Quantum.

Level One

Changing the Settlement Date of cash flows, or the insertion of Adhoc Interest or Adhoc Amount values, are considered to be level one customizations. These changes will not exclude the changed deal from the portfolio being processed by the WAC Processor, however, these customizations will not be transfered to the notional deal.

NOTES:

  • Adhoc Interest customizations will not be accounted for properly by the WAC Processor, therefore, it is recommended that you do not process deals that have changes made to the Adhoc Interest.
  • Adhoc Amount customizations can be accounted for on the actual deal only. The changes will not be transferred through the portfolio deal. The Posting Rule settings that relate to Adhoc Amounts can be set to post to the required G/L account on the actual deal rather than to the general G/L account and the Adhoc Amounts on the notional deal can be set to post to the general G/L account. Accruals will need to be run over the actual deals if accruals on Adhoc Amounts is required.

Level Two

Level two customizations are changes to an actual deal that change its value. For example, changing its deal map items or its price. These changes will result in the deal being excluded from the portfolio being processed by the WAC Processor.

Customization of notional deals is not supported.
