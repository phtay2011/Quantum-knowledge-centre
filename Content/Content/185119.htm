Scheduling Query Runs for Multiple Users

Queries can be scheduled to run at a certain time interval so that they are ready for multiple users to view the result. The query does not run at the time of retrieval, it just retrieves the latest run of the query and displays that. Retrieval of the result is therefore quick and independent of the time it takes to run the query.

To open the latest run for a scheduled query, choose Update to the latest scheduled run from the Auto-run Query dialog box.

The latest run will be displayed and when a new scheduled run is completed by the system, it will update automatically.

NOTES:

  • The report run in Scheduler must be marked as Run for Latest.
    Also, to reduce the load on the system, check Suspend execution when result not viewed which suspends the execution of these runs for latest when the result is not viewed.
    For example, a query which runs every 5 minutes when the results are being viewed could be paused for 60 minutes between runs when not being viewed using this setting. When a result is viewed again by any user, the frequency will return to 5 minutes. The report is therefore never older than 60 minutes and will refresh in 5 minutes when viewed.
  • A latest report cannot be selected from the Previous Runs dialog.
  • Update to the latest scheduled run is available for Analytics when resource security is not enabled.
  • Latest runs in reporting are filtered by resource security on retrieval.
