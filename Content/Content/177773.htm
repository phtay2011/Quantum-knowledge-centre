Interest Rate Options Accounting Rules

Use the Interest Rate Options tab to create and maintain accounting rules for Interest Rate Options (IROs).

There must be at least one Accounting Rule entry that matches each type of these deals that your organization uses. If more than one, the most specific matching rule entry takes precedence. For example, entries for Instrument groups take precedence over ALL Instrument Name entries (labeled For any ... in the rules grid). Individual instruments that are members of a group will apply settings defined for that instrument before those for the group, before those for an ALL entry.

You can add, edit, and delete specific rule entries using the tab's grid menu, as follows.

NOTE: You cannot add, edit, nor delete inherited entries.

To add a new rule entry

  1. In the Interest Rate Options tab's grid menu, select New (or select an entry row and in the grid menu, select Copy).
  2. In the dialog box under Matching Criteria select:
    • one or more Interest Rate Option Types (Simple, Bond Option European, Bond Option American, and/or Swaption European),
    • one or more Instrument Names or ALL, and
    • any other customized criteria that may apply to this transaction type.

      NOTE: You cannot specify a combination of criteria already covered by an existing entry.

  3. Under Contingent Accounting Settings optionally select Apply contingent accounting to post contingent accounting entries.

    NOTE: No contingent entries will be created for Swaptions.

  4. Under Accrual Settings, optionally select:
    • Accrue Option Premium - to take the premium to the Balance Sheet at deal save and post it to Profit and Loss when Accrual Processing is run, and/or...
    • Accrue Option Settlement - to post accruals on settlement of the option.
  5. If you selected Accrue Option Premium in Step 4:
    • select the Premium Translation date: POSTING DATE CCY TRANSLATION or CASHFLOW DATE CCY TRANSLATION, and...
    • select the Premium Accrual Period: NOTIONAL, EXERCISE, or FULL.
  6. If you selected Accrue Option Settlement in Step 4, select the Settlement Translation date: POSTING DATE CCY TRANSLATION or CASHFLOW DATE CCY TRANSLATION.
  7. Under Revaluation Settings, select Revalue deals* if you want to create revaluation postings, and as required, select:
    • Revaluation Method (as available for the selected Interest Rate Option Type),
    • Interest Rate* and Interest Rate Type*, and...
    • Volatility Rate Name* and Volatility Rate Type*.

      NOTE:* Mandatory settings when Hedge Accounting applies. They are the defaults on the Linked Deals tab of any Hedge Relationship based on an IRO using this Accounting Rule. The system can then revalue the IRO and perform effectiveness calculations.

  8. Click Save. The new rule entry is added to the grid.

To edit an existing rule entry

  1. Click an entry value in the grid (or select the entry and in the grid menu, select Edit).
  2. In the Edit window, edit the settings as required (see above), and click OK.
  3. Save your changes.

To delete an existing entry

  1. Select the entry in the grid and in the grid menu, select Delete.
  2. In the dialog box, click OK to confirm.
