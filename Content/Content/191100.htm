Changing the Current Alerts Definition

Use the following procedures to maintain an Alerts Definition record.

To add a new Alert object to the current Definition:

  1. In the Alerts grid menu, select New (or click the New button above the grid).

    The New Alerts dialog box appears.

  2. Enter a new Name for this Alert object and select a Column.
  3. Click Add Recipient and enter a recipient Name and E-Mail Address. Repeat to add more recipients (individuals nominated to receive Alert Messages whenever an Alert condition is met in a query result set).
  4. Click OK.

    The dialog box closes. Your new Alert object is added to the Alerts grid.

  5. In the Alerts grid, select your new Alert row.

    The Messages grid area opens, with the grid initially showing no messages for your new Alert object.

  6. Click the New button above the Messages grid to add messages to your new Alert object. See the next procedure for details.
  7. Click Save.

To add Messages to an existing Alert object:

Alert Rules are made up of one or many Conditions which are made up of one or many criteria.

  1. In the Alerts grid, select the row you want to add messages to.
  2. Click the New button above the Messages grid.

    The New Message dialog box appears.

  3. Enter a new Name for the message and the Resend Timeout in minutes. This is the number of minutes after which the Alert Message will be resent, even if the column has not changed. For example, to have an Alert send one message daily, set Resend Timeout to 1440 minutes (that is, 24 hours).

    CAUTION: Since multiple emails may be generated if the Resend Timeout field is left blank, it is strongly advisable to always specify a Resend Timeout value greater than zero.

  4. Click the Add Message link to open the Expression Builder window; type in your message text and use the + button and Browse window to select any data items to include, then click Done.

    This is an example of the condition in Expression Builder.

    COUNTERPARTY LIMIT BREACH

    Counterparty = {Limit Details/Char)
    Limit = {Limit Amount/Money}
    Utilization = {SumUtilization/Money}
    Available = {Available/Money}

    Where Utilization and Available are two calculated columns set up as follows:

    • Utilization = Market Value
    • Available = Limit Amount - Abs(SubTot(Limit Details , Utilization))

    An email like the following one will be sent when the criteria are met.

    NOTE: Unless all column types are Char (character), you must select the suffix for each field with its respective column type - for example, Money. This controls the formatting of the Alert message text.
    When triggered, the columns specified within the Message are populated with actual values from the returned result set.

  5. Click Add Criterion and enter a Criteria Name, Column, Column Type, Operator, Value, and Tolerance. Repeat to add more criteria.

    The Tolerance field determines the frequency with which subsequent Alert messages will be emailed to recipients. This is based on the Tolerance value specified in the original Alert. For example, when Tolerance is set to 10% an Alert Message will be resent if the Available amount varies by at least 10% below zero.

    Column Type defines the segment of data retrieved from the selected Column (for example, if Column = Currency and Column Type = Money, the amount along with the currency code is retrieved ("USD 10,000"); if Column = Currency and Column Type = Char, only the currency code is retrieved ("USD").

    Example:

    Column:

    Currency*

    Column Type:

    Money

    Operator:

    < (less than)

    Value:

    0

    Tolerance:

    10%

    *NOTE: The actual Column Name and Type will vary according to the column type on which the Alert is based.

    This condition triggers an Alert Message whenever the Available amount is less than zero. Additionally, the Alert will be resent to the recipient if the Available amount varies by at least 10% from the previous returned value.

  6. Click OK.

    The dialog box closes. The Messages grid lists your added Criteria entries.

  7. Click Save.

To edit an existing Alert object's settings:

  1. In the Alerts grid, select the row you want to edit.
  2. In the grid menu, select Edit (or click on a linked value in the row).

    The Edit Alerts dialog box appears.

  3. Edit the required fields and click OK.

To edit a Message for an existing Alert object:

  1. In the Alerts grid, select the row whose message you want to edit.
  2. In the Messages grid, select the message and in the Messages grid menu select Edit (or click on a linked value in the row).
  3. In the Edit Message dialog box:
    • Optionally edit the Name for the message and the Resend Timeout in minutes.

      CAUTION: Since multiple emails may be generated if the Resend Timeout field is left blank, it is strongly advisable to always specify a Resend Timeout value greater than zero.

    • To edit the message content, click the Message content link to open the Expression Builder window; edit the message text as required, using the keyboard and the + button's Browse window to select any new data items to include, then click Done.
    • To edit criteria, directly enter or select values in the Criteria Name, Column, Column Type, Operator, Value, and Tolerance fields in the relevant row/s. Repeat to add more.
    • To delete criteria, click the x button at the right of the relevant row.
  4. Click OK.

    The dialog box closes. The Messages grid lists your edited Criteria entries.

To delete an existing Alert object:

  1. In the Alerts grid, select the row you want to delete.
  2. In the grid menu, select Delete.

    The Confirm Delete dialog box appears.

  3. Click OK to confirm.

    The Alert row is removed from the Alerts grid.
