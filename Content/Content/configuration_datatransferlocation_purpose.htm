Purpose

Select one of the following incoming or outgoing data types.

  • Incoming data*:
    • BankImportation - Bank Statement items (account balances and transaction flows).
    • Confirmation - Confirmation Matching messages returned by the external matching service.
    • DealImportation - deal details
    • StaticImportation - Static Data
    • WorkflowIn (only when Location Type is FILE or QUEUE, at least one Parser is available, and the Process workflows Operational Setting is selected) - an input text file that will be parsed by the specified Parser (if any are installed) into a Data Source.
  • Outgoing data:
    • ConfirmationDelivery - Confirmation messages output from the Confirmations Processing page.
    • Eft - EFT messages output from the Processing > EFTs page or the Utility > EFTs Reprocessing page, where the External Method Source for the specified Payment method has Type Export set to Create File) or Queue.
    • GLInterface - General Ledger entries exported from the G/L Interface page, including FILE, QUEUE, and WEB API data location types.
    • Scheduler (only when Location Type is QUEUE) - required for Scheduler tasks wanting to use the Message Queue defined in this DTL record to output Quantum data.
    • WorkflowOut - an output text file formatted as defined in the specified Workflow Template.
  • Incoming/Outgoing data:

    • WorkflowIn/Out (available when Location Type is WEB API) - handles receiving requests and sending responses to external Web API services.

    • WorkflowOut/In (available when Location Type is WEB API) - handles sending requests and receiving responses from external Web API services.

TIP: An Incoming or Outgoing badge displays at the top right to show the direction of your selected data type.
