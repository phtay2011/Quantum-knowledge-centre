Swaption Instruments in Quantum

Swaptions (swap options) are used to create Interest Rate Options on forward-starting swaps. A Swaption gives the holder the right - but not the obligation - to enter into a particular Interest Rate Swap deal on a specified future date, and at a specified swap rate. For more details, see Swaption Deals in Quantum.

Default values for Swaption deals are provided when an Interest Rate Option deal uses an instrument with the following settings.

Details area

Option Type

Either:

  • Payer - for an option on a valid swap giving the buyer the right to pay a fixed interest rate and receive a floating interest rate, or
  • Receiver - for an option on a valid swap giving the buyer the right to receive a fixed interest rate and pay a floating interest rate.

Exercise Style

Read-only and set to EUROPEAN for Swaption instruments (when Option Type is Payer or Receiver).

TIP: European Interest Rate Options are exercised or lapsed on the option expiry date.

Settlement Type

The default method of settlement; either:

  • PHYSICAL - where settlement is by entry into a Swap deal, or
  • CASH - where settlement is by cash payment.

NOTE: You must define a Settlement Type before you can exercise a deal using this instrument.

Settlement Instrument (Swap)

(Mandatory for Swaptions.)

This instrument provides default settings for the Interest Rate Swap deal that is created when the Swaption deal is exercised (and Settlement Type is PHYSICAL). This setting is required to ensure the Swaption is valued correctly (even if Settlement Type is CASH).

Only Interest Rate Swap instruments valid for Swaptions will be available for selection. See Swaption Instruments in Quantum.

NOTES: For analysis and valuation purposes, ensure the underlying Swap instrument has a Revaluation Method set in the applicable Accounting Rule record (on the Interest Rate Swaps tab).

Premium Paid In

This setting gives the default Premium Date value for Interest Rate Option deals based on this instrument. Select:

  • ADVANCE - to use the Deal Date, or
  • ARREARS - to use the Exercise Date.

Days To Premium

(Available when Premium Paid In is set to ADVANCE.)

The number of days (business or calendar) between the Deal Date and the Premium cash flow date; determines the Premium Date for the deal. Enter B (for business) or C (for calendar) followed by the number of those days, from 0 to 7.

Show broker

Select this option to include the Broker field when entering a deal based on this instrument.

That Broker field can then be used to record a third party or intermediary Counterparty, where one is used to transact the deal.

Broker is required

(Available only when Show Broker is selected.)

Select this option to require users to select a Broker before saving deals based on this instrument.

Defaults area

The same fields apply to all Option types, and all are optional.

See Defaults Area Settings (all Interest Rate Option Types).
