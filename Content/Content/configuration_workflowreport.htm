Workflow Report page

Use this page to create and maintain your site-specific reporting pages, which can combine data from multiple Quantum Six data sources, and which can be placed anywhere in the Quantum menu structure.

Data items

  • Name - enter a unique module name for the target page; used in Module Security.
  • Title - the target page title; used also on browser tabs and in the site menu.
  • Url Name - the name to be used for the controller or last part of the target page URL. Valid characters are: a-z, A-Z, 0-9; with hyphen (-) separators (if any).
  • Menu Area - the area in the site menu where the target page title will appear (Analytics, Processing, Reporting, and so on).
  • Menu Group - the menu group under which the target page title will appear; also used in Module Security. If the group already exists, the variant is added to it; if not, a new group is created.

The Workflow Report screen consists of four tabs:

  • Report tab - grids with sources that enable configuring datasources for your reports.
  • Actions tab - additional actions made available in the target report's results view (as quick menu items or toolbar buttons).
  • Structures tab - defined structures that enable data in spreadsheet form to be included as Data Sources. The target report user can edit such data directly or via a spreadsheet application (as with Bulk Replace and Insert for Deal Map Items).
  • Sources tab - one or more entry rows each giving a data source used for the report, with settings to reduce the target report's result set, columns to be available in the target report, and optional mapping of fields to another source (if any).

Module Security

To quickly grant (or remove) all permissions for the target page and current User (if not exempt), select the relevant Add or Remove item in the page menu (under MODULE SECURITY). For other users, use the Module Security tab on the Users page as normal (the page will be labeled using the specified Menu Group and Name values).

TIP: To use this page itself, Workflow Report module permissions apply.
