FX Posting, Balance Sheet tab

  • Actual FX Position - This entry is the contra of the cash entry, and is posted on the deal maturity date, debiting the Balance Sheet account for the sale of a currency, crediting for purchase.
  • Settlement Payable -
  • Settlement Receivable -
  • Forward Points Payable - See Forward Points Receivable (B/S).
  • Forward Points Receivable - To use this default, in Accounting Rules for the applicable condition, select the Accrue forward points setting.

    The system determines which currency the forward points are accrued in from the Posting currency field in Accounting Rules. If it is set to:

    • Base - forward points are accrued in either the base or dominant currency, or if...
    • Non base - this is the non-dominant currency.

    If the Accrue forward points field in Accounting Rules is set to:

    • Posting date CCY Translation - the system posts the entry on the Value Date of the deal, and contras it by a bank account entry; or if...
    • Cashflow date CCY Translation - the system posts the entry on the deal date (or on the deal Spot Date where the Start Date is set to Spot), and contras it by a bank account entry

  • Base Reval Gain: Realized - This account is debited when a P&L Gain is calculated, and credited with a decrease in a P&L Gain, the contra account being Base Gain: Realized (P/L).

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component.

  • Base Reval Gain: Unrealized - This account is debited when a P&L Gain is calculated, and credited with a decrease in a P&L Gain, the contra account being Base Gain: Unrealized (P/L).

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component.

  • Base Reval Loss: Realized - See Base Reval Gain: Realized above (reverse the direction of the accounting entries where a P&L Loss is calculated).
  • Base Reval Loss: Unrealized - See Base Reval Gain: Unrealized above (reverse the direction of the accounting entries where a P&L Loss is calculated).
  • Revaluation Gain: Realized - This account is debited when a P&L Gain is calculated, and credited with a decrease in a P&L Gain, the contra account being Gain: Realized (P/L).

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component.

  • Revaluation Gain: Unrealized - This account is debited when a P&L Gain is calculated, and credited with a decrease in a P&L Gain, the contra account being Gain: Unrealized (P/L).

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component.

  • Revaluation Loss: Realized - See Revaluation Gain: Realized above (reverse the direction of the accounting entries where a P&L Loss is calculated).
  • Revaluation Loss: Unrealized - See Revaluation Gain: Unrealized above (reverse the direction of the accounting entries where a P&L Loss is calculated).
  • Reval Separate Buy Gain: Real - This account is debited when a P&L Gain is calculated on the Buy leg of the deal, and credited with a decrease in a P&L Gain, the contra account being Separate Buy Gain: Realized (P/L)

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component.

    NOTE: The system posts the entries in the currency of the leg of the deal the revaluation relates to.

  • Reval Separate Buy Gain: Unreal - See Reval Separate Buy Gain: Real contra account being Separate Buy Gain: Unrealized (P/L)).
  • Reval Separate Buy Loss: Real - This account is credited when a P&L Loss is calculated on the Buy leg of the deal, and debited with a decrease in a P&L Loss, the contra account being Separate Buy Loss: Realized (P/L)

    See Relationship between Accounting Rules & G/L Components for the Accounting Rule settings that use this G/L Component

    NOTE: The system posts the entries in the currency of the leg of the deal the revaluation relates to.

  • Reval Separate Buy Loss: Unreal - See Reval Separate Buy Loss: Real above (the contra account being Separate Buy Loss: Unrealized (P/L)).
  • Reval Separate Sell Gain: Real - This relates to the revaluation of the Sell leg of the deal. See Reval Separate Buy Gain: Unreal above (the contra account being Separate Sell Gain: Realized (P/L)).
  • Reval Separate Sell Gain: Unreal - This relates to the revaluation of the Sell leg of the deal. See Reval Separate Buy Gain: Real above (the contra account being Separate Sell Gain: Unrealized (P/L)).
  • Reval Separate Sell Loss: Real - This relates to the revaluation of the Sell leg of the deal. See Reval Separate Buy Loss: Real above (the contra account being Separate Sell Loss: Realized (P/L)).
  • Reval Separate Sell Loss: Unreal - This relates to the revaluation of the Sell leg of the deal. See Reval Separate Buy Loss: Real above (the contra account being Separate Sell Loss: Unrealized (P/L)).
