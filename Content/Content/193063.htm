Stock Flows for Security deals with Indexed Formulas

You can now enable the creation of stock flows for Quantum Security deals based on instruments with Formula Timing set to:

  • INDEXED ANNUITY ACTUAL END DATE,
  • INDEXED BOND PERIODIC, or
  • INDEXED GILT PERIODIC.

Implementation

On the Quantum Security Instrument page, the Allow stockflows option is now available when Formula Timing is one of the above INDEXED settings. If you select the option and save the instrument, Allow stockflows will become read-only as soon as the next deal using that instrument is saved.

On the Security Dealing page, when the selected Instrument's Formula Timing is one of the above and its Allow stockflows option is selected and the deal is saved, two stock flows will be created based on the FACE VALUE and REPAY PRINCIPAL events (if multiple REPAY PRINCIPAL events, the last one is used). The stock flows themselves are named STOCK FACE VALUE and STOCK REPAY FACE VALUE.

Repo Deals where the underlying Security instrument has Allow stockflows selected also support stock flows.

NOTE:

  • Closeouts adjust the REPAY FACE VALUE flow.
  • The sum of FACE VALUE and REPAY FACE VALUE flows is zero.
  • WACO processing is not affected.
  • Whether or not stock flows are linked to cash flows is determined by the applicable Stock Account Defaulting Rule.

Exclusions

Stock flows are not generated for Unwind and Indexation events.

History

Previously, stock flows were not supported for any INDEXED formula.
