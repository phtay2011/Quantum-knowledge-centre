Round Up and Down for FX Deals in Quantum Analytics

Quantum supports a Rounding setting on the Currency page which allows a user to define whether an amount for a given currency should be Round(5/4), Always Round Up or Always Round Down.

This setting is now available in the Currency page of Analytics. You can choose between the following options:

  • Use 5/4 Rounding - Rounding depends on the fractional amount:

    • Up to and including 4 is rounded downwards

    • 5 or more is rounded upwards

  • Always Round Down - Any fractional amount is always rounded downwards to the next currency unit.

  • Always Round Up - Any fractional amount is always rounded upwards to the next currency unit.

CAUTION: This feature requires a change in the static replication XML. Contact your FIS Consultant to upgrade the XML before using this feature.

In addition to the above, to provide a more flexible rounding on other calculated amounts in Quantum Analytics reporting, we have introduced two new calculated functions to the Query calculated columns RoundDown and RoundUp.

RoundDown

Description

Rounds a number downwards to the next currency unit.

Format

RoundDown(number,num_digits)

Arguments

  • number - The number or column to be rounded.

  • num_digits - The number of decimal places to round do.

Example

RoundDown(Interest Rate,6)

If Interest Rate is:

  • 5.0225000000 - the result is 5.022500

  • 5.76230749308 - the result is 5.762307, and so on

RoundUp

Description

Rounds a number upwards to the next currency unit.

Format

RoundUp(number,num_digits)

Arguments

  • number - The number or column to be rounded.

  • num_digits - The number of decimal places to round do.

Example

RoundUp(Interest Rate,6)

If Interest Rate is:

  • 5.0225000000 - the result is 5.022500

  • 5.76230749308 - the result is 5.762308, and so on
