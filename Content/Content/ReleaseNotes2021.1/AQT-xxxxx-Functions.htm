Expression builder - New and changed functions

The following are new functions used in Expression Builder.

DealLocation

Description

Returns the location of the deal.

Format

DealLocation(dealNo)

Arguments

dealNo – It can be a number.

NOTE: Argument must be an integer.

Example

DealLoation(17025)

The result populates locations Singapore and United States of the given deal.

Mapping

Description

Converts value using the given mapping

Format

Map(mapping,value)

Arguments

mapping – an entry in the mapping table header section.

value – a value from the mapping values.

Example

Map('Regis Day Basis','30E+/360')

Returns 30/360

MatchOurConfirmations

Description

Returns message id of our confirmation that best matches the message, using given criteria function

Format

MatchOurConfirmations(message, criteria)

Arguments

message – a valid their confirmation message

criteria – a user defined function that has only two input parameters (“Our” and “Their”)

Example

Created user defined function as “Match” with expression as –

SWIFTTagMatch([Inputs.Our], [Inputs.Their], "22A,36,30T,30V")

MatchOurConfirmation([Message],"Match")
Additional notes

  • This function performs matching on data available in Confirmation for Matching Table.

  • Function will not compare messages that are already processed or proposed.

  • If matching of messages is successful, then it will return the message ID. Otherwise, it returns zero.

Example

SWIFTTag

Description

Returns the value of the first occurrence of a tag within the SWIFT message or sequence when specified.

Format

SWIFTTag (message, tag, sequence)

Arguments

message – valid swift message

tag – swift tag

sequence (optional) – sequence name

Allowed sequence names are: "A", "B", "C", "D", "E", "F", "G", "H", "I", "J", "K", "L", "M", "N", "O", "P", "Q", "R", "S”.

Subsequences are also allowed (for example, B1 and B2).

Example

message:

{1:F01BPGBGB2LAXXX0000000000 {2:I320BKCHGB2LXXXXN}{4:

:20:FX56-60

:25:2100 0000 0000

:30:210121

:21:FX56-60

:32B:USD1222,

:52A:ABNBIC

-}

SWIFTTag (message, "20", "A")

The result is FX56-60

Additional Notes

  • Tags and sequences are case sensitive.
  • If a sequence is not specified, then the first occurrence of a tag's value is returned. Otherwise, the function returns the tag value from the specified sequence.
  • If there is no matching SWIFT tag or sequence found, then an empty string is returned.
  • To learn more about tags and in which sequence they are present, read the Format Specification section defined for each Message Type

SWIFTTagList

Description

Returns a list of values of all occurrences of tag within the SWIFT message or sequence when specified.

Format

SWIFTTagList (message, tag, sequence)

Arguments

message – valid swift message

sequence (optional) – sequence name

Allowed sequence names are: "A", "B", "C", "D", "E", "F", "G", "H", "I", "J", "K", "L", "M", "N", "O", "P", "Q", "R", "S”.

Subsequences are also allowed (for example, B1 and B2).

TargetAddressee

Description

Returns the unique addressee from Target Directory for a given value and date.

Format

TargetAddressee(value, date)

Arguments

value - specify the input field or string of the Target Directory Participant BIC

date - specify the date or input date field of the cashflow date that will map to 'Valid from' and 'Valid till' date range in Target Directory

Example

TargetAddressee([Counterparty.Swift BIC Code], [Date])

On the Target Directory page, if the matching Addressee = AACSDE33XXX, the result is: "AACSDE33XXX"

Translate

Description

Returns text with all characters replaced by the corresponding character from translations

Format

Translate(text,characters,translations)

Arguments

text - the text to translate characters in

characters - the characters that should be replaced

translations - the replacement characters in the same order

Example1

translate("This is a @string #with &special characters.", "@#&", "...")

Result

This is a .string with .special .characters.

Example2

Translate("This is a string.", " istT", "_at")

Result

"hat_at_a_trang."
