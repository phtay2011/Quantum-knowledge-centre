Intraday Matching

Use the Intraday Match page to view and reconcile Intraday* bank flows and cash flows. The page shows one query (the last viewed query is shown by default).

*TIP: Intraday matching applies to unconfirmed Cash Management bank-reported data for the current and still-open day. For matching with confirmed data for earlier, closed-off days, use Prior Day Matching.

Intraday matching enables the Cash Manager to get a quick view on the current day's cash position as soon as data is available from the bank. With Intraday matching, the bank-reported position is always the focal point, unlike other parts of Quantum, where bank information plays more of a supporting role to synchronize the system's view of the cash position.

When you run an Intraday match query:

  • the query summary shows the date range, the number of accounts with unmatched flows, and how many cash flows and bank flows are involved;
  • the Account summary shows the account number and the:
    • net total of unmatched bank flow amounts,
    • net total of unmatched cash flow amounts, and
    • any Detail Match scripts linked to the account;
  • when you select an account in the summary list, the Flows and Matches tabs then display;

On the Flows tab:

  • the Bank Flows and Cash Flows grids list the flows and their match statuses,
  • you can optionally configure each grid view (filters, displayed columns, and so on),
  • select one or more items in each grid to enable the manual matching functions,
  • each grid menu's items apply to the currently selected flows in that grid,
  • in the Match button's drop-down list, select the required function (Match, Possible Match, Probable Match, and Unmatch, each enabled only when your selected flows satisfy the required criteria in each case),
  • you can select a set of flows in one grid only and - provided they sum to zero - apply a Match to that set, by selecting Force a Match from that grid's menu, and
  • when you select a function, the match status of the selected flows changes accordingly;

On the Matches tab:

  • the Matches grid lists matching summaries, with one row per Match ID showing Match Status, Bank Flows and Cash Flows totals and numbers included in the match,
  • you can optionally change the match type by selecting one or more match rows In the Matches view grid, and selecting the new match type from the grid menu,
  • select a Match row to display the Bank Flows and Cash Flows grids on this tab (which list only those flows covered by your selected Match ID), and a Match button, and
  • in the Match button's drop-down list, select the required function (as on the Flows tab - see above).
