Customizing Rule Conditions

The Customize Rule Condition functionality has been provided so that you can change the items available in a rule condition. For example, you may have business units set up as Counterparties, and want different accounting rules for certain Instruments using these Counterparties. Quantum is predefined with the Instrument Name, so to include the Counterparty Name in the accounting rule you need to customize the rule condition.

Customizing the rule condition changes the fields displayed when creating or editing a rule. You can customize your rules by adding new fields, removing fields, and changing the priority order of fields.

You can customize your rule conditions, provided you have the appropriate Module Security permission.

CAUTION: Customizing rule conditions should not be undertaken lightly. Ideally the rule condition should be customized on initial set up of the accounting static data. Adding new fields, removing fields or changing the order of priority can change the accounting or posting rule that is used by a transaction. The accounting treatment changes are not applied to existing deals. For example, you might want to change the field priority order so that now an existing deal uses a new posting rule that posts G/L entries to different G/L accounts. The existing entries on the deal are not changed, however any subsequent actions to the deal, such as accruals or deal customization, use the new G/L accounts.

NOTES:

  • The NOTE area in the Customize pop up box displays information relevant to the rule you are customizing. For example, this area identifies the products for which the changes apply.
  • The system ensures that the integrity of the system is maintained. You cannot remove a field where its removal would cause duplicate rules to exist. See Validation Checks, within What happens when you customize rules.
The Level at Which You Can Customize Rules

You can customize your accounting rule and posting rule structure from the Master accounting type only. This changes the structure of all existing and subsequent rules for all accounting types, ledgers and entities, depending on the product type for which the rule applies. For further information, see What happens when you customize rules and Product groupings and customizing rule conditions.

NOTE: You cannot customize the:

  • inter-company rule structure
  • accounting rule structure for Bank Account Interest products
What Happens When You Customize Rules

When you customize a rule condition the change affects all existing and subsequent rules for all accounting types, ledgers and entities, for that product(s).

NOTE: You cannot customize rule conditions for individual rules.

Validation checks

When you customize rule conditions, Quantum checks that the change does not result in a duplicate rule. If it does you must remove one of the rules before you can proceed with the customization.

Product Groupings and Customizing Rule Conditions

Because financial products differ, not all products within Quantum reference the same Quantum fields. For example, in Accounting Rules, Futures and Exchange Traded Options reference Futures Contract and ETO Contract respectively, while all other products reference Instrument Name.

This means that before you customize your rule conditions you need to consider the product against which the rule applies. Depending on the product, the rule customization may apply to all rules for one product or for many products.

NOTES:

  • You cannot customize accounting rules for Bank Account Interest. Therefore, when you customize accounting rules for other products, Bank Account Interest accounting rules remain unaffected.
  • You cannot customize individual products that fall within the All other products group. In this instance, if you change the rule condition for one product you change the condition for all products within the group.

Futures and ETOS

CAUTION:

  • If you add Broker Name as a rule condition for Futures and ETOs, be aware that the list of Brokers includes both Executing and Clearing Brokers. Currently however, rules will only ever apply to Executing Brokers. This means if you create a rule where the Broker is a Clearing Broker, the rule will never be used.
  • To reduce the creation of redundant rules, it is good practice to ensure you select only Executing Brokers from the list.
