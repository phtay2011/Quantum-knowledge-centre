CashXplorer and Prior Day Matching

A prerequisite for accurate cash positions and cash flow forecasts is the availability of reconciled data for the previous day, upon which forecasts can be built.

Using Cash Management Data and Bank Reconciliation functionality, CashXplorer can:

  • import data from banks and other external sources,
  • validate the imported data,
  • reconcile Quantum's cash position with the imported data,
  • view summary information for the reconciliation and data import status,
  • automate the process of data import, reconciliation and Data Version creation, and
  • accurately represent any zero balance cash flows.

Import Data

Using the available data import methods, CashXplorer has access to:

  • External Cash Flows (cash movements referenced in another system), and
  • Bank Flows (cash movements referenced in an imported bank statement that may or may not be matched with a Quantum cash flow).

See Importing Bank Files for more information.

Validate Imported Data

It is crucial to the integrity of CashXplorer that imported bank files and external data files are accurate and appropriately formatted.

Bank Reconciliation includes a thorough validation of data imported in formats such as SWIFT and BAI.

The structure of the validation routines can be tailored according to the quality of the imported data and, where appropriate, include extra validation codes.

Reconciled Data

Accurate cash positions and cash flow forecasts can be made only if the previous day cash flows are reconciled. Through Bank Reconciliation, CashXplorer has access to reconciled bank flows and balances.

The Balances and Floats used as the starting position for the cash positions and cash flow forecasts can include or exclude unreconciled cash flows:

  • current balance/last of the day (excluding the Unreconciled cash flows of that day) and
  • first cash flow of the day (including the Unreconciled cash flows of that day).

To illustrate the accuracy of the starting position, CashXplorer can display unreconciled flows for the previous day as System-defined Horizons:

  • Unreconciled Bank Flows Prior to Start, and
  • Unreconciled External Cash Flows Prior to Start.

See Matching Cash Flows with Bank Flows

Accurately Represent Zero Balancing

Zero balancing is an account management practice in which money is transferred between accounts in order that preset target balances can be maintained.

Using Bank Reconciliation functionality, CashXplorer is able to accurately represent zero balancing within a Data Version.

See Zero Balancing and Bank Reconciliation.

Automate Import Data, Reconciliation, and Data Version Creation

The Import Cash Management Data form includes run-time parameters you can use to:

  • select the files you want to import (if you are importing Bank transaction data from ECHOS, no files are involved - see Importing Bank Data from ECHOS), and
  • after a successful import - run the auto-matching process and automatically create a new Data Version.

See:

  • Bank Reconciliation Auto-Matching Parameters
