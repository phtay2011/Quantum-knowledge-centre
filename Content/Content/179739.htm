EFT Plug-Ins

An EFT Plug-In DLL is required to produce EFT output, and must be located in the bin folder for your site. This can be a plug-in supplied by FIS (the default plug-in) or a custom plug-in.

You can also use workflow templates, created in the Enterprise Workflow (EW), to do the same tasks as these plug-ins.

CAUTION: If you are using a custom EFT Plug-In created prior to Release 6.2.1.273, it must be updated - see below.

About the Default EFT Plug-In

The default EFT plug-in provided with your system handles SWIFT External Methods, in conjunction with the supplied XML Data records in each case.

Implementing EFTs for other External Methods

In some cases - especially with SWIFT-style output - you may be able to use the default EFT plug-in with XML Data records that are modified versions of the supplied ones. For assistance in such cases, contact your FIS representative. See also XML Data records for other External Methods.

More complex and non-SWIFT-style EFTs may require specific plug ins. Also contact your FIS representative in such cases.

About a Custom Plug-In

IMPORTANT NOTICE FOR EFT PLUG-IN DEVELOPERS

From Release 6.2.1.273, the way that EFT custom plug-ins initialize or override some of the properties or methods was changed. This change was necessary because Quantum Six does some pre-processing before plug-in initialization.

Previously, the Initialise method was used in the custom plug-in to set up a CpartyDelLink value. By default it was set true which meant that the Counterparty details were required, as follows.

/// <summary>
/// Initialise
/// </summary>
bool Initialise(bool CpartyDelLink = true);

From Release 6.2.1.273, the new property EnsureCountepartiesHaveDeliveryInstructions replaces the one above.

This is a virtual property on the EFTPlugin class that returns a default value of true. If the plug-in needs to use a false value, then this virtual property must be overridden in the plug-in code, as follows.

protected override bool EnsureCountepartiesHaveDeliveryInstructions
{
    get { return false;}
}

The value returned could be hard-coded as above, or it can come from a database field, depending on the plug-in requirements.

The above property is usually overridden in the same class file with the OnRun() method, which is in the custom plug-in C# file.
