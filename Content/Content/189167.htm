Setting Up a Site to Use the WAC Processor

A number of tasks need to be undertaken to set up a Quantum site to use the Weighted Average Closeout (WAC) Processor program.

TIP: For definitions of the acronyms and terms used here, see WAC Terminology.

Update Money Market Instrument Types (WAC Processor)

On the Security Instrument page for the security that is to be weighted average processed by the Weighted Average Closeout (WAC) Processor program, make the following selections.

Details tab

  1. Select from the Formula Timing list, one of the WAC supported formulas. The supported formula types are:
    • BOND PERIODIC
    • BOND ROLLING
    • CERTIFICATE DEPOSIT PERIODIC
    • DISCOUNT PREPAID
    • FRN-CGL PERIODIC
    • SIMPLE DISCOUNT PREPAID
    • ZERO COUPON DISC ON MATURITY
  2. Select CLOSEOUT in the Closeout or Unwind list.
  3. Select any Closeout Method except WEIGHTED AVG.

    NOTE: Instruments with the setting of WEIGHTED AVG will be excluded from the WAC processor portfolios.

Dates tab

  1. (Mandatory) Select a Maturity Date - a date must be entered here.
  2. In Maturity Basis select ACTUAL DATE.

The securities will be processed by the WAC Processor, providing the deal setup meets all the settings above.

Create Notional Entities (WAC Processor)

For the Weighted Average Closeout (WAC) Processor program, you must create a notional Transaction Level entity for every actual Transactional Level entity to which weighted average closeout accounting is to be applied.

  • Create each notional Transactional Level entity in the usual way. Make sure that the notional Entity:
    • Has the same name as its original but that it is prefixed (for example, when prefixed by Z-, if the actual entity is "Dealing Entity" then the notional entity is "Z-Dealing Entity".

      TIP: Use any prefix you like, however, keep in mind that it might be most practical to use one that ensures that these notional entities are always listed last to keep them out of your way. For examples see Business Structure and the WAC Processor

    • Is assigned to the same Group Level entity as the actual entity it mirrors.
Create Notional Counterparties (WAC Processor)

Create up to four notional Counterparties for use by Weighted Average Closeout (WAC) Processorprogram in generating notional deals. For examples of each of the following records see Counterparties and the WAC Processor.

  1. Create a notional counterparty (for example, Z-WAC).

    This is the Counterparty on all the portfolio deals. Set up as Deal Internal.

  2. Create a notional closeout counterparty (for example, Z-WACC).

    This Counterparty is only used on the deals created to fully close out the position when the portfolio position is being increased. Set up as Deal Internal.

  3. Create a notional bank counterparty (for example, Z-WACO).

    This bank is the notional bank counterparty. Set up as Bank Internal.

  4. (Optional) Create a notional custodian counterparty (for example, Z-WACOC).

    Do this only if using stock flows (you can either set it up as separate Counterparty or select the Custodian check box for Z-WACO - see above).

NOTE: The prefix used for notional counterparties (for example, Z-, as above) can be anything you like. However, once you have chosen the prefix it must be used consistently for all static data set up for Weighted Average Closeout processing using the WAC Processor).

These Notional counterparties are now available for use during WAC processing.

Create Notional Bank/Stock Accounts & Bank Account Defaults (WAC Processor)
  1. Create a notional Bank Account for each combination of Currency (of deals in the Bank Account Owner), and Bank Account Owner.
  2. Set up Bank Account Rules for each notional Bank Account created.

    NOTE: As a general rule, you should consider setting up notional Bank Accounts aimed at Group Level Bank Account Owners (Entities) to minimize the number of notional Bank Accounts and Bank Account Defaults required.

  3. (Optional) Set up a notional Stock Account for each Custodian Account that will be processed for weighted average closeout.

Set up General Ledger requirements (WAC Processor)

The following tasks describe the requirements for setting up a General Ledger when using Weighted Average Closeout (WAC) Processor program.

Posting Rules

  • Set up a G/L posting rule for the notional deals with each of the following:
    • Notional counterparty (for example. Z-WAC) deals
    • Notional closeout counterparty (for example Z-WACC) deals

GL Templates

  • Set up a G/L template each for the following:
    • A general G/L account (for example, call this G/L Template "WAC Clearing") that will be used as a netting account for many of the accounting entries created by the posting rules set up above.
    • All notional Bank Accounts (for example, call this G/L Template "WAC Clearing Account").

Set up a Position Processing Parameter Format (WAC Processor)

A Position Processing parameter format is required if the Weighted Average Closeout (WAC) Processorprogram needs to run Position Processing.

On the Position Reporting page, make the following specific selections:

  • Set As At Date to TODAY.
  • Set Position Date to TODAY.
  • Clear the As at End of Day check box.
  • Where the Accrual Method setting in Accounting Rules is Straight-Line, Yield Cum, or Yield Ex, select Use management rule, then the Rule Set to the specific Management Control record you have created for use by the WAC Processor when running the Position Processing - for example Default (with GL Owner: ALL).
  • Where the Accrual Method setting in Accounting Rules is Amortized Cost, select Use accounting setup and in Position Processing, ensure the Update the General Ledger For options: Standard Accounting and Hedge relationship deals are NOT selected.
  • Save the Position Processing parameter format as WACC.

    IMPORTANT: This Position Inquiry parameter format must be called WACC to allow the WAC Processor to run it successfully.

Set up a Management Control record (WAC Processor)

This information applies when the Accrual Method setting in Accounting Rules is Straight-Line, Yield Cum, or Yield Ex only.

(Management Control does not support the Amortized Cost accrual method.)

When the Weighted Average Closeout (WAC) Processor program is used, a Management Rule record for selection in the Rate Set field in an Accounting > Position page has to be set up as follows:

  1. From Book Value, select ADJUSTED.
  2. From Accrual Method, select the same Accrual Method as is set in Accounting Rules for the deals to be processed by the WAC Processor.

For general information about management reporting control, see Management Reporting Control

For general information about Money Market instruments and management reporting control, see Money Market tab, Management Reporting Control.

For a example of a management control record set up specifically for use when the WAC Processor runs Position Processing, see Management Control and the WAC Processor

Switch an existing Position to the WAC Processor

CAUTION: It is essential that you read Switching an existing Position to the WAC processor in its entirety before using the Weighted Average Closeout (WAC) Processor program to process existing live deals that currently use the Weighted Average Closeout method (WEIGHTED AVG) in Quantum Money Market Instrument setup.

The following tasks are required to switch an existing Position to use the WAC Processor.

  1. Ensure the deals are up to date.

    That is, check that all actions on the deals that are currently in the portfolio are up to date. Back dated changes to these deals will not be reflected in the portfolio processed by the WAC Processor.

  2. Fully Closeout each portfolio.

    Closeout the entire portfolio at the weighted average price to get a zero profit and loss impact on the closeout. Save this deal using a notional counterparty.

    CAUTION: In this case do not use the notional counterparty setup for WAC Processor (for example, "Z-WAC").

  3. Update Instrument Closeout Method.

    That is, change the Closeout Method in the Money Market Instrument to be a method other thanWEIGHTED AVG.

  4. Update Accounting and Posting Rules.

    That is, change the Accounting Rules and Posting Rules as specified. See General Ledger and the WAC Processor.

    NOTE:The Weighted Average closeout method does process capital accruals. To get the same behavior on the new WAC deals ensure the Accrue Capital check box in Accounting Rules is unchecked.

  5. Save Portfolio deal.

    That is, save a new deal representing the entire portfolio at the weighted average currently reflected in the accounts. Save this deal using the same notional counterparty as in Step 2 above.

    This deal will be the same as the deal saved in Step 2 above, except for the BUY/SELL which will be the opposite of the deal.

  6. Run the WAC Processor.

    That is, run the WAC Processor to generate the replicated WAC position.

  7. Process new deals.

    That is, process new deals as described in this document. This will have the effect of moving the G/L entries from the original deals to the replicated entity, from the date of the switch to using the WAC Processor. The cashflows to settle from the date of the transfer to the WAC Processorwill effectively also transfer to the Portfolio deal (as saved in step 5 above). As new deals are added to the portfolio, the cashflows will be settled and recorded against these new deals.
