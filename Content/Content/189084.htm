How Inter-Company Transactions Work in Quantum

An Inter-Company transaction is created when a deal fulfills the Bank Account and Inter-Company Defaulting structure, as defined using the process summarized in Setting up for Inter-Company Transactions.

Transaction Elements and Entries

An Inter-Company transaction consists of a cash flow and deal element. The entries created for each element are described in the following table.

Deal

*NOTE: The inter-company account refers to either the inter-company bank account or inter-company G/L account, depending on the underlying transaction.

If an Inter-Company Defaulting Rule exists for both bank accounts and General Ledger accounts, and the selection criteria for those defaults are identical, then Quantum uses the inter-company bank account default.

If an Inter-Company Defaulting Rule does not exist, then Quantum either:

  • posts the entries to a suspense account, if suspense account posting is available; or
  • displays a message that suspense account posting is unavailable. In this instance, the deal is not saved until the inter-company default is defined.

NOTE: Suspense account posting is enabled by selecting Allow postings to suspense account in the the General Ledger Properties page.
