File Integrity Validation Parameters

The parameters below control the extent of data validation carried out by the Bank Import service on a BAI file before it is processed. If any of these parameters are set to Yes, and the file fails validation on one of these counts, the file will be rejected, and will be moved to the error directory.

By default, all these parameters are turned off (set to No).

TIP: Leave these validation parameters turned off if you experience incorrect validation check sums in BAI data you receive from data providers.

ValidationPass

Specifies whether or not any of the following individual file validation parameters are tested. This is effectively a file validation "master switch".

Syntax

ValidationPass = Yes | No

Default

ValidationPass = No

ValidateLinesInStatement

Specifies whether or not, when ValidationPass = Yes, to validate that the number of lines in each statement matches the check sum in the statement trailer record.

Syntax

ValidateLinesInStatement = Yes | No

Default

ValidateLinesInStatement = No

ValidateAccountsInGroup

Specifies whether or not, when ValidationPass = Yes, to validate that the number of Statements in a group matches the check sum in the group trailer record.

Syntax

ValidateAccountsInGroup = Yes | No

Default

ValidateAccountsInGroup = No

ValidateStatementsInFile

Specifies whether or not, when ValidationPass = Yes, to validate that the number of Statements in a file matches the statement check sum in the file trailer record.

Syntax

ValidateStatementsInFile = Yes | No

Default

ValidateStatementsInFile = No

ValidateLinesInFile

Specifies whether or not, when ValidationPass = Yes, to validate that the number of transaction lines in a file matches the transaction check sum in the file trailer record.

Syntax

ValidateLinesInStatement = Yes | No

Default

ValidateLinesInStatement = No
