Scheduler Jobs

New pages have been added to Quantum to perform limited processing and management of Scheduler Jobs.

Where previously you could only execute Scheduler jobs from the FIS Scheduler server, you can now use these pages instead.

Background

FIS Scheduler is a client/server application which enables you to execute and schedule certain Quantum functionality using predefined Task Providers.

Primarily, it enables the scheduling and execution of the following programs and processes:

  • Replication Manager, which enables information to be replicated from Quantum to Analytics (for example, Users, Static data, and so on).
  • Analytics Reconcile, which enables deals, cash flows and bank accounts to be reconciled between Quantum and Analytics.
  • Deal Importer / Remover, which enables batch importing (or removal) of deals into (or from) Analytics.
  • Reporting (Quantum or Analytics) and Processing, which enable you to combine, execute or schedule frequently run reports.
  • Static Data, used to schedule the generation of Quantum Static Data reports.
  • Market Value Importation (MVI), which process in Quantum allows Analytics to post valuation data to a table in the Quantum database (imp_mtom). These market values are then used by Quantum in place of its own valuation routines when running positions.

Implementation

Job Page

Subject to having the Inquire permission for the Job module in Module Security, a user will have access to the Job page from the Configuration menu.

The Job page lists all the custom modules of type Job created in the Custom Module page which a user has permission to see.

The Job page enables you to view and maintain information for existing Scheduler jobs.

From here you can:

  • Run (or re-run) a Job.
  • Modify the schedule.
  • View or clear the history and associated logs.

NOTE:

  • FIS Scheduler 3.2.0.731 or later must be installed on the Quantum Application Server.
  • The Quantum Site must be set up as a Company in FIS Scheduler before the new Scheduler pages are activated in Quantum.
  • For a custom module of type Job to appear in Quantum, the Name of that custom module must be exactly the same as a Category in Scheduler. As Quantum Module Security is defined at the level of the individual custom module, Quantum users will therefore be able to view, run or manage, as permitted, all Scheduler jobs allocated to that Category.
  • The timezone defined for the Company and Job in FIS Scheduler should match the timezone of the server on which FIS Scheduler is installed.

Scheduled Jobs Page

Subject to having the Inquire permission for the Scheduled Jobs module in Module Security, a user will have access to the Scheduled Jobs page from the Utility menu.

The Scheduled Jobs page enables you to view the history of execution of all scheduled Jobs.

From here you can:

  • Run (or re-run) a Job.
  • View the history and associated logs.

NOTE: Users will only see the jobs they have the permissions to see and/or run.
