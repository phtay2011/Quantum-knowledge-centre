The instructions in this Commands.config file (see below) will create the following page in Quantum:

Clicking Run will execute the instructions and display the results as notifications in Quantum.

Commands.config File

 

NOTES

 

 

 
  • Set an appropriate timeout value to ensure that processing and retrieval operations (such as file fetch operations) do not run indefinitely.
  • The module name must match that of the custom module defined in Quantum.

    TIP: You can create multiple <command> blocks, associated to the same or different module.

 

  • Set restrictTovalues="true" to create a drop-down list of values AND restrict manual entry in the field.
    When restrictTovalues="false", users can type in field values OR select from a browse list.

 

  • The datatype=Date variable's output will be in the format defined in Quantum's User Account Settings page. Set the output format as per your requirements.
    Date can have a default value (Today, Yesterday, and so on).

  • Include the variable name/s (defined above) in arguments to display the required controls or fields when the option is selected.
  • Use the outcome result to define two or more outcomes and display the associated message in the Notifications panel when true.

    TIP: Define multiple outcome result values in a batch file.
