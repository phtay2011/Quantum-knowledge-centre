Hedging the Spot Value of FX Forwards

Quantum recognizes the change in the forward element of the Forward Contract when only its spot risk is designated in a hedge relationship as follows.

  • For transaction related hedged items, the change in fair value is accumulated in a separate component of Equity and reclassified to profit and loss.
  • For time period related hedged items, the change in fair value is accumulated in a separate component of Equity and amortized on a rational basis over the term of the hedge relationship.

Designated Risk: Overall Market Risk v Foreign Exchange - Spot

When the Spot Value is designated in a hedge relationship with Designated Risk equal to Overall Market Risk, the changes to the Spot Value are accounted in accordance with the type of hedge (Fair Value or Cash Flow), whereas the changes to the Forward element are accounted in profit or loss, resulting in potentially significant profit or loss volatility.

When only the Spot Value is designated with Designated Risk equal to Foreign Exchange - Spot, the change in the fair value of the forward element is recognized in OCI and accumulated in a separate component of equity. The forward element that exists at inception is optionally amortized from the separate component of equity and classified to profit and loss on a straight line basis.

Hedging procedures

Create the Hedge Strategy page

Create a valid Hedge Strategy page with the following options:

  1. A Fair Value Hedge or Cash Flow Hedge for the Hedge Type; FOREIGN EXCHANGE - SPOT for the Designated Risk (Details tab).
  2. If required, set a release date for the adjustment amounts (Events tab).

Create the Hedge(s), Hypothetical(s) and Exposure(s)

Create the following deals in Quantum according to the requirements of your hedge:

  1. Hedge: Create one or more FX deals that represent the terms of the hedge.
  2. Hypothetical*: For each Hedge deal created in Step 1, create an FX deal that counteracts the purpose of the hedge - that is, Hedge (Buy) / Hypothetical (Sell) - but matches the terms of the hedge exactly.

    * NOTE: It is possible to account for the forward element component of the forward contract only to the extent it relates to the hedged item.When the terms of the hedge do not match at inception, the Aligned Forward Value is determined using the Hypothetical, which represents the risk being hedged.

  3. Exposure: Create one or more Actual Cash Flow deals to designate the risk being hedged.

Create the Hedge Relationship

Create a hedge relationship with the following options:

  1. Select the Hedge Strategy page with FOREIGN EXCHANGE - SPOT as the Designated Risk, created above.
  2. Select the Maturity Date, and ensure that the Designated Risk is set to FOREIGN EXCHANGE - SPOT (Deal Details tab).
  3. For time-period related hedges, select Amortize hedging costs (forward value) (Deal Details tab).
  4. Add the Hedges, Hypotheticals, and Exposures created above to the hedge relationship (Linked Deals tab).

IMPORTANT: Once a Foreign Exchange - Spot Hedge Relationship has been designated and postings have been started, it is currently not possible to add additional deals into the Hedge Relationship.

Create the Posting Rule(s)

For the entity, select the following accounts for the Posting Rule for hedge relationships (Posting Rule, BALANCE SHEET tab):

  • OCI Balance (Hedging Costs)
  • Deferred Tax (Hedging Costs)
  • Hedging Costs Reclass
