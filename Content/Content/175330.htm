Standard Cap/Floor Pricing Models

An interest rate Cap is designed to provide insurance against the interest expense from a floating rate loan exceeding a certain range. That is, a cap will have a positive payoff when the reference interest rate ends up above the fixed rate specified in the contract. The borrowing entity still pays an amount of interest based on periodic fixing of the floating rate, but when that exceeds the interest cap, the net cost of borrowing is partially offset by the payoff from the option.

Conversely, an interest rate Floor is analogous to a standard put option. The holder of the floor will receive a payoff when the reference interest rate falls below the benchmark rate. This structure can therefore be used to guarantee a minimum return for a lender.

A Collar is simply the combination of a long position in a Cap and a short position in a Floor, and is typically used to ensure that the cost of floating-rate debt does not fall outside a certain range.

Caps are an aggregation of a series of caplets, each of which provides insurance for discrete periods during the life of the cap. In general, the payoff function for the caplet that matures at time ti with payment to be made at ti+1 can be written as:

max(F - K, 0) * t * FV

...where:

F = current Forward Rate (value of the reference floating rate at reset date i)

K = fixed Strike (or Cap) Rate for the deal

t = Cap/Floor term in years; the time between the i reset date and payment date

FV = Face Value of the deal

     = P/(1 + Ft)

P = Principal

The payoff structure in the equation above is the standard payoff for a Call option. If the reference interest rate F is assumed to have a log-normal distribution with volatility then the caplet can be valued using a modified version of the standard Black (1976) model:

Cpu = DF(0,ti+1) * [F * N(d1) - K * N(d2)]

Cpos = Cpu *FV * t

...where:

Cpu = Cap per Unit Market Value

Cpos = Cap Position Market Value

N = Normal Distribution function

DF(0,ti+1) = Discount Factor for the period between time 0 and time ti+1

                = e-rT

e = 2.71828182... (the base of the natural logarithm)

ln = natural logarithm function (log to the base e)

si = Volatility

NOTE:For most standard options, the payoff occurs on the same date as the comparison between the spot and strike values, and the appropriate discount factor therefore has the same maturity as the expiration date. However for the caplet, the payoff occurs at the end of the period after the rateset has been made. This is consistent with the fact that bond interest payments are normally made in arrears.

Using the same logic, the Black model for floorlets is defined as:

Fpu = DF(0,ti+1) * [K * N(-d2 - F * N(-d1))]

Fpos = Fpu *FV * t

...where:

Fpu = Floor per Unit Market Value

Fpos = Floor Position Market Value

NOTE:In the definitions of d1and d2 above, the volatility measure for each caplet or floorlet has a subscript. This implies that the volatility measure can potentially differ for each of the optlets that make up caps or floors. Alternatively, if the reference interest rate F is assumed to have a normal distribution with volatility then the caplet can be valued using the Bachelier model:

Cpu = DF(0,ti+1) * [(F – K) N (d) + σ√(t )N’(d)]

Cpos = Cpu *FV * t

...where

Cpu = Cap per Unit Market Value

Cpos = Cap Position Market Value

N = Normal Distribution function

N’ = Probability Density Function of a Normal Distribution

DF(0,ti+1) = Discount Factor for the period between time 0 and time ti+1

                = e-rT

ln = natural logarithm function (log to the base e)

si = Volatility

The Bachelier model for floorlets is defined as:

Fpu = DF(0,ti+1) * [(K - F)N(-d) + σ√(t )N’(d)]

Fpos = Fpu *FV * t

...where:

Fpu = Floor per Unit Market Value

Fpos = Floor Position Market Value
