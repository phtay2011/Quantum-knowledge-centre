Bank Statement Import - New Transaction Rule Set page

(First introduced in Release 6.6.1 SP, March 2018.)

You can use the new Bank Transaction Rule Set page to create and maintain Bank Statement Import data manipulation details. This includes uploading existing Transaction Rule files.

Background

The Bank Import Service supports parameters (rules) you can use to manipulate imported bank statement and transaction data before it is loaded into Quantum. Clients have requested a way to manage these within the Quantum Six user interface.

Implementation

Bank Transaction Rule Set page

An example of this new page is given below.

Use the standard page menu functions to Add, Edit, Copy, Deactivate, or Delete a Rule Set.

Use the standard grid menu functions to Add, Edit, Copy, Deactivate, or Delete individual rule entries in the current Rule Set or its (or click on an existing grid row value to edit it). You can also use the grid menu's Upload Rules function to add one or more rule entries from an existing Transaction Rule TXT file (Legacy syntax only).

An example of the Edit dialog box is given below.

In each rule entry:

  • Order* indicates the order in which the rule is applied within the rule set,
  • Name is a text string identifying the entry within the rule set,
  • Bank Account gives the selected accounts the rule applies to (or ALL),
  • Additional Conditions* is a logical expression giving conditions the applicable data must meet,
  • Assignments* sets bank statement and transaction data values to values that you specify (when the conditions are met), and
  • In Use indicates whether the entry is currently active or not.

*NOTES:

  • If you change the Order of an existing rule, the Order number on all other rules will automatically update.
  • The Additional Conditions and Assignments text boxes must respectively contain a valid Condition Clause and Assignments Clause in Legacy syntax.
  • The ability to include Cash Flow Instrument (IN), Ticket Number (TN), and Entity (EN) field keywords in Condition and Assignment clauses (as in Rule entry 2 in the example page above) is also new from this release and applies to all methods of data manipulation.
  • Where rule entries specify the same Bank Account, the most specific rule will be applied.

BKIMPORT.INI Configuration File

To apply a Bank Transaction Rule Set, include the new TransactionRuleSet parameter in the BKImport.ini file, specifying the Rule Set Name. For example:

TransactionRuleSet = XYZTransRule

Your BKImport.ini file may contain a combination of included files and a single TransactionRuleSet parameter. Transaction rules in Included Files will be applied first, then the Transaction Rule Set's transaction rules will be applied.

Module Security

The Bank Transaction Rule Set Module has been added, with the following permissions available for assignment to users and/or groups.

Assigning this Permission...

Allows Users to...

Inquire

Access the page and view all Rule Sets

Add

Add a new Rule Set only (not individual rule entries)

Change

Edit an existing Rule Set only (not individual rule entries)

Delete

Delete an existing Rule Set, including all its rule entries

Deactivate

Deactivate/Reactivate an existing Rule Set

Transaction Rule Add*

Add a new Rule entry to a set

Transaction Rule Change*

Edit an existing Rule entry

Transaction Rule Delete*

Delete an existing Rule entry

Transaction Rule Deactivate*

Deactivate/Reactivate an existing Rule entry

Transaction Rule Upload Rules*

Import Transaction Rule entries from an external TXT file

Transaction Rule Clear*

Clear all Rule entries in the current set


* Rule Set Change permission also required.

Auditing

Changes to Bank Transaction Rule Set pages are audited.

You can view such changes by running a query from the Audit Reporting page. Then, in the Results View:

  • script changes are listed under a Table value of Bank Transaction Rule Set,
  • the Key Value gives the Rule Set name, and
  • Field gives the internal item name, with Bank Transaction Rule indicating grid entry changes.

Restrictions and Limitations

The Transaction Rule Set page does not support:

  • Iron Python syntax,
  • Summarizing Clauses, nor
  • specifying a contiguous range of accounts using a TO operator - for example, 123400 TO 123499 (you must select each account).

You can only Upload files that have a .TXT extension (if your Transaction Rules file currently has the .INI extension, change it to .TXT before uploading).

History

Previously:

  • Bank Importer supported data manipulation using INI or TXT files only, and
  • you could not assign values to the bank flow fields for Instrument, Ticket Number, or Entity.
