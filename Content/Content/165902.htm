Counterparty Bank Account - Defaults, Rules

TIP: Counterparty Account Defaults are not mandatory in order to save a deal. If no default is present, no defaults are set for new deals.

Use the Counterparty Bank Account Rule page to create and maintain the default rules that associate a counterparty's cash flows with appropriate settlement instructions.

NOTE: Cash flows here are seen from the Counterparty's perspective, so that the Receipts option selects negative cash flows (and Payments selects positive cash flows).

These rules provide a high level of control over the specific circumstances in which different defaults are used. For example, you can use different settlement instructions for funds paid and funds received. The defaulting process searches the default records to find the most complete match, and applies the selected account or External Method information to the deal. These instructions can also be used for confirmation purposes and/or the actual settlement of funds through an Electronic Banking interface.

NOTE: When Static Data Approval applies and you add, update, or delete a record:

  • until the change is approved (or rejected), no further changes are allowed to that record (on this page).
  • the record is flagged as pending approval and all items become read-only.
  • links to the Unapproved page are provided for that record.
  • an Approver must use the Unapproved page to approve or reject the change (you cannot approve your own changes).
  • further editing is allowed on the Unapproved page (for an add or update) - provided no approve action has yet been taken.

When Static Data Approval does not apply, all changes can be regarded as immediately approved.
In either case, Static Data and Rule changes are recorded in the audit log. Use Audit-Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.

CLS (Continuous Linked Settlement)

TIP: If no Counterparty Account Default rule is found, the deal is not CLS-eligible.

If you are using CLS (Continuous Linked Settlement) with FX Deals and Currency Options, you can define different Counterparty Bank Account Rules for CLS and non-CLS (Standard Settlement) deals. The methods differ between the two transaction types, as follows.

For FX deals

NOTE: To use CLS, you must have a matching CLS Cut-Off Time Rule for the FX deal and for each currency (or ALL).

For each applicable Counterparty/Instrument/Currency combination:

  • Create one Counterparty Bank Account Defaulting rule for CLS, setting the Transaction Type to FX Deals only and selecting the CLS option. Add an Instruction entry with the required Effective Date, Bank, CLS-eligible Account Number, and set Payment Category to EFT NOT REQUIRED.
  • Either select an existing rule that covers the applicable combination for Standard Settlement, or create a new rule with the Standard Settlement option either selected or not shown. The rule must have an Instruction entry with the required Effective Date, Account Number, and Payment Category set to STANDARD SETTLEMENT INSTRUCTION.

See also: CLS workflow enhancements

For Currency Options

If you are using separate Instruments solely for CLS deals (recommended):

  • For the CLS Instrument, set up a Counterparty Account Defaulting Rule with a Payment Category of EFT Not Required.
  • For the non-CLS Instrument, set up a Counterparty Account Defaulting Rule with a Payment Category of Standard Settlement Instructions or Debit/Credit Advice.

NOTE: A Physical Exercise action on a Currency Option creates a new FX deal, when the CLS rules for that FX deal then apply as above.
