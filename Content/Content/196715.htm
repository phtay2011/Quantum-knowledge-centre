Test DTL WebLogic Connections

Incoming Example

An example of a Data Transfer Location record for the Deal Import queue example is shown below, where the Location is the JNDI queue name (jms/WLDealImport in this case).

To test the connection:

  1. In WebLogic Server, go to WLDealImport queue -> Monitoring -> ShowMessages and press New.
  2. In the Body text field under Produce JMS Message, enter your message content in QXT format with the above DTL Name in the ticket no line, as in the following part-example.

    EXENAME=FXDEAL
    $DoMenu=mfNew
    ticket no=IncomingExample
    fxtype=moFXDeal
    sectype=FX Forward - Time Option
    cparty=ABCorp
    entity=....

  3. Press OK.
  4. On Quantum's FX Dealing page, you should see the new imported deal in the browse list.

Outgoing Example

An example of a Data Transfer Location record for the EFT queue example is shown below, where the Location is the JNDI queue name (jms/WLEft in this case).

To test the connection:

  1. On Quantum's EFT processing page, set File Destination to the above DTL Name (OutgoingExample in this case).
  2. Run an EFT query that returns at least one message (with a Sent value of Y).
  3. In WebLogic Server, go to WLEft queue -> Monitoring -> ShowMessages.

    You should see the messages received from Quantum.
