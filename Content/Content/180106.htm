/CASHMGMT Examples

Archiving Cash Management Data

Example 1 - SQLServer, one Worksheet, with verbose logging

ARCHIVE /DB SQLSERVER OURSERVR\SQL2015 AGQMAINDBS
        /CASHMGMT /WORKSHEET CMWBAL 2015-06-23 3
        /LOG "C:\Temp\ArchCMLog20151231.TXT" VERBOSE

Archives the Cash Management Data Version that belongs to Worksheet CMWBAL, has a start date of 23rd June 2015, and is version 3, where:

  • the server name is OURSERVR\SQL2015,
  • the database name is AGQMAINDBS,
  • no user name or password is given, so that integrated security is used, and
  • a run log file will be output,
    • named ArchCMLog20091231.TXT,
    • in the folder C:\Temp, and
    • listing reference numbers for individual item that violate the archiving validation rules.

Example 2 - Oracle, with basic logging

ARCHIVE /DB ORACLE SERVRORA11
        /USR ADMIN_CJ /PWD PA55W0RD
        /CASHMGMT /DAYS 120
        /LOG "C:\Temp\ArchCMLog20151031.TXT"

Archives Cash Management Data Versions that have a Start Date more than 120 days old, where:

  • the server name is SERVRORA11,
  • the Oracle database name is not required,
  • the username of the database account is USR ADMIN_CJ and the password is PA55W0RD, and
  • a run log file will be output:
    • named ArchCMLog201510131.TXT,
    • in the folder C:\Temp, and
    • listing only indications that violations exist (not the relevant item reference numbers).

Restoring Archived Cash Management Data

Example 3 - SQLServer, one Worksheet

ARCHIVE /DB SQLSERVER OURSERVR\SQL2015 AGQMAINDBS
        /CASHMGMT RESTORE
        /WORKSHEET CMWBAL 2015-06-23 3

Restores, from the archive back into the AGQMAINDBS SQL Server database, the Cash Management Data Version that:

  • belongs to Worksheet CMWBAL,
  • has a start date of the 23rd June 2015, and
  • is version 3.

(Other parameters are the same as in Example 1.)

Example 4 - Oracle

ARCHIVE /DB ORACLE SERVRORA11
        /CASHMGMT RESTORE
        /DAYS 365

Restores, from the archive back into the Oracle database, Cash Management Data Versions that have a Start Date 365 days old ot less, where:

  • the server name is SERVRORA11 (as in Example 2), and
  • no user name or password is given, so integrated security is used.
