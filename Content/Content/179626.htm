Importing Quotes and Quote updates

The Quotes Importer supports the following QID-format fields.

NOTE:

  • quotetype is required only when importing new quotes. It indicates the type of quote, as shown below in each case.
  • deal_no is required only for updates to existing quotes and identifies the quote concerned.
  • Bid fields in the shaded third columns are not required if bid_speed=NO BID for the bid row.
  • If you want to import updated bids for an existing quote, you should re-import all bid details, because resaving the quote may have changed the bid row order and Quantum cannot detect that. No unique row reference is stored in the database.

FX Deal Quotes, Import File Examples

Example 1

Import an FX Deal Quote with no bids recorded.

(QID format)

EXENAME=QUOTES
$DoMenu=mfnew
quotetype=FX
bid_dt=10-Jul-2012
bidclosedt=10-Jul-2012
bid_time=10:00 am
sectype=CM FX Forward
dealer=baseline
entity=Owner Exclusive 1
fx_ccy=GBP
fx_ccy2=USD
fx_mature_dt=30-Sep-2012
fx_fv=1,000,000.00
sscparty=ADV Counterparty2|Deal,ADV Counterparty3|Deal,ADV Counterparty4|Deal,
bid_row=1
bid_cparty=ADV Counterparty2
bid_speed=NO BID
bid_row=2
bid_cparty=ADV Counterparty3
bid_speed=NO BID
bid_row=3
bid_cparty=ADV Counterparty4
bid_speed=NO BID
$DoMenu=mfinsert
