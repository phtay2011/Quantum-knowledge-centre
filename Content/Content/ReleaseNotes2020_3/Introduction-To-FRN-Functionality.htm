IBOR replacement functionality

The work to support adoption of risk-free rates (RFR) has continued within this release.

The following functionality is available to support instruments that are set in arrears (at the end of the interest period) using an average of daily rates published during the period.

Average rate

We now support two calculation methods to calculate the value of the Average Rate in Security deals that use an overnight indexed FRN instrument:

  • COMPOUNDING ON BUSINESS DAY- the system calculates the average rate for the period using the compound formulas which can be found in the ISDA definitions, and
  • ARITHMETIC AVERAGE (Simple Average) - the system calculates the average rate for the period using the sum of the daily interest factors.

To control the way the system rounds the value of the Average Rate, use the following settings, found in Dealing > Securities:

  • Average Rate Rounding - to round the final value of the Average Rate, and
  • Average Rate Intermediate Rounding - to round the daily intermediate interest factor calculations. Applies to the Arithmetic Average calculation method only.

Margin treatment

The Average Rate is calculated on a "Spread Exclusive" basis.

The Margin is added to the Average Rate to determine the All Up rate for the coupon period.

Advance notice of payment

Instruments that are rate-set in arrears using a daily RFR include conventions to ensure that the administrative agent can calculate the value of the Average Rate ahead of the coupon or interest payment. Quantum supports the following settings relating to the advance notice of payments:

  • Lag - Rate Only (look back) - for every day in the current interest period, Quantum uses the RFR from k days earlier,
  • Lag - Rate & Weight (look back observation shift) - for every day in the current interest period, Quantum uses the RFR from k days earlier and the weighting (number of days) that applied to that rate,
  • Suspension (lock out) - Quantum uses the averaged RFR over the current interest period, with the last k rate observations set using the rate from the day prior to the suspension period, and
  • Payment Date Offset Days (payment delay) - Quantum delays payment by k business days after the coupon / interest period end date.

Bulk rate-set

You can now rate-set one or more overnight indexed instruments using the Processing > Rate Set Deals page. A new action, Average Interest Rate, has been provided on the Rate Set Deals page.

Click on a line item within the Rate Set Deal page's report result's grid to provide a link to the Rates > Average Interest Rate Calculator page. This provides the detail behind the average rate calculation.

Accrual method

In the Static > Accounting Rule page, we now support the yield Accrual Method options for these instruments:

  • Coupon / Interest Accrual - the coupon is accrued on an "as earned" accrual basis. The system calculates the average rate for the period between the last coupon date and accrual date, and then calculates the accrual amount on a basis consistent with the calculation of the coupon amount, and
  • Capital Accrual - the system determines the capital accrual by repricing the instrument on the accrual date using the purchased yield and swap rate. This results in an effective yield-based amortization calculation.

Supported day count conventions

Currently we support Actual/365 and Actual/360 day counts, which reflect the day count conventions associated with the RFR.
