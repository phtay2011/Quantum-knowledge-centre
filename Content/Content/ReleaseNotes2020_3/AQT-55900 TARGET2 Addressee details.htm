TARGET2 payment changes

From 2020.3, instead of using an EFT plug-in specific to TARGET2, you can store the TARGET directory content as a table in the Enterprise Workflow (EW). The EFT and Confirmation XML Framework also includes a new function that retrieves the addressee from this table.

To use this functionality, you'll need to add or configure a few pages to upload the details from the TARGET directory content. Please add:

  • a tab delimited Parser page,

  • and a WorkflowIn Data Transfer Location (DTL) page.

For a detailed example of how to configure the parser and DTL, search for the Knowledge Center topic "Details of TARGET Addressee".

We support an additional attribute in the XML Framework:

  • mapTable - This attribute maps an input value, specified in the Layout XML Data record, to a field in a Quantum table, and returns the required value in the EFT message from the table when specified in the EFT layout for a given input data defined in the XPath.

For an example of how to collect the addressee from the TARGET Directory page, search for the Knowledge Center topic "Details of TARGET Addressee". To collect the addressee from the TARGET directory content, you'll require:

  • "Includes" XML Data, with additional IncludeItems specifying which BIC to compare and retrieve for the "Addressee". Note, the example documented is using either the Deal Counterparty BIC or Bank’s BIC; however, this is modifiable, and you can change the logic to include the BIC of other parties.

  • Layout XML Data, referencing the modified IncludeDocument and referencing the IncludeItem to populate the ReceiverAddress.

    In the ReceiverAddress, use the new XML function mapTable to retrieve the "Addressee" from the TARGET directory table.

    Optionally, define a fallback XPath for EFTs where there is no entry present in the TARGET directory.

    The targetDate will observe a "valid from" and "valid till" date as the TARGET directory may contain what is otherwise perceived as a duplicate "Participant" record.

For more information, search for the Knowledge Center topic "Details of TARGET Addressee".

NOTE: We will retire the SWIFT TARGET2 EFT plug-in, and settings related to the TARGET2 address file, from 2020.4.

M 8 Type of Change 1x

A: added

M: modified

D: deleted

U: unchanged

01 - "Direct"

07 - addressable BIC - Branch of Indirect participant

08 - addressable BIC - Branch of correspondent

Future releases

The SWIFT TARGET2 EFT Plug-In and settings related to the TARGET2 address file will be retired from 2020.4.

There will also be a few enhancements in the roll-up (RU) immediately following this main Version 2020.3 release. These include:

  • Support for the parser without headings, removing the need so manually enter ‘LINE’ in the TARGET directory as the first row.
  • The ability to define multiple actions in the same DTL, and thus delete existing directory details as part of the upload process of the latest directory file. In this main Version 2020.3 release, you have to define a DTL to delete and define a separate DTL to upload.
