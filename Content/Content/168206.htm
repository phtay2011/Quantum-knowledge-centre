General Ledger and WAC Processing

Weighted Average Closeout (WAC) accounting has specific General Ledger setup requirements.

Accounting Rules and WAC Processing

Securities that are to be included in WAC-processed portfolios should have the Accrual Method in their Accounting Rules set to one of:

  • AMORTIZED COST
  • STRAIGHT LINE
  • YIELD CUM
  • YIELD EX

An Accrual Method should be set even if selecting not to accrue.

Making Changes to Accounting Rules

If you make a change to an accounting rule (other than to Accrual Method) but you require that the deals saved with the old rule continue to be processed separately from the deals saved with the changed rule (where Instrument, Entity, and so on are the same), you must create a new instrument - otherwise they will be processed as one portfolio.

A change to the accrual method on the accounting rule where the weighted average portfolio has an open position is not supported, as deals with different accrual methods will not close out against each other.

For more information about customized deals, see Customized Deals and the WAC Processor.

Posting Rules and WAC Processing

Separate G/L posting rules must be set up for the following When the Average Closeout (WAC) processing scenarios.

Off Balance Sheet is not impacted by the WAC process.

*TIP: A general G/L account here is one specifically set up for use as a netting account for the accounting entries of the actual deal (except the Bank Account with its related Currency Translation entries), and the Bank Account and related Currency Translation entries of the notional deal.

Actual Deals with Actual Counterparties

All entries created in the True Transactional Entity (TTE) will be posted to the general G/L account* except for:

  • Bank Account (cash) entries.
  • Currency Translation entries related to the Bank Account.
  • Ad hoc Amount (Balance Sheet and P/L).

    NOTE: Ad hoc Amount is only for G/L postings from Ad hoc Amount customization. Ad hoc Interest postings follow interest accounts.

The above three G/L entry types are set to post to their actual G/L account in accordance with the actual posting rules set up in Quantum.

Notional Deals with Notional Counterparties

The reverse of the G/L set up for the actual deals mentioned above will be created in the Notional Transactional Entity with the notional counterparty. These will be pointed to their actual G/L account except for:

  • Bank Account (cash) entries.
  • Currency Translation entries related to Bank Account.
  • Ad hoc Amount (Balance Sheet and P&L).

These above three entries will post to the general G/L account*.

Notional deals with the Notional Closeout Counterparty

This will be the same as the setup for the Notional Transaction Entity with the Notional Counterparty - see above, except for the following.

  • The counterparty is the Notional Closeout Counterparty.
  • The Realized Gain/Loss P&L account is pointed to the same G/L account used to capture Capital Premium/Discount P&L entries, because deals with this counterparty are created purely to close out Notional Counterparty positions in order to enable the re-statement of an increase in the position of the net weighted average trade. This Posting Rule setup will nullify any P&L implications of this fictitious closeout.
  • Interest Income and Interest Expense is pointed to the same G/L account used to capture the Net Value Investments Posting Rule. This will ensure that the Cum/Ex Profit/Loss net out in the Net Value Investments account with no P&L impact.

Repo deals

If you have Repo deals and also require Money Market deals to be processed by the Weighted Average Closeout (WAC) Processor program, then you will need to set up the Posting Rules with an additional customized rule condition for Strategy. This is because Repos share the Money Market Accounting and Posting Rules. Deals to be processed by the WAC Processor need to be saved with the Strategy used in the rule in order to pick up the correct posting rules.

NOTES:

  • Strategy is not a criteria for determining a portfolio. All Money Market deals of the same Instrument/CUSIP/ISIN/Currency in the same Entity will be processed as one portfolio whether or not there is a Strategy. If you do not want the Money Market deals that have been Repoed to be processed by the WAC Processor, it is recommended you save these deals to a different Entity.
  • Separate Accounting Rules can also be set up based on the defined Strategy if required.

G/L Templates and WAC Processing

When the Weighted Average Closeout (WAC) Processor is used, General Ledger Templates are required for:

  • a general G/L account specifically set up for use as a netting account for the accounting entries (except the Bank Account and the related Currency Translation entries) of the actual deal, and the Bank Account and related Currency Translation entries of the notional deal, and
  • all notional Bank Accounts that notional deals (created during WAC processing) post to - for example, a WAC Clearing Account.
