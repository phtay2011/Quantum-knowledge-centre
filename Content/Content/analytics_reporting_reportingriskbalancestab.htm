Balances tab

(This tab appears only when an existing Analytics Balances Definition record is specified on the Properties tab.)

Use this tab to configure filters on the balances considered for an Analytics (or Adaptiv, if installed) query run. For example, you may wish to report only on balances for specific Bank Accounts, Account Owners, Currencies, and so on.

The default settings on this tab are taken from the Balances Definition specified on the Properties tab. Any changes made on this tab will be saved in that Definition record when the query configuration is saved.

Analytics Balances Definitions are shared with Adaptiv.

A filter is based on logical operators and field-level elements. You can create multiple filters with one or more selection criteria under a logical "branch", include other saved Balances definition filters (by using the Add Filter Link function), and nest branches. A branch must be set to either Match All (Logical AND) or Match Any (Logical OR), and there is always one overall branch.

To add a logical branch - Select the "parent" branch, and in the Add Filter button's drop-down list, select Add Branch. A new branch is added under that parent. To change the branch type, select Match All or Match Any in the branch's drop-down list as required.

To remove a branch - Select the branch, and in its drop-down list, select Remove Branch.

To add a new filter row within a branch - Select the required branch, click the Add Filter button, and select the Bank Account filter field from the drop-down list (start typing to find your filter quickly). Specify the field's filtering criteria as follows:

  • Drill down on a link to show the fields within that data folder -Bank Account, then (within that) Bank, Ccy, and so on.
  • For a multi-select field - in the field's drop-down list, select one or more values. If required, click one of to change to not one of, and vice versa
  • For an alphanumeric field - type in one or more values, separated by commas. If required, click one of to change to not one of.
  • For a numeric field - in the operator drop-down list above the filter field, select the arithmetic operator (equal to, less than, between, and so on), and enter one or two values as required.
  • For a date field - in the operator drop-down list above the filter field, select the comparison operator (equal to, between, and so on), and use the calendar tool to select one or two dates as required.

To include an existing Balances Definition within a branch:

  1. Select the required branch and select Add Filter Link in the Add Filter button's drop-down list.
  2. In the browse window, select the Balances Definition record to also be applied at that point in the logical structure.
  3. (Optional) To negate the logic in the linked Balances Definition, change the equal to caption above the Filter Link field to not equal to.

To remove a filter row - In the drop-down menu on the filter field caption,select Remove Filter.
