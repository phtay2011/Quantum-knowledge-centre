Applying Resource Security to Bank Reconciliation

When Resource Security is enabled for a Quantum site, Bank Reconciliation filters resources according to the Resource Security settings for the user. The system provides Resource Security on Prior Day and Intraday Matching, using the Bank Accounts as the primary Resources that can be secured.

You will notice security behaviors throughout Bank Reconciliation appropriate to any Resources referenced by the page in which you are currently working. These behaviors are described in the following topics:

  • Quick Summary
  • Resource Security Indications on Matching Pages
  • Resource Security and Automatic Matching

Quick Summary (Resource Security & Bank Reconciliation)

In general, only View access to the Bank Account is required to view all information for Prior Day or Intraday Match queries. However, there are some specific access requirements, as detailed below.

To create adjustments or to match flows requires View/Update access to the Bank Account and to associated resources where Deals are being created. This is normal Resource Security functionality.

Resource Security Indications on Matching Pages

Resource Browse Lists

When a resource* is secured, the browse lists for those resources will list only those items you may access.

*NOTE: This applies to Bank Account, Bank/Counterparty, Entity, and Instrument browse lists, where they occur.

Menu Items and Control buttons

Where a user does not have View/Update access to the relevant secured resources, a menu item will be unavailable. This applies to the page menu, grid menus, and any function buttons in Accounts, Match, and Flows work areas.

For example, if you do not have View/Update access to a Bank Account you have selected, all adjustment and matching menu items and buttons will be disabled.

Displayed messages

User messages are displayed where appropriate - for example:

  • Query Configuration

    If resources such as Currency, Entity, and Counterparty are filtered due to your Resource Security permissions, a message displays informing you that some parameter values have been removed and which resources this applies to.

  • Edit Deal

    If you do not have the correct Resource Security permissions to edit a deal, an error message to that effect displays if you try to do so by clicking on a cash flow for that deal.

  • Delete Imported Data

    While you may delete an imported Cash Flow, Bank Flow, or statement for a Bank Account you have View/Update access to, a warning message displays if:

    • when it was imported, the item you want to delete was one of many in the import file, and
    • you do not have Write access to all Bank Accounts for all items in that import file.

(The message box buttons enable you to choose whether to continue with the Delete action or not.)

  • View Statements

    The following message displays when you have only View Access to the Account.

    You will not be able to modify this record as it has restricted access.

Logged messages

Message Log entries can also occur when you run a matching Script or Chain in any way, as follows.

  • If a script (for an Account you have View/Update access to) creates adjustments when run, that script will partiallyfail and a message will be logged if:
    • an Account Transfer adjustment is created, but you do not have View/Update access to the second Bank Account, or
    • an Adjustment of any sort is created, but you do not have View/Update access to any of the other resources for the Adjustment.

    (Other adjustments that do not break these rules will be created.)

  • Where a script is run automatically (for example via Importation) that script will not run and a message will be logged if you do not have View/Update access to the underlying Bank Account/s.

    (The script does not run at all.)

Resource Security and Automatic Matching

Edit Balance Scripts

The creation/edit or deletion of a Balance Script is impacted by Resource Security when an Adjustment is required - that is, a Bank Account Adjustment or an Actual Cash Flow.

Any Resources* on these forms will be filtered based on the Resource Security of the User.

*NOTE: This applies to Entity, Instrument, Bank/Counterparty, and Bank Account lookups, where they occur.

Edit Detail Scripts

The creation, edit, or deletion of a Detail Script is impacted by Resource Security when an Adjustment is required - that is, any of:

  • Bank Account Adjustment,
  • Actual Cash Flow,
  • Projected Cash Flow,
  • Bank Flow Adjustment, or
  • Bank Account Transfer.

Any Resources on these forms will be filtered based on the Resource Security of the User.

Edit Chains

The creation, edit, or deletion of a Chain is not impacted by Resource Security.

Link Scripts/Chains to Accounts

The Accounts shows only those accounts that the user has at least View access to.

  • View access is required to view/edit Linked Scripts and Chains.
  • View/Update access is required to run Scripts and Chains.

If a user re-saves settings where the Bank Accounts have been filtered, then the settings will only record the Bank Accounts that that user could see at that time. There is no impact on other Links to Bank Accounts that the user cannot see.

Running Scripts/Chains

Generally, a User needs View/Update access to all Bank Accounts affected by a script before that script can be run. Therefore, the Run Scripts/Chains dialog's Accounts shows only the Accounts the user has View/Update access to.

When you run a script that creates adjustments, that script will partiallyfail and a message will be logged if:

  • an Account Transfer adjustment is created, but you do not have View/Update access to the second Bank Account, or
  • an Adjustment of any sort is created, but you do not have View/Update access to any of the other resources for the Adjustment.

NOTE: Other adjustments that do not break these rules will be created.

Where a script is run automatically (for example via Importation) that script will not run at all, and a message will be logged if the initiator does not have View/Update access to the underlying Bank Account/s.
