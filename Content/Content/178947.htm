FX Accounting Rules

Use the FX tab to create and maintain accounting rules for FX deals.

There must be at least one Accounting Rule entry that matches each type of these deals that your organization uses. If more than one, the most specific matching rule entry takes precedence. For example, entries for Instrument groups take precedence over ALL Instrument Name entries (labeled For any ... in the rules grid). Individual instruments that are members of a group will apply settings defined for that instrument before those for the group, before those for an ALL entry.

You can add, edit, and delete specific rule entries using the tab's grid menu, as follows.

NOTE: You cannot add, edit, nor delete inherited entries.

To add a new rule entry

  1. In the FX tab's grid menu, select New (or select an entry row and in the grid menu, select Copy).
  2. In the dialog box under Matching Criteria select one or more Instrument Name, Counterparty Name, and Strategy Name values2,3 or ALL in each case , and select any other customized criteria that may apply for this transaction type.

    NOTE: You cannot specify a combination of criteria already covered by an existing entry.

  3. Under Contingent Accounting Settings optionally select Apply from deal date to start date to post contingent entries for the notional principal amount of the deal (both the deal date entries and maturity date reversal entries will be created when the deal is saved).
  4. Under Accrual Settings* optionally select Accrue forward points if you want to post forward point accruals to Profit & Loss.

    *NOTE: Accrual settings do not apply if Revalue deals is selected and Revaluation Method is IMPORTED SEP, NPV SEPARATE, or NPV SEPARATE ALT.

  5. If you DID NOT select Accrue forward points in Step 4 (or it did not apply - see Note above), SKIP TO STEP 9.
  6. Select the associated Accrual Start Date (the start of the accrual calculation period): Deal Date, Spot Date, or Settlement Date.
  7. Select the associated Forward Point Translation date:
    • POSTING DATE CCY TRANSLATION - to use the relevant FX Rate on the day the accrual entries are posted, or...
    • CASHFLOW DATE CCY TRANSLATION - to use the FX Rate on the day the cash flow to be accrued/amortized is posted. Where the cash flow being accrued is in the future, the posting date should be used.

      TIP: The accrual postings created by End of Day Accrual Processing are translated to base using the Accrual Rate Type setting for the relevant General Ledger record.

  8. Select the associated Posting Date for Forward Point accruals: MATURITY or ACCRUAL START; and select the Posting Currency: BASE or NON BASE.
  9. Under Revaluation Settings, optionally select Revalue deals* if you want to create revaluation postings, and as required, select:
    • Revaluation Method (FORWARD, FORWARD NPV, NPV SPOT, NPV SEPARATE,SPOT, IMPORTED, IMPORTED SEP, or NPV SEPARATE ALT),
    • Revaluation Currency (when applicable) (DOMINANT, BASE, or NON-BASE),
    • Posting Method (REALIZED or UNREALIZED),
    • FX Rate Type, and
    • Interest Rate* and Interest Rate Type*.

      * Mandatory settings when Hedge Accounting applies.

  10. Under Translation Settings, optionally select Apply deal level currency translation to apply deal-level translation to the specified ledger.
  11. Click Save. The new rule entry is added to the grid.

To edit an existing rule entry

  1. Click an entry value in the grid (or select the entry and in the grid menu, select Edit).
  2. In the Edit window, edit the settings as required (see above), and click OK.
  3. Save your changes.

To delete an existing entry

  1. Select the entry in the grid and in the grid menu, select Delete.
  2. In the dialog box, click OK to confirm.
