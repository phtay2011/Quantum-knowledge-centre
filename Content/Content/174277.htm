SWIFT Financial Center Location Codes

For Deal confirmations of Cross Currency or Interest Rate Swap deals matched using SWIFT, a valid SWIFT Financial Center code is required in each confirmations message. The code must be set up as a specific Identifier value in any applicable Location records.

NOTE: You can use only Identifier names that have been set up in External Methods with Method Type set to Mapping and Enable Identifiers selected.

On the Location page, for each location required:

  1. Click Select Identifier.
  2. Select the SWIFT Location identifier name.

    The SWIFT Location identifier field is added.

  3. In that field, enter the valid SWIFT Financial Center code for that location, as indicated below.

    NOTE: For other locations, you may specify OTHR, or a new unique code built in a specific way. See Other Locations, below.

When you specify one of these Locations in the Location field for either leg of a Cross Currency or Interest Rate Swap deal (on the Other Details tab), the corresponding SWIFT Financial Center code will be included in Tag 22B of the MT360 (or MT361) Confirmation message.

Other Locations

SWIFT Financial Center codes are based on the ISO country code and the English name of the location. Additional codes must be built as follows:

  • The first two characters represent the ISO country code.
  • The next two characters have one of the following formats:
    • If the location name is one word, they are the first two letters of the location.
    • If the location name consists of at least two words, they are the first letter of the first word followed by the first letter of the second word.

TIP: Each code must be unique, so rather than add new codes that might cause confusion (for example, Dallas and Dayton in the US), use the code OTHR.
