ECHOS Configuration Parameters (Bank Import)

The Bank Import service can process SWIFT and BAI Bank Statement data messages received from ECHOS.

From Release 6.8.0, the Automatch process is separate from the import process, so that the previous EchosAutoMatch and PerformAutoMatchAfter configuration parameters for Bank file importing are redundant.

To import and process bank statements from ECHOS, the following Configuration file parameters must be set:

  • DataManager = Echos
  • Parser = Echos

In addition, you will need to set the ECHOS-specific option parameters described in this section to control how the system communicates with the ECHOS Listener service. Using these default* settings, the following message types can be processed:

  • MT900, MT910, MT941, MT942, and camt.052 - By default* these messages are treated as Intraday.
  • camt.053, MT940, and MT950 - By default* these message are treated as Prior Day.
  • BAI:
    • Intraday if As-of-Date Modifier = 3 or 4 or (if blank) File Creation Date = As of Date
    • Prior day if As-of-Date Modifier = 1 or 2 or (if blank) File Creation Date NOT = As of Date

*NOTES:

  • You can also use the MAP CONFIG parameter to dynamically use different Configuration files depending on the message type and/or sender. For example, to apply different transformation rules from MT900 and MT910 messages, the following parameters will map MT900 messages to EchosIntradayRules900.ini and MT910 to EchosIntradayRules910.ini:

    MAP CONFIG {C:\BankImport\Echos\EchosIntradayRules900.ini, MT900}
    MAP CONFIG {C:\BankImport\Echos\EchosIntradayRules910.ini, MT910}

  • For Message Type (mType) BAI only, the Map Config has been extended to allow separate rules to be specified for Prior-day and Intra-day using a new mode (PRIORDAY/INTRADAY) indicator.

    The following setup is recommended if you wish to define different rules for Prior-day and Intra-day:

    1. The IntradayMode=ON parameter must be removed from the INI file (otherwise the IntradayMode will overwrite any MAP CONFIG parameters and the suggested solution will not work).

    2. The Import file must have the As-of-Date Modifier field specified, to determine if the applied rules pick up Prior-day or Same-day.

    3. Configure two different rules for Intra-day and Prior-day in the INI:

         c:\users\ECHOS BAI Import\ECHOSPriordayrules.ini, BAI, PRIORDAY
         c:\users\ECHOS BAI Import\ECHOSPriordayrules.ini, BAI, INTRADAY

    Example 1 - BAI - explicit mode parameter for both Prior-day and Intra-day

    The following will use ECHOSPriordayRules.ini for a prior-day statement, and ECHOSIntradayRules.ini for an intra-day statement:

         MAP CONFIG {C:\Users\Echos BAI Import\EchosPriordayRules.ini, BAI, PRIORDAY}
         MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY}

    Example 2 - BAI - explicit mode parameter for Intra Day only

    The following will use ECHOSRules.ini for a prior-day statement, and ECHOSIntradayRules.ini for an intra-day statement:

         MAP CONFIG {C:\Users\Echos BAI Import\EchosRules.ini, BAI}
         MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY}

    Example 3 - BAI - explicit mode parameter for both Prior-day and Intra-day with a Sender.

    The following will use EchosPriordayRules.ini for a prior-day statement and EchosIntradayRules.ini for intra-day statement, and also provides for a rule specific to a Sender:

         MAP CONFIG {C:\Users\Echos BAI Import\EchosPriordayRules.ini, BAI, PRIORDAY}
         MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY}
         MAP CONFIG {C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI,<Sender>, INTRADAY}

    In this example, the EchosIntradayRules firstly applies to a rule for a specified Sender and then for all other Senders picks up the C:\Users\Echos BAI Import\EchosIntradayRules.ini, BAI, INTRADAY rule.
  • If an IncludeFile parameter is present in the main file, then these values will be the defaults used by the dynamically generated options.For example, If an ini file reads:

         IncludeFile=C:\BankImport\Echos\EchosRules.ini
         MAP CONFIG {C:\BankImport\Echos\EchosIntradayRules900.ini, MT900}
         MAP CONFIG {C:\BankImport\Echos\EchosIntradayRules910.ini, MT910}

    ...then the EchosRules.ini file will be used to populate the settings initially and when a MT900 message is processed, the EchosRules settings will be overwritten by the settings contained in the EchosIntradayRules900.ini file.

The above settings work in addition to other existing Bank Import Configuration file settings and the default behavior related to message types and Intraday/Prior Day processing will be overwritten by the IntraDayMode setting if one is present.
