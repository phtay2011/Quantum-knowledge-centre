Using EFT-related workflow templates for ISO20022

Using the initial workflow template specific to EFTs, you can now format an ISO20022 message using the Workflow Template page.

There are some pre-configured EFT workflow templates available. You cannot modify these, but you can modify a copy of them in order to customize them - for example, for a particular bank.

The workflow templates are easily shared by exporting (and importing) as QXT files.

Initial seed files include:

  • FIToFICustomerCreditTransfer - pacs.008.001

    • The pacs.008.001 is the MX equivalent of the MT103 Single Customer Credit Transfer

  • FinancialInstitutionCreditTransfer - pacs.009.001

    • The pacs.009.001 is the MX equivalent of the MT200 Financial Institution Transfer for its own account and MT202 General Financial Institution Transfer.

    It is also used as a replacement for the MT203 Multiple General Financial Institution Transfer, but requires support for batched messages and/or Single File when generating the EFT.

 

The workflow template uses EFT-specific inputs and expressions, and existing data sources.

In this release there are new expressions specific to EFT. For more information, search for the Knowledge Center topic "Using Expression Builder - Functions".

NOTE: Initially we've introduced this functionality for EFT queries that are type cash (not stock) and:

  • are not batched, and
  • use EFT > Generate > File per EFT.

    • We plan to expand this functionality to cover other options in a later release.

EFT Sample Data and Preview tab

A dialogue specific to EFTs allows you to easily import initial EFT sample data.

Saving the workflow template also saves the sample data, so that if you change the workflow template you can see the changes immediately in the Preview tab.

For more information about using the sample data, search for the Knowledge Center topic "Using the Workflow Template", and look under the sub-heading "To add sample data - Import EFT Data".

Publish and use EFT Templates

An external method for type Payments can now be linked to a Workflow Template, whereas previously you could only select an EFT Plugin.

Before a template can be used as part of the EFT Process, the template must be ‘Published’. A warning badge is displayed for both the template and the external method when templates are unpublished; and any EFT Processing runs will be aborted for that reason.

Security

In addition to the standard module security permissions, the following permissions also apply to workflow templates with Purpose set to EFT:

  • EFTs – Required to add workflow templates with Purpose = EFT or modify existing EFT-related workflow templates.

  • Publish/Unpublish EFT Templates – Required to use the main menu item Publish (and to unpublish) the EFT-related workflow template. A published workflow template is always read only.
