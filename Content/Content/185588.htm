Settings, Deal Confirmation Rule Entries

These items define the general actions and related settings that will be applied to Deal Action confirmations meeting the Matching Criteria for the Confirmation rule entry.

Require approval

Select this option to require full approval of the deal action before a confirmation is created.

Clear the option to explicitly NOT require any prior approval.

External Method

Select the applicable External (payment) Method, or PAPER, or NONE. This will be applied to deal actions meeting the Matching Criteria for the rule entry, and is saved on the transaction at deal save time.

  • Only External Method records whose Method Type setting is Confirmation are available.
  • If you select an External Method, Confirmation messages for applicable deals will be sent to the appropriate External Matching system. Response messages received may indicate an automatic Match made by the external system, or an error.
  • PAPER means that Confirmations of applicable deals must be manually matched (no specific External Method nor automatic matching system applies).
  • NONE means "not applicable" -that is:
    • no specific External Method nor automatic matching system applies,
    • confirmations of applicable deals DO NOT need to be manually matched, and
    • the Print setting determines the confirmation output type (PRINT, EMAIL, FILE, or NONE).
    • For more details, see How the Matching and Actioning Settings Work.

Actioning

This setting controls whether or not if the confirmation must be matched before a cash flow can be changed to ACTIONED in Settlement Processing (manually or automatically).

Select one of the following types of action that will be applied to cash flows meeting the Matching Criteria, following deal approval and/or confirmation matching, if and when applicable.

  • Auto Action - to automatically action cash flows, regardless of the Confirmation status.

    CAUTION: This setting automatically actions all future cash flows of the same type, and is therefore unsuitable for transactions that may require any further actions - for example, Pending cashflows for Repo and Yield on Statement deals.

  • Manual Action - to require cash flows to be manually actioned (in Settlement Processing) following deal save or approval, regardless of the Confirmation status.
  • Match & Auto Action (not available If External Method is NONE) - to automatically action cash flows following deal approval, but only after their Confirmation status is set to Manually Matched or Automatically Matched.

    CAUTION: This setting automatically actions all future cash flows of the same type, and is therefore unsuitable for transactions that may require any further actions - for example, Pending cashflows for Repo and Yield on Statement deals.

  • Match & Manual Action (not available If External Method is NONE) - to require cash flows to be manually actioned following deal save or deal approval, but only after their Confirmation status is set to Manually Matched or Automatically Matched. The flow will not be automatically set from Unactioned to Actioned.

See also:How the Matching and Actioning Settings Work

Print Device

Select a network print device name from those available to the Confirmation Processing service (as installed on the appropriate server/s).

TIP: If the printer you want is not listed, consult your System Administrator.

Copies

Enter the number of copies to print to the specified Print Device. The minimum value is 1.
