UploadWhiteList

The default list of permitted file extensions is: dat, doc, docx, xls, xlsm, xlsx, ppt, pptx, pdf, png, jpg, qxt, qid, txt, xml, jrg, csv, sft, out, dat.

System administrators can add or remove file extensions from this list by editing the UploadWhiteList key in the Web.AppSettings.config file. However, there is a hard-coded blacklist of file extensions that can never be uploaded for security reasons - for example, exe and bat.

UseSecurityProtocolTLS12

Enforces the TLS 1.2 protocol for your Site.

Set this key to false to use the TLS 1.0 or TLS 1.1 protocols.

CAUTION: The TLS 1.0 and TLS 1.1 protocols are outdated and suffer from multiple cryptographic flaws. FIS does not recommend that you disable this key.

ValidClientRedirectUris

NOTE: Required for OAuth Authorization.

Configures the valid client identifier and the endpoint to be used by external systems to access Quantum.

This key accepts multiple identifier / endpoint values (delimited by a semicolon, ";") and must be set in the following format:

client1=redirecturi1;client2=redirecturi2;...

Where,

clientn matches the client_id specified in the endpoint of incoming token or authorization-related requests.

redirecturin is the endpoint used by external systems to request access to Quantum for a user who is already logged in. This is required for OAuth Authorization which will utilize the /OAuth/Authorize endpoint.

When the user accepts the request, they are redirected to the redirect endpoint. The access token is provided as part of the response header. Refer to the OAuth Usage Guide for more information.

 

Example

value="Test=http://localhost/apiconsumer/authcallback; Test1=http://localhost/apiconsumer1/authcallback"

Changing IIS Request Filter Settings in IIS Manager

If you want to increase the UploadFileMaximumSize key higher than the default, then you must use IIS Manager to edit the IIS Request Filter setting MaxAllowedContentLength.  The default for MaxAllowedContentLength is 30000000 B (28.6MB).

CAUTION: Increasing the upload size opens up a possible attack vector, so keep any increase in upload size to a realistic size.

To use IIS Manager to edit settings

  1. Open IIS Manager.
  2. Select the website that is serving Quantum.
  3. At the bottom of the page, select the Features View button.
  4. Select Requests Filtering and double-click the icon to open. The Request Filtering window opens.
  5. In the Actions pane on the right, click Edit Feature Settings... . The Edit Request Filtering Settings window opens.
  6. Enter the appropriate value in the field Maximum allowed content length (Bytes).
  7. Click OK.
  8. Restart IIS Manager.
