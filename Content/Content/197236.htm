Workflow Template

Use the Workflow Template page to create output that contains textual information to your specification. Quantum brings together the information and then creates an output by using a section of code that you write (the template) along with inputs and expressions that you define, all in the Workflow Template page. For example, you can set up a template page that substitutes values in outbound regulatory reporting files.

You don't run a template, but instead set up a workflow report that uses the template as a source. Find default templates and your templates by opening a workflow report and in the Select Source dialog box, look for the Values > Templates folder.

Terminology

In this topic we'll use the following terms:

  • Template - a section of code that refers out to the inputs.
  • Inputs - user-defined names of a property that you can use in the template, which later gets expanded.
  • Expressions - short descriptors that stand for mathematical expressions created in Expression Builder. The descriptor makes the template simple to read and the output easy to change.

  • Publish (only applicable to workflow templates with Purpose = EFT) - moves the EFT-related workflow template from an Unpublished state (i.e. editable) to a Published state (i.e. not editable). An EFT run will not be stopped from processing if the workflow template being used is in a Published state.

  • Unpublish (only applicable to workflow templates with Purpose = EFT) - moves the EFT-related workflow template from a Published state (i.e. not editable) to an Unpublished state (i.e. editable). An EFT run will be stopped from processing if the EFT template being used is in an Unpublished state. An "Unpublished" badge appears in the upper right corner of the page when the workflow template is in this state.

Export and import templates

You can export templates using a JSON file format. The workflow template's browse menu includes Import and Export options. The Import option is shown if the Can import data option is checked for the appropriate user in Configuration > User.

You can import templates in QXT and JSON formats via the Utility > Static Data Import Status page as well.

NOTE: QXT is no longer available as an export option, however the import action supports both QXT and JSON formats.

Success or failure message for the import is shown at the top of the page.

Edit schema

The Edit item in the main menu appears when you select Output Format = XML.

Add security

It is important to be able to lock EFT-related workflow templates (EFT templates), when required. Use one or all of the following:

Configuration > User > Module Security tab > Workflow Template module > EFTs:

  • The option EFT in the Purpose lookup is only available to users with this permission.

  • This user has edit access to Unpublished EFT templates and read-only access to Published EFT templates.

  • Users without this permission have read-only access to EFT templates.

Configuration > User > Module Security tab > Workflow Template module > Publish/Unpublish EFT Template:

  • This user only ever has read-only access to EFT templates and does not have the EFT option in the Purpose lookup.

  • This user can publish and unpublish EFT templates.

  • For users with the 'Publish / Unpublish' permission, the page menu contains Publish or Unpublish menu items, depending on the current state.

  • Published EFT templates are read-only for all users.

Users must have module security rights to either EFT and/or Publish/Unpublish in order to see existing EFT Templates.

Template and Preview tabs

The Template tab includes:

  • Inputs grid
  • Expressions grid
  • Validations grid
  • Sort field - select one or more Inputs and Expressions grid items, to order the data coming from source. Your Template output will be generated accordingly.
  • Text box showing numbered lines ("1", "2", etc.). Type the template code that refers to the Inputs and Expressions and that will create the output.
  • Sources grid

The Preview tab enables you to enter sample inputs and see the output.

Reserved words

Reserved words include the following lower-case statements:

  • else - use an (else) keyword within a #if... /if section.
  • #each - repeat the output structure for each input. See below for more information about #each.
  • #if - standard conditional statement; must use a Yes/No type of expression.
  • #unless - conditional statement opposite to #if. Must use a Yes/No type of expression. #if and #unless can be used with an input as a condition. For example, where TicketNumber is an input:
{{#if TicketNumber}}	
    <TicketNumber>{{TicketNumber}}</TicketNumber>
{{/if}

Text box

The general format for the code in the text box is:

<Header>
  <Details>
    <Input>
      {{#if expression1}}<Input2>{{Token 2}} </Inputn2>{{/if}}
    </Input>
  </Details>
</Header>

Instead of creating expressions for formatting strings and numbers, you can format inputs and expressions inline in the text box.

All the formatting options supported by the .NET toString function can be used here.

Text box - #if and #unless

Use the #if and #unless statements in the text box, for example:

{{#if TicketNumberHasValue}}<TicketNumber>{{TicketNumber}}</TicketNumber>{{/if}}{{#unless TicketNumberHasValue}}<TicketNumber>{{TicketNumber}}</TicketNumber>{{/unless}}

Where:

  • TicketNumberHasValue is a short descriptor used in place of an expression in the template
  • <TicketNumber> is an element
  • TicketNumber - is an input defined in the template

Make sure the expressions referred to in the statement are saved in the template.

Text box - else

Use else in the text box, for example:

// {{#if two}} test {{one}} test{{else}}{{one}} {{/if}}

Cannot be used as an input name.

Text box or Preview tab - #each

In the Template text, you can use nested #each statements on expressions and inputs. This enables you to iterate over expressions or inputs and access or iterate on the underlying grouped data.

The general syntax for #each is:

{{#each Expression}}{{#each Input1}}

Semantics:

  • #each statements can be nested to any level.
  • Only the deepest #each statement can be without an input or expression.
  • You can use an @index keyword to access the current index of the innermost #each loop in which it is used. You can also use ~@index keyword.
  • A parent path indicator (../) can be used to access the index number of parent #each loops. For example: @../index for the direct parent, @../../index for the parent above that.
  • Input and Expression names cannot contain an @ symbol.
  • You can use the new Sort field to select one or more items in the Inputs and Expressions grids, so that you can order the data coming from source. Your Template output will be generated accordingly.

First and last item in #each loop

Definitions:

  • @first returns true when the current item in each loop is the first item
  • @last returns true when the current item in each loop is the last item

@../last and @../first refer to their parent same as @../index to any nested level.

Use with conditional statements if and unless, if required

Example input:

{{#each people}}
{{#if @last}}
    <li> {{@first}} {{people}}</li>
{{/if}}
  {{/each}}
-----------
  {{#each people}}
{{#if @first}}
    <li> {{@first}} {{people}}</li>
{{/if}}
  {{/each}}

{   people: [     
	"Yehuda Katz",     
	"Alan Johnson",     
	"Charles Jolley"] 
}

Example output:

    <li> false Charles Jolley</li>
-----------
    <li> true Yehuda Katz</li>

Text box - Comments

You can comment inputs and expressions in the text box.

Use the {{!-- delimiter to mark the beginning of the comment, and the --}} to mark the end.

Example input:

<div class="entry">
	<h1>{{!title}}</h1>
	{{!--<div class="body">
		{{body}}
	</div> --}}
</div>
{
title:"My New Post",
body:"This is my first post!"
}

Example output:

<div class="entry">
	<h1></h1>
</div>
    NOTE:
  • Comment is not displayed in the template preview.
  • Comment is not rendered when the template is executed.

About whitespace and newline

Quantum handles whitespace and newline in a manner consistent with Handlebar.js.

For #each, /each, #if, /if, #unless and /unless:

  • if on a line with only whitespace, don't output the line.
  • if there is any none whitespace on the same line (before or after), output whitespace and none whitespace before and after (if matching), and output newline.
