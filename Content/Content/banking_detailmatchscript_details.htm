Details tab, Detail Match Script page

Use this tab to view and maintain the following parameters for a Detail-level Match script.

Data Scope - Select one of:

  • Include all data - to run the script over all loaded bank account data that has a current Detail Match Status of Possible or Unmatched (to include other Match statuses, select one or more of the Reset options below), or
  • Limit data by Date Range - to run your script over loaded bank account data dated between the Start Date and End Date you specify in the adjacent fields.

Run script over bank flows only (not available if the script contains pass types other than Bankflow Only) - Select this option to run the script over Bank Flows only; clear it to run the script over both cash flows and bank flows (when present).

Automatically match zero netted cash flows - Select this option to automatically mark as Matched any netted group of cash flows totaling zero.

Print interim reconciliation report - Select this option to automatically print the reconciliation report after the script has run. Clear the option if you do not want an automatic report.

Rollback Reconciliation Marks - Select this option to roll back any pre-existing reconciliation marks in the included data, before the script runs. Clear this option to retain pre-existing reconciliation marks.

Reset items set to 'Matched' - Select this option to first change all pre-existing Detail Match statuses from Matched to Unmatched, so you can include such items in your script processing. Clear the option to retain all pre-existing Matched statuses.

Reset items set to Probable - Select this option to first change all pre-existing Detail Match statuses from Probable to Unmatched, so you can include such items in your script processing. Clear the option to retain all pre-existing Probable statuses.

Reset items set to Possible - Select this option to first change all pre-existing Detail Match statuses from Possible to Unmatched, so you can include such items in your script processing. Clear the option to retain all pre-existing Possible statuses.

Sort Order - Select one of:

  • Automatic Sorting - to apply the default sort order within each pass - that is, sort both bank flows and cash flows by the first two indexed fields (in order), as specified for each Script pass (in the Criteria table on the Script Passes tab), or
  • Manual Sorting - to specify your own sort orders for each script pass, in the following grids.

Sort Bank Flows by (available when Manual Sorting is selected) - Use this grid to specify, in order, the fields by which you want to sort bank flows for each script pass.

  • To add a field to the sort sequence - Click Add and select the field in the Browse window. That field takes the next available position in the sort order. Repeat to add other fields further down the order.
  • To remove a field from the sort sequence - Click the x button to the right of the field in the grid row.
  • To change a field's sort direction - Click the field name in the grid; in the Edit dialog box, select Sort by Ascending or Sort by Ascending as required, and click OK.

Sort Cashflows by (available when Manual Sorting is selected) - Use this grid to specify, in order, the fields by which you want to sort cash flows for each script pass. The procedures for this grid are identical to those above for the Bank flows grid.

NOTE: You must specify one sort order for bank flows and one for cash flows. The default in each case is to sort by the first indexed field for the first script pass (as specified in the Criteria table on the Script Passes tab).
