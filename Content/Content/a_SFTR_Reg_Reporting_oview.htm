Securities Financing Transactions Regulation (SFTR) Reporting

Securities Financing Transactions (SFTs) refer to the build-up of leverage, procyclicality, liquidity and transformation, and interconnectedness in financial markets.

At the moment, Quantum supports the EMIR Regis reporting of the eight formats of Repurchase Agreements (Classic, Reverse, Buy / Sell Back, Sell / Buy Back, Fee Based Borrowing, Fee Based Lending, Security Borrowing and Security lending).

NOTE: Limited Fixed Rate Repo instruments are supported for EMIR REGIS.

Components of the SFTR Reporting

SFTR reporting uses the existing workflow Components of the DRS, which are already available for the DRS reporting. Specific template and parsers for SFTR are stored in Quantum:

  • Template - EMIR_REGIS_SFT_RA

  • Parser - EMIR REGIS SFT Response

Collateral tags for reporting SFTs

We provide support for adding collateral tag values in the existing EMIR Regis SFT template. The following tag values are available: "Cash collateral currency", "Identification of a security used as collateral", "Classification of a security used as collateral", "Price per unit", "Maturity date of the security".

Reporting an SFTR transaction

Reporting an SFTR transaction is similar to reporting derivative transactions. You can find more information on how to report the SFTR transaction on the How to Report Trades page.

Disclaimer

  • Only mandatory (M) and conditionally mandatory fields are reported.

  • Optional (O) and not applicable (-) fields are not reported.

  • Floating Rate, Commodities, Margin lending related fields are not reported as their support is not available in Quantum.

  • Margins and Re-use details are not reported.

2.18

General collateral indicator

Indication whether the secured financing transactions is subject to general collateral arrangement. ‘GENE’ is populated for general collateral. General collateral specifies a collateral arrangement for a repurchase transaction in which the security lender may choose the security to provide a collateral with the cash provider among a relatively wide range of securities meeting predefined criteria. ‘SPEC’ shall be populated for specific collateral. Specific collateral specifies a collateral arrangement for a repurchase transaction in which the buyer requests a specific security commodity (individual ISIN) to be provided by the seller.
