EFTs - Reporting, Processing, and Reprocessing Separated

Quantum EFT Reporting, Processing, and Reprocessing functions have been split into three pages, accessible via different menus (namely, the Reporting, Processing, and Utility menus, respectively).

This greatly reduces the risk of accidentally generating duplicate payments by processing query results for already SENT items.

Implementation

EFTs Processing page

On this existing page:

  • the Flow Status filter value is now read-only and set to UNSENT, so that only EFTs that have not yet been created can be included in a Processing run;
  • the Advanced Option Repeat Earlier Run has been removed (this function has been moved to the EFTs Utility page); and
  • existing validation remains unchanged.

EFTs Reporting page

On this new page, accessed via the Reporting menu:

  • no EFT files can be generated regardless of the Flow Status setting (UNSENT, SENT, or ALL);
  • the Advanced Option Repeat Earlier Run is available for re-running previous queries;
  • validation is otherwise as for EFT Processing; and
  • a new RUN HISTORY section has been added to the page menu, containing a View command that will show a log of recent EFT Processing and Reprocessing Update runs.

    (The 50,000 most recent records will be listed. Preview runs and EFT reporting runs are not included.)

EFTs Re-processing Utility page

On this new page, accessed via the Utility menu:

  • the Flow Status filter value is read-only and set to SENT, so that only EFTs that have already been created can be included in a Processing run;
  • the Cancel Requests option is NOT supported;
  • the Batching option is available (as for EFT Processing);
  • the Advanced Option Repeat Earlier Run is available for re-running previous queries; and
  • validation is otherwise as for EFT Processing.

Data Transfer Location page

A Data Transfer Location record with Purpose set to Eft applies to the output from both EFTs Processing and EFTs Reprocessing runs.

XML data files

Create XML data files from the EFTs - Query page in the Reporting menu item and not from the EFTs - Query page in the Processing menu item.

Module Security Changes

For the EFTs - Processing module, the permissions:

  • Inquire, Configure Public Queries and Views, View File Destination, and Cancel Messages apply as before; while
  • Flow Status and Repeat Earlier Run are now redundant (they are replaced by access to either EFT Utility or EFT Processing, respectively).

For the new EFTs - Reporting module, the same permissions apply as above plusConfigure Private Queries.

For the new EFTs - Utilities module, the applicable permissions are Inquire, Configure Public Queries and Views, and View File Destination.

Scheduler Changes

You can now use the Scheduler's Reporting Task Provider to schedule EFT Reporting query runs, specifying any Public EFT Queries and Views defined via the EFT Reporting page. However, you cannot schedule EFT Reprocessing (Utility) runs.

(The use of Scheduler's Processing Task Provider to schedule EFT Processing Preview or Update runs is unchanged.)

Upgrading from an Earlier Quantum Six Release

If you upgrade to this release from an earlier one, existing EFT Processing:

  • queries - are all created as Public queries in EFT Reporting;
  • views - are all retained in EFT Processing AND created as Private or Public views in EFT Reporting and Reprocessing (Utility), with any Crystal Views also mapped to those modules (EFTRPT and EFTRERUN) in Crystal Mapping;
  • queries and Previous Runs with Flow Status set to SENT and/or Repeat Earlier Run selected - are moved to Reprocessing;
  • queries and Previous Runs with Flow Status set to ALL - are deleted from EFT Processing;

History

Previously, EFT reporting, processing, and re-processing functions were all executed using the EFTs Processing page.
