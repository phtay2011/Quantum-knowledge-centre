Queries and Resource Security

NOTE: This topic describes how Resource Security affects query parameters and the query results. See Applying Resource Security to Queries and Reports for information on how queries are secured as a Resource.

The following rules are applied when Resource Security filtering is used with Quantum queries.

Generic Rules

Generally, all Resource parameter fields will have full filtering applied. If the User creates a parameter not using a securable field, the result will still be filtered based on the User's resource permissions and therefore only return results the User is permitted to view. However, for a Resource parameter field to be filtered, it must have a direct link back to the resource table. For example, if you select a:

  • Counterparty - the lookup will be filtered;
  • Counterparty owner - a Counterparty that can be secured, and that would be filtered;
  • Counterparty fax number - a data field that can be directly linked to a specific Counterparty, and that would be filtered; or...
  • Counterparty currency - not a data field in the Counterparty record as it references the currency table; therefore, since there is no direct link between Counterparty and currency, the lookup would not be filtered. However, if the Currency resource is enabled then the Counterparty Currency list will be filtered based on the currencies the user has permissions for.

Resources

Individual Resources (as opposed to Resource groups) will be used for data filtering when creating the result set and/or running the query.

NOTE: You can create a result set that contains individual Resource items that the User is permitted to view for a Resource group that the User is not permitted to view.

Parameter drop-down lists

All Resource parameter selection lists (individual Resource and Resource group fields) will be filtered. However, not all Resource parameter fields are used for filtering the data during the result creation. For example, there are two different types of parameter fields that can be specified for a query:

  • Pre-processing - fields used in creating and Resource filtering the report results for of the results.

    Resource Security applies to these parameter fields, even if not specifically used (as filters) for creating the results. Generally in such cases, the system assumes ALL - which, with Resource Security, means all records that the User is permitted to view and/or modify.

  • Post-processing - fields used to filter the results after they are created (not before). All post-processing Resource related fields will be filtered as usual.

Recalling Saved Query Configurations

When recalling saved Query configuration parameters, the parameter information is filtered, not the actual saved parameter record. Invalid Resource items are removed from the fields when the parameter is recalled. If any information is removed due to Resource Security, a message will be displayed informing you that this has occurred, and any fields that were removed are listed.

Query Result Views

Re-saving a recalled View will save the View parameters as they are displayed. This is the case even when the User has not made any physical changes to the View and the only changes were due to Resource Security removing invalid items.

Recalling Report Results and Re-running

When recalling a saved report, the report results will be filtered based on the View parameters and User permissions (as usual). However, if the User re-runs the report, the new report result set will be filtered based on the Query configuration parameters and User permissions. Other data in the original report result set that may have been accessible by other Users will be lost.

Specific Queries - Rules and Exceptions

Accruals

Resource Security is applied against both Bank Accounts and Deals.

In Processing mode:

  • if the Preview function is used - the result set includes all data the User has Read access to, while
  • if the Update function is used - the result set includes only data the User has Write access to.

In Reporting mode, if the Posting type report option is:

  • cleared - the result set includes all data the User has Read access to, or...
  • selected - the result set includes only data the User has Write access to.

Bank Account Statements

In Reporting mode, the result set includes all data the User has Read access to.

In Processing mode, for new statement runs or re-runs of previous statements:

  • if the Preview function is used - the run will be performed for all banks and bank accounts the User has Read access to, while...
  • if the Update function is used - the run will be performed only for banks and bank accounts the User has Write access to.

Cash Flows Query

Resource Security is applied against both cash flows and Deals.

When creating a broker cash flow (projected or actual), the Broker field is saved in the database as the Counterparty (even though it is displayed as Broker). The Broker database field represents the Counterparty of the Broker cash flow. For this reason, when the Counterparty Filtered Resources option is set to Enabled (on the Data Security page), the Broker cash flow browse will always be filtered by Broker, even if the Broker category option is selected (under Additional Counterparty Categories for Deal Browse Filtering on the Data Security page), is disabled in the Defaulting Rule.

Cost of Funds

Resource Security is applied against both Bank Accounts and Deals.

Currency Translation

In Processing mode:

  • if the Preview function is used - the result set includes all data the User has Read access to*, while...
  • if the Update function is used - the result set includes only data the User has Write access to*.

In Reporting mode, if the Posting type report option is:

  • cleared - the result set includes all data the User has Read access to*, or...
  • selected - the result set includes only data the User has Write access to*.

*NOTE: When the User has access to a G/L Owner, then any sub-entities of that G/L Owner will be included, irrespective of the User's access permission to those individual sub-entities.

Limits Query

For the limit amounts to be useful, the full utilization amount for a given Counterparty and/or Entity is provided in the result set. However, drill down to the deal level will have Resource Security filtering applied.

Position

In Processing mode:

  • if the Preview function is used - the result set includes all data the User has Read access to, while...
  • if the Update function is used - the result set includes only data the User has Write access to.

In Reporting mode, if the Posting type report option is:

  • cleared - the result set includes all data the User has Read access to, or...
  • selected - the result set includes only data the User has Write access to.
