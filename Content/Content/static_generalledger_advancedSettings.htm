Advanced Settings, General Ledger Properties page

* Starred options are unavailable for Posting Entities (those not G/L Owners).

Allow postings to suspense* - Select this option to have the system create Suspense Account entries for any transactions that do not have defaults defined.

NOTE: Where user suspense accounts are defined, these will be used before the system suspense account.

Allow backdated deals* - This option controls when backdated deals are posted to the G/L. When the option is:

  • selected - backdated G/L entries will be posted on either the actual date or the first open date of the accounting period, whichever is later, or...
  • cleared - backdated G/L entries will be posted on Today's date.

    NOTE: The first date of the accounting period is based on both the type of G/L Close (interim or full) and the individual user's rights to post into an interim close period.

Enable G/L Audit sequence number (only available for G/L owner entities) - Select this option to enable the recording of audit sequence numbers for all G/L entries for the entity. When selected, you can use the Assign Audit Sequence page to assign a unique number to each G/L entry for the entity, such that all its entries are numbered in an unbroken sequence, suitable for auditing purposes.

Post G/L entries for Initial Margin* - Select this option to enable the system to generate Initial Margin G/L entries for all Broker accounts set up for this G/L Owner.

Extend accrual end date to include non business days* - Select this option to include subsequent non-business days in the calculation of accruals. The accrual amounts will be posted to the G/L on the first prior business day. For example, when this option is selected and you run accruals on a Friday, the total accrual amount calculated includes the accruals from Saturday and Sunday and is posted to the ledger on Friday.

When this option is selected, the following three options are made available.

  • Exclude non business days in next period - Select this option to enable the calendar month end rule to take precedence over the non business days rule when calculating accruals. For example, if the calendar month end date falls on Saturday, all accruals run on Friday include Saturday, as it is month end. However, Sunday's accruals will be included in the following calendar month.
  • Use accounting period not calendar month - Select this option to enable accounting periods for this ledger to take precedence over the non-business days rule. For example, if the calendar month end date falls on a Saturday and the accounting period end date for the ledger is Friday, then the accrual amount is as at end of day Friday only. Saturday and Sunday accruals will be included in the following accounting period.
  • Use base currency holidays only - Select this option to observe only those non-business days that are specified in the base currency.

    NOTE: If they are selected, the other two options take precedence over this option.
