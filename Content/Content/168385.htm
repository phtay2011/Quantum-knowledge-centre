EFT Message Output

TIP: EFT message output may be suppressed in some circumstances. This is controlled by the Counterparty Payment Category setting of EFT Not Required. on the Counterparty Bank Account record. For example, this is used with CLS (Continuous Linked Settlement) processing.

Output Handling

EFT processing output is handled in one of two ways, depending on the Type Export setting in the External Method Sources record for the relevant External Method (Payment type).

  • Create File (the default)

    Flat files are output directly to a file in the File Destination location, which you can set to any network location you have access to OR a predefined Data Transfer Location that has Purpose set to Eft (File or Queue type).

    For applicable EFT Processing runs, you can also:

    • set the Generate option to create a Single file, File per EFT, or File per day, and
    • (if available) set the Batch Messages option to:
      • NONE - for no batching of messages (the only option when Generate is File per EFT),
      • PAY - to batch MT101 and MT202 Payment messages with the same bank account and currency,
      • RECEIVE - to batch MT210 Advice-to-Receive messages with the same bank account and currency, or
      • BOTH to batch both Payment messages and MT210 Advice-to-Receive messages, as above.
  • Publish To ECHOS (SWIFT and other External Methods where supported)

    EFT messages are sent via ECHOS, using the Data Publisher and ECHOS Gateway services (no files are created).

    In an EFT processing run for the relevant External Method, File Destination, File Name, Generate, and Batch Messages option fields are not shown.

    TIP:

    • You can use the EFT Messages page to view (and possibly cancel) outgoing SWIFT EFT messages (including those published to ECHOS).
    • You can use the ECHOS Messages page to view message status updates, and ACK and NAK response messages from ECHOS.
    • On the Settlements processing page, you can select an individual flow and mark it for reprocessing, using the Mark as Exception action.

Output File Naming

(Not applicable if the Type Export setting is Publish To ECHOS in the External Method Sources record for the relevant Payment Method.)

The default file name format is obtained from the External Method Sources record. For example, for SWIFT, this is usually *.SFT, which produces file names of the form nnnnnnDDMM.SFT, where:·

  • nnnnnn is the Run Number, in the range 000001 to 999999,
  • DDMM gives the Value Date day and month, and
  • SFT is the required file extension.

NOTE: The default file name will be superseded if a value is specified in the Output File Name field. This value may also include the asterisk wildcard symbol. For example, if you enter A*.TEX; and both From Value Date and To Value Date are 31 October 2009; and the system-generated run number is 456, the output file name will be A0004563110.TEX.

Set up a retry timer

When publishing EFT messages to ECHOS using the Data Publisher Service, you can now set up a timer associated with the EFT messages. If the connection drops while the system is sending the EFT message, the timer wh6>s a set number of minutes and then resends the message.

To turn on the timer, add "retryerrorqueuetimer" to the AGDataPublisherService.config.

Example: In AGDataPublisherService.config enter <add key="retryerrorqueuetimer" value="30" /> This resends the EFT message 30 minutes after the failed message.

To view messages and errors, open Processing > EFT and run a query using the EFT Messages page.
