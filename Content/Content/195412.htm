Cut-off Time functions in expressions

When configuring Quantum report result views, you can now include functions that provide Cut-Off date and time information, as follows.

These functions can also be used in:

  • appropriate Processing page result views (particularly Settlements) - to add columns giving cut-off date and time information, and
  • workflow page Variants - for example, to apply cut-off date validation. See the Validations grid example under Page Customization enhancements.

Cut-Off Time, Rules

Using Expression Builder in Calculated Columns

DealCutOffDate

Description

Returns the minimum deal cut-off date from today based on the best-matching Cut-Off Time rule with a Scope of Deal, where:

  • if cut-off time not yet passed - returns (Today + (Cut-off days) business days), where Today is based on the rule location and business days are based on the locations of the deal Entity, Counterparty, and Bank Account, and Currency;
  • if Cut-off time is passed, returns (Today + (Cut-off days +1) business days), where Today and business days as above; or
  • if no matching rule is found - returns Today's date at the user's location.

Format

DealCutOffDate(transactionType, dealType, instrument, counterparty, entity, currency, bankAccount, otherCurrency)

Arguments

Each argument may also be a field name within square brackets (which will provide the actual value in that field at run-time) or a valid nested function.

transactionType - a valid Transaction Type to be matched (with valid abbreviations allowed): "FX", "Money Market", "UNH", and so on)

dealType - "Deal, "Exposure", "Quote", or "Request"

instrument - one Instrument name

counterparty - one Counterparty name

entity - one Entity name

currency - one Currency name

bankAccount (optional) - blank, "ALL" or one account name

otherCurrency (optional where applicable) - blank, "ALL" or one or more* Currency names

Examples

DealCutOffDate("AT", "Deal", [Instrument], [Counterparty], [Entity], [Currency], [Bank Account.Account Number] )

DealCutOffDate("AT", "Deal", "Bank Account Transfers", "XYZ Corp", "SBU Entity", "GBP", "", )

SettlementCutOffDate

Description

Returns the minimum flow cut-off date from today, based on the best-matching Cut-Off Time rule with a Scope of Settlement, where:

  • if cut-off time not yet passed - returns (Today + (Cut-off days) business days), where Today is based on the rule location and business days are based on the locations of the deal Entity, Counterparty, and Bank Account, and Currency;
  • if Cut-off time is passed, returns (Today + (Cut-off days +1) business days), where Today and business days as above; or
  • if no matching rule is found - returns Today's date at the user's location.

Format

SettlementCutOffDate(flowType, transactionType, instrument, counterparty, entity, currency, paymentMethod, flowDirection, bankAccount)

Arguments

NOTE: Each argument may also be a field name within square brackets (which will provide the actual value in that field at run-time) or a valid nested function.

transactionType - a valid Transaction Type to be matched (with valid abbreviations allowed): "FX", "Money Market", "UNH", and so on)

flowType - "CASH", "STOCK, or "NET"

instrument - one Instrument name

counterparty - one Counterparty name

entity - one Entity name

currency - one Currency name

paymentMethod - one External Method name

flowDirection - "Payments", "Receipts", or "Both" ("P" or "R" or "B" are also valid)

bankAccount (optional) - blank, "ALL" or one account name

Examples

SettlementCutOffDate("FX", "CASH",[Instrument], [Counterparty], [Entity], [Currency], [Bank Account.Default Payment Method], "Payments", [Bank Account.Account Number] )

SettlementCutOffDate("AT", "CASH", "Bank Account Transfers", "XYZ Corp", "SBU Entity", "EUR", "SWIFT", If([Amount] > 0, "R","P"), "")

IsDealCutOffTimeNear

Description

Returns true or false depending on whether the cut-off time, based on the best-matching Cut-Off Time rule with a Scope of Deal, is within the specified number of minutes (MM say) of the deal location's time or not, where:

  • if (Cut-off time - deal time) is positive and less than or equal to MM - returns true,
  • if (Cut-off time - deal time) is negative, zero, or greater than MM - returns false, or
  • if no matching rule is found - returns false.

Format

IsDealCutOffTimeNear(minutes,transactionType, dealType, instrument, counterparty, entity, currency, bankAccount, otherCurrency)

Arguments

minutes - the number of minutes defining "near"

Each of the following arguments may also be a field name within square brackets (which will provide the actual value in that field at run-time) or a valid nested function.

transactionType - a valid Transaction Type to be matched (with valid abbreviations allowed): "FX", "Money Market", "UNH", and so on)

dealType - "Deal, "Exposure", "Quote", or "Request"

instrument - one Instrument name

counterparty - one Counterparty name

entity - one Entity name

currency - one Currency name

bankAccount (where applicable) - blank, "ALL" or one account name

otherCurrency (where applicable) - one Currency name

Examples

IsDealCutOffTimeNear("30", "AT", "Deal", [Instrument], [Counterparty], [Entity], [Currency], [Bank Account.Default Account Number] )

IsDealCutOffTimeNear("45", "Actual Cashflows", "Deal", "Std AC Deals", "XYZ Corp", "SBU Entity", "AUD", "OzBank Settle", )

IsSettlementCutOffTimeNear

Description

Returns true or false depending on whether the cut-off time, based on the best-matching Cut-Off Time rule with a Scope of Settlement, is within the specified number of minutes (MM say) of the flow's deal location's time or not, where:

  • if (Cut-off time - deal time) is positive and less than or equal to MM - returns true,
  • if (Cut-off time - deal time) is negative, zero, or greater than MM - returns false, or
  • if no matching rule is found - returns false.

Format

IsSettlementCutOffTimeNear(minutes, flowType, transactionType, instrument, counterparty, entity, currency, paymentMethod, flowDirection, bankAccount)

Arguments

minutes - the number of minutes defining "near"

Each of the following arguments may also be a field name within square brackets (which will provide the actual value in that field at run-time) or a valid nested function.

flowType - "CASH", "STOCK, or "NET"

transactionType - a valid Transaction Type to be matched (with valid abbreviations allowed): "FX", "Money Market", "UNH", and so on)

instrument - one Instrument name

counterparty - "ALL" or one or more* Counterparty name

entity - one Entity name

currency - one Currency name

paymentMethod - one External Method name

flowDirection - "Payments", "Receipts", or "Both" ("P" or "R" or "B" are also valid)

bankAccount (where applicable) - blank, ALL" or one account name

Examples

IsSettlementCutOffTimeNear("30", "CASH", "AT", [Instrument], [Counterparty], [Entity], [Currency], [Bank Account.Default Payment Method], "Both", [Bank Account.Account Number] )

IsSettlementCutOffTimeNear("30", "CASH", "Bank Account Transfers", "Std BA Instr", "XYZ Corp", "SBU Entity", "EUR", "SWIFT", If([Amount] > 0, "R","P"), "ALL")
