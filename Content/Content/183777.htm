Delete, Reset, and Reject

You can

  • Delete a deal in full,
  • reject an approved deal (change the status from approved to rejected), or
  • reset an approved deal.

Create the deal's Audit History and Image

To create the audit history and support functionality such as creating a rollback, viewing commodity swap legs, reporting at a given date, and calculating Bank Statement opening balances, Quantum records all actions performed and changes affected to the deal.

Certain actions that occur in the life of a deal create a record of the deal, known as a deal image. This is a snapshot of significant aspects of the deal as it was before the action occurred.

A deal image is primarily used for calculations based on the deal as it was at that time. This information is used by rollbacks, for example.

An image is not the same as an Audit History, which itemizes what the situation was before an action, and what it was afterwards. See the Details tab of the Deal Details utility.

NOTE: Not all fields are imaged.

For more information about how to view the deal audit history, see Viewing Deal Audit History and Imaging.

Notes, Attachments, and Comments in deal's Audit History

The Deal Audit History for individual Dealing pages displays the following details for notes and attachments:

  • Action performed on the note or attachment (Add, Update, or Delete),
  • Deal Number the note relates to,
  • date and time of the action, and
  • user who performed the action.

The deal image stores all comments, notes, and attachments through the life cycle of a deal. You can access the saved comments, notes, and attachments for a previous version of a deal by loading the deal image.

NOTE:

  • You can only add, edit, or delete comments, notes, and attachments for the Current Version of the deal.
  • When an existing comment or note is edited, the edited version is visible on the Current Version of the deal and on all future versions until it is edited or deleted, after which, the previous version of the note or comment is visible only in previous images.
  • When an existing comment or note is edited several times for the same deal image, only the latest version is visible on the Current Version of the deal and on all future versions until it is edited or deleted.
  • When an existing comment, note, or attachment is deleted for a deal image, it is visible only the deal images that are older than the one that it has been deleted from.
  • Existing deal attachments are visible and moved across all images until being deleted.
