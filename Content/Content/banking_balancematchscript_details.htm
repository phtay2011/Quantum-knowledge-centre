Details tab, Balance Match Script page

Use this tab to view and maintain the following parameters for a Balance-level Match script.

Data Scope - Select one of:

  • Include all data - to run the script over all loaded bank account data that has a current Balance Match Status of Possible or Unmatched (to include other Match statuses, select Reset balance matches - see below), or
  • Limit data by Date Range - to run your script over loaded bank account data dated between the Start Date and End Date you specify in the adjacent fields.

Tolerance Level - The maximum amount by which balances may differ when allowing a Balance-Level match. An adjusting entry will be generated for the actual difference. If the balances differ by more than this amount, a Balance-Level match will not be achieved.

TIP: When you enter a non-zero value, adjustment-related fields are displayed (see below). If zero, a Balance-level match is allowed only when the balances are equal, so no adjustment is required.

Reset balance matches - Select this option to first change all pre-existing Balance Match statuses to Unmatched, so you can include such items in your script processing. Clear the option to retain all pre-existing Balance Match statuses.

Rollback reconciliation marks - Select this option to roll back any pre-existing reconciliation marks in the data you are including in your script processing. Clear the option to retain pre-existing reconciliation marks.

Print interim reconciliation report - Select this option to automatically print the reconciliation report after the script has run. Clear the option if you do not want an automatic report.

Calculate Balance Using - Select the balance field to match against when running the script: Ledger Balance or Available Balance.

[The following items appear only when Tolerance Level is greater than zero.]

Adjust Using - The type of automatic adjustment that will be created for the difference, to complete a Balance-Level match when the balances differ by no more than the Tolerance Level value. Select:

  • BANK ACCOUNT ADJUSTMENT - to create a Bank Account Adjustment (enter the required Adjustment Detail items), or
  • ACTUAL CASH FLOW - to create a new Actual Cash Flow (enter the required Adjustment Detail and Analysis Code items).

Dates As At - the default Deal and Settlement Date for any adjustment: Today's Date (the default) or Statement Balance Close (as imported, if available).

Also adjust for external cash flows - Select or clear this option to allow or prevent an automatic Balance-Level adjustment entry when an external cash flow is involved, respectively.

TIP: Clearing this option prevents the Treasury position being affected by automatic adjustments created to reconcile external (rather than Quantum) cash flows.

In most cases where External Cash flows do not exactly match, you should notify the organization or system from which they were imported.

For more details, click on an item or tab when the Help panel is open.
