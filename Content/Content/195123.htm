Import function on Static Data pages

(First introduced in Release SP 6.7.1, August 2018.)

You can now import deal data files directly from Quantum Counterparty and Entity pages, subject to the Module Security and Resource Security permissions assigned to your User Name.

NOTE: The importing of static data files using the import Service and the Upload File function on the Static Data Import Status page remain unchanged (see History below). The Import Static Data operational setting applies only to that importing method (NOT to the new Counterparty and Entity page function).

Implementation

User page

On the User Details tab under User Rights, the Can import data option has been added, as highlighted below. This option must be selected for each User who needs the Import function to be available on Static Data (and Dealing) pages.

This option has also been added to the User tab on the Static Data Transfer spreadsheet.

Module Security

Each User who needs to use the Import function on the Counterparty and Entity pages must be assigned (or inherit) Inquire, Add, Update, and Delete Module Security permissions for each of those static data modules (or be exempt from Module Security).

Resource Security

If Resource Security is enabled, the data a user can import will be subject to the permissions applicable to that User name and the relevant resources in each case.

Counterparty and Entity pages

Subject to the required security settings (see above), an Import function will appear in the Browse menu of the Counterparty and Entity pages, as highlighted in the example below.

When you select this function, an Import dialog box is displayed for you to browse for and enter a File to Upload name and location. This file must contain either Counterparty or Entity static data records, in a supported format (QXT, QID, or tab-formatted).

When you enter a file name and click OK, data records in the file are imported into the Quantum database provided the data is valid and you have the required security rights in each case. When the import operation is finished, you will receive a status notification (see below), where:

  • Static data file imported successfully indicates that all deal records were imported from the file,
  • Static data file imported Partially indicates that only some records were imported, or
  • Static data file failed to import indicates that no records were imported.

If Static Data Approval applies to a record you imported in this way, the Approve dialog box shows your User name as the Creator.

Notifications

The Notifications list shows your recent Static Data importation run results, as in the following examples.

The Notifications page will show the numbers of records imported and that failed to import for each of your static data importations, as in the example below.

Clicking on an imported successfully notification returns you to the page from which you did that import. Clicking on a failed to import or imported partially notification downloads the Import Error log file for that run, which details the reasons for failed records.

In the run results of Message Log queries that include static data Importation runs a user initiates from the Counterparty or Entity page, that User Name is shown. Restrictions and Limitations

The Import function is not currently supported for any other Quantum and Analytics Static Data pages.

History

Previously, the importing of deal data could be done only from the Static Data Import Status page, and only by users with Upload File Module Security rights for that module.

TIP: This method still applies. It uses a service that picks up files arriving in the file system and uploads them using the Service Account User, who is typically exempt from Module and Resource security. So, for static datas imported in this way, both the Approve dialog box (if applicable) and the Message Log indicate the Service Account user.
