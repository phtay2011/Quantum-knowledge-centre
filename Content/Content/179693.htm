Field Keywords in Expressions

Field Keywords can be used to assign values to bank flow record fields prior to being imported into Quantum. There are two types:

  • Read-only Statement Header fields, whose values can be used, but not changed; and
  • Modifiable Transaction detail fields, whose values may be changed, and whose assignment code names can be used in data manipulation rules.

The TX and TL field codes view the text in different ways:

  • TL returns the text with any line delimiters (CRLF's) still in the text. Functions such as the LINE() below can be used to extract individual lines of text.
  • TX returns the text without line delimiters. This is useful where you want to search through the text and the words you want to search for may be split across line boundaries.

2

The EXCLUDE marker field can be used to control the loading of individual transaction records. When set to true, the transaction record will not be loaded.

3

The IN, EN, and TN field codes were introduced in Release 6.6.1.

IN must be set to an existing Quantum Actual Cash Flow Instrument.

EN must be set to an existing Quantum Entity.


Other fields specific to particular data formats are defined under Parser parameter and Bank Import file formats.
