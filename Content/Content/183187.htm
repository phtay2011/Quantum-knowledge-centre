Resetting a deal

When at least one deal action has been approved on a deal:

  • the financial fields show up as read-only, and
  • the Reset menu option is made available in the deal's page menu for users who have the required permission, namely:
    • the Module Security Setting for Change enabled for the User and Module, and
    • Can change approved deals selected for the User.
    • This applies to the page menu in workflow pages created in the Enterprise Workflow (EW) and to core pages.

NOTE: The Reset option is:

  • disabled when the user has the required permission, but the deal is not yet in a state that needs resetting, or
  • always hidden for users who do NOT have the required permission.

The Reset page menu option applies to:

  • all Dealing pages except Fees, Internal Swap, and Unit Holding, and
  • the Actual Cash Flow, Cash Transfer, and Stock Transfer pages in the Cash Management menu.

To reset a deal

NOTE: Resetting a deal affects the EFTs and you lose any customization of the deal.

  1. Select the Reset option to display a warning message as follows:

    This warning appears if:

    • cashflows are in either the EFT Preparation or Completion phase,
    • cashflows are in an Amended state, or
    • someone has customized the deal map.

    Check the list of flows and customization before proceeding.

  2. If you select Yes, the system will:
    • re-enable the financial fields,
    • set the deal to not customized, and
    • regenerate the Deal Map when you save the deal.

In addition:

  • If a deal cash flow is a member of a net - the cash flows that change are automatically removed from the Net and the Flow Status on both the removed cash flows and the Net is set to UNACTIONED with the matching Flow Lifecycle set to Awaiting Settlement Action (as when manually removing flows from a Net), and
  • If an EFT has already been created:
    • the Flow Lifecycle on the cash flow is set to Amended and the EFT (Pay/Rec) status to Exception, and
    • if a Net is involved - the EFT (Pay/Rec) status is set to Exception for both the removed cash flows and the Net (all remaining cash flows in the Net) should get the EFT (Pay/Rec) status is Exception.

For more details, see Changing a Flow after Payment has been Sent.

NOTE: Without selecting Reset, you may still:

  • make changes to non-financial fields (these fields are always available and changing them will not lead to Deal Map regeneration), and
  • make further customizations (this can always be done, as the Deal Map grid is editable).
