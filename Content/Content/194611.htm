Import the Groups a User belongs to, and the Users in a Group

(First introduced in Release SP 6.7.1, August 2018.)

In User or Group static data to be imported into Quantum Six, you can now specify:

  • for each User - the Short Names of Groups that user belongs to, and
  • for each Group - the Short Names of Users in that Group.

This is supported for all methods of importing static data (spreadsheet or formatted file).

Implementation

In the Static Data Transfer spreadsheet:

  • the User worksheet now has a Groups column, which may contain zero, one, or more Group names, with multiple names separated by commas, and
  • the UserGroup worksheet now has a Users column, which may contain zero, one, or more User names, with multiple names separated by commas.

In QID or QXT files to be imported:

  • User records now have a Groups data row, which may contain zero, one, or more Group names, with multiple names separated by commas, and
  • UserGroup records now has a Users data row, which may contain zero, one, or more User names, with multiple names separated by commas.

In tab-separated files to be imported:

  • User data files now have a Groups field name in the Field Description row, and the corresponding position in following data rows may contain zero, one, or more Group names, with multiple names separated by commas, and
  • UserGroup data files records now have a Users field name in the Field Description row, and the corresponding position in following data rows may contain zero, one, or more User names, with multiple names separated by commas.

Examples

(Relevant changes highlighted)

Spreadsheet

QXT file format

@User
$new
LogOnName = JohnDoe
ShortName = JDoe
FullName = Johnathan Z Doe
EmailAddress = john.doe@ourcorp.com
Groups = Admin,Traders,Payments
Location = NONE
Dealer = Y
Approver = Y
ServiceAccount = N
CanPostIntoAnInterimClosedPeriod = N
LdapExempt = Y
$Insert

@UserGroup
$new
GroupName = Receipts
ShortName = Rcpts
FullName = Accounts Receivable
Users = FredBloggs,LeeChing,AnnaBell
$Update

History

Previously, while you could import both users and user groups, you could not import the information about which groups a user belonged to, nor which users belonged to a group, respectively.
