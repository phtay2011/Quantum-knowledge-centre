Changing Actual Cash Flow Counterparty Instructions

On the Cash Flows tab, you can optionally view and/or change the Counterparty Delivery Instructions for a selected cash flow (subject to applicable Module Security and Resource Security rights and the Restrictions below).

This includes Actual Cash Flow deals generated from CashXplorer or Bank Reconciliation matching.

To view Counterparty Instructions for a single cash flow when the EFT has already been created:

  1. Select the required cash flow and in the grid menu select View.

    The Counterparty Instructions page is displayed in a new window (as when the same page is invoked from Settlements Processing). All details are read-only.

  2. When finished, close the window.

To view and change the Counterparty Instructions for a single cash flow when the EFT has NOT been created:

  1. Select the required cash flow and in the grid menu select Change.

    The Counterparty Instructions page is displayed in a new window.

  2. Optionally change the Counterparty Instructions mode via the page menu. The available modes are:
    • Account (the default) - allows limited changes,
    • Adhoc Instruction - allows full changes, or
    • Third Party (SWIFT or SWIFT-compatible External Methods only) - as Adhoc Instruction mode plus entry of Ordering Customer details for Third Party Payments.
  3. Change specific settings as required and as allowed for the current mode.
  4. Click Save.

    Your changes (if any) can result in:

    • the Payment Instruction setting for the cash flow changing to either Modified SSI or Ad Hoc);
    • a different Settlement Approval Defaulting Rule being applied to the cash flow (because Payment Instruction is included in rule Matching Criteria);
    • changes to the content of EFT output messages, or EFT output being suppressed, for the cash flow; and/or
    • changes to the Flow Lifecycle setting for the cash flow, which will also apply to a linked stock flow. See Counterparty Instruction Changes and EFT Lifecycle Phase Validation.

Restrictions

The Change function is available only on a single cash flow for an Actual Cash Flow deal, where:

  • the deal has been saved,
  • the cash flow is not an inter-company one,
  • the EFT has NOT already been created,
  • the user has the required Module Security rights (see below), and
  • Resource Security restrictions apply as usual.

The View function is available when a single cash flow is selected AND the EFT has already been created for that flow.

Audit Log

Details of all Counterparty Instruction changes are recorded in the Audit log and can be viewed using the dealing page's Audit History function.
