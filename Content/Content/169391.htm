Position Fee Template Requirements

This topic summarizes Base Template specific requirements when the Fee Category is Position fee.

TIP: A Position fee applies when the Fee Base Amount is based on a deal position or series of deal positions - for example, Face Value, Settlement Value, Capital Value, Security Value, or Interest Value.

Specific Base Template settings for Position fees are summarized below.

  • Fee Category - Position fee
  • Fee Base Amount - define the deal values used to determine the Fee Base Amount. Select from:Â·
    • Face Value - the current Face Value of the deal on a Base Reset Date; generally equals the principal value of the deal at settlement date, but it can change during the life of the deal
    • Settlement Value - either the Settlement Value for discount instruments or the Settlement Price of bonds multiplied by the Face Value as at the deal Settlement Date.
    • Capital Value - the capital component of the Settlement Price of the bond, as at the deal Settlement Date. This represents the sum of the Face Value and the discount / premium
    • Interest Value - for an interest-bearing instrument: the daily-accrued interest amount; for a discounted instrument: the daily-accrued discount amount on a base reset

      TIP: This can be calculated only on a daily basis.

    • Security Value - the sum of Capital Value, Purchased Income, and Premium/Discount Accrual, where:
      • Capital Value is as defined above.
      • Purchased Income is the accrued income/coupon as at the settlement date of the security. This amount becomes zero once the next coupon is paid.
      • Premium/Discount Accrual is the total discount accrued or premium amortized for a given date over the life of the Quantum deal (can be positive or negative).
  • % Adj - no special requirements
  • Base Computation - specify arithmetically how the Fee Base Amount is calculated for the Transaction fee:
    • Sum - the sum of the value of all qualifying deals (those satisfying the Selection criteria) as at the Base Reset Date
    • Average - the average value (Face, Settlement, or Capital Value, as defined in the Base Amount field) of all qualifying deals as at the Base Reset Date
    • Minimum - the minimum deal value of all qualifying deals, as at the Base Reset Date
    • Maximum - the maximum deal value of all qualifying deals as at the Base Reset Date

      TIP: The Base Reset Date is defined on the Schedule tab of the Fee Deal Rule page.

  • Formula - select the method for calculating the Position fee:
    • Percent per Annum - on each Calculation Date and for each qualifying deal (satisfying the Selection criteria), the fee amount is the Unit Fee Rate (%) value multiplied by the calculated Fee Base Amounts on Base Reset Dates within the Calculation Period, adjusted for the number of days in that period
    • Simple Percentage - on each Calculation Date and for each qualifying deal, the fee amount is the sum of each calculated Fee Base Amount on Base Reset Dates within the Calculation Period multiplied by the Unit Fee Rate (%) value (with no adjustment for the number of days in the Calculation Period)
    • Flat Fee - a flat fee is calculated, and must be paid, if the absolute value of the Fee Base Amount is greater than zero on a Base Reset Date

      On each Calculation Date and for each qualifying deal, the fee amount is the sum of the Flat Fees that were calculated on each Base Reset Date within the Calculation Period

      The Flat Fee amount is defined in the Unit Fee Amount ($) field.

  • Other Definition area items - no special requirements
  • All Selection Criteria items - no special requirements
