Zero Balancing Passes

Any number of Zero Balancing passes may be included in standard scripts at the Matched and/or Probable level of certainty (they do not apply to Bankflow-only scripts, nor to the Possible level of certainty).

When a Zero Balancing pass type is run in Prior Day or Intraday Matching:

  1. The applicable Match Status is the detail-level status for the run's Matching type (Prior Day or Intraday).
  2. The system searches for valid Zero Balancing adjustment (ZBA) bank flow items for the account that:
    • satisfy the defined matching criteria for the pass, which specify one or more bank flow Generic fields and a value or field to compare with in each case, and
    • do not have already the applicable Match Status set to Matched or Probable.

      TIP: To include items previously matched at the Matched and/or Probable levels (for all passes of the script), select the Reset items set to 'Matched' and/or Reset items set to Probable options (Details tab), respectively.

  3. When such a ZBA bank flow is found, cash flows are then searched for a single Account Transfer item with the same date and amount (and not already matched). Any specified match criteria again apply, so that:
    • one or more bank flow Generic fields can be compared to appropriate cash flow fields in each case, and
    • for numeric values, a maximum allowed difference (which may be zero) can be specified.

      IMPORTANT NOTE: This caters for the situation where the Bank Account Transfer has already been created by earlier processing of the account on the other side (of the transfer).

  4. When a cash flow is found that satisfies the pass criteria:
    • the two items are assigned the applicable Match Status and a new Detail-level Match ID, and
    • if (and only if) the Bank Value Date of the matching cash flow differs from that of the ZBA bank flow, AND the pass Match Status is Matched, AND the Update bank value date on cash flow when matched Operational Setting is selected, the cash flow's Bank Value Date is changed to the bank flow's date (otherwise it is not changed).
  5. However, if no match is found:
    • a Bank Account Transfer is generated with the Adjustment Detail values (Ticket Number, Instrument, Dealer, Source Entity, and Destination Entity) specified for the pass;
    • the Account Transfer cash flow (for the account being processed) and the ZBA bank flow are assigned the applicable Match Status and a new Detail-level Match ID; and
    • the generated Bank Account Transfer's Bank Value Dates are set to that of the ZBA bank flow (on both sides of the transfer).

      TIP: The account on the other side of the transfer will now have a new unmatched cash flow of the transferred amount.
