Using the Finastra Confirmation Matching Service

The Finastra Confirmation Matching Service enables you to electronically match and confirm some deal actions.

NOTE: Finastra was formerly known as Misys.

Quantum supports the following transaction and SWIFT message types for Finastra confirmations.

NOTE: Quantum Six does not support confirmations for Quotes. The message status for deal quotes is set to Not Required.

Preparing Your System to Use Finastra

NOTE: Your organization must be registered with Finastra, and it is your responsibility to establish the data communications link to them, as per their specified requirements.

For a list of prerequisites and the required static data setup, see Implementing Confirmations - Summary.

Configuring to use the Confirmation Matching Service

Begin by creating a page that will support the Confirmation Matching Service. Open the Utility > External Method page and create a new page using these terms:

Name: Misys

Short Name: Misys

External Name: Misys

Match Type: Confirmation

Language: Default

Options: (optional)

The system can now match an external status in a match report with the internal status in Quantum.

Sending and Receiving Finastra Confirmations

Export, import, and matching of Electronic Confirmations are automatically handled by system services running on the Application Server.

The Process Confirmations service forwards new Confirmation message output files to the matching service.

The Match Confirmations service "listens" for any new files returned by the matching service, uploads and matches them, and updates the confirmation match status.

Tracking the progress of Finastra Confirmations

When Finastra Confirmation responses are imported into Quantum, matching and status-changing are automatically performed by the system.

Use the Confirmations Processing page to track the progress of your Finastra (and other) confirmations within Quantum, and to optionally change Unmatched Confirmation statuses to Matched.

NOTE: If exported deal actions are rejected by the Finastra matching service itself, Quantum will not receive notification of these in any subsequently imported responses. If Confirmation responses are missing, you can check for possible reasons on the Finastra website's Import results page, by viewing the Finastra Log.

You can manually enter these deal actions directly into Finastra if required, using their website's Import page.

Requesting Amendments

You can also use the Electronic Messages Processing page to:

  • view Confirmation messages output by Quantum, and/or
  • send Confirmation Amendment requests for selected messages.
