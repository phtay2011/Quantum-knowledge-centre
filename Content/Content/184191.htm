Specifying Match Criteria, Detail Match Script

Use the Criteria grid to set up individual field criteria for your currently selected pass. These field criteria must all be satisfied to achieve a successful match at the Match Status level specified for the pass.

You can specify up to 25 grid entries for one pass.

NOTES:

  • If the Pass Type is Match, Match Status is Matched, and if the Allow flows to match Operational Setting is On same Value Date only, the Criteria grid will always include an exact match of Value Date Equal to Value Date, which you cannot remove. Otherwise (that Operational Setting is NOT On same Value Date only), you can optionally specify the Value Date field in the Criteria grid (as for any other field).
  • The Bank Value Date is also avaiable for matching.
  • For other Pass Types, only Extra Information and Generic bank flow fields are available.

Example

Examples of simple criteria for a Match pass follow.

Match Status

Example Criteria

Matched

Identical cash flow and BANK flow dates, amounts, deal numbers, and cash flow types

Probable

Identical cash flow and bank flow dates, amounts, and deal numbers

Possible

Identical cash flow and bank flow deal numbers, and:

  • dates within 2 days of each other, and
  • amounts within 0.1% of each other.

Procedure Summaries

To add a Criteria entry

  1. In the Criteria grid menu, select New.
  2. In the dialog box:
    1. select the Bank Flows Field and Match type (see Field Descriptions below);
    2. where applicable, enter the associated Tolerance, Percentage, Cash Flows Field, or Match To value; then
    3. click OK.
  3. Save your changes.

To edit a Criteria entry

  1. In the Criteria grid, click a data item in the entry row.
  2. In the Edit dialog box:
    1. edit the criteria settings as required (see Field Descriptions below), then
    2. click OK.
  3. Save your changes.

To delete a Criteria entry

  1. In the Criteria grid, select the entry row.
  2. In the grid menu, select Delete.
  3. in the dialog box, click OK to confirm.
  4. Save your changes.

Field Descriptions (in dialog boxes)

Generic bank flow fields here include Generic 1 through Generic 10 and Extra Information.

Bank Flows Field

Select a bank flow field to be matched (or filtered on) for your currently selected pass.

NOTES:

  • Other fields and selections will vary, depending on the field you select.
  • Only Generic fields are available when Pass Type is Bankflow Only, Bulk Bankflow Only, or Zero Balancing.

Match

Depending on the data type of your selected Bank Flows Field (as indicated below), select one of the following (where available):

  • Equal to* (all except Generic) - to require an exact match with the corresponding cash flow field (ignoring spaces before and after),
  • Equal to Field* (Generic only) - to require an exact match with the specified Cash Flows Field (ignoring spaces before and after),
  • Equal to Value (Generic only) - to require an exact match with the specified Match To value (ignoring spaces before and after),
  • Contained in (all except dates and Generic) - to require an exact match with part of the corresponding cash flow field,
  • Contained in Field (Generic only) - to require an exact match with part of the specified Cash Flows Field (for example, ABC will match against ABCD, XABCYY, and so on),
  • Contained in Value (Generic only) - to require an exact match with the part of the specified Match To value,
  • Within Percentage (amounts only) - to require the percentage difference in amounts to be no more than the specified Percentage value,
  • Within Tolerance (amounts only) - to require the absolute difference in amounts to be no more than the specified Tolerance value,
  • Within Business Days (dates only) - to require the difference in dates to be no more than the specified number of days, or
  • Within Calendar Month (dates only) - to require the dates to be in the same calendar month.

*TIP: For matching to a cash flow Instrument, do not select Equal To (because Instrument names are unique across all transaction types, and while your bank flow Instruments may be similarly named, they cannot be identical to any cash flow Instrument names).

Cash Flows Field

(Available only when Bank Flows Field is a Generic field and Match is Equal to Field or Contained in Field; not supported for Bankflow Only or Bulk Bankflow Only passes.)

Select the cash flow field you want to match with the specified Generic Bank Flows Field, either exactly (when Match is Equal to Field) or in part (when Match is Contained in Field).

TIP: Deal Analysis Codes will be included only when Show on cash flow detail during bank reconciliation is selected for those codes (on the Analysis Code configuration page).

Match To

(Available only when Bank Flows Field is a Generic field and Match is Equal to Value or Contained in Value.)

Enter the literal text value that, for the bank flow to be included, you require the specified Generic Bank Flows Field to either be exactly equal to (when Match is Equal to Value) or in part (when Match is Contained in Value); ignoring spaces before and after.

Percentage

(Available only when Bank Flows Field is Amount and Match is Within Percentage.)

Enter the maximum percentage difference in the bank flow and cash flow amounts that will be allowed, for a match to occur.

If Match Status is Match or Probable, an adjustment will automatically be made for the difference, and you must specify its type , related options, and details in the relevant fields that then appear - namely, Adjust Using, Set From Bankflow Record and other options, and Adjustment Detail fields.

Tolerance

(Available only when Bank Flows Field is Amount and Match is Within Tolerance.)

Enter the maximum absolute difference in the bank flow and cash flow amounts that will be allowed, for a match to occur.

If Match Status is Match or Probable, an adjustment will automatically be made for the difference, and you must specify its type, related options, and details (as for Percentage).

Within Business Days

(Available only when Bank Flows Field is Value Date and Match is Within Business Days.)

Enter the maximum number of business days up to which in the bank flow and cash flow dates may differ, for a match to occur.

TIP: For details of matching bank flows processed on non business days, see Non Business Day Cash Flows and Bank Reconciliation.
