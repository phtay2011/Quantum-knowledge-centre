Workflow Template page

Use this page to find data, alter the data as needed, and then create an output in a format you specify - for example, for finding data from Quantum and creating an output for a dialog. Set up a report through which certain actions run the dialog.

Inputs make it possible to create one output from data that comes from multiple records.

The output has a defined format and also a purpose, so you can use the output for regulatory reporting or EFT messages, for example.

Data items

  • Name - the internal name that the system uses to recognize the template.
  • Output Format - the format of the output of the template. Select Text, XML, or HTML.
    • Select HTML and you have the option to convert the resulting HTML to PDF in output files when the template is applied.
    • Select XML and an Edit item appears in the main menu. Click Edit to open a dialog box to edit the schema.
  • Purpose - the reason for the output. Select from Any, EFT (where the template will be used to structure generated EFT messages - where an EFT template is defined on the External Method page), or Purpose (where the template will be used a function in expression builder).
  • Descrption - a short description of the template.

Menu items:

  • Publish – (only available for EFT templates) Select to make the EFT template read-only. This will move the EFT template from an Unpublished state (i.e. editable) to a Published state (i.e. not editable). The system will not stop an EFT run from processing if the EFT template being used is in a Published state.

  • Unpublish – (only available for EFT templates) Select to make the EFT template editable. This will move the EFT template from a Published state (i.e. not editable) to an Unpublished state (i.e. editable). An “Unpublished” badge will appear in the upper right-hand corner of the template. The system will stop an EFT run from processing if the EFT template being used is in an Unpublished state.

  • Edit - To view the schema, select Output Format = XML and then select Edit.
  • Bulk Replace - applies to any Purpose. To use a spreadsheet application, for example, Excel, to create and maintain Sample Data which can be viewed in the Preview tab.
  • Import EFT Data - (only available for EFT templates) – Brings up a dialog that mimics the Reporting EFT query page fields and filters. Use the Query field to select from your existing Reporting EFT queries. Selecting a Query will default the rest of the fields for you, but you can still make changes. After populating the fields, select the Import which runs the query in the background and sends the filtered query results over to the template's Sample Data section in the Preview tab.

In the Template tab:

  • Inputs grid - add a new input or edit an existing one by selecting New or Edit in the grid menu.
    • Name - a unique name for the input.
    • Description - a short description of the use of the input.
    • Type - select Text, Date, Yes/No, Amount, or Number.

    • Default Value - a value of the specified Type (for Yes/No, select or clear the check box).

      NOTE: The system stops you deleting an input that is in use.

  • Expressions grid - add a new expression or edit an existing one by selecting New or Edit in the grid menu.
    • Name - a unique descriptor for the expression.
    • Type - select Text, Date, Yes/No, Amount, or Number.
    • Expression - click the Add Expression link and use Expression Builder to create or edit the associated mathematical expression, using Functions, Operators, and Inputs.
  • Validations grid (optional) - add conditions that either block the system from processing invalid data or warn about exceptions in the data.
  • Sort field - select one or more items from the Inputs and Expressions grid, to order the data coming from source. The order persists in the output.
  • Text box (optional) - enter the section of code that refers to the Inputs and Expressions and that creates the output.
  • Sources grid - the columns and rows from the database, used as the source of the data for the template. Add a new source or edit an existing one by selecting New or Edit in the grid menu.
    • All Data - select to show only Columns and Settings from the specified source.
    • Aggregated - select to show fields to Group By, Columns to aggregate by, and Settings from the specified source.
    • Source - select the database record type.
    • Name - a unique descriptor for the source.
    • Columns tab and grid - click Add and select columns from the specified source that you want to be available in your template.
    • Settings tab and grid - settings to apply to the data, including a mathematical operator and an expression.

In the Preview tab:

  • by clicking the Generate Preview button you can view the sample data and the output of the template resulting from the inputs you enter, and
  • if you have defined validation conditions that the sample data meets, the Validation Messages grid displays the applicable message type and text.
