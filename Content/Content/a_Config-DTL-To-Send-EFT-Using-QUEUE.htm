Configure and use the DTL to send EFT messages using QUEUE

From 2020.2, in the Quantum Configuration wizard, Echos Message Queue Settings dialog box, we removed the Queue Type radio button Publisher.

This topic describes how to set up the system from 2020.2.

To use message queues to import or export data

In general:

  1. Configure the MessageQueueSettings.config file for the Message Queue servers. The MessageQueueSettings.config file is in the \config directory of the Site folder.

  2. Set up the SITE to point to the required Quantum site that you want to use.

    NOTE: For a given Site, there can be a number of Message Queue servers.

  3. Configure Quantum to use the correct Data Transfer Location (DTL).

    TIP: You can also use the DTL to set up Quantum to send confirmations to ECHOS.

Configure the MessageQueueSettings.config

The Echos Gateway Settings no longer supports Publisher settings, so you must migrate Publisher settings to the config file. The example below demonstrates how to set up the MessageQueueSettings.config file.

NOTE: The ECHOSGatewaySettings.config still supports Listener settings specific to the AvantGard ECHOS Message Listener Service.

ECHOSGatewaySettings.config - Legacy

This is a segment of a legacy EchosGatewaySettings.config file prior to 2020.2 that we not be used: 

<Publishers> 
<Publisher> 
<Site>RPACK</Site> 
<WebSphereMQProfileXms> 
<ProfileName>T.Q1.FIN.QNTNZ.IN</ProfileName> 
<Channel>CHANNEL_MCA</Channel> 
<QueueName>T.Q1.FIN.QNTNZ.IN</QueueName> 
<QueueManager>T.ECH1.QNT.FIN</QueueManager> 
<Host>10.244.249.44</Host> 
<Port>1617</Port> 
</WebSphereMQProfileXms> </Publisher> 
</Publishers>

MessageQueueSettings.config - Current

This is a segment of a current MessageQueueSettings.config from 2020.2, setting up a message: 

<sites>
<site name="mySiteName"> 
<servers> 
<server 
name="ECHOS_FIN" 
uri="xms:wmq://10.244.249.44:1617?QueueManagerName=T.ECH1.QNT.FIN&amp;ChannelName=CHANNEL_MCA" 
deliveryMode="Persistent"/> 
</servers> 
</site>
...
</sites>

name

Use “name” to distinguish one server from another. Maximum 100 characters. The “name” forms the name of the record in the Queue Connection browse list on the DTL page.

For example:

name="ECHOS_FIN"

uri

The connection to the Message Queue Broker, using IBM WebSphere queue client, is: xms:wmq://WebSphereServer?QueueManagerName=QUEUENAME&amp;ChannelName=CHANNELNAME

Where:

  • WebSphereServer is the IP address of the machine hosting the IBM WebSphere MQ server, for example, the combined <Host/>:<Port/> settings from a legacy Publisher setting.
  • QueueManagerName matches the queue manager you created in IBM WebSphere. Must be unique. For example, the <QueueManager/> setting from the legacy Publisher setting).
  • ChannelName matches the IBM WebSphere channel created for the queue manage, for example, the <Channel/> setting from a legacy Publisher setting. deliveryMode="Persistent" - Persistent or NonPersistent.
  • Not all Queue Providers support this functionality. MSMQ only supports persistent queues, whereas ActiveMQ supports both persistent and non-persistent queues.
  • Non-persistent queues only maintain their messages in memory and these messages are lost after a service restart.

For example:

uri="xms:wmq://10.244.249.44:1617?QueueManagerName=T.ECH1.QNT.FIN&amp;ChannelName=CHANNEL_MCA"

Configure Quantum to use the DTL etc.

As well as setting up the MessageQueueSettings.config, you must complete the following:

  1. Set up a DTL:

    • Location Type = QUEUE
    • Connection = ECHOS_FIN - server name of a specific queue as defined in the MessageQueueSettings.config file
    • Purpose = Eft
    • Location = TA.QN.FIN.ABCD.IN - for example the <QueueName/> setting from legacy Publisher setting

    The "Outgoing" badge indicates that the purpose is an outgoing type.

  2. If the queue location does not exist or does not respond, the DTL record will show the "Not Operational" badge and the location is not currently available.

  3. To set up an External Method Source for EFT processing:

    • Type Export = QUEUE
    • Destination = EFTQueue (in this example, the name of the active queue DTL shown above)

    If upgrading from a prior release, the system does not remove a Publish To ECHOS setting from Type Export on any existing External Method Source records. We ask that you change this setting before editing existing records because:

    • if you re-save with Publish To ECHOS selected, the system returns an error message, and
    • if the system processes an EFT using the External Method Source record, the system logs an error message.

    The system cannot change Type Export to QUEUE for you, because you have to first set up the DTL page, as above.

  4. To set up a DTL to send EFT Messages using files, set:

    • Location Type = FILE
    • Purpose = Eft
    • Location = SG (this is the name of an existing sub-folder of Eft\Processed\)

    The "Outgoing" badge indicates the purpose is an outgoing type.

  5. If the folder location does not exist, the DTL record shows the "Not Operational" badge and Location is not available.

  6. If sending EFTs to ECHOS via a queue, then open Reporting > EFTs and add Generate = File per EFT, and only send one transaction at a time to the queue. Set Generate = File per EFT as opposed to Single File or File per Day when processing the EFT run.

View message received from ECHOS

To view messages received from ECHOS, open Processing > ECHOS and set up the ECHOS Messages page to show these messages.
