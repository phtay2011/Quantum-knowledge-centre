Report View enhancements

Calculated Columns as Filters (all reports)

When setting up a query, previous versions allowed you to add simple expressions in the View > Configure > Columns tab > Calculated Columns. You can now add more complex expressions, such as "OR" or "[Currency] = "AUD" AND [ContractRate] > 5".

If you then open the Filters tab, you can now select the Display Name of the calculated column from the Add Filter dialog box and use the calculated column to set up a filter.

This applies to all views that support calculated columns.

Additionally, if you:

  • rename a calculated column, the system reflects this change in the filter, or...
  • remove a calculated column, the system removes the filter.

Views – Show rounded amounts

You can now specify Rounding settings related to Amount fields - but not numeric fields - that will be shown in a report View.

This applies to all reports, including custom reports created in the Enterprise Workflow (EWF). It applies to pivots and tables and the amounts will be exported as you see them in the View. On the View’s Properties tab there is a new section called Rounding which contains the following fields: Columns, Rounding Method, Rounding Amounts, and Decimals.

The View heading will display the rounding unit in smaller font. The amounts will be exported as you see them in the view. The rounding unit will be exported as part of the View heading.

This is available for Table views only.

Previously, when displaying Amounts fields these would always show as whole numbers.

See more: Configuring Table Views

Distinct Count in Pivot or Chart

In the View’s Layout tab, you can now configure a pivot or chart to display a distinct count of a specified field.

The Values – aggregate on lookup now contains all data types instead of just amounts and numbers. When the data type is not an amount or number, then the list of aggregation functions is restricted to Count, Distinct Count, and None.
