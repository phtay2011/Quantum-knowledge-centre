Actual or Projected Cash Flows - Payment/Receipt option

On the Actual Cash Flow and Projected Cash Flow pages, you can now default the value for the Receipt or Payment option, by using a customized page Variant. The Amount dictates the behavior of the Payment/Receipt radio buttons.

The changes include:

  • If you select Payment and then enter data in another field on the Deal Details tab, the Payment setting now does not change. (Previously, the option switched to Receipt.)
  • If you select Receipt, and enter an amount that is:
    • positive - the Receipt option remains, but if...
    • negative - the option switches to Payment.

Example

To set up Payment as the default, and set up validation messages

  1. Open a new workflow page.
  2. Select Variant, and choose the Core Module to be Actual Cash Flow or Projected Cash Flow.
  3. In the XML area, add XML that sets up the default for the Payment/Receive radio buttons. For example:

    <Fields>

    <Field id="PaymentReceive" defaultValue='Payment' />

    </Fields>

  4. In the Validation grid, set up any validation messages you require and whether the validation process blocks the deal or just displays a message. For example, block the deal if the Receipt value is negative and display a message to explain why this is blocked.
  5. Complete the other fields and save.
