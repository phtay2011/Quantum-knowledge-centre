Bulk Bankflow Only Passes

Any number of Bulk Bankflow Only passes may be included in standard scripts (they do not apply to Bankflow-only scripts).

A Bulk Bankflow Only pass is similar to a Bankflow Only pass, but creates only one adjustment for each Bank Value Date, with that adjustment matched to all bank flows with the same Bank Value date. (A Bankflow Only pass creates a separate adjustment and Match ID for each individual bank flow).

When a Bulk Bankflow Only pass type is run in Prior Day or Intraday Matching:

  1. The applicable Match Status is the detail-level status for the run's Matching type (Prior Day or Intraday).
  2. The system searches only for Bank flow items that:
    • satisfy the defined matching criteria for the pass, which specify one or more bank flow fields and a value to compare with in each case, and
    • have the applicable Match Status set to Unmatched.

      TIP: To include items previously matched at the Matched, Probable, and/or Possible levels (for all passes of the script), select the corresponding Reset items set to... option/s as required.

  3. When such items are found:
    • one adjustment (see below) is made per Bank Value Date, of the type specified in Adjust Using (except that, for Prior Day Matching only,Projected Cashflow adjustments are ignored),
    • the applicable Match Status and a unique detail-level Match ID are assigned to the adjustment and its matching set of bank flows with the same Value Date.

TIP: Items matched by one pass are thus ignored in following passes.

Adjustments

For a Bulk Bankflow Only pass, an adjustment must always be specified.

To specify the adjustment settings:

  1. In Adjust Using, select the type of adjustment - one of:
    • ACTUAL CASHFLOW - to create a new Actual Cash Flow with Bank Value Date set to that of the matched bank flow, adjusted if necessary according to the Non-Business Days setting for the specified Instrument, or
    • BANK ACCOUNT ADJUSTMENT - to create an adjusting Bank Account cash flow.
  2. Optionally select the Copy into Comments Field option to automatically copy the More Information field contents from the Bank Flow item into the Comments field of the created adjustment entry (whatever its type). Clear the option to exclude this information.
  3. Optionally select the Also adjust for external cash flows option to prevent an automatic Detail-level adjustment entry if an external (imported) cash flow is involved. This prevents the Treasury position being affected by automatic adjustments made when matching external (non-Quantum) cash flows.

    Select this option to allow an automatic Detail-level adjustment entry whether or not an external cash flow is involved. The Treasury position will then be affected by any deal adjustments created when matching external (as well as Quantum) cash flows.

    NOTE: In most cases where external cash flows do not exactly match, you should notify the organization or system from which they were imported.

  4. DO NOT select any of the Set From Bankflow Record options (they are not supported for Bulk Bankflow Only passes).

    â€  NOTE: If not all mandatory fields are valid for an adjustment, no adjustment will be created and no match will be made.

  5. In the Adjustment Detail area, optionally select static data values for the adjustments generated when the script pass runs. The available data fields vary depending on the selected adjustment type (in Adjust Using), as indicated below.

    NOTE: None of these settings will be affected by any Set From Bank Flow Record option selections.

    • Ticket Number - optional transaction ticket information - for example, a reference for interfacing to external systems.
    • Instrument (Actual Cash Flows only) - select an instrument the transaction is based on. Only instruments defined for the adjustment type are available.
    • Entity (Actual Cash Flows only) - the entity (legal party) on your side of the transaction.
    • Counterparty (Actual Cash Flows only) - the counterparty you are transacting with. The drop-down list includes all transaction-level Counterparties with Category set to Deal external or Deal internal in Counterparty Setup.
    • Dealer (Actual Cash Flows only) - select a dealer; defaults to the current user if they are set up as a Dealer in User Maintenance.
    • Strategy (Actual Cash Flows only) - optionally select, for the adjusting deal, a Strategy group.
    • G/L Code (Bank Account Adjustments only) - the General Ledger code for the adjustment entry.
    • Comments - optional text giving further information about the adjustment.
  6. Save your changes.
