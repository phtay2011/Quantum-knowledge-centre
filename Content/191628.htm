Procedures, Adaptiv Queries

These procedures give generic steps covering all types of Adaptiv Query and all View types. For more details, refer to the See also links at the end of this topic.

TIP: To help users locate queries of each type, the names of queries can (and should) include an indication of the type. For example, Base Valuation query names could include the abbreviation BV, Cash Flow query names could include CF, and so on. This also applies to folders that contain queries of only that type.

To find an existing query:

If you have adopted a naming convention as above, then:

  • On the Queries browse page, search for the standard designation for that type of query - for example BV for a Base Valuation query.

To create a new query (from the Queries browse page):

  1. Navigate to the folder (or root) level where you want to locate the new query.
  2. Click New* and in the drop down list select the query type. (Or in the page menu, select the New function for the query type.)

    A new Query page will open for your selected query type.

  3. Configure the required query parameter Defaults and Properties, including a unique Name, the required Definition elements and Expiry Strategy settings; then click Save.

    The query will be saved in the current location.

To create a new query (from any Query page):

  1. In the page menu, select the New function for the query type (New Base Valuation, New Cash flow, and so on).

    A new Query page will open for your selected query type.

  2. Configure the required query parameter Defaults and Properties, including a unique Name, the required Definition elements and Expiry Strategy settings; then click Save.

    The query will be saved in the same location as the query you started from (whatever its type).

To run the current query:

  1. Check the query's parameter Defaults and configuration.
  2. Click Run.

    When the run is completed, the report results are displayed (in either the default View or the last View you used for the query).

  3. If required, use the > and < buttons to page through the results.

To run the current Query with automatic updating:

  1. In the Run Query button's drop-down menu, select Auto-run....
  2. In the Auto-run Query dialog box, select a run option - one of:
    • Auto-run after a pause (and enter the Pause time and units),
    • Auto-run immediately,
    • Update to the latest scheduled run, or
    • Update live on change in the system.

      TIP: If any of these options have been used before on the page, the system shows the last three of these in the Run Query button's drop-down menu.

  3. Click OK.

    An Auto-run badge is displayed next to the query name and, if it is updating automatically (Live, Latest, or Auto-run), a Pause button is displayed next to the Run Query button. For more details, see Auto-Update a Query.

To pause and resume automatic updating:

  1. Click Pause to pause the automatic update. The button changes to Resume.
  2. Click Resume to have the system automatically update the query results again.The button changes back to Pause.

To apply a different View to the current run results:

  • Click on the View heading and select the View you want from the lookup.

    The current report results are redisplayed in your selected View.

To change the current View settings:

  1. Click the View Configure link.
  2. Edit the View settings as required.
  3. To see the effect of your changes, EITHER :
    • optionally enter a new View Name on the Properties tab and click Save, OR
    • click Apply (which does not change stored View settings).

    The current report results are redisplayed using the changed View settings.

To create a new View for the current Query:

  1. In the View menu, select the required New function for the type of View: New Table, New Pivot, or New Chart (or select Copy to create a new View based on the current one).
  2. Edit the View settings as required.
  3. Enter a new View Name on the Properties tab and click Save.

    The current report results (if any*) are redisplayed in your new View.

*TIP: You can create, reconfigure, and remove Views without having to run the query.
