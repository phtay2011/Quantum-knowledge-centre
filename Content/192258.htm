CMS Spread Notes

CMS Spread Note deals can be added to the Analytics database using the External Portfolio File utility.

Requirements

A CMS Correlation Price Factor is required for deal valuation - that is:

  • an Analytics Swaption Volatility rate record is required to define the CMS Correlation curve, and
  • that Rate Name must be selected in the CMS Correlation field on the Var Rates Definition page.

Example Deal Data Format in .AAP file

<Deal>
Object=CMSSpreadNote,Reference=3361-Test CMS SNote,MtM=&lt;undefined&gt;,
Currency=EUR,Discount_Rate=EUR-Default Zero Curve 6M OIS EOD,
Discount_Rate_Volatility=,Forecast_Rate1=EUR-Default Zero Curve 6M OIS EOD,
Forecast_Rate2=EUR-Default Zero Curve 6M OIS EOD,Forecast_Rate1_Volatility=,
Forecast_Rate2_Volatility=,Buy_Sell=Buy,Effective_Date=26Jan2018,
Maturity_Date=26Jan2038,Principal=1000000,Payment_Frequency=1Y,
Payment_Offset=0,Accrual_Day_Count=ACT_365,First_Coupon_Date=26Jan2023,
Penultimate_Coupon_Date=26Jan2037,Roll_Direction=Backward,Amortisation=,
Rate1_Tenor=10Y,Rate2_Tenor=2Y,Rate1_Day_Count=_30_360,
Rate2_Day_Count=_30_360,Rate1_Frequency=6M,Rate2_Frequency=6M,Rate_Offset=0,
Known_Rate1s=26Jan2025=0.018\26Jan2026=0.019\26Jan2027=0.08\,
Known_Rate2s=26Jan2025=0.006\26Jan2026=0.005\26Jan2027=0.001\,
Rate1Fixing=,Rate2Fixing=,Accrual_Calendars=Target Settlement Day,
Payment_Calendars=Target Settlement Day,
Rate1_Calendars=Target Settlement Day,Rate2_Calendars=Target Settlement Day,
Rate_Publication_Calendars=Target Settlement Day,Principal_Exchange=None,
Payment_Timing=End,Reset_Type=Standard,Fixed_Rate=0 bp,Rate1_Multiplier=4,
Rate2_Multiplier=4,Is_Floored=Yes,Floor_Rate=300 bp,Is_Capped=Yes,
Cap_Rate=1000 bp
</Deal>

NOTES: The Reference attribute value has the format deal id - description, where:

  • deal id must refer to an existing deal in Analytics, and
  • description is text describing the deal.
