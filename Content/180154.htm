Static Data Server Configuration

  1. Run the Analytics Service Manager by executing AGRSvcMgr.exe from within the GUI folder.
  2. Confirm that the Datasource is defaulted to the correct database.

    NOTE: The default data source will be that which is nominated within the Client settings. The drop down box lists all Mrc11dh!data sources that have been configured within the Site Settings. Select the correct data source from the drop down list to configure its server information.

  3. Select the Static Data Server from the Risk Server lookup list.

  4. Make sure the Server is the Application Server, if not this will need to be changed.

    NOTE: You will need to have administrative rights on the Application Server to use this application from a remote machine.

  5. For the Static Data server to run properly it needs to be run under a user account as specified in the Installation Instructions.
  6. Select This Account and enter a domain user account, password and domain.
  7. Click Configure and select the additional options as required.

    Initialize on Startup

    When checked, whenever the service is started, the service will initialize the currently selected database. This may occur because a user accessed a database, or the service was started either manually or automatically.

    Risk Server

    When the server is initialized on startup, the Risk server it should start up against needs to be specified.

    Use Monitor

    When checked information is sent to the monitor.

    Send Requests to Monitor

    When checked the timings on each individual call are displayed within the monitor.

  8. Select Apply to commit the changes.
  9. Select OK to save the Static Data Server configuration.
  10. Select Start to execute the Static Data server, the Current Status will change from Stopped to Running.

    NOTE: The current status of the Static Data Server can be viewed within the Analytics Monitor.
