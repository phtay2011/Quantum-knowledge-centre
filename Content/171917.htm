Revaluing an FX Deal

When a deal is revalued in Quantum, the difference between its market value and its book value represents the gain or loss (realized or unrealized) on that deal.

Revaluation method

The following settings determine the revaluation method applied to an FX Deal:

  • Revalue FX deals using setting in FX Accounting Rules
  • Mkt Revals column in Management Rules

    NOTE: A Hedge Relationship is revalued using the method selected in the Component column on the Linked Deals tab of the Hedge Relationship page.

Book value

The book value of an FX Deal is always calculated by the system, whether the market value is calculated or imported. The selected revaluation method determines how the book value is calculated. See Book value, FX Deals.

Revaluation currency

The revaluation currency and posting currency settings determine which currency is used for revaluation calculations, and which currency is used to post the revaluation result.

If a market revaluation is imported, the specified revaluation currency is also used to validate the imported record.

Results are automatically translated from the calculation currency to the revaluation posting currency where necessary. A Revaluation Analysis shows both the revaluation calculation currency and the revaluation posting currency for the deal.

Forward points

If the forward points on an FX Deal are revalued and accounted for separately, the currency used for their revaluation is determined by:

  • Fwd Post CCY setting in Management Rules, and
  • Accrue forward points setting in Accounting Rules.

In Quantum, it is recommended that you revalue an FX Deal and its forward points in the same currency. To achieve this, use the same setting for:

  • Fwd Post CCY and Reval CCY columns in Management Rules, and for
  • Revalue FX deals using and Accrue forward points options in Accounting Rules.
