Configuring a CashXplorer Query

The query configuration is used to create the CashXplorer report. It stores information about the time periods, how cash flows are to be translated into other currencies, how the cash flows are grouped, and which cash flows are to be retrieved from the Quantum database.

If you have the correct security access rights, you can create as many queries as you require within each of the access types (Public, Group, or Personal).

Use the Query Configure link to open the Configuration area, and use the tabs as follows.

  • Horizons tab - to specify the time periods over which the cash flows are calculated and forecast.
  • Filters tab - to specify the cash flows to be retrieved from the Quantum database, selecting bank and/or broker accounts, flow types, and balances.
  • Pools tab - to specify multi-dimensional cash flow groupings (in addition to the hierarchical grouping structure that CashXplorer uses).
  • Adjustments tab - to specify minor corrections or changes to a query configuration - for example, when creating a new Data Version, refreshing existing Data Versions, defining currency translation, deleting previous data versions, or for message logging.
  • Properties tab - to name the Query, select Business Day and Access options, specify currency translation, deletion of previous data versions, and message logging.

    TIP: When you create a new query, the default group to which it belongs is the group in which you created it. You can override this default by selecting a different Access Type on the Properties tab.

NOTE: In addition to the information defined in the query itself, the data retrieval properties for all bank accounts accessed in CashXplorer must be defined. These properties are set automatically with default values; to check the values and make any necessary changes see the Bank Account Rules module.

Defining new queries

There are two quick ways to define a new Query in sufficient detail to enable Data Versions to be created:

  • Using default settings that are available in any newly created query
  • Copying an existing query that is similar to the new query you want to define

Both these methods create a good quick-start query that once created you can tailor to more specific needs.

Security access rights

Users with Maintain security access rights are able to create, edit, delete, and copy the queries. However, the access rights must be specific to each part of the Worksheet and to the Worksheet's owning group. For example, users with Data Retrieval Parameters Maintain (Public) security access rights may create, edit, delete, and copy Data Retrieval Parameters in Public queries.

Users with Inquire security rights may look at queries. But again, the access rights must be specific to each part of CashXplorer and to the query's owning group. For example, users with View Inquire (Group) Access rights may see the Group queries.

Security access rights are described in detail in CashXplorer Security.
