Valuation Rule enhancements

The Valuation rules matching criteria have been extended to allow users the ability to apply separate Valuation models, Zero curves and/or Margin curves based on:

  • Counterparty or Counterparty Group
  • Currency Pair (CCY Pair)
  • Interest Payment (Payment Frequency) and/or Rateset Frequency

Find these in the Valuation Rule page.

Counterparty

You can now see Counterparty as a matching criteria element in most of the transaction types supported on the Valuation Rule (Analytics) page. This includes the "...Pair" tabs that exist for transaction types with split tabs e.g. the Cross Currency Swap tab and its pair tab Cross Currency Swap Pair. Also applies to Counterparty Group.

Payment Frequency and Rateset Frequency

You can now see Payment Frequency as a matching criteria element for the following transaction types: Money Market, Security (Bond, FRN, Discount), Cross Currency Swap, Interest Rate Swap, and Cash Flow. Excludes the Cross Currency Swap Pair tab.

You can now see Rateset Frequency as a matching criteria element for the following transaction types: Money Market, Security (Bond, FRN, Discount), Cap/Floor, Cross Currency Swap, and Interest Rate Swap. Excludes the Cross Currency Swap Pair tab. Selecting a value here will immediately exclude any fixed rate deals from using this rule, across the transaction types. Given other criteria are also in line, any floating rate deals with a rateset frequency matching this value will match to this rule.

The frequencies available include: Daily, Weekly, Fortnightly, Monthly, Quarterly, Semi Annual, Annual, and Maturity.

Money Market Payment Frequency also includes Statement.

Currency Pair (CCY Pair)

You can now see Currency Pair (CCY Pair) as a matching criteria element for the following transaction types: Cross Currency Swap, Currency Option, and Foreign Exchange. This allows you to use a different set of curves, based on the currency as part of a pair, rather than the currency alone. For example, the USD side of a USD/EUR swap may be valued with different curves to the USD side of a USD/NZD swap.

Order of matching

When the system matches a deal against a rule, it applies these criteria in the following order, with a match to multiple criteria having further priority over a single match or a match to a lower number of criteria.

In addition to this, there is a diminishing priority applied for grouping structures. For example, where a transaction level instrument is used as the only criteria in a rule, this will take precedence over a rule configured with either the instrument group, the deal entity, currency, or pay/receive setting.

  • Instrument
  • Entity
  • Counterparty (excludes Futures and Future Options)
  • Currency (excluding CCS Pair, CO Pair, FX Pair, Futures, Futures Option)
  • Currency Pair (CCS, CCS Pair, CO, CO Pair, FX, FX Pair only)
  • Pay/Receive (IRS, CCS only)
  • Payment Frequency (MM, Security, CCS, IRS, Cash Flow only)
  • Rateset Frequency (MM, Security, Cap/Floor, CCS, IRS only) currency, or pay/receive setting. But this will not take precedence over a rule configured with both entity and currency.
