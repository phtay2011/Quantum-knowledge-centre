Data Manipulation Examples (File)

The following example correspond to the example entries under Bank Transaction Rule Entry Examples. That topic shows how these examples would appear in the Edit Transaction Rule dialog box following an Upload of the Configuration file containing them.

IMPORTANT: Although each clause is shown here on a separate line for clarity, all the clauses of one data manipulation parameter must be entirely on one line in the Configuration file.

  1. For account 123456, if the strings BILL and BEN appear somewhere in the transaction source record, then set CP to the string FPM and set CN to the number 67890:

    #Example 1
    FOR ACCOUNT 123456
    WHERE TX LIKE "*BILL*" AND TX LIKE "*BEN*"
    SET CP="FPM", CN=67890

  2. For accounts from 601012 to 607080, unconditionally set generic field 10 to INTRADAY, and the Check Number field to 45678 (no condition clause is present, so the condition is always true):

    #Example 2
    FOR ACCOUNTS 601012 TO 607080
    SET G10="INTRADAY", CN=45678

  3. For ALL accounts, if the Customer Reference field holds a CUSIP number in the format CUSIPnnnnnn, and the Transaction Type is 242, then assign the Customer Reference numeric portion nnnnnn to Generic field 9:

    #Example 3
    FOR ALL ACCOUNTS
    WHERE TT = "242" AND MID(CR,1,5) = "CUSIP"
    SET G9=MID(CR,6,9)

  4. For all accounts, always set Generic field 9 to the Bank Reference, and Generic field 8 to the Customer Reference:

    #Example 4
    FOR ALL ACCOUNTS
    SET G9=BR, G8 = CR

  5. For all accounts, if the transaction date is prior to today, set it to today's date.

    #Example 5
    FOR ALL ACCOUNTS
    WHERE DT < TODAY
    SET DT = TODAY

  6. For all accounts, set the Zero Balance Account transaction field (ZBA) to the Bank Reference if the Transaction Type code is 275 (ZBA Credit) or 575 (ZBA Debit):

    #Example 6
    FOR ALL ACCOUNTS
    WHERE TT=275 OR TT=575
    SET ZBA = BR

  7. For all accounts, exclude a transaction if the Customer Reference field value does not begin with CUSIP:

    #Example 7
    FOR ALL ACCOUNTS
    WHERE MID(CR,1,5) <> "CUSIP"
    SET EXCLUDE = TRUE

  8. For all accounts, summarize the individual transactions with a Transaction Type of 208 into one summary transaction, and import only that summary transaction (containing only the specified fields):

    #Example 8
    FOR ALL ACCOUNTS
    WHERE TT = 208
    SUMMARIZE TRANSACTIONS SUM(AM), MAX(DT), COUNT(G1) GROUP BY TT, DC
    SET SR = "This is a summarized transaction", G02 = TT

    NOTE: A SUMMARIZE clause must explicitly specify all of the fields that Quantum requires.

  9. For all accounts, unconditionally set the Entity field (EN) to the string MYENTITY, the Ticket Number field (TN) to the string MYTICKET, and the Cash Flow Instrument (IN) to the string FEES (no condition is present, so the condition is always true):

    #Example 9
    FOR ALL ACCOUNTS
    SET EN = “MYENTITY”, TN = “MYTICKET”, IN = “FEES”
