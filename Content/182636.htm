Analysis Code Format Types

The new Data Type field on the Analysis Code setup page enables you to specify the following format types for a Dealing analysis code field when entered on the Analysis Code tab on the appropriate Dealing page (or pages).

  • TEXT - no specific format (previous functionality),
  • Number - numeric values with optional Minimum and Maximum,
  • Amount - numeric amounts with optional decimals*, Minimum and Maximum, or
  • Date - valid date formats* only.

*TIP: For a Date or Amount code value, the respective date format or decimal and digit-grouping symbols will be taken from the regional setting for the logged-on User.

Database Implementation

To support the new data types a new dynamic database field is created when an Analysis Code is created of Data Type Amount, Number, or Date. See Dynamically Add Dealing Analysis Codes.

Reporting Pages

Reporting pages have been enhanced to handle the new format types for Analysis Codes, as follows:

  • different Filter operators (equal to, not equal to, less than, and so on) are available based on the Data Type of the Analysis Code when configuring a query,
  • sorting of the field values in the view will use the correct data type (previously, text-based Analysis Code values could only be sorted in text order), and
  • you can now configure Calculated Columns using the values for Analysis Codes with an Amount or Number data type.

Importing Values for Dealing Analysis Codes

When importing deals using the QXT deal import format:

  • Use $$ANALYSIS to identify an analysis code line item.
  • The Analysis Set name, hash separator, and value are used to set the analysis code for the deal.
  • Examples:

    $$ANALYSIS = DealsAnalysisCode01#Abcdefg

    $$ANALYSIS = DealsAnaAmount01#12345.123

    $$ANALYSIS = DealsAnaDate01#30-Sep-2015

    $$ANALYSIS = DealsAnaNum01#50000

When importing deals using the QID deal import format:

  • Use analyseNN, anadateNN, anaamtNN, or ananumNN to identify an analysis code line item (where NN is the code number).
  • The Analysis Set name, pipe separator, and value are used to set the analysis code for the deal.
  • Examples:

    analyse01 = DealsAnalysisCode01|Abcdefg

    anaamt01 = DealsAnaAmount01|12345.123

    anadate01 = DealsAnaDate01|30-Sep-2015

    ananum01 = DealsAnaNum01|50000

Exclusions and Restrictions

Predefined Values are valid only when Data type is TEXT.

The Data type setting is:

  • read-only and set to TEXT for a Static Data analysis code - that is, when Scope is Bank Accounts, Counterparty, Entity, or Instrument Type, and
  • unable to be changed after the record is saved*.

Analysis Code Defaulting Rules may be set up only for Analysis Codes with a Data Type of TEXT.

*TIP: A new Analysis Code will need to be created in order to migrate an existing Analysis Code to a Data Type other than TEXT.
