Importing G/L Journal Entries

G/L Adjustments differ from the other import formats in that each G/L Adjustment record consists of a header and multiple spreadsheet rows.

The Importer uses the SSRow command to support this. To understand how SSRow is used in both tab-separated and QID formats, see G/L Journal Entries, QID Format Example and the table below.

Field Identifier

Description

deal_no

Deal number

Specify the deal number to modify or delete an existing deal.

comments

Optional free format narrative, relating to the whole transaction

cparty

Counterparty

external_ref_id

Optional external reference identifier

external_ref_text

Optional external reference text

deal_dt

Date for which the Journal entries apply

(Trans. Date on the G/L Adjustment form.)

reverse

Indicator to set Automatic reversal on or off

reverse_dt

The date to use on the automatically generated reversals

This should only be specified if Automatic reversal is required.

ssrow

Row increment to position the current row for the remaining spreadsheet fields

The header section in a G/L Adjustment record relates to multiple rows in the spreadsheet section of the record. The Importer uses this command to separate the header and spreadsheet rows from each other.

ssentity

Entity name for this Journal as on the spreadsheet

ssccy

Currency code for this Journal as on the spreadsheet

ssglcode

GL Code for this Journal as on the spreadsheet

ssbase

Base setting for this Journal as on the spreadsheet

If Yes, then the entry amount is expressed in terms of the base CCY of the GL owner, otherwise it will be the nominated currency.

ssdrorcr

Specifies whether the entry is debit (DR) or credit (CR)

ssamount

Journal Entry Amount

sscomment

Journal Entry Comments

ssbankaccount

Enables you to specify which bank account to use when importing a journal entry for a GL code that is attached to more than one bank account

Intercompany Entries

To prevent intercompany G/L adjustments posting to a general suspense account, you cannot use the Importer in Auto Intercompany mode.

Example

(QID format)

exeName=gladj
*QID file, horizontal format
*Header
comments=Test Import with 2 entries in the spreadsheet
cparty=ABC Corp
deal_dt=2/3/98
reverse=yes
reverse_dt=3/3/98
*Spreadsheet first row
ssrow=1
ssentity=Accounting Ltd
ssccy=AUD
sscomment=first ba
ssglcode=1000-1001-00
ssbankaccount=11111[ABCBNKACC
ssdrorcr=Dr
ssamount=55,555.25
*Spreadsheet second row
ssrow=2
ssentity=Accounting Ltd
ssccy=AUD
sscomment=second ba
ssglcode=General Suspense
ssbankaccount=12345[DFGBNKACC
ssdrorcr=Cr
ssamount=55,555.25
$DoMenu=mfinsert
