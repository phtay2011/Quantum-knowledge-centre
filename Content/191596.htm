Analytics Reporting

An Analytics menu is available in all perspectives*. The menu shown below is available in the Default and Analytics perspectives, and provides access to all Analytics Reporting functionality.

*NOTE: In the Adaptiv perspective (available only when Adaptiv Analytics is installed with your Quantum Six site), the Analytics menu is extended to provide access to Adaptiv Reporting pages (in addition to those above). See Adaptiv Features below.

The main features of Analytics Reporting are summarized in the following subsections.

Reports Hierarchy

Use the FIS Quantum Analytics Explorer's Queries page to drill down through Analytics query folders and queries to view and run reports.

NOTE: If you upgrade to Quantum Six from Quantum and Risk Version 5, your previous Query hierarchy is replicated, with some reorganization. See Upgrading to Quantum Six below.

Folders are used to organize the different Analytics queries you can build and run to produce the different types of report. You can create sub-folders under a folder, and within these one or more queries. This enables you to organize reporting by categories such as functionality, deal types, user responsibility, and so on. Queries may reside at the root level or within folders and must have a unique name.

You can navigate to folders and queries using the breadcrumb trail structure or the Search tool.

Configuration and Definition Elements

Before you can create an Analytics query, you need to set up at least the two mandatory Definition elements defining the data you want to report on. You then run a particular query to produce the full result set, and you can apply different views over those results - for example, to produce different formats, subsets of the results, and so on.

An Analytics Query page therefore has two configuration sections (as for Quantum queries).

  • Use Query Configure to specify the query Defaults and Properties, including all the query definitions required, which determine the data reported on and the full result set.
  • Use View Configure to provide report display format/s (Table, Pivot, Chart, or Crystal views, with filters, rows, and columns as required), enabling users to view the same result data in different ways without having to re-run the query.

Limit Audit Reports

The following reports are available in the Analytics menu under Limit Audit Reports:

  • Limit Definition Changes,
  • Breach Approval Changes, and
  • Breaches.

After installing or upgrading to Quantum Six, your system administrators will need to associate a default Crystal report with each of these Limit Audit Reports, using the Quantum Crystal Mapping page. The standard Crystal reports provided are, respectively:

  • LimitAudit.rpt,
  • LimitQueryControl.rpt, and
  • LimitBreachApproval.rpt.

Query Security

The Query Security page is available in the Analytics menu under Security. An example of this page is given below.

Use this page to assign Users and Groups the required access rights to Analytics queries and definitions at various levels, from specific Queries, Definitions, or Explorer folders through to all Queries, all Definitions, or to everything.

NOTE: By default, the initial group EVERYONE has full access at all levels. As new users are added, they must be explicitly assigned the appropriate access rights.

Upgrading Notes

If you upgrade to Quantum Six from Quantum and Risk Version 5, your previous Queries hierarchy is replicated, with some reorganization, as follows.

  • Folders and Workbooks become folders in Analytics.
  • A Worksheet in the Analytics Explorer will become both a Query and a Pivot View in Quantum Six with the previous Worksheet name.
  • The Query definition names in previous versions get changed to the names of the corresponding Worksheets. Where the same Query was attached to multiple Worksheets, the Query definition is cloned and renamed to the applicable Worksheet name.
  • If Queries were not attached to a Worksheet, they will retain the same name, but will be placed in a newly created Lost and Found folder, which appears at the first level of the Reports hierarchy.
  • With the exception of the Lost and Found folder, your new Queries hierarchy should have the same number of parent and child items in the same order as before in the Analytics Risk Explorer.

After upgrading, your systems administrator will need to associate applicable Crystal reports to Analytics Query pages where required, using the Crystal Mapping page. The relevant pages and Crystal reports are indicated in the previous subsections above.
