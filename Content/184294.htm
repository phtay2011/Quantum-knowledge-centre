Handling Exception Flows

Exception flows are those with Pay/Rec set to Exception. They apply only to cash flows or netted groups.

The system will automatically set a cash flow Exception status when a deal save involves a financial change and the cash flow has entered the EFT Lifecycle Phase.

You can also manually do this (in the Settlements Processing Results grid) when:

  • your User record has Module Security rights (for Settlements) to Allow Mark as Exception,
  • only one item (a standalone cash flow or single Netted group) is selected in the Settlements query results grid,
  • the Flow Lifecycle has moved from the EFT Preparation Phase to the EFT Creation Phase (see About Flow Lifecycles), and
  • you select Mark as Exception from the quick menu.

When you select Mark as Exception:

  • Flow Status is set to UNACTIONED,
  • Flow Lifecycle is set to Amended,
  • Pay/Rec is set to Exception, and
  • for a netted group, this applies to the group and all its members.

Actioning Exception Flows

The following quick menu functions (under EXCEPTIONS) enable you to manually reset the Flow Lifecycle for an Exception flow:

  • Action, EFT Created, and
  • Action, EFT Not Created.

These functions are available when:

  • your User record has Module Security rights (for Settlements) to AllowEFT Amendments,
  • only one item (a standalone cash flow or single Netted group) is selected in the Settlements query results grid, and
  • Pay/Rec for that item is Exception.

TIP: The "standard" Action button and Quick Menu function does not apply to, and will ignore, Exception flows.

When you select Action, EFT Created for an Exception item:

  • Pay/Rec is set to Created,
  • Flow Status is set to ACTIONED,
  • Flow Lifecycle is set to EFT Created, and
  • no further processing is required (the item is regarded as fully processed).

When you select Action, EFT Not Created for an Exception item*:

  • Pay/Rec is set to Not Created;
  • Flow Status is set to ACTIONED;
  • the item is re-submitted for Settlement Approval (if required ) and EFT processing;
  • if Settlement Approval is:
    • not required - Flow Lifecycle is set to EFT Ready for Creation, or
    • required - Flow Lifecycle is set to Awaiting Settlement Approval(s), then, following Settlement Approval, to EFT Ready for Creation;
  • If an EFT Cancelation (MTn92) is required, ENSURE THAT before you re-process the EFT, you cancel the previous EFT message and output the Cancelation message, as follows:
    • select the item in the Settlements page's Results grid and select EFT Message from the Quick menu, then click Cancel EFT in the dialog box (this adds a Cancelation message to the EFT processing queue and sets Pay/Rec to Cancel Pending)*; then
    • on the EFT Processing page, run a query to output the Cancelation message; and (lastly)
    • on the EFT Processing page, run a query to output the new EFT message.

*TIPS:

  • You can also cancel a previous EFT message via the EFT Electronic Message page. Run a query that shows the EFT message you want to cancel, select that message in the Results grid, and click Cancel EFT. Then on the EFT Processing page (as above), run a query to output the Cancelation message.
  • Cash flows with Pay/Rec set to Cancel Pending cannot be modified in Settlements. If you change the deal and such a cash flow is affected, the EFT Cancel Request is removed from the EFT Cancel Pending queue.

Exclusions

Exception flows do not apply to stock flows, cash flows linked to stock flows, nor to other deal actions such as RATE FIX, DIARY and so on.
