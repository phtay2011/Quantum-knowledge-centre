Defining Settlement Approval Rule Entries

Use the Approval and Authorization Rule page's Settlement tab to create and maintain, for an Entity (or ALL Entities*), different rule entries for each cash flow, stock flow, and cash flow netted group* Settlement action that has separate approval requirements.

*NOTE: Netted groups of cash flows can potentially contain a mixture of Entities, so a Settlement Approval record with Entity set to ALL is required for these. See also Cash Flow Netting and Settlement Approval Rules.

Key Points

  • Approval Rules can apply to Entities at any level in the business structure.
  • Make sure that your rule entries cover any transactions that might be included in External Confirmation Matching and EFT Processing. Approvals are mandatory with these systems.
  • The first Settlement Approval Rule entry for an Entity should have every selection field set to ALL. This ensures that the system can find at least one approval record to cover every type of action. (Where more specific records exist, the most specific one will be used instead.)
  • To set up tiered approval, specify multiple approval entries with identical Matching Criteria except for the different Limit Amount values giving the minimum threshold amount for each tier, one of which must be zero. See Tiered Approvals Based on Amounts.
  • Only those Approvers in the specified Signature Lists (1st Approval and 2nd Approval) with authority for the specified Entity will be permitted to approve the relevant actions (subject also to their Approval Limit Amount, if any).

    TIP: Some Approvers may have authority to action both approvals at the same time.

  • When approving/rejecting stock flows or cash flows that are part of a linked "package" of flows, more than one Settlement Approval Rule entry may match. In such cases:
    • the entry applying to the stock flow is used to obtain the approval requirements for all the linked flows, and
    • approving/rejecting any one flow in the package approves/rejects all flows in that package.
  • Changes made to Approval Rule records are recorded in the audit log. Use Audit Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.
