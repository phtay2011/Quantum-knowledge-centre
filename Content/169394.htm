Formula Options for Transaction fees

Formula calculations for Transaction fees depend on your selected Base Computation and Base Event, as follows.

Base Computation

Base Event

Selected Formula and calculation details

Sum

Deal Date, Settlement Date, or Maturity Date

Flat Fee

A flat fee is calculated and must be paid if the absolute value of the aggregate Fee Base Amount is greater than zero on a Base Reset Date.

Sum

Deal Date, Settlement Date, or Maturity Date

Simple Percentage

On each Calculation Date, the fee amount is the sum of the calculated Fee Base Amounts on each Base Event Date in the Calculation Period multiplied by the Simple Percentage rate.

There is no adjustment for the number of days in the Calculation Period.

Sum

Deal Date or Settlement Date

Percent per Annum

On each Calculation Date, the fee amount is the Percent per Annum rate multiplied by the calculated Fee Base Amounts on each Base Event Date in the Calculation Period, adjusted for the number of days in that period and the Days Convention specified for the Base template.

Sum

Maturity Date

Percent per Annum

On each Calculation Date, the fee amount is the Percent per Annum rate multiplied by the Fee Base Amount value of each deal for each Base Event Date.

A separate fee is calculated for each applicable deal, and is adjusted for the term of the deal and the Days Convention specified for the Base template.

Count

Deal Date, Settlement Date, or Maturity Date

Flat Fee

A flat fee is calculated and must be paid for each deal with a Base Event occurring on the Base Reset Date. A separate fee amount is calculated for each deal, and is paid on the Payment Date for the Calculation Period.

On the Schedule tab of Fee Deal Schedule, select a Base Reset Frequency of Daily to ensure no Base Event dates are missed.

NOTE: Flat Fee is the only option when Base Computation is Count.
