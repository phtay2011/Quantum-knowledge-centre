Currency Option Posting, Profit and Loss tab

  • Premium Expense - This G/L tag is used to record the payment of premium. The Accounting Rules option determines the accounting treatment.
    • Expense premium - The Premium Expense account is debited at deal save, the contra account being Bank Account or Premium Payable (B/S), depending on the settlement date of the premium.
    • Accrue premium - The Premium Expense account is debited when Accrual Inquiry is run in posting mode, the contra account being Premium Expense Accrual (B/S).
    • Defer premium - The Premium Expense account is debited when Accrual Inquiry is run in posting mode after an exercise, lapse, expiry or knock-out event, the contra account being Premium Expense Accrual.
  • Premium Income - See Premium Expense (Reverse the direction of the accounting entries).
  • Rebate Expense - This G/L tag is used to record the payment of rebate, after an option is knocked out. The following occurs depending on the setting of the Apply Rebate Accrual check box in Accounting Rules:
    • If Apply Rebate Accrual is selected the Rebate Expense account is debited when Accrual Inquiry is run in posting mode after the option has been knocked out and the contra account used is Rebate Expense Accrual (B/S).
    • If Apply Rebate Accrual is cleared the Rebate Expense account is debited when the option is knocked out and the contra account used is Bank Account or Premium Payable (B/S), depending on the settlement date of the rebate.
  • Rebate Income - See Rebate Expense (Reverse the direction of the accounting entries).
  • Base Gain: Realized - This G/L tag is used with the following Revaluation settings in Accounting Rules:
    • Posting method - Realized
    • Revaluation currency - Base

    This account is credited when a P&L gain is calculated, and debited with a decrease in a P&L Gain, the contra account being Base Reval Gain: Realized (B/S).

  • Base Gain: Unrealized - This G/L tag is used with the following Revaluation settings in Accounting Rules:
    • Posting method - Unrealized
    • Revaluation currency - Base

    This account is credited when a P&L gain is calculated, and debited with a decrease in a P&L Gain, the contra account being Revaluation Gain: Realized (B/S).

  • Base Loss: Realized - See Base Gain: Realized. (Reverse the direction of the accounting entries).
  • Base Loss: Unrealized - See Base Gain: Unrealized. (Reverse the direction of the accounting entries).
  • Gain: Realized - This G/L tag is used with the following Revaluation settings in Accounting Rules:
    • Posting method - Realized
    • Revaluation currency - Non base or Premium

    This account is credited when a P&L gain is calculated, and debited with a decrease in a P&L Gain, the contra account being Revaluation Gain: Realized (B/S).

  • Gain: Unrealized - This G/L tag is used with the following Revaluation settings in Accounting Rules:
    • Posting method - Unrealized
    • Revaluation currency - Non base or Premium

    This account is credited when a P&L gain is calculated, and debited with a decrease in a P&L gain, the contra account being Revaluation Gain: Unrealized (B/S).

  • Loss: Realized - See Gain: Realized. (Reverse the direction of the accounting entries).
  • Loss: Unrealized - See Gain: Unrealized. (Reverse the direction of the accounting entries).
  • CCY Translation Gain: Realized - There is no currency translation functionality at deal level for currency options. This is taken up at the G/L account templates level.
  • CCY Translation Gain: Unrealized - There is no currency translation functionality at deal level for currency options. This is taken up at the G/L account templates level.
  • CCY Translation Loss: Realized - There is no currency translation functionality at deal level for currency options. This is taken up at the G/L account templates level.
  • CCY Translation Loss: Unrealized - There is no currency translation functionality at deal level for currency options. This is taken up at the G/L account templates level.
