Roll/Renegotiate form for Repo Deals

The Roll/Renegotiate form enables you to:

  • Change the principal amount or extend the maturity date of an existing Repo (Roll or Renegotiate the Repo).
  • Amend certain information about the Repo without renegotiating it (for example, the ticket number, and the setting that controls whether or not the Repo should be included in bulk Renegotiation).

NOTE: You can later roll back (undo) these actions if necessary - see Rolling Back to an Earlier Version of a Deal.

Items on the form

NOTE: Some of these items described below may not be visible for certain types of Repo.

Ticket Number - This field displays the Ticket Number of the Repo deal.

If you change the Ticket Number on this form, the change will be updated on the original Repo deal.

Renegotiation / Amendment - Select Renegotiation to change the principal or extend the maturity of the Repo; select Amendment to change the Bulk Renegotiate setting for the Repo, as well as secondary information such as the ticket number and comments.

Effective Date - Enter the date you want the Renegotiation to take effect. This defaults to the Maturity Date of the original Repo, but you can change it if necessary.

For example, if you want to use the Renegotiation form to alter the principal of a Repo part way before maturity, you would enter the date that the principal change is to occur in this field.

New Maturity Date - If you are renegotiating the Repo to extend the maturity date, enter the new date in this field.

NOTE: You cannot make the new Maturity Date earlier than the existing Maturity Date on the Repo.

Term - This field displays the new term for the Repo, in Business Days. This is the number of business days by which the term of the Repo has been extended.

Currency - Read-only.

The Currency of the existing Repo.

NOTE: This field cannot be changed, even if the Repo or underlying instruments are modifiable.

Instrument - The name of the Repo Instrument Type.

Bulk Renegotiate - Select YES (the default option) to make this Repo or Rolled deal available for renegotiation in the Roll Deals page; select NO to prevent this deal from being renegotiated using the Roll Deals page.

Interest Options - Specify how to treat the interest outstanding on the original deal maturity date, as follows.

Select …

To …

Pay

Settle the interest payment or receipt on the original maturity date.

Accumulate

Defer the interest payment or receipt to the new maturity date.

Capitalize

Add the interest amount to the principal value of the Repo on the original maturity date.

(Not available for Buy/Sell Backs or Sell/Buy Backs.)

No Repo Rate - If the renegotiation requires a new Repo Rate, enter the new rate in this field.

Interest to Date - This is the Repo interest calculated up to the original Maturity Date.

Principal Adjusted - You can change increase or decrease the amount of principal on the Repo.

  • Enter a positive amount to increase the principal.
  • Enter a negative amount (for example, -1,000,000.000) to reduce the principal.

Principal Rolled - This field shows the amount of principal to be rolled. This is equal to the original principal amount plus any adjustments entered in the Principal Adjustments field.

NOTE: If you change this field directly, the Principal Adjustments field will be recalculated (when you change the focus).

Interest Amount - This field displays the calculated additional interest resulting from the renegotiation.

Total Outstanding - This field displays the total interest on the Repo - that is, the original principal amount adjusted to account for the effects of Principal Adjustments and changes in Maturity Date as a result of the renegotiation.

The remaining fields summarize the details of the underlying cash or security transactions on the Repo.
