Market Data Archive page (Adaptiv)

System Administrators use the Market Data Archive page to configure the Market Data Archive utility (MDA) and (re)generate MDA data.

Market Data Archive data contains histories of the values of (series of dates and corresponding factor values). Usually these numbers are the points making up price factors, but they don’t need to be. Price factor point histories are used by the Market VaR (Historical Simulation) calculation and by some of the calibrations for price factor models. Market price point histories are used for rate fixings.

You can create multiple records with different sets of parameter values. These values specify the market data base currency, the start and end dates for extracting data, which price factors to archive, and so on. The system provides three system-defined configurations, named Default, RiskMngt, and VaR. Use Default and VaR in combination with the Default and Valuation Control records, respectively.

When you run the MDA utility:

  • The system extracts historical price factor data that matches the current MDA record and stores this in the database. You can then use that data to calibrate models or calculate correlations between price factors by specifying the MDA Configuration name for Model Calibration or Correlation Feed runs, respectively.
  • You can select or clear the Incremental Update run option to specify whether Adaptiv incrementally updates or overwrites the MDA when you regenerate it. Default = selected.

    TIP: Selecting this option enables large databases to run more quickly.

  • You can optionally export your run results in CSV format or the proprietary ADA format by selecting Export CSV or Export ADA in the page menu, respectively. For more details, see CSV file format details and ADA file format detailsbelow.
  •  When you run from the Command line, End Date and As At Date are set to Today.

Data descriptions

  • Code - the unique identifier for the current MDA record
  • Name - a descriptive name for the current MDA record
  • Base Ccy - the Base Currency of the market data to be archived
  • Adaptiv Analytics Rate Configuration - select an Adaptiv VaR Rates Definition record for the rate to be archived
  • Valuation Models - select an Adaptiv Valuation Model record
  • Valuation Control - select an Analytics Valuation Rule record
  • Archive Zero Curve Grid - enter the Zero Curve tenors, in order, separated by spaces - for example, the Default: 0d 1d 2d 1w 2w 1m 3m 6m 9m 1y 1y6m 2y 3y 4y 5y 6y 7y 8y 9y 10y 15y 20y
  • Archive Start Date - the earliest date of the market data to be archived. Blank if no previous successful run.
  • Archive End Date - the latest date of the market data to be archived. Blank if no previous successful run.
  • As At Date - the date at which to view the database. Any changes after this date will not be included.
  • Last Run Date – The date that a run (successful or otherwise) was last performed. Blank if no previous successful run.
  • Last Run Status – The status (Failure/Success/none) of the last run. Blank if no previous successful run.
  • Run Start Date - The date you want the MDA run to start.
  • Run End Date - The date you want the MDA run to go up to and include. Must be greater than Archive End Date from Message Bar.
  • Valuation Models - the Adaptiv Valuation Model record to be used.
  • Valuation Control - the Analytics Valuation Rule record to be used.
  • Archive Zero Curve Grid - the Zero Curve tenors for the rate data to be archived. These are entered as a series of time points in time order, separated by spaces, and made up of a number and unit abbreviation. The abbreviation is in the format nx, where:
    • n is the number of time units and
    • x is the unit type: d, w, m, or y (for days,weeks, months, or years).

      • Compound points for part-months and/or part-years are allowed - for example 2y6m.

CSV file format details

This format is comma-delimited and is designed to be easily opened and copied to and from Excel. Details of the format follow.

  • The first row represents the header record.
    • The first item in the header record will always be DATE followed by the header items.
    • The header items are formatted as <factor type><subtype>.<factor ID>[;<point ID.]. For example InterestRateCreditSpread.USD.AA;6m where: <factortype> is InterestRate, <subtype> is CreditSpread, <factor ID> is USD.AA and <point ID> is 6m.

      Point ID is used if there is more than one point in the price factor and it consists of a list of 0 or more coordinates depending on the factor type, For example:

      • InterestRate.EUR-Default Zero Curve 3M End of Day;1m has a single coordinate Point ID representing the term, and
      • InterestRateVol.EUR-IO Normal Volatility 1M End of Day;-0.07502614715054;0M;1.75342465753425 has a list of three coordinates for Point ID representing Time to Expiry, Tenor and Moneyness.
  • Subsequent rows represent the data, where:
    • the first value is the date in YYYY-MM-DD format,
    • rows will be in ascending order,
    • data values are in Decimal format,
    • missing data is indicated by an empty cell, and
    • the final row is an empty row.

ADA file format details

This format is tab-delimited, must use UTF-8 encoding, and also can be opened with Excel or a text editor.

Details of the format follow.

  • The first two lines are taken as comments.
  • Only the first two fields are read from line 3. These fields are the number of columns (excluding the DATE column) and the number of date rows.

  • Line 4 identifies the item columns and is formatted as <factor type>.<factor ID>[,<point ID>]. <point ID> is used if there is more than one point in the price factor, and consists of a comma-separated list of 0 or more coordinates depending on the factor type.

    For example:

    • InterestRate.USD.AA,6m

    ...where <factor type> is InterestRate, <factor ID> is USD.AA, and <point ID> is 6m.

  • Subsequent rows represent the data, where:
    • the first value is the date in the Excel convention of number of days since 31 December 1899,
    • rows must be in ascending date order,
    • data values are in Decimal format with a period as the decimal separator,
    • missing data is indicated by an empty cell, and
    • trailing separators are not required after the last non-empty cell.
