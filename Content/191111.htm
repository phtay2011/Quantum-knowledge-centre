Limit Detail and Matching Criteria

Each Limits Definition record has one or more Limit entries. This topic describes the Limit Detail and Matching Criteria values defining a Limit entry.

Matching Criteria

These values determine the result set records to which the Limit Details will apply.

TIP: Include an entry with All Matching Criteria settings to catch deals that do not fall under any other Limit.

Counterparty - The lookup lists counterparties defined as Deal External or Deal Internal. Select All or a specific Counterparty or Counterparty group.

Country - Select All or a specific Country or Country group.

Industry - Select All or a specific Industry or Industry group.

Counterparty Rating - Used in conjunction with the Counterparty setting, select All or a specific rating.

Country Rating - Used in conjunction with the Country setting, select All or a specific rating.

Currency - Select a All or a specific transaction Currency or Currency group.

Entity - Used in conjunction with other Limit definitions, select All or a specific Entity or Entity group.

Transaction Type - Select All or a specific transaction type.

Instrument - Select All or a specific Instrument or Instrument group.

Dealer - Select All or a specific Dealer or Dealer group.

Facility - Used in conjunction with the Instrument setting, select All or a specific Facility or Facility group.

Unit Issue - Used in conjunction with the Instrument setting, select All or a specific Unit Issue.

Term Start and Term End

Limits can optionally be set up to apply only over a specific term, relative to transaction maturity. Use these field pairs to specify the start and end of the maturity date range (relative to the applicable As At Date).

In each field pair:

  • leave the lookup field on the right blank for no start or end point, or
  • in that lookup, select Spot (the Spot Date), Overnight, or a time unit (Days, Weeks, Fortnight, Months, Quarter, Half Year, or Years), and
  • if you selected a time unit, enter the number of those units in the numeric field on the left.

When result set records are being checked against a Limits Definition, the As At Date is used as the base date from which actual dates are derived.

If you enter both values, the End point cannot precede the Start.

Limit Detail

These values define the limit to be applied when result set records meet the Matching criteria for the Limit entry.

Limit Currency - Select All or a specific currency in which the Limit Amount is denominated.

Limit Amount - Enter the Limit amount.

Start Date -and End Date

A Limit entry can optionally be made active for a defined period of time relative to transaction maturity. Use these field pairs to specify the start and end of the maturity date range (relative to the applicable As At Date).

In each field pair:

  • leave the lookup field on the right blank for no start or end point, or
  • in that lookup, select Spot (the Spot Date), Overnight, or a time unit (Days, Weeks, Fortnight, Months, Quarter, Half Year, or Years), and
  • if you selected a time unit, enter the number of those units in the numeric field on the left.

When result set records are being checked against a Limits Definition, the As At Date is used as the base date from which actual dates are derived.

If you enter both values, the End point cannot precede the Start.

Review Date - Optionally enter a date when the Limit entry details should be reviewed. This is typically set a few days prior to the Limit Detail End Date.

Utilization Method - Specific to Limit Checking, select from the available options:

  • Deal: Limit utilization is the utilization for a single deal.

    LimitUtilization = DealUtilization

  • Sum - Limit utilization is the sum of utilizations for deals that map to the limit record.

    = sum(Limit RecordUtilization)

  • Percent - Limit utilization is the sum of utilizations for deals that map to the limit record as a percent of the sum of utilizations for the query.

    = (sum(Limit RecordUtilization)/sum(QueryUtilization))*100

  • Weighted Average - Limit utilization is a weighted average of utilizations for deals that map to the limit record.

    = sum(Limit RecordUtilization * Limit RecordWeight)/sum(Limit RecordWeight)

    If the Weighted Average utilization method is being used, the query must also contain a column called Weight.
