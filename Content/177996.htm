For EFT Output

Category 1 - Customer Payments and Cheques

Format

Used for...

MT101

Payments (*batched or unbatched) when:

  • The Payment Category for the Counterparty is Standard Settlement Instructions

-and-

  • Payment is NOT to a Financial Institution

-and-

The Counterparty is mapped to MT101 in the SWIFT Mappings.csv file, or (if unspecified or no file) MT101 is the default.

 

Bank Account Transfers (*batched or unbatched) between Financial Institutions when:

  • the Payment Category for the Counterparty is Standard Settlement Instructions, and
  • the SWIFT Message Type option for the Bank Account Transfer instrument is set to MT101.
  • The message type is NOT overwritten by the use of EFT Message Defaults settings for the applicable External Method Source - namely, Non-Financial Transfer and/or Counterparty Mapping.

*NOTE: Non-netted cashflows for the same deal on the same day, with the same settlement instructions (and no AdHoc instructions) are automatically batched into one MT101 message (up to ten cash flows per message).

MT103

Payments (unbatched)

  • When NOT all parties are Financial Institutions

-and-

  • The Counterparty is mapped to MT103 in the SWIFT Mappings.csv file, or (if unspecified or no file) MT103 is the default.

 

Bank Account Transfers between Financial Institutions when:

  • The SWIFT EFT run is started without the AT_DEFAULT=MT200 command line parameter specified.

-and-

  • The SWIFT Message Type option for the Bank Account Transfer instrument is set to MT103.

MT103+

Payments (unbatched)

  • When NOT all parties are Financial Institutions

-and-

  • The Counterparty is mapped to MT103+ in the SWIFT Mappings.csv file, or (if unspecified or no file) MT103+ is the default

Category 2 - Financial Institution Transfers

Format

Used for...

MT200

Payments (unbatched) for Bank Account Transfers only when:

  • all parties are Financial Institutions, and
  • the SWIFT Message Type option for the Bank Account Transfer instrument is set to MT200.

MT202

Payments (unbatched)

  • When all parties are Financial Institutions

-and-

  • Batch Messages is set to Receipts or None (or does not apply)

-and-

  • The MT200 alternative (Bank Account Transfers only) does not apply - see above.

MT203

Payments (batched up to ten per message)

  • When all parties are Financial Institutions

-and-

  • Batch Messages is set to Payments or Both

-and-

  • Payment Method is not for TARGET

MT210

Advice-to-Receive messages (for cashflow amounts greater than zero):

  • Un-batched (one advice per message) if Batch Messages is set to Payments or None (or does not apply)
  • Batched (up to ten advices per message) if Batch Messages is set to Receipts or Both

Category 5 - Securities Markets

Format

Used for...

MT540

Advice messages for receipt of Securities (stock) free of payment - for example for:

  • Stock Account Transfers
  • Repo deals with underlying Money Market Instruments that have stock flows specified

MT541

Advice messages for receipt of Securities (stock) against payment made

  • For Security deals based on Instruments with stock flows specified
  • Also covers Equity transaction types (Unit Deals, Unit Holdings)
  • Output includes the associated cash information (no separate cash payment message is output)

MT542

Advice messages for delivery of Securities (stock) free of payment - for example, for:

  • Stock Account Transfers
  • Repo deals with underlying Money Market Instruments that have stock flows specified

MT543

Advice messages for delivery of Securities (stock) against payment received

  • For Security deals based on Instruments with stock flows specified
  • Also covers Equity transaction types (Unit Deals, Unit Holdings)
  • Includes the associated cash information (no separate cash receipt advice message is output)

Category 6 - Treasury Markets - Commodities

NOTE: These message types are NOT supported for Commodity Swap deals.

Format

Used for...

MT604

MT604 Commodity Transfer / Delivery Order

Output only for Commodity currency types when:

  • an XML Template is specified for the Payment Method (in External Method Sources)
  • the flow amount is positive

MT605

Commodity Notice to Receive

Output only for Commodity currency types when:

  • an XML Template is specified for the Payment Method (in External Method Sources)
  • the flow amount is negative

Category 9 - Cash Management and Customer Status

Format

Used for...

MT900

Confirmation of Direct Debit

Applies for a debit to the Counterparty account, where the Payment Category is set to Debit / Credit Advice, as follows:

  • In Settlements, if specified for the individual flow in Cparty Delivery Instructions (Adhoc Instructions or Third-Party Payments mode),

    -otherwise-

  • In Counterparty Delivery Instructions, for the Counterparty bank account.

NOTE: The default is Standard Settlement Instructions, which outputs standard SWIFT messages (MT103, 103+, 200, 202, 203, and 210).

MT910

Confirmation of Direct Credit

Applies for a debit to the Counterparty account, where the Payment Category is set to Debit / Credit Advice, as for MT900 (above).

Category n - Common Group Messages

  • MTn92- Cancellation messages

    The MTn92 Cancellation message was updated in v2019.4 (6.8.2).

    Format

    Used for...

    MT192

    Cancellation message for a previous MT103 or MT103+ message

    MT292

    Cancellation message for a previous MT202, MT203, or MT210 message

    NOTE: An MT292 may not request cancellation of an MT 200. Instead, the Sender must send a new message to the account with institution of the initial message, requesting a re-transfer of the funds.

    MT692

    Cancellation message for a previous MT604, or MT605 message

    MT992

    Cancellation message for a previous MT900 or MT910 message

    NOTE: Created manually using the Free Format MTn99 page.

    NOTE: Created manually using the Free Format MTn99 page.

ISO 20022 Messages

Payments Clearing and Settlement (pacs)

pacs.008

FIToFICustomerCreditTransfer

The message is sent by the debtor agent to the creditor agent, directly or through other agents and/or a payment clearing and settlement system. It is used to move funds from a debtor account to a creditor.

MX-equivalent to the MT103.

Unbatched message via File per EFT processing.

pacs.009 (core)

FinancialInstitutionCreditTransfer

The message is sent by a debtor financial institution to a creditor financial institution, directly or through other agents and/or a payment clearing and settlement system. It is used to move funds from a debtor account to a creditor, where both debtor and creditor are financial institutions.

MX-equivalent to the MT202 and MT200.

Unbatched message via File per EFT processing.

Payments Initiation (pain)

pain.001

CustomerCreditTransferInitiation Supported SWIFT Message Formats

The message is sent by the initiating party to the forwarding agent or debtor agent. It is used to request movement of funds from the debtor account to a creditor.

MX-equivalent to the MT101

Batched or Unbatched message via File per EFT, Single File, or File per day processing.
