Schema File (Commands.xsd)

Command

A command defines a server-side executable or script that can be started from a web page. The page will prompt the user with fields appropriate to the variables defined by the command.

variables

Optionally define one or more variables that can be used by the arguments on the command or the option.

options

Optionally define one or more variations of how the command can be executed. Each option can have different predefined values.

outcomes

Optionally define one or more possible outcomes of the command so that the web page can return a user-friendly message for each outcome.

name

The name of the command. This is the text that will display in the menu and as the title of the web page.

description

Optionally provide a description that is displayed at the top of the web page when the user runs this command interactively.

module

The custom module associated with the command. Use this to set up module security. The custom module must be of type Command.

filename

The file name of the executable or script that needs to be run on the server.

workingDirectory

The working directory that will be used when the command is running.

arguments

The arguments that will be presented to the command. Use the syntax %variable% to refer a variable. For example, use /StartDate %From% /verbose to define where the value of the variable called From should be substituted in the arguments.

timeout

Optionally provide a timeout period in seconds to ensure the command definitely ends. Use this when the executable or script depends other components that may not always be available.

Variable

Define the variable details.

output

Define how the value of the variable is presented to the executable or script.

constraints

Define constraints on the variable.

values

Optionally provide possible values for the variable.

name

The name of the variable.

description

Optionally provide a description of the variable. This text will be displayed with the input field on the web page.

datatype

Define the data type of the variable (Amount, Date, Number, or Text).

Value

A possible value for the variable. This value will be passed to the command and presented on the web page unless a text attribute is defined.

default

Optionally define whether this value is the default for the variable. A default value will be put into the input field when the web page comes up.

text

Optionally provide a text representation that needs to be displayed on the web page instead of the value.

Option

Define a variation of how the command is to be executed.

name

The name of the option. This will appear on the web page as a choice. Make the name descriptive.

arguments

The arguments that will be presented to the command. Use the syntax %variable% to refer a variable. For example, use /StartDate %From% /verbose to define where the value of the variable called From should be substituted in the arguments.

default

Optionally define whether this option is the default. On the web page, this option will be selected when the page displays.

Output

Define a variation of how the command is to be executed.

format

The format of the date or decimal variable.

Constraints

Define constraints on the variable.

decimalPlaces

Optionally provide the number of decimal places that need to be used in the input field.

min

Optionally provide the minimum value of the variable.

max

Optionally provide the maximum value of the variable.

restrictToValues

Optionally define whether this variable can only be set to the predefined values.

Outcome

Define the possible return codes of the command and define a user message for each of them.

result

The numeric return code that can be returned by the command.

status

Define whether the message should be represented as a success or failure in the Notifications panel.

message

The message that needs to be displayed in the Notifications panel when this outcome is returned by the command.

default

Optionally define whether the outcome is to be used when an unknown result is returned.
