Functions (Expressions in Calculated Columns)

This list applies to Analytics.

Functions are predefined commands whose arguments are evaluated in a specific order. They provide the foundation for calculating columns within queries.

Abs

Description

Returns the absolute value of a number (the unsigned value)

Format

Abs(number)

Arguments

number - The number or column name whose absolute value you want.

Example

Abs(Market Value)

If Market Value is -123,456.00, the result is 123, 456.00.

AverageCommodityPrice

Description

Returns the average price, over a specified date range, for a given Commodity, Currency, Measure, and Commodity Rate curve.

NOTE: Supported only for Analytics Position queries.

For more details, see Average Commodity Price function.

Format

AverageCommodityPrice(ccy,rate_name,basis_rate_name,commodity,
measure,start_date,end_date,position_date,as_at_date,term,
term_type,frequency,bdc,fixing_date,interpolation,period_average,
method,bidoffermid,holiday_center1|holiday_center2|...,rounding)

Arguments

ccy

The currency linked to the Commodity Rate to be averaged.

rate_name

The rate name which together with ccy, measure, and commodity determine the Commodity curve used in the calculation for historical points (up to position_date) and, if a rate is specified for it, on position_date also.

basis_rate_name

The rate name which together with ccy, measure, and commodity determine the Commodity curve used in the calculation for forward points (after position_date); if omitted, defaults to rate_name.

commodity

The commodity linked to the Commodity Rate to be averaged.

measure

The measure linked to the Commodity Rate to be averaged.

start_date

The start of the averaging period. The first rate record used in the calculation will have a Rate Date equal to, or immediately preceding this date**.

end_date

The end of the averaging period. The last rate record used in the calculation will have a Rate Date equal to or preceding this date**.

position_date

The date before which rates will be extracted from historical Commodity rate records, and after which Forward Rates will be interpolated from a single (last available) rate record.

For dates on or before this date, the average of the Bid/Offer rates from each rate record will be extracted.

For dates after this, the position date rate record will be used to interpolate Forward Rates at the same frequency. The last Forward Rate interpolated from this rate record will be that on or before the end_date.

as_at_date

The effective date that determines the "time slice" of the database - that is, the data used reflects the values in the database at that time.

term

Zero or a positive integer that together with term_type gives the point on the Commodity Rate curve for selecting the rate to be averaged.

term_type

Spot, Overnight, Tom Next or a time unit (Days, Weeks, Months, or Years) that together with term gives the point on the Commodity Rate curve for selecting the rate to be averaged.

frequency

The frequency at which the rate will be averaged. Currently only "DAILY" is supported.

bdc

(Business Day Convention) Determines which rate/forward date to select if the rate fixing falls on a non-business day (NBD): A (Actual; the default), F (Following), MF (Modified Following*), or P (Preceding). This convention will also apply to the start_date, end_date, and position_date arguments if any of these are set to a Non Business day.

fixing_date

Not currently used, a zero value 0 is required for this argument.

interpolation

The interpolation method between points: "L" (Linear) or "F" (Flat; the value at the end point).

period_average

Not currently used, a value "FULL" is required for this argument.

method

The type of average (mean) to apply to the extracted rates: A (Arithmetic) or G (Geometric).

bidoffermid

Currently only "MID" (Mid) is supported. This is the average of the Bid and Offer values in the Rates grid row with the matching term and term_type settings.

holiday_center1 | holiday_center2 | ...

Optional predefined Holiday Center identifiers of specific holiday calendars also to be applied, in tandem with the bdc value, to determine the behavior when start_date, end_date, position_date, or one of the averaging (fixing) dates falls on** a holiday**. Multiple identifier values must be separated by the pipe character |.

rounding

The number of decimal places to round the result to, using the 5/4 method, with -1 indicating NO ROUNDING.

**NOTE: If there is no rate for the Start Date and it is a holiday, the applicable Business Day Convention (BDC) will be ignored if it would adjust the Start Date to an earlier date; instead, the Start Date point is treated as missing rate and will not be included in the calculation.

Similarly, if there is no rate for the End Date and it is a holiday, the applicable BDC will be ignored if it would adjust the End Date to an later date; instead, the End Date point is treated as missing rate and will not be included in the calculation.

Examples and Results

Example 1

AverageCommodityPrice("USD", "LME Zinc Closing Daily Market Rates","LME Zinc Closing Comm" ,"Zinc", "Pound", QueryStartDate, QueryEndDate, QueryPositionDate, QueryAsAtDate, 0, "SPOT", "DAILY", "A", 0, "LINEAR", "FULL", "A", "MID", "United States",-1)

Result 1

Calculates the arithmetic mean price calculated from the LME Zinc Closing Daily Market Rates and LME Zinc Closing Comm records for a pound of Zinc in USD currency, using:

  • the MID of Spot rates,
  • the United States Holiday Center,
  • the query's As at Date as the Effective Date,
  • rates taken daily between the query's specified Start Date and End Date,
  • the Actual business day convention, and...
  • with no rounding of the result value.

Example 2

AverageCommodityPrice("USD", "LME Zinc Closing Daily Market Rates","LME Zinc Closing Comm" ,"Zinc", "Pound", QueryStartDate, QueryEndDate, QueryPositionDate, QueryAsAtDate, 3, "DAYS", "DAILY", "MF", 0, "LINEAR", "FULL", "A", "MID", "United States | NZ", 5)

Result 2

Similar to Example 1, except that:

  • the Term and TermType specify 3 Days,
  • there are two Holiday Centers (United States and NZ),
  • the result is rounded (5/4) to 5 decimal places, and...
  • BDC is MF (Modified Following) meaning that holidays will be excluded (See Note above**).

AverageInterestRate

Description

Returns the average rate as a multiplier for a given currency, Yield Rate curve, over a specified date range.

For more details, see Average Interest Rate Function in Analytics Calculated Columns.

Format

 

AverageInterestRate(ccy, rate_name, start_date, end_date, as_at_date [, term, term_type [, method, denominator, margin, suspension, holidays_list, rounding, lag, lagtype] ] ])

Fixed values must be within quotes (see the Examples below).

Arguments

 

ccy

The currency linked to the Yield Rate to be averaged.

rate_name

The rate name which together with ccy determines the Yield Rate curve used in the calculation.

start_date

The start of the averaging period. The first rate record used in the calculation will have a Rate Date equal to, or immediately preceding this date.

end_date

The end of the averaging period. The last rate record used in the calculation will have a Rate Date equal to or preceding this date.

as_at_date

Determines the "time slice" of the database - that is, the data used reflects the values in the database at that time.

term

Zero or a positive integer that together with term_type gives the point on the Yield Rate curve for selecting the rate to be averaged.

The applicable mid rate is used. This is the average of the Bid and Offer values in the Rates grid row with the matching Term and Term Type settings. If there is no matching row, an error will be reported.

Optional.

term_type

Overnight (the default), Tom Next, or a time unit (Days, Weeks, Months, or Years) that together with term gives the point on the Yield Rate curve for selecting the rate to be averaged.

Required if term is present, must be absent if term is absent.

method

The type of average (mean) to apply to the extracted rates:

  • G or Geometric (the default) for the geometric mean (for compounding rates), or
  • A or Arithmetic for the arithmetic mean (the sum of the rates divided by the number of rates),
  • B or BusinessGeometric is like Geometric but the compounding is applied on Business Days only: it applies simple interests formula on non business days, or
  • R or ReturnGeometricAverage, that calculates the Geometric Average Return,
  • optional if term and term_type are present,
  • not required (and must be absent) if term and term_type are absent.

margin

Margin to apply to the rate and to compound with the same frequency. Default value is zero (0).

rounding

Number of decimal digits used to round the intermediate rates using 5/4 rounding. Default value is -1, that means no intermediate rounding. This parameter is ignored for "R" method.

lag

Number of Business Days (k): for every day in the interest period we use the rate from k business days earlier.

lagtype

Rate Only (the default): only the rate date is shifted, the day weights remain the ones from the interest period. Rate & Weight: the rate dates are shifted and the day weight are calculated on the new observation period

denominator

Number of days per year (DPY). If not present or 0, this value is taken from the rate "Day Count". Ignored for "R" method.

suspension (or lockout)

Number of Business Days (N+1) at the end of the interest period for which the rate is constant and equal to the one fixed N Business Days before the period ends. Default is 0.

last_int_period_accrual_date

The last day of the whole coupon period that the interest is accrued. If not present, its value is 0; otherwise its value should be equal to “Coupon End Date” – minus 1 Calendar Day. This date, together with end_date, dictates how we use Suspension to calculate the Average Rate.

holidays_list

List of holidays: list of holiday centers separated by "|". For example, "GBP|London". If not present, the rate currency is used.

Example 1

AverageInterestRate("EUR","EURIBOR",QueryStartDate,
QueryEndDate,QueryAsAtDate)

Result 1

Gives the geometric (compounded) average EURIBOR Yield rate for the EUR currency, using the Overnight rates.

The rates will be taken between the query's specified Start Date and End Date, and the query's As At Date applies.

The term, term_type, and method parameter values have not been specified, so the Overnight and Geometric default settings apply.

Example 2

AverageInterestRate("AUD","BBSW","3","MONTHS",QueryStartDate,
QueryEndDate,QueryAsAtDate,"A")

Result 2

Gives the arithmetic mean BBSW Yield rate for the AUD currency, using the 3 MONTHS rates.

The rates will be taken between the query's specified Start Date and End Date, and the query's As At Date applies.

Example 3

AverageInterestRate ("GBP","SONIA",StringToDate("28-01-2019","dd-mm-yyyy"), StringToDate("22-02-2019","dd-mm-yyyy"),QueryAsAtDate,
"0","OVERNIGHT","B",0,0.075,0,"London")

Result 3

Gives the average compounded (avRb) SONIA rate for the GBP currency, using the Overnight rate curve points. The rates are taken between 28/01/2019 and 22/2/2019, and the query's As At Date applies.

The denominator is 0, so it is taken from the SONIA rate definition.

A 0.075 margin is applied each day.

The London calendar is used to retrieve the Business Days.

Example 4

AverageInterestRate ("USD","SOFR",StringToDate("28-01-2019","dd-mm-yyyy"), StringToDate("22-02-2019","dd-mm-yyyy"),QueryAsAtDate,
"0","OVERNIGHT","B",360,0,2)

Result 4

Gives the average compounded (avRb) SOFR rate for the USD currency, using the Overnight rate curve points. The rates will be taken between 28/01/2019 and 22/2/2019, and the query's As At Date applies.

The denominator is 360.

0 margin is applied.

2 Business Days suspension (lockout) are used. This means that the rate of the 20/2/2019 is applied until the end of the period.

The Holidays List is not specified, so the currency is used as Holiday Center to retrieve the Business Days.

no intermediate rounding.

Example 5

AverageInterestRate ("AUD","BBSW",QueryStartDate,QueryEndDate,QueryAsAtDate,"0","OVERNIGHT","A",0,0.025,2,"",7)

Result 5

Gives the arithmetic mean (Rarith) BBSW Yield rate for the AUD currency, using the Overnight rate curve points. The rates will be taken between the query's specified Start Date and End Date, and the query's As At Date applies.

The denominator is 0 so it is taken from the BBSW rate definition.

0.025 margin is applied.

2 Business Days suspension (lockout) are used, that means that the rate of the (QueryAsAtDate – 2 BD) is applied until the end of the period.

The Holidays List is not specified so the currency is used as Holiday Centre to retrieve the Business Days.

The intermediate rates are rounded to 7 decimal digits.

Description

Returns a business day (date) for a given date and business day convention.

Format

BusinessDay(date,"bdc","holiday_center1","holiday_center2",…)

Arguments

  • date - The base date.
  • bdc - The business day convention - one of:

    A - Actual

    F - Following

    MF - Modified Following

    P - Preceding

  • holiday_center1, holiday_center2, ... - The holiday center/s or calendar/s used to determine business days.

Example

Description

Converts date values to text strings.

Format

Arguments

  • date - The date you want to convert to text.
  • format - The date format to be displayed; one or more of the following with optional inserted characters (if not specified, defaults to the short date format set in Windows Regional Options):

    d - Day of month number with no leading zero if under 10

    dd - Day of month number with leading zero if under 10

    M - Month number with no leading zero if under 10

    yyyy - Year's full four digits

FXAverageRate

Description

Returns the average FX rate as a multiplier for a given currency pair, rate curve, and date range.

Format

FXAverageRate(ccy1,ccy2, "fx_rate_name", start_date, end_date, position_date, as_at_date, "frequency", "bdc", fix_date, "method", "holiday_center1", "holiday_center2",,,)

Arguments

 
  • ccy1

The first currency of the pair (normally a commodity where applicable)
  • ccy2

The second currency of the pair.
  • fx_rate_name

The FX rate name that, with the currency pair, determines the FX rate curve used in the calculation.
  • start_date

The start of the averaging period. The first rate record used within the averaging calculation will have a rate date equal to this date, unless a separate fixing_date is specified or this date is a holiday (see below).
  • end_date

The last point for the average calculation. The last rate record or interpolated Forward Rate used within the averaging calculation will have a rate/forward date equal to or preceding this date, depending on the fixing_date and holiday settings (see below).
  • position_date

The date before which Spot Rate fixings will be extracted from historical FX rate records, and after which Forward Rates will be interpolated from a single (last available) rate record.

  • For dates before the position_date, the average of the Bid/Offer Spot rates from each rate record will be extracted.
  • For dates after the position_date, the position date rate record will be used to interpolate Forward Rates at the same frequency. The last Forward Rate interpolated from this rate record will be that on or before the end_date.
  • as_at_date

Determines the "time slice" of the database - that is, the data used reflects the values in the database at that time.
  • frequency

The frequency at which rates will be averaged: D (daily), W (weekly), M (monthly), Q (quarterly), SA (semi-annual), or A (annual).

With a daily frequency, the first rate record used for averaging will be on the start_date (unless a holiday - see below).
  • bdc

(Business Day Convention) Determines which rate/forward date to select if the rate fixing falls on a non-business day (NBD): A (Actual; the default), F (Following), MF (Modified Following*), or P (Preceding). This convention will also apply to the start_date, end_date and position_date arguments if any of these are set to a NBD.
  • fix_date

An optional argument that sets the day of the week or month on which to extract rates for averaging.

  • Does not apply if frequency is daily.
  • If not set, this will default to the start_date.
  • If frequency is weekly, this may be a number between 1 and 7 (1 for Monday, and so on).
  • if frequency is monthly, quarterly, semi-annual, or annual, this may be a number between 1 and 31 (31 is equal to end of month).
  • method

The type of average (mean) to apply to the extracted rates: either A (Arithmetic) or G (Geometric).
  • holiday_center1, holiday_center2, ...

Optional predefined Holiday Centre identifiers of specific holiday calendars also to be applied, in tandem with the bdc value, to determine the behavior of the function when start_date, end_date, position_date, or one of the averaging (fixing) dates falls on a holiday. When included, each of these values must be within double quotes.

Example 1

FXAverageRate ("XAL",BaseCurrency,"Foreign Exchange End of Day",QueryStartDate,QueryEndDate,QueryPositionDate,QueryAsAtDate,"D","MF",,"A","XAL")

Result 1

Finds the Average FX rate for the XAL/USD currency pair (assuming USD Base Currency) using the Foreign Exchange End of Day rates.

The frequency is daily and the position date is between the start and end dates, which means Spot rates will be taken between QueryStartDate and QueryPositionDate, and Forward rates will be interpolated between QueryPositionDate and QueryEndDate.

Holidays will be excluded due to the MF (Modified Following) setting. This will be an arithmetic mean.

Example 2

FXAverageRate ("XAL","USD","Foreign Exchange End of Day",QueryStartDate,QueryEndDate,QueryPositionDate,QueryAsAtDate,"M","MF",31,"A","XAL")

Result 2

As Example 1 except that the frequency is monthly, cycling off the 31st (end of month). The first rate record used will be that for the final day in the month within which the QueryStartDate falls. The last rate found (interpolated) for averaging will fall on or before the QueryEndDate and at the end of month.

PercentileCalc

Description

Returns the percentile for each item in a group of values.

Format

PercentileCalc (data,groupby,method)

Arguments

  • data - The column containing the values. Values cannot be characters, only numbers.
  • groupby - The group by which you want to aggregate the values (optional): elements that belong to the same group will have the same percentile.
