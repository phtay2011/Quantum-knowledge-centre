Users and Security

In order to manage to manage your organizational structure in Quantum, you need to be able to determine which users and/or user groups will have Resource Security and/or Module Security enabled. Some users or groups can have Resource Security, others Module Security, and some can have both.

Module Security for Users

See Applied Module Security.

Resource Security and User Maintenance

When you save a change to a User record that affects current Resource Security permissions, the status of the change will depend on the filtering applied for resources. This includes additions, deactivations, reactivations, group changes and deletions.

TIP: If you intend to make extensive changes to users and/or groups, it may be easier to disable Resource Security while doing so.

Adding a User

The permissions of new users will be inherited from any groups they belong to. If none, the default permission for the user will be No access.

Modifying Users in Groups

When a user is added to, or removed from, a group, they will lose all permissions that were applied directly. If they are moved to another group, their permissions will be applied using inheritance.

Deleting a User

CAUTION: This is not recommended - for auditing reasons - as it removes the record from the database. Deactivate the user record instead.

Deactivating a User

The permissions that a user had when deactivated will remain constant while they are deactivated (you cannot change them). This provides a historical audit of the User's last permissions.

A deactivated user cannot log on.

(Any active users in a deactivated group will not inherit from that group.)

Reactivating a User or Group

When a user is reactivated, the permissions will be applied using inheritance. Any direct permissions that a user had when deactivated will be replaced by inheritance.

When a group is reactivated, the permissions it had before it was deactivated will re-apply. The members of the group will inherit the permissions of the group, and any direct permissions they had are replaced.

Resetting a User or Group Account

A user's security credentials can be reset by another user with the Control User Access module security permission. The available Administration options in the page menu work as follows:

  • Reset Questions and Password clears the user's security questions and emails* the user a temporary password.
  • Force Password Change prompts the user to create a new password when they log in.

* The email address provided in the Email Address field to send temporary credentials and other information directly to the user.

Single Sign-On Enabled Sites
When Single Sign-On is enabled for a Quantum Site, user authentication is processed by the Identity Service Provider (IdP). Options to reset or change user passwords are not available within Quantum and these actions must be initiated via the IdP. See User and User Group Maintenance Under Single Sign-On (SSO) for more information.

Enabling User's Two-Step Authentication

Two-Step Authentication must be added for each user expected to use the additional security available in the EFT Processing and EFT Reprocessing Utility pages.

The authentication relies upon an Time-based One-Time Password (TOTP) supplied the user's authentication device that matches security built into Quantum. This process verifies that the current user can run the required process.

To enable the Two-Step Authentication for a user, populate the User fields with a current user, and then from the page menu select the Generate Two-Step Authentication Code from the Administration options.

An email is sent to the address listed in the User settings and contains a link that, when clicked, opens Quantum's Two-Step Authentication Code page. The user then scans the QR code with their authentication device to establish a match via a shared key to the Two-Step Authentication algorithm.

NOTE: The QR code is valid for 24 hours after being generated.

Removing User's Two-Step Authentication

To remove the two-step authentication permission for a user, select Clear Two-Step Authentication Code from the page menu.
