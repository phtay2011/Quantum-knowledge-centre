Notes

  1. Standard transactions store the purchase or sale capital price as a decimal in the bond_price field. Prices expressed in fractional terms (for example in Yield to Maturity instruments) are stored in a different field; bond_pric_frac. For this reason:
    • Make sure that you use the correct field when importing transactions which support fractional pricing.
    • When importing a mix of decimal and fractional transactions, separate them into two batches to avoid having blanks in the pricing fields.
  2. Do not use Carriage Return/Line Feed in the Comments field. For multi-line comments, use the pipe character | to represent a new display line. For example, the following import file content:

    comments=Multi-line comment:|Second Line|Third line.

    ...would appear in the Comments text box on the Money Market Deals form as:

    Multi-line comment:
    Second Line
    Third line.

  3. IMPORTANT: The ex_cum_new field must be the last field in each deal record.
  4. To successfully import data into the Relationship field you must set the facility field to specify the Facility and Counterparty names, separated by the pipe character, as follows:

    Facility Name | Counterparty Name

    This is required because the Counterparty Name for the Facility may be different from the Counterparty Name for the deal, which is a separate importation field.

Changing Deal Map Items (QXT)

All items in the deal map are associated with an Event Number. The Event Number uniquely identifies the Deal Map item and is automatically generated for cash flows and events in the life cycle of the deal.

When changing the deal map using a QXT file, the Event Number determines its treatment within the Deal Map; when the Event Number is set to 0 (zero) or not present, a new Deal Map item is created; when the Event Number matches an existing Deal Map item, the item is updated in the Deal Map.

IMPORTANT:

  • Retain the Event Number of an existing deal map item to ensure that Quantum does not treat it as a new Deal Map item.
  • Ensure that the date and number formats match those used for Quantum (for example: DD/MM/YYYY for dates and N,NNN.NN for amounts).
