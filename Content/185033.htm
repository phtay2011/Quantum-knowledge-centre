Enhancements to Pivot Views

Two new check boxes have been added to the Layout tab of the Pivot configuration area for Reporting and Processing pages:

  • Drill down to a table view displays the data behind the aggregation in a table view.
  • Split view by level 0 groups automatically splits the pivot into separate views.

Implementation

Drill Down to Table View

A Pivot view, by default, shows Level 0 data. When there is a next level configured, this will include aggregated data that may be calculated from a single row of the query result or from many.

To display the data behind the aggregation in a table view:

  1. Select the Drill down to table view option.
  2. Select the fields to display in the table view and the sort order of the data.

The pivot will now have an additional drill-down level that will use the defined Table view to show the data making up any aggregated value users click on.

Split Views in Pivots

The data displayed in a pivot can be automatically split into separate views. This organizes the data at a higher level and allows for selection of an individual group directly from the view selection dialog.

Select the Split view by level 0 groups option to split the Pivot view, so that Level 0 will be included in the View selection list and the Pivot will be displayed at the next level.
