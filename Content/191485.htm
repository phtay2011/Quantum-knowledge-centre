Data Entry and Presentation

Data Fields

Page fields have the following attributes:

  • They will reposition and re-size appropriately when you re-size the window, to always present an optimal layout.
  • You can use the keyboard as usual to enter data and to move between fields (TAB and SHIFT-TAB).

Lookup fields

Clicking in a lookup field opens the lookup window. Lookup windows give access to all possible values in a grid and may display more information about them. See Grids for more information.

Fields with lookups can be typed into and matching values will display automatically.

Drop-down list fields

Clicking in a field opens a list of the only available values. Click the value you want.

Fields with drop-down lists can be typed into and the first matching value will display automatically.

Date fields

NOTE: Some Internet browsers allow you to change the language displayed within the browser interface. This will override the system language settings, resulting in dates shown in a different format within Quantum Six pages.
Click here for instructions on changing these settings in Google Chrome.

Date fields accept relative dates like Tomorrow, End of Month, and Start of Year, and date calculations can be done in the field. For example, typing +3m will add three months to the current date or the relative date.

TIPS:

  • Days can be expressed as: d, D, day, Day, days, or Days.
  • Weeks, months, and years can be similarly expressed.

Amount fields

Amount fields accept shortcut keys - T, M, and B indicate thousands, millions, and billions, respectively.

Badges

Badges indicate the status, associations, errors, or available supplementary data for a record or data item. They are displayed in the upper right-hand corner of the page and also within grids.

For example, badges are used to indicate the following:

  • The approval status of new static data (if static data approval is enabled).
  • Linked deals; in such cases, a clickable badge is displayed to the deal.
  • The type of bank flow (that is, Stock or Cash).
  • When a new record is created (or an existing record is changed) and an approval is required to be effective.
  • The status of each flow in the life cycle of the deal on the Currency Options Processing page.

Browse Lists

All data entry pages have a Browse button . to navigate to one piece of data for that page. The button opens a configurable grid or a Select xxx dialog box (a multi-select Browse list)..

Users can search Browse grids and add fields to the layout, and position and sort the results. They can add and save multiple views with separate names. They can apply selection filters to Browse views if desired.

Selection lists display records in a grid from which a record can be selected and displayed in the application page.

To list the data items - click the Browse button . Click it again to hide the list.

To view/edit details of an item - select it in the list, optionally edit the data fields, and click Save.

Multi-Select Browse Lists

Multi-select Browse lists have selection check boxes at the left of each item and your selected items are displayed at the bottom of the Browse window, as shown in the following example.

To select multiple items one by one: click on each item. A check mark indicates selected items. Click again to de-select a row. The check box is cleared.

To select a contiguous group of items: click the first item, then SHIFT-click on the last item.

To de-select a contiguous group of items: click the first selected item, then SHIFT-click on the last selected item.

Configurable Grid

This applies to application pages other than Reporting pages where you can select records from a list (displayed by clicking ). (Reporting pages have more extensive configuration - see Configure Query and Configure View.

To toggle display of the Configure panel - click the Configure link when the selection list is showing.

To save a view for later re-use (where allowed) - on the Properties tab, name the view and click Save.

Use the tabs as follows.

Layout tab

Use this tab to change the columns displayed, their sequence from left to right, and (optionally) the default sort columns.

To reposition a column in the grid - drag-and-drop that column name in the Columns box.

To add an additional column - select it from the list (start typing to find columns quickly).

To remove a column from the grid - in the Columns box, click the x button for the column.

To apply one or more default sort columns - in the Sort box, select the columns, in order of precedence. Drag a column to change this order. Click a column in the box to reverse its sort order ( or for ascending or descending, respectively). To cancel sorting on a column, click the x button for that column.

TIP: The default sorting can be overridden by clicking a column heading in the list.

Filters tab

Use this tab to configure filters on the data that comes back from the query. This enables you to further extract data from the query without having to re-run it.

NOTE: You must save the view before the grid will update to reflect any changes.

To add a filter field to the view

  1. Click the Add Filter button and select the field you want from the list of those available (start typing to find your filter quickly).
  2. In the field label's drop-down list, select an applicable logical operator to apply filter values, depending on the type of field.
  3. In the field itself, select one or more filter values to apply.

To remove a filter field from the view - In the drop-down menu on the filter field's caption, select Remove Filter.

Columns tab

Use the Columns tab to choose the headings for the columns in the results grid. Add extra columns called "calculated columns" that show the calculated values that are the results of an expression associated with the column. This tab uses Expression Builder to create the expressions.

Colors tab

Use the Colors tab to add colors and conditions to columns and rows in the results grid.

Properties tab

Use the Properties tab to:

  • Enter a new Name for the view (required before you can save it).
  • Optionally set this view as the default for the current query. This will be the first view you'll see when you select the current query.
  • Set the Access Permission for the view (Public or Private).
  • Change the way any amounts appear (columns), the type of rounding, whether the amounts are in thousands, millions etc., and what decimal format to use.
  • Embed the units in the results table.

To save a view for later re-use - Make your required changes (on all tabs) and click Save.

To alter how amounts appear - in the Rounding area, select Columns for this to apply to, enter a rounding method (5/4, Down, Up), a unit (called a Round Amount and in Whole Amounts, Thousands, Millions, Billions or with the unit Embedded) and how many decimal places.

Once saved, Views for queries like this one - that can be accessed in either Reporting or Processing mode - are available for the query in both modes.

The export to Excel uses the raw numbers, but other exports use what is in the View.

Grids

Grids display multiple items, records, and report results of various types in tabs, selection lists, and lookup tables. Grids often have a Configure link at the top.

Grid Menu

The grid menu, when available, is located to the right and just above the grid. Grid menus vary depending on the context - that is, the type of data displayed. They usually include grid view functions where applicable (Configure, Save, Copy, and so on), and may contain:

  • specific functions to be applied to one or more selected grid items (or the grid as a whole if no items are selected),
  • drill-down functions to view related data,
  • grayed-out functions if not valid for the selected grid items, and
  • functions related to copying or exporting selected item data.

Quick Menu

Quick menus apply only to some Processing page grids (for example, Settlements, Confirmations, EFTs, and so on). They:

  • are not available in selection or lookup lists,
  • appear when you click on a linked value in a grid row,
  • can differ depending on the type of item in that row,
  • list only functions valid for that row, and
  • when used, apply the selected function only to that row.

Sorting

You can set none, one, or more default sort columns when configuring the grid view, and you can override these on the displayed grid. When you first display a grid, items are listed in the default sort column sequence, if any (if not, rows are displayed in the order the data is returned by the application).

The current sort columns (if any) have column headings with either an up or down chevron or (for ascending or descending sequence, respectively).

Filters

You can filter out current grid items not matching a text string, by typing a value In the Search box. The listed items will be filtered to show only items whose column values contain the characters you have entered.

You can use a partial number (~ ) search as well. This searches through numeric fields, such as the Amount column, but does not search through text fields, such as Bank Account.

For query result grids, you can also filter the result View OR filter the query itself (by changing the Query Filters and Options) and re-running it to reduce the total size of the result set.

Actions

After each processing action you apply to a row in a grid:

  • a vertical bar appears to the left of the row, colored gray for successful processing, red for an error,
  • clicking the bar displays a status message for the processed row, and
  • the background color of the message indicates its severity (error, information, warning).

You can also configure the grid view to display two additional columns that you can also use for filtering:

  • Response Message – the message associated with the bar/row, and
  • Response Severity – the severity of the message.

Navigating to Grid Pages

NOTE: Items are paged according to the current sort column sequence (if any).

Use the following features to quickly locate specific items in multi-page grids.

  • Click or to go to the next or previous page , respectively.
  • Click and then or to go to the last or first page, respectively.
  • Click and select the page to display that page. If the grid is currently sorted, the drop-down list is in that sequence, with distinct values of that column listed (if unsorted, only page numbers are shown).

Exporting Grid Data

If the Export option is available in the associated menu, you may be able to can export grid data to Microsoft Excel, CSV, XML, PDF, JSON, and QXT formats. You can also click the Quick Export to Excel function above the grid to export the grid data as a Microsoft Excel file.

The data exported is restricted to the records in the current data set of the grid (that is, filters and search criteria will also affect the data exported).

Pop-Out Links

The individual deals, instruments, and static data that make up a record are clickable links that take you to the Quantum form for those individual data items, from where you can edit the data item or create a new one. Forms are displayed in separate pop-up windows, without interrupting your current workflow.

  • Within a page, click a linked field caption to open the form for the data item.
  • Within grids, click a linked record to open the form for the data item.
