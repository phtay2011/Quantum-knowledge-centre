Filtering Hedge Accounting Queries

Use the Filters area to change the current values for selecting the Hedge Relationships (HRs) you want to report on, as follows.

TIP: You can change these as required, and optionally save the changed settings, either updating the current query settings, or as a new query. Or you can create a new query, either from scratch or by copying an existing query.

  1. Select the type of filtering -one of the following:
    • By deal number - to report on specific HR deal numbers over a specified period, or
    • By transaction details - to report on HR deals matching entered parameters over a specified period.

    Select a date from the calendar or clear the field for the drop down menu to appear, then choose from Today, Next Business Day, End of Month, and so on and optionally type + or - and a number for an offset. For example, Today + 3bd is today's date plus 3 business days; Yesterday - 5d is yesterday's date minus 5 calendar days.

    TIP: The end date is treated as an As At Date in some of the result view types.

  2. If you selected By deal number in Step 1, enter one or more HR deal numbers in the Deal Number field, separated by commas. Go to Step 9.

    TIP: To find deal numbers, use the browse tool on the Hedge Relationship page.

    Otherwise (you selected By transaction details in Step 1), continue at the next step.

  3. Under Hedge Type, select either Cash Flow, Fair Value, or both.
  4. Under Nature of Risk, select Exchange Rate Risk and/or Interest Rate Risk and/or Overall Market Risk.
  5. Use the Entities drop-down list to select one or more entities, or select ALL*.
  6. Use the Ticket Number drop-down list to select one or more ticket numbers, or select ALL*.
  7. Use the Instruments drop-down list to select one or more underlying Hedge Strategies, or select ALL*.
  8. Optionally save your query settings. Click the Configure link, on the Properties tab enter a Name for the query, select Public or Private, and click Save.

You have entered query Filters values. To run the query now, either click Run Query (to run without stopping on an error),or in that button's drop-down list, select Stop on error.

*TIP: Leaving any of the multi-select fields blank is equivalent to selecting ALL.
