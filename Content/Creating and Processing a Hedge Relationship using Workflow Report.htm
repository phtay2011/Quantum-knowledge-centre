Creating and Processing a Hedge Relationship using Workflow Report

To create a Hedge Relationship:

  1. Open Configuration > Workflow > Report > the Actions tab.
  2. In the menu select New and the New Action dialog box appears.
  3. In Action Name click the lookup. Select Create > Create Hedge Relationship.
  4. Add appropriate inputs.

You need to map at least Hedge Strategy, Deal Date, Maturity Date, Entity, and deals that you want to add to that Hedge Relationship. All inputs that you don’t map are directly pulled from Hedge Strategy.

    NOTE:

  • If you add an invalid input, an error message will be displayed. If you map fields that are not associated with the parameter that you chose, those fields will be ignored, and a new valid Hedge Relationship will be created. If you add an invalid input expression while mapping Revaluation Method, Leg 1/Leg 2 and Clean/Dirty fields, that expression will be ignored, and default values for these fields will be saved.

  • Dollar Offset Lower and Dollar Offset Upper fields on the Hedge Relationship page (the Effectiveness tab) are equivalent to Lower Limit and Upper Limit on the Workflow Report page.

Add Linked Deals action

Use the Add Linked Deals action to link a deal to an existing HR.

To add linked deals to an existing HR:

  1. Open Configuration > Workflow > Report > the Actions tab.

  2. In the menu select New and the New Action dialog box appears.

  3. In Action Name click the lookup. Select Process > Hedge Relationship > Add Linked Deals.

  4. Add appropriate inputs.

You need to map at least the Hedge Relationship Deal Number and Deal Number of the deal that you want to link.

 

You can map Linked Deals by:

  • Entering the Deal Number of the deal that you want to link to an HR.

  • Choosing a deal that you want to link from the report that you created (input Deal Number > Add Expression > the Columns folder)

  • Combining actions. You can create a deal which you want to add to an HR. You need to set its Output = Deal Number. Then in the Create Hedge Relationship action, you create a Hedge Relationship and set its Input to be the Output of the deal that you created.

NOTE:

  • If you want to link a Projected Cashflow deal, you need to map the Transaction Type (PC) and Deal Number of that deal.
  • 1. If you add a Commodity Swap deal and map its Deal Number only, all periods of that deal will be added, and you won’t be able to update them.

    • 2. If you want to add specific periods of that Commodity Swap deal, you need to map the Period Numbers of those periods. Each period acts like a separate linked deal.

The Hedge Accounting Results source is added to Source > the Reporting folder.

Dedesignate Full

To perform the Dedesignate Full action:

  1. Open Configuration > Workflow > Report > the Actions tab.

  2. In the menu select New and the New Action dialog box appears.

  3. In Action Name click the lookup. Select Process > Hedge Relationship Dedesignation > Dedesignate Full.

  4. You need to map the Hedge Relationship Deal Number and Dedesignation Date.

Dedesignate Partial

To perform the Dedesignate Partial action:

  1. Open Configuration > Workflow > Report > the Actions tab.

  2. In the menu select New and the New Action dialog box appears.

  3. In Action Name click the lookup. Select Process > Hedge Relationship Dedesignation > Dedesignate Partial.

  4. You need to map the following:

    1. Hedge Relationship Deal Number,

    2. Linked Deal Number,

    3. Dedesignation Date

    4. Dedesignation Amount, New Allocation or New %.

     

For Currency Option - Intrinsic, Foreign Exchange – Spot, Cross Currency Swap – Basis, and Commodity Price - Defer/Immediate and Release Account are optional fields, but you need to set it from the originally listed Inputs grid.

For Interest Rate Risk, Exchange Rate Risk and Overall Market Risk all inputs need to be chosen from Add > Linked Deal.

NOTE:

  • If you want to perform the Dedesignate Partial action on a Projected Cashflow deal, you need to map the Transaction Type (PC) as well.
  • If you want to perform the Dedesignate Partial action of a specific period of a Commodity Swap deal, you need to map the Period Number of that period. Each period acts like a separate linked deal.

Update Hedge Relationship action

To update a Hedge Relationship:

  1. Open Configuration > Workflow > Report > the Actions tab.
  2. In the menu select New and the New Action dialog box appears.
  3. In Action Name click the lookup. Select Update > Update Hedge Relationship.
  4. You need to map the HR Deal Number to be able to perform this action.

You can update everything that you can update on the HR screen. If you don’t want to update linked deals, you need to set 0 in the Number of LinkedDeals field. If you want to update Linked Deal you need to map it’s deal number and OriginalLinkStart (equivalent to Start Date of that deal on the HR screen).

NOTE:

  • If you want to update a Projected Cashflow deal, you need to map the Transaction Type (PC) and Deal Number of that deal.
  • If you want to update a specific period of a Commodity Swap deal, you need to map the Period Number of that period. Each period acts like a separate linked deal.
