/GL

/GL "gl_owner_name"
        /DATE yyyy-mm-dd [RESTORE]
        [/TYP "transaction_type" ["instrument_name"] ]

Use with the ARCHIVE command to archive or restore General Ledger (G/L) entries, where:

  • Only G/L entries for qualifying deals will be considered. These are matured deals for:
    • the explicitly specified transaction type - if /TYP is included and Instrument name if one is also specified, or
    • all transaction types and Instruments - if /TYP is not included except for Unit Deals (which have no Maturity Date) and Futures or Securities deals involved in closeouts.
  • gl_owner_name (within enclosing double quotes) is the mandatory first argument that specifies the G/L Owner name for the archive or restore session. Only one G/L Owner may be specified.

    TIP: If you wish to archive GL entries from multiple GL Owners you will need to run the archive tool multiple times altering the GL Owner name for each processing session.

  • /DATEyyyy-mm-dd is mandatory and specifies the date of oldest G/L entries you wish to retain in the database.
    • This date must NOT be later than the last close date for the ledger.
    • On archive, G/L entries for qualifying matured deals with a Maturity Date before the specified date will be removed from the database and archived.
    • On restore, G/L entries for qualifying matured deals with a Maturity Date on or after the specified date will be restored to the database.

      Enter "THE BEGINNING" instead of a date (with RESTORE only) to restore from the beginning point.

  • RESTORE specifies that you want to restore previously archived G/L entries. The default (when RESTORE is not included) is to archive.
  • /TYP "transaction_type" is optional and specifies a particular transaction type (and optionally, and Instrument name) for matured deals whose G/L entries are to be archived or restored.

    The default (when /TYP is not included) is all transaction types with qualifying G/L entries.

  • transaction_type (mandatory with /TYP only, and within enclosing double quotes) is one of the following transaction type codes:
    • AA - Bank Account Adjustment
    • AC - Actual Cashflows
    • AI - Bank Account Interest
    • AT - Bank Account Transfers
    • CO - Currency Options
    • FR - FRA Deals
    • FU - Futures Deals (except those involved in Closeouts)
    • FX - FX Deals
    • GL - G/L Journals
    • HR - Hedge Relationship
    • IM - Initial Margin
    • IO - Interest Rate Options
    • RA - Repurchase Agreement
    • SC - Currency Swaps
    • SE - Money Market Deals
    • SI - Interest Rate Swaps
    • ST - Stock Account Transfers
  • instrument_name (optional with /TYP only, and within enclosing double quotes) specifies a particular Instrument name for matured deals whose G/L entries are to be archived or restored. If that Instrument is not one for the specified transaction type, nothing will be archived or restored.

    The default (when an instrument_name is not included) is all instruments of the specified transaction type.

See also: /GL Examples.
