Additional AFB120 Configuration files

In addition to the .ini configuration file, the following files can be use to define parameters that modify the way in which Bank Importer interprets AFB120 files as they are being processed:

  • AFB120.config, AFB120,xsd, AFB120.xsl - to define the rules for locating and processing information in AFB120 statements.
  • ZBAmap.txt - to map ZBA numbers.
  • qtmap.txt - to optionally specify bank account numbers.

These files must be placed in the same folder as the Bank Import Configuration file BkImport.ini.

Defining Rules (AFB120.config)

The AFB120 parser can be configured to locate and process information in the Bank Statement file according to certain rules.

The rules are defined in the <Rules> node within the AFB120.config file.

CAUTION: The AFB120.config file must be placed in the same folder as the .ini configuration file for AFB120.

A rule is composed of two nodes: Searches and Results, as described below.

Searches node

The <Searches> node can contain as many <Search> nodes as required to validate a rule.

A <search> is composed of the following nodes:

  • <Field> - which must contain one of the following values (follow the Ref link to see the field position in the Bank Statement record):
    • BankCode (Ref 1.2),
    • OperationCode (Ref 4.9),
    • AccountNumber (Ref 4.8 and optionally mapped according to the value defined in qtmap.txt file),
    • Label (Ref 4.13), or
    • ExtraLabel (Ref 5.13).
  • <Type> - which must contain one of the following operators:
    • EQUALTO - to match the value/s to all of the field,
    • CONTAINS - to match the value/s to part of the field, or
    • STARTSWITH - to match the value/s to the start of the field.
  • <Value> - which must contain a value to match to, in the format <![CDATA[value]]>.

    You can define as many <Value> nodes as required. Any of the defined values will validate the rule.

Example

<Searches>
<Search>
<Field>BankCode</Field>
<Type>Contains</Type>
<Value><![CDATA[Aust]]></Value>
</Search>
<Search>
<Field>OperationCode</Field>
<Type>EQUALTO</Type>
<Value><![CDATA[94]]></Value>
<Value><![CDATA[95]]></Value>
</Search>
</Searches>

Result node

A <Result> node is composed of either a <Constant> or <Find> node, and an optional <Copy> node. The value returned will be copied to the Generic 10 Bank Flow field.

Only one <Result> node may be defined in each Rule.

Examples

<Result>
<Constant><![CDATA[SWEEP]]></Constant>
</Result>

<Result>
<Find>
<Field>Label</Field>
<Position>13</Position>
<Length>16</Length>
</Find>
<Copy>
<Field>Extra label</Field>
<Position>3</Position>
<Length>25</Length>
</Copy>
</Result>

Rules Examples

The following AFB120 configuration defines two rules:

Rule 1: Calyon - Virement

If...

  • the Bank Code is 31489, and
  • the Operation Code is 94 or 95, and
  • the Label starts with "VIREMENT",

then...

  • the result will be extracted from the Label, from position 13 and length 16, and
  • in the Bank Flow Record's Generic 10 field, a substring of the extra label will be copied (from position 3 and length 25).

Rule 2: HSBC Sweep

If...

  • the Bank Code is 30056, and
  • the operation code is 94, and
  • the complementary information (Extra Label) starts with "SWEEP DE"

- or -

  • the Operation Code is 95, and
  • the complementary information (Extra Label) starts with "SWEEP A"...

then the result will be the constant "SWEEP".

Mapping ZBA Numbers (ZBAmap.txt)

Once the ZBA is identified, the AFB120 parser will look for a ZBA mapping file named ZBAmap.txt.

CAUTION: The ZBAmap.txt file must be placed in the same folder as the .ini configuration file for AFB120.

If the file is found, the parser will send the mapped ZBA Number given in the file (if not, the original ZBA Number is sent).
