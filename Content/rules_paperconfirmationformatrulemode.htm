Paper Confirmation Format Rule page

Use this page to define selection criteria that determine which Confirmation Template files are to be used to format "Paper" (print/email/fax formatted) Confirmations for applicable deal and/or cash flow actions and specific parameter combinations (Transaction Type, Entity, Instrument, and so on).

To add a new Format record - click New; and:

  1. Enter the Matching Criteria*: Transaction Type*, Entity, Relationship, Counterparty (one only or ALL in each case), and Instruments*, (one, several, or ALL).

    *NOTE: If Cash Flow Netting is being used and Settlement Confirmation requirements differ - for example, across Counterparties - you must include Rule entries with Transaction Type and Instruments both set to ALL for each case (because Netted groups can potentially contain mixtures of these values).

  2. (Optional) To override the default subject text that the system places in e-mail confirmations, under Email Subject select User Defined and in the associated text box enter text and/or the predefined [Transaction Type], [Deal Number], and/or [Counterparty] placeholders as required.
  3. (Optional) To specify, for paper Confirmations being saved to file, the structure and content of the file names, under File Name Mask select User Defined and in the associated text box enter text and/or one or more of the predefined placeholders for [Transaction Type], [Deal No], [Date], and [Time].

    CAUTION: The placeholders must be entered exactly as above. Any other values will be taken as literal text. Also, if you remove [Deal No], [Date], and/or [Time], you could get duplicated file names, and earlier files would then be overwritten.

  4. In the Settings - Default Paper Formats area, select a suitable Crystal report* for each event applicable to the selected Transaction Type.

    *NOTE: Only Crystal reports mapped to the CONFFMTP module may be selected. Standard reports are provided for most deal types and actions, named accordingly (CO Deal, FX RateFix, and so on), and already mapped. You can modify these or create new ones (which must be mapped to the CONFFMTP module before you can select them here).

  5. Click Save.

To list records - Click to open the browse window. (Click the Configure link to change browse list properties.)

To change a record's details - Select it in the browse list, optionally change Selection and/or Paper Format values as required, then click Save.

To delete a record in full (not recommended; Deactivate instead) - Select it in the browse list; in the page menu, select Delete; then confirm.

To deactivate a record - Click , select the record, and in the page menu select DeActivate. You can later reactivate it in the same way.

For details of data items, click on an item when the Help panel is open.
