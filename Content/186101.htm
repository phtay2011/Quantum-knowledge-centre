Maintaining FX Requests

To enter and maintain FX requests, open the FX Request page and follow the procedures below, as required.

Creating, making financial changes to, or deleting a Request can be subject to Approvals. See Requests and Approvals.

To list and find current FX Requests

(You need Inquire rights to the FX Request module.)

  1. Click and in the Browse list:
    • Use the < and > buttons to browse through longer lists.
    • Enter Search text to list only matching items.
    • Click the Configure link on a tab to change the browse list properties, including filtering**.
  2. Select the Request you want.

**TIP: In particular, you can add in the Fill Link and Fill Date columns (both for Requests and actual deals) to indicate filled Requests, and you can filter the browse list to show only filled or unfilled Requests.

To create a new FX Request (from scratch)

(You need Add rights to the FX Request module.)

TIP: Typing a single character in a lookup field sets it to the first list value starting with that character (if any) and the lookup, when opened, will be positioned at that value.

  1. Click New.
  2. Select an FX Instrument.

    TIP: Only FX Instruments with the following settings will be available:

    • Type is OUTRIGHT or SPOT & FORWARD, and
    • Allow Requests is selected.
  3. On the Request Details tab, enter at least the following mandatory field values:
    • Buy Ccy and Sell Ccy,
    • Buy amountorSell amount,
    • Deal Date and Value Date, and
    • (for NDFs) Settlement Ccy, and Fix Date.

    (Other fields are optional, including Entity, Counterparty, Dealer, Spot Rate, Contract Rate, and Forward Points.)

  4. On all other tabs, optionally enter any other values specifically required for the request.
  5. Click Save.

    When applicable, the FX Request will be replicated into Analytics by the Replication process as an FX Deal with a Deal Type of Request). The closest-matching Analytics FX Deal Request Rule record will be used to supply any missing field values that are required in Analytics Pricing (but optional in Quantum Requests). The applicable fields are Counterparty, Entity, Dealer, Spot Rate, Contract Rate, and Forward Points.

To create a new request by Copying an existing one

(You need Add rights to the FX Request module.)

  1. Click and in the Browse list, select the request you want to copy.
  2. In the page menu, select Copy.
  3. Edit the data fields on each tab as required.
  4. Click Save.

    When applicable, the FX Request will be replicated into Analytics by the Replication process as an FX Deal , as in the procedure above.

Any details relating to a Fill of the original request are NOT copied.

To change a request's details (including Notes)

(You need Change rights to the FX Request module.)

  1. Click and in the Browse list, select the request you want to edit.
  2. Edit the data fields on each tab as required.
  3. (Optional) To add a note:
    1. in the Page menu under NOTE, select New.
    2. In the dialog box, enter your Note text.
    3. Click OK.
  4. (Optional) To attach a file as a note:
    1. in the Page menu under NOTE, select Attach File.
    2. In the file browse window, navigate to and select the file.
    3. Click OK.
  5. Click Save.

    Your changes are saved and will flow through the Replication process (when applicable) to update the FX Request (an FX Deal with type Request) in Analytics.

    A Notes badge will show on the request if you entered a Note or attached a file.

To delete a request

(You need Delete rights to the FX Request module.)

  1. Click and in the Browse list, select the request you want to delete.
  2. In the dialog box, optionally enter Reason text.

    NOTE: If a Filled request, you are warned that deleting the request will disconnect it from the deal.

  3. Click Delete to confirm.

    Results:

    • a Deleted badge will show on the request,
    • a Notes badge will also show if you entered Reason text,
    • if a Filled request - any link is removed, and
    • the deletion will flow through the Replication process (when applicable) to delete the FX Request (an FX Deal with type Request) in Analytics.
