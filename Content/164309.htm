Automatic Allocation in hedge relationships

Automatic Allocation attempts to achieve a specified Face Value Hedge Ratio. This can occur:

  • when deals are linked to or removed from a hedge relationship,
  • when changes occur to a deal, which is part of a hedge relationship, or...
  • an 'as required' basis.

Automatic Allocation will occur in the above circumstances only when the linked deals:

  • have matching currencies

    The hedge ratio is based on the Face Values of the deal or deal leg that is being linked to the hedge relationship. See Leg 1/Leg 2 defaulting. If the deal being linked does not have a currency match on the deal or deal leg being linked, then the Auto Allocation rule cannot be applied.

  • are unique to a hedge relationship

    Automatic Allocation does not occur for any hedge relationship where a linked deal forms part of another hedge relationship.

Automatic Allocation is also unavailable when:

  • a hedge relationship requires Spot to Spot Adjustments, or...
  • Hypothetical derivative deals are part of the hedge relationship.

Using Automatic Allocation

At system level

Automatic Allocation must be enabled at system level to allow Quantum to adjust the allocation percentage of deals to achieve the target ratio.

Automatic Allocation is not used (and not available) when the Designated Risk in the Hedge Relationship page is Commodity Price Risk.

To enable automatic allocation

  1. On the Operational Settings page, select the Enable auto allocation check box.
  2. Enter the percentage of the face value ratio that Auto Allocation is to achieve in the Target Face Value Ratio field.

For each hedge relationship

Enable Automatic Allocation for deal changes that occur outside the relationship

This function re-allocates the percentage of linked deals on an ongoing basis whenever changes occur to individual deals outside the hedge relationship.

For example, when a deal is unwound, Quantum automatically re-allocates the percentages of all remaining deals within the hedge relationship, to maintain the system face value ratio.

NOTE Although this function is optional, it is recommended for all hedge relationships. You cannot manually initiate Automatic Allocation in these circumstances on an as-required basis, as the system does not recognize any change made to the deals outside the hedge relationship.

To enable automatic allocation

  1. From the Deal Details tab of the Hedge Relationship page, select the Enable auto allocation when changing linked deals check box.
  2. Click Save to enable the function.

Initiate Automatic Allocation each time you add a new deal to a hedge relationship

Prior to saving a hedge relationship, you must run Automatic Allocation if you add a new deal. This ensures that future Automatic Allocation calculations use the correct percentiles.

NOTE: You cannot run Automatic Allocation after saving a hedge relationship unless changes such as the addition of further linked deals occur.

To enable automatic allocation

Before you save the deal:

  1. From the Linked Deals tab of the Hedge Relationship page, select Auto-allocate from the grid menu.
  2. Click Save to apply the change.

TIP: If you forget to run Automatic Allocation prior to saving the relationship, determine the applicable percentages for each deal and manually change the allocation percentage amounts as required.

Initiate Automatic Allocation when deal changes occur within the hedge relationship

This manually initiated option re-allocates the percentage of linked deals as a result of deal changes that occur within the hedge relationship, such as changing the date from which a linked deal takes effect or removing a deal from the relationship.

CAUTION: This option does not recognize any deal changes that occur outside the hedge relationship, such as a change in value. Selecting the Allow Automatic Allocation When Changing Linked Deals check box stops this occurring (see Stage 2).

To enable automatic allocation

  1. From the Linked Deals tab of the Hedge Relationship page, select Auto-allocate from the grid menu.
  2. Click Save to apply the change.

NOTE This is unnecessary when deals are removed at deal level, using the applicable deal page.
