Hedge Only the Spot Value of FX Forwards

(First introduced in Release 6.5.1 SP, September 2017.)

In accordance with the IFRS 9 standard for accounting of Forward Contracts in a hedge relationship, Quantum recognizes the change in the forward element of the Forward Contract when only its spot risk is designated in a hedge relationship as follows.

  • For transaction related hedged items, the change in fair value is accumulated in a separate component of Equity and reclassified to profit and loss.
  • For time period related hedged items, the change in fair value is accumulated in a separate component of Equity and amortized on a rational basis over the term of the hedge relationship.

Background

Under the IAS 39 accounting standard when only the Spot Value is designated in a hedge relationship, the changes to the Spot Value are accounted in accordance with the type of hedge (Fair Value or Cash Flow), whereas the changes to the Forward element are accounted in profit or loss, resulting in potentially significant profit or loss volatility.

Under the IFRS 9 accounting standard, an entity can continue to use the IAS 39 approach when designating the full fair value of the forward contract; however, when only the Spot Value is designated, the change in the fair value of the forward element can be optionally recognized in Other Comprehensive Income (OCI) and accumulated in a separate component of equity. The forward element that exists at inception is amortized from the separate component of equity and classified to profit and loss on a straight line basis.

Implementation

Hedge Strategy

The Designated Risk field includes a new Foreign Exchange - Spot option to hedge only the spot risk in a forward contract (under IFRS 9). When selected, the forward value is accounted in a separate component of equity.

NOTE: When selected, the following fields are hidden:

  • Hedge Dedesignation (value set as DEFERRED),
  • Enable Recognition Reclassifications (value set as False), and
  • For Fair Value Hedges only, Posting Based On (value set as Hedge).

Hedge Relationships

The Designated Risk field includes a new Foreign Exchange - Spot option to hedge only the spot risk in a forward contract (under IFRS 9). When selected, the forward value is accounted in a separate component of equity.

NOTE: When selected, the following fields are hidden:

  • Hedge Dedesignation (value set as DEFERRED),
  • Enable Recognition Reclassifications (value set as False),
  • For Fair Value Hedges only, Posting Based On (value set as Hedge), and
  • Perform Spot to Spot Adjustment.

When Foreign Exchange - Spot is selected as the Designated Risk, you can select Amortize hedging costs (forward points) to amortize the forward points to Profit & Loss for time period related hedges.

Implementation Notes

Hypothetical Derivative

It is only possible to account for the forward element component of the forward contract to the extent it relates to the hedged item. This is called the aligned forward element. To measure the aligned time value Quantum requires a hypothetical deal to be added to the hedge relationship. The Hypothetical deal should represent an FX Forward that has critical terms that perfectly match the underlying hedged item.

Inception Valuations

If the date the FX Forward is designated into the hedge relationship coincides with the FX Forward's Deal Date, then an Inception valuation is calculated using the Dealt Terms of the FX Forward or Hypothetical Derivative as follows:

  • The Market Value of the FX Forward is set equal to 0.
  • The Intrinsic Value of the FX Forward is set equal to the discounted difference between the Forward Rate and Spot Rate.

Exclusions and Restrictions

Deal Linking

Once a Foreign Exchange - Spot Hedge Relationship has been designated and postings have been started, it is currently not possible to add additional deals into the Hedge Relationship.
