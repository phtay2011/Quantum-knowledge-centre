Creating a New Template from a Workflow Template

Set up the following three pages to create a new Workflow Template page.

The order is important: first set up the template, then the location for the data to be sent to (the Data Transfer Location), then the report that creates the output and sends it to the location.

Step 1: Set up a template

NOTE: For more detail on a field, click the ? button to open the help panel, and click in the field.

  1. Open Configuration > Workflow > Template.
  2. Enter a Name for the new template.
  3. Select an Output Format of Text, XML, or HTML.

    4. TIP:

    • When Output Format is HTML:you will have the option to convert the resulting HTML to PDF in output files when the template is applied, and

    • the generated HTML is prefixed with a <link> tag to a template.css file in the QuantumWeb/Resources folder, enabling you to apply your organization's styles to the output (if the file exists). The tag is:

      <link href="Resources/template.css" rel="stylesheet" />

  4. In the Inputs area, select New and add the inputs you want to use in this template. Inputs are case insensitive. For Type = Yes/No, tick Default Value for Yes (true).
  5. In the Expressions area, select New, add the Name (unique) and data Type, and define the conditions under which your template will apply. When adding an expression, in the Expression Builder choose from Functions, Operators, or Inputs.
  6. In the Validations area, optionally specify one or more grid entries, each with a Condition, Message, and Type. The Type can be Block or Warn, where:
    • a Block validation prevents execution of the template, an error is returned by the Workflow Action, and a Data Source using the template returns no result; while
    • a Warn validation will not prevent execution of the template and will log the warn message in the Message Log if the Condition is met.
  7. In the Sort field, select one or more Inputs and Expressions grid items, to order the data coming from source if required. Your Template output will be generated accordingly.
  8. In the Sources area, enter the columns and rows in the database that will be used as the source if.. then data for the template.
  9. Select Validate to check the expression before saving.
  10. In the text box, enter the code for the template (optional).
  11. Select Save. The system checks for invalid expressions and inputs. At the most basic level, an invalid input is one that is not defined in the template.

Step 2: Set up a Data Transfer Location page linking to the template

  1. Open Configuration > Data Transfer Location.
  2. Give the location a unique Name.
  3. Select a Location Type of either FILE or QUEUE and select a Purpose of WorkflowOut.
  4. If Location Type is FILE, enter the Location of the folder for the output. The location is relative to the designated folder location for the processing type selected in Purpose. Go to Step 6.

    For example, if Purpose = WorkflowOut, entering a value of Default in Location will place files in the Quantum site's AppData folder, under ProgramData\Sites\WorkflowOut\Processed\Default.

  5. If Location Type is QUEUE, enter the Connection configured for the message Queue output.
  6. In Template, select an existing template.
  7. Complete Digital Signature and Encryption areas if required.
  8. Select Save.

Once you've set up the Data Transfer Location page referring to the template, the name of the location will appear in the Transfer folder in the Select Action Name dialog box in Configuration > Workflow > Report > Actions area. If Location Type = QUEUE, saving the DTL creates an action with the same name as the DTL in the Transfer folder. The action has settings for each input, but does not include a filename. When the action happens, the template output is sent on to the queue.

Step 3: Set up a Report page with an action that creates the output

NOTE: Link the template to the Data Transfer Location first (see above).

  1. Open Configuration > Workflow > Report.
  2. Set up the source of data used by the report.
  3. In the Actions area, select New to open the New Action dialog box.
  4. Click the Action Name lookup and select Transfer, then the name of the Data Transfer Location linking to the template (see above).

    The Inputs grid appears with a default input, File Name.

  5. Set Display In to either Quick Menu or Toolbar.
  6. If required, click File Name and add an expression. For example, [Entity] = "-" Today() . This expression gives the file the name of the entity plus the date. The default extension is .txt. The system does create blank files if there is no content.
  7. Click Add and in the Select Inputs dialog box, tick the names of the inputs you want the template to use in this action, then click OK. The inputs are from the template linked to your selected Data Transfer Location.
  8. Click each input to open the Edit Settings dialog box and add a mandatory expression that relates to the input.
  9. Validate the expression, then click OK. Repeat as needed.
  10. Complete the report and then click Save.
  11. Find the report and click Run Query.
  12. Select the row or rows you want.
  13. In the View, click the action (toolbar button or menu item). The action takes place, sending the resulting file to the Data Transfer Location.
  14. Open the location specified in the Data Transfer Location and look for the file name, for example, [Entity] = "-" Today() might create a file called "ABC Corporation - 5 Oct 2019".

NOTE: In v2020.1, we renamed the "Workflow" folder to "WorkflowOut". Please check any automated processes prior to v2020.1 that used the "Workflow" folder and update the name. The folder location is: C:\ProgramData\SunGard\AvantGard Quantum\Site\Site_name\WorkflowOut.
