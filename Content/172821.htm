Specify Assumptions (Spectrum Analysis Query)

NOTE: The default settings on the Assumptions tab depend on whether or not an existing Assumptions Definition record is specified on the Properties tab, and:

  • if so - the defaults are taken from that Definition record, but...
  • if not - a new Definition record is created with the same Name as the Query (as on the Properties tab) and system defaults apply.

In either case, any changes made on this tab are saved in the specified Definition record when the query configuration is saved.

On the Assumptions tab, edit the following settings as and if required.

  • Valuation area
    • Rule - the required Valuation Rule record.
    • Revaluation Timing - Start of Day, Intra Day, or End of Day
    • Duration Weight - Gross PV or Net PV to exclude or include the first settlement cash flow from the PV calculation of transaction duration.
    • Risk Allocation - the date for "bucketing" transaction cash flows when a Time Profile is used: Maturity Date or Repricing Date.
  • Risk area
    • Shift Method - Par, Zeros, IR Return, or Derivative.
    • Shift Direction - Up, Down, or Average.
    • Shift Frequency (applies if Shift Method is IR Return or Derivative) - Quarterly, Semi Annual, Annual, or Continuous.
    • Delta Shift Points - the number of shift points giving the sensitivity of the option price to a change in price of the underlying asset.
    • Gamma Shift Points - the number of shift points giving the sensitivity of the option price to the square of the change in price of the underlying asset.
    • Vega Shift Points - the number of shift points giving the sensitivity of the option price to a change in implied volatility.
  • Risk Method area
    • Spectrum - Individual or Cumulative.
    • Generic - Preserve PV and Delta or Preserve Delta and Gamma.
  • Options area (link)
    • Complete Time Profile - Select this option to include, in query results, all Time Profile buckets, whether or not they have any transactions assigned to them.
