Schedule a Query Run

Schedule a report to run a query at a certain time interval so that the results of the query are ready for multiple users to view on demand. With Run for Latestenabled, the query does not run at the time of retrieval, the system just retrieves the latest run of the query and displays those results. This makes the process quick and independent of the time the system takes to run the query.

NOTE: The queries must be public queries for Scheduler to run them, and the queries produce public reports viewable by all users.

Set up Scheduler to "Run for Latest"

  1. In Scheduler, set up the job to run the report. This job will run continually, according to the schedule. For example:

  2. Mark the run in Scheduler as Run for Latest.

    TIP: To reduce the load on the system, enable Suspend execution when result not viewed, which suspends the scheduled task if noone views the results.
    For example, you can set up a query that runs every 5 minutes to pause for 60 minutes between runs if users do not view the last results. When any user views the results, the frequency of the task returns to 5 minutes. The report is therefore never older than 60 minutes and will refresh in 5 minutes when viewed.

  3. The reporting system filters latest runs by Resource Security on retrieval, so enable Query resource filtering if you want to limit the availability of the report to some users.
  4. To view the scheduled latest run, select the report from the browse list. These reports have the prefix Latest: in the Summary column of the browse list.
  5. The system automatically refreshes the list with the latest run when the results become available.

NOTE: Latest runs are not available in Analytics when Resource Security is enabled.

Background

Use the Run for Latest feature to schedule reports that have a reasonably high refresh rate and are viewed regularly by multiple users.

For example, imagine you have a report that takes about a minute to generate, the data changes reasonably often during the day, and multiple users need this data during the day.

In the scheduled task, enable Run for Latest, with a run every 10 minutes during business hours and a suspend of one hour. The following occurs:

  • The Scheduler tries to run the report every 10 minutes, but if no one has opened the report in the browser recently, then Scheduler runs the report once per hour. This means there is always a run available to see.
  • When the user opens the latest report in the browser (the one marked Latest:), the report immediately shows the most recent results created through the scheduled task. It also shows how long ago the system generated the report. The user is not supposed to run this query manually.
  • When the user has the latest run open in the browser and Scheduler generates a new run, the browser automatically displays the most recent run. This is similar to live or auto-run modes.
  • Different users can open the same report and get the same auto-refresh behavior.

This setup makes reasonably live data available fast for multiple users without using too many resources. The resource use is balanced out and predictable. The alternative would be to have multiple users manually run the report frequently during the day, potentially creating high load at certain times.

NOTE: Export options are not available in this scenario, because the system would email out every generated report.

Enabling Run for Latest does not impact the data being retrieved. The same data appears if you tick the box, or not.

Enabling Run for Latest impacts the expiry of reports, once generated. Run for Latest was designed to use if you want reasonably frequent reports, so the latest run will replace the one before, effectively immediately expiring earlier runs.

However, if you disable Run for Latest, the expiry strategy of the query applies, and previous runs are available.
