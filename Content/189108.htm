Accounting Rules, NDF FX Deals

Use the Accounting Rules for FX transaction type to set up the accounting rules for NDF FX deals.

Contingent Accounting, NDF FX Deals

To apply contingent accounting to NDF FX deals, select Apply from deal date to expiry date.

Contingent entries will record the exposure between deal (trade) date and Fixing date*. They will be posted according to the Posting Rules set up for the FX transaction type (Off Balance Sheet tab).

*Fixing Date is the date on which the difference between the prevailing market exchange rate and the agreed exchange rate is calculated. The date by which this difference is due, is settlement date. Applicable to Non Deliverable Forward FX deals (NDF).

On Deal Save

On Deal Save, G/L entries are only posted for Contingent Accounting or Accruing/Amortizing Forward points.

On Fixing Date

For NDF FX deals, when the deal is fixed, the following entries will be posted to the Balance Sheet:

  • Fixing Date

    DR/CR      Settlement Receivable/Payable

    CR/DR      Realized Gain/Loss

  • Settlement Date

    DR/CR      Bank Account

    CR/DR      Settlement Receivable/Payable

Accruals Accounting, NDF FX Deals

To apply accrual accounting to NDF FX deals, select Accrue forward points then make selections for the other accrual accounting items as for FX deals.

When the NDF is fixed, the settlement amount is calculated using the Contract Rate. The posting to the P&L Gain/Loss account includes the Forward Points.

Where the Forward Points are being accrued/amortized, G/L entries are required on the Fix date to avoid an over/understatement of the Forward Points amount in the P&L Gain/Loss accounts.

Because the Forward Points on NDF deals are accrued/amortized through to the Settlement date, the Forward Points Payable/Receivable account cannot be used as the balancing account for the P&L gain/loss on the Fix date.

  • When amortizing Forward Points, the Actual Position account is used.
  • When accruing Forward Points, a contra account is used. On Settlement date the Forward Points Payable/Receivable are moved from the contra account to the Forward Points Payable/Receivable account.

Accrue Forward Points

  • Accrual entries (from Start to Settlement date)

    DR/CR      Forward Points Expense/Income

    CR/DR      Forward Points Payable/Receivable

  • Fixing Date

    DR/CR      Settlement Receivable/Payable

    CR/DR      Realized Gain/Loss  (calculation based on the Contract Rate)

    DR/CR      Forward Points Payable contra/Receivable contra

    CR/DR      Realised Gain/Loss (to offset Forward Points taken to P&L)

  • Settlement Date

    DR/CR      Forward Points Payable/Receivable

    CR/DR      Forward Points Payable contra/Receivable contra

    DR/CR      Bank Account

    CR/DR      Settlement Receivable/Payable

Amortize Forward Points

  • Accrual entries (from Start to Settlement date)

    DR/CR      Forward Points Expense/Income

    CR/DR      Forward Points Payable/Receivable

  • Start Date

    DR/CR      Forward Points Payable/Receivable

    CR/DR      Actual Position

  • Fixing Date

    DR/CR      Actual Position

    CR/DR      Realized Gain/Loss

    DR/CR      Settlement Receivable/Payable

    CR/DR      Realized Gain/Loss (calculation based on the Contract Rate)

  • Settlement Date

    DR/CR      Bank Account

    • CR/DR      Settlement Receivable/Payable

Revaluations, NDF FX Deals

Although an NDF FX deal is notional in principle, Quantum still provides FX amounts for revaluation purposes.

To revalue and post accounting entries for NDF FX deals, select Revalue Deals and a Revaluation Method. All revaluation methods available for FX deals are available for NDF FX deals.

Revaluations can be posted up to the Fix Date (in Position Inquiry this is the day before fixing date but with the End Of Day flag on). If Position Inquiry is run after Fix Date there will not be any revaluation postings - only reversing revaluation entries will be generated.

NOTE:

  • If NPV Separate is selected and revaluation is carried out on fixing date, the gain/loss will not equal the settlement value.
  • The name of a revaluation method suggests the order in which calculations are performed.

Make selections for the other revaluation accounting items as for FX deals taking into account the points below.

  • Posts can be made in Base, Dominant, or Non Base currency.
  • The posting method can be Realized or Unrealized.

Currency Translation, NDF FX Deals

To apply Currency Translation, select Apply deal CCY translation.

For NDF FX deals, the Currency Translation process applies the following:

  • Unrealized Deal Level Currency Translation gain/loss is calculated on MTM Revaluations up to the Fixing Date.
  • Deal Level Currency Translation gain/loss is calculated on the accrual of Forward Points from accrual Start date to Value date.
