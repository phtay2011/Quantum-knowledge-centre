Defining Analysis Code defaults for deals

If any Analysis Codes have been set up for deals, you can define rules for providing default Analysis Code values on the applicable deal entry pages, for deals matching rule criteria.

NOTE: These default values can be overridden on Customized dealing pages if specific XML is included to do so. See Dealing Page Customizations.

Prerequisite

Before defining any Analysis Code default records, a system-wide Analysis Code Currency must be defined. This setting specifies the single currency in which any non-zero Face Values are expressed in Analysis Code defaulting criteria. See Defining a Currency for Analysis Code Defaulting.

To create a new Analysis Code Defaults record (from scratch)

  1. In the site menu's Rules submenu, select Analysis Code.

    The Analysis Code Defaulting Rule page opens.

  2. Click New.
  3. Select the following mandatory values to match deals the default Analysis Code values will apply to.
    • Transaction Type - if you select:
      • a specific type of deal - the listed Analysis Codes are those whose Scope is set to that deal type, or to Deal
      • ALL - the listed Analysis Codes are those whose Scope is set to any deal type or to Deal
    • Instrument - select a specific Instrument valid for the Transaction Type, or select ALL.
    • Counterparty - select a specific deal counterparty, counterparty group, or select ALL.
    • Entity - select a specific deal entity or select ALL.
    • CCY - select a specific deal currency or select ALL.
    • CCYLeg (Available only when Transaction Type is FX DealsandCCY is notALL) - select the deal leg the CCY value must match: BOTH, BUY, or SELL.

      TIP: If two default records (with different CCY Leg settings) both match a deal, the order of precedence is as above.

  4. Optionally specify other settings where applicable, as follows.
    • Commodity (applicable to Commodity Swap or Exposure deals only) - select a specific Commodity static data record or select ALL.
    • Face Value - either leave this field blank to apply the defaults to any deal value, or enter an amount (in the Analysis Code Currency) to apply the defaults only to deals with face values greater than or equal to that amount, after conversion to the system-wide Analysis Code Currency (as defined in Operational Settings).
    • Deal Term (time between the deal's Value date and Maturity date) - either leave this field as zero to apply the defaults to deals with any term, or enter a number of days to apply them only to deals with terms longer than that.
    • Maturity Offset Type - leave as blank or ALL to apply the defaults to deals with any Maturity date, or select ACTUAL DATE or DAYS OFFSET and enter the applicable offset as below.
    • Maturity Offset Date (available when Maturity Offset Type is ACTUAL DATE) - enter a date, to apply the defaults to deals with later Maturity dates.
    • Maturity Offset (available when Maturity Offset Type is DAYS OFFSET) - enter a number of days, to apply the defaults to deals whose Maturity date is more than that number of days later than the Settlement Date.
  5. In the Analysis Codes list:
    1. Select an entry row and in the list menu select Edit (or click a data value in the list).
    2. In the Edit dialog box, optionally enter a default Value, select or clear the Fixed option to specify whether the code is mandatory or not, and click OK.
    3. Repeat for each Analysis Code, as required.

      TIP: The listed Analysis Codes are those that apply to the specified Transaction Type, as determined by the Scope setting in Analysis Code setup. See Setting Up Analysis Codes.

  6. Click Save.

    You have defined a rule to provide default Analysis Code values for deals satisfying the rule criteria.

To create a new record based on an existing one

  1. On the Analysis Code defaults page, click and select the record you want to copy from the list.

    TIPS: Use the < and > buttons to page through the list, or the Search box to reduce the list. Click the Configure link to change list properties.

    The page fields are populated with the values for your selected record.

  2. Edit the record details as in the Edit procedure below. You must change at least one selection field setting.
  3. Click Save.

    You have defined a new rule to provide default Analysis Code values for deals satisfying the rule criteria.

To edit an Analysis Code Defaults record

  1. On the Analysis Code defaults page, click and select the record you want from the list.

    TIPS: Use the < and > buttons to page through the list, or the Search box to reduce the list. Click the Configure link to change list properties.

    The page fields are populated with the values for your selected record.

  2. Edit the selection criteria field values as required (see the procedure above).
  3. In the Analysis Codes list:
    1. Select an entry row and in the list menu select Edit (or click a data value in the list).
    2. In the Edit dialog box, optionally change default Value, select or clear the Fixed option to specify whether the code is mandatory or not, and click OK.
    3. Repeat for each Analysis Code, as required.

      TIP: The listed Analysis Codes are those that apply to the specified Transaction Type, as determined by the Scope setting in Analysis Code setup. See Setting Up Analysis Codes.

  4. Click Save.

    You have edited the Analysis Code default record. The changed criteria and default values will apply to new matching deals satisfying the new rule criteria.

To delete an Analysis Code Default record completely

CAUTION: Not recommended - use Deactivate instead (see below).

  1. On the Analysis Code setup page, click and select the record you want from the list.

    The page fields are populated with the values for your selected Analysis Code.

  2. In the page menu, select Delete.
  3. Click OK to confirm.

    The Analysis Code default record is removed from the database and will no longer be applied.

To deactivate an Analysis Code Default record

  1. On the Analysis Code setup page, click and select the record you want from the list.

    The page fields are populated with the values for your selected Analysis Code.

  2. In the page menu, select Deactivate.
  3. Click OK to confirm.

    The Analysis Code default will no longer be applied to matching deals, but the default record is retained in the database.

To reactivate a deactivated Analysis Code Default record

  1. On the Analysis Code setup page, click and select the deactivted record you want from the list.
  2. In the page menu, select Reactivate.
  3. Click OK to confirm.

    The Analysis Code default will again be applied to matching deals.
