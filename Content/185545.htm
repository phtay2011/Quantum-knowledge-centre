Matching Criteria, Deal Approval Rules

These items define which deals and deal actions the approval rule entry applies to.

  • Transaction Type - select one transaction type or ALL.
  • Deal Type - select DEAL, ALL, or:
    • QUOTE - when Transaction Type is ALL, Currency Options, FX Deals, or Money Market, or
    • REQUEST - when Transaction Type is ALL, FX Deals, or Money Market.

      TIP: See Approval and Authorization Rules for Requests.

  • Instrument - select one instrument (filtered by Transaction Type) or ALL.
  • Action - select one Deal action (filtered by Transaction Type and Instrument), ALL, or REQUEST (valid when Instrument is ALL, an FX instrument, or a Money Market instrument - see Approval and Authorization Rules for Requests).

    CAUTION: A DELETE deal action takes immediate effect and cannot be undone, even if subject to approval or rejection. See Deleting a Deal.

  • Counterparty - select one counterparty or ALL.
  • Category - select the category of deal counterparty: INTERNAL, EXTERNAL, or ALL.
  • Dealer - select one dealer or ALL.
  • Date Type - select Forward Dated, Back Dated, Same Day, or ALL.
  • Ccy - select the currency of the Limit Amount OR NONE for non-tiered approvals (no lower limit).

    For Tiered Approvals, you must specify the same currency in entries with identical Matching criteria except forLimit Amount.

  • Limit Amount - enter the minimum threshold amount for deals you want this Approval Rule entry to apply to. Limit Amounts are compared to EITHER:
    • the remaining Face Value - only if Transaction Type is Money Market and Action is ROLLBACK or UNWIND, OR
    • the original Face Value - in all other cases.

    For no lower limit, set Limit CCY to NONE and Limit Amount to zero. Approval will then be required for flows of any amount.

    For Tiered Approvals, the lowest Limit Amount equaled or exceeded by that deal amount determines the applicable rule entry (the approval tier).

    NOTE: Tiered approvals are NOT currently supported for Exposure deals.
