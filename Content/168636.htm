Interest Rate Options

Use the Interest Rate Option Rate page to record the volatility rates used in the valuation of Interest Rate Options. You can define time intervals and record bid/offer spreads or mid rates.

The combination of Rate Name, Rate Type, Currency, and Date uniquely identifies this type of rate record in the system, and is displayed above the Rate Header.

Rates area, Interest Rate Option Volatility Rate

Use this area to enter and/or change Interest Rate Option Volatility Rate values.

To add a new interval definition from scratch:

  1. In the grid menu, select New. The New Rate dialog box opens.
  2. Enter the required values (see below) for the new time band and click OK. The new interval definition is added to the grid.

To add a new interval definition by copying an existing one:

  1. Select the row you want to copy and in the grid menu, select Copy. The Copy Rate dialog box opens.
  2. Enter a new Interval, change other values as required (see below), and click OK. The new interval definition is added to the grid.

To edit an existing interval definition:

  1. Select the row you want to edit and in the grid menu, select Edit. The Edit Rate dialog box opens.
  2. Change values as required (see below), and click OK. Your changes will show in the grid.

To remove an interval definition:

  1. Select the row you want to remove and in the list menu, select Delete. The Confirm Delete dialog box opens.
  2. Click OK. The interval definition is removed from the grid.

Field Descriptions

Interval - the time band against which rates or prices are recorded. If you require a specific date, leave this field blank and enter the required date here (do this if a security has a specific Maturity Date).

Input (Bid/Offer) - the rate entered either as a spread with the Bank's Bid and Offer rates separated by a slash, OR as a single rate (without a slash).

Bank Bid (read-only) - the rate you entered to the right of the slash in Rate Input, or the single actual rate.

Bank Offer (read-only) - the rate you entered to the left of the slash in Rate Input, or the single actual rate.

Date - the actual date up to which the Interval definition applies. If a Date Interval or Days value is entered, this date is that period added to the record Date (Rate Header area). This is an actual days calculation based on the calendar for this currency.

Days (read-only) - the number of days from the record Date (Rate Header area) to the specified rate date (in the Date column).Basis (mandatory) - the timing basis the system will use to calculate the rate: CASH, QUARTERLY, SEMI ANNUAL, and ANNUAL.

High / Low / Close - the highest, lowest, and closing rates traded on a specific day.
