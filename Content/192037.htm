Applying FX Cover Rules

The FX Cover Processor:

  • Looks only for new or changed (or reset) Projected cash flows with the Exposure Approval analysis code set to Approved.
  • Initially matches rules on the Projected Cash Flow's Counterparty (or parent counterparty, if one exists).

    When an FX Cover Rule Set record is found for that Counterparty, the FX Cover Processor:

    • Looks for matches against the Cash Flow Instrument (Anticipated or Committed),
    • Checks if dates are in range and uses the first date range matched - see Date range matching below
    • Applies currency matching conditionally - see Currency matching below
    • When a match is made, generates a new FX Request record from the Projected Cash Flow data (based on the mapping provided in the hdgfxmap.txt file - see FX Cover Setup)

NOTE: If the Projected Cash Flow has been previously matched, the FX Cover Processor first tries to match against existing rules. If none are found, it retrieves the old rule that the cash flow was initially matched against.

Date range matching

Exposure From/To Dates are calculated as offset from Today (for example, Today + 6 months). These dates are inclusive - for example, for a range spanning 6 months from Today to 9 months from Today, where Today is 13/03/09, the time range limits are 13/09/09 and 13/12/09, respectively. To match, the Projected Cash Flow's Settlement Date must be on or after the From date and before the To date.

Deal dates are matched against Deal From/To dates and are inclusive at both ends of the time range - that is, to match, the Deal date must be on or after the From date and before or on the To date.

Currency matching

The last check for a positive rule match is comparing the Buy currency of the resulting FX Request with the Currency field of the rule selected above (based on Instrument and Date range).

IMPORTANT: The Sell currency (the second currency) is assigned based on the Dual Currency setting:

  • If Dual Currency is set to N (a Single currency applies), the Sell currency is the Domestic Currency Analysis Code of the Counterparty.
  • If Dual Currency is set to Y, the Sell currency is the ccy2 value of the Projected Cash Flow.

Reversal logic

A reversal is created when any changes are made to the Projected cash flow's:

  • Currency
  • Entity
  • Counterparty
  • Settlement Date (if outside the Date Threshold)
