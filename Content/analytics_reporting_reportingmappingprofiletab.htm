Mapping Profile tab (Configure Analytics Query)

(This tab is mandatory for Monte Carlo and Parametric VaR queries, otherwise it appears only when an existing Mapping Profile Definition record is specified on the Properties tab.)

Use this tab to view and optionally change the Mapping Profile Definition settings for the current query. A Mapping Profile specifies one or more custom groupings as Matching Criteria and Settings for each set of criteria (see below).

Initial settings depend on whether or not an existing Mapping Profile Definition record is specified on the Properties tab. If so, the defaults are taken from that Definition record, but if not, then - for Monte Carlo and Parametric VaR queries only - a new Definition record is created with the same Name as the Query (as on the Properties tab) and a standard set of defaults apply. In either case, any changes made on this tab are saved in the specified Definition record when the query configuration is saved.

To add a new set of Matching Criteria and Settings:

  1. In the Matching Criteria grid menu, select New. The New Matching Criteria dialog box appears.
  2. Under Settings, enter a new Mapping Code. The other Settings values show the corresponding defaults (to change these, see Step 5).
  3. Under Matching Criteria, select specific values or All for Currency, Entity, Transaction Type, Instrument, Counterparty, Issuer, Curve, and Unit Issue. Under Matching Order, enter the Order Number that specifies the order in which this criteria is to be applied.
  4. Click OK. A new row is added to the Matching Criteria grid, showing the new values, and - only if no Settings grid row already exists for the selected Mapping Code - a new row is also added to that grid.
  5. In the Settings grid, select the row for the new Mapping Code and in the grid menu, select Edit. The Edit Settings dialog box appears.
  6. If required, select a different Yield curve to be used in the calculation, and EITHER (for a General type) select a Time Profile and Risk Type, OR (for a VaR type) select an appropriate Asset Class and enter a Probability(%) between 1 and 100.
  7. Click OK. The Edit Settings dialog box closes.
  8. (VaR type only) Under VaR Files, enter a Correlation Data File and/or Volatility Data File.

*TIP: You can also create a new set by selecting an existing one and selecting Copy. Enter a new name and edit the Matching Criteria and Settings values as required.

To edit existing Matching Criteria:

  1. In the Matching Criteria grid, select the row you want to edit and in the grid menu, select Edit.
  2. In the dialog box, edit the required fields and click OK.

To edit the Settings for existing Matching Criteria:

  1. In the Settings grid, click on the Mapping Code for the criteria.
  2. In the Edit Settings dialog box, edit the values as required and click OK.

To change the order of the Matching Criteria Columns:

  1. In the Matching Criteria grid menu, select Reorder Columns.
  2. in the dialog box, drag and drop the column buttons as required and click OK.

To delete an existing set of Matching Criteria and Settings:

  1. In the Matching Criteria grid, select the row and in the grid menu, select Delete.
  2. When prompted, click OK to confirm. The Matching Criteria and corresponding Settings rows are removed from the grids.
