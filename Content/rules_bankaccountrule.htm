Bank Account Rule page

TIP: Bank Account Rules were previously referred to as Bank Account Defaults in Quantum.

Use this page to create and maintain rules that apply particular bank account instructions to Quantum deal cash flows, depending on deal values such as Entity, Transaction Type, Instrument, Counterparty, CLS and so on, as specified in the Account Rule Selection area. You can define multiple Bank Instructions for a single rule selection, each with a different Effective Date.

NOTE:

  • You cannot save a transaction unless the appropriate accounts and account rules are set up. Account rules are required even if the currency is a commodity.
  • When you change Account Rules, existing cash flows are not changed. To update them, run Bank/Counterparty Bank Account Conversion.

To add a new rule record - click New; select the Bank the rule applies to; specify the criteria used to select cash flows (Entities, Transaction Types, and so on); add one or more rule entries defining the settlement instructions with their Effective Date (see below); then click Save.

To list records - Click to open the browse window. (Click the Configure link to change browse list properties.)

To change a record's details - Select the record in the browse list; optionally change the selection criteria* if required; add, edit, and/or delete rule entries (grid rows) as follows; then click Save.

  • To add a new rule entry - In the grid menu, select New; in the dialog box, enter the Effective Date; select the Bank Account and Payment Category; then click OK. A grid row is added for the new entry.
  • To edit a rule entry - click a data value in the grid row; in the Edit window, make the required changes; then click OK.
  • To delete a rule entry - select the grid row; in the grid menu select Delete; then confirm.

*TIP: Do not change the Bank setting. Instead, use the Copy function to create a new rule record based on the current one (or create a New record), enter the new bank, and edit that record as required.

To delete a rule record in full (not recommended; Deactivate instead) - Select it in the browse list; in the page menu, select Delete; then confirm.

To deactivate a record - Click , select the record, and in the page menu select DeActivate. You can later reactivate it in the same way.

NOTE: When Static Data Approval applies and you add, update, or delete a record:

  • until the change is approved (or rejected), no further changes are allowed to that record (on this page).
  • the record is flagged as pending approval and all items become read-only.
  • links to the Unapproved page are provided for that record.
  • an Approver must use the Unapproved page to approve or reject the change (you cannot approve your own changes).
  • further editing is allowed on the Unapproved page (for an add or update) - provided no approve action has yet been taken.

When Static Data Approval does not apply, all changes can be regarded as immediately approved.

In either case, Static Data and Rule changes are recorded in the audit log. Use Audit-Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.

For details of data items, click on an item when the Help panel is open.
