FX

Use the FX Rate page to record foreign exchange rates for valuing foreign exchange deals translating foreign currency exposures.

You can define time intervals and record bid/offer spreads or mid rates. Quantum also supports direct rate feed data for live rate feed access and (for these rates) bulk Insert/Replace, enabling you to edit rates directly or via a spreadsheet application (as with Bulk Replace and Insert for Deal Map Items).

The Currency, Rate Type, and Date key values together uniquely identify a saved rate record, and are displayed at the top of the page and in the web browser tab. Changing any of these values loads an existing record if one matches those values, otherwise a new empty one is started.

NOTES:

  • FX rates are entered against U.S. Dollars, not as cross rates.
  • The Bank Bid and Offer rates are distinguished based on the rules defined in the applicable Currency Pair record, as this establishes direct and indirect currency pairs.

Forward Points grid, FX Rate

Use this grid to set up interval definitions and record foreign exchange rate forward points.

To add a new interval definition from scratch:

  1. In the grid menu, select New. The New Rate dialog box opens.
  2. Enter the required values (see below) for the new time band and click OK. The new interval definition is added to the grid.

To add a new interval definition by copying an existing one:

  1. Select the row you want to copy and in the grid menu, select Copy. The Copy Rate dialog box opens.
  2. Enter a new Interval, change other values as required (see below), and click OK. The new interval definition is added to the grid.

To edit an existing interval definition:

  1. Select the row you want to edit and in the grid menu, select Edit. The Edit Rate dialog box opens.
  2. Change values as required (see below), and click OK. Your changes will show in the grid.

To remove an interval definition:

  1. Select the row you want to remove and in the list menu, select Delete. The Confirm Delete dialog box opens.
  2. Click OK. The interval definition is removed from the grid.

Field Descriptions

  • Rate Interval - the time band against which the FX rate forward points are recorded. Enter the required prefix and a number may be entered, for example M 6 (6 Months), D 7 (7 Days), Y 3 (3 years).
  • Bid Input /Offer Input - the forward points, entered as a decimal or a whole number, depending on the method defined in Currency Setup. A preceding minus sign is valid if negative forward points apply. If a spread is entered, identify the descending rates as negative and increasing rates as positive. Use a positive bid or offer in a spot rate.
  • Bid - the bank bid rate if a slash was used (spread), or the actual rate if a slash was not used.

    NOTE: The Bank Bid and offer rates are distinguished based on the rules defined in the Currency Control for each currency pair, as this establishes direct and indirect currency pairs.

  • Offer - the bank offer rate if a slash was used (spread), or the actual rate if as slash was not used.
  • Date - the actual date up to which the Interval definition applies. If an Interval or Days value is entered, this date is that period added to the record Date (Rate Header area). This is an actual days calculation based on the calendar for this currency.
  • Days - the number of days over spot, from the rate record Date to the Spot Date. This field cannot be modified.
  • High / Low / Close (read-only) - the highest, lowest, and closing forward points traded on the specific Date, as recorded from the electronic rates feed source.

Spot Rate area, FX Rate

Use this area to set up the Spot Rate details.

  • Bid & Offer - the bank's actual bid rate and offer rate.
  • The Bank Bid and Offer rates are distinguished based on the rules defined in the applicable Currency Pair record, as this establishes direct and indirect currency pairs.
  • Spot Date - the actual date to which the date interval indicated by the Days field applies. If a Days value is entered, this date will represent the number of days added to Spot value from the record Date. This is an actual days calculation based on the calendar for this currency.
  • Spot Days - the number of days over spot, from the rate record Date to the Spot Date. This field cannot be modified.
  • High / Low / Close (read-only) - the highest, lowest, and closing Spot rate traded on the Spot Date, as recorded from the electronic rates feed source.
