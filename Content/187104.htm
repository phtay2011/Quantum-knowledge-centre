Filling a Request

You can fill a Request by either:

  • creating a new actual deal based on the Request settings (Fill action), or
  • linking the Request to an already existing actual deal (Mark as Filled action).

These actions are enabled for a saved Request when:

  • the logged in user has Fill rights to the particular Request page in Module Security, and
  • the Request does not require approval, or
  • the Request is fully approved, or
  • the request is NOT fully approved but the Allow Processing option is set to YES in the applicable Deal Approval Rule.

Fill action

When you select Fill:

  • a new FX Deal or Money Market Deal window is opened (according to the Request type),
  • the Deal fields are populated with settings from the Request,
  • all Deal fields are enabled for you to enter additional values as required for the actual deal,
  • only Save and Cancel buttons are available (no other actions are available), and:
  • if you click Cancel - the Deal window is closed, and the Request remains in an unfilled state, or
  • if you click Save:
    • all deal save validation applicable to the deal type applies (for example, Counterparty, Entity, and Dealer must be set),
    • the new actual deal will be saved in the system (it is badged Fills request mmm on the Deal page, where mmm is the Request deal number),
    • the Deal window is closed,
    • on the Request page, the Fill and Mark as Filled actions are disabled in the menu,
    • the Request is badged Filled by nnn, where nnn is the new actual deal number,
    • the date/time of the Fill action is saved with the Request, and may be included in the Request browse view as a column or used as a filter, and
    • if an FX Request and the Replication process is active - then, in Analytics:
      • the linked FX Deal details will be replicated as a new FX Deal,
      • the linked FX Deal will be linked to the FX Request at a database level and via a visible tag Fills request Reqid (where Reqid is the FX Request's ID), and
      • the FX Request will be linked to the new FX Deal at a database level and via a visible tag Filled by Dealid (where Dealid is the FX Deal's ID).

Mark as Filled action

When you select Mark as Filled:

  • the Mark Request as Filled dialog box is displayed, with Mark as Filled and Cancel buttons and an Available Deals grid in which you can select an existing FX or Money Market deal from those deals that:
    • are consistent with your Resource Security permissions,
    • have not been not deleted,
    • are not yet linked to a Request, and
    • are open (have a Maturity or Value Date equal to or later than the current date), or
    • have matured within the last 14 days (have a Maturity or Value Date no more than 14 days prior to the current date).

      NOTE: In Release 6.3.1, deals with a Maturity or Value Date earlier than the current date were NOT available.

  • if you click Cancel - the dialog box is closed, and the Request remains in an unfilled state, or
  • if you click Mark as Filled (after selecting a deal):
    • all deal save validation applicable to the deal type applies (for example, Counterparty, Entity, and Dealer must be set),
    • the linked actual deal will be badged Fills request mmm on the Deal page (where mmm is the Request deal number),
    • the dialog box is closed,
    • on the Request page, the Fill and Mark as Filled actions are disabled in the menu,
    • the Request is badged Filled by nnn, where nnn is the linked actual deal number,
    • the date/time of the Fill action is saved with the Request, and may be included in the Request browse view as a column or used as a filter, and
    • if an FX Request and the Replication process is active - then, in Analytics:
      • the FX Request will be linked to the existing FX Deal (as above),
      • the linked FX Deal details will be updated, and
      • the linked FX Deal will be linked to the FX Request (as above).
