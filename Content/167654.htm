Breach Approval Rule

Use the Breach Approval Rule page to define the Breach Criteria for a Limits query. The same level of granularity is provided as for the Limit Definition, which enables Breach Criteria of varying degrees of complexity to be defined.

For each Limits Query, you can have only one Rule record, which specifies:

  • the required Number of Approvals, and if this is:
    • None - rule entries do not apply,
    • One - one or more rule entries must be entered on the First Approver tab, or...
    • Two - one or more rule entries must be entered on both the First ApproverandSecond Approver tabs;
  • whether or not to send Limit Breach and Breach Approval notifications;
  • if the deal's Creator and/or Dealer may or may not b Approvers; and
  • whether or not to activate the rule.

Limits Query - Select the name of the Limit Query this rule applies to.

NOTE: Only Limit Queries that have been enabled for Limit Breach Management will be available for selection in the drop down list.

No of Approvals - This field determines the number of approvers that will be required to approve a Limit Breach. The values are: None, One, or Two.

Activate Process - Select Yes or No to determine whether or not to activate the rule.

TIP: Instead of deleting a rule no longer needed, set Activate Process to No, so that redundant rule details are saved for future reference (and for possible reactivation or copying).

Send Notification Message - This field will determine if Limit Breach notification and Limit Breach Approval notification will be sent. The values are: Yes, No.

Breach Approval Currency - This is the currency that the Breach Amount for each Approver will be defined in. The drop down list will be all the currencies defined in the system.

Notification Recipients - This field will determine who gets the notification messages. The values are:

  • Approvers - all approvers defined for the approval of the Limit Breach.
  • Dealers - the dealer of the deal causing the breach.
  • Both - all approvers of the Limit Breach and the dealer of the deal causing the breach will receive the notifications.

First Approver tab, Second Approver tab - Use these tabs to define the Approvers for the LImit Breach.
