Cash Flow Netting and Settlement Approval Rules

The following business rules apply for determining which Settlement Approval Rule record and entry will be applied to a netted group of cash flows when the netted group's status is set to ACTIONED.

All cash flows in the group are treated together to determine the one Settlement Approval Rule entry to be applied to them all, as follows (in this order).

  • If all cash flows in the group have the same Entity, then match to that Entity record; otherwise, use the ALL record.
  • Use the best matching NET Flow Type rule, but if none, use the best-matching CASH rule.
  • All cash flows in the group must have the same Flow Ccy, Payment Method, Counterparty, and Account, so match to those values if specified; otherwise use the ALL rule in each case.
  • If all cash flows in the group have the same Matched Status, then match to that status; otherwise match to the highest-priority (most conservative) status in the group, in the following order:
    1. Unmatched (includes Not Created),
    2. Manual Matched,
    3. Automatched (Electronically Matched), then
    4. ALL.
  • If all cash flows in the group have the same Payment Instructions, then match to those; otherwise match to the highest-priority instructions in the group, in this order:
    1. Modified SSI,
    2. SSI, then
    3. ALL.

      NOTE: Ad Hoc Payment Instructions cannot be included in a netted group of cash flows.

  • The applicable Flow Direction is that of the Net Amount, with a zero amount matching eitherReceipt or Both.
  • If all cash flows in the group have the same Transaction Type, then match to that Transaction Type; otherwise, use ALL.
  • If all cash flows in the group have the same Instrument, then match to that Transaction Type; otherwise, use ALL.

If the applicable Settlement Approval Rule entry specifies a currency limit, only the total Net Amount is validated against that limit (if necessary, the net amount is converted to the limit currency first).
