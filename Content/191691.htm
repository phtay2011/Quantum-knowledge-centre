Stock Flows in Quantum

Quantum supports SWIFT MT540 through MT543 EFT messages, which cover the receiving and delivery of Securities (stock). These movements of stock are termed Stock Flows, and the stock is held in, or taken out of, a Stock Account.

NOTE: SWIFT MT518 Confirmation messages may also be created for Market-Side Securities Trade Confirmations (supported only for Money Market and Repo deal Confirmations to SWIFT Accord using XML-based message templates).

MT541 and MT543 EFT messages respectively cover receiving and delivery of stock against payment, with the cash payment or receipt advice information included in the same message (no separate cash payment or receipt message is output), for the following:

  • Security deals
  • Repos involving Securities that create stock flows where the Repo Format is:
    • Buy / Sell Back
    • Classic
    • Reverse
    • Sell / Buy Back

MT540 and MT542 EFT messages respectively cover the receiving and delivery of stock free of payment for:

  • Repo deals involving securities that create stock flows where the Repo Format is:
    • Fee Based Borrowing
    • Fee Based Lending
    • Security Borrowing
    • Security Lending
  • Stock Account Transfers

NOTE: If a financial change occurs (for example, a different amount), then reversal stock flows and new stock flows will be generated and a CANC and a NEW MT540/541/542/543 EFT message will be created, as relevant.

Examples

MT541

Example 1

You have a Stock account with Custodian 1 and buy AUD Government Bonds from Counterparty X. When you settle the deal, an MT541 message is output to Custodian 1 advising them to receive these bonds from Counterparty X, with the cash payment information included in that message (no separate cash payment message is output).

Example 2

A change to the amount is required for the stock flow in Example 1. An MT541 Cancelation message is sent containing a link to the previous MT541 message and with Tag 23G set to CANC. A new MT541 message is then sent with the new Amount and with Tag 23G set to NEWM.

MT543

Example 1

You have a Stock account with Custodian 2 and sell USD Government Bonds to Counterparty Y. When you settle the deal, a MT543 message is output to Custodian 2 advising them to deliver these bonds to Counterparty Y's Custodian, with the receipt advice information included in that message (no separate receipt message is output).

Example 2

Incorrect CUSIP and ISIN values were sent in the Example 1 stock flow. An MT543 Cancelation message is sent containing a link to the previous message and with Tag 23G set to CANC. A new MT543 message is then sent with the correct CUSIP and ISIN, and with Tag 23G set to NEWM.

MT540 and MT542

Example 1

You want to simply transfer the stock itself from Custodian 3 to Custodian 4 (without any associated payment) either way.

When you settle the Stock Account Transfer "deal":

  • An MT542 message is output to Custodian 3 advising them to deliver the stock to Custodian 4.
  • An MT540 message is output to Custodian 4 advising them to receive the stock from Custodian 3.

Example 2

You have a Stock account with Custodian 5 and make a Repo deal, agreeing to borrow, free of payment, a Security from Counterparty Z until the end of 2010.

The underlying security Instrument is for AUD Government Bonds, and is set up with stock flows.

  • When the first leg of the Repo deal is settled, an MT540 message is output to Custodian 5 advising them to receive these bonds from Counterparty Z.

    (Any associated cash payments/receipts are handled independently of the stock flows.)

  • When the second leg of the Repo deal is settled (on December 31, 2010), an MT542 message is output to Custodian 5 advising them to deliver these bonds to Counterparty Z.

    (Any associated cash payments/receipts are handled independently of the stock flows.)

Stock Flows in Quantum

Quantum can generate and output MT540 through MT543 messages, as above.

To support this, the concepts of Custody accounts and Stock Flows are incorporated into Quantum. This provides you with a release/control vehicle where you can approve Stock Flow delivery/receipt instructions, action them, and finally generate SWIFT message output (as for cash payments/receipts).

Supported Deal types

Quantum stock flows are supported for the following:

  • Security Deals based on instruments with the Allow stockflows option selected* and any of the following Formula Timing settings:
    • BOND - PERIODIC,
    • CERTIFICATE DEPOSIT - PERIODIC,
    • CIB-RBA - PERIODIC,
    • DISCOUNT - PERIODIC,
    • DISCOUNT - PREPAID,
    • FRN-CGL - PERIODIC,
    • INDEXED ANNUITY ACTUAL END DATE,
    • INDEXED BOND - PERIODIC,
    • INDEXED GILT - PERIODIC,
    • SIMPLE DISCOUNT - PREPAID, or
    • ZERO COUPON DISC - ON MATURITY.
  • Repo Deals where the underlying Security instrument has the Allow stockflows option selected and any of the above Formula Timing settings.

    All Repo formats support stock flows (some in association with cash flows, some not).

  • Stock Transfers

Prerequisites and Setup for Stock Flows

Before you can produce SWIFT stock flow message output, you must do the following:

  • Set up MT540/1/2/3 preferred format templates (seed templates are provided).
  • Set up physical locations for Securities stock.
  • Set up at least one Instrument specifically for Security deals involving stock flows by selecting the Allow stockflows option on the Security Instrument page.
  • Set up at least one Security Identifier for for Security deals involving stock flows.
  • Set up Instruments for Stock Transfers if required.
  • Specify at least one Counterparty as a Custodian by selecting the Custodian option on the Counterparty page.
  • Set up your own custody (stock) accounts (and Stock Account Defaults if required).
  • Specify Counterparty Delivery Instructions for Stock (and Counterparty Stock Account Defaults if required).
  • Set up Approvals for Deals with stock flows if required, using Approval and Authorization Rules.

Stock Flow Processing Overview

After you have fulfilled the above requirements, you can then:

  • Create actual deals based on Instruments set up for stock flows.

    For example, when you save a Security or Repo deal that uses an Instrument defined to create stock flows, a stock flow is created in addition to the normal cash flows of a deal.

    NOTES:

    • Stock flows are only linked to cash flows if the Link to settlement cash flows is checked for the Stock Account Defaulting Rule.
    • Stock Account Transfers and Repo deals free of payment do not have any linked cash flows.

    The stock flow is used to track the physical flow of the Stock, and is a representation of the Face Value of the stock. If the Face Value is changed, the stock flow will change; and if a stock is held until maturity, the net position of stock flows will be zero.

  • Settle and approve the deal stock flows (with associated cashflows, if any).

    As for cash flows, stock flows will have the Custodian settlement information defaulted as standard settlement instructions, and will appear in both Settlements Processing and Settlement Approvals.

  • Use EFTs Processing to produce SWIFT MT541/MT543 stock flow message output.

    Once actioned and (optionally) approved, the stock flows will be accessed by the SWIFT EFT Payments Interface for processing as follows.

SWIFT Output Format Templates (MT540-543)

Depending on Custodian requirements and the relationship between the custodians of the dealing parties, you can vary the SWIFT message output format, by specifying different SWIFT Output Format templates. These templates control the output message format and content, and allow you to include optional fields.

Stock Account Transfers

Two stock flow messages are created:

  • An MT542 message is sent to the "Source" Custodian advising them to deliver the stock.
  • An MT540 message is sent to "Destination" Custodian advising them to receive the stock.

The Source and Destination Custodians are those specified, for that transfer, in the Source and Destination areas on the Deal Details tab in Stock Transfers.
