Areas, Workflow Report page

This topic covers the different tabs seen in the Workflow Report page and describes how to use them.

Report Tab

The Report tab has the following grids:

  • Joined Sources - the grid with the main source and the sources that are joined to the main source or other joined sources.

  • Appended Sources - a grid with sources that you can append to the result of the joined sources.

    NOTE: The Appended Sources grid is available when there is at least one Joined Source as you cannot add appended sources without at least one joined source.

    Each column in this the Appended Sources corresponds with one of the columns of the joined sources (for example, have the same display name and data type). Once you run the report with appended sources, the result of the each appended source is added to the final results of the report.

The Joined Sources grid has the following columns:

  • Source - the data source used for the report. Includes internal data sources such as Quantum and Analytics deals, reports, and static data; deal and settlement approval rules (in the Reporting folder); database views (in the Database folder, with the prefix QVW_DATASOURCE_%); and DTLs with WorkflowOut/In purpose (in the Web API folder). Also includes Workflow Templates, other Workflow Reports, Workflow Components, and Workflow Component Problems.
  • Name - a name for the source. Defaults to the Source value. Used to:
    • shorten the source name, or
    • include the same data source twice so you can compare the source under different conditions.
  • Columns - selected columns from the data source to be available in your target report's result set, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level. Formats available within the selected column include:
    • Amount - the length varies with the Currency selected (default 2dp),
    • FX Rate - 8 dp,
    • Interest Rate - 12 dp,
    • Number - an integer,
    • Deal Number - a link to the deal,
    • Date -your date format, and
    • Text.
  • Settings - use these settings to reduce the target report's result set. Either fixed values or linked to query filters on the target page.

    TIP: You will get an error message if you have not included a required (key) data item in the Settings.

  • Joined To (optional) - This appears when two or more sources exist. Use to map fields to another source so that values from both sources - including (for Dealing, Static, Rates, and Reporting sources) values of linked items down to any level - will appear in your custom report results when the joined items match.

Select New to open the New Source dialog box.

New Source dialog box

Use the New Source dialog box to add, aggregate, or join sources, and to edit columns and settings.

To aggregate data

  1. Select Aggregated and the Columns tab shows both:
    • a Group By grid - enables you to select one or more fields to group by (or none), and
    • the Columns grid – enables you to select fields to aggregate by.

    The available Aggregate By functions depend on the Data Type (as in pivots):

    Data Types

    Aggregate By

    Amount, Number

    Count, Distinct Count, Max, Min, None, Running Sum, Sum

    Reference, Date, Text

    Count, Distinct Count, Max, Min, None

  2. You can aggregate within a source, allowing Actions against aggregated data.
  3. The data source returns the data aggregated by the aggregation functions and grouped by the fields.
  4. Use the aggregated results in joins with other data sources - that is, use only the Group By fields and selected columns to define a join with an other data source.

Select a Source and two tabs appear: Columns and Settings.

Columns tab

In the Columns tab, select Add to select the data item within the data source.

Data Type of the source

The columns that have the Data Type as Amount or Number have some default formatting.

If the data type for the source is Amount and, using the Edit Column dialog box, you chose to display it as:

  • Amount - the length varies with the Currency selected. If no Currency selected, then 2dp;
  • FX Rate - the length is 8 dp; or
  • Interest Rate - the length is 12 dp.

If the data type for the source is Number, using the Edit Column dialog box, you can display it as:

  • Number - the system displays an integer, or
  • Deal Number - the column includes the link to the deal.

Select the Currency that has the format you want to use in the column. The Currency browse list only shows the Data Types of Reference and Text and currencies that are from the same data source.

Join tab

In the Join tab, select the types of sources to join.

Select one of:

  • Join on Columns - joins based on column names
  • Join on Expressions - select fields from both of the sources to use in the expression, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level. Validation is Boolean (Yes/No).

About Deal and Settlement Approval Rules

Using these data sources identifies what deal or settlement approval rules apply to a deal.

Deal approval rules

You can input a deal number, action number, or any of the 11 other input fields and the system displays which deal approval rules apply to that deal action or action type, ordered by the priority.

Input fields include: Action, Action Number, Amount, Counterparty, Counterparty Category, Currency, Date Type, Deal Number, Deal Type, Dealer, Entity, Instrument, or Transaction Type.

Columns include: 1st Approval, 2nd Approval, Action, Allow CF Action, Allow Processing, Approval, Counterparty, Counterparty Category, Currency, Date Type, Deal Type, Dealer, Entity, Excluded to Approve, Instrument, Limit, Amount, Priority, or Transaction Type.

The source is called Deal Approval Rules.

Settlement approval rules

You can input a cash flow, stock flow, or net flow number or any of the 12 other input fields and the system displays which settlement approval rules apply to that flow. Flow Number will default to cash flow number, unless you use Flow Type to choose another type.

Input fields include: Flow Number, Flow Type, Transaction Type, Instrument, Payment Method, Matched Status, Counterparty, Bank Account, Payment Instructions, Currency, Amount, or Entity.

Columns include: 1st Approval, 2nd Approval, Approval, Bank Account, Counterparty, Flow Ccy, Flow Direction, Flow Type, Instrument, Limit Amount, Limit Ccy, Match Status, Payment Instruction, Payment Method, Priority, or Transaction Type.

The source is called Settlement Approval Rules.

Troubleshoot workflow reports

Quantum does three things to help you diagnose and fix broken workflow reports.

  • A message on the workflow report's page notifies you that the page is broken. The error message is:

    There is a problem retrieving the Source.

  • Scheduler keeps working.

  • There is a configuration setting in the web.config called WorkflowMaximumRowsPerSource. This constrains the maximum number of sources that can be joined. The default is ALL.

    If any source returns more than the configured number of rows, the error message in the workflow report is:

    Unable to run report - data source qvw_datasource_audit has more rows than in the system settings.

Filters Grid (Report tab)

The Filters grid shows the filters to be displayed as Filters fields on the report page, and enables you to select one or many items from a filter's browse window.

Set up sources in the Sources grid and these appear when the filter's Type = Lookup.

Calculated Columns Grid (Report tab)

A calculated column is an additional column in the results grid. When you run the report query, the system adds the column, and the calculated values, into the s the results grid of the report.

In the Calculated Columns grid menu, select New and use the New Column dialog box to set up an additional column in the results grid.

The Display Name must be unique.

The Display As formats include: Text, Date, Number, Deal Number, Amount, FX Rate, Interest Rate, Yes/No, and Countdown Timer.

Click Save and the system validates the expressions. You can use the Number column in the expression for a calculated column without it being included in the report. You can select items for the Filters area that will appear in calculated columns.

Run the report and the results are similar to this.

You can use expressions in the calculated column so that you can have a more complex logic creating the column. You can use the calculated column as a Column (data source) in the New Action dialog box, by selecting the Update > Bulk Update folder.

In Expression Builder you'll find some functions that are designed to aggregate data, such as Sum(values, groupBy).

Example

When selecting the source, there is a data source - Sequence - in the Values folder, available to select as the primary source. This allows you to apply one or more calculated columns to the result set based on a sequence number. The settings include: Rows, Start, Step.

TIP: To find the Values folder, type "Values" in the search box.

Actions tab

The Actions tab shows the list of tasks you have configured the report to do. For example, you might have configured the report to include a toolbar button that creates a new Actual Cash Flow deal.

To add a (general) action

  1. Open Configuration > Report.
  2. In the menu select New and the New Actions dialog box appears.
  3. In Action Name click the lookup. The top level folders are organized as follows:
    • Create - the action creates a new record for your selected page type without opening the relevant page, You can select any Dealing or Request page (including variants), a supported Static Data or Rule page (for example Counterparty, Entity, User, Static Approval, and so on), the XML Data page, plus other site-specific pages.
    • Delete - make a button or menu item that will delete a record of your selected page type (for a subset of the pages available for Create above).
    • Merge - merge sets of data.
    • Open - open a new page, as selected: Analytics > Deal Analyze; Cash Management > Actual Cash Flow (or variant); Dealing > Quantum deal, request (or variant); Dialog > templates created with Purpose set to Any (if there is no template with Purpose set to Any, the Dialog folder is not available); Pricing > Bond, plus other site-specific pages under Merge or External.

      • Each template (with Purpose set to Any) automatically gets a workflow action that opens a dialog box and renders the output of the template. These dialog boxes are based on the selected rows from the report results. You can drill-down into the selected row(s) and give related or aggregated information.

      Two tabs are available in the dialog box:

      • Details - displays outputs of records that passed validation. If each record passes validation, the entire output is displayed in the dialog box, and both tabs are hidden.

      • Error - displays validation error messages of records that failed validation. If each record fails validation, the Details tab is blank.

    • Process - to perform a specified action for the selected deal or request type (including variants), supported static data item, or Workflow report that has one or more multi-select actions configured. For example Interest Rate Swap > Add Deal Map Items, Rate Set, Unwind, or Update Deal Map Items,
    • Transfer - choose a data transfer location (DTL). Populated with locations that have a template or template and parser (DTLs with WorkflowOut/In purpose) associated with them.
    • Update - to update the specified deal or request type (including variants), static data or rule type (as for Create above), or:
      • Bulk Update - to use the calculated column as a Column (data source),
      • Save Holiday - to use the report to add a new day to your holidays table. The inputs include: Currency, Location, Holiday Name, Observable, Start Date, Expiry Date, Fixed Date, Based on, Day, Day Name, Month, Day and Month.
      • [custom table name] - to use the report to update data in a row in a custom table. You can also use Actions to update and delete rows in a custom table.

Examples

To add a row to a custom table

This applies to a custom table (also called a workflow table) created in "Quantum_20xx.x_Database_Installation_Instructions.pdf".

  1. Open Configuration > Report.
  2. In the menu select New and the New Actions dialog box appears.
  3. Set Action Name to Create > Create [custom table name]. An action is available for each custom table.
  4. In Name, enter the name the user will see.
  5. Set Display In.
  6. Select OK and continue to create the report.

The action creates a row identified by the NK fields (or PK fields if no NK fields). If there are no NK fields, the PK fields are output settings. No action is available for tables without PK or NK fields.

NOTE: You can also use actions to update and delete rows in a custom table.

To delete rows from a header table or detail table

This applies to a custom table (also called a workflow table) created in "Quantum_20xx.x_Database_Installation_Instructions.pdf". Also available in workflow pages.

  1. Open Configuration > Report.
  2. In the menu select New and the New Actions dialog box appears.
  3. In the Delete folder, select the name. The name of the action relating to the parts of a custom table is in the format "Delete <title of custom table>".
  4. Select OK, finish configuring the report and save.
  5. Run the report. The action deletes the row identified by the NK fields (or PK fields if no NK fields).

The action works only if there are NK or PK fields. Deleting the header table deletes the details table too. If you've mapped the NK or PK fields to a header and detail table input, the system only deletes the detail table entries. If mapped to a header table input only, then the system deletes the header row and all rows from the details table for that record.

To update rows in a header table or detail table

This applies to a custom table (also called a workflow table) created in "Quantum_20xx.x_Database_Installation_Instructions.pdf". Also available in workflow pages.

  1. Open Configuration > Report.
  2. In the menu select New and the New Actions dialog box appears.
  3. In the Delete folder, select the name. The name of the action relating to the parts of a custom table is in the format "Update <title of custom table>".
  4. Select OK, finish configuring the report, and save.
  5. Run the report. The action updates the row identified by the NK fields (or PK fields if no NK fields).

The action works only if there are NK or PK fields provided for update. Choose either a PK or NK field and all fields, except SYS, are available as input. All NK fields are available, but PK fields are only available when there are no NK fields. All NK fields are available as recommended settings, but PK fields are only available if there are no NK fields.

To add, update, or delete holidays

The Save Holiday action is a way of uploading a large list of holidays.

  1. Open Configuration > Report.
  2. In the menu select New and the New Actions dialog box appears.
  3. In the Update folder, select Save Holiday.
  4. In the Inputs area, select Add and choose the settings: currency, holiday name, location, etc.
  5. Add expressions that set up the conditions to Add, Update, or Delete either singly or in bulk.
  6. Select OK, finish configuring the report, and save.

Input parameters:

  • "add" - Adds a new record without checking the existing entries. 
  • "update" - Based on the settings Holiday Name and Fixed Date, finds the existing record and replace it with a new entry.
  • "delete" - Deletes the record that matches the Holiday Name and Fixed Date provided in the input.

Workflow action that calls on a deal’s action

You can set up a workflow page or workflow report to call on actions that are already in a particular type of deal. For example, the actions Fill or Mark as Filled that are already associated with FX Request deals.

In the Select Action Name dialog box there is a new Process folder, with sub-folders for each transaction type and supported static data item, including Variant pages, and for any Workflow reports that have multi-select actions.

The actions available include, for example:

  • the Dealing > FX actions Drawdown, Predeliver or Extend, Rate Fix, Rollback
  • the Dealing > FX Request actions Fill and Mark as Filled
  • deal customization actions for those deal types currently supporting them (Cross Currency and Interest Rate Swaps, Money Market and Securities)
  • Create, Update, and Delete actions for some deal types and Static Data (Currency Options, Cross Currency and Interest Rate Swaps, Money Market and Security deals, Counterparty, Entity, and User)
  • Workflow Templates (reports only), enabling users to apply a template to the report results when output to file

    • NOTE: All existing templates are available in the Download folder and the file can be downloaded in a browser, in either the output format set for the template if Convert to PDF is No or PDF format if Convert to PDF is Yes.

  • Workflow Components and Workflow Component Problems (reports only), enabling users to report on problems with any Workflow page, report, template, action, function, parser or event
  • multi-select actions configured on Workflow reports (using a toolbar button), which enable you to run the report with specified input values passed into the filters, and execute the action on all results from the report

NOTE: A deal can have more than one action associated with it, and actions appear in the list even if they are disabled - for example, a disabled XML menu action such as Save still appears in the list.

To set up a workflow action that calls on a deal action

Set up an action in a workflow report or workflow page that calls on an action already in a deal. This uses the Process folder in the Select Action Name dialog box.

  1. Open Configuration > Page.
  2. Select Core or Variant and enter the initial details for the page.
  3. In the Workflow menu, select New.
  4. When adding an Action Name, in the Process folder select the name of the page in Quantum that you want to call actions from. This Process folder includes all page names for core and variant pages in the allowed deals, for example, FX deals or FX Request deals.
  5. Set up the input for this action.
  6. Complete the set up.

Similar instructions apply to Configuration > Report.

Example

In this example, if the workflow page has an action called “Mark as Filled”. If the analysis code “additional conditions” is populated with the phrase “mark as filled”, then the system marks the FX Request deal #359195 as filled by deal #7733.

To create a deal that is partially populated

You might regularly use a workflow page to create a certain type of deal, for example your FX deals. Or maybe you import certain deals using JSON. From 2020.2, you can populate fields as you create the deal using either of these methods. For example:

  • If all of your FX deals have the same Counterparty, then you might design a specific workflow page to both create the FX deal and partially populate the deal with the Counterparty name at the same time.
  • Or you might populate fields with the result of an expression, for example the sum of related deals, or the result of calculating a particular rate.

Prior to 2020.2, this functionality was only available when you saved a deal.

  1. Open Configuration > Page.
  2. Select Core or Variant and enter the initial details for the page.
  3. In Core Module, select the deal type, in this example an FX Deal.
  4. In the Workflow menu, select New.
  5. In Action Name find the Update folder and select "Set Context".
  6. In Action Timing select After New.
  7. Set up the input for this action. For example, set the counterparty to always be "[SelectTheCounterparty]".
  8. Complete the set up for the page.

Each time you use the workflow page to create a new FX deal, the deal will already contain the values you set up.

Similar instructions apply to Configuration > Page.

Structures Tab (Workflow Report)

The Structures tab shows one or more optional entry rows that define structures that enable you to include data in spreadsheet form as a data source. The report user can edit such data directly or via a spreadsheet application (as with Bulk Replace and Insert for Deal Map Items).

To configure a structure

Define each structure to make it available for users to select as a data source.

  1. In the main menu, select New and the New Structure dialog box appears.
  2. Each structure's definition has a Name and one or more mandatory columns, each with a display Name, Type (Text, Number, Date, Yes/No), optional Default Value (in tab-delimited form).
  3. Each structure's definition has either the Hidden or Bulk Insert option set, where:
    • Hidden indicates set data that users will not be able to change, while...
    • Bulk Insert enables users to use a spreadsheet application (e.g. Excel) to create and maintain the tabular data, including the column headings.
  4. In the target report, the source will be represented as a Bulk Insert link. Click to open a Bulk Insert dialog box and use to transfer data values to and from a spreadsheet application and see them in report results.

Sources

This tab consists of standalone sources that are not used as report data but for expressions and lookups of filters.
