Currency Options Accounting Rules

Use the Currency Options tab to create and maintain accounting rules for Currency Options.

There must be at least one Accounting Rule entry that matches each type of these deals that your organization uses. If more than one, the most specific matching rule entry takes precedence. For example, entries for Instrument groups take precedence over ALL Instrument Name entries (labeled For any ... in the rules grid). Individual instruments that are members of a group will apply settings defined for that instrument before those for the group, before those for an ALL entry.

You can add, edit, and delete specific rule entries using the tab's grid menu, as follows.

NOTE: You cannot add, edit, nor delete inherited entries.

To add a new rule entry

  1. In the Currency Options tab's grid menu, select New (or select an entry row and in the grid menu, select Copy).
  2. In the dialog box under Matching Criteria select:
    • the Currency Option Type (Vanilla American, Vanilla European, Asian, Barriers, or Double Barriers),
    • one or more Instrument Names, Entity External Names (if offered), and
    • any other customized criteria that may apply to this transaction type.

      NOTE: You cannot specify a combination of criteria already covered by an existing entry.

  3. Under Contingent Accounting Settings optionally select Apply from deal date to expiry date to post contingent Settlement receive and pay amounts calculated from Deal date to Exercise/Lapse/Knock-out date, and set Amount to SETTLEMENT.
  4. Under Accrual Settings select one of the Premium accrual options:
    • Expense Premium - if premium accrual/amortization is not required (the premium will be taken directly to Profit and Loss when a deal is saved), or
    • Defer Premium - to hold the premium on the Balance Sheet until the option is exercised, lapsed, or knocked out (at which time the premium is taken to Profit and Loss in full, using Accrual Processing), or
    • Accrue Premium - to take the premium to the Balance Sheet at deal save and post it to Profit and Loss when Accrual Processing is run.
  5. If you selected Defer Premium or Accrue Premium in Step 4, select the Premium Translation date:
    • POSTING DATE CCY TRANSLATION - to use the relevant FX Rate on the day the accrual entries are posted, or
    • CASHFLOW DATE CCY TRANSLATION - to use the FX Rate on the day the cash flow to be accrued/amortized is posted.

      TIPS:

      • Where the cash flow being accrued is in the future, the POSTING DATE should be used.
      • The accrual postings created by End of Day Accrual Processing are translated to base using the Accrual Rate Type setting for the relevant General Ledger record.
  6. If the Currency Option Type is a Barrier and you selected Accrue Premium in Step 4, optionally select Apply Rebate Accrual to determine how the rebate amount should be treated*, and select the Rebate Accrual Trans Date (the options are as for Step 5).

    *TIP: UP & IN and UP & OUT Currency Options only.

  7. Under Revaluation Settings, select Revalue deals* if you want to create revaluation postings, and as required, select:
    • Revaluation Method (as available for the selected Currency Option Type),
    • After Knock In Use method (applies to Barrier options only),
    • Revaluation Currency (BASE, NON-BASE, or PREMIUM),
    • Risk Free Rate Format (YIELD or ZERO curve),
    • Posting Method (REALIZED or UNREALIZED),
    • Interest Rate* and Interest Rate Type*, and
    • Volatility Rate* and Volatility Rate Type*.

      * Mandatory settings when Hedge Accounting applies.

  8. Click Save. The new rule entry is added to the grid.

To edit an existing rule entry

  1. Click an entry value in the grid (or select the entry and in the grid menu, select Edit).
  2. In the Edit window, edit the settings as required (see above), and click OK.
  3. Save your changes.

To delete an existing entry

  1. Select the entry in the grid and in the grid menu, select Delete.
  2. In the dialog box, click OK to confirm.
