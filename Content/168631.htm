Issuer Instrument Rate

NOTE: The combination of Rate Name, Rate Type, Currency, and Date uniquely identifies this type of rate record in the system, and is displayed above the Rate Header.

Use the Issuer Instrument Rate page to store and update the market value of securities, to revalue positions based on either Security Identifier OR the Issuer and Instrument in addition to the Maturity date and (optionally) Coupon value, as below.

  • Security Identifier

    In this case, other identifying values (Primary Identifier, Maturity, and Coupon) are populated from the associated Security Identifier record, and cannot be changed. If the associated Instrument is non-modifiable, this also applies to the Instrument and Issuer values; otherwise they are left blank. To make all cells available again, clear (blank out) the Security Identifier.

-or-

  • Issuer and Instrument, in addition to the Maturity date, and (optionally) the Coupon value

    In this case:

    • If a Security Identifier record exists that matches with those settings, the Security Identifier and Primary Identifier values are populated from that record and become read-only.
    • If no matching Security Identifier record is found, the Security Identifier and Primary Identifier fields are set to blanks and read-only.
    • To make all cells available again, blank out the Instrument.

When revaluing Security deals, the system gets the market value of the instruments from this Issuer/Instrument rates table.

NOTE:

  • When you update a rate record that involves an Instrument, make sure that it includes all instruments that will require revaluation. When the system finds an Issuer/Instrument record with the correct rate date, it searches for a rate for the specified Instrument Type, and cannot proceed if this is missing. See Updating an Existing Issuer/Instrument Rate Record below.
  • If you create an Issuer/Instrument record and a Yield record using the same combination of Rate Name, Rate Type, Currency, and Date, the Issuer/Instrument curve will always be used in preference to the Yield curve.

Rates area, Issuer/Instrument Rate

Use the grid to enter and/or change Issuer/Instrument Rate entry values.

To add a new rate entry from scratch:

  1. In the Actions menu, select New. The New Rate dialog box appears.
  2. Select the Curve type, enter the other values (see below) and click OK. The new entry is added to the grid.

To add a new rate entry by copying an existing one:

  1. Select the row you want to copy. In the Actions menu, select Copy. The Copy Rate dialog box appears.
  2. Select a new Curve type, change other values as required (see below), and click OK. The new entry is added to the grid.

To edit an existing rate entry:

  1. Select the grid row you want to edit. In the Actions menu, select Edit. The Edit Rate dialog box appears.
  2. Change values as required (see below), and click OK. Your changes are shown in the grid.

To remove a rate entry:

  1. Select the grid row you want to remove. In the Actions menu, select Delete. The Confirm Delete dialog box appears.
  2. Click OK. The entry is removed from the grid.

Field Descriptions

Security Identifier (not available if you have selected an Issuer and Instrument) - the Security Identifier you want to load this rate record against. When selected, other identifying values in that row are then populated from the associated Security Identifier record, and cannot be changed. This applies to Primary Identifier, Maturity, and Coupon - and, if the associated Instrument is non-modifiable, to Instrument and Issuer also (otherwise, those two fields are left blank). To make all cells available again, set the Security Identifier to blanks.

Primary Identifier (read-only) - the Primary Identifier value in the specified Security Identifier record. The type of identifier (CUSIP, ISIN, or Other) is also specified in the Security Identifier record.

Issuer (not available If you have selected a Security Identifier) - the Issuer for the specific rate record. (Select this before selecting an Instrument.) The drop-down list shows all transaction-level counterparties that have the Issuer/Acceptor Category specified (in Counterparty Setup). The issuer can be a single counterparty or ALL, but you cannot select a group-level issuer.

Instrument (not available If you have selected a Security Identifieror have not selected an Issuer) - the Instrument for the specific rate record. The drop-down list shows Instruments associated with the specified Issuer. The instrument can be a single instrument or a group of instruments, but you cannot select ALL.

When you select an Issuer and instrument:

  • If a Security Identifier record exists for that Instrument and Issuer, the Maturity, Coupon, Security Identifier, and Primary Identifier values are populated from that record, and become read-only.
  • If no matching Security Identifier record is found, the Security Identifier and Primary Identifier fields are set to blanks and read-only.

Maturity (read-only) - the maturity date from the selected Instrument and/or Security Identifier record.

Coupon (read-only) - the coupon rate for the selected instrument and/or Security Identifier, if applicable.

Rate Input (Offer/Bid) - the interest rate, entered either as a spread (Bid rate, a slash (/), then the Offer rate) or as a single rate without a slash. You can enter a positive or a negative number (which allows you to use a negative curve in Revaluation Analysis).

Bank Offer (read-only) - the rate you entered to the left of the slash in Rate Input, or the single actual rate.

Bank Bid (read-only) - the rate you entered to the right of the slash in Rate Input, or the single actual rate.

Basis (optional) - the timing basis the system will use to calculate the interest rate for the selected instrument and/or Security Identifier: CASH, QUARTERLY, SEMI ANNUAL, and ANNUAL. For Zero Rate or Discount Factor curves, the mandatory value is ANNUAL.

High / Low / Close - the highest, lowest, and closing rates at which the rate is traded on a specific day.

Creating a New Issuer/Instrument Rate Record

New record vs updated record

When entering an entirely new Issuer/Instrument rate record, follow the steps described below, ensuring that the new record includes every instrument that will require a rate record.

TIP: When updating rates for an existing record, follow the steps described in Updating or Editing an Issuer/Instrument Record below. Whether you are changing one or more rates, or adding and deleting Instrument Types, it is best to copy and amend an existing record. This ensures that no instruments are accidentally omitted.

To create a new record

  1. In the Issuer/Instrument Rate page's Rate Header area, select a unique combination of Rate Name, Rate Type, Currency, and Date for the curve.

    The rate record is 'for' the date specified in the Date field. This is the date for which the rates were quoted, and is independent of the date the system enters in the Entry Date field.

  2. Do one of the following:
    • If you are loading the rate against a Security Identifier, go to Step 5.
    • If you are loading the rate against a specific Instrument, continue with Step 3.
  3. On the Rates tab, in a cell in the Issuer column, select an issuer from the drop-down list.

    This list shows all transaction-level counterparties that have the Issuer/Acceptor Category specified (in Counterparty Setup). The issuer can be a single counterparty or ALL. You cannot select a group-level issuer here.

  4. In the adjacent Instrument cell, select an instrument from the drop-down list.

    This list shows all instruments associated with the selected Issuer. The instrument can be a single instrument or a group of instruments. You cannot select ALL here.

    When you select an Issuer and Instrument:

    • If a Security Identifier record exists for that Instrument and Issuer, the Maturity, Coupon, Security Identifier, and Primary Identifier values are populated from that record, and become read-only.
    • If no matching Security Identifier record is found, the Security Identifier and Primary Identifier fields are set to blanks and read-only.

    Go to Step 6.

  5. Alternatively, in the Rates grid, in a cell in the Security Identifier column, select a Security Identifier from the drop-down list.

    Other identifying values in that row are then populated from the associated Security Identifier record, and cannot be changed. This applies to Primary Identifier, Maturity, and Coupon - and, if the associated Instrument is non-modifiable, to Instrument and Issuer also (otherwise, those two fields are left blank). To make all cells available again, clear (blank out) the Security Identifier.

  6. Do one of the following.
    • Record the price for the reference or instrument in the Rate Input (Offer/Bid) column.

    -or-

    • Link the price of the reference or instrument to a benchmark in the Link To column. See Linking below.

    Whichever method you choose, make sure that a price is recorded for every reference or instrument that is listed.

    TIP: The Rate Input (Offer/Bid) and Spread (Offer/Bid) columns support the Offer/Bid format. If you enter a single rate in these columns, Quantum applies it to both the Offer and Bid columns.

  7. You can record a High, Low, and Close price for each instrument. This is optional, as the system does not use these values.
  8. Enter any other details that are required.

    See Rates area above for details of all columns on this tab.

    NOTE:

    • List one Security Identifier or Issuer/Instrument combination per row. If necessary, use the next row to select the same issuer with a different instrument, or vice versa.
    • When creating or updating an Issuer/Instrument record, it is important that you include all instruments or references that could require revaluation.
  9. Save the record.

    A read-only Entry Date is automatically entered, giving the date the record was saved.

  • Position Query - to use the main revaluation tool in Quantum

Updating an Existing Issuer/Instrument Rate Record

New record vs updated record

The following steps describe how to update rates by using an existing record. To enter an entirely new Issuer/Instrument rate record, see Creating a New Issuer/Instrument Rate Record above.

NOTE: Whether you are changing one or more rates, or adding and deleting Instrument Types and/or Security Identifiers, it is always best to copy and amend an existing record. This ensures that no instruments or Security Identifiers are accidentally omitted.

To update a record involving Instrument Types and/or Security Identifiers:

  1. Copy the previous version of the record that is to change.
  2. Amend the rate on the copied record (if this is the reason for the update).
  3. Add or delete instrument types and/or Security Identifiers as required, always making sure that there is a rate in place for every instrument or Security Identifier that will require one.
  4. Save the updated record.

NOTE:

  • When the system searches for an Issuer/Instrument rate record, it first finds the record with the correct Rate Date. Within that, it finds the rate for the specified Instrument Type or Security Identifier. It cannot revalue that instrument or Security Identifier if the rate is missing.
  • If a rate record is updated as described here, even instruments and/or Security Identifiers that do not have a new rate are included. They are copied to each successive record with their existing rates, so there is always a valid rate for all instruments and/or Security Identifiers.
  • This is less important if a rate record does not include an Instrument Type or Security Identifier. In that case, if for example a six-month rate is missing from the selected record, the system can interpolate using the three-month and nine-month rates from the selected rate record.

To edit a record:

TIP: You may need to amend a saved rate record - for example, if an incorrect rate has been entered manually.

  1. Click the button and select the record from the list.
  2. Make the required changes.
  3. To add, change, copy, or delete entries in the Rates grid, use the Actions menu.
  4. Save the record.
