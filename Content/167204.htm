About Deal Sets

Deal Sets are used for two functions, depending on the Mode option or options selected on the underlying Deal Set instrument.

  • Standard - Deal Sets based on an instrument with this selected are accessed via the Dealing > Deal Set page.
  • Forecast - Deal Sets based on an instrument with this selected are accessed via the Cash Management > Cash Forecast page.

Deal Sets based on an instrument with both options selected can be accessed on either page, and additions and changes made to a record on one page are reflected on the other.

Depending on the mode, deal sets enable you to:

  • group multiple deals together, and
  • enter, maintain, generate, convert, or obtain an overview of, multiple related cash flows (actual or projected).

    For example, enter Projected cash flows to forecast future cash flows, and later (optionally) change them to Actual cash flows.

The functionality is flexible in that you can link any combination of deals together (provided they do not already belong to a deal set). For further information on the relationship between Deal Sets and deals, see Deal Set / Deal Relationship below.

Once you have created your deal sets, you can:

  • monitor and report on their deals as a group, using deal-related queries - for example, Position;
  • use CashXplorer to report and scenario-build, allowing you to generate predictable forecasts and to capture and manage multiple sets of related deals or forecast cash flows;
  • link deals to existing deal sets using Quantum, the Importer, or an API, and restrict updates to one of these Maintenance Modes - see Deal Update Restrictions);
  • limit the use of a Deal Set to specific Transaction Types, and to specific Instruments within those Transaction Types (see Supported Transaction Types and Excluded Transaction Types below); and
  • optionally view Deal Set numbers and names for relevant deal actions and flows in Deal Approvals and Settlements Processing;

TIP: Resource Security can affect the information displayed for deals in a Deal Set.

Implementing Deal Sets

Use the Deal Set page to view, edit, and create deal sets, which group related deals for supported Transaction Types (see below). You can then run queries on these deal sets, or perform reporting and scenario-building within CashXplorer to generate predictable forecasts.

You can also use the Cash Forecast page to maintain Deal Sets. Cash Forecast records are Deal Set records. Additions and changes made on that page are reflected on the Deal Set page, and vice versa. The main difference is that the Cash Forecast page provides Convert and Recalculate functions for the cash flows.

Deal Set / Deal Relationship

A deal may be linked to only one Deal Set; however, a Deal Set can have from zero to many deals linked to it.

Update Restrictions

You can use Quantum, the Importer, or an API to create deal sets and the deals that link to them. You can set up Deal Set Rules to restrict updates to only one of these Maintenance Modes.

For more information, see Deal Set Update Restrictions (under Deal Set Rules)..

Supported Transaction Types

You can use deal sets for the following Transaction Types:

  • Actual Cash Flows (AC)
  • Bank Account Adjustment (AA)
  • Bank Account Interest (AI)
  • Bank Account Transfers (AT)
  • Currency Options (CO)
  • Currency Swaps (SC)
  • Foreign Exchange Deals (FX)
  • Forward Rate Agreement Deals (FR)
  • Futures Deals (FU)
  • G/L Journals (GL)
  • Initial Margin (IM)
  • Interest Rate Options (IO)
  • Interest Rate Swaps (SI)
  • Letters of Credit (LC)
  • Money Market Deals (SE)
  • Projected Cash Flows (PC)
  • Repurchase Agreement (RA)
  • Stock Account Transfers (ST)

Excluded Transaction Types

You cannot use Deal Sets for the following Transaction Types:

  • Bank Flows (BF)
  • Concentration Flows (CF)
  • External Cash Flows (EC)
  • Hedge Relationship (HR)
  • Intraday Bank Flows (IB)
  • Unit Deals (UN)
  • Unit Holdings (UNH)
