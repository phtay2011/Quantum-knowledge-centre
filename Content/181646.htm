Custom Modules

(First introduced in Release 6.1.7 SP, October 2015.)

Custom modules can now be set up to control who has access to scheduled jobs, commands, and custom Crystal Reports from the site menu.

Enabling Custom Modules

To be able to see the Custom Module page, or to set up Custom Modules, users will need to have Module Security permissions for Custom Modules.

The available permissions are Inquire, Add, Change and Delete.

With the necessary permissions, the menu option for Custom Modules will appear in the Configuration menu in Quantum.

Custom Module Page

The Custom Module page identifies the Type of Custom Module and controls where commands and reports appear in the site menu.

The Name of the Custom Module is used as the Program/File Name in Quantum, and is restricted to 8 characters.

The Type field defines the following types of modules that can be created using the Custom Module page:

  • Report - to create and manage custom site report modules.
  • Job - to create and manage FIS Scheduler jobs.
  • Command - to create and run server-side commands.

The Area field defines which functional area of Quantum the Custom Module will appear in (for example, Reporting, Processing, and so on).

In the case of Reports and Commands, this will define where the Custom Module appears in the site menu. For Jobs, the Area description appears in the Scheduled Jobs page only.

The Description is what is used as the Module in Module Security.

Module Security for Individual Custom Modules

Once created via the Custom Module page, each custom module will appear separately in Module Security with different permissions available depending on the Type:

  • Report - sole permission = Inquire
  • Job - permissions are View, Run and Manage
  • Command - sole permission = Run

NOTE: A user can have permissions for individual custom modules without having permission for Custom Modules generally, and thereby access to the Custom Module page.
