Base Deals tab, Process Fees page

Use this tab as follows.

To generate base deal amounts and supporting data for the next fees period:

RECOMMENDATION: Do this daily, as the deal value information in the Fee Position History table is maintained on a daily basis. Normally, no dates should be skipped between successive Get Base Details runs.

  1. Click Get Base Details.
  2. In the Generate Base Deals dialog box, select or enter the Position Report and Accruals Report set up for Fees processing and the As At Date to be used for those report runs; accept the default Start Date and End Date of the period (enter new dates only if necessary); then click OK.

    The Base Deals grid lists the applicable deals, giving key data and fees-related values from the resulting Fee Deal Position History table.

To test a Fee Base Template against the current Base Deals:

  1. Under Evaluate, select Use Base Template and select the Base Template you want to test.
  2. Under Filters, select the Position Date and click Run Query.

    The Base Deals grid lists the qualifying deals used to calculate the Fee Amount and Base Amount. For a Facility fee, read-only fields also show the Facility name and total notional amount.

    TIPS: The grid will be empty for a Manual fee or if there are no qualifying deals. For an Unutilized Facility fee, Fee and Base amounts can show when no deals are listed (because the Facility is not being utilized).

To filter the current Base Deals:

  1. Under Evaluate, select User Defined.
  2. Under Filters, select the Position Date and select specific values for your required filter parameters (Transaction Type, Instrument, and so on), leaving others set to ALL.
  3. Click Run Query.

    The Base Deals grid displays the filtered qualifying deals, and the Fee Amount and Base Amount are changed accordingly; also the Facility-related values if applicable).
