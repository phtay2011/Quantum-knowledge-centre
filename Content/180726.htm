Process and data flow - Bank Reconciliation

This diagram gives an overview of the main Quantum Bank Reconciliation processes and data flow. It illustrates how Bank Reconciliation prepares data for the CashXplorer module.

Data Load

This process imports Bank data into Quantum, using the Bank Import Service and the Data Load component function. Bank data* includes, bank balances, and any external (non-Quantum) cash flows.

*TIP: Either confirmed prior-day bank data (for Prior Day Matching) OR unconfirmed Intraday bank data (for Intraday Matching).

For more details, see Importing Cash Management data.

Prior Day and Intraday Matching

Prior Day Matching does the actual reconciliation of a bank account, by matching confirmed cash flow and bank flow amounts and data for earlier, completed and closed-off days.

Intraday Matching enables you to provisionally match unconfirmed cash flow and bank-reported data for the current and still-open day, for facilitating the subsequent Prior Day Matching and actual reconciliation.

The rules and criteria for matching are defined in one or more Match scripts that are linked to the bank account. A script can specify one or more passes through the data, where the criteria for later passes are always wider, thus producing matches with lower levels of certainty. These levels, from highest to lowest, are Matched, Probable, and Possible.

There are two main types of Matching script.

  • A Detail-match script compares Intraday and/or Prior Day cash flows to bank flows, looking for matching data, which is then labeled as either matched (Prior Day)or temporarily matched (Intraday), in each case at a specified level of certainty. Adjustments can be automatically generated where required for matches within specified criteria.
  • A Balance match script compares only Prior Day cash flow balances with bank-reported ledger balances. A single adjustment can be automatically generated if required, depending on specified criteria.

You can link each bank account to a script of each type (or to a script chain- see below).

You can run the Matching process in the following ways:

  • automatically (following successful data loading),
  • manually for all unmatched data, or...
  • manually for selected data.

This matching function enables you to select one or more cash flows, and then find the best matching bank flow/s, applying the script rules for that data only - or vice versa.

For more details, see Matching Cashflows with Bank Flows

Edit Match Scripts/Chains

Use this function to set up and maintain your scripts. You can also chain scripts together, so they can run in sequence.

For more details, see Automatic Matching using Scripts.

Mark as Reconciled

(Prior Day Matching only.)

This process checks that all cash flows for the specified period are matched, either to loaded bank flows (if any) and/or ledger balances. If so, the bank account is marked as Reconciled up to the relevant date.

Although you can proceed without fully reconciled data, your CashXplorer will be more accurate if the previous day's cash flows are fully reconciled first.

For more details, see Mark as Reconciled.
