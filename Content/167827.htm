Horizons

Horizon Definitions are an important aspect of CashXplorer as they define the time periods over which cash flows are calculated and forecast.

There are two types of Horizon:

  • user-defined and
  • system-defined.

User-defined Horizons

These are the time periods you select for the cash flow calculations and forecasts. They can range from one day through to several years.

You can create user-defined Horizons in any order, but they will be automatically re-ordered in the tree view - for example Days followed by Weeks, Fortnights, Months, and so on.

User-defined Horizons are calculated from the start date of the Data Version.

System-defined Horizons

These are the first Horizons to appear in a View. They cover the period up to and including the Previous Day. You can select whether your query creates system-defined Horizons using the Included Flows area in the Filters tab.

System-defined Horizons are visible only with a Forecast View.

The available System-defined Horizons are:

  • Previous Day Reconciled Cashflow
  • Unreconciled Cashflows Prior to Start
  • Unreconciled Bankflows Prior to Start
  • Projected Cashflows Prior to Start
  • Unreconciled External Cashflows Prior to Start

Horizon Start and End Dates

The start and end dates for Horizons are calculated by the system and follow these basic rules:

  • the first user-defined Horizon starts from the start date of the query;
  • second and subsequent user-defined Horizons start one day after the previous Horizon; and
  • system-defined Gap Horizons are created automatically to display the cash flows between user-defined Horizons of varying Date Range Types.

    For example, if a View displays 3 one-day Horizons starting from Monday, followed by 3 one-week Horizons, then Thursday and Friday of the first week will be missing and cause a two-day gap in the cash flow display.

Procedure Summaries

To create a new Horizon Definition

  1. On the Horizons tab, select New from the grid menu.
  2. In the New Horizon dialog box, enter the horizon details (see Field Descriptions below).
  3. Click OK.

    The new Horizon is added to the grid

  4. Save your change.

To edit an existing Horizon Definition

  1. In the Horizons tab's grid, click on a data item in the Horizon definition row you want to edit (or select the row and in the grid menu, select Edit).
  2. In the Edit Horizon dialog box, change the horizon details as required (see Field Descriptions below).
  3. Click OK.

    Your changes show in the grid row.

  4. Save your change.

To remove a Horizon Definition

  1. In the Horizons tab's grid, click the x button to the right of the Horizon definition row entry you want to remove.
  2. Save your change.

Field Descriptions

Occurrences

This item has two parts:

  • a lookup (on the right) giving the type of occurrence - select either Remainder or a time unit (Days, Weeks, Fortnights, Months, Quarters, Semi Annual, or Years), and
  • the number of occurrences (not available if the type is Remainder) - enter the number of Horizons to be calculated of the selected type. You can select up to 400 Horizons.

NOTES:

  • Only one of each type of occurrence may be used in a Horizon definition.
  • Remainder is a limited Horizon that collects all the flows from the end of the last user-defined Horizon, or - if there is no user-defined Horizon - from the start date of the query. The end date is the maximum end date of the current database.
  • When Months is selected, you must also enter an appropriate Start Date (see below).

Start Date

(Available when the Occurrences type is Months.)

Enter the day of the month (1 to 31).

NOTE: If the value selected is greater than 28, the date may exceed the number of days in some months. In these instances, the occurrence will be moved to the last day of the month

Caption

The caption text that will appear above each Horizon. You can enter up to 40 characters, which may include the following variables:

  • @Start - inserts the Horizon start date,
  • @End - inserts the Horizon end date, or
  • @NewLine - inserts a line break).

There can be only one @Startand one @Endvariable per caption, but you can use any number of @NewLinevariables.

TIP: To include an actual @ symbol in your caption, follow it with a space.

Enter the caption to appear above the Horizons.

Gap Caption

(Available when the Occurrences type is NOTRemainder.)

Sometimes there may be a gap between the end of one user-defined Horizon and the start of the next. For example, if your View displays three one-day Horizons starting from Monday, followed by 3 one-week Horizons (which start on Monday), then Thursday and Friday of the first week will not be displayed and cause a four-day gap.

In such cases, CashXplorer will displays a Gap Horizon to cover the cash flows for the gap period.

Use the Gap Caption field to enter the caption to appear above such Gap Horizons. The same variables and rules apply as for the Horizon Caption (see above)
