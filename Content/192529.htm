Bank Transaction Rule Set

Use the Bank Transaction Rule Set page to create and maintain Bank Statement Import data manipulation details.

This includes uploading any existing Transaction Rule file rules into Rule Sets in Quantum Six,.

A Rule Set can then be applied to the Import process by including a TransactionRuleSet parameter in the BKImport.ini file for your site. For example:

TransactionRuleSet = XYZTransRule

For data manipulation, your BKImport.ini file could contain a combination of IncludeFile parameters and a single TransactionRuleSet parameter. Transaction rules in Included Files will be applied first.

However, if you have existing Legacy syntax Transaction Rule files, it is recommended that you upload them into one Bank Transaction Rule Set record in Quantum Six and replace the IncludeFile parameter with a single TransactionRuleSet parameter specifying the Rule Set name. You will then be able to keep all your rules in one place and use only one method of data manipulation. If you do this, note that the Transaction Rule Set page currently does not support:

  • Iron Python scripting,
  • Summarizing Clauses, and
  • specifying a contiguous range of accounts using a TO operator - for example, 123400 TO 123499 (you must select each account).

Data item descriptions

Name - the unique value identifying the record in the system,.

Description - a longer text describing the Rule Set.

Comments - any optional text.

Bank Transaction Rule grid

Use the standard grid menu functions to Add, Edit, Copy, Deactivate, or Delete individual rule entries (or click on an existing grid row value to edit it). You can also use the grid menu's Upload Rules function to add one or more rule entries from an existing Transaction Rule file (Legacy syntax only) into the current Rule Set record.

In each rule entry:

  • Order* indicates the order in which the rule is applied within the rule set,
  • Name is a text string identifying the entry within the rule set,
  • Bank Account gives the selected accounts the rule applies to (or ALL),
  • Additional Conditions* is a logical expression giving conditions the applicable data must meet,
  • Assignments* sets bank statement and transaction data values to values that you specify (when the conditions are met), and
  • In Use indicates whether the entry is currently active or not.

*NOTES:

  • If you change the Order of an existing rule, the Order number on all other rules will automatically update.
  • The Additional Conditions and Assignments text boxes must respectively contain a valid Condition Clause and Assignments Clause in Legacy syntax.
  • Where rule entries specify the same Bank Account, the most specific rule will be applied.
  • All changes to Bank Transaction Rule Set records are audited.
