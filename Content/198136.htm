Areas, Workflow Template page

This topic covers the different grids seen in the Workflow Template page and describes how to use them.

Headers

Name - enter the name of the template

Output Format - choose from: Text (simple text output), HTML (html output), and XML (an XML schema)

Purpose - choose from: Any and EFT. Templates with Purpose = Any are available on the Workflow Report page in Actions > Open > Dialog.

Inputs Grid (Workflow Template)

The Inputs grid (Template tab) shows a list of inputs to the template and their type, which depend on the Sources specified.

To enter an input

  1. Select a Name for the input. The name appears in the template.
  2. Write a longer Description for your information.
  3. Select a Type: Text, Date, Yes/No, Amount, or Number.
  4. Add a Default value.

Expressions Grid (Workflow Template)

The Expressions grid (Template tab) shows a list of expressions you have built and that are available to this template.

To enter an expression

  1. Select a Name for the expression. The name appears in the template.
  2. Decide whether the Type is Text, Date, Yes/No, Amount, or Number.
  3. Add the mathematics for the Expression. For more information, see Using Expression Builder in Calculated Columns.

You can use an expression in text where the string is blank (empty string).

Validations grid (Workflow Template)

In the grid menu, select New to add a new condition, Edit to change an existing one, Delete to remove one. Add validations in the order you want them to occur.

The New Validation or Edit Validation dialog box has the following items.

  • Condition - Click the Add Condition link to open Expression Builder so you can enter any validation conditions. These can include your Template Inputs, Expressions, and Sources.
  • Message - The message text to be displayed when the Condition is met.
  • Type - Select Terminate, Block or Warn, where:
    • Terminate applies at the data set level, validating all records in the data set at the same time, terminating execution of the template if any record matches the validation condition, outputting no records, and returning a validation error for each one;
    • Block applies to single records, validating one record at a time, terminating execution of the template and returning a validation error only for each record that matches the validation condition, and outputting those records that pass validation; while
    • Warn applies to single records, does not prevent execution of the template, and just returns a warning message for each record that matches the validation condition.
    • NOTE: Terminate is excluded from EW templates with purpose of EFT.

Text box (Workflow Template)

Underneath the Sort box is the place to enter template content that refers to the inputs, expressions and sources.

Some things to note:

  • Choosing elements from the SeedXMLDataTemp_EFTandFlows_InclSource.xml in the Input area, allows you to access any generic XML EFT Data to support ISO20022 messages.

Sources grid (Workflow)

Use the Sources area to provide data for use in the template.

In the grid menu, select New to add a new source, Edit to change an existing one, Delete to remove one.

The New Source or Edit Source dialog box has the following items.

  • All Data / Aggregated - Select:
    • All Data - to show only Columns and Settings from the specified Source, or
    • Aggregated - to show fields to Group By, Columns to aggregate by, and Settings from the specified Source.
  • Source - The database record type used for the template.
  • Name - A unique name for the source (defaults to the Source value).
  • Columns tab, Columns grid - Click Add and select columns from specified Source that you want to be available in your template, including - for Dealing, Static, Rates, and Reporting sources - linked items down to any level.
  • Settings tab and grid - Settings to apply to the source data, including a mathematical operator and expression.

    TIP: You will get an error message if you have not included a required (key) data item in the Settings.

  • Columns tab, Group By grid (only when Aggregated is selected) - select one or more fields to group by (or none).

Preview Tab (Workflow Template)

The Preview tab enables you to enter sample inputs into the Sample Data area, and see the resulting output in the text box below that.

If you have defined validation conditions that the sample data meets, the read-only Validation Messages grid appears, displaying the applicable message type and text, in each case.

For more information, see Using the Template.

Schema Box (Workflow Template)

The schema box appears when you select Output Format = XML.

To add or edit schema

  1. In the main menu select Edit Schema. The Edit dialog box opens.
  2. Add or upload the XML schema, or find the existing schema in the list. You can edit (or export) workflow templates in any of these formats (CSV, Excel, JSON, QXT, PDF, and XML where valid).
  3. Click OK to validate and save the schema in the database. An error message displays if not valid.

For more information, see Using the Template.
