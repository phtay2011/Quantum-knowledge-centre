Rate Name (Analytics)

Use the Rate Name page to maintain unique Rate Names for all rates used in Analytics.

To add a new record (from scratch) - click New, enter a new Code value, fill in the remaining data fields (see below), and click Save.

To list records - click to open the browse window. (Click the Configure link to change browse list properties.)

To add a new record (by copying the current one) - click Copy, enter a new Code value, edit the remaining data fields as required (see below), and click Save.

To change a record's details - select it in the browse list, edit the data fields (except Code*) as required (see below), and click Save.

*NOTE: If you do change the key Code value and Save, the record you accessed is unchanged. If your entered value is:

  • a new value - a new record is created (equivalent to a record Copy), or
  • an existing value - that record is changed.

To delete a record (see Note*)- select it in the browse list and in the page menu, select Delete, then click OK to confirm.

*NOTE: If you delete a Rate Name record:

  • that Rate Name will no longer be available for selection in applicable Rate fields, including Rate or Zero Curve records, and
  • any Analytics Rate or Zero Curve records using that Rate Name will no longer be available for selection for valuation or other purposes.

Field Descriptions

Code - the unique Rate Name code or ID identifying this record in Analytics; a value that:

  • will populate the Name field in Analytics Rate or Zero Curve records (and Rate Name in Analytics Zero Curve record components);
  • if created by static data importation from Quantum - is the concatenated Quantum Rate Name and Rate Type for each combination of those Quantum records available* and
  • cannot be changed on an existing Rate Name record.

    TIP: If you did change a Code value and click Save, the record you accessed will not be affected. If your entered value is:

    • a new value - a new record is created, or
    • an existing value - that record is overwritten.

Name - the mandatory name (up to 25-characters); the name of the curve Analytics refers to when this rate is selected for a deal.

Reference Rate - set to Yes if the rate is a reference rate - that is, an official rate used for rate setting; otherwise set to No.

External Rate Name - the external or published rate name - for example, the Quantum Rate Name for records imported from Quantum*.

External Rate Type (optional) - the type of the external rate; a value with meaning to an external system - for example, the Quantum Rate Type for records imported from Quantum*.

*NOTE: When Rates-related static data is imported from Quantum, the Replication Manager creates one Analytics Rate Name record for each combination of Quantum Rate Name and Rate Type, with the Code value set to the concatenated Quantum key values in each case. For example, if you have three Quantum Rate Names (A, B, and C, say) and two Rate Types (X and Y), then six AnalyticsRate Name records will be created (with Code values AX, AY, BX, BY, CX, and CY).
