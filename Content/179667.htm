Importing data into Quantum

Data can be imported into Quantum as follows:

  • Import deal data and some static data by using the Quantum Public API. Import one file or multiple files.
  • Import static data and rates by using Data Transfer spreadsheets. You can also import rates in correctly formatted files - see Import File Formats and Commands
  • Import Analytics rates via the Analytics Rate Server.
  • Import Bank, Confirmation Match reply, deal, workflow, and static data - either automatically when the relevant Import Service is running, or manually using the corresponding Import Status utility. See Import Services and Utilities.

    NOTE: Static Data here also covers Rules and Rates.

  • Import Counterparty and Entity static data files in supported formats (QXT, QID, or tab-formatted) by using a command in Quantum Dealing page Browse menus - see Import function on Static Data pages.
  • Import structured addresses, set up in Configuration > Address Structure, by using QXT import/export processes.
  • Deal data files in supported formats (QXT, QID, or tab-formatted) can also be imported using a command in Quantum Dealing page Browse menus - see Import function on Dealing pages. Some deals allow you to import
  • Deals added or updated in Analytics will be replicated in the Quantum database by the Analytics Deal Export Server - either automatically or manually, as set in Analytics Service Manager Configuration.

    • TIP: Similarly, deals added or updated in Quantum will be replicated in Analytics by the Integration layer - see the Integration guide (Quantum_20xx.x_Analytics_Integration.pdf).

  • Previously archived data can be restored using the Archive utility with the RESTORE option (see Archiving Quantum Data).
  • Workflow Templates can be imported in QXT format via the Static Data Import Service. Template sample data can be included.
