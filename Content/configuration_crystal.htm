Crystal Mapping page

System Administrators or Database Administrators must use this page to attach either a Crystal Report format supplied with your Quantum system or a user-created custom one, to any Quantum application page that can produce report results.

Crystal Mapping Details

  • View Name - the unique name for the Crystal Report format (this will appear in the drop-down list when printing).
  • Module - select a Quantum application page to link to, or type the executable name in UPPER case.
  • Crystal Report - select a Crystal report name from the lookup (which references the Report directory specified for the system).

    NOTES:

    • Report formats must be in this directory to work within Quantum.
    • If Resource Security is enabled, the Crystal Report name must be unique when used for Static Data reports and Resource Security must be applied to each Table Alias.
  • Report Table - select the database table from the drop-down list.

    TIP: Ensure you select the correct table name where a query can output results to more than one table - for example, a Position reporting (or processing) run creates both a posnrpt table and a pairrpt table.

  • Description - meaningful text describing the report.
