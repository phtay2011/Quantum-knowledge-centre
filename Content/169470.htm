Parametric VaR Reporting in Analytics

The Parametric VaR of a portfolio is the volatility of the portfolio. It is calculated as the portfolio volatility multiplied by the current market value of the portfolio.

Advantages of Parametric VaR:

  • It is easy to implement and understand.
  • Riskmetrics data sets are easily accessible.

Disadvantages of Parametric VaR:

  • Data sets are based on normal distribution.
  • Parametric VaR does not handle option products (non-linear instruments) correctly and tends to underestimate the results.

Use the Analytics Query page to create, maintain, and run Parametric VaR queries.

TIP: To help users locate queries of each type, the names of queries can (and should) include an indication of the type. For example, Parametric VaR query names could include the word Parametric or the abbreviation PVaR, and so on. This also applies to folders that contain queries of only that type.

To find an existing Parametric VaR query:

If you have adopted a naming convention as above, then:

  • On the Queries browse page, search for the standard designation for Parametric VaR queries - for example PVaR.

To create a new Parametric VaR query (from the Queries browse page):

  1. Navigate to the folder (or root) level where you want to locate the new query.
  2. Click New and in the drop down list select Parametric VaR. (Or in the page menu, select New Parametric VaR.)

    A new Query page will open for your new Parametric VaR query.

  3. Configure the required query parameter Defaults and Properties, including a unique Name, the required Definition elements and Expiry Strategy settings; then click Save.

    The query will be saved in the current location.

To create a new Parametric VaR query (from any Query page):

  1. In the page menu, select New Parametric VaR.

    A new Query page will open for your new Parametric VaR query.

  2. Configure the required query parameter Defaults and Properties, including a unique Name, the required Definition elements and Expiry Strategy settings; then click Save.

    The query will be saved in the same location as the query you started from (whatever its type).
