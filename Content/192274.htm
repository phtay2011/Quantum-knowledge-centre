Mortgage Backed Swap Deals

Mortgage Backed Swap (MBS) Adaptiv-specific deals can be added to the Analytics database using the External Portfolio File Upload AND Mortgage Backed Swap Update utilities.

Requirements

MBS deals can be reported on in Quantum Six when all the following requirements are met.

  1. A "placeholder" Interest Rate Swap deal exists in Analytics for each Adaptiv MBS Swap deal.
  2. Using the MBS Deal Mapping page, Note Classes (FRN deal IDs) have been mapped for every one of these Analyticsdeals (all zeros if no FRN deals apply).
  3. Using the External Portfolio File Upload page, an External Portfolio (.AAP) file containing the original Adaptiv MBS data has been uploaded into Quantum Six, with the first (deal ID) part of each Reference field in the uploaded AAP file matching an existing Analytics placeholder deal ID.
  4. CSV files containing additional mortgage data for the MBS deals are available in the location specified in the MbsFolderPath key in your site's AAI.AAIS.exe.config file, and have filenames in the format dealID-description, where:
    • dealID matches an existing Analytics placeholder deal ID, and description is descriptive text, and
    • the filename fully matches a Reference field in the Portfolio (.AAP) file.
  5. The Mortgage Backed Swap Update utility is then run, specifying the uploaded Portfolio file.

    When the MBS Update run is completed, the .AAP file is updated and a file suffixed _old.aap is created in the same folder, representing the .AAP file before it was updated.

  6. Relevant Adaptiv Query Configurations include an External Portfolio file grid entry (on the Assumptions tab) specifying the uploaded Portfolio file.

    When you run a query that specifies that Portfolio file, the updated file is sent to the Adaptiv Analytics application along with the report results, enabling it to map the mortgage cash flow data and Notes information from that file to the relevant MBS deals and calculate the results in each case.
