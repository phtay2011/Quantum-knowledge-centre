/CASHMGMT

/CASHMGMT [RESTORE [reference] ]
    [/DAYS ndays]
    [/WORKSHEET name [yyyy-mm-dd  [version_no] ] ]

Optional. Use with the ARCHIVE command to specify the Cash Management Worksheet data version/s to be archived or restored, where:

  • RESTORE specifies that previously archived Cash Management Data Version information is to be brought back into the database.
  • If RESTORE is absent, the default action applies - that is, the specified Worksheet data is archived and removed from the database.
  • reference (optional with RESTORE only) specifies the Reference Number of a particular previous Archive action to be restored.
  • /DAYSndays (optional) specifies the oldest Worksheet Data Versions you wish to retain in the database.
    • On archive, Data Versions that have a Start Date more than ndays older than the current date will be removed.
    • On restore, Data Versions that have a Start Date ndays old or less will be restored.

    The default number of days is 180.

  • /WORKSHEETname (optional) specifies a particular Cash Management Worksheet name that you want to archive or restore, as applicable.
  • yyyy-mm-dd (optional with /WORKSHEET only) specifies the Worksheet Start Date for archiving or restoring.
    • On archive, Data Versions that have a Start Date older than that date will be removed.
    • On restore, Data Versions that have a Start Date on or after that date will be restored.
  • version_no (optional with /WORKSHEET when yyyy-mm-dd is included only) specifies a particular Worksheet Data Version Number for the specified date.

For examples including this parameter, see /CASHMGMT Examples.
