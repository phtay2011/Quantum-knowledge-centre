Deal Margin Rates (Analytics)

Use the Deal Reval Margin Rate page to enter a yield margin against a specific deal, or deal side in the case of Swap deals.

  • Use the Rate Definition area to enter a Name for a new Analytics Deal Revaluation Margin Rate record, OR to change an existing Rate record name (when you must acknowledge the risk involved in changing the record's identifying value).
  • Use the Rate Dates area to specify an effective Date for a set of rates for the current Rate record.
  • Use the Rates grid to enter and/or change individual rate term values to create the set of rates for that effective Date.

Deal Margins:

  • may be entered directly into Analytics or imported from Quantum using the Rates import tab in Replication Manager - see Integration with Quantum below; and
  • are applied to the rates used in deal valuations when, in the applicable Analytics Valuation Rule record, a Deal Margin Rate name is specified in the Margin Rate Type setting in the matching rule entry for that deal type, instead of a Security Margin or Yield Margin rate. Setting the applicable Valuation Rule entry's Margin Rate Type value to None will produce Deal Valuations minus margins.

Deal Margin valuations are returned through the Pricing, Deal Analyze, and Reporting functions within Analytics. See Deal Revaluation Results, below.

TIPS:

  • The Rate Definition Name alone identifies a unique Deal Reval Margin Rate record.
  • If possible, use Yield Margins instead of Deal Margins, as they are more consistent with Analytics functionality.
  • If Deal Reval Margins from Quantum are required, all parts of each Rate record should be imported from Quantum (Name and all rate entries for all effective Dates).
  • Importation from GTM is not supported.

Rate Definition area

Use this area to:

  • select a Name* for a new Analytics Deal Revaluation Margin Rate record, OR
  • change an existing Rate record name (when you must acknowledge the risk involved in changing the record's identifying value).

The Rate Name alone identifies a unique Deal Reval Margin Rate record in the system.

*NOTE: A Rate Name Record must be created before you can select a rate Name.

To list Rate records - click the page browse button (the top one) to open the browse window. (Click the Configure link to change browse list properties.)

To add a new (empty) Rate record - in the page menu, select New; in the New Rate Definition dialog box enter new Rate Definition values, enter the required settings (see Field Descriptions below); Then click Save, and:

  • use the Rates Date area to specify an effective Date for a set of rates for the Rate record, then
  • use the Rates grid to enter and/or change individual rate term values to create the set of rates for that effective Date.

Each Rate record may have one or more effective dates, each with a set of rate values. The number of these dates is indicated in the badge at top right of the page and the Date value shows the currently selected effective date for the set of rates displayed in the grid.

To view and edit another Rates record - in the main browse list, select the Rates record you want and add, edit, and/or remove complete rate sets and their individual rate entries (grid rows) as required.

To delete a Rate record - in the main browse list, select the Rates record you want to remove; in the Rates menu, select Delete; then click OK to confirm.

Field Descriptions

Name - the name of the required rate record (the Code value of an existing Analytics Rate Name record).

I understand that editing this rate definition will affect all current and historical rate records associated with this rate definition (applies and mandatory only when editing the Rate Name) - you must select this check box to accept the stated risks of such a change.

Rate Dates

Each Rate record may have one or more effective dates, each with a set of rate values.

The Date value shows the currently selected effective date for the set of rates displayed in the grid below. This is the date those rates apply from (up until the next effective date, if any). The default for a new set of rates is Today's date and time.

TIP: The number of these dates is indicated in the badge at top right of the page.

To add a new set of rates (with a new effective Date) to the current Rate record:

  1. In the Rates menu (the middle menu), select New.
  2. In the New Rate Date dialog box, enter the new effective Date.
  3. Click Save.
  4. Add individual Rate term entries to the Rates grid as required.

TIP: You can also add a new set of rates by selecting an existing effective Date set, selecting Copy Rates in the Rates menu, changing the Date and clicking Save, then adding, editing, and/or removing individual rate entries as required.

To view and edit another set of rates in the current Rates record:

  1. Click the Rates browse button (the lower one) and select the effective Date for the rates set you want.
  2. Add, edit, and/or remove single rate term entries in the grid as required.

To remove a set of rates from the current Rates record:

  1. Click the Rates browse button (the lower one) and select the effective Date for the rates set you want to remove.
  2. In the Rates menu, select Delete.
  3. In the Confirm Delete dialog box, click OK to confirm.

    The rate set for that effective Date is removed from the current Rates record.

Rates grid

Use the Rates grid to enter and/or change individual rate values in the set of rates for the currently selected effective Date (for the current Rate record).

To add a single rate term entry to the current set of rates:

  1. In the grid menu, select New*.
  2. In the New Rate dialog box, enter a new Term and Term Type, and enter the remaining values as required (see below).
  3. Click Save.

    The new rate entry is added to the Rates grid.

*TIP: You can also add a new row by selecting an existing one, selecting Copy in the grid menu, changing the values as required, and clicking Save.

To edit a rate term entry in the current set of rates:

  1. Click a cell value in the entry's grid row (or select the row you want to edit and in the grid menu, select Edit).
  2. In the Edit Rate dialog box, change the values as required (see below).
  3. Click Save. Your changes are shown in the Rates grid.

To remove a rate term entry from the current set of rates:

  1. Select the grid row you want to remove.
  2. In the grid menu, select Delete.
  3. In the Confirm Delete dialog box,click OK to confirm. The term definition is removed from the Rates grid.

Field Descriptions

Deal ID - the Analytics deal identification number that the margin will be applied to.

Side (applies to Swap and FX deals only) - specifies whether the margin will apply to the PAY or RECEIVE side, or to both sides. If left blank, the margin will apply to both sides.

Margin - the margin that is to be applied to the deal.

Integration with Quantum

The Analytics Replication Manager (AGRReplMgr.exe) provides an interface for integrating static and deal data from Quantum into Analytics.

Deal integration between Quantum and Analytics must be up to date prior to the importation of Deal Revaluation Margin rates.

After the database has been upgraded, the Deal Margin information from Quantum will need to be integrated to Analytics. Importation of the following data is required.

  1. On the Static Data tab, select Import selected objects, then select:
    • Rate Names,
    • Rate Definition (Deal Margin),
    • Valuation Control Names (Management), and
    • Valuation Control (Management).

      NOTE: The Valuation Control Names (Management) and Valuation Control (Management) records will only be updated to Analytics if new records are added in Quantum. Additions or amendments to existing records in Quantum must be manually updated within Analytics.

  2. On the Rates tab, select Import selected curves, then select Deal Reval Margins.
  3. Click Do Import.
  4. Check the displayed messages to ensure all data has been successfully imported.

    NOTE: Deal Margins will only be imported for deals that exist in Analytics. If a Deal Margin appears to be missing in Analytics, first check that the deal has been correctly imported into Analytics. If the deal does not exist in Analytics, the deal must be imported prior to re-importing the Deal Margin.

Deal Revaluation Results

Deal Margin valuations are returned through the Pricing, Deal Analyze, and Reporting functions within Analytics.

Deal Analyze

Setting the applicable Valuation Rule entry's Margin Rate Type value to None will produce Deal Valuations minus margins.

Reporting

The Rule query in the Valuation area of the Assumptions Definition page must be set to a Valuation Rule record that contains Deal Margin information.

Valuation Marginsmust be checked to calculate Deal Margin revaluations. The Assumption setting provides the flexibility of including or excluding Deal Revaluation Margins at the time of running the query, without the need to adjust Valuation Rule records.

Pricing

Pricing pages do not have a Valuation Rule setting - they use the Site's Valuation Control setting. This is defined during configuration of your site's Analytics database, and defaults to the Default Valuation Rule.

To change the Site setting:

  1. Start the Quantum Configuration Wizard (QuantumConfig.exe) on your Analytics server.
  2. Click Next until you reach the Configure Site Sections page.
  3. Select the section name and click Edit.

    The Edit Site Section dialog is displayed.

  4. On the Analytics tab, click Advanced.
  5. In the Valuation Control field, select the Valuation Rule record to be used for Pricing.
  6. Click OK to return to the Edit Site Section dialog.
  7. Click OK to return to the Configure Site Sections page.
  8. Click Next until you reach the Confirmation dialog.
  9. Review the Summary. If all is correct, select the I have reviewed the above actions check box, and then click Apply.

    A green tick displays, showing that the wizard has successfully completed.

  10. Click Finish.
