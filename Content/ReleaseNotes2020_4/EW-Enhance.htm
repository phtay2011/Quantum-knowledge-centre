Enterprise Workflow enhancements

The following enhancements apply to the Enterprise Workflow (EW).

Diagnose and fix broken workflow reports

Quantum does three things to help you diagnose and fix broken workflow reports.

  • A message on the workflow report's page notifies you that the page is broken. The error message is similar to the following:

    There is a problem retrieving the Source.

  • Scheduler keeps working.

  • To control the impact of potentially large volumes of data causing performance issues on the system, there is a configuration setting in the web.config called WorkflowMaximumRowsPerSource which is set to ALL. This default value will not impose a restriction. Override this default by adding this setting to the Web.AppSettings.config file and specifying the maximum number of rows that can be processed.

    If any source returns more than the configured number of rows, the error message in the workflow report is similar to the following:

    Unable to run report - data source qvw_datasource_audit has more rows than in the system settings.

    For more information about related settings, search for the Knowledge Center topic "Web.AppSettings.config".

Customize "Pricing" variant pages in EW

You can now use the Enterprise Workflow (EW) to specify which Pricing variant pages include data from Pricing core pages. This allows you to customize these pages more heavily. Use the Pricing variant page options Include core data and Publish to core, with behavior mimicking existing behavior on EW dealing variant pages.

Works for both combined and separate Quantum and Analytics database structures.

Actions - Pass the complete selection into each sub action

A workflow action can now pass the complete selection into each sub action, so that the sub actions can process the whole selection rather than each item individually. For example, a workflow action that calls a template action passes the whole selection into the template action. This results in a file that contains all items rather than a single file for each item, which is the current behavior.

If the process fails validation, the whole selection is not processed, i.e. all already-processed sub actions are rolled back. The failure is logged.

Expression Builder - new functions

There are new functions, one to convert an input to a number ("Number"), and one to convert and input to an amount ("Amount").

For information about functions, see Expression builder - New and changed functions.

New validation types in workflow actions - "Block", "Terminate"

We have added the following validation types to the workflow actions list:

  • Block - Block causes the system to skip that particular record and process the other remaining rows.

  • Terminate - Terminate causes the system to terminate all actions and none of the rows from the workflow report are processed.

Collateral tags to support EMIR REGIS-TR and EMIR DTCC

We now provide support for collateral tags in the workflow templates "EMIR REGIS-TR" and "EMIR DTCC". This allows you to report the compulsory Collateral tags with a new trade submission, for example, when using the regulatory reporting functionality in Quantum.

To use this on deals, in Configuration > Analysis Codes create an analysis code named "Collateral Type".

Add Predefined Values for "Fully" (FC),"OneWay" (OC), "Partially" (PC), or "Uncollateralized" (UC). "U" means Not Set.

Using Set Context action triggers fields

If you use the Set Context action to fill in default values, (for example, after a New action occurs), the system triggers the field dependencies in the same way as if you had filled out those fields manually.
