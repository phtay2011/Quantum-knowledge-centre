Workflow Template enhancements

Template to File Download

Each workflow template automatically gets a workflow action (in the Download folder) that downloads a file into the browser containing the output of the workflow template . These downloads are available on the workflow reports and apply to the selected rows from the report results.

Use these file downloads to drill-down into selected rows and give related or aggregated information.

The file is created in the template output format and can be automatically converted to PDF (using the Convert to PDF setting on the action).

Validations grid

We've added a Validations grid to the Workflow Template page, so that you can now prevent invalid data being processed or warn about exceptions in the data when the system applies a template.

Define Validations grid entries, each with a Condition expression, Message, and Type. The Type can be:

  • Terminate - when any item in the data set matches the terminate condition, none of the items in the data set will be processed by the template and a single error will be returned for the data set;
  • Block - any item matching the block condition will not be processed and an error is logged for it;
  • Warn - any item matching the warn condition, will be processed but a message is logged for it.

Preview tab - Validation Messages grid

If you defined conditions in the Validations grid that the sample data meets, the read-only Validation Messages grid appears, displaying the applicable message Type and Text in each case. This enables you to check your validations before the system executes the template.

  • if the Entity Code is OL3 - the first warning message is added to the Message Log and execution proceeds,
  • if the Entity Name is Praetor - the second error message is added to the Message Log and the item is not processed by the template, and
  • if the Entity Code is more than five characters - the third error message is added to the Message Log and execution is blocked.

CSS File for HTML Template

When a template generates HTML output, it includes the template.css file from the QuantumWeb/Resources folder. This enables you to apply your organization's style to the HTML output. In combination with the Convert to PDF setting on the template action, this allows you to generate styled PDF documents.
