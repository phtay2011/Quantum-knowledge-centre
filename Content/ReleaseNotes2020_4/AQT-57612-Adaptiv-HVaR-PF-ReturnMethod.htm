Adaptiv CVA and HVaR queries - AAI.AAIS.exe.config file changed

NOTE: From Version 2020.4, use the new Calculation Properties page to define multiple sets of any Calculation Property values, which you can then specify at the Assumptions Definition and query level. Those settings then override any settings (like those below) in the AAI.AAIS.exe.config file, which apply system-wide.

ReturnMethods

For Adaptiv Historical VaR queries, you can now add the ReturnMethods configuration option to change the default settings of the AA Studio Calculate dialog value for Price Factor ReturnMethods.

The settings available for each Price Factor are:

  • None
  • Historical
  • Absolute
  • Absolute Cumulative
  • Proportional
  • Proportional Cumulative
  • Rebootstrap
  • Pass Though Absolute (Adaptiv Studio has a spelling mistake so this option also does)
  • Pass Through Proportional
  • Pull-to-One
  • Vol Weighted Fractional Return
  • Vol Weighted Absolute Return
  • Vol Weighted Log Return

Example

To configure InflationRate and InterestRate Price Factor Return Methods to Absolute and Historical respectively, and leave the remaining Return Methods at the default of Proportional, include the following property setting in the AAI.AAIS.exe.config file:

  • <add key="ReturnMethods" value="InflationRate=Absolute;InterestRate=Historical" />

TIP: See Calculation Properties page for an equivalent example setting using that new page.

For more details, refer to the AA Studio documentation or help.

History

Previously, all Price Factors defaulted to Proportional and none could be altered via the query.

ScenariosPath

For Adaptiv CVA queries, you can now add the ScenariosPath configuration option to define your Deterministic Scenarios from file, by specifying the file pathname. For example:

<add key="ScenariosPath" value="C:\DetScens\DSFactorArchive.xml"/>

The above line is included,the system passes this setting to Adaptiv through the integration layer and sets the calculation properties on the CVA query as follows:

  • Deterministic Scenario is set to FromFile, and
  • Scenario Path is set to C:\DetScens\DSFactorArchive.xml.

If the scenario path line is commented out or absent, or the path setting is invalid, the system sets Deterministic Scenarios to Off (the pre-existing functionality).

Exclusions

The following Deterministic Scenarios are NOT supported:

  • Generate
  • From Database

The Database_Name and Archive_ID settings are NOT supported.

The setting is not applicable to PFE queries.

History

Previously, Deterministic Scenarios was always set to Off.
