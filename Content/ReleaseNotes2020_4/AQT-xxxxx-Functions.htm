Expression builder - New and changed functions

The following are new functions used in Expression Builder.

Amount

Description

Returns the amount representation of the value

Format

Amount(value)

Arguments

value - the input is the value that you want to convert to an amount

Example

Amount(“12345”)

Where 12345 is an input in type "Text" and the function converts the string into an amount value.

Example

Amount([Deal Number])

The number value of the Deal Number field is converted to an amount value.

BankToBankInstruction

Use in Configuration > Template pages as part of the Enterprise Workflow (EW).

Description

Returns a bank to bank instruction field value from a counterparty account based on the provided cashflow number and bank to bank instruction line name
Format BankToBankInstruction(CashflowNumber, name)

Arguments

name - specify the bank to bank instruction line name. Bank to bank instruction line names are the part of the bank to bank instruction details entered in the Bank to Bank Instructions free-format text box on a counterparty account. The line name is the name inside the two "/" characters.

Leading and/or ending spaces will be trimmed from the returned value.

CashflowNumber - specify the cashflow number or select an input / column from the available sources
Example

BankToBankInstruction([CashflowNumber], “B2B”)

If the Bank to Bank Instructions text box has lines:

  • /B2B/ Some Bank to Bank Instructions /

  • /B2B LINE2/ B2B extra line info /

then this returns a value of "Some Bank to Bank Instructions" (with leading / ending spaces trimmed).

ClearingField

Use in Configuration > Template pages as part of the Enterprise Workflow (EW).

Description

Returns a clearing field value based on the provided account number and the clearing field name
Format ClearingFields(accountNumber, clearingFieldName, optional isIntermediary)

Arguments

accountNumber – Specify the account number or select an input / column from the available sources

clearingFieldName – Specify the clearing field caption. Clearing field captions are the system’s field captions for clearing fields, as dictated by the currency.

isIntermediary (optional) – can be set to TRUE or FALSE:

  • if TRUE, the clearing field value from the intermediary account is returned

  • if FALSE, the clearing field value from the account is returned
Example

ClearingFields([Account], “FedWire”, TRUE)

Because clearing fields are defined at a currency level, we recommend that you use expressions to specify which clearing field to use based on the currency.

Example

If([Currency] = "USD", ClearingFields([Account] , "FedWire"), "")

ClearingFieldName

Description

Returns a clearing field name based on the provided currency and list of excluded clearing field names
Format ClearingFieldName(currency, {clearingFieldName} )

Arguments

currency - specify the currency as clearing fields are defined at currency level

clearingFieldName - specify the clearing field captions that are required to be excluded. Clearing field captions are the system’s field captions for clearing fields, as dictated by the currency.
Example

ClearingFieldName([Currency], {"CLS", "CHIPS Universal", "CHIPS Participant","RU CBIC","Swiss BC"} )

CounterpartyClearingField

Use in Configuration > Template pages as part of the Enterprise Workflow (EW).

Description

Returns a clearing field value based on the provided cashflow number, the clearing field name, and the isIntermediary argument
Format CounterpartyClearingFields(cashflowNumber, clearingFieldName, isIntermediary)

Arguments

cashflowNumber – specify the cashflow number or select an Input / column from the available sources

clearingFieldName – specify the clearing field caption. Clearing field captions are the system’s field captions for clearing fields, as dictated by the currency.

isIntermediary – TRUE or FALSE:

  • if TRUE, the clearing field value from the counterparty intermediary account is returned

  • if FALSE, the clearing field value from the counterparty account is returned
Example

CounterpartyClearingFields([CashflowNumber], “FedWire”, TRUE)

Because clearing fields are defined at a currency level we recommend that you use expressions to specify which clearing field to use based on the currency.

Example

If([Currency] = "USD", CounterpartyClearingFields([CashflowNumber] , "FedWire", TRUE), "")

CounterpartyNarrative

TIP: Including the delivery narrative in the data available for an EFT template created in the Enterprise Workflow (EW) allows you to include the delivery narrative values in the EFT ISO message, given certain conditions.

Description

Returns a delivery narrative field value from a counterparty account based on the provided cashflow number and narrative line name
Format CounterpartyNarrative(cashflowNumber, name)

Arguments

cashflowNumber - specify the cashflow number or select an input / column from the available sources

name - specify the narrative line name. Narrative line names are the part of the special delivery details entered in the Delivery Narrative free-format text box on a counterparty account. The line name is the name inside the two "/" characters typically at the beginning of a line.

Leading and/or ending spaces are trimmed from the returned value.
Example

CounterpartyNarrative([cashflowNumber], “DELNAR”)

If the Counterparty Bank Account page's Delivery Narrative text box has lines:

  • /DELNAR/ Some Delivery Narrative /

  • /DELNARLINE2/ extra lines of narrative /

then this returns a value of "Some Delivery Narrative" (with spaces trimmed).

CurrencyPair

Description

Returns the base and quote currency pair for currency1 and currency2
Format CurrencyPair(currency,currency)

Arguments

currency – the applicable currencies
Example

CurrencyPair("USD","EUR")

Returns the correct order of the quote basis pair as it appears on the Static > Currency Pair page - USD / EUR or EUR / USD, as the case may be.

If the inputs do not match with the currency pair defined in the system, the function returns a blank value and logs an error in the Audit log.

Identifier

Use in Configuration > Template pages as part of the Enterprise Workflow (EW).

Description

Returns an identifier field value for a given scope based on the provided name and identifier name
Format Identifier(scope, name, identifierName)

Arguments

scope - the scope defined for the identifier: entity, counterparty, or location

name- specify the entity / counterparty / location name or select an input / column from the available sources

identifierName - specify the name of the identifier
Example

Identifier(“Location”, [Datasource Entity.Location], “SWIFT Code”)

On the Location page, if SWIFT Code = OTHR, then this returns "OTHR".

Example

Identifier("Entity", [Entity] , "SWIFT")

On the Entity page, if SWIFT =ENT01BIC, then this returns "ENT01BIC".
Example

Identifier("Counterparty", [Counterparty] , "SWIFT Confirmation")

On the Counterparty page, if SWIFT Confirmation = MYCPYSWF, then this returns "MYCPYSWF".

Identifiers are External Method records that have a Method Type of Confirmation, Payment, or Mapping and have Enable Identifiers selected.

Identifiers are returned:

  • for the Bank (counterparty record) of the Bank Account,

  • for the Entity for multiple entities,

  • if the system finds a matching party name, for example, counterparty intermediary bank is matching that of an existing counterparty record.

IsFXSpot

Description

Determines whether the given deal is an FX spot
Format IsFXSpot(DealNo)

Arguments

Deal No - the deal number of the FX Deal
Example

IsFXSpot(60)

Returns "Yes" if the FX deal is a spot deal. If there is no deal or an invalid deal number, returns "No".

LineNumber

Description

Returns the line number of text in a given input, or returns zero if the text is not found.

The function counts the line number for empty lines as well.
Format LineNumber(within_text, find_text)

Arguments

within_text - the text to search in

find_text - the text whose line number you want to find

Example

LineNumber([Data],"RUNDATE")

The following lines represent [Data]:

START-OF-FILE

RUNDATE=20141028

FIRMNAME=dl586041

SN=851169

WS=1

The result is 2.

Lower

Description

Returns a string in which all characters in the given argument are converted to lower case
Format Lower("text") or Lower([element])

Arguments

text - a string

element - an item in the database, for example, [name]
Example

Lower("a2B3c") = "a2b3c"

Converts characters in "a2B3c" to lower case.

Narrative

TIP:Including the delivery narrative in the data available for an EFT template created in the Enterprise Workflow (EW) allows you to include the delivery narrative values in the EFT ISO message, given certain conditions.

Description

Returns a narrative field value from an account, based on the provided cashflow number and narrative line name
Format

Narrative(cashflowNumber, name)

The outcome value will be trimmed.

Arguments

cashflowNumber - specify the cashflow number or select an input / column from the available sources

name - specify the narrative line name. Narrative line names are the part of the special delivery details entered in the Narrative free-format text box on an account. The line name is the name inside the two "/" characters.

Leading and/or ending spaces are trimmed from the returned value.
Example

Narrative([cashflowNumber], “ACC”)

If the Bank Account page's Narrative text box has lines:

  • /CABLE/1234567890abcdefghij

  • /PHONEBEN/abcdefghij12345678901

  • /ACC/ 1234567890abcdefghij1234567890

  • /BENONLY/123

then this returns a value of "1234567890abcdefghij1234567890" (with space trimmed).

Number

Description

Returns the number representation of the value.

Format

Number(value)

Arguments

value - the text value you want to convert to a number

Example

Number(“12345”)

where 12345 is an input of type "Text" and the function converts the string into a number.

Example

Number([Amount])

where Amount is an input of type "Amount" and the function converts the string into a number.

Text After

Description

Returns the text between the find_text up to the end of line or end_text.

Format

Text After(within_text, find_text, end_text(optional))

Arguments

within_text - the text to search in

find_text - the result starts after text specified here

end_text (optional) - the result ends before the text specified here.

If not specified, the result ends before the end of line.

Example

TextAfter(":A:Lorem:B:ipsum",

":A:", ":B:")

The result is 'Lorem'.
