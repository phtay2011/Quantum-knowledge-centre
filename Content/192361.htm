Confirmation Processing Functions

After you run a query, deal action confirmation messages matching your specified filters and options are listed.

Two function menus are available for result list items.

  • Grid menu functions apply when one or more items are selected (click the selection box at the left of the row/s and click the menu button above the grid), and
  • Quick menu functions apply to only one item (click a value in the item row to show this menu).

The table below summarizes the functions available, which menu/s, and the types of message they apply to.

NOTE:

  • Buttons also apply to the first five functions below (in bold).
  • Buttons and menu, items are disabled when the function is invalid for your currently selected item/s.
  • You must have the required security rights to perform most functions. These rights are determined by settings on the relevant User or User Group record's Module Security tab for the Confirmations module.

About FX Settlement Switch functions

The Switch to quick menu functions apply only to FX deal actions and enable you to switch between CLS and Standard Settlement from that point on.

The function is available only on a single FX deal action* when:

  • no actions other than the initial DEAL action and CHANGE actions have been performed;
  • the confirmation External Method is Electronic, Misys (for Finastra), or SWIFT Confirmation; and
  • the current User has the Switch CLS Module Security permission for the FX Deal module and Inquire permission for both the FX Deal and Confirmations modules (or is exempt).

NOTE: If the selected FX deal action is for any leg of an FX Swap or FX Back to Back deal, the change of settlement option will apply only to that leg's linked FX deal. A message will be displayed informing you of this and prompting you to confirm (or to cancel).

After a successful Switch action:

  • when the deal is switched to CLS, the Is CLS value is set to Yes and the CLS badge will appear on the FX deal page;
  • when the deal is switched to Standard Settlement, the Is CLS value is set to No and the CLS badge will not appear;
  • Bank Account and Counterparty Bank Account Defaulting Rules are reapplied based on the new value of Is CLS;
  • the Cut-Off Time will not impact the Is CLS value for subsequent actions, but every action does record an Is Past CLS CutOff value.
  • the action is treated as a financial change and will trigger a CHANGE action being created;
  • the CHANGE action will follow exception processing for the associated cash flows;
  • the Deal Map is not affected;
  • a ROLLBACK is not allowed on any deal actions prior to the Switch;
  • the Flow lifecycle status is changed to Amended;
  • the Cash Flow Status becomes Unactioned if it was Actioned earlier;
  • the Pay/Rec status will change to Exception if EFT files were created earlier;
  • if netted earlier, cash flows are removed from the netted group;
  • if no Confirmation file had been created, a new MT300 or MT304 file is created based on the Is CLS value; and
  • if a Confirmation file was already created, a Cancel file and new MT300 or MT304 file are created.
