Applying Resource Security to Crystal Mapping

Crystal Mapping has functionality that is specific to Crystal Reports attached to non-inquiry and non-dealing programs. This functionality applies to both Quantum seed reports and any user-defined reports, and involves global Resource Security settings for each Crystal Report.

When Resource Security is enabled, you can select Security in the page menu on the Crystal Mapping page to display the Security dialog box and configure additional Resource Security setting options, as shown below.

Check boxes in the first column can be used to apply Resource Security to each Table Alias and Table Name combination where Resource Security is applicable.

This feature is provided because the Resource Security permissions you define for a User may not be appropriate when the User generates a Crystal Report. If every Resource-related data field had Resource Security filtering applied in the report, this filtering could inadvertently exclude data records from the report that the User would normally be able to view in an Quantum program.

For example, in the example report above, Resource Security is applied to the Table Alias bankacc (which corresponds to the Bank Account number), but is not applied to any of the other Resource Table Alias. The report that is created based on these settings would only display Bank Account data for those Bank Account numbers the User running the report has at least Viewpermissions for.

However, the Bank Account data displayed in the report will include other Resource data that the User may not have at least View permission as defined in the dialog box. For example, Table Alias bank refers to the Counterparty field on the Entities tab within the Bank Account program. For a given Bank Account that the User has at least View permission to, they may not have View permission for this Counterparty Resource as specified by the tables alias bank. Therefore they would be able to view the Counterparty defined in this field in both the Bank Account program and the static report but not in the Counterparty program.

So in this example, the setting could be valid for this report, but this is not necessarily true for all custom Crystal Reports a User has access to. In addition, the same Crystal Report can be used in more than one program and have different report formats.

This does not apply to Inquiry Crystal Reports that report only off the Inquiry's result set. These are filtered based on the User's permissions.

See Crystal Seed Reports and Resource Security for specific information on the seed reports that have additional Resource Security.
