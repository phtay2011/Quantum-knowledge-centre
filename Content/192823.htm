Command Line support for all defined Query filter parameters

Queries with additional filters defined can now have the value of those filters overwritten on the command line (in addition to the default parameters).

AGCommandLineSupport executes the query with the filter property values specified.

Implementation

The AGCommandLineSupport program has been modified as follows.

  • The value of all filters specified for a query can now be overwritten using the command line.
  • Filters that have not been added to the query or are marked as invisible cannot be overwritten on the command line.
  • Setting the value on the command line will be in the format of name=value where name is the descriptive name (with spaces removed) of the filter as specified in the Quantum API for the required module. Refer to the API documentation for more information.
  • When a filter value is overwritten, the same operator as defined in the query will be used (equal to, less than, and so on).
  • When a list operator is used (one of), multiple values can be specified using a comma separator.
  • When the between operator is used, the two values can be specified using a comma separator.
  • If the query's filter parameters are not specified or their names do not match any in the command line, then the default values of the filter parameters will apply.

In addition, the export of report results (for example to PDF) now shows the actual filter values for the report run (previously only the query's saved filter values were shown).

Examples

(Filter parameters highlighted in bold)

Message Log

Query CommandLineTest is created and saved with the following filters and default values:

Type one of = "FUNCTIONAL"
Date Message Logged equal to = "Today"
Long User Name equal to = "Supervisor"                          [ This is an invisible parameter. ]
Module one of = "MSGENQ"
Time between = "", ""
Message Severity greater than = "1"

The following command line overrides the values in the saved filters when the query is run (except forLONGUSERNAME, which is an invisible parameter that cannot be overwritten - that is, Supervisor always applies):

AGCommandLineSupport MSGENQ SITE=RPACK USERNAME=baseline PASSWORD=baseline QUERY="CommandLineTest" VIEWNAME="Default" FILETYPE="PDF" DIRECTORY="C:\Projects\AQT\trunk\Site\RPACK\CommandLine" FILENAME="MYREPORT" STARTDATE="2017-11-10" TYPE="FUNCTIONAL,SYSTEM" DATEMESSAGELOGGED="Today" LONGUSERNAME= "Baseline Tests Logon ID" MODULE="MSGENQ,ACRLENQ" TIME="10:00,11:00"

Audit

Query CommandLineTest is created and saved with the following filters and default values:

Action Date between = "Yesterday, Today"

The following command lines override the values in the saved filters when the query is run:

AGCommandLineSupport AUDENQ SITE=RPACK USERNAME=baseline PASSWORD=baseline QUERY="CommandLineTest" VIEWNAME="Default" FILETYPE="PDF" DIRECTORY="C:\Projects\AQT\trunk\Site\RPACK\CommandLine" FILENAME="MYREPORT" STARTDATE="10-Nov-2017" ENDDATE="Today" TABLENAME="ALL" ACTIONDATE="28-FEB-2017,28-FEB-2018"

Transactions By Action

Query CommandLineTest is created and saved with the following filters and default values:

Customized? equal to = "NO"

The following command lines override the values in the saved filters when the query is run:

AGCommandLineSupport TRNSACTN SITE=RPACK USERNAME=baseline PASSWORD=baseline QUERY="CommandLineTest" VIEWNAME="Default" FILETYPE="PDF" DIRECTORY="C:\Projects\AQT\trunk\Site\RPACK\CommandLine" FILENAME="MYREPORT" STARTDATE="10-Nov-2017" ENDDATE="Today" CUSTOMIZED?="YES"

History

Previously:

  • from Release 6.6.0 - only the query's default filter parameters were supported by the AGCommandLineSupport program,
  • prior to Release 6.6.0 - no filter parameters were supported, and
  • the export of report results showed only the query's saved filter values.
