How to implement Confirmation XML Data records

Quantum Six currently provides seed XML Data records for all supported SWIFT and Finastra (Misys) Confirmation message types.

Change Message Format / Content

To change output message format and content

  1. Ensure that you:
    • contact your Quantum representative if you need assistance,
    • keep copies of the original seed XML Data records supplied with Quantum,
    • have the required expertise in maintaining XML code, and
    • co-ordinate your "live" testing plans and cutover times with the external message service.
  2. Take copies of all the seed XML Data records for the message type you want to change, and rename them.
  3. Edit your copies of the XML Data records as required.
  4. (Optional) Contact your Quantum representative for assistance in pre-testing your changes.
  5. (Recommended) To test message output without affecting production output, set up special Electronic Confirmation Format records to assign your changed XML Data records to a "Test" combination of selection criteria and Deal actions.
  6. Test your new XML Data records with low numbers of "dummy" deals and deal actions before fully implementing them.
  7. When your testing has been successfully completed, change over to your new XML Data records and Electronic Confirmation Formats.

Implement New Message Types

To implement new message types

  1. First contact your Quantum representative to check if the message type you want is available in a later release than yours, or planned for an upcoming release.
  2. Base the XML Data records for your new message type on copies of one of the supported seed XML Data records.
  3. Change these by following the general procedure to Change Message Format / Content.

Review Confirmation Message Output

To review output message content

  1. On the Confirmation Messages page, enter a Message Date Range and optional filter values (External Reference, Deal Number/s, and/or Action Number/s).
  2. Click Run Query.

    Output messages meeting your query parameters are listed in the results grid, using the default View.

    TIP: You can change the result list view by selecting a different View or configuring a new one, if required. To change the full result list itself, rerun the query with different parameters.

  3. Check that expected output messages are present in the results grid.
  4. To view the content of a message, select the message and in the grid menu, select View (or click a link in the message row and select View in the popup menu).

    The Confirmation dialog box displays a summary of the message, followed by its full text content, line by line.

    TIP: While you can select and copy text from this dialog box, you cannot edit it.

  5. When you have finished viewing that message, click Close.
  6. If the message content is as expected, repeat the procedure for different deals, deal actions, and/or message types until you are satisfied that all message output is correct.

If no message output is created

  1. Check that the Confirmations and Matching services are running.
  2. If message data or formatting is incorrect, check the applicableXML Data records, starting with the applicable Data Template record content first.
  3. If expected messages are missing, check the applicable Electronic Confirmation Format and Confirmation Rules record settings.
  4. Check if an error message has been posted in the Message Log giving details of the problem.

If message content is not as expected

  1. Check the applicable XML Data records, starting with the applicable Data Template record content first.
  2. Check if an error message has been posted in the Message Log giving details of the problem.
