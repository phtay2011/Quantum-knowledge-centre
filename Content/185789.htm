Quantum Request Pages

The new Money Market and FX Request pages contain a subset of the items required for the corresponding live deals.

TIP: The pre-existing FX Cover Requests page is still also supported. Pre-existing users can continue using that method, but if it is not required, consider suppressing it using Module Security.

The number of fields required to create a Request is minimal.

For a Money Market Request:

  • Mandatory fields are Instrument, Borrow/Invest, Currency, Face Value, Deal Date, Settlement Date, and Maturity Date.
  • Entity, Counterparty, Dealer, and all other fields are optional.
  • Analysis Codes are as for an actual Money Market Deal with the same settings.

For an FX Request:

  • Mandatory fields are Instrument, Buy Ccy, Sell Ccy, Buy amount or Sell amount, Deal Date, Value Date, and (for NDFs) Settlement Ccy and Fix Date.
  • Entity, Counterparty, Dealer, Spot Rate, Contract Rate, Forward points, and all other fields are optional.
  • Analysis Codes are as for an actual FX Deal with the same settings.

Page Menu Functions

CATEGORY/
Item

Available when...

REQUEST

 

New
  • User has Add rights for the Request module.

Reset
  • User has Change rights for the Request module,
  • Request has been fully approved, AND
  • Request is not Deleted.

Save
  • User has Add rights for the Request module AND
  • Request is not Deleted.

Copy
  • User has Add rights for the Request module.

Delete
  • User has Delete rights for the Request module.

Approve
  • A matching Deal Approval Rule exists,
  • User has Change rights for the Request module,
  • User is a qualifying Approver, AND
  • Request is not already approved.

Reject
  • As for Approve, AND
  • Request is not already rejected.

Audit History
  • User has Inquire rights for the Request module AND
  • Request has been saved.

Fill
  • User has Fill rights for the Request module,
  • User has Add rights for the Dealing module, AND
  • Request is fully approved OR Allow Processing option is selected in the matching Deal Approval Rule.

Mark as Filled
  • User has Fill rights for the Request module,
  • User has Change rights for the Dealing module, AND
  • Request is fully approved OR Allow Processing option is selected in the matching Deal Approval Rule.

Attach File
