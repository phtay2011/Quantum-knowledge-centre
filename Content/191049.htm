Confirmations - Structured Fields for NDFs

(First introduced in Release 6.5.1 SP, September 2017.)

Specific structured and validated fields have been added for support of Non-Deliverable Forward (NDF) opening and valuation confirmation messages, as summarized in the table below and detailed in the following subtopics.

All Seed Layout files for MT300, MT304, and MT305 have been updated accordingly, for both SWIFT and Finastra (formerly Misys) confirmations.

TIP: All fields listed below are in mandatory Sequence A in the respective messages.

MT300 Foreign Exchange Confirmation

77D Terms and Conditions

In this optional existing field:

  • the use of NDF codes in this field is no longer supported, and
  • the codes FIX, SETC, SRCE, and VALD must not be present at the start of any line.

17F Non-Deliverable Indicator

This new field:

  • is mandatory and contains Y for an NDF (that is, the deal will be cash-settled),
  • is optional and contains N for a deliverable trade, and
  • when present, follows 14C Year of Definitions.

17O NDF Open Indicator

This new field is mandatory for an NDF, else it is not allowed.

When present, it follows 17F and contains:

  • Y for the NDF opening (DEAL, CHANGE, EXERCISED), or
  • N for other NDF deal actions (RATE FIX, DELETE, and so on).

32E Settlement Currency

This new field is mandatory for an NDF, else it is not allowed.

When present, it follows 17O and contains the three-character Settlement Currency of the NDF (which must not be of type Commodity),

30U Valuation Date

This new field is mandatory for an NDF, else it is not allowed.

When present, it follows 32E and contains the Rate Fix Date of the NDF, in the format YYYYMMDD.

14S Settlement Rate Source

This new field is mandatory for an NDF, else it is not allowed.

When present, it follows 30U and contains the value of a Deal Analysis Code or FX Deal Analysis Code named Settlement Rate Source, in the format XXXnn[HHMMCCLL] where:

  • XXXnn is the mandatory rate source code* (EMT00 for confirmations where the terms are governed by an EMTA template),
  • HHMMCCLL represents an optional Time and Location for the settlement,
  • HHMM is the time in hours and minutes,
  • CC is a two-character ISO country code, and
  • LL is a two-character location code - either:
    • the first two characters of the location name if it is one word, or
    • the first letters of the first two words of the location name.

For some cross-currency NDFs, two separate settlement rate sources may be quoted, in which case the fields must appear in alphabetical order of Rate Source.

*REFERENCE:Annex A of the 1998 FX and Currency Option Definitions (published by the International Swap and Derivatives Association Inc, the Emerging Markets Traders Association and the Foreign Exchange Committee) as amended and supplemented from time to time.

21A Reference to Opening Confirmation

This new field is mandatory for an NDF when 17O contains N (RATE FIX), else it is not allowed.

When present, it follows 14S and contains the 16-character identification of the NDF Opening Confirmation message to which the current Valuation message is related - that is, the Sender's Reference in field 20 of the opening confirmation (if more than one previous opening message, the external reference from the most recent is used).

The field must not start or end with a slash /, and must not contain two consecutive slashes //.

MT304 Advice/Instruction of a Third Party Deal

94A Scope of Operation

This mandatory existing field identifies the type of foreign exchange contract and contains one of the following four-character codes.

  • AFWD - Forward Currency Contract (FFC) or Hedge executed for a future date at a set price. The contract is opened and closed for the same value date resulting in a gain or loss. The result can be settled by netting the local base or both currencies or by settling the gross/principal amounts.
  • ANDF - Non-Deliverable Forward (NDF). The trade will be cash-settled in the agreed settlement currency.
  • ASET - Settlement Contract; a physical settlement in which both the Buy and Sell currencies will move on the settlement (value) day.

17O NDF Open Indicator

This new field is mandatory for an NDF or FCC (94A = ANDF or AFWD), else it is not allowed.

When present, it follows 94A and contains:

  • Y for the NDF or FCC opening, or
  • N for the valuation of an NDF or the partial or final close of an FCC.

17F NDF Final Close Indicator

This new field is mandatory for an NDF or FCC close (17O = N), else it is not allowed.

When present, it follows 17O and contains:

  • Y for a final close, or
  • N for a partial close.

32E Settlement Currency

This new field is optional for an NDF (94A = ANDF), , else it is not allowed.

When present, it follows 14C and contains the three-character Settlement Currency of the NDF (which must not be of type Commodity),

30U Valuation Date

This new field is mandatory for an NDF (94A = ANDF), else it is not allowed.

When present, it follows 32E (or 17F if 32E not present) and contains the Valuation date of the NDF, in the format YYYYMMDD.

14S Settlement Rate Source

This new field is mandatory for an NDF (94A = ANDF), else it is not allowed.

When present, it follows 30U and contains the value of a Deal Analysis Code or FX Deal Analysis Code named Settlement Rate Source, in the format XXXnn[HHMMCCLL].

(For more details, see under MT300 above.)

21A Reference to Opening Confirmation

This new field is mandatory for an NDF when 17O contains N (RATE FIX), else it is not allowed.

When present, it follows 14S and contains the 16-character identification of the related NDF Opening Confirmation message.

(For more details, see under MT300 above.)

MT305 Foreign Currency Option Confirmation (Vanilla)

17F Non-Deliverable Indicator

This new field:

  • is mandatory and contains Y for an NDF (that is, the deal will be cash-settled),
  • is optional and contains N for a deliverable trade, and
  • when present, follows 26F Settlement Type.

14S Settlement Rate Source

This new field is mandatory for an NDF (17F = Y), else it is optional.

When present, it follows 17F and contains the value of a Deal Analysis Code or FX Deal Analysis Code named Settlement Rate Source, in the format XXXnn[HHMMCCLL].

(For more details, see under MT300 above.)

32E Settlement Currency

This new field is mandatory for an NDF (17F = Y), else it is not allowed.

When present, it follows 14S and contains the three-character Settlement Currency of the NDF (which must not be of type Commodity).

History

Previously, codes in a free-format field were used for confirmation of NDF trades, which led to manual processing and errors.
