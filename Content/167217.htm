Deal Set Update Restrictions

Use Deal Set Rules to define your deal set update restrictions, which control whether the following changes are allowed using a Maintenance Mode other than the one specified for the Deal Set:

  • changing the Deal Set a deal links to,
  • saving a deal in the Deal Set, or
  • a deal processing action.

Not all Maintenance Modes support all actions.

Deal Save actions

The way the system treats a deal save when restrictions exist is common across most deal actions - see How restrictions affect deal actions, below. In some instances however, further system checks may also be made - see Additional system checks, below.

Deal set update restrictions affect the following deal save actions.

  • new deal
  • edit deal (includes changing the Deal Set)
  • change the Value Date and Amount in Settlement processing
  • convert a Projected Cash Flow to an Actual Cash Flow*
  • convert Live to Underlying
  • customize deal
  • Rollback
  • delete deal

*TIP: If a Projected Cash Flow in a Deal Set is converted to an Actual Cash Flow, the Actual Cash Flow is automatically added to that set.

Deal Process actions

Deal Set Update Restrictions affect the following Process actions:

  • Approval
  • Fix Average Rate
  • Rate Fix
  • Close Out
  • Indexation
  • Rate Set
  • Corporate Actions†
  • Knock In
  • Roll Renegotiation
  • Draw Down
  • Knock Out
  • Settle
  • Exercise
  • Lapse
  • Settle Approval
  • Extend / Pre-deliver
  • Profile
  • Unwind

† Corporate Actions cover Dividend, Split, and Reverse Split actions on Equity Transaction Types (Unit Deals and Unit Holdings).

How restrictions affect deal actions

The way the system applies restrictions is common across all deal actions. This section describes the common treatment. In some instances however, additional system checks may also be made (see below).

If you are licensed for Deal Sets, the system determines whether a deal set exists when a deal save or process action occurs, and:

  • if a deal set does not exist, the deal saves as per normal, but
  • if a deal set does exist, the system checks the specified Deal Set Rule to see whether the deal or process action is restricted. The following table describes how the action is treated, depending on the Deal Actions or Process Actions setting specified in the Deal Set Rule, as applicable.

    Rule setting

    Treatment

    Are Allowed

    The deal update continues as usual.

    Need Confirmation

    Treatment varies depending on the source of the action (the Deal Set's Maintenance Mode):

    • Interactive (Quantum)
      A message is displayed and the user can decide whether or not to continue with the action.
    • Import
      A message is sent to the message log and the action continues.
    • API
      A message is displayed and the action continues.

    Are Blocked

    Treatment varies depending on the source of the action:

    • Interactive (Quantum)
      A message is displayed and the action is canceled.
    • Import
      A message is sent to the message log and the action is skipped.
    • API
      A message is displayed and the action is canceled.

Additional system checks

For some actions, additional system checks may be made, as follows.

  • Bulk FX Deal Extend and Bulk Renegotiation (Mature Deals, Rate Set Deals, and Roll Deals)

    Deals are excluded if Process Actions is set to Are Blocked in the applicable Deal Set Rule.

  • Closeouts

    Deals cannot be closed out if Process Actions is set to Are Blocked in the applicable Deal Set Rule.

  • Exercising a deal

    If an exercised deal is in a Deal Set, any deals created as a result of the exercise are not automatically added to that set.

    When a physical exercise occurs, the exercised deal and the physically created deal may potentially belong to different deal sets, with different restrictions. This can have implications when rolling back an exercised deal. See Rolling back a deal below.

  • Index Update

    Deals are excluded if Process Actions is set to Are Blocked in the applicable Deal Set Rule.

  • Rolling back a deal

    A rollback will occur provided there is no Deal Set Rule Process Actions Block restriction on:

    • the deal you are rolling back, or
    • any associated deal - for example the deal physically created when you exercise a deal.
