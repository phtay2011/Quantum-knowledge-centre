Tiered Approvals Based on Amounts

Using the Approval and Authorization Rules page, you can set up multiple approval entries that have identical selection criteria except for different Limit Amounts. This enables you to set up approval scenarios like the following (for Deal and/or Settlement approvals).

Amounts of...

Require...

USD1,000,000.00 and over

Two Approvers

From USD10,000.00 to 999,999.99

One Approver

Less than USD10,000.00

No approvals

NOTE: You must specify the same Limit Currency in such cases.

During the approvals process, when such criteria are being applied for matching:

  • The relevant deal and cash flow amounts will be converted to the Limit Currency if necessary.
  • Deal Approvals generally uses the Face Value amount from the deal.
  • The Rate used to convert differing currency amounts is sourced from Management Reporting Control, using the applicable Default record and CCY Translation Rate Type.
    • Settlement Approvals uses the cash flow date as the Rate Date.
    • Deal Approvals use the deal date as the Rate Date.
  • With multi-legged deals such as Swaps, the first leg amount and currency are used.
  • For Bank Account Adjustments with multiple amounts, the sum of the amounts is used.
  • For General Ledger Adjustments:
    • If only one Entity/currency, the sum of the Debit-side amounts is used.
    • If multiple Entities/currencies, an amount of zero is used.
  • The system first tests if the amount equals or exceeds the largest Limit Amount, and if so, that Approval Control record applies. If not, it tests if the amount equals or exceeds the next-largest Limit Amount and if so, applies that record.

Exclusions:

  • No allowance is made for Deal Actions that reduce or increase the Face Value of the deal.
  • Hedge Relationships will continue to be automatically approved.
  • Tiered approvals are not currently supported for Commodity Swap deals.
