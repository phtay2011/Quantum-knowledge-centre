Netting Cash Flows in Settlements Processing

When you run a Settlements Processing query, stand-alone cash flows meeting the Basic Criteria for Netting may be EITHER proposed for netting OR automatically netted, by using the following run options:

  • select Use counterparty defaults - to apply applicable Counterparty Netting Rules,
  • select Use facility defaults - to apply Facility Netting where specified,
  • set Net by Same to Deal Number - to add the same Deal number to the netting criteria, or
  • set Net by Same to Security ID - to add, for security cash flows such as coupons, the same Primary Identifier value to the netting criteria, then
  • select Net automatically to apply any applicable netting immediately, OR
  • select Preview Net proposal (the default) - to show Net Proposals* that you can review before actually creating the nets (by selecting Net Proposal items in the results grid and clicking Net as Proposed).

    *NOTE: The only available actions on flows currently Proposed for Netting are either Net as Proposed or Clear Net Proposal.

Where two or more types of netting apply, Counterparty Netting Rules are applied first, followed by any netting by Facility, then netting by Deal Number or Security Identifier.

TIPS:

  • You can optionally net by Deal Set by selecting the Is part of a deal set option in the applicable Counterparty Netting Rule records. See Netting by Deal Set.
  • A limit on the maximum number of cash flows that may be netted could apply, depending on the NetItemsMax setting in the Web.AppSettings.config file.

Manually Netting

After you run a Settlements query, you can select flows in the results grid that meet the Basic Criteria for Netting and manually Net them (or append flows to an existing netted group), using the netting functions available.

  • If your selected flows all have the same Payment Method, Bank Account, and Counterparty Account, they are immediately netted.
  • If these differ for any flows, a dialog box will be displayed indicating the differences, and giving you the option to select which single Payment Method, Bank Account, and/or Counterparty Account is to apply to the netted group as a whole, or to cancel.

For more details, see Netting Functions in Settlement Processing.

Changing Netted Cash Flows

When EFT Not Yet Created

If the EFT has not yet been created for a netted group (Pay/Rec is Not Created) and the net's Flow Status is UNACTIONED:

  • you can manually add or remove cash flow items (in the Settlements Processing results grid),
  • if a financial change is made to a cash flow item in the group, that cash flow is automatically removed from the netted group.

Once a netted group has entered the EFT Lifecycle stage (Flow Status is ACTIONED), the Flow Lifecycle can be either EFT Ready for Creation or Awaiting Settlement Approval(s), and any change to a cash flow member OR manually removing items from the group is treated as an amendment, and:

  • the changed cash flow is removed from the netted group, and
  • the Flow Status is reset to UNACTIONED and the Flow Lifecycle to Awaiting Settlement Action for the changed cash flow, the net itself, and all its remaining members.

When EFT Created

Once a netted group has completed the EFT Lifecycle stage (Flow Status is ACTIONED and Flow Lifecycle is EFT Created):

  • you may not manually remove items from the netted group (in the Settlements Processing results grid),
  • you may not manually change any* item's Flow Status to UNACTIONED,
  • you may manually apply the Mark as Exception and then make changes to the net.

    Mark as Exception results in Pay/Rec set to Exception, Flow Status to UNACTIONED, and the Flow Lifecycle to Awaiting Settlement Action for the net and all the cash flow items in the net.

*NOTE: Once an EFT has been created for an item, you cannot manually change its Flow Status to UNACTIONED. This applies to netted or un-netted cash flows, netted groups, stock flows, and cash flows linked to stock flows.

When EFT Marked as Created

You cannot change Netted flows after using the Mark as Created function UNLESS there is a subsequent financial; change that changes the EFT Lifecycle Status to Amended.
