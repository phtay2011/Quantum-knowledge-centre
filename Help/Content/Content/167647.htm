Limits Definition

Use the Limits definition page to view, create and delete Limits Definition records, which enable you to satisfy specialized Limit reporting requirements.

A Limits Definition contains one or more entries, each with a unique set of Matching Criteria and associated Limit Details (Currency, Amount, applicable Dates, and so on). This enables you to assign multiple limits to the same deal when a query runs . For example, you could assign both Counterparty and Counterparty Group limits.

TIP: Include an entry with All Matching Criteria settings to catch deals that do not fall under any other Limit.

Limits Definitions are optional for all types of query. A Limits Definition is included when a query configuration specifies that record's name in the Limits field on the Properties tab. A Limits tab then appears in the query's Configure area, initially showing that Definition record's settings, and any changes made to the settings on that tab will be saved in that Definition record when the query configuration is saved.

NOTE: If the Use with Limit Check option is selected for the query (on the Properties tab), then (provided the Limit Checking server is running) deals will automatically be checked against the specified Limit entries when they are saved or changed.

When a Limits Definition is used with a query, each deal in the result set is checked against each set of Matching Criteria. When matches are found:

  • the specified Limit Detail settings are applied according to the specified Utilization Method - EITHER to single deals (Deal), or over all matching deals (Sum, Percent, or Weighted);
  • if a deal matches more than one Limit entry, it will show that many times as a row in the result dataset
  • one Group By and three Point in Time (PIT) columns are populated and when included in the Columns Definition for the query (Columns tab), the relevant values from the matching Limit entry will be included in each row of the raw results dataset:
    • Limit Details (Group By) - a pipe-delimited string of the Matching Criteria values,
    • Limit Amount (PIT),
    • Limit Currency (PIT), and
    • Limit Description (PIT) - a list of the Matching Criteria items being specifically applied; those not set to All. For example, if only Counterparty is not All, this field will return Counterparty. If multiple criteria, a list.

      TIP:Limit Description gives the criteria labels (for example Counterparty, Dealer), while Limit Details gives the actual values (for example NE Bank, Jane Doe).

As shown in the example below, you can use Calculated Columns to:

  • define your Utilization calculation (fully customizable),
  • strip out individual Matching Criteria values from the Limit Details string (by using the GetString function),
  • create a column indicating a Limit breach, the remaining amount available, and so on.
