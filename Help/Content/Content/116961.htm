About XTC Configuration XML Data records (EFTs)

Overview

An XTC Configuration XML Data record*:

  • is accessible only via the XML Data page*,
  • has Type set to XTC Configuration (corresponding to the earlier filename suffix*),
  • contains XML code;
  • applies to a specific External Method when its name is specified in the Transformation Configuration File field of the associated External Method Source record (for SWIFT, the default is the seed XTC Configuration record provided for each External Method - that is, SeedXMLConfig_EFT);
  • is optional and if NOT specified (in the Transformation Configuration File field as above), no XML formatting is applied - however, an ATS transformation may still be applied IF output is directed to a File Destination matching a Data Transfer Location name that specifies one;
  • allows optional specification of an alternative separator between payment messages - see the delimiter attribute under the Messages element below; and
  • gives the name of the Data Template, Layout, and Stylesheet XML Data records.

    NOTE: An ATS Transformation XML Data record is optional, and may be included EITHER in place of the Layout and Stylesheet records (in which case the ATS transformation is applied directly to the raw output in the Data Template record format) OR in addition to them (in which case the ATS transformation is applied after full XML transformation).

See also: Supplied EFT XML Template Files and XML Template Files for Other External Methods.

Configuration Element

Only one Configuration element is allowed.

No attributes are supported.

Messages element and delimiter attribute

Only one Messages element is allowed per Configuration element.

The optional delimiter attribute is supported. Include this attribute to specify a separator between payment messages.

If the attribute is not present - the default dollar sign delimiter $ applies.

If the attribute is present, the separator value may:

  • be null (delimiter="") - no separator is output,
  • include one or more alphabetic characters, numbers, spaces, and/or symbols (or none),
  • include one or more of the following special ASCII control characters within brackets (or none):
    • (CR)
    • (LF)
    • (CRLF)
    • (TAB)

For example, to output, between payment messages:

  • no separator:

<Messages delimiter ="">

  • a new line:

<Messages delimiter ="(CRLF)">

  • a dollar sign followed by a new line:

<Messages delimiter ="$(CRLF)">

Message element

Only one Message element is allowed per Messages element.

No attributes are supported.

DataTemplate, Layout, AtsTransformation, and Stylesheet elements

Each Message element must have a DataTemplate element, and can include the optional AtsTransformation, Layout, and Stylesheet elements.

Each of these elements are XML-based, and can be either a linkage to a Data Template type of XML Data record, or contain the required XML content.

Where it is a linkage to an XML Data record, each element must contain the Name attribute giving the name of the relevant Data Template record.

For the AtsTransformation element, name="ATS".

Examples (SWIFT)

1. Full XML transformation without ATS

<?xml version="1.0" encoding="UTF-8"?>
<!--EFT Configuration File-->
<Configurations>
<Configuration>
<Messages delimiter="(CRLF)">
<Message>
<Input>
<DataTemplate name="SeedXMLDataTemp_EFTandFlows.xml"/>
<Layout name="SeedXMLLayout_EFT.xml"/>
<Stylesheet name="SeedXMLStylesheet_SWIFTFIN.xsl"/>
</Input>
</Message>
</Messages>
</Configuration>
</Configurations>

Note the inclusion of the delimiter attribute in the Messages element, which will output a new line between payment messages. If this attribute was not present, the default dollar sign delimiter ($) would be output.

2. Full XML transformation plus ATS

<?xml version="1.0" encoding="UTF-8"?>
<!--EFT Configuration File-->
<Configurations>
<Configuration>
<Messages>
<Message>
<Input>
<DataTemplate name="SeedXMLDataTemp_EFTandFlows.xml"/>
<Layout name="SeedXMLLayout_EFT.xml"/>
<Stylesheet name="SeedXMLStylesheet_SWIFTFIN.xsl"/>
<AtsTransformation name="ATStransform.ats"/>
</Input>
</Message>
</Messages>
</Configuration>
</Configurations>

In this example, the ATS transformation will be applied after the full XML transformation.

3. DataTemplate and ATS transformations only

<?xml version="1.0" encoding="UTF-8"?>
<!--EFT Configuration File-->
<Configurations>
<Configuration>
<Messages>
<Message>
<Input>
<DataTemplate name="SeedXMLDataTemp_EFTandFlows.xml"/>
<AtsTransformation name="ATS"/>
</Input>
</Message>
</Messages>
</Configuration>
</Configurations>

In this example, the ATS transformation replaces the Layout and Stylesheet transformations, so will be applied directly after the Data Template transformation.

4. No XML transformation (no XTC Configuration record)

If the Transformation Configuration File field of the associated External Method Source record is blank, no XML formatting is applied.

However, an ATS transformation may still be applied if output is directed to a File Destination matching a Data Transfer Location name that specifies one.
