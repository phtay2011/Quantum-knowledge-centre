How Static Data Approval Works

Entity and Instrument approvals are implemented via the Static Data Approval page, while all others are currently implemented using an Unapproved page in each case, to list items of the relevant type that are pending approval.

Static Data Approval does not apply when the relevant Approval Level is set to None (on the Data Security page), for:

  • Entity, Instrument, and Strategy - the Static Data Approval page will not list any changes, while for...
  • all other types - the Unapproved page/s for the relevant static data types are redundant and no changes need approval. For example, if Bank Account Approval Level is set to None, the Unapproved Bank, Broker, Stock, and Money Market Fund Account pages are all redundant.

Static Data Approval does apply when the relevant Approval Level is set to Single or Dual. Either the Static Data Approval page or the Unapproved page/s for the relevant Static Data must be used to approve (or reject) any updates* and deletes for that type of data.

*NOTE: Updates here cover add, edit, and deactivate actions.

When Static Data Approval applies and a record update or delete remains unapproved:

  • on the "main" static data page:
    • the record is badged as Pending Approval with a link to the Unapproved page for that record,
    • all items are read-only, and
    • the record status may not be changed (for example, to Deactivated);
  • on either the Static Data Approval page or the Unapproved static data page, as applicable:
    • the update or delete awh6>s an Approve or Reject action (by a qualified Approver who is not the initiator of the change),
    • if an update - further changes may be made, provided no approve action has yet been taken (users making such changes may not then approve or reject the update or delete),
    • if a delete - no further changes may be made,
    • after the first Approve action is performed (by an allowed Approver):
      • if Approval Level is Single - the update or delete is processed (see below) and the record is removed from the Static Data Approval or Unapproved page (as applicable), or
      • if Approval Level is Dual - the update or delete is part-approved, awaiting the second Approve action (or a Reject),
    • if a Reject action is performed (by an allowed Approver) - the update or delete is removed from the Unapproved page, and the situation reverts to that before the update/delete (this applies at any stage - a Reject is immediate, even when the Approval Level is Dual),
    • if part-approved:
      • no further changes may be made,
      • after the second Approve action is performed (by an allowed Approver who cannot also be the first) - the update or delete is processed (see below) and the record is removed from the Static Data Approval or Unapproved page (as applicable), or
      • if a Reject action is performed (by any allowed Approver) - the update or delete is processed (see below) and the record is removed from the Static Data Approval or Unapproved page.

When a record update has been fully approved:

  • on the main page - the updated record is no longer flagged and is available for update or delete, and
  • on the Static Data Approval or Unapproved page (as applicable) - the record is removed.

When a record delete has been fully approved:

  • on the main page - the record is removed, and
  • on the Unapproved page, the record is removed.

When a record update or delete has been rejected:

  • on the main page - the situation reverts to that before the update/delete, and
  • on the Static Data Approval or Unapproved page - the record is removed.

TIP: Static data changes are recorded in the audit log. Use Audit Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.
