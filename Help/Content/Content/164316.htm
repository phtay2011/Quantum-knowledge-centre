Validating a deal when saving in hedge relationships

Quantum validates the following information whenever you save a hedge relationship (HR):

  • Hedge Ratio
  • Hedge Type
  • Nature of Risk

Other validation and warnings on Deal Save is summarized below.

Testing and posting ccy

This setting must be either:

  • the base currency of the Entity value selected for the hedge relationship, or
  • a currency present in one of the linked deals in the hedge relationship.

NOTE: If Amortization is already set at Accounting Control level, the consequence is a double impact Amortization amount on the Profit and Loss.

Impact on revaluation entries

Some changes to a hedge relationship can result in the reversal of existing revaluation entries. If you make a financial change to the hedge relationship, we will reverse the revaluation entries. We'll display warning messages to notify you.

Example 1

Suppose you add a deal where the deal's Start Date, for example, 1 August 2019, is prior to the latest revaluation date for the hedge relationship, for example, 10 August 2019.

When you save the hedge relationship, Quantum automatically unwinds all revaluation calculations that occurred after the new deal's Start Date and reverses any G/L postings. This means that your revaluation entries applicable for 10 August no longer exist. The next time you run position processing, the system creates new entries for the reversed period.

Example 2

If you add a new Linked Deal to a previously saved hedge relationship and the hedge relationship has existing G/L posting entries generated by a Position Processing run, the Save Deal confirmation window prompts you to:

  • Click Yes to reverse the existing G/L postings. This is the normal scenario.
  • Click No to keep the existing G/L postings. Enter a new Maturity Date balance to ensure that the G/L postings will be correct.
  • Click Cancel to cancel the save.

Forward point amortization

If selected, this option is automatically reset to NO AMORTIZATION if:

  • no Foreign Exchange deals form part of the hedge relationship, and/or
  • no Freeze Date exists.

An alert provides details of why this change was made.
