About Bank Account Rules

NOTE: Bank Account Defaults are mandatory. If no defaulting rule is found for a deal, it cannot be saved.

Use the Bank Account Rule page to define rules that apply particular bank account instructions to deal cash flows, depending on deal values such as Entity, Transaction Type, Instrument, Counterparty, and so on.

You can define multiple instructions for a single rule selection, each with a different Effective Date. These can reflect changes in your banking situation. For example, if an entity changes banks, you can save existing deals using the previous bank account and new deals using the new bank account, depending on whether the cash flow date is before the Effective Date for the new account or not.

When you save a deal, Quantum searches the rule records by priority to find the most complete match and relevant Effective Date record, and then applies those account instructions to the deal cash flows. These are shown on the deal page's Cash Flows tab (if a deal has associated stock flows, the stock account instructions are shown on the deal page's Other Details tab).

NOTE:

  • You cannot save a transaction unless the appropriate accounts and account rules are set up. Bank Account rules are required even if the currency is a commodity.
  • When you change Bank Account Rules, existing cash flows are not changed. To update them, run Bank/Counterparty Bank Account Conversion.
  • Changes made to Rule records are recorded in the audit log. Use Audit Query to list details of records added, changed, and deleted, including the user who made the change and the date and time.
  • Specific conditions apply with Actual Cash Flow deals - see Bank account defaulting for Actual Cash Flow deals

CLS (Continuous Linked Settlement)

If you are using CLS (Continuous Linked Settlement) with FX Deals and Currency Options, you can define different Bank Account Rules for CLS and non-CLS (Standard Settlement) deals. The methods differ between the two transaction types, as follows

For FX deals

NOTE: To use CLS, you must have a matching CLS Cut-Off Time Rule for the FX deal and for each currency (or ALL).

For each applicable Counterparty/Instrument/Currency combination:

  • Create one Bank Account Defaulting rule for CLS, setting the Transaction Type to FX Deals only and selecting the CLS option. Add an Instruction entry with the required Effective Date, Bank, CLS-eligible Account Number, and so on.
  • Either select an existing rule that covers the applicable combination for Standard Settlement, or create a new rule with the Standard Settlement option either selected or not shown. The rule must have an Instruction entry with the required Effective Date, Bank, Account Number, and so on.

TIP: The CLS and Standard Settlement options are used in combination with CLS Cut-Off Time Rules.

See also: CLS workflow enhancements

For Currency Options

Depending on whether or not a CLS Bank Account is used only for CLS deals, the desired Bank Account defaulting rule can be selected at an instrument level.

If you are using different Bank Accounts, then provided you are using separate Instruments solely for CLS deals:

  • set up a Bank Account Defaulting Rule specifying the CLS Instrument and the CLS Bank Account, and
  • set up another Bank Account Defaulting Rule specifying the non-CLS Instrument and the non-CLS Bank Account.

If you are not using different Bank Accounts, you should still set up two Bank Account Defaulting Rules: one specifying the CLS Instrument and the other the non-CLS Instrument.

NOTE: A Physical Exercise action on a Currency Option creates a new FX deal, when the CLS rules for that FX deal then apply as above.
