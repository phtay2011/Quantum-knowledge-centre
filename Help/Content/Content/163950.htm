Counterparty Categories

Select the appropriate options on the relevant Counterparty page's Categories tab to indicate the categories that apply to that counterparty. These categories are used for various purposes throughout the system.

TIPS:

  • Categories do not apply to Counterparty groups.
  • Several categories apply only in the Default perspective. See Counterparties - Quantum v Analytics.
  • For Counterparties with more than one Category, only the Deal external or Deal internal options apply when matching Category selections of External or Internal (respectively) in Confirmation Control. Other Counterparty categories such as Issuer/Acceptor, Tax Authority, and so on, do not affect this.

Bank internal (Quantum only)

Select this category if the Counterparty provides internal bank accounts.

These would generally be internal counterparties. Bank accounts may be created and operated for this type of Counterparty.

Selecting this field displays the BIC Code field on the Identifiers tab.

Bank external (Quantum only)

Select this category if the Counterparty provides external bank accounts.

Bank accounts may be created and operated for this type of Counterparty.

NOTE: Selecting this field displays the BIC Code field on the Identifiers tab.

Issuing and Paying Agent

Select this category to enable this Counterparty to appear in the Issuing and Paying Agent lookup for Securities instruments and deals.

Related to financial securities, an Issuing and Paying Agent (IPA) performs the following functions:

  • issuing securities upon Issuer orders,
  • collecting funds from the Issuer to make interest and principal payments to bondholders,
  • arranging payment of coupons and matured bonds, and for cancelation and destruction, and
  • processing transfers of ownership of securities.

Deal internal

Select this category if you intend to use the Counterparty to record internal deals against.

Counterparties must be categorized as Deal internal (or Deal external) to allow deals to be recorded against them.

TIP: This category determines that a deal is internal to (within) your organization (there are no other real differences between internal and external deals in the system).

Deal external

Select this category if you intend to use the Counterparty to record external deals against.

Counterparties must be categorized as Deal external (or Deal internal) to allow deals to be recorded against them.

TIP: This category determines that a deal is external to (outside of) your organization (there are no other real differences between internal and external deals in the system).

Exchange

Select this category if the Counterparty is an exchange that trades securities or units (Equities and/or Mutual Funds).

Counterparties of this category will populate Exchange lookups for Unit Holding instruments and deals, and for exchange-traded Security Identifiers.

TIPS:

  • Only an Exchange-category Counterparty can be used for Unit deals.
  • For futures exchanges (in the Default perspective), use the dedicated Futures Exchange page (not this Counterparty category).

Debtor / Creditor

Select this category to enable the Counterparty to be used for Actual or Projected Cash Flow transactions.

Issuer / Acceptor

Select this category to enable the Counterparty to appear in the Issuer lookup for Security instruments, Security deals, and Security Identifiers.

Custodian

Select this category if the Counterparty is to be specified as the custodian of any Stock accounts.

A Stock account is used to handle Stock Flows (the receiving and delivery of Securities stock against payment).

Only a Custodian Counterparty can be specified when you set up a Stock account. You cannot clear this option if any Stock accounts have already been set up for the Counterparty (you must delete those Stock accounts first).

Broker

Select this category if the Counterparty is a Broker (or, in Futures, an Executing Broker).

Broker-category Counterparties are included in the following deal lookups:

  • Executing Broker in Futures - to indicate the broker who makes the futures deal on the exchange floor. (Clearing Brokers are indicated by the Clearing Broker category.)
  • Broker in non-Futures deals - to enable users to record a Broker for the deal, if one was used.

Clearing Broker

Indicates that this Counterparty is a valid broker to clear Futures deals with.

The Clearing Broker clears the futures deal through the exchange. Conversely, an Executing Broker (as controlled for Futures deals by the Broker category) is a party with whom future contracts are dealt.

Tax Authority

Select this category to identify the Counterparty as a Tax Authority, for withholding tax purposes.

SWIFT Financial Institution

This option, together with the corresponding one for the transaction Entity, controls the format of SWIFT cash flow message output for the selected Counterparty and Entity in a transaction.

  • When this option is selected for both the Counterparty and the Entity, an MT202 (or MT203 when batched) message will be output.
  • If only one, or neither, of these is selected, the standard format applies (MT101, MT103, or MT103+).

Settlement Bank

Select this category if the Counterparty is only to be used for settlement purposes.

This removes them from Counterparty lookups for deal entry.

TIP: If the Bank internal and Bank external categories are not set, then this category also controls whether or not the BIC Code field is displayed on the Identifiers tab.

Fund Manager

Select this category if the Counterparty is a Money Market Fund Manager.

You can set up Fund accounts for this category of Counterparty in Bank Accounts (when Account Type is set to Fund). Those accounts may then be selected when:

  • defining fund details (in AG-Money Market Funds)
  • defining an Interest Rate record for the fund (in Bank Interest Rates)
  • establishing a fund's balance (in AG-Money Market Funds)
  • setting a fund's daily Dividend Yield (in Bank Account reporting)

Clearing House

Indicates that this Counterparty is designated a clearing house for deal reporting purposes.

TIP: When the same Counterparty is also specified as the Clearing Broker on a deal, all flows will be cleared through that Counterparty.
