Add a Query Filter

TIP: Rather than change an existing query you may want to re-use, Copy it (or create a New one), configure that query, and Save it under a new name.

The Quantum and Analytics procedures differ. When installed, Adaptiv filtering procedures are as for Analytics and the Filters Definitions are shared.

To add a Query filter (Quantum):

  1. Click the Query Configure link.

    The Configure Query area opens.

  2. On the Defaults tab, click the Add Filter button.
  3. In the Add Filter browse window, select the field you want. You can, where applicable:
    • start typing in the Search box to find a field quickly,
    • click a field name to select it,
    • click a data table icon to show the fields in that data table, and/or
    • click a data table name to select that table's Reference field (the record key), where applicable.

    That new field is added to the query filters.

  4. If required, click the default operator above the new field and select the operator you want (one of, equal to, less than, between, and so on) and enter one or more values as required.
  5. (Optional) To make the new filter field invisible when the report is run, click the field name and in the drop-down menu, select Make Filter Invisible.

    Your changes will be saved as part of the query configuration when (and if) you click Save.

To add a Query filter (Analytics or Adaptiv)

  1. Click the Query Configure link.

    The Configure Query area opens.

  2. (Optional) On the Properties tab, ensure that the Filters field is set to the Filters Definition record you want to apply to the query run. The default is the Query Name, but you may select another Definition record.

    NOTE:

    • Any changes you make to the Filters will be saved to the selected Definition record when you save the query.
    • Filters Definitions are shared between Analytics and Adaptiv.
  3. On the Filters tab, select the required Match All or Match Any branch* in the filter logic structure and click the Add Filter button.

    *TIP: You can add, change the branch type, and remove branches. For more details, see Filters Definition Procedures.

  4. In the Add Filter browse window, select the filter: a data field, data table, or (Analytics only) an applicable Mapping Profile).

    The browse window closes and your new Filter field appears, labeled with the selected item's name and the default operator.

    TIP:

    • Start typing in the Search box to list matching fields.
    • Click a field name or )Analytics only) an applicable Mapping Profile) to select it.
    • Click a data table icon to show the fields within that data table.
    • Click a data table name to select the Reference field (the record key) within that table (where applicable).
  5. Depending on your new Filter field type, specify its filter criteria. First select the required logical operator, then enter or select one or more values to apply that operator to.

NOTE: When (and if) you save the query, your changes will be saved to the selected Definition record (Filters field, Properties tab).
